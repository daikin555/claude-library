## 原文（日本語に翻訳）

フルスクリーンモードでリストがモーダルをオーバーフローする場合に、`/skills`メニューがスクロールしない問題を修正しました。

## 原文（英語）

Fixed /skills menu not scrolling when the list overflows the modal in fullscreen mode.

## 概要

Claude Code v2.1.110では、フルスクリーンモードで`/skills`コマンドを実行した際に表示されるスキル一覧モーダルが、スキルの数が多くてモーダルの高さを超えた場合にスクロールできなかった問題が修正されました。この問題により、フルスクリーンモードで多くのスキルがインストールされている場合、一部のスキルが表示されず選択できない状態になっていました。今回の修正でスキル一覧を上下にスクロールして全スキルにアクセスできるようになります。

## 基本的な使い方

```bash
# /skills メニューを開く（フルスクリーンモード）
/skills

# v2.1.110以降: スキルが多くてもスクロールして全スキルを確認・選択可能
# 矢印キーまたはvim風キー（j/k）でスクロール
```

## 実践例

### ユースケース1: 多数のスキルがある環境での操作

```bash
# フルスクリーンモードでスキル一覧を表示
/skills

// 修正前の表示（フルスクリーンモード、スキルが多い場合）:
// ┌─────────────────────────────────────┐
// │ スキル一覧                          │
// ├─────────────────────────────────────┤
// │ 1. code-review                      │
// │ 2. git-helper                       │
// │ 3. debug-assistant                  │
// │ 4. doc-generator                    │
// │ 5. test-writer                      │
// │ ...（以降スクロールできず不可視）   │
// └─────────────────────────────────────┘
// ※ 6番目以降のスキルが表示・選択できなかった

// 修正後の表示:
// ┌─────────────────────────────────────┐
// │ スキル一覧 (スクロール可能)         │
// ├─────────────────────────────────────┤
// │ 1. code-review                      │
// │ 2. git-helper                       │
// │ 3. debug-assistant                  │
// │ 4. doc-generator                    │
// │ 5. test-writer                      │
// │ ↓ スクロールで続きを表示            │
// └─────────────────────────────────────┘
```

### ユースケース2: キーボード操作でのスクロール

```bash
# /skills メニューでのキーボード操作
/skills

# スクロール操作:
# ↓ (下矢印) または j  → 次のスキルへ
# ↑ (上矢印) または k  → 前のスキルへ
# Page Down または Ctrl+f → ページ下へ
# Page Up または Ctrl+b  → ページ上へ
# Home または gg        → 先頭へ
# End または G          → 末尾へ
# Enter                 → 選択されたスキルを実行
# Esc または q          → メニューを閉じる
```

### ユースケース3: スキル管理の効率化

```bash
# スキルを多数インストールしている場合の管理
/plugin install additional-skills-pack

# インストール後にスキル一覧を確認（フルスクリーンでもスクロール可能）
/skills

# 特定のスキルを検索（フィルタリング）
/skills code  # "code" を含むスキルをフィルタリング
```

### ユースケース4: スキルの整理と管理

```bash
# インストール済みスキルの一覧（コマンドライン）
claude skills list

# スキルの詳細確認
claude skills info code-review

# 不要なスキルの削除（メニューを管理しやすくする）
/plugin remove unused-skill

# 削除後にスキル一覧を確認
/skills
```

### ユースケース5: フルスクリーンモードの設定

```json
// .claude/settings.json でフルスクリーンモードを設定
{
  "ui": {
    "defaultMode": "fullscreen"
  }
}
```

```bash
# フルスクリーンモードで起動
claude --fullscreen

# または実行中に切り替え
/tui
```

## 注意点

- **影響を受けていた条件** — フルスクリーンモード（`/tui`または`--fullscreen`で有効化）で、かつスキルの数がモーダルの高さを超えている場合に発生していました。
- **通常ウィンドウモードでは正常** — 通常のウィンドウモード（非フルスクリーン）では正常にスクロールできていました。
- **スキル数の目安** — ターミナルの高さにもよりますが、概ね7〜10個以上のスキルがある場合にスクロールが必要になります。
- **キーバインド** — スクロールのキーバインドはカスタマイズ可能です（`/keybindings`で確認）。
