## 原文（日本語に翻訳）

- 長時間セッションでのCPU使用効率を改善

## 原文（英語）

- Improved CPU utilization in long sessions

## 概要

長時間にわたるClaude Codeセッションにおいて、CPU使用率が最適化されました。以前は長時間のセッションでCPU使用率が徐々に上昇する問題がありましたが、この改善により安定したパフォーマンスが維持されるようになります。バックグラウンドタスクやエージェントを使った長時間の作業セッションで特に効果を実感できます。

## 基本的な使い方

この改善は自動的に適用されるため、特別な設定は不要です。Claude Codeを通常通り使用するだけで恩恵を受けられます。

```bash
# 通常通りClaude Codeを起動
claude

# 長時間セッションでもCPU使用率が安定
```

## 実践例

### 長時間の開発セッション

大規模なリファクタリングや複数ファイルにまたがる変更作業で、長時間セッションを維持しても以前のようなCPU使用率の上昇が発生しにくくなりました:

```bash
# 長時間のセッションでも安定
claude
> このプロジェクト全体のTypeScript移行を手伝って
# 数時間にわたる作業でもCPU使用率が安定
```

### バックグラウンドタスクとの併用

バックグラウンドエージェントやタスクを並行して実行する場合にも、CPU使用率が適切に管理されます:

```bash
# 複数のバックグラウンドタスクを実行しても安定
claude
> Ctrl+B でバックグラウンドタスクを開始しつつ、メインの作業を継続
```

## 注意点

- この改善は内部的な最適化であり、ユーザー側での設定変更は不要です
- 特に長時間セッション（数時間以上）で効果が顕著です
- バックグラウンドタスクを多数実行している場合にも改善効果があります
- システムリソースが限られた環境（CI/CDやリモートサーバー）でも恩恵があります
