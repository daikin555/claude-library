## 原文（日本語に翻訳）

他のサブエージェントがまだ作業中である間に、アイドル状態のサブエージェントがagentパネルから消えてしまう問題を修正。余剰のアイドルエージェントは、展開可能なサマリー行にまとめられるようになった

## 原文（英語）

Fixed idle subagents vanishing from the agent panel while other subagents were still working; surplus idle agents now collapse into an expandable summary row

## 概要

複数のサブエージェント（agent teamsのteammateを含む）を並行して動かしていると、プロンプト入力欄の下にある「agentパネル」に各エージェントの行が一覧表示されます。これまでは、あるエージェントがアイドル（待機中）になると、他のエージェントがまだ作業中であっても、そのアイドル行がパネルから消えてしまうことがありました。これでは、待機中のエージェントを選んでトランスクリプトを確認したり追加の指示を送ったりできなくなってしまいます。修正後は、パネル内に作業中のエージェントが1つでもいる間はアイドル行が表示され続けるようになりました。さらに、アイドルなエージェントが4つ以上になった場合は、先頭の3行を除いた余剰分が「N idle agents」のような1つの折りたたみ行にまとめられ、Enterキーで展開・再度折りたたみができるようになりました。

## 基本的な使い方

```
修正前:
1. 複数のサブエージェント/teammateを並行して動かす
2. 一部がアイドル状態になると、他がまだ作業中でも
   ❌ そのアイドル行がagentパネルから消えてしまう
3. 待機中のエージェントを選んで確認する手段がなくなる

修正後:
1. 同じように複数のエージェントを並行して動かす
2. ✅ 誰か1つでも作業中であれば、アイドル行はパネルに残り続ける
3. 全員がアイドルになった場合のみ、パネル全体が30秒後に非表示になり
   次のターンで再表示される
4. アイドルなエージェントが4つ以上になった場合は、先頭3行以外が
   「N idle agents」の折りたたみ行にまとまる
```

## 実践例

### 複数teammateのうち一部が先に完了するケース

```
1. `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` を有効化し、
   5人のteammateにコードレビューを分担させる
2. 3人が先にレビューを終えてアイドルになる
3. ✅ 残り2人がまだ作業中なので、先に終わった3人の行も
   パネルに残り、いつでも選んで結果を確認できる
```

### 大人数のteammateでアイドル行が折りたたまれるケース

```
1. 6人のteammateを起動し、そのうち5人が先にアイドルになる
2. ✅ 先頭3行はそのまま表示され、残り2人分は
   「2 idle agents」という1行にまとめられる
3. その行を選んでEnterを押すと展開され、個別に選択できる
4. 作業中のteammate、失敗したteammate、現在表示中のteammateは
   常に個別の行として表示され続ける
```

### アイドル行が隠れた後に再度メッセージを送るケース

```
1. 全teammateがアイドルになり、30秒後にパネルの行が非表示になる
2. 名前を指定してteammateにメッセージを送る
3. ✅ 該当のteammateは非表示中も動作・応答可能な状態を保っており、
   次のターンで行が再表示される
```

## 注意点

- v2.1.181〜v2.1.198では「自分のターンが終わってから30秒」で個別に行が隠れる仕様でしたが、v2.1.199ではパネル全体がアイドルになってから30秒後にまとめて隠れる挙動に変わりました
- v2.1.181より前のバージョンではアイドル行は隠れません
- 折りたたみの対象になるのは「アイドル状態」のエージェントのみで、作業中・失敗・表示中のエージェントは常に個別行として表示されます
- 本機能はagent teams機能（実験的機能、`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` で有効化）およびサブエージェントのパネル表示に関するものです
