## 原文（日本語に翻訳）

- プランモードの権限プロンプトのテキスト入力で、数字キーが入力ではなくメニューオプションの選択になってしまう問題を修正

## 原文（英語）

- Fixed digit keys selecting menu options instead of typing in plan mode permission prompt's text input

## 概要

プランモード（`/plan`）で権限プロンプトが表示された際、テキスト入力フィールドに数字を入力しようとすると、その数字がメニューオプションの選択として解釈されてしまうバグが修正されました。この修正により、テキスト入力フィールドにフォーカスがある場合は数字キーが正しくテキストとして入力されるようになります。

## 基本的な使い方

プランモードで権限プロンプトが表示された際に、テキスト入力フィールドに数字を含むパスやコマンドを入力できます。

```
/plan ポート8080で起動しているサービスを調査
```

権限プロンプトが表示された場合、テキスト入力欄に数字（例: ポート番号やファイルパス）を問題なく入力できるようになりました。

## 実践例

### テキスト入力に数字を含むケース

権限プロンプトのテキスト入力で、以下のような数字を含む入力が正しく処理されます:

```
# パスに数字を含む場合
/tmp/test123/output.txt

# ポート番号の入力
3000

# バージョン番号の指定
v2.1.72
```

### プランモードでの操作フロー

1. `/plan` でプランモードに入る
2. 権限が必要な操作でプロンプトが表示される
3. テキスト入力フィールドに自由に数字を含む文字列を入力
4. 数字がメニュー選択にハイジャックされることなく入力される

## 注意点

- この修正はプランモードの権限プロンプト内のテキスト入力フィールドに限定されます
- メニュー選択画面では引き続き数字キーでオプションを選択できます
- テキスト入力フィールドにフォーカスがあるかどうかで挙動が切り替わります
