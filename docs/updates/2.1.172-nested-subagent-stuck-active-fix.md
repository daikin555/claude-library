## 原文（日本語に翻訳）

バックグラウンドサブエージェントが起動したネストされたエージェントが停止した後も、エージェントパネルで「アクティブ」のままスタックする問題を修正しました

## 原文（英語）

Fixed a background sub-agent staying stuck as "active" in the agent panel after a nested agent it spawned was stopped

## 概要

バックグラウンドサブエージェントが入れ子（ネスト）でさらにエージェントを起動した場合、そのネストされたエージェントを停止してもバックグラウンドサブエージェントのパネル表示が「アクティブ」のまま変化しない問題が修正されました。今後はネストエージェントの停止が正しくステータスに反映されます。

## 基本的な使い方

この修正は自動的に適用されます。入れ子のエージェントを停止した際に、エージェントパネルのステータスが正しく更新されます。

```
# エージェントパネルの表示
claude agents

修正前:
[アクティブ] バックグラウンドサブエージェントA  ← ネストされたエージェントを停止後も残る
  └── [停止済] ネストエージェントB

修正後:
[完了] バックグラウンドサブエージェントA  ← 正しくステータスが更新される
```

## 実践例

### マルチエージェントワークフローでのステータス管理

```bash
# ワークフローを起動
claude agents

# ネストされたエージェント構造:
# 親エージェント
#   ├── サブエージェント1（完了）
#   └── サブエージェント2（停止）← ここを停止
# 修正後: 親エージェントのステータスも正しく更新される
```

## 注意点

- この問題は2.1.172で追加された「サブエージェントが独自のサブエージェントを起動できる」機能と関連しています
- エージェントパネルのステータスが正確に表示されることで、リソース管理が適切に行えます
