## 原文（日本語に翻訳）

ロゴ下の「セットアップ問題」起動行を削除しました。設定の問題を確認するには `/doctor` を実行するか、`--debug` を使用してください。

## 原文（英語）

Removed the startup "setup issues" line under the logo — run `/doctor` to see configuration issues or use `--debug`.

## 概要

これまで Claude Code の起動時にロゴの下に表示されていた「セットアップ問題」の通知行が削除されました。起動画面がよりすっきりしました。設定の問題を確認したい場合は、明示的に `/doctor` コマンドを実行するか、`--debug` フラグを使用してください。

## 基本的な使い方

### 設定問題の確認

```bash
# /doctor で設定の状態を確認（以前は起動時に自動表示）
/doctor

# --debug フラグで詳細な診断情報を表示
claude --debug
```

## 実践例

### セットアップ問題の診断

```bash
# Claude Code を起動
claude
# ロゴが表示されるがセットアップ問題の行は非表示になった

# 問題があると思われる場合は /doctor を実行
/doctor
# 出力例：
# ✓ API key: 設定済み
# ✓ Node.js: v20.x.x
# ⚠ MCP サーバー 'my-server': 接続できません
# ...
```

### デバッグ情報の確認

```bash
# 詳細なデバッグ情報を表示して起動
claude --debug
# 設定問題を含む詳細ログが表示される
```

### CI/CD 環境での活用

```bash
# ヘルスチェックスクリプト例
claude /doctor 2>&1 | grep -E "✓|✗|⚠"
if [ $? -eq 0 ]; then
  echo "Claude Code の設定は正常です"
else
  echo "設定に問題があります"
  exit 1
fi
```

## 注意点

- 起動時の自動表示がなくなったため、設定の問題に気づきにくくなる可能性があります。定期的に `/doctor` を実行することを推奨します
- `/doctor` は設定の問題だけでなく、MCP サーバーの接続状態や権限の設定なども確認できます
- `--debug` フラグはより詳細な情報を提供しますが、多量の出力が生成されます
- この変更は起動時の視覚的なノイズを減らし、よりクリーンな体験を提供します
