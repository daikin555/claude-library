## 原文（日本語に翻訳）

plan modeが状態を変更するブラウザツール呼び出しに対して確認を求めていなかった問題を修正。読み取り専用の`browser_batch`呼び出しは正しく自動許可されるようになった。

## 原文（英語）

Fixed plan mode not prompting for state-changing browser tool calls; read-only `browser_batch` calls are now correctly auto-allowed

## 概要

plan modeでは、ファイルを編集したりコマンドを実行したりせずに調査・計画のみを行うことが期待されており、Chrome連携でもページを読み取るだけの操作（`read_page`や`get_page_text`など）は確認なしで実行され、クリックや入力、ナビゲーションなど状態を変更する操作は承認を求める設計になっています。修正前はこの判定にいくつか不備があり、状態を変更するブラウザ操作なのに確認プロンプトが出ないケースや、逆に読み取り専用のはずの`browser_batch`（複数のブラウザ操作をまとめて実行するツール）が本来の意図どおりに自動許可されないケースがありました。v2.1.199でこの判定ロジックが修正され、状態変更を伴う呼び出しは確実に確認を求め、中身がすべて読み取り専用の`browser_batch`呼び出しのみ確認なしで実行されるようになりました。

## 基本的な使い方

**修正前:**
```
plan mode中に以下のようなブラウザ操作が発生した場合:
- クリックや入力、ページ遷移などの状態変更操作
  → ❌ 本来なら承認を求めるべきだが、確認なしで実行されてしまうことがあった
- read_page などの読み取りのみで構成された browser_batch
  → ❌ 読み取り専用のはずなのに承認プロンプトが出ることがあった
```

**修正後（v2.1.199）:**
```
plan mode中のブラウザ操作:
- クリック・入力・ページ遷移・タブ操作・GIF録画などの状態変更操作
  → ✅ 必ず承認プロンプトが表示される
- read_page / get_page_text / find / コンソールやネットワークの読み取り /
  スクリーンショット撮影など、純粋に読み取りのみの browser_batch
  → ✅ 確認なしで自動的に実行される
- createIfEmpty（tabs_context_mcp）、console/networkリーダーのclear、
  スクリーンショットの save_to_disk など、一見読み取り系でも
  状態を変更するフラグが立った呼び出し
  → ✅ こちらも承認プロンプトが表示されるようになった
```

## 実践例

### plan modeでのWebアプリ調査

plan modeでコードベースを分析しながら、実際の挙動を確認するためにローカルのWebアプリをブラウザで開いてページ内容やコンソールログを読み取るような使い方をしている場合、読み取りだけの`browser_batch`呼び出しは確認なしでスムーズに実行され、計画作成のための調査を妨げません。

### plan mode中に誤ってフォームを送信してしまう事故の防止

plan modeは「編集・実行前に承認を得る」ことが前提のモードです。修正前は、クリックや入力といった状態を変更するブラウザ操作が確認なしに実行されてしまう可能性があり、意図せずフォーム送信やボタンクリックなどの副作用を引き起こすリスクがありました。修正後は、こうした操作について確実に承認を求めるため、plan modeの「変更を加えない」という前提が正しく守られます。

### スクリーンショットの保存やコンソールログのクリアを伴う操作

スクリーンショットをディスクに保存する（`save_to_disk`）操作や、コンソール・ネットワークログをクリアする操作は一見「読み取り系」に見えますが、実際には状態を変更するフラグです。v2.1.199以降はこうした呼び出しも承認プロンプトの対象になるため、plan mode中に気づかないうちにローカルへファイルが保存されたり、ログが消去されたりすることを防げます。

## 注意点

- plan modeでの読み取り専用/状態変更の分類は、`browser_batch`にまとめられた個々の操作すべてが読み取り専用である場合にのみ自動許可されます。1つでも状態変更を伴う操作が含まれていれば、バッチ全体が承認対象になります。
- サイト単位のブラウザ権限（どのサイトを閲覧・操作してよいか）はChrome拡張機能側の設定から引き継がれるため、plan modeの承認プロンプトとは別に拡張機能側の設定も確認しておくとよいでしょう。
- この修正はplan modeとClaude in Chromeの組み合わせに関するものです。plan mode自体の基本的な挙動（ファイル編集やコマンド実行が承認制になる点）に変更はありません。
