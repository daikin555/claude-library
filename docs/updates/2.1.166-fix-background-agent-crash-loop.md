## 原文（日本語に翻訳）

git worktree に入ったバックグラウンドエージェントセッションが `claude agents` から再開しようとすると「No conversation found」というエラーでクラッシュループするバグを修正しました。

## 原文（英語）

Fixed background agent sessions that entered a git worktree crash-looping with "No conversation found" when reopened from `claude agents`

## 概要

git worktree を使用しているバックグラウンドエージェントセッションを `claude agents` から再開しようとすると、「No conversation found」エラーが繰り返し発生してクラッシュループに陥るバグを修正しました。この修正により、git worktree を活用したマルチブランチ並行作業の再開が正常に機能します。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
```

## 実践例

### 修正前の問題

以前は以下のワークフローでクラッシュループが発生していました：

```bash
# worktree を使用したバックグラウンドエージェントを起動
claude --bg --worktree "feature/my-feature" "テストを書いてください"

# agents から再開しようとするとクラッシュループ
claude agents
# → セッションを選択すると "No conversation found" エラーが繰り返し発生
```

### 修正後の動作

v2.1.166 以降では：

```bash
# worktree を使用したバックグラウンドエージェント起動
claude --bg "feature ブランチでテストを書いてください"

# agents から正常に再開できる
claude agents
# → セッションを選択するとスムーズに再開
```

### git worktree を使った並行開発フロー

```bash
# 複数のブランチで並行して作業
git worktree add ../feature-a feature/branch-a
git worktree add ../feature-b feature/branch-b

# 各 worktree でバックグラウンドエージェントを起動
claude --bg "feature-a の実装を進めてください"
# → 別の worktree でも同様に起動

# agents で全セッションを確認・再開
claude agents
```

## 注意点

- この修正は git worktree を使用したバックグラウンドエージェントに特有の問題でした
- 通常の（worktree を使わない）バックグラウンドエージェントは影響を受けていませんでした
- git worktree と `claude agents` を組み合わせた使用が安定するようになりました
