## 原文（日本語に翻訳）

スキルの説明と `/skills` メニューにプラグイン名を追加し、見つけやすさを改善しました。

## 原文（英語）

Added plugin name to skill descriptions and `/skills` menu for better discoverability

## 概要

スキルの説明文と `/skills` コマンドのメニュー表示に、そのスキルが属するプラグイン名が表示されるようになりました。これにより、どのプラグインがどのスキルを提供しているかが明確になり、スキルの管理と発見が容易になります。

## 基本的な使い方

`/skills` コマンドを実行すると、各スキルの説明とともにプラグイン名が表示されます。

```bash
claude
# プロンプト内で
/skills
```

表示されるスキルリストには、各スキルがどのプラグインから提供されているかが明示されるようになりました。

## 実践例

### スキル一覧の確認

プラグイン名が表示されることで、スキルの出所が明確になります。

```bash
# スキル一覧を表示
/skills

# 出力例（プラグイン名付き）：
# - plugin-name:skill-name - スキルの説明
# - document-skills:pptx - プレゼンテーション作成と編集
# - frontend-design:webapp - Webアプリケーション開発
```

### プラグインごとのスキル管理

どのプラグインがどのスキルを提供しているかを把握できます。

```bash
# プラグイン一覧の確認
/plugins

# 特定のプラグインのスキルを確認
# /skills コマンドで plugin-name: プレフィックスを確認
```

### スキルの衝突回避

プラグイン名前空間により、同名のスキルが衝突しません。

```markdown
# 異なるプラグインから同じスキル名を使用可能
plugin-a:skill-name
plugin-b:skill-name
```

## 注意点

- プラグインスキルは `plugin-name:skill-name` の名前空間を使用します
- この形式により、他のレベルのスキルと衝突しません
- スキルが多すぎる場合、文字数制限（デフォルト15,000文字）を超える可能性があります
- `/context` コマンドで除外されたスキルの警告を確認できます
