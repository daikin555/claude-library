## 原文（日本語に翻訳）

サブエージェント/フッターパネルを表示中にプロンプト入力欄をクリックしてもフォーカスが戻らない問題を修正

## 原文（英語）

Fixed clicking the prompt input not returning focus from the subagent/footer panel

## 概要

サブエージェントの表示やフッターパネルが開いている状態でプロンプト入力欄をクリックしても、入力フォーカスがプロンプト欄に移動しない問題がありました。ユーザーはテキストを入力しようとしても何も入力できない状態になっていました。この修正により、パネルが表示されている状態でもプロンプト入力欄をクリックすると正しくフォーカスが移動するようになりました。

## 基本的な使い方

この修正は自動的に適用されます。サブエージェントの結果やフッターパネルを参照しながら、プロンプト入力欄をクリックすることで即座に入力できるようになりました。

```
# 操作フロー
1. サブエージェントの結果またはフッターパネルを表示
2. プロンプト入力欄をクリック
3. → フォーカスが即座に入力欄に移動
4. → 入力が可能になる
```

## 実践例

### サブエージェント結果を参照しながら次の指示を入力

```
# シナリオ：サブエージェントがコード分析を完了
[サブエージェント分析結果が表示されている]
"以下の問題が見つかりました：
  1. パフォーマンスの問題
  2. 型エラー"

# プロンプト入力欄をクリック（修正後：フォーカスが戻る）
# → "1番目の問題を修正してください" と入力可能になる
```

### フッターパネル操作後の入力

```
# Ctrl+R でコンテキストビューを開いた後
# プロンプト入力欄をクリックすると
# → フォーカスが戻り、すぐに入力開始できる
```

## 注意点

- この問題はサブエージェントパネルとフッターパネルの両方で発生していました
- 以前はEscキーでパネルを閉じることでフォーカスを戻す回避策が必要でしたが、修正後はその必要がなくなりました
- マウス操作だけでなく、Tab/Shift+Tabキーでの移動も正常に機能します
