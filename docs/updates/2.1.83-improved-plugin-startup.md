## 原文（日本語に翻訳）

プラグイン起動を改善しました。コマンド、スキル、エージェントを再フェッチなしにディスクキャッシュから読み込むようになりました。

## 原文（英語）

Improved plugin startup — commands, skills, and agents now load from disk cache without re-fetching

## 概要

プラグインが提供するコマンド、スキル、エージェントが、起動時にネットワークから再取得（re-fetch）されず、ディスクキャッシュから直接読み込まれるように改善されました。これにより、プラグインが多い環境やネットワークが遅い環境での Claude Code 起動が大幅に速くなります。

## 基本的な使い方

プラグインをインストールしている場合、起動が速くなります。追加の操作は不要です。

```bash
# プラグインが多くインストールされている場合でも
$ claude
# 修正後: ディスクキャッシュから読み込むため素早く起動する
```

## 実践例

### ユースケース: 多数のプラグインが入った環境での起動

複数のプラグイン（スキル、エージェント、コマンド提供プラグインなど）がインストールされている場合に効果的です。

```bash
# 例: 5 つのプラグインをインストール済み
$ claude plugin list
plugin-1  # コマンド提供
plugin-2  # スキル提供
plugin-3  # エージェント提供
plugin-4  # MCP サーバー
plugin-5  # 複合プラグイン

$ claude
# 修正後: 全プラグインがキャッシュから読み込まれ素早く起動
```

### ユースケース: オフラインや低速ネットワーク環境

ネットワークが不安定な環境でも、キャッシュから読み込むため安定して起動します。

```bash
# モバイル通信など遅いネットワーク
$ claude
# 修正後: ネットワーク状態に関わらずキャッシュから高速起動
```

## 注意点

- ディスクキャッシュは一定期間後または更新が必要な場合に自動更新されます
- プラグインを更新した場合は次回起動時にキャッシュが更新されます
- キャッシュが古くなっている場合、バックグラウンドで更新が行われることがあります
