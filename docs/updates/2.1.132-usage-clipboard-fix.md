## 原文（日本語に翻訳）

Linux/X11上でクリップボードへの統計スクリーンショットのコピー時に `/usage` の Ctrl+S がハングする問題を修正しました。

## 原文（英語）

Fixed `/usage` Ctrl+S hanging when copying the stats screenshot to the clipboard on Linux/X11

## 概要

Linux環境でX11ディスプレイサーバーを使用している場合、`/usage` コマンドで表示される統計画面において `Ctrl+S` でスクリーンショットをクリップボードにコピーしようとすると、操作がハング（応答なし）する問題がありました。この修正によりLinux/X11環境でもクリップボードへのコピーが正常に機能するようになります。

## 基本的な使い方

`/usage` コマンドでトークン使用量の統計を表示し、`Ctrl+S` でスクリーンショットをクリップボードにコピーできます（修正後はLinux/X11でも動作）。

```
# 使用量統計を表示
/usage

# 統計画面が表示されたら
# Ctrl+S でスクリーンショットをクリップボードにコピー
# 修正前（Linux/X11）: ハングして操作不能になる
# 修正後（Linux/X11）: 正常にコピーされる
```

## 実践例

### /usageコマンドの基本的な使い方

```
# トークン使用量の統計を確認
/usage

# 表示される情報の例:
# ─────────────────────────────────────────
# セッション統計
# ─────────────────────────────────────────
# 入力トークン:    45,230
# 出力トークン:    12,450
# キャッシュヒット: 8,900
# 合計トークン:    57,680
# 推定コスト:     $0.85
# ─────────────────────────────────────────

# Ctrl+S でこの統計画面をクリップボードにコピー
# Ctrl+Q または q で画面を閉じる
```

### Linux環境でのクリップボードツールの確認

```bash
# Linux/X11でのクリップボードサポートを確認

# xclipが利用可能か確認
which xclip
# または
xclip -version

# xselが利用可能か確認
which xsel
# または
xsel --version

# xclipのインストール（Debian/Ubuntu）
sudo apt-get install xclip

# xclipのインストール（Fedora/RHEL）
sudo dnf install xclip

# xclipのインストール（Arch Linux）
sudo pacman -S xclip
```

### Wayland環境での使用

```bash
# Wayland環境（X11の代替）を使用している場合

# wl-clipboardが利用可能か確認
which wl-copy
wl-copy --version

# wl-clipboardのインストール（Debian/Ubuntu）
sudo apt-get install wl-clipboard

# DISPLAY変数の確認（X11）
echo $DISPLAY  # 例: :0

# WAYLAND_DISPLAY変数の確認（Wayland）
echo $WAYLAND_DISPLAY  # 例: wayland-0
```

### 統計データの活用

```
# クリップボードにコピーした統計データの活用例

# /usageでCtrl+S後、スプレッドシートに貼り付け
# テキストとしてコピーされるため、ドキュメントに貼り付けてコスト管理ができる

# 例: 日次のClaude Code使用状況レポート
日付       | 入力トークン | 出力トークン | コスト
2026-05-06 | 45,230      | 12,450      | $0.85
2026-05-05 | 32,100      | 8,900       | $0.60
```

## 注意点

- この問題はLinux環境でX11ディスプレイサーバーを使用している場合に発生していました。macOSおよびWindowsでは発生しません。
- Waylandを使用している場合も同様の問題が発生する可能性がありましたが、本修正で対応されています。
- クリップボードへのコピーには `xclip` または `xsel` パッケージが必要な場合があります。インストールされていない場合はエラーメッセージが表示されます。
- SSH経由でリモートサーバーにアクセスしている場合、クリップボードへのコピーはX11フォワーディングの設定が必要な場合があります。
