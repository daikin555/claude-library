## 原文（日本語に翻訳）

権限を持つユーザーに対して `opusplan` モデル設定がプランモードで1Mコンテキストを使用しない問題を修正しました。`opusplan[1m]` の回避策もプランモードでOpusに正しく切り替わるようになりました。

## 原文（英語）

Fixed `opusplan` model setting not shipping with 1M context in plan mode for entitled users; the `opusplan[1m]` workaround now also correctly switches to Opus in plan mode

## 概要

プランモード専用のモデル設定 `opusplan` が、1Mコンテキストを使用する権限を持つユーザーでも1Mコンテキストなしで起動していた問題が修正されました。また、回避策として使用されていた `opusplan[1m]` 設定も、プランモードで正しくOpusモデルに切り替わらない問題が同時に修正されました。

## 基本的な使い方

```json
// .claude/settings.json
{
  "model": "opusplan"
}
```

```
/plan
# 修正後: 1Mコンテキスト権限があれば、自動的に1Mコンテキスト付きOpusでプランモードが起動
```

## 実践例

### プランモードで大規模コードベースを設計する

```json
// settings.json
{
  "model": "opusplan"  // プランモード時にOpus + 1M contextを使用
}
```

```
# プランモードを起動
/plan
# または
/plan コードベース全体のリアーキテクチャを設計してください

# 修正後:
# - 1Mコンテキスト権限がある場合: Opus + 1Mコンテキストで起動
# - 1Mコンテキスト権限がない場合: 標準のOpusで起動
```

### opusplan[1m]の回避策が不要になった

```json
// 修正前: 回避策として必要だった
{
  "model": "opusplan[1m]"
}

// 修正後: シンプルに opusplan のみで1Mコンテキストが有効になる（権限がある場合）
{
  "model": "opusplan"
}
```

## 注意点

- `opusplan` はプランモード（`/plan`）での実行時にのみOpusモデルを使用します。通常の会話では別のモデル設定が使用されます
- 1Mコンテキストを使用するには、アカウントに対応する使用クレジットまたは権限が必要です
- `opusplan[1m]` 形式も引き続き使用できますが、`opusplan` のみで十分です
