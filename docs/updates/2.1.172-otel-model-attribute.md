## 原文（日本語に翻訳）

`claude_code.lines_of_code.count` OTELメトリクスに `model` アトリビュートが追加されました

## 原文（英語）

Added `model` attribute to the `claude_code.lines_of_code.count` OTEL metric

## 概要

OpenTelemetry（OTEL）の `claude_code.lines_of_code.count` メトリクスに、使用モデルを示す `model` アトリビュートが追加されました。これにより、どのモデル（Sonnet、Opus、Haiku等）がどれだけのコード行数を処理したかをモデル別に集計・可視化できるようになります。

## 基本的な使い方

OTELエクスポーターを設定すると、`claude_code.lines_of_code.count` メトリクスに `model` タグが自動的に付与されます：

```bash
# OTEL設定例
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318
export OTEL_SERVICE_NAME=claude-code
claude
```

メトリクスのサンプル出力：
```
claude_code.lines_of_code.count{model="claude-sonnet-4-6"} 1250
claude_code.lines_of_code.count{model="claude-opus-4-8"} 320
claude_code.lines_of_code.count{model="claude-haiku-4-5"} 580
```

## 実践例

### Grafanaでモデル別コスト分析

```promql
# モデル別の処理行数を集計するPromQLクエリ
sum by (model) (
  rate(claude_code_lines_of_code_count[5m])
)
```

### Datadogでのモデル利用傾向の可視化

```yaml
# datadog-agent.yaml の設定例
init_config:
logs:
  - type: otlp
    service: claude-code
    tags:
      - env:production
# ダッシュボードでmodel別にグループ化して分析
```

### モデル切り替えの効果測定

```bash
# 異なるモデルで同じタスクを実行してコード処理量を比較
ANTHROPIC_MODEL=claude-haiku-4-5-20251001 claude "コードを生成してください"
# vs
ANTHROPIC_MODEL=claude-sonnet-4-6 claude "コードを生成してください"
# OTELダッシュボードで両モデルの行数処理効率を比較
```

## 注意点

- OTELの設定が有効になっている場合のみ `model` アトリビュートが付与されます
- `OTEL_EXPORTER_OTLP_ENDPOINT` を設定することでメトリクスのエクスポートが有効になります
- モデルIDはAnthropicのAPIモデルID形式（例: `claude-sonnet-4-6`）で記録されます
