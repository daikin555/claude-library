## 原文（日本語に翻訳）

「review my auth changes」のような説明的な引数を渡すと`/ultrareview`が失敗する問題を修正しました。現在は、現在のブランチに対してレビューを実行し、渡されたテキストは所見（findings）へのメモとして適用されます

## 原文（英語）

Fixed `/ultrareview` failing on descriptive arguments like "review my auth changes" — they now run a review of your current branch with the text applied as a note to the findings

## 概要

`/ultrareview`は、クラウド上でより深いコードレビューを実行するコマンドです。以前は、PR番号やブランチ名のような構造化された引数を想定しており、「review my auth changes」のような自然言語の説明文を引数として渡すとコマンドが失敗していました。修正後は、こうした説明的なテキストが渡された場合、それをレビュー対象の指定ではなく「レビュー結果に付与するメモ」として扱い、現在のブランチに対して通常どおりレビューを実行するようになりました。

## 基本的な使い方

```bash
# 現在のブランチに対してクラウドレビューを実行し、
# 「認証まわりの変更に注目してほしい」という意図をメモとして添える
/ultrareview review my auth changes
```

## 実践例

### 特定の観点を伝えながらレビューを依頼する

```bash
# 現在のブランチの変更をレビューしつつ、
# 「認証周りの変更」に注目してほしい旨をメモとして残す
/ultrareview review my auth changes

# 実行結果:
# - 現在のブランチに対してクラウドレビューが開始される
# - レビューの所見（findings）に「review my auth changes」というメモが添付される
```

### 引数なしで実行する

```bash
# 引数を省略した場合は、これまでどおり現在のブランチに対して
# レビューが実行される
/ultrareview
```

### PRなど構造化された引数と組み合わせる

```bash
# 構造化された引数（PR番号など）は従来どおりそのまま解釈される
/ultrareview #123
```

## 注意点

- Claude Code v2.1.218で修正
- 説明的なテキストはレビュー対象の指定としては使われず、あくまで所見へのメモとして扱われる
- レビュー自体は常に「現在のブランチ」に対して実行される
- `/ultrareview`はクラウド上で実行される深いレビュー機能であり、ローカルのみで完結する通常のコードレビューとは挙動が異なる場合がある
