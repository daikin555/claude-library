## 原文（日本語に翻訳）

`/config` のトグル動作を変更しました：Enter と Space の両方で選択した設定を変更できるようになり、Esc キーが元に戻すのではなく保存して閉じるようになりました。

## 原文（英語）

Changed `/config` toggle behavior: Enter and Space both change the selected setting, and Esc now saves and closes instead of reverting.

## 概要

`/config` のインタラクティブ UI におけるキーボード操作が改善されました。トグル型の設定項目（オン/オフ切り替えなど）に対して、Enter キーと Space キーの両方が同じ動作（設定の変更）を行うようになりました。また、Esc キーの動作が「変更を元に戻す（リバート）」から「保存して閉じる」に変わりました。

## 基本的な使い方

### /config UI での操作

```bash
# /config を起動
/config

# キーボード操作：
# ↑↓ : 設定項目を移動
# Enter または Space : トグル設定を切り替え
# Esc : 現在の状態を保存して閉じる（以前は変更を元に戻していた）
```

## 実践例

### 以前の動作との比較

**以前の動作（変更前）：**
```
1. /config を開く
2. ↑↓ でトグル設定を選択
3. Enter で設定を変更
4. Space は無効（何も起きない場合があった）
5. Esc で変更を元に戻して閉じる
```

**新しい動作（変更後）：**
```
1. /config を開く
2. ↑↓ でトグル設定を選択
3. Enter または Space で設定を変更（どちらも同じ動作）
4. Esc で保存して閉じる（変更は保持される）
```

### 典型的な設定変更フロー

```
/config
# ↓↓ で目的の設定項目に移動
# Space を押してトグル（例：テーマのダーク/ライト切り替え）
# Esc で閉じる → 変更が保存される
```

## 注意点

- **重要な変更**：Esc キーの動作が変わりました。以前は「キャンセル」でしたが、現在は「保存して閉じる」になっています
- 誤って設定を変更してしまった場合は、もう一度 Enter または Space で元に戻してから Esc で閉じてください
- この変更は `/config` のインタラクティブ UI のみに適用されます。`/config key=value` 形式のコマンド入力には影響しません
- Space キーによるトグルは多くのターミナルアプリケーションで一般的な操作です（例：ファイル選択ダイアログ）
