## 原文（日本語に翻訳）

`/model`、`/config`、およびその他の選択画面でスクロール時にヘッダーが消える問題を修正

## 原文（英語）

Fixed headers disappearing when scrolling `/model`, `/config`, and other selection screens

## 概要

`/model` や `/config` などのインタラクティブな選択画面でリストをスクロールすると、画面上部のヘッダー（タイトルやナビゲーション情報）が消えてしまうUIバグが修正されました。ヘッダーが消えることで、どの選択画面にいるか分かりにくくなったり、操作方法が見えなくなる問題がありました。v2.1.90 以降では、スクロール中もヘッダーが正しく表示され続けます。

## 基本的な使い方

修正内容はUIバグフィックスのため、v2.1.90 にアップデートするだけで問題が解消されます。影響を受けていたコマンドは通常通り使用できます：

```bash
# モデル選択画面（ヘッダーが正常に表示される）
/model

# 設定画面（ヘッダーが正常に表示される）
/config
```

## 実践例

### モデル選択での使用

`/model` コマンドで多数のモデルが表示される場合にスクロールしても、ヘッダーが維持されます：

```
/model
# 表示例（v2.1.90以降）:
# ╔══════════════════════════════╗
# ║  モデルを選択してください    ║  ← ヘッダーがスクロール中も表示
# ╚══════════════════════════════╝
# > claude-opus-4-6
#   claude-sonnet-4-6
#   claude-haiku-4-5
#   ... スクロールしてもヘッダーは消えない
```

### 設定画面での使用

`/config` 画面で設定項目が多い場合のスクロール操作：

```
/config
# 設定項目が多くてスクロールが必要でも
# 画面上部のヘッダーが常に表示される
```

## 注意点

- このバグは選択肢が多くスクロールが必要な画面でのみ発生していました
- 特に多数のモデルや設定が表示される環境で顕著でした
- v2.1.90 へのアップデートで全ての選択画面でヘッダーが安定表示されます
