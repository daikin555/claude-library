## 原文（日本語に翻訳）

モデルが思考ブロックのみを返したときに、ターンが目に見える出力なしに静かに完了していた問題を修正しました。Claude は今後、この場合に一度再プロンプトを行います。

## 原文（英語）

Fixed turns silently completing with no visible output when the model returned only a thinking block; Claude now re-prompts once.

## 概要

Extended Thinking（拡張思考）機能を使用する際、モデルが思考ブロック（`<thinking>` の内容）のみを生成して通常のテキスト出力を返さない場合、ターンが何も表示せずに無音で完了するバグが修正されました。修正後は、このような場合に Claude が自動的に一度再プロンプトを実行し、実際のレスポンスを取得します。

## 基本的な使い方

この修正は自動的に適用されます。思考ブロックのみが返された場合：

```bash
# 以前の動作
ユーザー: "複雑な問題を解決してください"
Claude: [内部で思考するが、テキスト出力なし → 無音でターン終了]

# 修正後の動作
ユーザー: "複雑な問題を解決してください"
Claude: [内部で思考] → [自動的に再プロンプト] → "こちらが解決策です..."
```

## 実践例

### Extended Thinking が有効な環境での動作

```bash
# Extended Thinking が有効なモデル（例：Claude Opus）を使用
claude

# 複雑な問題を尋ねた場合
ユーザー: 「この複雑なアルゴリズムの時間計算量を分析してください」

# 修正前：思考のみで応答なし（画面は空白のまま）
# 修正後：思考後に自動再プロンプト → 適切な回答が表示される
```

### 影響を受けやすいケース

- 非常に複雑な問題で Extended Thinking が深く実行される場合
- モデルが思考プロセスで問題を「完結」させてしまう場合
- 長い思考ブロックが生成される設定での動作

## 注意点

- 再プロンプトは **一度だけ** 自動実行されます（無限ループを防ぐため）
- Extended Thinking が無効な設定では、この問題は発生しません
- この修正により、一部のケースでレスポンス時間がわずかに長くなる場合があります（再プロンプトの分）
- ユーザーから見ると、レスポンスが来ない状態が改善され、必ず何らかの出力が返るようになります
