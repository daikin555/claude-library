## 原文（日本語訳）

セッション中に変更したスキルやコマンドが、再起動するまでスラッシュメニューに表示されない問題を修正しました。

## 原文（英語）

Fixed skills and commands changed during a session not appearing in the slash menu until restart

## 概要

これまでは、Claude Codeのセッションを実行中に `.claude/skills/` や `.claude/commands/` 配下のファイルを追加・編集しても、`/` を入力したときに表示されるスラッシュコマンドメニューにはその変更が反映されず、Claude Codeを再起動するまで古い一覧のままになる不具合がありました。今回の修正により、セッションを継続したままスキルやコマンドを変更しても、スラッシュメニューが最新の状態に自動的に更新されるようになります。

## 基本的な使い方

この修正は自動的に適用される挙動であり、特別な設定は不要です。セッション実行中に以下のようなファイルを追加・変更・削除すると、次に `/` を入力してメニューを開いたときに最新の内容が反映されます。

```bash
# セッションを実行したまま、別ターミナルで新しいスキルを追加
mkdir -p .claude/skills/my-new-skill
cat > .claude/skills/my-new-skill/SKILL.md <<'EOF'
---
name: my-new-skill
description: 新しく追加したスキル
---
このスキルの説明本文
EOF
```

再起動せずにClaude Codeのセッションで `/` を入力すると、`my-new-skill` がメニューに表示されます。

## 実践例

### スキルを開発しながらその場で動作確認する

スキルやコマンドを作成中は、Claude Codeを起動したまま `SKILL.md` を編集し、そのまま `/` メニューで反映を確認できるようになりました。従来のように「編集 → 再起動 → 確認」というサイクルを回す必要がなくなり、開発イテレーションが高速化します。

```bash
.claude/skills/deploy-helper/SKILL.md を編集
→ Claude Codeセッションはそのまま
→ "/" と入力してメニューを開く
→ 変更後の deploy-helper がすぐに一覧に表示される
```

### プロジェクトコマンドをチームで共有・調整する

複数人でリポジトリの `.claude/commands/` を編集している場合、他のメンバーが追加したコマンドファイルをpullしてきた際にも、セッションを再起動せずにそのままメニューに反映されるため、コラボレーション時の手間が減ります。

### プラグインやスキルの一時的な無効化・有効化を試す

スキルファイルをリネームしたり一時的に別ディレクトリへ退避したりして有効/無効を切り替える際も、メニューの表示が即座に追従するため、動作確認のたびにセッションを立て直す必要がありません。

## 注意点

- 反映されるのはメニューの表示更新のタイミングのみで、実際にスキル/コマンドの内容が変わったこと自体を検知する仕組みです。ファイルのフォーマットや`SKILL.md`のfrontmatter自体が不正な場合は引き続きエラーになります
- プラグイン経由のスキル・コマンドについても同様に反映対象です
- 反映タイミングはあくまで次回メニューを開いたときであり、既に実行中のツール呼び出しなどには影響しません
