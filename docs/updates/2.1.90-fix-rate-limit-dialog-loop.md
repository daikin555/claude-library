## 原文（日本語に翻訳）

使用量上限に達した後、レート制限オプションダイアログが繰り返し自動的に開き続け、最終的にセッションがクラッシュする無限ループを修正

## 原文（英語）

Fixed an infinite loop where the rate-limit options dialog would repeatedly auto-open after hitting your usage limit, eventually crashing the session

## 概要

使用量上限（レート制限）に達した際に表示されるオプションダイアログが、自動的に何度も開き続け、最終的にClaude Codeセッションがクラッシュするバグが修正されました。このバグはセッションの強制終了を引き起こし、進行中の作業を失う可能性がありました。v2.1.90 以降では、ダイアログは適切に一度だけ表示されます。

## 基本的な使い方

修正内容はバグフィックスのため、特別な操作は不要です。v2.1.90 にアップデートすることで、レート制限に達した際のダイアログが正常に動作するようになります：

```bash
# Claude Codeのアップデート
npm update -g @anthropic-ai/claude-code

# バージョン確認
claude --version
```

## 実践例

### レート制限到達時の正常な動作

v2.1.90 以降では、使用量上限に達した際に以下のように正常動作します：

1. レート制限ダイアログが**一度だけ**表示される
2. ユーザーが選択肢を選ぶ（待機、APIキー入力など）
3. ダイアログが閉じ、選択に応じた処理が行われる

### 長時間のセッションでの作業継続

レート制限が発生しやすい長時間・大量のコード生成作業でも、セッションが安定して継続できるようになりました：

```bash
# 長時間作業でもセッションが安定
claude
# ... 大量の処理 ...
# レート制限に達した場合でもクラッシュしない
```

## 注意点

- v2.1.90 以前のバージョンを使用している場合は、このバグが発生する可能性があります
- レート制限に達した場合は、表示されるダイアログの指示に従ってください
- 使用量上限はプランによって異なります。詳細はAnthropic公式サイトで確認してください
