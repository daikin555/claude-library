## 原文（日本語に翻訳）

折りたたまれた読み取り/検索グループが画面外でスクロール後に完了したときに、スクロールバックが飛ぶ問題を修正しました。

## 原文（英語）

Fixed scrollback jumping when collapsed read/search groups finish after scrolling offscreen

## 概要

会話履歴を上にスクロールして過去のメッセージを閲覧中、折りたたまれたファイル読み取りや検索のグループが完了すると、スクロール位置が突然別の場所に飛んでしまう問題が修正されました。これにより、過去の会話を確認しながら Claude の応答を待つ際のスクロール体験が安定します。

## 基本的な使い方

会話履歴を参照しながら Claude が作業を完了するまで待つ際、スクロール位置が意図せず変わらなくなりました。

```bash
$ claude
> （いくつかのタスクを実行した後）
> 別のタスクをお願いします
# Claude が作業中に、以前の会話を確認するためにスクロールアップ
# 修正前: 完了時にスクロール位置が突然変わった
# 修正後: スクロール位置が安定して維持される
```

## 実践例

### ユースケース: 複数タスク実行中の履歴確認

Claude がバックグラウンドで作業している間に過去の会話を参照する場合に安定します。

```bash
$ claude
> ファイルA, B, C を順番に読んで分析してください
# 分析中...
# 上にスクロールして以前の指示を確認
# 修正後: 各ファイル読み取り完了時にスクロール位置が飛ばない
```

### ユースケース: 長いセッションでのナビゲーション

多数のツール使用（Read、Bash、検索など）を含む長いセッションで特に効果があります。

```bash
# 長い作業セッション後に履歴を確認中
# 折りたたまれたツール呼び出しが完了しても
# スクロール位置が安定して維持される
```

## 注意点

- 「折りたたまれたグループ」は、読み取り操作や検索操作が UI 上でまとめて折りたたまれた状態のことです
- この問題はスクロールアップして画面外の位置を表示している場合のみ発生していました
- 最新のメッセージを表示している状態（スクロールしていない状態）では問題は発生していませんでした
