# --channelsで起動したインタラクティブセッションでplanモードツールが使えない問題を修正

## 原文（日本語に翻訳）

`--channels` で起動したインタラクティブセッションでplanモードのツールが使えなかった問題を修正しました。

## 原文（英語）

Fixed plan-mode tools being unavailable in interactive sessions launched with `--channels`

## 概要

`--channels` フラグを使用して起動したインタラクティブセッションでPlan modeに切り替えた際、Plan modeのツール（`EnterPlanMode`、`ExitPlanMode`など）が利用できず機能しない問題が修正されました。`--channels` による起動でもPlan modeが正常に動作するようになります。

## 基本的な使い方

```bash
# --channels フラグでインタラクティブセッションを起動
claude --channels

# v2.1.126以降: Plan modeが正常に動作する
/plan
# → Plan modeに切り替わり、計画ツールが利用可能になる
```

## 実践例

### --channelsセッションでの計画立案

```bash
# チャンネル接続でセッションを起動
claude --channels ws://localhost:3000

# セッション内でPlan modeを使用
/plan
# → planモードが正常に動作
# → EnterPlanMode / ExitPlanMode ツールが使える

# 計画の立案
"このリファクタリングの実装計画を立てて"
```

## 注意点

- `--channels` は特定のWebSocket接続でClaude Codeを使用する場合のフラグです
- Plan modeは実装に入る前に計画を立案・承認するためのモードです
- この修正は `--channels` を使用するシステム統合で重要です
