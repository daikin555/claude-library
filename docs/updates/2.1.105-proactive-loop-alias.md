## 原文（日本語に翻訳）

`/proactive`が`/loop`のエイリアスになりました。

## 原文（英語）

`/proactive` is now an alias for `/loop`

## 概要

`/proactive`コマンドが`/loop`のエイリアスとして追加されました。`/loop`はプロンプトを指定したインターバルで繰り返し実行するスケジューラーですが、`/proactive`という名前により「Claudeが能動的に定期タスクを実行する」という意図がより明確に伝わります。どちらのコマンドも同じ動作をします。

## 基本的な使い方

```
/proactive 5m /check-status
```

これは5分ごとに`/check-status`スキルを実行します。インターバルの単位：

| 単位 | 説明 | 例 |
|------|------|---|
| `s` | 秒（最小1分に丸め） | `30s` |
| `m` | 分 | `5m`, `30m` |
| `h` | 時間 | `1h`, `2h` |
| `d` | 日 | `1d` |

## 実践例

### ビルド状況を定期確認

```
/proactive 10m /check-build-status
```

10分ごとにビルド状況を確認し、問題があれば報告します。

### デプロイ完了を監視

```
/proactive 2m check if the deployment at staging is complete and report the status
```

2分ごとにデプロイ状況を確認し、完了したら通知します。

### コード品質の定期チェック

```
/proactive 1h /lint-and-report
```

1時間ごとにlintを実行してレポートを生成します。

### `/loop`との比較

```
# どちらも同じ動作
/loop 5m /my-task
/proactive 5m /my-task
```

`/proactive`はClaudeが積極的（proactive）に行動するというコンセプトをより直感的に表現しています。

## 注意点

- `/proactive`と`/loop`は完全に同等です。どちらを使っても同じ結果になります
- ループを停止するには、実行中のプロセスを中断するか、セッションを終了します
- インターバルが短すぎると（例: `1s`）、APIコストが増加する可能性があります
- バックグラウンドモニターとは異なり、`/proactive`はプロンプトを定期的に再送信します
