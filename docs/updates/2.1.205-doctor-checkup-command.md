## 原文（日本語訳）

`/doctor` が、問題を診断して修正できる本格的なセットアップチェックコマンドになった。`/checkup` はそのエイリアスとして追加された。

## 原文（英語）

`/doctor` is now a full setup checkup that can diagnose and fix issues; `/checkup` is its alias

## 概要

`/doctor` はこれまでインストール状態などの簡易的な確認コマンドでしたが、今回のアップデートでインストール種別・バージョン・設定ファイル（settings.json）・MCPサーバー設定・コンテキスト使用量などを一括で監査し、問題を検知したうえで修正まで提案・実行できる本格的なセットアップチェックアップ機能に強化されました。あわせて、より覚えやすい `/checkup` というエイリアスも追加されています。

## 基本的な使い方

Claude Codeのセッション内で以下のどちらかを実行するだけです。

```text
/doctor
```

```text
/checkup
```

数秒で診断結果が一覧表示され、各項目が緑（正常）・黄（警告）・赤（エラー）のステータスで示されます。修正可能な問題が見つかった場合は、案内に従ってキー操作（例: `f`）を押すとClaudeが該当箇所を自動的に修正してくれます。

## 実践例

### 導入直後の初回セットアップ確認

Claude Codeを新しくインストールした直後や、マシンを乗り換えた際に、インストールタイプ（npmグローバル、ローカル、ネイティブなど）や重複インストールの有無、`ripgrep` の状態などをまとめて確認したい場合に使います。

```text
/checkup
```

問題があれば具体的な原因と対処法が提示されるため、手動でドキュメントを探し回る前にまず実行するのがおすすめです。

### MCPサーバーや設定ファイルの不調時の診断

MCPサーバーが読み込まれない、Claudeの起動が不安定、`settings.json` を編集してから挙動がおかしい、といったトラブルが起きたときに実行します。

```text
/doctor
```

設定ファイルの構文エラーやMCP設定の不備、コンテキストを圧迫している巨大な `CLAUDE.md` やカスタムエージェント定義などが検出され、修正の提案が表示されます。Claudeがまったく起動しない場合は、Claude Code内からではなくシェルから `claude doctor` を実行することでも同様の診断が可能です。

## 注意点

- 自動修正の対象は主に設定ファイルの不備やインストール環境に関する問題であり、すべての不具合が自動修正できるわけではありません。修正できない項目については原因と手動対応の指針が示されます。
- `/checkup` は `/doctor` の完全なエイリアスであり、機能面での違いはありません。好みで使い分けて構いません。
- Claude Code自体が起動できないほど深刻な状態では、セッション内のスラッシュコマンドではなくターミナルから `claude doctor` を実行する必要があります。
