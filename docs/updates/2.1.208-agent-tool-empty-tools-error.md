## 原文（日本語訳）

サブエージェントの `tools` リストがどのツールにも解決できない場合にAgentツールがツールなしで起動してしまう不具合を修正した。現在は、認識できなかった項目名を明示したエラーを返すようになった。

## 原文（英語）

Fixed the Agent tool launching with no tools when a subagent's `tools` list resolves to nothing — it now returns a clear error naming the unrecognized entries

## 概要

サブエージェントの定義ファイルではYAMLフロントマターの `tools` フィールドで、そのサブエージェントが使えるツールを許可リストとして指定できます。これまでは、`tools` に列挙したすべての項目がスペルミスなどで実在しないツール名だったり、サブエージェントには渡せないツール（`AskUserQuestion` など）だけだったりして、結果として1つもツールが解決できない場合でも、サブエージェントはツールを一切持たない状態でそのまま起動してしまい、空の結果や意味不明な結果を返すことがありました。今回の修正により、Claude Codeはこのようなサブエージェントの起動自体を拒否し、Agentツールが「どの項目が解決できなかったか」を名指しした明確なエラーを返すようになりました。

## 基本的な使い方

サブエージェントの `tools` フィールドには、使用を許可するツール名をカンマ区切りで指定します。

```yaml
---
name: safe-researcher
description: 制限付きの調査用エージェント
tools: Read, Grep, Glob, Bash
---
```

## 実践例

### ツール名のスペルミスに気づける

`tools` フィールドに誤ったツール名を書いてしまうケースです。

```yaml
---
name: broken-agent
description: ツール名を間違えて指定してしまった例
tools: Reed, Grpe
---
```

修正前はこのサブエージェントが呼び出されるとツールなしで起動し、原因が分かりにくい失敗をしていました。修正後は、Agentツールが `Reed`, `Grpe` のような未解決の項目名を含んだエラーを返すため、フロントマターの記述ミスにすぐ気づけます。

### サブエージェントに渡せないツールだけを指定してしまった場合

`AskUserQuestion` や `EnterPlanMode` のように、メインの会話のUIやセッション状態に依存しておりサブエージェントには渡せないツールだけを `tools` に指定してしまうケースです。

```yaml
---
name: interactive-only
description: サブエージェントでは使えないツールだけを指定した例
tools: AskUserQuestion
---
```

この場合も1つもツールが解決できないため、修正後は起動時に「Agent would be spawned with zero tools」というエラーが返り、定義ファイルの見直しが必要だと分かります。

## 注意点

- `tools` の一部だけが解決できて残りが未解決の場合はエラーにならず、解決できたツールだけでサブエージェントが起動します。全項目が未解決の場合のみ、このエラーが発生します。
- `disallowedTools` と `tools` を併用している場合、まず `disallowedTools` が適用されてから `tools` が残りのツールプールに対して評価されます。
- Skillsをプリロードしたい場合は `tools` に `Skill` を列挙するのではなく、専用の `skills` フィールドを使います。
