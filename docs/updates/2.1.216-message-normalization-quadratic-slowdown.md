## 原文（日本語訳）

長時間セッションにおいて、メッセージの正規化コストがターン数に対して二次関数的に増加し、数秒単位の停止やレジューム時の遅延を引き起こしていた問題を修正しました。

## 原文（英語）

Fixed a slowdown in long sessions where message normalization cost grew quadratically with the number of turns, causing multi-second stalls and slow resumes

## 概要

Claude Codeはターンを重ねるたびに会話履歴のメッセージを内部的に正規化（normalization）していますが、この処理のコストがセッションのターン数に対して二次関数的（O(n²)）に増加してしまう不具合がありました。その結果、会話が長くなるほど毎ターンの応答前に数秒単位の停止が発生したり、`claude --resume` でセッションを再開する際に読み込みが遅くなったりしていました。今回の修正でこの計算コストの増大が解消され、長時間・多ターンのセッションでも応答性が維持されます。

## 基本的な使い方

この修正はClaude Codeの内部処理に対するものであり、ユーザー側で追加の設定を行う必要はありません。アップデート後は、長時間稼働しているセッションや大量のターンを重ねたセッションでも、ターンごとの応答速度やセッションのレジューム速度が改善されます。

## 実践例

### 長時間の連続セッションでの体感速度改善

数十〜数百ターンにわたる長いペアプログラミングセッションや、エージェントを長時間動かし続けるワークフローでは、以前はターンが増えるごとに応答前の待機時間が伸びていく傾向がありました。アップデート後は、ターン数が増えてもメッセージ正規化にかかる時間がほぼ線形に保たれるため、待機時間の急激な増加が起こりにくくなります。

### セッションレジュームの高速化

```bash
# 長時間動かしていたセッションを再開する
claude --resume
```

これまで、ターン数の多いセッションを `--resume` で再開する際に読み込みが遅くなることがありましたが、今回の修正によりレジューム時の待機時間が改善されます。

### バックグラウンドエージェントの長時間実行

バックグラウンドで長時間稼働させているエージェントセッションでも、正規化コストの増大が原因で応答が段階的に遅くなる現象が緩和されます。長時間タスクを `claude` のバックグラウンドセッション機能で運用している場合に特に効果を実感しやすい修正です。

## 注意点

- この修正はパフォーマンス改善のバグ修正であり、動作や設定項目に変更はありません
- 体感できる改善幅は、セッションのターン数や会話履歴のサイズに依存します。短いセッションでは違いを感じにくい場合があります
- 大幅にターン数が多い場合は、引き続き `/compact` などでコンテキストを整理することが推奨されます
