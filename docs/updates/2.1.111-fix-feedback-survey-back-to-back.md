## 原文（日本語に翻訳）

フィードバックサーベイを一つ閉じた後に次のサーベイが連続して表示される問題を修正しました。

## 原文（英語）

Fixed feedback surveys appearing back-to-back after dismissing one.

## 概要

Claude Code v2.1.111では、フィードバックサーベイ（アンケート）を閉じた直後に別のサーベイが表示されてしまうバグが修正されました。複数のサーベイが連続して表示されることで作業の中断が発生していたため、この修正によりユーザー体験が改善されます。

## 基本的な使い方

この修正はユーザーによる設定変更が不要な自動改善です。

### 修正前後の動作比較

```
# 変更前（バグ）：
# 1. フィードバックサーベイが表示される
# 2. 閉じる/後でをクリック
# 3. すぐに別のサーベイが表示される
# 4. これを繰り返すことがあった

# 変更後（修正済み）：
# 1. フィードバックサーベイが表示される
# 2. 閉じる/後でをクリック
# 3. 適切な間隔が空いてから次のサーベイが表示される
```

## 実践例

### フィードバックサーベイへの適切な対応

```
# サーベイが表示された場合の選択肢：
# 1. 回答する → フィードバックを送信して閉じる
# 2. 後で → 後で表示されるようにスキップ
# 3. 閉じる → このサーベイをスキップ

# 変更後：どの選択をしても連続してサーベイが表示されない
```

### フィードバックサーベイを無効化したい場合

```json
// .claude/settings.json（またはグローバル設定）
{
  "survey": {
    "enabled": false
  }
}
```

```bash
# 環境変数での無効化（確認が必要）
export CLAUDE_DISABLE_SURVEYS=true
```

## 注意点

- フィードバックサーベイはClaude Codeの改善に役立てられます。可能であれば回答することを推奨します
- この修正はサーベイが「閉じる」操作後に適切な間隔が空くよう修正するものです。サーベイ自体を無効化するものではありません
- サーベイの表示頻度はClaude Codeの使用頻度や期間に基づいて決定されます
