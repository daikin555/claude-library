## 原文（日本語訳）

Windowsで、素の（オプションなしの）`claude --resume` の前にセットアッププロンプトが表示された場合に、agents view（エージェント一覧）でキーボード入力が無視されてしまう不具合を修正した

## 原文（英語）

Fixed keyboard input being ignored in the agents view when a setup prompt appeared before a bare `claude --resume` on Windows

## 概要

Windows環境で `claude --resume`（セッションIDなどを指定しない素の状態）を実行すると、初回起動時などに表示されるセットアップ用のプロンプトが先に表示されることがあります。このセットアッププロンプトの後にagents view（複数エージェント／セッションを一覧表示する画面）が開くと、キーボード入力が正しく画面に伝わらず、矢印キーでの選択やフィルタ入力などの操作を受け付けなくなる不具合がありました。今回の修正でこの入力フォーカスの問題が解消され、Windows上でもagents viewの操作が正常に行えるようになりました。

## 基本的な使い方

Windows環境で、セッションを指定せずに `--resume` を実行してagents viewを開きます。

```powershell
claude --resume
```

セットアッププロンプトが表示された場合はそれに従って進めた後、通常通りagents view上で矢印キーやショートカットキーを使って操作できます。

## 実践例

### 初回起動直後にセッション再開を試みる場合

Windows環境にClaude Codeを新規インストールした直後など、初期セットアップがまだ完了していない状態で `claude --resume` を実行するケースです。

```powershell
claude --resume
```

修正前は、セットアッププロンプトの後にagents viewへ遷移すると、矢印キーや `?`（ショートカット一覧表示）などのキー入力が一切反映されないことがありました。修正後は、セットアッププロンプトを経由した場合でもagents view上でのキーボード操作が正しく機能します。

### agents view上で複数セッションを絞り込みたい場合

agents viewでは、ディスパッチ入力欄に文字を入力してセッションをフィルタする操作が可能ですが、これも入力が無視される不具合の影響を受けていました。

```powershell
claude --resume
```

修正後は、セットアッププロンプト経由でagents viewに入った場合でも、フィルタ入力や一覧内の移動といったキーボード操作が期待通りに動作します。

## 注意点

- この不具合はWindows環境かつ「セットアッププロンプトが先に表示された後にagents viewが開く」という特定の順序でのみ発生していました。
- macOS/Linuxや、セットアッププロンプトが表示されない通常の起動フローでは影響がありませんでした。
- 現象が再発する場合は、Claude Codeのバージョンが2.1.206以降になっているか確認してください。
