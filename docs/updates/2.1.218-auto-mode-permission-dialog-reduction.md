## 原文（日本語に翻訳）

auto modeを改善しました。危険なrmコマンド、`&`によるバックグラウンド実行、不審なWindowsパスのチェックが許可ダイアログを開かなくなり、代わりにauto modeの分類器がこれらを判定するようになりました。

## 原文（英語）

Improved auto mode: the dangerous-rm, background-`&`, and suspicious-Windows-path checks no longer open permission dialogs; the auto-mode classifier adjudicates them instead

## 概要

auto modeは、ツール呼び出しのたびに人間が許可/拒否を判断する代わりに、ML分類器（別インスタンスのClaude）がリスクを評価して自動的に承認・ブロックを行う権限モードです。これまでは、危険な`rm`コマンド、`&`を使ったバックグラウンド実行、不審に見えるWindowsパスなど、特定のパターンに一致するBashコマンドは、auto modeが有効であっても静的なチェックによって強制的に許可ダイアログが開かれていました。v2.1.218では、これらのチェックがダイアログを開くのをやめ、代わりにauto modeの分類器がコンテキストを踏まえて判定するようになりました。これにより、実際には安全な操作（一時ファイルの削除、意図的なバックグラウンドジョブなど）での不要な中断が減り、本当に危険なコマンドは引き続きブロックまたは確認対象となります。

## 基本的な使い方

### auto modeの有効化

```bash
# CLIでauto modeを有効にして起動
claude --permission-mode auto

# または対話中に切り替え
claude
> /permission-mode auto
```

### 変更前後の挙動の違い

```bash
claude --permission-mode auto

> ビルドキャッシュディレクトリを削除して

# 変更前: rm を含むコマンドは静的チェックにより
#         常に許可ダイアログが開いていた
# 変更後: auto modeの分類器がコマンド内容と文脈
#         （対象パスがプロジェクト内の一時ディレクトリか等）
#         を評価し、安全と判断すれば自動承認される
```

## 実践例

### バックグラウンドで開発サーバーを起動する

```bash
claude --permission-mode auto

> 開発サーバーをバックグラウンドで起動して、ログを監視して

# npm run dev & のような & を使ったコマンドも、
# 分類器が「開発サーバーの起動」という文脈から
# 安全と判断すれば、確認ダイアログなしで実行される
```

### 一時ファイルのクリーンアップ

```bash
claude --permission-mode auto

> node_modules と dist ディレクトリを削除してクリーンビルドして

# rm -rf を含むコマンドでも、プロジェクト内の
# ビルド成果物・依存関係ディレクトリが対象であれば
# 分類器が安全と判断し自動承認されやすくなる
```

### 本当に危険なコマンドは引き続きブロックされる

```bash
claude --permission-mode auto

> ホームディレクトリ全体を削除して

# スコープを逸脱した破壊的な rm コマンドは、
# 分類器がリスクありと判断し、実行をブロックする
# （3回連続、または合計20回ブロックされると
#   auto modeは一時停止し、通常の確認プロンプトに戻る）
```

## 注意点

- この改善はClaude Code v2.1.218で実装されました
- 対象となるのは「危険なrmコマンド」「`&`によるバックグラウンド実行」「不審なWindowsパス」の3種類のチェックです。これらは以前は静的ルールにより無条件で確認ダイアログを開いていましたが、現在はauto modeの分類器の判断に委ねられます
- auto modeが無効な通常モード（デフォルトの確認プロンプトモード）では、この変更の影響を受けず、従来通り確認ダイアログが表示されます
- 分類器はスコープ逸脱・信頼できないインフラへのアクセス・プロンプトインジェクションの疑いを評価基準としており、判断に迷う場合は引き続き確認プロンプトを表示します
- 分類器が連続してブロックした場合はauto modeが自動的に一時停止するため、意図せず危険な操作が繰り返し実行され続けることはありません
