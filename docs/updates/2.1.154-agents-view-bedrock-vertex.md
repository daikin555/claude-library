## 原文（日本語訳）

エージェントビューを開く `←←`（左矢印キー2回）が、Bedrock、Vertex、Foundry、およびテレメトリ無効の環境でも動作するようになりました。

## 原文（英語）

`←←` to open the agents view now works on Bedrock, Vertex, Foundry, and with telemetry disabled

## 概要

`←←`（左矢印キーを素早く2回入力）でエージェントビューを開くキーボードショートカットが、Amazon Bedrock・Google Vertex AI・Azure AI Foundry といったエンタープライズ向けプロバイダー経由での利用時や、テレメトリを無効化した環境でも正常に動作するようになりました。これまではこれらの環境でショートカットが機能しない問題がありましたが、本バージョンでの修正により、プロバイダーや設定に依存せずに一貫した操作感が実現されます。

## 基本的な使い方

### `←←` でエージェントビューを開く

```
# Claude Code のチャット画面で
# 左矢印キーを素早く2回入力する

←←

# エージェントビューが開き、実行中・完了済みのエージェントセッション一覧が表示される
```

この操作は以下のすべての環境で動作します。

- Anthropic API（直接アクセス）
- Amazon Bedrock 経由
- Google Vertex AI 経由
- Azure AI Foundry 経由
- テレメトリ無効化設定時

## 実践例

### ### Amazon Bedrock 経由で利用している場合

```bash
# Bedrock を使うよう設定した環境で Claude Code を起動
export CLAUDE_CODE_USE_BEDROCK=1
claude

# チャット画面で ←← を入力するとエージェントビューが開く
# 実行中のエージェントを管理・監視できる
```

### ### Google Vertex AI 経由で利用している場合

```bash
# Vertex AI を使うよう設定した環境で Claude Code を起動
export CLAUDE_CODE_USE_VERTEX=1
claude

# ←← でエージェントビューを開いて並列セッションを確認
```

### ### テレメトリ無効化環境で利用する場合

セキュリティポリシーやプライバシー要件によりテレメトリを無効化している組織での利用例。

```bash
# テレメトリを無効化した状態で起動
claude --no-telemetry

# または設定ファイルでテレメトリを無効化
# .claude/settings.json に telemetry: false を設定

# ←← でエージェントビューへのアクセスが正常に動作する
```

### ### エージェントビューで複数セッションを管理する

```
# 複数のエージェントを並列実行中に ←← でビューを開く
←←

# エージェント一覧が表示される
# 例:
#   [実行中] テストスイートの修正
#   [完了]   ドキュメントの更新
#   [実行中] リファクタリング作業

# 確認したいセッションを選択してアタッチ
```

## 注意点

- `←←` は左矢印キーを素早く2回入力するショートカットです。テキスト入力中に誤作動しないよう、入力の速度が判定に使われます。
- このショートカットはチャット画面の入力フィールドが空の状態か、特定の状態でのみ有効になる場合があります。
- Bedrock・Vertex・Foundry で利用する場合は、各プロバイダーの認証設定が正しく行われている必要があります。
- テレメトリを無効化している場合でも、エージェントビューの機能自体は完全に利用できます。テレメトリの有無はデータ収集に関わるもので、機能の動作には影響しません。
- 本修正以前のバージョンでは、上記の環境でショートカットが動作しないことがあったため、v2.1.154 へのアップデートを推奨します。
