## 原文（日本語に翻訳）

`CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` を設定している状態で、セッションからエージェントビューに戻る際に、重なり合った「ゴーストフレーム」が残ってしまう問題を修正しました。

## 原文（英語）

Fixed returning to the agents view from a session leaving overlapping ghost frames with `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1`

## 概要

`CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1`（またはクラシックレンダラー相当の `/tui` 設定）を有効にすると、Claude Codeは代替スクリーンバッファを使わず、ターミナルのネイティブなスクロールバックに描画内容を残す方式で動作します。この方式では、あるセッションからエージェント一覧（agents view）へ戻る操作を行った際に、直前のセッション画面の描画がクリアされずに残り、新しいエージェントビューの描画と重なって表示が乱れる「ゴーストフレーム」が発生する不具合がありました。v2.1.210では、ビュー遷移時に画面を正しく再描画し、古いフレームが残らないよう修正されています。

## 基本的な使い方

### 修正前の問題

```bash
# クラシックレンダラー（代替スクリーン無効）で起動
export CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1
claude

# エージェントビューから特定のセッションを開く
> (agents viewでセッションAを選択して開く)

# セッションA内で作業した後、エージェントビューに戻る
> Esc または戻る操作でagents viewに復帰

# 修正前:
# 直前のセッションAの画面内容が消えずに残り、
# 新しいagents viewの描画と重なって
# テキストが二重に表示される・レイアウトが崩れる
```

### 修正後の動作

```bash
# 同じ操作を行っても、
# ビュー遷移時に画面が正しくクリア・再描画されるため、
# ゴーストフレームは発生しない

export CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1
claude
> (セッションを開いて作業 → agents viewに戻る)
# → クリーンな画面でagents viewが表示される
```

## 実践例

### 複数セッション間を行き来する運用

```bash
# クラシックレンダラーを使い、ターミナルのCmd+F検索や
# tmuxのコピーモードを活用しながら複数セッションを行き来する運用

export CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1
claude

# セッションA → agents view → セッションB → agents view ...
# と繰り返し遷移しても、各遷移で画面が正しく再描画され、
# 過去のセッション内容が視覚的に混在しない
```

### tmux環境での利用

```bash
# tmux内でクラシックレンダラーを使う典型的なユースケース
tmux new-session -s claude-work
export CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1
claude

# tmuxのスクロールバックやコピーモードと相性の良い
# クラシックレンダラーでも、agents view復帰時の
# 描画崩れが解消され、快適に使えるようになった
```

### `/tui` コマンドでの切り替えとの関係

```bash
# 起動後に /tui コマンドでクラシックレンダラーに切り替えた場合も
# 同様の効果が得られる（環境変数と /tui 設定は等価）
claude
> /tui
# → クラシックレンダラーに切り替え
# → agents viewとの往復でゴーストフレームが発生しない
```

## 注意点

- この修正はClaude Code v2.1.210で実装されました
- この問題は `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` を設定している場合、または `/tui` コマンドでクラシックレンダラーに切り替えている場合にのみ発生していました（デフォルトのフルスクリーンレンダリングでは代替スクリーンバッファが使われるため影響を受けません）
- バックグラウンドセッションを `agents view` から開いたり `claude attach` した場合は、常にフルスクリーンレンダリングが使われるため、この環境変数や `/tui` 設定の影響を受けません
- クラシックレンダラーはターミナルのネイティブなスクロールバックを利用するため、Cmd+F（macOSでの検索）やtmuxのコピーモードとの相性を重視するユーザー向けの設定です
- 表示が乱れた場合の一時的な回避策としては、ターミナルの再描画（多くのターミナルでは `Ctrl+L` や画面リサイズ）で改善することがありました
