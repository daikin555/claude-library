## 原文（日本語に翻訳）

ツール呼び出し中に CLAUDE.md ファイルが自動ロードされる際、フルスクリーンのスクロールバックで折り畳まれた検索/読み取りサマリーバッジが複数回表示される問題を修正

## 原文（英語）

Fixed collapsed search/read summary badge appearing multiple times in fullscreen scrollback when a CLAUDE.md file auto-loads during a tool call

## 概要

フルスクリーンモードでスクロールバックを表示している際、ツール呼び出し中に CLAUDE.md ファイルが自動ロードされると、折り畳まれた検索・読み取り操作のサマリーバッジが複数回重複して表示されるUIバグが修正されました。この問題は表示上の乱れを引き起こし、会話の流れを追いにくくしていました。v2.1.90 以降では、バッジは正しく一度だけ表示されます。

## 基本的な使い方

修正内容はUIバグフィックスのため、特別な操作は不要です。v2.1.90 にアップデートするだけで問題が解消されます。

CLAUDE.md の自動ロードは、プロジェクト設定ファイルがある場合に自動的に行われます：

```
project/
├── CLAUDE.md          # プロジェクト設定（自動ロード）
├── src/
│   └── CLAUDE.md      # サブディレクトリ設定（必要に応じて自動ロード）
└── ...
```

## 実践例

### CLAUDE.mdを使ったプロジェクト設定

CLAUDE.md の自動ロードが正常に機能するプロジェクト構成：

```markdown
# CLAUDE.md
## プロジェクト概要
このプロジェクトはReactアプリケーションです。

## コーディング規約
- TypeScriptを使用
- ESLintとPrettierでフォーマット

## よく使うコマンド
- `npm run dev`: 開発サーバー起動
- `npm run build`: ビルド
- `npm test`: テスト実行
```

v2.1.90 以降、このファイルがツール呼び出し中に自動ロードされても、UIバッジの重複は発生しません。

### フルスクリーンモードでの作業

フルスクリーンモードで長い会話履歴を確認する場合でも、バッジ表示が正常になりました：

```bash
# フルスクリーンモードでClaude Codeを起動
claude
# Ctrl+Lなどでフルスクリーン表示切り替え
```

## 注意点

- このバグは CLAUDE.md の自動ロードが発生するプロジェクトでのみ影響していました
- フルスクリーンのスクロールバックを多用するユーザーに特に影響があったバグです
- v2.1.90 へのアップデートで解消されます
