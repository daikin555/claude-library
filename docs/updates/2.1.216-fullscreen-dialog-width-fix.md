## 原文（日本語訳）

フルスクリーンモードにおいて、ダイアログがパネルの右端を越えて伸びてしまう不具合を修正しました。

## 原文（英語）

Fixed dialogs in fullscreen mode stretching past the right-hand edge of their panel

## 概要

[フルスクリーンレンダリングモード](https://code.claude.com/docs/en/fullscreen)（`/tui fullscreen` で有効化）では、選択メニューや確認ダイアログなどが専用のパネル内に描画されます。今回の修正前は、一部のダイアログの幅計算が正しく行われず、パネルの右端（境界線）を越えてテキストや枠が伸びてしまうことがありました。v2.1.216ではこのレイアウト崩れが修正され、ダイアログが常にパネル幅に収まって描画されるようになりました。

## 基本的な使い方

見た目の修正のため、特別な操作は不要です。フルスクリーンモードを利用している場合、Claude Codeをv2.1.216以降にアップデートするだけで自動的に改善されます。

```
/tui fullscreen
```

## 実践例

### 修正の確認方法

フルスクリーンモードに切り替えた状態で、権限プロンプトや `/model`、`/config` などの選択ダイアログを開き、パネルの右端でテキストや枠線がはみ出していないかを確認します。

```
/tui fullscreen
/config
```

### 端末幅を変えて表示崩れがないか確認

ターミナルのウィンドウ幅を変更（リサイズ）した状態でもダイアログがパネル内に収まるかを確認すると、この修正の効果を体感しやすいです。

```
# ターミナル幅を狭めたり広げたりしてから
/model
```

### 通常レンダリングモードへの切り替え

もし引き続き表示崩れが見られる場合は、一時的にクラシックレンダラーに戻すことも可能です。

```
/tui default
```

## 注意点

- フルスクリーンレンダリングは現時点でリサーチプレビュー（研究的な先行提供）機能であり、ターミナルエミュレータによっては別の表示上の問題が残っている可能性があります
- 表示崩れを見つけた場合は `/feedback` で報告するか、[claude-code GitHubリポジトリ](https://github.com/anthropics/claude-code/issues)にIssueを立てることが推奨されています
- 本修正はレイアウト計算のみに関わるものであり、ダイアログの操作方法や機能自体に変更はありません
