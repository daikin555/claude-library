## 原文（日本語に翻訳）

`isolation: 'worktree'` を指定したサブエージェントが、自分専用の分離済みworktreeではなくメインのリポジトリチェックアウトに対してgitを変更するコマンドを実行できてしまう不具合を修正しました

## 原文（英語）

Fixed `isolation: 'worktree'` subagents being able to run git-mutating commands against the main repo checkout instead of their own isolated worktree

## 概要

`isolation: 'worktree'` は、サブエージェントごとに独立したgit worktreeを作成し、ファイル編集やコミットをメインのリポジトリチェックアウトから隔離するための機能です。しかしこれまでは、一部のgit変更系コマンド（コミット・ブランチ操作・リセットなど）がサブエージェントの作業ディレクトリ設定を無視して、意図せずメインチェックアウトに対して実行されてしまうケースがありました。Claude Code v2.1.210ではこの不具合が修正され、`isolation: 'worktree'` を使うサブエージェントのgitコマンドは確実に自分専用のworktree内でのみ実行されるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
claude

> isolation: worktree でサブエージェントを起動して、
> auth.ts をリファクタリングして

# サブエージェントは専用worktreeで作業しているはずだが、
# 一部のgitコマンド（例: git commit, git checkout）が
# メインリポジトリのチェックアウトに対して実行されてしまうことがあった
# → メインの作業ツリーに意図しない変更やコミットが混入するリスク
```

### 修正後の動作

```bash
claude

> isolation: worktree でサブエージェントを起動して、
> auth.ts をリファクタリングして

# サブエージェントが実行するすべてのgit変更コマンドは
# 専用worktree（例: .git/claude-worktrees/agent-xxx）内でのみ実行される
# メインチェックアウトのHEADやインデックスは一切変更されない
```

## 実践例

### 複数サブエージェントを並行実行する場合

```bash
claude

> agent A に src/auth.ts を、agent B に src/payments.ts を
> それぞれ isolation: worktree で並行してリファクタリングさせて

# 修正後:
# - agent A のコミットは worktree-A にのみ反映
# - agent B のコミットは worktree-B にのみ反映
# - メインチェックアウトはどちらの変更の影響も受けない
# → レビュー後にどちらのブランチを採用するか選べる
```

### カスタムサブエージェントのfrontmatterで指定する場合

```markdown
---
name: refactor-agent
isolation: worktree
---

あなたはリファクタリング専門のサブエージェントです。
```

```bash
claude

> refactor-agent を使ってレガシーコードを整理して

# frontmatterで isolation: worktree を指定した
# カスタムサブエージェントでも、修正後は
# git変更コマンドが専用worktreeの外に漏れることはない
```

### 誤ってメインチェックアウトを壊すリスクの排除

```bash
# 修正前に起こり得たシナリオ（現在は発生しない）
claude

> isolation: worktree のサブエージェントに
> git reset --hard を含む大掛かりな変更をさせて

# 修正前: 最悪の場合メインの作業ツリーの未コミット変更が
#         意図せず失われるリスクがあった
# 修正後: reset・checkout・commitなどの変更コマンドは
#         常にサブエージェント専用worktree内にスコープされる
```

## 注意点

- この修正はClaude Code v2.1.210（2026年7月15日リリース）で行われました
- `isolation: 'worktree'` を指定したサブエージェントは、プランモード・メモリ・フック・権限設定・会話トランスクリプトもすべてそのworktreeに紐づきます
- 未コミットの変更や新規ファイル、新規コミットがない場合、そのworktreeとブランチはサブエージェント終了時に自動的に削除されます
- 変更が残っている場合は、worktreeを保持するか削除するかを確認するプロンプトが表示されます
- この機能を使うには「エージェント向けにworktreeを使って」のように依頼するか、カスタムサブエージェントのfrontmatterに `isolation: worktree` を追加します
- 本修正はセキュリティ上重要な修正であり、メインリポジトリを誤って変更されたくない場合は積極的に `isolation: worktree` の利用を検討してください
