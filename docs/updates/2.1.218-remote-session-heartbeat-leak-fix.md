## 原文（日本語に翻訳）

リモートセッションが、自身のワーカーが入れ替わった後もハートビートを送信し続けてしまい、長時間起動したままのデスクトップアプリやIDEのプロセスが、拒否されるリクエストを数秒おきに永久にリトライし続ける問題を修正しました。

## 原文（英語）

Fixed remote sessions continuing to send heartbeats after their worker was replaced, which left long-lived desktop and IDE processes retrying a rejected request every few seconds forever

## 概要

リモートセッション（Remote Controlやデスクトップアプリ、IDE拡張から接続されるセッション）は、裏側で動作する「ワーカー」プロセスと結び付いており、再接続やフェイルオーバーなどでワーカーが入れ替わることがあります。この修正前は、ワーカーが入れ替わった後も古いセッションが引き続きハートビート（生存確認のping）を送り続けてしまい、サーバー側からは既に無効なリクエストとして拒否され続けるにもかかわらず、デスクトップアプリやIDE拡張のプロセスが数秒おきに同じリクエストを永久にリトライしてしまう状態になっていました。v2.1.218では、ワーカーが入れ替わった後は古いセッションがハートビートの送信を停止するように修正され、この無限リトライによる無駄な通信やリソース消費が解消されました。

## 注意点

- この修正は内部的な安定性向上であり、ユーザーが設定を変更する必要はありません
- 長時間起動したままのデスクトップアプリやIDE拡張（VSCode、JetBrains系など）で、ネットワークログや通信量に不自然な定期リトライが見られていた場合、本修正によって解消される可能性があります
- 問題が再発するように見える場合は、アプリやIDE拡張、Claude Code CLI自体を最新バージョンに更新した上で再現するか確認してください
