## 原文（日本語に翻訳）

`/config` メニューで検索カーソルとリスト選択が同時に表示される問題を修正しました。

## 原文（英語）

Fixed `/config` menu showing both the search cursor and list selection at the same time

## 概要

`/config` コマンドで設定メニューを開いた際、検索入力フィールドのカーソルと、リスト内で選択中のアイテムのハイライトが同時に表示されてしまい、視覚的に混乱する問題が修正されました。修正後は検索モードとリスト選択モードが適切に排他的に表示されます。

## 基本的な使い方

`/config` コマンドを使って設定を変更する際、UI が正しく表示されるようになりました。

```bash
$ claude
> /config
# 設定メニューが開く
# 検索カーソルとリスト選択が同時に表示されなくなった
# すっきりとした UI で設定を操作できる
```

## 実践例

### ユースケース: 設定の検索と選択

設定メニューで特定の設定を検索する際、カーソルとハイライトが混在しない明確な UI になります。

```bash
$ claude
> /config
# 設定メニュー:
# [ 検索: _____ ]  ← 検索モード時はここにカーソル
# または
# > theme         ← 選択モード時はリストハイライトのみ
#   model
#   voice
# 修正後: 両方が同時に表示されなくなった
```

### ユースケース: テーマやモデルの変更

設定変更の操作が視覚的にわかりやすくなります。

```bash
$ claude
> /config
# ↓ キーで設定項目を選択
# 検索入力とリスト選択が明確に区別されて表示される
# Enter で選択した設定を開く
```

## 注意点

- この問題は純粋に表示上のバグであり、設定の保存や適用には影響していませんでした
- `/config` メニューの機能自体は変更なく、見た目の修正のみです
