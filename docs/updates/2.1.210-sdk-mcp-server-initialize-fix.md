## 原文（日本語に翻訳）

`initialize` コントロールリクエスト経由で登録されたSDK MCPサーバーが、次のターンまで接続を開始しない状態で待機してしまう問題を修正しました。

## 原文（英語）

Fixed SDK MCP servers registered via an `initialize` control request waiting until the next turn to start connecting

## 概要

Claude Agent SDKでは、セッション開始後にコントロールプロトコルの `initialize` リクエストを通じてMCPサーバーを動的に登録できます。以前のバージョンでは、この方法で登録されたMCPサーバーへの接続処理が、登録直後ではなく次の会話ターンが始まるまで開始されない不具合がありました。そのため、登録直後にそのMCPサーバーのツールを使おうとすると、接続がまだ確立されておらずツールが利用できない、あるいは不要な遅延が発生していました。v2.1.210では、`initialize` リクエストを受け取った時点で即座に接続処理を開始するように修正されています。

## 基本的な使い方

### 修正前の問題

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

// セッション開始後にMCPサーバーを動的に登録
const session = query({
  prompt: "作業を開始してください",
  options: {
    mcpServers: {}, // 初期状態では空
  },
});

// 途中でコントロールリクエストを送りMCPサーバーを追加登録
await session.setMcpServers({
  myTools: {
    type: "stdio",
    command: "node",
    args: ["./my-mcp-server.js"],
  },
});

// 修正前: 登録直後にツールを呼び出そうとしても
// 接続処理がまだ開始されておらず、
// 次のターンが始まるまでツールが使えない状態だった
```

### 修正後の動作

```typescript
// 修正後: setMcpServers（内部的にはinitializeコントロールリクエスト）
// を送信した時点で、即座にサーバーへの接続処理が開始される
await session.setMcpServers({
  myTools: {
    type: "stdio",
    command: "node",
    args: ["./my-mcp-server.js"],
  },
});

// 同一ターン内でも接続が進行しているため、
// ツールが利用可能になるまでの待ち時間が短縮された
```

## 実践例

### セッション途中でのMCPサーバーのホットスワップ

長時間稼働するSDKセッションで、タスクの内容に応じて必要なMCPサーバーを動的に切り替えるケース。

```typescript
// タスクAの間はデータベースツールのみ登録
await session.setMcpServers({
  db: { type: "stdio", command: "npx", args: ["-y", "db-mcp-server"] },
});

// タスクBに移行するタイミングで別のMCPサーバーを追加登録
await session.setMcpServers({
  db: { type: "stdio", command: "npx", args: ["-y", "db-mcp-server"] },
  deploy: { type: "stdio", command: "npx", args: ["-y", "deploy-mcp-server"] },
});

// 修正後は登録直後から接続処理が始まるため、
// 次のプロンプト送信時にはツールが使える可能性が高くなった
```

### CI/CDパイプラインでの自動化スクリプト

```typescript
// ヘッドレスで動くAgent SDKスクリプトが、
// 実行フェーズごとに必要なMCPサーバーを切り替える例
for (const phase of ["build", "test", "deploy"]) {
  await session.setMcpServers(getMcpConfigForPhase(phase));
  await session.sendMessage(`${phase}フェーズを実行してください`);
}

// 各フェーズ切り替え直後の接続待ちによるタイムロスが軽減される
```

### レイテンシに敏感なマルチエージェント構成

```typescript
// オーケストレーターが複数のサブセッションに
// 異なるMCPサーバーをその場で割り当てる構成でも、
// 接続開始のタイミングが早まったことで
// 全体のスループットが向上する
```

## 注意点

- この修正はClaude Code / Claude Agent SDK v2.1.210で実装されました
- 影響を受けるのは、セッション開始時の初期設定ではなく、セッション途中で `initialize` コントロールリクエスト（SDKの `setMcpServers` などのAPI経由）によって登録されたMCPサーバーです
- 接続開始タイミングが早まるだけで、MCPサーバー自体の起動時間やハンドシェイクにかかる時間そのものが短縮されるわけではありません
- stdioトランスポートのサーバーはプロセス起動、HTTPトランスポートのサーバーはネットワーク接続が必要なため、登録直後でも即座にツールが使えるとは限りません
- MCP SDK自体の接続タイムアウトはデフォルトで60秒程度であるため、極端に起動が遅いサーバーでは依然としてタイムアウトが発生し得ます
