## 原文（日本語に翻訳）

`/code-review --fix` がレビュー後にワーキングツリーへレビュー結果を適用するようになりました。再利用性・シンプルさ・効率性の提案が反映されます。`/simplify` は `/code-review --fix` を呼び出すようになりました。

## 原文（英語）

`/code-review --fix` now applies review findings to your working tree after the review, surfacing reuse, simplification, and efficiency suggestions; `/simplify` now invokes `/code-review --fix`

## 概要

`/code-review --fix` フラグを使うことで、コードレビューを実行した後、その結果（再利用性・シンプルさ・効率性の提案）をワーキングツリーに自動的に適用できるようになりました。また、既存の `/simplify` コマンドは内部的に `/code-review --fix` を呼び出すよう変更され、コードの簡略化ワークフローが一本化されました。

## 基本的な使い方

```bash
# コードレビューを実行し、結果を自動的にワーキングツリーに適用
/code-review --fix

# /simplify コマンドでも同様の処理が実行される
/simplify
```

## 実践例

### バグ修正後のコード整理

機能実装やバグ修正後に、コードの品質を素早く向上させる用途に使えます。

```bash
# 機能実装後にレビューと修正を一括実行
/code-review --fix

# → レビュー結果がそのまま差分として反映される
# git diff でどのような変更が加えられたか確認可能
git diff
```

### PR提出前の最終確認と自動修正

プルリクエスト提出前に `/code-review --fix` を実行することで、手動では気づきにくい冗長なコードや非効率なパターンを一括修正できます。

```bash
# PR提出前の最終整理
/code-review --fix

# 変更内容を確認してコミット
git diff
git add -p
git commit -m "refactor: apply code review suggestions"
```

### /simplify による即時コード整理

`/simplify` を使うと、`/code-review --fix` と同等の効果が得られます。複雑になってきたコードベースの整理に便利です。

```bash
# 現在のファイルまたはセレクションを簡略化
/simplify

# 変更が自動的に適用される
```

## 注意点

- `--fix` フラグを付けない `/code-review` は従来通りレビュー結果を表示するのみで、ファイルへの変更は行いません
- 自動適用された変更は `git diff` で確認し、意図しない変更がないかレビューすることを推奨します
- `/simplify` が `/code-review --fix` を呼び出すようになったため、以前と挙動が変わる場合があります
- 大規模なコードベースでは適用範囲が広くなる可能性があるため、適用後は必ずテストを実行してください
