## 原文（日本語に翻訳）

ネストされた `.claude/` ディレクトリ: エージェント、ワークフロー、出力スタイルは、名前が衝突した場合に作業ディレクトリに最も近いものが優先されるようになりました。プロジェクトスコープのワークフロー保存先は、最も近い既存の `.claude/workflows/` ディレクトリが対象になります。

## 原文（英語）

Nested `.claude/` directories: the agent, workflow, and output-style closest to the working directory now wins when names collide; project-scope workflow saves now target the closest existing `.claude/workflows/`

## 概要

モノレポやネストされたプロジェクト構造において、エージェント・ワークフロー・出力スタイルの設定が「最近傍優先」ルールで解決されるようになりました。作業ディレクトリに最も近い `.claude/` の設定が優先されるため、サブプロジェクトごとに独自の設定を持てます。ワークフロー保存時も、最も近い既存の `.claude/workflows/` ディレクトリが対象になります。

## 基本的な使い方

ディレクトリ構造の例:

```
root/
├── .claude/
│   ├── agents/
│   │   └── reviewer.md     # 全体共通のレビューエージェント
│   └── workflows/
│       └── deploy.js       # 全体共通のデプロイワークフロー
└── packages/
    └── api/
        └── .claude/
            ├── agents/
            │   └── reviewer.md  # API専用のレビューエージェント（優先）
            └── workflows/
                └── deploy.js    # API専用のデプロイワークフロー（優先）
```

`packages/api/` 配下で作業中は、`packages/api/.claude/` の設定が優先されます。

## 実践例

### サブパッケージ専用エージェントの設定

```
monorepo/
├── .claude/agents/
│   └── code-reviewer.md    # 全体: 汎用レビューエージェント
└── services/
    └── auth/
        └── .claude/agents/
            └── code-reviewer.md  # auth専用: セキュリティ重視のレビュー
```

`services/auth/` 内でエージェントを呼び出すと、セキュリティ重視のレビュアーが起動します。

### ワークフロー保存先の自動選択

新しいワークフローを保存する際、Claude Codeは作業ディレクトリから上位に向かって最初に見つかった `.claude/workflows/` ディレクトリを使用します:

```bash
# packages/api/ で作業中にワークフローを保存
# → packages/api/.claude/workflows/ に保存される
# (root/.claude/workflows/ ではなく)
```

### 出力スタイルのプロジェクト別カスタマイズ

```
company-app/
├── .claude/
│   └── output-style.md     # デフォルト: 英語で詳細に
└── ja-docs/
    └── .claude/
        └── output-style.md  # 日本語ドキュメント用: 日本語で簡潔に
```

`ja-docs/` 配下では日本語スタイルが自動適用されます。

## 注意点

- 優先度は「作業ディレクトリに最も近い `.claude/`」が最高
- 同名の設定ファイルが複数階層にある場合、下位（より近い）のものが上位（遠い）のものを完全に上書きします
- ワークフローの保存先は「最も近い既存の `.claude/workflows/`」です。存在しない場合は通常のプロジェクトルートが使われます
- この変更により、以前は意図しない設定が適用されていたケースが修正されます
