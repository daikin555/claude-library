## 原文（日本語に翻訳）

- `/plugin` マーケットプレイス閲覧ペインに予測コンテキストコスト（ターンごとおよび呼び出しごとのトークン見積もり）を追加

## 原文（英語）

- Added projected context cost (per-turn and per-invocation token estimates) to the `/plugin` marketplace browse pane

## 概要

`/plugin` コマンドのマーケットプレイス閲覧画面に、各プラグインがコンテキストに与えるトークンコストの見積もりが表示されるようになりました。ターンごとのコストと呼び出しごとのコストの両方が確認でき、プラグインを有効化する前にコンテキスト使用量の影響を把握できます。

## 基本的な使い方

```
# Claude Code 内でプラグインマーケットプレイスを開く
/plugin
```

マーケットプレイスの各プラグイン一覧に以下の情報が表示されます：

- **ターンごとのトークン見積もり**（per-turn token estimate）：各ターンのシステムプロンプトに追加されるトークン数
- **呼び出しごとのトークン見積もり**（per-invocation token estimate）：プラグインツールが実行されるたびに消費するトークン数

## 実践例

### プラグイン選択時のコスト比較

```
/plugin browse
# 一覧例：
# ┌─────────────────────────────┬──────────────┬──────────────────┐
# │ Plugin                      │ Per-turn     │ Per-invocation   │
# ├─────────────────────────────┼──────────────┼──────────────────┤
# │ code-formatter@team-tools   │ ~200 tokens  │ ~50 tokens       │
# │ deployment-tools@team-tools │ ~800 tokens  │ ~300 tokens      │
# │ test-runner@team-tools      │ ~150 tokens  │ ~100 tokens      │
# └─────────────────────────────┴──────────────┴──────────────────┘
```

### 大規模プロジェクトでのコスト最適化

コンテキストウィンドウが限られている場合、重いプラグインを一時無効化することでより多くのコードやドキュメントをコンテキストに収めることができます：

```bash
# コストの高いプラグインを確認してから判断
/plugin browse
# → トークン見積もりを参照して、不要なプラグインを特定

# 不要なプラグインを無効化
claude plugin disable deployment-tools
```

## 注意点

- 表示されるトークン数はあくまで「見積もり」であり、実際の使用量は入力内容やプラグインの動作によって異なります
- ターンごとのコストはシステムプロンプトへの追加分のため、会話が長くなるほど影響が大きくなります
- 複数のプラグインを有効化している場合、コストは累積されます
- コンテキストコストを抑えるため、必要なプラグインだけを有効化する運用が推奨されます
