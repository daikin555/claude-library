## 原文（日本語に翻訳）

`/plugin` でプラグインが不正確にインストール済みと表示される問題を修正しました。

## 原文（英語）

Fixed plugins showing as inaccurately installed in `/plugin`

## 概要

`/plugin` コマンドでプラグインの一覧を表示した際、実際にはインストールされていないプラグインが「インストール済み」として誤って表示される問題が修正されました。プラグインのインストール状態の検出ロジックが改善され、実際のインストール状態が正確に反映されるようになりました。

## 基本的な使い方

```bash
# プラグイン一覧の表示
/plugin

# 修正前: 実際にはインストールされていないプラグインが
#         「インストール済み」として表示されることがあった
# 修正後: 実際のインストール状態が正確に表示される
```

## 実践例

### プラグイン一覧の確認

```bash
/plugin

# 修正後の表示例:
# ✓ plugin-name-a    インストール済み
# ✗ plugin-name-b    未インストール
# ✗ plugin-name-c    未インストール

# 修正前: plugin-name-b や plugin-name-c が
#         誤って ✓ マーク付きで表示されることがあった
```

### プラグインのインストール状態の確認

```bash
# 特定のプラグインを検索
/plugin search database

# 表示されたプラグインのインストール状態が正確になった
# インストールと表示されているものは実際にインストール済み
# 未インストールと表示されているものは実際に未インストール
```

### プラグインの管理操作

```bash
# プラグインのインストール
/plugin install my-plugin

# インストール後に一覧を確認
/plugin
# → インストールしたプラグインが正確に「インストール済み」と表示される

# プラグインのアンインストール
/plugin uninstall my-plugin

# アンインストール後に一覧を確認
/plugin
# → アンインストールしたプラグインが正確に「未インストール」と表示される
```

## 注意点

- この修正はプラグインのインストール状態の「表示」の問題を修正したもので、プラグインの実際の動作への影響はありません
- プラグインが誤ってインストール済みと表示されていた場合、実際には機能していなかった可能性があります
- 修正後に `/plugin` でインストール済みと表示されているプラグインは確実にインストールされています
- プラグインのインストール状態に不整合が生じた場合は、再インストールすることで解消できます
- マーケットプレイスからのインストール後の表示更新については別の修正（2.1.70-plugin-marketplace-refresh-fix）も参照してください
