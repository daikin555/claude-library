## 原文（日本語に翻訳）

再開したセッションの履歴に破損したdelta attachmentが含まれていた場合に、毎ターン失敗する、あるいは再開時にクラッシュしてしまう問題を修正しました

## 原文（英語）

Fixed a resumed session failing every turn, or crashing on resume, when its history held a malformed delta attachment

## 概要

`--resume`/`--continue`フラグや`/resume`コマンドで過去のセッションを再開できますが、以前は保存されている会話履歴の中に破損した「delta attachment」（差分の増分更新を記録するデータ）が含まれていると、再開直後にクラッシュしたり、再開はできても以降のすべてのターンでエラーが発生し続けたりすることがありました。この修正により、そうした破損した履歴エントリが含まれていても、それを安全に扱って正常にセッションを再開・継続できるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 影響を受けるのは、履歴データに破損したdelta attachmentが含まれている一部のセッションのみ
- ユーザー側での修復作業は不要で、アップデート後は該当セッションを再開しても通常どおり動作するようになる
- `--resume`/`--continue`や`/resume`によるセッション再開全般の安定性向上に関わる修正
