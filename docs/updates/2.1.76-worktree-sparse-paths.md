# worktree.sparsePaths：大規模モノレポでの部分チェックアウト設定

## 原文（日本語に翻訳）

大規模モノレポで `claude --worktree` 使用時に、git sparse-checkout を通じて必要なディレクトリのみをチェックアウトするための `worktree.sparsePaths` 設定を追加

## 原文（英語）

Added `worktree.sparsePaths` setting for `claude --worktree` in large monorepos to check out only the directories you need via git sparse-checkout

## 概要

Claude Code v2.1.76で、`worktree.sparsePaths` 設定が追加されました。大規模なモノレポで `claude --worktree` を使用する際に、リポジトリ全体ではなく必要なディレクトリのみをチェックアウトできます。git sparse-checkout を内部的に使用することで、worktreeの作成時間とディスク使用量を大幅に削減し、巨大なリポジトリでもworktreeベースの並列開発を快適に行えるようになります。

## 基本的な使い方

プロジェクトの `.claude/settings.json` に `worktree.sparsePaths` を設定します：

```json
{
  "worktree": {
    "sparsePaths": [
      "packages/frontend",
      "packages/shared",
      "config"
    ]
  }
}
```

この設定を行った状態で `claude --worktree` を実行すると、指定したディレクトリのみがチェックアウトされたworktreeが作成されます：

```bash
claude --worktree my-feature
# packages/frontend, packages/shared, config のみがチェックアウトされる
```

## 実践例

### フロントエンド開発に必要な部分のみチェックアウト

大規模モノレポでフロントエンド関連のパッケージだけを扱う場合：

```json
{
  "worktree": {
    "sparsePaths": [
      "packages/web-app",
      "packages/ui-components",
      "packages/shared-types",
      "packages/config",
      "package.json",
      "tsconfig.json"
    ]
  }
}
```

```bash
# フロントエンドの作業に必要なファイルのみでworktreeを作成
claude --worktree frontend-feature
```

### マイクロサービスの特定サービスのみ

マイクロサービスアーキテクチャのモノレポで、特定のサービスに集中する場合：

```json
{
  "worktree": {
    "sparsePaths": [
      "services/auth-service",
      "libs/common",
      "libs/database",
      "proto/auth.proto",
      "docker-compose.yml"
    ]
  }
}
```

```bash
# 認証サービスに関連するファイルのみでworktreeを作成
claude --worktree auth-refactor
```

### チーム・ロールごとの設定

ユーザーの `~/.claude/settings.json` にチームの担当領域に合わせて設定：

```json
{
  "worktree": {
    "sparsePaths": [
      "services/payment",
      "services/billing",
      "libs/stripe-client",
      "libs/common",
      "tests/integration/payment"
    ]
  }
}
```

## 注意点

- **gitの sparse-checkout を内部で使用**: この設定は内部的に `git sparse-checkout` を利用しているため、git 2.25.0以降が必要
- **パスの指定はリポジトリルートからの相対パス**: `sparsePaths` にはリポジトリルートからの相対パスを指定する
- **依存関係に注意**: ビルドやテストに必要なファイル（`package.json`、設定ファイルなど）も忘れずに含める必要がある
- **設定レベル**: プロジェクトの `.claude/settings.json` に設定するとチーム全体で共有でき、`~/.claude/settings.json` に設定すると個人の担当範囲に合わせてカスタマイズできる
- **worktreeフラグ必須**: この設定は `claude --worktree` で起動した場合にのみ適用される。通常起動時には影響しない
