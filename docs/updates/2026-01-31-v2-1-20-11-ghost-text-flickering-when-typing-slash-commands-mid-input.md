## 原文（日本語に翻訳）

入力途中でスラッシュコマンドを入力した際のゴーストテキストの点滅を修正

## 原文（英語）

Fixed ghost text flickering when typing slash commands mid-input

## 概要

Claude Code v2.1.20では、テキスト入力の途中でスラッシュコマンド（`/help`、`/config`など）を入力した際に、オートコンプリートのゴーストテキストが点滅する視覚的な問題が修正されました。以前は、コマンド候補が表示される際にテキストが激しく点滅し、使いにくく目に優しくない表示でした。この修正により、スムーズで安定した入力体験が提供されます。

## 基本的な使い方

修正は自動的に適用され、スラッシュコマンドの入力がより快適になります：

```bash
# プロンプトの途中でコマンドを入力
> このファイルを確認して /he

# "he" と入力すると "/help" の候補が表示される
# 修正前：ゴーストテキストが激しく点滅
# 修正後：スムーズに表示される
```

## 実践例

### コマンドのオートコンプリート使用時

スラッシュコマンドを入力する際：

```bash
# "/" を入力
> /

# "c" を追加
> /c

# オートコンプリート候補が表示：
# /config
# /context
# /commit
# /copy

# 修正前の問題：
# - 候補が表示される度にゴーストテキストが点滅
# - 視覚的にストレスフル
# - 目が疲れる

# 修正後：
# - スムーズに候補が表示される
# - 点滅なしで読みやすい
```

### 長いプロンプト中にコマンドを挿入

文章の途中でコマンドを使用する場合：

```bash
# プロンプトを入力中
> このプロジェクトの設定を確認したいので /con

# "con" まで入力すると "/config" や "/context" が候補に
# Tab で補完、または続けて入力

# 修正により：
# - 候補表示が安定
# - 入力フローが中断されない
# - より直感的な操作感
```

### 複数のコマンドを連続入力

素早くコマンドを切り替える際：

```bash
# 1つ目のコマンド
> /he
# Tab で /help に補完

# Ctrl+U でクリア、次のコマンド
> /co
# Tab で /config に補完

# さらに次
> /com
# Tab で /commit に補完

# 修正前：
# - 各入力で点滅が発生
# - 素早い入力時に特に目立つ

# 修正後：
# - 点滅なしでスムーズ
# - 高速入力でも快適
```

### スキルコマンドの検索

カスタムスキルを探す際：

```bash
# スキルコマンドを入力
> /rev

# 候補：
# /review-pr
# /review-code

# さらに入力
> /review-p

# 候補が絞り込まれる：
# /review-pr

# 修正により：
# - リアルタイムフィルタリングが滑らか
# - 視覚的なノイズなし
```

### ヘルプコマンドの頻繁な使用

初心者がコマンドを学ぶ際：

```bash
# 何度も /help を入力して確認
> /help
> /help コマンド名
> /help 別のコマンド

# 修正前：
# - 毎回の入力で点滅
# - 学習体験が低下

# 修正後：
# - ストレスフリーな学習
# - コマンド入力がより快適
```

## 注意点

- この修正はスラッシュコマンドのオートコンプリート表示にのみ影響します
- 通常のテキスト入力には影響しません
- ゴーストテキストは Tab キーで確定できます
- Esc キーでオートコンプリート候補を閉じることができます
- この問題は特に高速タイピング時や、高リフレッシュレートのディスプレイで顕著でした
