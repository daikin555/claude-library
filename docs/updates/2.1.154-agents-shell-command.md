## 原文（日本語訳）

- `claude agents`：`! <コマンド>` と入力することで、シェルコマンドをアタッチ・デタッチ可能なバックグラウンドセッションとして実行できます。`claude --bg --exec '<コマンド>'` としても利用可能です。
- `claude agents`：`/logout` が、バックグラウンドセッションに送られるのではなく、サインアウトするようになりました。

## 原文（英語）

- `claude agents`: type `! <command>` to run a shell command as a background session you can attach to and detach from. Also available as `claude --bg --exec '<command>'`
- `claude agents`: `/logout` now signs you out instead of being sent to a background session

## 概要

`claude agents` 画面から `! <コマンド>` と入力するだけで、任意のシェルコマンドをバックグラウンドセッションとして起動できるようになりました。起動したセッションにはアタッチして出力を確認したり、デタッチして他の作業を続けたりすることが可能です。コマンドラインからは `claude --bg --exec '<コマンド>'` という形式でも同じ機能を利用できます。また、`/logout` コマンドの挙動が修正され、誤ってバックグラウンドセッションに送信されることなく、正しくサインアウト処理が実行されるようになりました。

## 基本的な使い方

### `claude agents` 画面からの操作

```
# claude agents を起動
$ claude agents

# エージェント一覧画面で ! に続けてコマンドを入力
! npm run build

# バックグラウンドセッションとして起動される
# セッションにアタッチして出力を確認する
# デタッチして他の作業に戻ることも可能
```

### コマンドラインフラグを使う方法

```bash
# --bg --exec フラグでバックグラウンドセッションとして起動
claude --bg --exec 'npm run build'

# 長時間かかるコマンドも同様に実行可能
claude --bg --exec 'pytest tests/ -v'
```

## 実践例

### ### ビルドをバックグラウンドで実行しながら別の作業を続ける

長時間かかるビルドやテストを裏で走らせながら、メインの開発作業を継続できます。

```
# claude agents 画面で
! npm run build:production

# バックグラウンドセッションが起動
# 別のエージェントセッションを開いてコーディング作業を続ける
# 必要に応じてビルドセッションにアタッチして進捗を確認
```

### ### テストスイートをバックグラウンドで実行する

```bash
# バックグラウンドでテストを実行しつつ、別のタスクを進める
claude --bg --exec 'pytest tests/ --tb=short 2>&1 | tee test-results.log'

# 後から claude agents 画面でセッションにアタッチしてログを確認
```

### ### 長時間の処理をデタッチしながら監視する

```
# claude agents 画面で重いスクリプトをバックグラウンド起動
! python scripts/data_migration.py --batch-size 1000

# 処理中はデタッチして他の作業
# 完了が近づいたらアタッチして結果を確認
```

### ### `/logout` で確実にサインアウトする

```
# claude agents 画面で
/logout

# 以前は誤ってバックグラウンドセッションに転送されることがあったが、
# 今後は確実にサインアウト処理が実行される
```

## 注意点

- `! <コマンド>` 構文は `claude agents` 画面（エージェント一覧）でのみ使用できます。通常のチャット画面では動作しません。
- バックグラウンドセッションはアタッチ・デタッチが可能ですが、`claude agents` 画面からセッションを選択して操作します。
- `--bg --exec` フラグを使う場合、コマンドはシングルクォートまたはダブルクォートで囲んでください。
- コマンド内にシングルクォートが含まれる場合は、適切なエスケープ処理が必要です。
- `/logout` の挙動変更は `claude agents` 画面での操作に影響します。これまで `/logout` がバックグラウンドセッションに転送されていたユーザーは、新しい挙動（即時サインアウト）に注意してください。
