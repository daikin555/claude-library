# Remote Control初回接続失敗時にエラー理由が通知に表示されない問題を修正

## 原文（日本語に翻訳）

初回接続失敗に対してRemote Controlの失敗通知にエラー理由が表示されない問題を修正しました。

## 原文（英語）

Fixed Remote Control failure notification not showing the error reason for initial connection failures

## 概要

`/remote-control` での初回接続が失敗した際、失敗通知にエラーの理由（「接続拒否」「タイムアウト」「認証失敗」など）が表示されず、「接続に失敗しました」という情報の少ないメッセージしか表示されなかった問題が修正されました。今後は接続失敗の原因が通知に含まれるため、問題の診断が容易になります。

## 基本的な使い方

```bash
/remote-control connect ws://remote-server:3000

# v2.1.126以前（初回接続失敗時）:
# ✗ Remote Control connection failed
# （原因が不明）

# v2.1.126以降:
# ✗ Remote Control connection failed: Connection refused (ECONNREFUSED)
# ✗ Remote Control connection failed: Authentication failed (401)
# ✗ Remote Control connection failed: Connection timed out
```

## 実践例

### エラー原因別の対処方法

```bash
# ECONNREFUSED → リモートサーバーが起動していない
# → サーバーを起動してから再試行

# 401 Authentication failed → 認証トークンが無効
# → 正しいAPIキーまたはトークンを設定

# Connection timed out → ネットワークの問題またはファイアウォール
# → ネットワーク設定やファイアウォールルールを確認
```

## 注意点

- エラーの詳細情報を使って適切な対処を行ってください
- セキュリティ上センシティブな情報（認証トークンなど）はエラーメッセージに含まれません
