## 原文（日本語に翻訳）

新しい Claude Code バージョンが書き込んだバックグラウンドジョブが `claude agents` から消えたり、データが失われる問題を修正

## 原文（英語）

Fixed background jobs disappearing from `claude agents` or losing data when written by a newer Claude Code version

## 概要

新しいバージョンの Claude Code がバックグラウンドジョブを書き込んだ後、古いバージョンで `claude agents` を開くとジョブが表示されなくなったり、データが失われる問題が修正されました。これはバージョン間の互換性問題であり、チーム内で複数のバージョンを使用している場合や、アップデート後に以前のジョブを確認しようとした際に発生していました。修正後は後方互換性のあるフォーマットでジョブデータが保存されます。

## 基本的な使い方

```bash
# バックグラウンドエージェントの一覧表示
claude agents

# バックグラウンドでタスクを実行
claude --background "長時間かかるタスクを実行してください"
```

## 実践例

### バージョンアップ後の既存ジョブ確認

```bash
# Claude Code をアップデート前に開始したジョブ
claude --background "コードベース全体の分析を行ってください"

# Claude Code をアップデート
claude update

# 修正前: アップデート前に開始したジョブが表示されないことがあった
# 修正後: 以前のジョブも正しく表示される
claude agents
```

### チーム環境でのバックグラウンドジョブ管理

```
シナリオ: チームメンバーが異なるバージョンの Claude Code を使用

メンバー A (v2.1.195): バックグラウンドジョブを作成
メンバー B (v2.1.194): 同じプロジェクトで claude agents を確認

修正前: メンバー B がジョブを見つけられないことがあった
修正後: ✅ バージョンに関わらずジョブ情報が正しく表示される
```

### 長時間バックグラウンド処理の管理

```bash
# 長時間かかる大規模分析をバックグラウンドで実行
claude --background "10万行のコードベースを分析してリファクタリング提案を作成してください"

# 後で確認（バージョンアップしても結果が保持される）
claude agents
```

## 注意点

- バージョン 2.1.195 以降のジョブデータは後方互換性を持つフォーマットで保存されます
- 既に失われたジョブデータは復元できません（今後の新規ジョブから適用）
- `claude agents` の表示に問題がある場合は、Claude Code を最新版にアップデートしてください
- チーム全体で同じバージョンを使用することをお勧めします
