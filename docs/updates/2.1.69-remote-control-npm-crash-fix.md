## 原文（日本語に翻訳）

npm インストール時に「bad option: --sdk-url」というエラーで `claude remote-control` が即座にクラッシュしていた問題を修正しました

## 原文（英語）

Fixed `claude remote-control` crashing immediately on npm installs with "bad option: --sdk-url"

## 概要

`claude remote-control` コマンドを npm 経由でインストールした際に、`--sdk-url` オプションが認識されず「bad option: --sdk-url」というエラーメッセージとともに即座にクラッシュしていました。この問題により、リモートコントロール機能が利用できない状態になっていたため、修正が適用されました。

## 基本的な使い方

修正後は、`claude remote-control` が正常に動作します。

```bash
# npm インストール後に正常起動できるようになった
npm install -g @anthropic-ai/claude-code
claude remote-control
```

## 実践例

### リモートコントロールの基本的な使用

```bash
# Claude Code をリモートコントロールモードで起動
claude remote-control

# または特定のセッションに接続
claude remote-control --session-id <session-id>
```

### Android アプリとの連携

```bash
# ローカルマシンで remote-control を起動
claude remote-control

# Android アプリから接続して Claude Code を操作できる
# → 以前はこの起動ステップでクラッシュしていた
```

### トラブルシューティング（修正前の回避策）

```bash
# 修正前に試みられていた回避策（現在は不要）
# npm install の代わりに直接バイナリをダウンロードするなど

# 修正後は通常の npm install で問題なし
npm install -g @anthropic-ai/claude-code@latest
claude remote-control  # 正常に起動
```

## 注意点

- この修正は npm 経由のインストール時に発生していたバグへの対応です
- 最新版（2.1.69 以降）にアップデートすれば問題は解消されます
- `npm install -g @anthropic-ai/claude-code@latest` でアップデートできます
- リモートコントロール機能はモバイルアプリとの連携に使用されます
