## 原文（日本語に翻訳）

`/clear` やコンパクション後に `/effort` または `/model` を変更した際に、キャッシュミス警告が不正に表示される問題を修正しました。

## 原文（英語）

Fixed cache-miss warning appearing spuriously after `/clear` or compaction when changing `/effort` or `/model`

## 概要

`/clear` コマンドで会話をクリアした後、またはコンパクションが実行された後に `/effort` や `/model` コマンドで設定を変更すると、不必要なキャッシュミス警告が表示される問題が修正されました。これらの警告は誤ったものであり、実際のキャッシュの問題を示すものではありませんでした。修正後は正当なキャッシュミスのみが警告として表示されます。

## 基本的な使い方

この修正は自動的に適用されます。`/clear` 後や `/effort`・`/model` 変更時に不要な警告が表示されなくなります。

```
# 修正前の誤った動作
/clear
/model claude-opus-4-5
# → ⚠ キャッシュミス警告（不正） ← 誤った警告

# 修正後の正しい動作
/clear
/model claude-opus-4-5
# → 警告なし（正常）
```

## 実践例

### /clear 後のモデル切り替え

```
# 会話をリセットして別のモデルに切り替える
/clear
/model claude-haiku-3-5

# 修正前: 不必要なキャッシュミス警告が表示される
# 修正後: 警告なしで正常に切り替わる
```

### コンパクション後の設定変更

```
# コンパクションが自動実行された後
# （コンテキストウィンドウが一定量を超えた場合）

/effort max
# 修正前: キャッシュミス警告が表示される（誤り）
# 修正後: 正常に effort レベルが変更される
```

### ワークフローへの影響

```
# 日常的なワークフロー例
/clear              # セッションをリセット
/model claude-opus-4-5    # 重い作業用モデルに切り替え
/effort max         # 最大の思考量に設定

# 修正後: 上記の操作後に誤警告が表示されなくなる
# 実際のキャッシュ問題が発生した場合のみ警告が表示される
```

## 注意点

- 実際のキャッシュミス（プロンプトキャッシュが期待通りに機能しない場合）では引き続き警告が表示されます。
- この修正により、警告の信頼性が向上します。警告が表示された場合は実際の問題である可能性が高くなりました。
- コンパクションは自動または `/compact` コマンドで手動実行できます。コンパクション後は/clearと同様の動作になります。
