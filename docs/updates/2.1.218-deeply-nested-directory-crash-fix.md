## 原文（日本語に翻訳）

監視対象の深くネストしたディレクトリツリーが削除・移動された際、および深くネストしたUIツリーを描画する際に発生する「maximum call stack exceeded（最大コールスタック超過）」クラッシュを修正しました

## 原文（英語）

Fixed crashes (maximum call stack exceeded) when a deeply nested watched directory tree was deleted or moved, and when rendering deeply nested UI trees

## 概要

Claude Code v2.1.218で修正された、2件の関連するスタックオーバーフロー系クラッシュのバグです。1つ目は、Claude Codeのファイル監視機能が監視している非常に深い階層のディレクトリツリーを一度に削除・移動すると、再帰的な処理でコールスタックの上限を超えてクラッシュすることがありました。2つ目は、ターミナルUIで深くネストしたコンポーネントツリーを描画する際にも同様のリスクがありました。修正後は、いずれのケースも無制限な再帰ではなく反復的な処理などで対応し、クラッシュしなくなりました。

## 注意点

- Claude Code v2.1.218で修正
- 影響範囲は非常に深いディレクトリ構造の監視・削除・移動、および深くネストしたUI描画
- 通常のプロジェクト構成では問題は起きにくいが、大規模・深い階層のリポジトリでの安定性が向上
- ユーザー側で特別な設定変更は不要
