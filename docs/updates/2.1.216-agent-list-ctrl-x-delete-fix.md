## 原文（日本語訳）

エージェント一覧でCtrl+Xを2回押してもセッションが削除に失敗する問題、および削除済みセッションのバックグラウンドワーカーが停止していた場合にセッションが再表示されてしまう問題を修正しました。

## 原文（英語）

Fixed pressing Ctrl+X twice in the agent list failing to delete a session, and deleted sessions reappearing when their background worker had died

## 概要

Claude Codeのエージェント一覧（agent view）では、対象セッション上でCtrl+Xを押すとセッションが停止し、2秒以内にもう一度Ctrl+Xを押すとそのセッションが削除されます。しかし、この操作を行っても削除処理が正しく完了しない場合があり、さらにセッションのバックグラウンドワーカーが既に停止（死亡）していた場合には、一度削除したはずのセッションが一覧に再度表示されてしまう不具合がありました。今回の修正により、Ctrl+Xの2回押しによる削除が確実に行われ、ワーカーが停止しているセッションの削除状態も正しく維持されるようになりました。

## 基本的な使い方

この修正はバグ修正であり、通常のエージェント一覧の操作フローに変更はありません。

```
# エージェント一覧でセッションを選択した状態で
Ctrl+X          # セッションを停止
Ctrl+X（2秒以内） # セッションを削除
```

## 実践例

### 不要になったバックグラウンドセッションの整理

複数のバックグラウンドセッションを並行して走らせたあと、不要になったものを一覧から削除してクリーンアップします。

```
# エージェント一覧を開く
claude
# （エージェント一覧画面で）対象セッションにカーソルを合わせて
Ctrl+X
Ctrl+X
```

削除すると、そのセッションに紐づくワークツリー（未コミットの変更を含む）も削除されるため、残しておきたい変更は事前にコミットしておく必要があります。

### CLIから直接セッションを削除する

エージェント一覧を開かずにセッションを削除したい場合は、CLIコマンドを利用することもできます。

```bash
claude rm <session-id>
```

Ctrl+Xによる削除、`claude rm` による削除のいずれの場合も、削除操作はスーパーバイザーのセッション一覧から確実にクリアされ、スーパーバイザー再起動後も削除状態が維持されます。

### ワーカーが停止したセッションの状態確認

セッションのバックグラウンドワーカーが異常終了していないかを確認したい場合は、エージェント一覧の状態表示を確認します。

```

```

## 注意点

- Ctrl+Xによる削除はワークツリーごと削除するため、未コミットの変更は失われます。削除前に必要な変更はコミットしてください。
- 2回目のCtrl+Xは1回目から2秒以内に押す必要があります。時間が経過すると通常の停止操作として扱われます。
- この修正以前は、バックグラウンドワーカーが既に停止しているセッションを削除しても、一覧の再描画時に復活してしまうことがありましたが、今回の修正で解消されています。
