## 原文（日本語訳）

Remote Control（モバイル／Web）からインタラクティブセッションに送信されたコマンドが「Unknown command（不明なコマンド）」エラーで失敗する不具合を修正しました。

## 原文（英語）

Fixed commands sent from Remote Control (mobile/web) into an interactive session failing with "Unknown command"

## 概要

これはバグ修正のリリースです。Remote Control機能を使い、スマートフォンやWebブラウザ（claude.ai/codeやClaudeモバイルアプリ）からインタラクティブに起動しているClaude Codeセッション（`claude --remote-control`やセッション中の`/remote-control`で開始したセッション）に対してコマンドを送信すると、本来サポートされているはずのコマンドであっても「Unknown command」というエラーになり実行できないことがありました。今回の修正により、モバイル／Webから送信したコマンドがローカルのインタラクティブセッションで正しく認識・実行されるようになりました。

## 基本的な使い方

Remote Controlは、手元のマシンで動いているClaude Codeセッションを、スマートフォンやタブレット、別のPCのブラウザから操作できるようにする機能です。処理自体はあくまでローカルマシン上で実行され、クラウドに移行するわけではありません。

セッションを開始する主な方法は次の3つです。

```bash
# 1. サーバーモード（複数セッションを待ち受け）
claude remote-control

# 2. インタラクティブセッションをRemote Control有効で開始
claude --remote-control
# 短縮形
claude --rc
```

```text
# 3. 既存のセッション内からRemote Controlを開始
/remote-control
```

インタラクティブセッション（上記2, 3の方法）で接続すると、ターミナルのフッターに`/rc active`という接続状況インジケーターが表示されます。表示されたセッションURLやQRコードから、claude.ai/codeまたはClaudeモバイルアプリで同じセッションに接続できます。

## 実践例

### 外出先からタスクの進捗を確認し、コマンドで制御する

デスクで`claude --remote-control`を使って長時間かかるリファクタリングタスクを開始し、外出先でスマートフォンのClaudeアプリからセッションに接続したとします。この状態で会話の要約を取得するために`/compact`をモバイルから送信すると、今回の修正前は「Unknown command」エラーとなり実行できないケースがありました。修正後は、モバイルから送った`/compact`や`/context`、`/usage`といったテキスト出力系コマンドが、ローカルのインタラクティブセッション側で正しく解釈され、期待通りの結果が返ってきます。

### ブラウザ（claude.ai/code）からMCPサーバーの状態を確認する

会社のPCで`/remote-control`を実行して既存セッションをリモート化し、自宅のブラウザからclaude.ai/codeにアクセスして作業を引き継ぐ場面を考えます。ここで`/mcp`コマンドを送信すると、ローカルCLIのようなインタラクティブなピッカーは開きませんが、テキストでMCPサーバーの状態サマリーが返るはずです。しかし修正前は、このようにWebから送ったコマンドがインタラクティブセッション側で「Unknown command」として扱われ失敗することがありました。修正後は`/mcp`や`reconnect`・`enable`・`disable`といったサブコマンドも含め、モバイル／Webから送信したコマンドが正しくローカルセッションに伝わり実行されます。

## 注意点

- この修正はあくまで「モバイル／Webからインタラクティブセッションへ送信したコマンドが誤ってUnknown command扱いになる」不具合の修正です。`/plugin`や`/resume`のようにインタラクティブなピッカーをターミナルに表示するコマンドは、引き続きローカルCLIからのみ実行可能で、モバイル／Webからは利用できません。
- モバイル／Webから利用できるのは、`/compact`、`/clear`、`/context`、`/usage`、`/exit`、`/usage-credits`、`/recap`、`/reload-plugins`などのテキスト出力系コマンドと、`/mcp`（バージョン2.1.166以降はサブコマンド対応）、`/config`（バージョン2.1.181以降は`key=value`形式での設定変更に対応）です。
- Remote Controlの利用にはPro・Max・Team・Enterpriseプランでのclaude.ai OAuthログインが必要で、APIキー認証や、Amazon Bedrock・Google Cloud・Microsoft Foundry経由の利用では使用できません。
- コマンドが実行できない場合は、まず`/status`でログイン方法やサブスクリプションの状態を確認し、それでも解決しない場合は`claude --verbose`で詳細ログを確認することが推奨されています。
