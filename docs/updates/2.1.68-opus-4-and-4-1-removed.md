# Opus 4および4.1がファーストパーティAPIから削除 — 自動的にOpus 4.6へ移行

## 原文（日本語に翻訳）

Claude CodeのファーストパーティAPIからOpus 4およびOpus 4.1が削除されました。これらのモデルを固定設定していたユーザーは、自動的にOpus 4.6に移行されます。

## 原文（英語）

Removed Opus 4 and 4.1 from Claude Code on the first-party API — users with these models pinned are automatically moved to Opus 4.6

## 概要

Claude Code v2.1.68より、ファーストパーティAPI（Anthropic直接API）においてOpus 4およびOpus 4.1が利用できなくなりました。旧モデルをピン留め設定していたユーザーは自動的にOpus 4.6へ移行されます。Opus 4.6は1Mトークンコンテキスト、アダプティブ思考、エフォート制御など大幅に強化されており、後継モデルへの移行は品質面でもメリットがあります。

## 基本的な使い方

### 現在使用しているモデルの確認

```bash
# セッション内でモデルを確認
/model
```

### モデルを明示的にOpus 4.6に設定する

```bash
# 起動時に指定
claude --model claude-opus-4-6

# 環境変数で設定
export ANTHROPIC_MODEL=claude-opus-4-6
```

### 設定ファイルの確認・更新

古いモデル名が設定ファイルに残っている場合、Opus 4.6への更新が必要です。

```bash
# ユーザーレベルの設定確認
cat ~/.claude/settings.json

# プロジェクトレベルの設定確認
cat .claude/settings.json
```

旧モデル名が記載されている場合は更新してください：

```json
// 旧設定（削除されたモデル）
{
  "model": "claude-opus-4"
}

// 新設定（Opus 4.6に更新）
{
  "model": "claude-opus-4-6"
}
```

## 実践例

### 自動移行後の動作確認

```
# セッション開始後、/modelで現在のモデルを確認
/model
```

Opus 4またはOpus 4.1がピン留めされていた場合、自動的にOpus 4.6が表示されるはずです。

### Amazon BedrockやVertex AIユーザーの場合

ファーストパーティAPI以外（Bedrock・Vertex AI）でOpus 4/4.1を使用していた場合、自動移行の対象外となる可能性があります。それぞれのプラットフォームのモデル提供状況を確認してください。

```bash
# Bedrockを使用している場合の確認
echo $ANTHROPIC_BEDROCK_BASE_URL

# Vertex AIを使用している場合の確認
echo $ANTHROPIC_VERTEX_PROJECT_ID
```

### Opus 4.6の主な改善点

移行後に利用できる新機能：

| 機能 | Opus 4/4.1 | Opus 4.6 |
|---|---|---|
| コンテキストウィンドウ | 200K | 1Mトークン（ベータ） |
| 最大出力トークン | 32K | 128K |
| エフォートレベル | なし | low/medium/high/max |
| アダプティブ思考 | なし | あり |
| エージェントチーム | なし | あり（リサーチプレビュー） |

## 注意点

- **自動移行はファーストパーティAPIのみ**: Bedrock・Vertex AIなど他のプラットフォームは別途対応が必要な場合があります。
- **ANTHROPIC_DEFAULT_OPUS_MODELの確認**: 環境変数でモデルをピン留めしている場合は、`ANTHROPIC_DEFAULT_OPUS_MODEL` の設定も確認・更新してください。
- **Enterprise管理者への影響**: `availableModels` でOpus 4/4.1を明示していた場合は、ポリシー設定の更新が必要です。
- **コスト**: Opus 4.6はOpus 4/4.1と同価格帯（$5/$25 per million tokens）のため、コスト面での影響はほぼありません。
