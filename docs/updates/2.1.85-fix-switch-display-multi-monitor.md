## 原文（日本語に翻訳）

マルチモニター環境でcomputer-useツールの `switch_display` が「このセッションでは利用できません」を返す問題を修正しました。

## 原文（英語）

Fixed `switch_display` in the computer-use tool returning "not available in this session" on multi-monitor setups

## 概要

computer-useツールの `switch_display` アクションがマルチモニター環境で正常に動作せず、「not available in this session（このセッションでは利用できません）」というエラーを返すバグが修正されました。複数のモニターを使用している環境でも、Claudeが正しくディスプレイを切り替えられるようになります。

## 基本的な使い方

computer-useツールでディスプレイを切り替える：

```json
{
  "action": "switch_display",
  "display_number": 2
}
```

修正前はマルチモニター環境でこの操作が失敗していましたが、修正後は正常に動作します。

## 実践例

### ユースケース: マルチモニター環境でのコンピューター操作

複数のモニターを使用した環境でClaudeにコンピューター操作を指示する場合：

```
サブモニター（ディスプレイ2）に表示されているアプリケーションを確認して、必要な情報を取得してください
```

Claudeは `switch_display` を使って適切なモニターに切り替えて操作を行います。

### ユースケース: デュアルモニターのワークフローの自動化

メインモニターでコードを書きながら、サブモニターでブラウザを操作するワークフロー：

```
メインモニターのコードを確認して、サブモニターのブラウザでそのドキュメントを検索してください
```

## 注意点

- この修正はcomputer-useが有効な環境でのみ関連します
- `switch_display` のディスプレイ番号は1から始まります
- マルチモニター環境では、各ディスプレイの解像度と配置が正確に認識される必要があります
