## 原文（日本語に翻訳）

一部のターミナルで、ユーザーメッセージのバブル内の特定の絵文字の背景色が欠ける問題を修正しました。

## 原文（英語）

Fixed missing background color behind certain emoji in user message bubbles on some terminals.

## 概要

Claude Codeのチャット画面でユーザーのメッセージが表示されるバブル（吹き出し）内に絵文字が含まれる場合、一部のターミナルエミュレータ（例: iTerm2、Alacritty、WezTermなど）でその絵文字の背景色がバブルの背景色と一致せず、透明または異なる色で表示されるレンダリングバグが修正されました。

## 基本的な使い方

この修正は自動的に適用されるため、特別な設定は不要です。絵文字を含むメッセージが正しい背景色でレンダリングされるようになります。

```
# 修正後は絵文字が正しくレンダリングされる例
ユーザー: 🚀 このコードを最適化してください
         ✅ テストが通っていることを確認して
         🐛 バグを修正してください
```

## 実践例

### ユースケース: 絵文字を使ったプロンプト

プロンプトに絵文字を使用するユーザーの場合：

```
修正前の問題（一部ターミナル）:
「🎯 目標を達成するコードを書いて」と入力すると
絵文字「🎯」の周囲に背景色が欠けた不自然な表示になる

修正後の動作:
絵文字がバブルの背景色と統一されたきれいな表示になる
```

### ユースケース: 絵文字を含む複数行メッセージ

```
1️⃣ まずコードをリファクタリングして
2️⃣ 次にユニットテストを追加して
3️⃣ 最後にドキュメントを更新して
```

修正後は、番号絵文字（1️⃣ 2️⃣ 3️⃣）も含めて、すべての絵文字の背景色が正しく表示されます。

## 注意点

- この問題は特定のターミナルエミュレータでのみ発生していました
- ターミナルによって絵文字のレンダリング方法が異なるため、すべての環境での正確な表示が保証されます
- Claude Codeの機能的な動作には影響しない、純粋に視覚的な修正です
