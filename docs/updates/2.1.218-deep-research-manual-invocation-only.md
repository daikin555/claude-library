## 原文（日本語に翻訳）

`/deep-research`を手動で実行した場合にのみ開始するよう変更しました。Claudeが自らそれを起動することはなくなりました。

## 原文（英語）

Changed `/deep-research` to start only when invoked manually; Claude no longer launches it on its own

## 概要

`/deep-research`は、あるトピックについて広範囲かつ時間のかかる調査プロセスを開始するスラッシュコマンドです。これまでは、会話の流れの中でClaudeが「詳しく調べたほうがよい」と自律的に判断し、ユーザーが明示的に指示していないにもかかわらず`/deep-research`相当の処理を自ら開始してしまうことがありました。v2.1.218では、この自律的な起動が廃止され、`/deep-research`はユーザーが明示的にコマンドを実行した場合にのみ開始するようになりました。これにより、ユーザーが意図しないタイミングで長時間の調査タスクが走り出すことがなくなり、いつ・どの程度の処理が始まるかをユーザー自身がコントロールできるようになります。

## 基本的な使い方

`/deep-research`を使うには、ユーザーが明示的にコマンドを実行します。

```bash
claude
> /deep-research 生成AIにおけるエージェント評価手法の最新動向
```

Claudeは会話中に「これはdeep researchが必要そうな話題だ」と判断しても、ユーザーの明示的な指示なしに自動でこの調査プロセスを開始することはありません。

## 実践例

### 通常の会話中に深い調査が必要だと感じた場合

```bash
claude
> 最近の分散システムにおけるコンセンサスアルゴリズムの
> 研究動向を幅広く調べてレポートにまとめて

# 変更前: Claudeがこの依頼を「deep researchが適切」と
#         自律的に判断し、内部的に長時間の調査プロセスを
#         勝手に開始してしまうことがあった
# 変更後: 通常の応答として調べられる範囲で回答し、
#         本格的な深掘り調査が必要な場合はユーザーに
#         /deep-research の利用を提案するにとどまる
```

### ユーザーが明示的に深い調査を依頼する場合

```bash
claude
> /deep-research 量子耐性暗号アルゴリズムの標準化状況と
> 主要クラウドベンダーの対応ロードマップ

# 明示的にコマンドを実行した場合のみ、
# 広範囲な調査・複数ソースの参照・
# 長時間にわたる処理が開始される
```

### 長時間タスクの実行タイミングを自分でコントロールしたい場合

```bash
# 通常の質問は素早い応答を期待し、
# 本格的な調査は都合の良いタイミングで
# 明示的に起動したい、という使い分けができる

claude
> このライブラリの使い方を簡単に教えて
# → 通常の応答（自動でdeep researchは始まらない）

> /deep-research このライブラリの設計思想と代替ライブラリとの比較
# → 明示的な指示に基づき深い調査を開始
```

## 注意点

- この変更はClaude Code v2.1.218で実装されました
- 以前のバージョンでは、複雑な調査系のリクエストに対してClaudeが自律的にdeep research相当の処理を開始することがありましたが、現在はその挙動が廃止されています
- 深い調査が必要だとClaudeが判断した場合でも、まずはユーザーに`/deep-research`の利用を提案する形にとどまり、実際の起動はユーザーの明示的なコマンド実行を待つようになります
- この変更により、意図しない長時間処理によるトークン消費や待ち時間の増加を防ぎやすくなります
