## 原文（日本語訳）

バックグラウンドサブエージェントの起動処理中に高優先度メッセージが届くと、そのサブエージェントがキャンセルされてしまう不具合を修正しました。

## 原文（英語）

Fixed background subagents getting cancelled when a high-priority message arrives during their startup window

## 概要

Claude Codeでは、サブエージェントをバックグラウンドで実行しながらメインスレッドで別の作業を続けることができます。しかし、バックグラウンドサブエージェントが起動処理中（スタートアップウィンドウ）にある間に、割り込みなど優先度の高いメッセージが届くと、まだ実質的な処理を始めてもいないそのサブエージェントが誤ってキャンセルされてしまう不具合がありました。今回の修正により、起動処理中のバックグラウンドサブエージェントが高優先度メッセージの到着によって不必要にキャンセルされることがなくなりました。

## 基本的な使い方

この修正はバグ修正であり、特別な操作は不要です。バックグラウンドでサブエージェントを起動した直後にメインスレッドへメッセージを送っても、そのサブエージェントは正常に起動を完了し、処理を続行します。

```
# バックグラウンドでサブエージェントを起動
「この調査をバックグラウンドで実行して」

# 起動直後にメインスレッドへ別の指示を送っても、
# バックグラウンドサブエージェントはキャンセルされずに継続する
```

## 実践例

### 複数のバックグラウンド調査タスクを立て続けに起動する

大きな調査タスクをバックグラウンドで開始した直後に、別の緊急対応をメインスレッドに指示するようなワークフローで、この修正の効果を確認できます。

```
# 1つ目のタスクをバックグラウンドで起動
「リポジトリ全体の依存関係を調査して」（バックグラウンド実行を指示）

# 直後に高優先度の割り込みメッセージを送信
「いま実行中のコマンドを止めて、先にこちらを直して」
```

修正前はこのようなタイミングで1つ目のバックグラウンドサブエージェントが起動途中にキャンセルされることがありましたが、修正後は影響を受けずに完了まで実行されます。

### frontmatterで常時バックグラウンド実行するエージェントの起動

`background: true` を設定したエージェントは常にバックグラウンドで起動しますが、起動直後にメインスレッドで他の操作を行っても正しく起動が完了します。

```markdown
---
name: researcher
description: コードベース調査用のバックグラウンドエージェント
background: true
---

あなたはコードベース調査を専門に行うエージェントです。
```

### `/tasks` での起動状況の確認

バックグラウンドサブエージェントが起動直後にキャンセルされていないか不安な場合は、`/tasks` で状況を確認できます。

```
/tasks
```

正常に起動していれば、一覧に "Working" として表示されます。

## 注意点

- この修正は起動処理中（スタートアップウィンドウ）という短い時間帯に限定された競合状態の修正です。起動が完了した後のバックグラウンドサブエージェントの挙動には影響しません。
- バックグラウンドタスクはセッションで既に許可されている権限の範囲で動作し、それ以外の操作が必要になった場合は自動的に拒否されます。
- バックグラウンドタスク機能全体を無効化したい場合は、環境変数 `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS` を `1` に設定します。
