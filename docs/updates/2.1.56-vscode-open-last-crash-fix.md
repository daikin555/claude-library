## 原文（日本語に翻訳）

VS Code: `command 'claude-vscode.editor.openLast' not found` クラッシュの別の原因を修正しました

## 原文（英語）

VS Code: Fixed another cause of "command 'claude-vscode.editor.openLast' not found" crashes

## 概要

Claude Code VS Code拡張機能において、`claude-vscode.editor.openLast` コマンドが見つからないというエラーによるクラッシュの、さらなる原因を修正しました。このエラーはVS Codeでの作業中に突然クラッシュを引き起こしていたもので、特定のシナリオでも発生していた問題が修正されています。これにより、VS Code上でのClaude Code利用の安定性が向上しました。

## 基本的な使い方

この修正はVS Code拡張機能の自動アップデートで適用されます。手動で最新バージョンに更新する場合は以下の手順に従います。

```bash
# Claude Code CLIを最新バージョンに更新
npm install -g @anthropic-ai/claude-code@latest

# バージョン確認
claude --version
# 2.1.56 以上であることを確認
```

VS Code拡張機能は Claude Code CLIと連携して動作します。CLIを更新後、VS Codeを再起動することで最新の修正が適用されます。

## 実践例

### VS Codeで最後のセッションを開く

`openLast` 機能は、前回のClaude Codeセッションを再開するために使用されます。

```bash
# VS Codeのコマンドパレットから（Ctrl+Shift+P / Cmd+Shift+P）
# "Claude: Open Last Session" を検索して実行

# または VS Code キーボードショートカットから呼び出す場合
# 設定に応じたショートカットを使用
```

### 拡張機能の状態確認

```bash
# VS Code拡張機能タブで Claude Code 拡張機能が最新か確認
# Extensions (Ctrl+Shift+X) → "@installed claude" で検索

# クラッシュが続く場合は VS Code の開発者コンソールを確認
# Help → Toggle Developer Tools → Console タブ
```

### 問題が解消されない場合のトラブルシューティング

```bash
# VS Code拡張機能をリロード
# コマンドパレット: "Developer: Reload Window"

# Claude Code CLIを再インストール
npm uninstall -g @anthropic-ai/claude-code
npm install -g @anthropic-ai/claude-code@latest

# VS Codeを完全に再起動
```

## 注意点

- このクラッシュは v2.1.56 で修正されています（最初の修正は以前のバージョンで実施済み）
- VS Code拡張機能は Claude Code CLI に依存しているため、両方を最新バージョンに保つことを推奨します
- 同様のクラッシュが引き続き発生する場合は、GitHub Issues にて報告してください
- エラーが発生した場合はVS Codeの開発者コンソール（Help → Toggle Developer Tools）でエラー詳細を確認できます
