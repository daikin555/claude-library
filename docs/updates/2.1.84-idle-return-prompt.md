## 原文（日本語に翻訳）

75分以上離席して戻ってきたユーザーに `/clear` を促すアイドルリターンプロンプトを追加。古いセッションでの不要なトークン再キャッシュを削減します。

## 原文（英語）

Added idle-return prompt that nudges users returning after 75+ minutes to `/clear`, reducing unnecessary token re-caching on stale sessions

## 概要

Claude Codeセッションを75分以上放置した後に作業を再開すると、`/clear` コマンドの使用を促すプロンプトが表示されるようになりました。長時間の中断後は古いコンテキストが不要なトークンを消費する可能性があるため、セッションをクリアしてフレッシュな状態で再開することでコスト効率が改善されます。

## 基本的な使い方

75分以上アイドル状態になった後にClaude Codeに戻ると、以下のようなメッセージが表示されます。

```
75分以上経過しました。/clear でセッションをリセットすることをお勧めします。
古いコンテキストは不要なトークン消費につながる可能性があります。
```

`/clear` を実行してセッションをリセットするか、そのまま作業を続けることができます。

```
/clear
```

## 実践例

### 長時間離席後の推奨ワークフロー

```bash
# 朝の作業開始時（前日のセッションが残っている場合）
claude

# プロンプトに従って /clear を実行
/clear

# フレッシュなセッションで新しいタスクを開始
"今日のタスクを始めましょう..."
```

### コンテキストを保持したい場合

アイドルリターンプロンプトが表示されても、作業途中のコンテキストが重要な場合はそのまま続けられます。

```bash
# プロンプトを無視して継続
# 表示されたメッセージをスキップして、そのまま入力を続ける
"前回の続きから..."
```

### プロンプトキャッシュとコスト管理

長時間セッションを継続するとキャッシュが無効化され、トークンが再計算されます。

```bash
# コスト効率を重視する場合
# 75分後のプロンプトに従い /clear を実行してから再開

/clear
"新しいタスク: APIの実装を手伝ってください"
```

## 注意点

- このプロンプトはあくまで**提案**であり、強制ではありません。そのまま作業を続けることもできます
- 長時間の複雑な作業で中断が多い場合、コンテキストを保持した方が効率的なこともあります
- `/clear` を実行すると**現在の会話履歴がすべて削除**されます。重要な情報は事前にメモしておいてください
- 75分という閾値は、プロンプトキャッシュの有効期限（通常1〜5分）を考慮したものです。長時間離席するとキャッシュが失効し、再読み込みのコストが発生します
