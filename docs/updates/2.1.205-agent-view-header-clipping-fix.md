## 原文（日本語訳）

ジョブ一覧が画面をわずかにはみ出した際に、agent viewが1行分上にずれて描画され、ヘッダーが欠けて表示される問題を修正した。

## 原文（英語）

Fixed the agent view rendering one line too high and clipping its header when the job list slightly overflowed the screen

## 概要

agent view（`claude agents`）は、バックグラウンドで動作する複数のClaude Codeセッションを1つの画面から一覧・操作できる機能で、画面上部のヘッダーにバージョンやモデル、作業ディレクトリ、セッション数のサマリーが表示されます。今回の修正前は、`Working` や `Needs input` などのグループに属するセッション（ジョブ）の一覧が画面の縦幅をわずかに超えるだけで、描画位置が1行分上にずれてしまい、本来一番上に表示されるべきヘッダー行が切り取られて見えなくなる不具合がありました。v2.1.205ではこのレイアウト計算が修正され、ジョブ一覧がわずかにオーバーフローする状況でもヘッダーが正しく全体表示されるようになっています。

## 基本的な使い方

agent viewはシェルから次のコマンドで開きます。

```bash
claude agents
```

画面上部にはヘッダー（バージョン、使用モデル、作業ディレクトリ、セッション数などのサマリー）が表示され、その下に `Pinned` / `Ready for review` / `Needs input` / `Working` / `Completed` といったグループごとにセッション（ジョブ）が一覧表示されます。矢印キーで行を選択し、`Space` でピーク（覗き見）、`Enter` または `→` でアタッチできます。`Esc` でいつでもシェルに戻れます。

## 実践例

### ターミナルの高さいっぱいまでバックグラウンドセッションを走らせている場合

多数のタスクを同時にバックグラウンド実行していると、`Working` や `Completed` のセッション行が積み重なり、ターミナルの縦幅をわずかに超えることがあります。修正前はこの「わずかな超過」だけで描画が1行分ずれ、ヘッダーの一部が見切れてバージョンやモデル名が確認しづらくなっていました。

```bash
claude agents
# 多数のセッションを起動した状態で開くと、
# 以前はヘッダーの一部が欠けて見えることがあった
```

v2.1.205移行では、ジョブ一覧が画面の高さぎりぎりになってもヘッダーが正しい位置・高さで描画されるため、バージョンや現在のモデル設定などの情報を常に確認できます。

### 小さめのターミナルウィンドウでagent viewを常用している場合

ノートPCの画面や分割ペインなど、ターミナルの行数が少ない環境でagent viewを使う場合、この不具合の影響を受けやすい状況でした。なお、`Completed` グループはもともと画面の残り縦幅を埋めるように調整され、短いターミナルではヘッダーが1行のサマリーに自動的に圧縮される仕様になっていますが、今回の修正はその調整計算自体にあった「1行分ずれる」バグを解消するものです。

```bash
claude agents
# ウィンドウの高さを縮めてジョブが数件だけ並ぶ状態でも、
# ヘッダーが欠けずに表示されることを確認できる
```

## 注意点

- この修正は表示上のレイアウト不具合の修正であり、セッションの状態管理やジョブ一覧のグルーピングロジック自体（`Ready for review` / `Needs input` / `Working` / `Completed` の分類）には影響しません。
- agent viewは本記事執筆時点でリサーチプレビュー機能であり、インターフェースやキーボードショートカットは今後変更される可能性があります。
- 表示が乱れて見える場合は `claude update` で最新版に更新してください。
