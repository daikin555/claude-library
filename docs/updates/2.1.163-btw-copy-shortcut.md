## 原文（日本語に翻訳）

`/btw` に「c でコピー」ショートカットを追加。生の Markdown 回答をクリップボードにコピーし、他の場所に貼り付けた際もフォーマットを保持できます。

## 原文（英語）

Added a "c to copy" shortcut to `/btw` that copies the raw markdown answer to the clipboard, preserving formatting when pasted elsewhere

## 概要

`/btw` コマンドで表示された回答を `c` キーを押すだけでクリップボードにコピーできるようになりました。コピーされるのは生の Markdown テキストのため、Notion、Obsidian、GitHub などの Markdown 対応ツールに貼り付けるとフォーマットが正しく維持されます。

## 基本的な使い方

1. Claude Code で `/btw` コマンドを使用して質問する
2. 回答が表示されたら `c` キーを押す
3. 生の Markdown テキストがクリップボードにコピーされる

```
/btw このプロジェクトのアーキテクチャを説明して
```

回答が表示されたら `c` キーで Markdown ごとコピーできます。

## 実践例

### Notion や Obsidian へのエクスポート

`/btw` で生成した説明文やコードブロックを含む回答を Markdown 形式のまま Notion や Obsidian に貼り付けたい場合：

1. `/btw` で質問する
2. 回答が表示されたら `c` キーを押す
3. Notion または Obsidian のページに貼り付ける（Ctrl+V / Cmd+V）

Markdown の見出し、コードブロック、リストなどがすべて保持されます。

### GitHub の Issue や PR の説明文を作成

コードの変更内容の説明を `/btw` で生成して、GitHub にコピーペーストする場合：

1. `/btw` でコードの変更点や設計上の判断を質問する
2. `c` キーで Markdown テキストをコピー
3. GitHub の Issue や PR の説明欄に直接貼り付ける

### ドキュメントの草案をすばやく転記

```
/btw この関数の JSDoc コメントを書いて
```

回答後に `c` キーを押すと、フォーマット済みの JSDoc がクリップボードに入り、エディタに貼り付けるだけで使用できます。

## 注意点

- `c` キーは `/btw` の回答が表示されているときにのみ有効です。
- コピーされるのは「生の Markdown テキスト」です。レンダリングされた HTML ではありません。Markdown 非対応のツール（プレーンテキストのみのツールなど）に貼り付けると、Markdown の記法（`#`、`**` など）がそのまま表示されます。
- `/btw` は Claude Code の現在のコンテキスト（開いているファイルやコードの状態）を踏まえた回答を提供するコマンドです。
