## 原文（日本語に翻訳）

- `/ultrareview` を追加 — 並列マルチエージェント分析と批評を使用してクラウドで包括的なコードレビューを実行します。引数なしで現在のブランチをレビュー、または `/ultrareview <PR番号>` で特定の GitHub PR を取得してレビューします

## 原文（英語）

- Added `/ultrareview` for running comprehensive code review in the cloud using parallel multi-agent analysis and critique — invoke with no arguments to review your current branch, or `/ultrareview <PR#>` to fetch and review a specific GitHub PR

## 概要

`/ultrareview` は複数の AI エージェントが並列でコードを分析・批評するクラウドベースの包括的コードレビューツールです。現在のブランチの変更をレビューしたり、GitHub の特定の PR を指定してレビューを実行できます。通常のコードレビューより深い分析が可能です。

## 基本的な使い方

### 現在のブランチをレビュー

```
/ultrareview
```

引数なしで実行すると、現在のブランチの変更に対して包括的なレビューが実行されます。

### 特定の GitHub PR をレビュー

```
/ultrareview 1234
```

PR 番号を指定することで、GitHub 上の特定のプルリクエストをレビューします。

## 実践例

### 機能ブランチのレビュー前チェック

```bash
# feature ブランチに切り替え
git checkout feature/new-payment-system

# PR 作成前にレビュー
/ultrareview
```

マージ前の自己チェックとして使うと、レビュアーへの負担を減らせます。

### レビュー待ちの PR を事前確認

```
/ultrareview 567
```

レビュアーとして PR を確認する前に、AI による分析を先に受け取ることで効率的なレビューが可能です。

### CI/CD パイプラインとの組み合わせ

```bash
# CI が通った後、マージ前にウルトラレビュー
git push origin feature/my-feature
gh pr create --title "新機能: 決済システム"
# PR 番号を確認してレビュー
/ultrareview <PR番号>
```

## 注意点

- レビューはクラウドで実行されるため、コードがクラウドに送信されます。機密性の高いコードには注意が必要です
- GitHub PR をレビューする場合、リポジトリへのアクセス権限が必要です
- 並列マルチエージェント分析のため、通常のレビューより時間がかかる場合があります
- `/review` スキルと比較して、より包括的・詳細な分析を提供します
