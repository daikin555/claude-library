## 原文（日本語に翻訳）

大規模セッションの `--resume` でのメモリ使用量と起動レイテンシを改善しました。

## 原文（英語）

Improved `--resume` memory usage and startup latency on large sessions

## 概要

長い会話履歴を持つ大規模なセッションを `--resume` で再開する際のメモリ使用量と起動時間が改善されました。以前は大きなセッションを再開する際にメモリを大量に消費し、起動に時間がかかる問題がありましたが、内部の処理が最適化されてより効率的に動作するようになります。

## 基本的な使い方

大きなセッションを再開する際に、より速く、より少ないメモリで起動できるようになりました。

```bash
# 大規模なセッションを再開
$ claude --resume <large-session-id>
# 修正後: メモリ効率よく、より速く起動する
```

## 実践例

### ユースケース: 長い作業セッションの再開

数百回のやり取りがある長いセッションを再開する場合でも、スムーズに起動します。

```bash
# 長期にわたるプロジェクト作業セッションを再開
$ claude --resume proj-session-2024
# 修正後: 大量の会話履歴があっても効率的に読み込まれる
```

### ユースケース: 複数ファイルを扱った大規模セッション

多くのファイルを読み込み・編集した履歴を持つセッションの再開が速くなります。

```bash
# 大規模コードベースを扱ったセッションを再開
$ claude --resume codebase-analysis-session
# ツール使用履歴が多くても効率的に復元される
```

## 注意点

- `--resume` で再開できるセッション ID は `claude --list` や前回セッション終了時に確認できます
- 改善はセッションサイズが大きいほど効果が顕著です
- 具体的な改善量は会話の内容（テキスト量、ツール使用回数など）によって異なります
