## 原文（日本語訳）

エージェントビューを改善。各行には生のツール呼び出しテキストの代わりに、色付きの状態ワードと分類器（classifier）が生成したヘッドラインが表示されるようになった。また、peek（覗き見）パネルを開くと、ブロックされたセッションについては完全なステータス（Claudeが実際に尋ねている内容そのもの）を含む情報が表示されるようになった。

## 原文（英語）

Improved agent view: rows now show a colored state word and a classifier-written headline instead of raw tool call text, and the peek opens with full status including the exact ask for blocked sessions

## 概要

`claude agents` で開くエージェントビューは、バックグラウンドで走る複数のClaude Codeセッションを一覧管理するための画面です。今回の改善では、各行の表示がより読みやすくなりました。これまでは状態を色付きのアイコンで示し、行の一行サマリーには実行中のツール呼び出しの生テキストがそのまま表示されることがありましたが、今回からは「Working」「Needs input」「Completed」といった状態を色付きの単語（state word）として明示し、さらにその行が何をしているかを分類器モデルが自然文のヘッドラインとして書き起こすようになりました。加えて、行を選択してSpaceキーでpeekパネルを開いたとき、入力待ち（ブロック）状態のセッションについては、Claudeが実際に尋ねている質問文そのもの（exact ask）を含む完全なステータスが表示されるようになり、何を聞かれているのかを推測する必要がなくなりました。

## 基本的な使い方

```bash
claude agents
```

エージェントビューを開くと、バックグラウンドセッションが状態ごと（Working / Needs input / Idle / Completed / Failed など）にグループ化されて一覧表示されます。矢印キーで行を選択し、`Space` キーを押すとpeekパネルが開き、そのセッションの最新の状況を確認できます。

```text
↑ / ↓   行の選択
Space   peekパネルの開閉
Enter   セッションへのアタッチ（フル会話に入る）
```

ブロックされている（入力待ちの）セッションをpeekすると、これまでの要約的な表示ではなく、Claudeが尋ねている質問や許可確認の文言がそのまま表示されるため、内容を確認してすぐに返信するかどうかを判断できます。

## 実践例

### 複数のバックグラウンドエージェントを一覧監視する

複数のタスク（バグ修正、PRレビュー、フレーキーテストの調査など）を同時にバックグラウンドで走らせている場合、エージェントビューの各行を見るだけで「今何をしているか」を把握できます。

```text
claude agents
```

以前は行のサマリーに実行中のツール呼び出し（ファイルパスやコマンドの断片など）がそのまま表示されることがあり、内容を読み解くのに時間がかかりましたが、改善後は分類器が「テストを実行して失敗原因を調査中」のような自然な一文のヘッドラインを生成するため、一覧を流し見するだけで各セッションの進捗が把握しやすくなります。状態も色付きの単語としてはっきり表示されるため、Working / Needs input / Completed / Failed の区別が一目で分かります。

### ブロックされたセッションの内容をpeekで確認する

一部のセッションが権限確認や選択式の質問で止まっている場合、該当行を選択して `Space` を押すとpeekパネルが開きます。

```text
↑ / ↓ で該当セッションの行を選択
Space  でpeekパネルを開く
```

改善後は、そのセッションが実際にどんな確認・質問をしているか（exact ask）がそのままパネルに表示されるため、選択式の質問であれば数字キーで選択肢を選び、その他の場合は `Tab` で提案された返信を入力欄に呼び出して編集してから送信する、といった対応がスムーズに行えます。フルの会話（トランスクリプト）を開かなくても、ブロック理由を正確に把握したうえで判断できるのがポイントです。

## 注意点

- エージェントビューは研究プレビュー（research preview）段階の機能であり、インターフェースやキーボードショートカットは今後変更される可能性があります。
- 行のヘッドラインは分類器（Haiku系などの小型モデル）によって生成されるため、実行中は一定間隔（他のバージョンの挙動では最大15秒間隔、かつターン終了時）でしか更新されない場合があります。詳細な最新情報は公式ドキュメントの更新履歴を確認してください。
- peekパネルで表示される「exact ask」はセッションが実際に尋ねている質問文そのものであり、要約や言い換えではありません。多肢選択の質問では数字キーでの選択、それ以外のブロックでは `Tab` による提案返信の呼び出しが利用できます。
- 本記事作成時点（2026年7月8日）の公式ドキュメント（agent view ページ）の更新履歴には、バージョン2.1.203までの変更が記載されており、2.1.205時点でのこの改善（色付き状態ワード・分類器ヘッドライン・peekでの完全ステータス表示）についての詳細な記載は確認できませんでした。将来的にドキュメントが更新される可能性があるため、最新情報は公式ドキュメントを参照してください。
