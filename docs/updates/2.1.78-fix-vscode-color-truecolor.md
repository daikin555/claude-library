## 原文（日本語に翻訳）

トゥルーカラーサポートを通知しないVS Code / Cursor / code-serverターミナルで、Claudeのオレンジ色が薄く表示される問題を修正しました

## 原文（英語）

Fixed washed-out Claude orange color in VS Code/Cursor/code-server terminals that don't advertise truecolor support

## 概要

VS Code、Cursor、code-serverなどの統合ターミナルで、ターミナルがトゥルーカラー（24ビットカラー）対応を明示的に通知しない場合、Claude Codeのブランドカラーであるオレンジが薄く・くすんで表示されていた問題が修正されました。これらの環境でもClaudeのオレンジが正しい色で表示されるようになります。トゥルーカラー対応を自動検出できない環境でもフォールバック処理が適切に機能するよう改善されています。

## 基本的な使い方

この修正は自動的に適用されます。VS Code、Cursor、またはcode-serverの統合ターミナルでClaude Codeを起動するだけで、正しいオレンジ色が表示されます。

```bash
# VS Code統合ターミナルでClaude Codeを起動
claude

# ターミナルの色設定を確認したい場合
echo $COLORTERM   # truecolor または 24bit が理想的
echo $TERM        # xterm-256color など
```

## 実践例

### VS CodeのターミナルでClaude Codeを使用する場合

```bash
# VS Code統合ターミナル（COLORTERM が設定されていない場合でも）
# 修正後はClaudeのオレンジが正しく表示される
claude --version

# ターミナルのカラー設定を手動確認
tput colors   # 256色対応かどうか確認
```

### code-serverリモート環境での確認

```bash
# リモートのcode-serverにアクセスしてClaude Codeを使用
# トゥルーカラーが通知されない環境でも正しい色で表示される
claude chat
```

### Cursorエディタの統合ターミナルでの使用

```bash
# Cursor統合ターミナルを開き、Claude Codeを実行
# 修正前: オレンジが薄くくすんだ色で表示
# 修正後: 正しいAnthropicオレンジで表示
claude
```

## 注意点

- **影響環境**: VS Code、Cursor、code-serverの統合ターミナルが対象です
- **外部ターミナル**: iTerm2、macOSターミナル、Windowsターミナルなど通常のターミナルアプリは影響を受けていません
- **COLORTERM環境変数**: 問題が継続する場合、シェルの設定ファイルに `export COLORTERM=truecolor` を追加することで解決できる場合があります
- **ターミナルテーマ**: 一部のターミナルテーマがカラー表示に影響する場合がありますが、今回の修正でその影響が軽減されます
