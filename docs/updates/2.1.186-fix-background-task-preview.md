# バックグラウンドタスクのプレビューにエージェントのプランが読み込まれる前に生ツール名が点滅する問題を修正

## 原文（日本語に翻訳）
バックグラウンドタスクのプレビューに、エージェントのプランが読み込まれる前に生のツール名が点滅する問題を修正しました。

## 原文（英語）
Fixed background task previews flashing raw tool names before the agent's plan loaded

## 概要
バックグラウンドエージェントの起動直後に、タスクプレビュー表示に `Bash`, `Write`, `Edit` などの生のツール名が一瞬表示される（点滅する）問題がありました。この修正により、エージェントのプランが読み込まれるまでプレビューが適切に待機するようになりました。

## 基本的な使い方

この修正は自動的に適用されます。バックグラウンドエージェントを起動した際のプレビュー表示が改善されます。

## 実践例

### 影響を受けていたシナリオ
1. バックグラウンドエージェントを起動
2. タスクリストにプレビューが表示される
3. （以前）「Bash」「Write」などの生のツール名が一瞬点滅
4. （以前）その後エージェントのプランテキストに置き換わる

### 修正後
バックグラウンドエージェント起動後、プレビューにはエージェントのプランが適切に表示されます。生のツール名が点滅することはありません。

## 注意点
- UIの表示に関するバグ修正です
- 複数のバックグラウンドエージェントを並行実行する場合に特に効果的です
- 機能的な動作への影響はありません

## 関連情報
- バックグラウンドタスクの管理: `claude agents`
- バックグラウンドへの切り替え: Ctrl+B
