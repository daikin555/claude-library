## 原文（日本語に翻訳）

Cowork Dispatchでメッセージが配信されない問題を修正しました

## 原文（英語）

Fixed messages in Cowork Dispatch not getting delivered

## 概要

Cowork Dispatchは、スマートフォンやWebブラウザからデスクトップのClaude Codeエージェントにタスクを送信できる機能です。今回の修正では、モバイルやWeb経由でキューに入れたメッセージが実際にデスクトップで実行されないサイレントな配信失敗バグが解消されました。エラーメッセージも警告も表示されず、タスクが無言で失敗していた問題です。

## 基本的な使い方

最新バージョンへのアップデートで修正が適用されます。

```bash
npm install -g @anthropic-ai/claude-code@latest
```

バージョン確認:

```bash
claude --version
# 2.1.87 以降であることを確認
```

## 実践例

### モバイルからデスクトップにタスクを送信する

Cowork Dispatchを使ったリモートタスク送信の基本フロー：

1. デスクトップでClaude Code（またはCowork）を起動し、Dispatchセッションを開始する
2. iOSまたはAndroidのClaudeアプリ、もしくはclaude.aiのWebインターフェースを開く
3. 同じアカウントでサインインした状態でタスクを送信する

```
# 例：外出先からコードレビューを依頼する
「src/auth/login.tsのセキュリティレビューを行い、
問題があればPRを作成してください」
```

デスクトップが起動したまま席を外していても、Claudeがタスクを実行し、完了時にプッシュ通知で結果を受け取れます。

### 非同期ワークフロー

修正前は以下のようなシナリオで配信が無言で失敗していました：

```bash
# デスクトップ側でDispatchセッションを維持しておく
claude

# 別デバイス（スマホ）から送信したタスクが
# 2.1.87以前: デスクトップに届かず、エラーも出ない
# 2.1.87以降: 正常にデスクトップで受信・実行される
```

### マルチデバイス開発フロー

```
[スマートフォン]
  ↓ タスク送信「テストを全て実行してエラーを修正してください」
[Cowork Dispatch]
  ↓ メッセージ配信（今回の修正で信頼性向上）
[デスクトップのClaude Code]
  ↓ ローカルファイル・ツールを使ってタスク実行
[スマートフォンに完了通知]
```

## 注意点

- **デスクトップを起動したままにする必要がある**: Cowork Dispatchはデスクトップが稼働中であることが前提です。スリープや電源オフの状態ではタスクを受信できません
- **リサーチプレビュー段階**: 2026年3月時点でCowork DispatchはリサーチプレビューのためComplex Taskの成功率は約50%程度です
- **サイレント失敗の診断が困難**: 今回のバグはエラー表示なしに発生していたため、2.1.87未満を使っている場合はアップデートを強く推奨します
- **対応プラン**: ProプランおよびMaxプランのユーザーが利用できます
- **デスクトップUIのクラッシュ**: 関連する既知の問題として、Dispatchタブが応答をレンダリング時にクラッシュするケース（GitHub #40142）も報告されています。最新版へのアップデートで対処できます
