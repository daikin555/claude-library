## 原文（日本語に翻訳）

Write ツールが、承認前に IDE の差分ビューで提案内容を編集した場合にモデルへ通知するようになった

## 原文（英語）

Write tool now informs the model when you edit the proposed content in the IDE diff before accepting

## 概要

Claude が Write ツールでファイルへの変更を提案した際、IDE の差分ビューで内容を編集してから承認すると、Claude に「編集が行われた」ことが自動的に通知されるようになりました。これにより、Claude は自分の提案が修正されたことを把握でき、以降の会話でより適切な回答を生成できるようになります。

## 基本的な使い方

1. Claude がコードの変更を提案する
2. IDE の差分ビューで内容を確認する
3. 必要に応じて差分ビューで内容を手動編集する
4. 変更を承認する
5. → Claude が「あなたが内容を編集した」ことを自動的に把握する

```
[Claude がコード変更を提案]
      ↓
[IDE 差分ビューで確認・編集]
      ↓
[承認 (Accept)]
      ↓
[Claude に編集通知が送信される]
```

## 実践例

### Claude の提案を部分的に採用する場合

Claude が長いコードブロックを提案したが、一部だけ採用したい場合：

```bash
# Claude に実装を依頼
"この関数に型アノテーションを追加してください"

# Claude が変更を提案
# → IDE の差分ビューが開く

# 差分ビューで、追加された型アノテーションの一部を修正
# （例: Claude が推測した型が実際の型と異なる場合）

# 変更を承認

# Claude が通知を受け取り、修正内容を把握した上で回答
# "型アノテーションを追加しましたが、一部修正が入ったようです。
#  次の実装でも同様の型を使用しますか？"
```

### セキュリティ上の変更を加える場合

Claude が生成したコードに認証チェックを追加したい場合：

```bash
# Claude に API エンドポイントの実装を依頼

# 差分ビューで Claude の提案に認証コードを追加して承認

# Claude が通知を受け取り、以降の実装でも認証パターンを考慮
```

### チーム標準に合わせる場合

Claude の提案をチームのコーディング規約に合わせて調整：

```bash
# Claude にコードを生成させる

# 差分ビューでチーム規約に合わせてスタイルを修正
# （例: 命名規則、インデント、コメントスタイルの調整）

# 承認後、Claude は修正パターンを学習し次回から考慮
```

## 注意点

- この通知機能は IDE 拡張（VS Code、JetBrains）での差分ビュー使用時に機能します
- Claude は通知を受け取るだけで、編集内容の詳細を自動学習するわけではありません
- 編集した内容については、必要に応じて口頭でも説明すると Claude がより適切に対応できます
