## 原文（日本語に翻訳）

貼り付け操作の終了マーカーがターミナルにドロップされた後、キーボード入力が永続的に無応答になる問題を修正。

## 原文（英語）

Fixed keyboard input becoming permanently unresponsive after a paste operation whose end marker is dropped by the terminal

## 概要

一部のターミナルエミュレータでペースト操作の終了マーカーが正しく処理されない場合に、その後のキーボード入力が一切受け付けられなくなるバグが修正されました。特定のターミナル環境で発生していた問題で、修正後は貼り付け後もキーボードが正常に動作します。

## 基本的な使い方

修正後は特別な操作は不要です。Claude Code に長いテキストを貼り付けた後もキーボードが正常に動作するようになります。

```bash
claude
# 長いコードやテキストを貼り付け（Ctrl+V / Cmd+V）
# 修正前: その後のキー入力が無応答になる場合があった
# 修正後: 貼り付け後もキーボードが正常に動作する
```

## 実践例

### 長いコードの貼り付け後の作業

1. Claude Code のチャット欄を開く
2. 長いコードスニペットをクリップボードから貼り付ける（Ctrl+V / Cmd+V）
3. 修正前：キーボードが無応答になり、追加のコメントや質問を入力できなかった
4. 修正後：貼り付け後もキーボードが正常に動作し、続けて入力できる

## 注意点

- この問題は特定のターミナルエミュレータ（ブラケットペーストモードの終了マーカーを正しく送信しないもの）で発生していました。
- 発生した場合の回避策として、Claude Code を再起動する必要がありました。修正後はこの問題が解消されました。
