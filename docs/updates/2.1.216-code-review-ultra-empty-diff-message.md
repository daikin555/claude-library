## 原文（日本語訳）

`/code-review ultra` の差分が空の場合のメッセージを改善し、実際に使用されたベース参照（base ref）の名前を明示し、明示的なベースを指定するよう提案するようにしました。

## 原文（英語）

Improved `/code-review ultra` empty-diff message to name the exact base ref and suggest passing an explicit base

## 概要

`/code-review ultra`（クラウド上で実行される深いマルチエージェントレビュー、旧 `/ultrareview`）は、現在のブランチとリポジトリのデフォルトブランチとの差分をレビュー対象とします。しかし、ブランチ構成によっては比較対象のブランチとの差分が実質的に無い（空）と判定されることがあり、これまでは原因が分かりにくいメッセージになっていました。今回の改善で、エラーメッセージに実際に比較に使われたベース参照名が明示され、意図したブランチと異なる場合は明示的にベースを指定するよう提案されるようになりました。

## 基本的な使い方

```bash
/code-review ultra
```

差分が空と判定された場合、以下のようなメッセージが表示されます（イメージ）。

```
No changes found when comparing against base ref 'origin/main'.
If this isn't the branch you meant to compare against, pass an explicit base:
  /code-review ultra <base-branch>
```

このメッセージにより、「どのブランチと比較して空だったのか」がすぐに分かります。

## 実践例

### デフォルトブランチが意図と異なる場合

リポジトリのデフォルトブランチが `main` ではなく `develop` などである場合、明示的にベースを指定して再実行します。

```bash
# デフォルトのベース参照では差分なしと判定された場合
/code-review ultra develop
```

### フォーク・リモート構成でベースがずれている場合

`origin/main` ではなく別のリモートやタグを基準にしたい場合も、表示されたベース参照名を手がかりに指定し直せます。

```bash
# 表示されたベース参照（例: origin/main）が意図と違う場合
/code-review ultra upstream/main
```

### PR番号を指定してレビューする

ローカルブランチの比較ではなく、特定のPRに対してレビューしたい場合はPR番号を渡します。

```bash
# PRモードでレビュー（大きなリポジトリや特定PRを対象にしたい場合）
/code-review ultra 1234
```

## 注意点

- 差分が空になる典型的な原因は、ベースブランチの取り違えや、既にマージ済みの変更を対象にしていることです
- メッセージに表示されるベース参照名を確認し、意図したブランチと一致しているか必ずチェックしてください
- 明示的にベースを指定する場合は `/code-review ultra <base-branch>` の形式で、ブランチ名またはPR番号を渡せます
- ローカルの高速レビューであれば `/review` や `/code-review <level>` の利用も検討してください
