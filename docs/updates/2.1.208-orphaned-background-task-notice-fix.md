## 原文（日本語訳）

セッション再開時に繰り返し表示されていた「No completion record was found」という通知を修正した。孤立したバックグラウンドタスクは、これまでのように何度も表示されるのではなく、1つのサマリーにまとめて表示されるようになった。

## 原文（英語）

Fixed repeated "No completion record was found" notices on session resume — orphaned background tasks now collapse into a single summary

## 概要

Claude Codeのバックグラウンドタスク（バックグラウンドで実行するBashコマンドやサブエージェントなど）は、セッションのプロセスが停止・再起動された際、次に起動したプロセスがその作業を引き継ぐ仕組みになっています。しかし、何らかの理由でタスクの完了記録が見つからない「孤立した（orphaned）」バックグラウンドタスクが複数存在すると、セッションを再開するたびに「No completion record was found」という通知がタスクの数だけ繰り返し表示され、ノイズになっていました。特に `claude -p --resume` を1ターンごとに繰り返し呼び出すようなラッパー構成では、この通知が毎ターン蓄積して表示されることもありました。今回の修正により、複数の孤立タスクは1つにまとめられたサマリー通知として表示されるようになりました。

## 基本的な使い方

バックグラウンドタスクはセッション内で `Ctrl+B` を押すか、Claudeに「バックグラウンドで実行して」と依頼することで開始できます。

```text
このビルドをバックグラウンドで実行して
```

セッションを再開する際は通常どおり `--resume` を使います。

```bash
claude --resume <session-id>
```

## 実践例

### `claude -p` を繰り返し呼び出すラッパーでの利用

1ターンごとに `claude -p ... --resume <id>` を呼び出すような自動化スクリプトでは、前のターンで開始したバックグラウンドタスクが完了記録を残さないまま次のプロセスに引き継がれないケースがありました。

```bash
session_id=$(claude -p "ビルドをバックグラウンドで開始して" --output-format json | jq -r '.session_id')

# 後続ターンのたびに孤立タスクの通知が繰り返し出ていた
claude -p "進捗を教えて" --resume "$session_id"
claude -p "他のタスクも進めて" --resume "$session_id"
```

修正後は、こうした孤立タスクの通知が複数回ではなく1回のサマリーとしてまとめて表示されます。

### インタラクティブセッションでの再接続

ターミナルをうっかり閉じてしまい、`claude --resume` で同じセッションに戻ったときにも、バックグラウンドで動いていた複数のタスクが正しく引き継がれなかった場合に同様の通知が出ることがあります。

```bash
claude --resume
```

修正後は、引き継ぎに失敗した複数のタスクがあっても通知が1つに集約されるため、再開時の画面が読みやすくなります。

## 注意点

- 通知が1件に集約されるだけで、孤立したタスク自体の扱い（引き継げない場合は完了記録なしとして扱われる点）は変わりません。
- バックグラウンドタスク機能自体を無効化したい場合は、環境変数 `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1` を設定します。
- 通常のインタラクティブ利用では発生頻度が低く、`claude -p` をターンごとに繰り返し呼び出す自動化構成でより顕著に見られていた問題です。
