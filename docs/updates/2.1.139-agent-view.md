## 原文（日本語に翻訳）

エージェントビュー（リサーチプレビュー）を追加: 実行中・入力待ち・完了済みのすべてのClaude Codeセッションを一覧表示。`claude agents`で起動。詳細はhttps://code.claude.com/docs/en/agent-view

## 原文（英語）

Added agent view (Research Preview): a single list of every Claude Code session — running, blocked on you, or done. Run `claude agents` to get started. See https://code.claude.com/docs/en/agent-view

## 概要

`claude agents`コマンドで起動するエージェントビューは、実行中・入力待ち・完了済みのすべてのバックグラウンドClaude Codeセッションをダッシュボード形式で一覧表示するリサーチプレビュー機能です。各セッションにアタッチして返答し、再びデタッチすることができます。アタッチ時には、離れている間に行われた作業のリキャップが表示されます。

## 基本的な使い方

```bash
# エージェントビューを起動
claude agents
```

ビューには以下の状態でセッションが表示されます：
- **実行中**: 現在処理中のセッション
- **入力待ち**: ユーザーの応答を待っているセッション
- **完了済み**: タスクが完了したセッション

## 実践例

### バックグラウンドでタスクを実行して後から確認

```bash
# バックグラウンドでエージェントを起動
claude --bg "テストスイートをすべて実行して失敗したものをレポートして"

# 他の作業をしながら、後でエージェントビューを確認
claude agents
# → 実行中のセッションがリスト表示される
# → セッションを選択してアタッチ、リキャップを確認
```

### 複数の並行タスクを管理

```bash
# 複数のバックグラウンドタスクを起動
claude --bg "フロントエンドのビルドエラーを修正して"
claude --bg "データベースのマイグレーションスクリプトを作成して"
claude --bg "APIドキュメントを更新して"

# エージェントビューで一元管理
claude agents
# → 3つのセッションが状態別に表示される
```

### セッションへのアタッチ・デタッチ

エージェントビューでセッションを選択すると、そのセッションにアタッチして会話を続けられます。再びデタッチすることでバックグラウンドに戻し、Claude は作業を継続します。

## 注意点

- リサーチプレビュー段階の機能のため、仕様が変更される可能性があります
- バックグラウンドサービスが起動している必要があります（`claude --bg`で自動起動）
- 各セッションはターミナルにアタッチされていない独立したClaude Code会話として動作します
- アタッチ時には離席中の作業内容のリキャップが自動的に提供されます
