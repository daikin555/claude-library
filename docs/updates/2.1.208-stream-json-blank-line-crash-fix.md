## 原文（日本語訳）

Windowsスタイルの SDK ホストから送られる、空行や CRLF のみ・空白のみの行を stream-json 入力で受け取った際にセッションが強制終了してしまう不具合を修正した。

## 原文（英語）

Fixed stream-json input killing the session on blank CRLF or whitespace-only lines from Windows-style SDK hosts

## 概要

Claude CodeをSDK/ヘッドレスモードで動かす際、`--input-format stream-json` は標準入力からNDJSON（改行区切りJSON）形式でメッセージを受け取ります。Windows環境で動くSDKホストの中には、メッセージとメッセージの間に空行やCRLF（`\r\n`）のみの行、あるいは空白文字だけの行を送出するものがありました。これまでは、そうした行をJSONとしてパースしようとして例外が発生し、セッション全体が異常終了していました。今回の修正で、こうした無害な空行・空白行は単純に読み飛ばされるようになり、Windows由来のSDKホストと接続しているセッションが不意に落ちなくなりました。

## 基本的な使い方

`claude --input-format stream-json --output-format stream-json` でヘッドレスセッションを起動し、NDJSON形式のメッセージを標準入力に流し込みます。

```bash
claude -p --input-format stream-json --output-format stream-json --verbose
```

```bash
echo '{"type":"user","message":{"role":"user","content":"List all TODO comments"}}' | \
  claude --input-format stream-json --output-format stream-json --verbose
```

## 実践例

### Windows製SDKラッパーからパイプ入力する

Node.jsやPowerShellベースの独自ラッパーが標準入力へJSONメッセージを書き込む際、改行コードにCRLFを使っていたり、フラッシュのタイミングで余分な空行を挟んでしまうことがあります。

```powershell
# PowerShellから改行区切りJSONを送信する例
$msg = '{"type":"user","message":{"role":"user","content":"Summarize this file"}}'
$msg | claude --input-format stream-json --output-format stream-json --verbose
```

修正前はこの余分な空行やCRLFのみの行が原因でセッションが落ちることがありましたが、修正後は問題なく無視されて処理が継続します。

### CI/CDパイプラインでの多段ストリーミング

複数のClaude Codeプロセスをパイプでつなぐマルチフェーズパイプラインでも、中間で生成されるログ改行やツールの出力バッファリングにより空行が混入することがあります。

```bash
claude -p "Phase 1" --output-format stream-json --verbose | \
  claude -p --input-format stream-json --output-format stream-json --verbose --continue
```

こうした構成でも、空行・空白行が原因で後続プロセスがクラッシュしなくなりました。

## 注意点

- 影響を受けるのはあくまで空行・CRLFのみ・空白のみの行であり、不正なJSON構造を持つ行は引き続きエラーとして扱われます。
- Windows特有の問題として報告されていますが、修正自体はプラットフォームを問わずstream-json入力全般に適用されます。
- `stream-json` を使う場合は `--verbose` の付与が必須という既存の制約は変わりません。
