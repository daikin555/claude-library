## 原文（日本語に翻訳）

セッション中にMCPサーバーが再同期される際、プラグインが提供するMCPサーバーが強制終了されてしまう問題を修正しました

## 原文（英語）

Fixed plugin-provided MCP servers being torn down when MCP servers are re-synced mid-session

## 概要

Claude Code v2.1.210で修正された、プラグインとMCP連携に関するバグです。セッション実行中にMCPサーバーの設定が再同期（re-sync）されるタイミングで、以前はプラグインが提供しているMCPサーバー（プラグイン経由でインストールされたMCPサーバー）まで誤って一緒に停止（tear down）されてしまい、そのプラグインが提供するツールが使えなくなる問題がありました。今回の修正により、MCP再同期の際にプラグイン提供のMCPサーバーが不要に停止されることがなくなりました。

## 基本的な使い方

### 修正前の問題

```bash
# プラグイン経由でMCPサーバーを利用しているセッション
claude

> /mcp
# github, sentry などプラグイン提供のMCPサーバーが表示される
```

```text
1. セッション中に何らかの理由でMCPサーバーの再同期が発生
   （例: settings.jsonの変更検知、新規MCPサーバーの追加など）
2. 修正前: 再同期処理がプラグイン提供のMCPサーバーまで
   誤って停止（tear down）してしまう
3. プラグインのツール（例: mcp__plugin_github_...）が
   突然使えなくなる
4. ユーザーはセッションを再起動しない限り復旧できない
```

### 修正後の動作

```bash
claude

> /mcp
```

```text
1. MCPサーバーの再同期が発生
2. 修正後: 再同期の対象は変更があったMCPサーバーのみに限定され、
   プラグイン提供のMCPサーバーは稼働したまま維持される
3. プラグインのツールは引き続き利用可能
```

## 実践例

### 外部連携プラグイン（GitHub、Sentryなど）を使い続けるセッション

```bash
claude

> /plugin
# github プラグインをインストール（MCPサーバーを含む）

> issueを一覧して
# mcp__plugin_github_... のツールが呼び出される

# セッション中に別のMCPサーバー設定を追加・変更
> .mcp.json に新しいMCPサーバーを追加して

# 修正前: この再同期でgithubプラグインのMCPサーバーまで
#         停止してしまい、以降 issue 一覧などが使えなくなっていた
# 修正後: githubプラグインのMCPサーバーは影響を受けず稼働し続ける
```

### 複数プラグインを併用する場合

```bash
claude

> /mcp
# slack, linear, figma など複数プラグインのMCPサーバーが稼働中

# 何らかのトリガーでMCP再同期が発生した場合でも
# 修正後は各プラグインのMCPサーバーが個別に維持される
```

### プラグイン開発中のホットリロード確認

プラグイン開発者がローカルでMCPサーバーを含むプラグインを開発する際の確認例です。

```bash
claude --debug

> /plugin reload my-plugin

# --debug ログで、再同期時に他のプラグインのMCPサーバーが
# 巻き込まれて停止していないかを確認できる
```

## 注意点

- この修正は Claude Code v2.1.210 で実装されました
- 影響を受けていたのは、プラグイン経由でインストールされたMCPサーバー（`mcp__plugin_<plugin-name>_<server-name>__<tool-name>` の形式で呼び出されるツール）
- MCPサーバーの稼働状況は `/mcp` コマンドでいつでも確認できます
- プラグインのMCPサーバーが応答しないなど不審な挙動があれば、`/plugin reload` やセッション再起動で復旧できます
- 組織でMCPサーバーへのアクセスを管理している場合は、管理者ポリシーの設定も合わせて確認してください
