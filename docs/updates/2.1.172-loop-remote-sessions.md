## 原文（日本語に翻訳）

保留中のループがコンテナを生かし続けないリモートセッションでは `/loop` の宣伝を停止しました

## 原文（英語）

Stopped promoting `/loop` in remote sessions, where pending loops don't keep the container alive

## 概要

`/loop` コマンドがリモートセッション（クラウド環境で実行されるバックグラウンドエージェント等）では正しく機能しないため、リモートセッション内での `/loop` の宣伝・推奨が停止されました。リモートセッションでは、ループが保留中であってもコンテナが生き続けないため、スケジュールされたループが実行されません。

## 基本的な使い方

```
# リモートセッション内での動作
# 修正前: /loop が提案・宣伝されていた（ただし実際には機能しない）
# 修正後: リモートセッションでは /loop の宣伝が表示されない

# ローカルセッションでは引き続き /loop が利用可能
/loop 5m /code-review  # ローカルセッションでは正常に動作
```

## 実践例

### リモートセッションでの定期タスクの代替手段

```yaml
# リモートでの定期タスクには cron ジョブを使用する
# GitHub Actions scheduled workflow の例
name: Scheduled Code Review
on:
  schedule:
    - cron: '0 9 * * 1-5'  # 平日9時
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: claude --no-interactive "/code-review"
```

```bash
# または /schedule スキルを使用
/schedule
# クラウドエージェントのスケジュール設定を通じて定期実行が可能
```

## 注意点

- `/loop` はローカルセッションでは引き続き正常に使用できます
- リモートセッションでの定期実行には、外部スケジューラー（cron、CI/CD）や `/schedule` を使用してください
- この変更は動作を変えるものではなく、誤解を招くUI表示を修正したものです
