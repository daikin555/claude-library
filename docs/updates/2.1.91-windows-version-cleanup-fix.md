## 原文（日本語に翻訳）

Windowsのバージョンクリーンアップがアクティブバージョンのロールバックコピーを保護しない問題を修正しました。

## 原文（英語）

Fixed Windows version cleanup not protecting the active version's rollback copy

## 概要

Windowsでバージョンのクリーンアップ処理が実行される際、現在使用中のアクティブバージョンに紐づくロールバックコピーまで削除してしまうバグがありました。これにより、問題発生時に前のバージョンに戻すことができなくなる可能性がありました。今回の修正で、アクティブバージョンのロールバックコピーはクリーンアップの対象外となり、安全に保持されます。

## 基本的な使い方

この修正は自動的に適用されます。Windowsユーザーは特別な操作なしに恩恵を受けられます。

バージョン管理の確認：

```powershell
# 現在のバージョンを確認
claude --version

# バージョン一覧（ロールバック可能なバージョンを含む）
claude --list-versions
```

## 実践例

### アップデート後にロールバックが必要な場合

```powershell
# 最新バージョンへのアップデート
claude update

# 問題が発生した場合のロールバック
# 修正前: ロールバックコピーが削除されていて戻れない場合があった
# 修正後: ロールバックコピーが保持され、安全に戻せる
claude rollback
```

## 注意点

- この修正はWindows環境のみに影響します
- macOS・Linuxでは発生しない問題でした
- バージョンのロールバックには十分な空きディスク容量が必要です
- 複数世代前へのロールバックが必要な場合は、サポートにお問い合わせください
