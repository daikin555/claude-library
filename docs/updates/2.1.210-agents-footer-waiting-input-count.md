## 原文（日本語に翻訳）

エージェントフッターのヒントに、入力待ちのバックグラウンドエージェントの数が表示されるようになり、その数が変化した際には一瞬色が強調表示されるようになりました

## 原文（英語）

The agents footer hint now shows how many background agents are waiting on your input, with a brief color emphasis when the count changes

## 概要

通常のClaude Codeセッション内では、プロンプト入力欄のフッターにバックグラウンドで動いているエージェント（`/bg` などで起動したバックグラウンドタスクやサブエージェント）へのショートカットヒントが表示されます。v2.1.205以降、このヒントには入力待ちのエージェント数（例: `← 2 agents`）が表示されるようになっていましたが、v2.1.210ではその件数が変化したとき（新たにエージェントが入力待ちになった、または入力待ちが解消された等）に、ヒントの色が一瞬強調表示されるようになりました。複数のエージェントを並行して走らせているときに、どれかが自分の対応を待っている状態へ変化したことに気づきやすくなります。

## 基本的な使い方

```bash
claude

# 通常のフッター表示（誰も入力待ちでない場合）
# ← for agents

# バックグラウンドエージェントが2つ入力待ちになると
# ← 2 agents
# のように件数が表示され、表示が切り替わった瞬間だけ
# 色が一瞬強調される
```

`←`（左矢印キー）を押すとagent viewに切り替わり、待機中のエージェントの一覧を確認できます。

## 実践例

### 複数のバックグラウンドタスクを走らせながらメインセッションで作業する

```bash
claude

> 大きめのリファクタリングをバックグラウンドで進めて。
> 私はその間、別の小さいバグ修正を続ける

# バックグラウンドエージェントが権限確認や質問で
# 入力待ちになると、フッターが
# ← for agents → ← 1 agent
# に変化し、その瞬間だけ色が強調表示される
# → メイン作業に集中しつつ、見逃さずに気づける
```

### 複数エージェントの状態変化をリアルタイムに把握する

```bash
claude

# フッターのヒントはフォーカスがある間は約10秒ごとに更新され、
# フォーカスが戻った瞬間にも即座に更新される

# ← 1 agent   (1件目が入力待ち)
# ← 2 agents  (2件目も入力待ちに → 色が一瞬強調)
# ← 1 agent   (1件対応が完了 → 色が一瞬強調)
# ← for agents (全件解消)
```

### agent viewへ移動して待機中のエージェントに応答する

```bash
claude

# フッターに ← 2 agents と表示されたら
# 左矢印キーを押してagent viewへ

# agent view内で該当セッションを選択し、
# → キーでアタッチして質問に回答する
```

## 注意点

- この機能自体（フッターへの件数表示）はv2.1.205で導入されました。v2.1.210の変更は、件数が変化した際の「一瞬の色強調」の追加です
- `prefersReducedMotion` 設定を有効にしている場合、この色の強調表示は行われません
- スクリーンリーダーモードでは、このフッターヒント自体が非表示になります
- Amazon Bedrock、Google CloudのAgent Platform、Microsoft Foundry経由で利用している場合は、件数を伴わない `← for agents` の表示に固定されます
- 件数が99を超える場合は `99+` と表示されます
