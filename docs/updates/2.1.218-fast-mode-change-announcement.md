## 原文（日本語に翻訳）

`/config model=<x>` またはRemote Control経由でモデルを切り替えた結果としてfast modeが変化した場合に、アナウンスを表示するようにしました。

## 原文（英語）

Added an announcement when fast mode changes as a result of switching models via `/config model=<x>` or Remote Control

## 概要

fast modeは、Opusなど一部のモデルで利用できる高速レスポンスモードです。`/config model=<x>` コマンドやRemote Control（Webやモバイルからローカルセッションを操作する機能）でアクティブなモデルを切り替えると、切り替え先のモデルがfast modeに対応しているかどうかによって、fast modeのオン・オフが暗黙的に変化することがあります。これまではこの変化がユーザーに通知されず、気づかないうちにfast modeがオフになっていたり、逆に意図せずオンになっていたりすることがありました。v2.1.218では、モデル切り替えによってfast modeの状態が変わった場合にその旨をアナウンスするようになり、意図しない挙動の変化に気づきやすくなりました。

## 基本的な使い方

```bash
# fast modeがオンの状態でOpusを使っている場合
/fast
# → Fast mode: on

# fast mode非対応（あるいは挙動の異なる）モデルに切り替える
/config model=claude-opus-4-8

# 切り替えの結果fast modeがオフになった場合、
# 以下のようなアナウンスが表示される
# → "モデルの切り替えにより、fast modeがオフになりました"
```

## 実践例

### `/config model=<x>` でのモデル切り替え

```bash
/config model=claude-opus-4-8
# モデル切り替えの結果としてfast modeの状態が変化した場合、
# 切り替え完了メッセージと合わせてアナウンスが表示される
# 例: "モデルをclaude-opus-4-8に変更しました（fast modeはオフになりました）"
```

### Remote Control経由でのモデル切り替え

```bash
# ローカルでRemote Controlを有効化
claude --remote-control

# スマートフォンやWebブラウザから /model コマンドでモデルを変更
# → ローカルのターミナル側にも、
#   fast modeの状態変化を知らせるアナウンスが表示される
```

### fast mode非対応モデルへの切替時の挙動確認

```bash
/fast
# → Fast mode: on

/config model=claude-sonnet-4-5
# fast modeに対応していないモデルへ切り替えると、
# 自動的にfast modeがオフになり、その旨がアナウンスされる
# → "モデルをclaude-sonnet-4-5に変更しました。fast modeはこのモデルでは利用できないためオフになりました"
```

## 注意点

- アナウンスはあくまで通知のみであり、fast modeの状態自体を自動で元に戻すものではありません。必要であれば、モデル切り替え後に改めて `/fast` でfast modeを切り替えてください
- fast modeの対応状況やコストはモデルによって異なります。切り替え前にどのモデルがfast modeに対応しているか確認しておくと、意図しない挙動の変化を避けやすくなります
- Remote Control経由での操作でも、ローカルのターミナルセッションと状態は同期されるため、アナウンスはローカル側にも表示されます
