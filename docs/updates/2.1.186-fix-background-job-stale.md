# 返信後にclaude agentsのバックグラウンドジョブのステータスが古い「入力待ち」を表示し続ける問題を修正

## 原文（日本語に翻訳）
返信後に `claude agents` のバックグラウンドジョブのステータスが古い「入力待ち」メッセージを表示し続ける問題を修正しました。

## 原文（英語）
Fixed background job status in `claude agents` showing a stale "needs input" message after replying

## 概要
バックグラウンドエージェントが入力を求めており、ユーザーが返信した後も、`claude agents` のステータス表示に古い「入力待ち（needs input）」メッセージが残り続ける問題がありました。この修正により、返信後にステータスが正しく更新されるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。`claude agents` のステータス表示がリアルタイムで正確になります。

### 影響を受けていたシナリオ
1. バックグラウンドエージェントが入力待ち状態
2. `claude agents` で「needs input」と表示
3. エージェントに返信
4. （以前）ステータスが「needs input」のまま表示され続ける

### 修正後
返信後、エージェントのステータスが正確に更新されます（例：「実行中」「完了」など）。

## 注意点
- UIの表示に関するバグ修正です
- `claude agents` の表示とエージェントの実際の状態が正確に同期するようになります
