## 原文（日本語訳）

`/context` が、会話がコンテキストウィンドウを超過した場合に明示的な警告を表示するようになりました。また、失敗した `/compact` はエラーとして表示されるようになりました。

## 原文（英語）

`/context` now shows an explicit warning when the conversation exceeds the context window, and a failed `/compact` displays as an error

## 概要

`/context` はカテゴリ別のトークン使用量を可視化するコマンドですが、これまでは会話が実際にコンテキストウィンドウの上限を超えてしまっている状態でも、その旨がはっきりとは示されませんでした。今回のアップデートで、超過時には `/context` の出力に明示的な警告が表示されるようになりました。また、`/compact`（会話を要約して圧縮するコマンド）が何らかの理由で失敗した場合、これまで分かりにくかった失敗が、はっきりとしたエラーとして表示されるようになりました。

## 基本的な使い方

```bash
# コンテキスト使用量を確認
/context
```

会話がコンテキストウィンドウを超過している場合、出力の先頭付近に警告が表示されます（イメージ）。

```
⚠ Warning: This conversation exceeds the context window limit.
  Consider running /compact or /clear to continue.

Context usage:
  System prompt:     2.1K tokens ( 1%)
  Messages:        185.4K tokens (92%)
  ...
```

`/compact` が失敗した場合はエラーとして表示されます。

```bash
/compact
# → 失敗時: Error: /compact failed — <理由>
```

## 実践例

### 長時間セッションでの定期確認

長く続くセッションでは、定期的に `/context` を実行して超過警告が出ていないか確認します。

```bash
/context
# ⚠ Warning: exceeds the context window limit と表示されたら
/compact
```

### /compact失敗時の対処

`/compact` がエラーとして表示された場合は、原因を確認した上で `/clear` や新規セッションへの切り替えを検討します。

```bash
/compact
# Error: /compact failed — <詳細な理由>

# 対処例: 会話を完全にクリアしてやり直す
/clear
```

### CI・自動化スクリプトでの状態確認

自動化されたワークフローの中で、超過警告やcompact失敗をログとして検知し、セッションの健全性をモニタリングする際の参考にできます。

```bash
/context
# 出力に "exceeds the context window" が含まれていないかチェックする運用も可能
```

## 注意点

- 超過警告が表示された場合、そのまま会話を続けると古いメッセージが自動的に切り詰められたり、リクエストが失敗したりする可能性があります
- `/compact` が失敗する場合、要約対象のメッセージ量が多すぎる、あるいは一時的なAPIエラーなどが原因であることがあります
- 大幅にコンテキストが逼迫している場合は `/compact` よりも `/clear` で会話をリセットする方が確実な場合があります
- `/context` はあくまで現在のセッションのスナップショットであり、実行のたびに数値が変動します
