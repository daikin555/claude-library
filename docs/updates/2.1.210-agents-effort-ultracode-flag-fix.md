## 原文（日本語に翻訳）

`claude agents --effort ultracode` の指定が、ディスパッチされたセッションに届いていなかった不具合を修正しました。値が黙って（サイレントに）破棄されていました。

## 原文（英語）

Fixed `claude agents --effort ultracode` not reaching dispatched sessions; the value was silently dropped

## 概要

`claude agents` コマンドは、Agent View（複数のClaude Codeセッションを一覧・管理するための操作画面）を開き、`--effort` を含む各種フラグを指定してバックグラウンドセッションをディスパッチ（起動）できます。`--effort` には `low`/`medium`/`high`/`xhigh`/`max` に加え、xhigh相当の推論レベルと動的ワークフロー（ダイナミックワークフロー）によるサブエージェントの自動オーケストレーションを組み合わせた `ultracode` を指定できます。この修正前は、`claude agents --effort ultracode` としてディスパッチしても、`ultracode` の指定がディスパッチ先のセッションに正しく伝わらず、値が黙って破棄されて通常のeffortレベルで起動されてしまうバグがありました。v2.1.210でこの伝播処理が修正され、指定通り `ultracode` がディスパッチ先セッションに反映されるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
# ultracodeを指定してバックグラウンドセッションをディスパッチ
claude agents --effort ultracode --add-dir ./my-project

# 修正前:
# ディスパッチされたセッション内では ultracode が有効になっておらず、
# 通常のデフォルトeffortレベル（例: high）で実行されてしまう
# → 動的ワークフローによる大規模サブエージェントの並列オーケストレーションが
#   期待通りに発動しない
```

### 修正後の動作

```bash
claude agents --effort ultracode --add-dir ./my-project

# 修正後:
# ディスパッチされたセッションでも ultracode が正しく有効化される
# セッション内で /effort を確認すると ultracode になっている
> /effort
# → ultracode (xhigh相当の推論 + 動的ワークフローの自動判断)
```

## 実践例

### Agent Viewから大規模リファクタリングをultracodeでディスパッチ

複数ファイルにまたがる大規模な変更を、Agent View経由でultracode指定のセッションとして裏で走らせる。

```bash
# Agent Viewを開く（左矢印キーでも可）
claude agents

# Agent View内、あるいはCLIから直接ディスパッチ
claude agents --effort ultracode --add-dir ./backend --add-dir ./frontend

# 修正後:
# ディスパッチされたセッションはultracodeとして起動し、
# 必要に応じて動的ワークフロー（並列サブエージェント）を自動的に組み立てて実行する
```

### CI/バッチ処理としてultracodeセッションを複数ディスパッチ

夜間バッチなどで複数リポジトリに対してultracodeセッションを一括ディスパッチするケース。

```bash
for repo in service-a service-b service-c; do
  claude agents --effort ultracode --add-dir "./$repo" \
    --permission-mode acceptEdits
done

# 修正前: 各セッションでultracode指定が無視され、
#         通常のeffortで動作するため一貫性がなかった
# 修正後: すべてのディスパッチ先セッションで確実にultracodeが有効になる
```

### Agent Viewでディスパッチ済みセッションのeffort設定を確認

ディスパッチ後、実際にultracodeが反映されているかをAgent View上で確認する運用。

```bash
claude agents
# Agent View一覧から該当セッションを選択してアタッチ（→ または Enter）

> /effort
# ultracode になっていることを確認

# もし依然としてultracodeが反映されていない場合は
# Claude Codeのバージョンがv2.1.210以上であることを確認する
claude --version
```

## 注意点

- この修正はClaude Code v2.1.210で導入されました
- `ultracode` は `--effort` フラグの値として指定しますが、`CLAUDE_CODE_EFFORT_LEVEL` 環境変数では従来通り受け付けられません
- `ultracode` はセッション単位の設定であり、セッションを新しく開始するとリセットされます（永続化されません）
- ultracodeは通常のeffortレベルよりトークン消費量・実行時間が増える傾向があるため、バッチディスパッチする際はコストに注意してください
- 動的ワークフロー機能自体はClaude Code v2.1.154以降で利用可能です
