## 原文（日本語に翻訳）

スキル一覧の切り捨て警告が、影響するスキルの説明の数を表示するように改善されました。

## 原文（英語）

Improved the skill listing truncation warning to show how many skill descriptions are affected

## 概要

コンテキストウィンドウの制約でスキル一覧が切り捨てられる場合に表示される警告メッセージが改善されました。以前は単に「切り捨てが発生した」という通知のみでしたが、今回の改善で「何件のスキルの説明が省略されているか」が具体的に表示されます。これにより、どの程度スキルが利用不可能な状態かをすぐに把握できます。

## 基本的な使い方

スキル数が多い場合や、各スキルの説明が長い場合に切り捨てが発生します。警告は自動的に表示されます:

改善前の警告:
```
⚠️ Some skill descriptions were truncated due to context limits.
```

改善後の警告:
```
⚠️ 3 skill descriptions were truncated due to context limits.
```

## 実践例

### 多数のスキルを持つプロジェクトでの確認

スキルが多いプロジェクトで `/skills` または スラッシュコマンドリストを開いた際:

```
Available skills:
- deploy: Production deployment workflow
- test: Run test suite with coverage
- review: Code review checklist
... (2 more)

⚠️ 2 skill descriptions were truncated due to context limits.
Use /skills to see the full list.
```

### スキルの整理に活用

切り捨てが頻繁に発生している場合は、スキル説明を短くすることで解消できます:

```markdown
---
# スキルファイル例: .claude/skills/deploy.md
# 説明は簡潔に（切り捨てを避けるため）
description: "本番デプロイ"
---
```

### コンテキスト使用量の管理

```bash
# スキル使用量を確認
/context

# スキルの一覧確認
/skills
```

切り捨て数が大きい場合は、使用頻度の低いスキルを一時的に削除・無効化することを検討してください。

## 注意点

- スキルの切り捨てはコンテキストウィンドウの制約によるものです。スキル自体が壊れているわけではありません
- 切り捨てられたスキルは明示的なスキル名（`/deploy` など）で直接呼び出すことは引き続き可能です
- スキルの説明を簡潔にするか、スキル数を減らすことで切り捨てを防げます
- 数値が表示されることで、どの程度の整理が必要かを判断しやすくなりました
