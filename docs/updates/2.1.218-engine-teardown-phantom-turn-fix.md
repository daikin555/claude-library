## 原文（日本語に翻訳）

エンジン終了処理中に発生するレースコンディションを修正し、幽霊ターン（開始直後に破棄されるターン）が発生しないようにしました。また、クローズ後に送られた入力が一貫して拒否されるようにしました

## 原文（英語）

Fixed an engine teardown race that could start and abandon a phantom turn, and made input pushed after close consistently rejected

## 概要

Claude Code v2.1.218で修正された、セッション終了処理に関する内部バグです。以前は、エンジンの終了（テアダウン）処理が進行している最中にタイミングが重なると、新しいターン（リクエストと応答の1サイクル）が開始されてしまい、そのまま完了できずに破棄される「幽霊ターン」が発生することがありました。あわせて、セッションが閉じられた後に送信された入力の扱いが不安定で、状況によって挙動が異なっていました。修正後は、終了処理中のレースが解消されて幽霊ターンが発生しなくなり、クローズ後に送られた入力は常に一貫して拒否されるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- ユーザーが直接操作する必要はなく、内部的な安定性向上のための修正
- セッション終了時やタスクの中断時に発生しうる不整合なイベント処理を防止する
- クローズ後の入力送信は今後常に拒否される（挙動が予測可能になった）
