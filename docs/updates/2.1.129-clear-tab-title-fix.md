## 原文（日本語に翻訳）

会話後に `/clear` でターミナルのタブタイトルがリセットされない問題を修正しました。

## 原文（英語）

Fixed `/clear` not resetting the terminal tab title after a conversation

## 概要

`/clear` コマンドで会話をリセットした後、ターミナルのタブタイトルが以前の会話内容に基づいたタイトルのまま残ってしまう問題が修正されました。修正後は `/clear` を実行すると、タブタイトルもデフォルトの状態に戻ります。複数の会話を並行して管理している際のターミナルタブの識別が正確になります。

## 基本的な使い方

`/clear` コマンドを使用するだけで、タブタイトルも自動的にリセットされます。

```
# 会話中にタブタイトルが "my-project - Claude Code" などになっている状態で
/clear

# 修正後: タブタイトルもデフォルト（例: "Claude Code"）にリセットされる
```

## 実践例

### 複数のターミナルタブでの作業

複数のタブでClaude Codeを使用している場合、`/clear` 後のタブタイトルが正確になります。

```
# タブ1: プロジェクトAの会話中
タイトル: "ProjectA バグ修正 - Claude Code"

# /clear を実行
/clear

# 修正後のタイトル: "Claude Code"（デフォルトに戻る）
```

### 新しい会話を始める前のリセット

一連の作業を終えて新しいトピックの会話を始める際に、タイトルが正しく更新されます。

```
# 旧会話
> このコードをリファクタリングしてください
タイトル: "コードリファクタリング - Claude Code"

# クリア後
/clear
タイトル: "Claude Code"（リセット済み）

# 新しい会話
> 新しいAPIを設計してください
タイトル: "API設計 - Claude Code"（新しいタイトルに更新）
```

### /rename と組み合わせた使用

```
# 会話に名前をつけた場合
/rename "バグ調査セッション"

# 完了後にクリア
/clear
# タイトルがリセットされ、次の /rename まで新しいタイトルが付かない
```

## 注意点

- この修正はターミナルがタイトル変更をサポートしている場合のみ有効です（ほとんどのモダンなターミナルエミュレータが対応しています）。
- tmux などのターミナルマルチプレクサを使用している場合、タイトルの動作が異なる場合があります。
- `/compact` や他のコマンドとの組み合わせでのタイトル動作は個別に確認が必要です。
