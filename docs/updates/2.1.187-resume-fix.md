## 原文（日本語に翻訳）

`--resume` が「No conversation found」で失敗する問題を修正。元の `-p` 実行がモデルターンを生成しなかった場合に発生していた。

## 原文（英語）

Fixed `--resume` failing with "No conversation found" when the original `-p` run produced no model turns.

## 概要

`-p`（プロンプト）フラグを使用してClaude Codeを実行し、そのセッションでモデルが一度も応答しなかった場合（エラーや入力の問題などで即座に終了した場合）、その後 `--resume` でセッションを再開しようとすると「No conversation found」エラーが発生する問題が修正されました。これにより、中断されたセッションや不完全なセッションをより確実に再開できるようになりました。

## 基本的な使い方

`--resume` フラグを使用してセッションを再開します：

```bash
# セッションIDを指定して再開
claude --resume <session-id>

# または最後のセッションを再開
claude --resume
```

この修正により、以前は失敗していたケースでも正常にセッションを再開できます。

## 実践例

### -p フラグを使ったバッチ実行からの再開

```bash
# バッチ処理でセッションを開始（ただし何らかの理由で応答なしで終了）
claude -p "大量のファイルを処理してください" --output-format json
# → セッションIDが発行されるが、モデルターンが発生しなかった

# 修正後: このセッションを再開できるようになった
claude --resume <session-id>
```

### セッションIDの確認

```bash
# セッション一覧を確認
claude sessions list

# 特定のセッションを再開
claude --resume abc123def456
```

### CI/CDパイプラインでの活用

自動化されたパイプラインで `-p` を使用している場合、失敗したセッションを後から再開して診断できます：

```bash
# CI環境でのバッチ実行
SESSION_ID=$(claude -p "コードをレビューしてください" --output-format json | jq -r '.session_id')

# 問題が発生した場合、後から再開して調査
claude --resume $SESSION_ID
```

## 注意点

- この修正は `-p` フラグ（非インタラクティブモード）を使用したセッションに特に関連します
- セッションIDは通常 `~/.claude/sessions/` ディレクトリに保存されます
- モデルターンが存在しないセッションでは、再開後に最初から入力を提供する必要があります
- `--resume` はセッションの状態（ファイルの変更、コンテキストなど）を復元しますが、会話履歴が空の場合は新規会話として扱われます
