## 原文（日本語に翻訳）

`/fork` を `/branch` にリネームしました（`/fork` はエイリアスとして引き続き動作します）

## 原文（英語）

Renamed `/fork` to `/branch` (`/fork` still works as an alias)

## 概要

セッション分岐機能のコマンドが `/fork` から `/branch` に変更されました。これはGitのブランチ概念に合わせた、より直感的な命名への移行です。既存のワークフローを壊さないよう、`/fork` は引き続きエイリアスとして利用でき、従来通り動作します。

## 基本的な使い方

```bash
# 新しいコマンド名（推奨）
/branch

# 従来のコマンド名（引き続き動作します）
/fork
```

`/branch` コマンドは現在の会話セッションを分岐させ、別の方向で探索できる新しいセッションを作成します。元のセッションはそのまま保持されます。

## 実践例

### 実装アプローチの比較探索

異なる実装方法を並行して試したい場合に活用できます。

```bash
# 現在の状態でセッションを分岐
/branch

# 分岐したセッションで別アプローチを試す
"Reactを使って実装してください"

# 元のセッションに戻って別アプローチを試す
"Vue.jsを使って実装してください"
```

### 危険な変更を試す前にバックアップ

```bash
# 大きな変更を加える前にセッションを分岐
/branch

# 分岐先でリスクのある変更を試す
"データベースのスキーマを大幅に変更してください"

# 問題があれば元のセッションに戻れる
```

### 複数の設計案を検討

```bash
# アーキテクチャ設計の議論中にブランチを作成
/branch

# 各ブランチでそれぞれの設計案を深掘り
"マイクロサービスアーキテクチャで詳細設計してください"
```

## 注意点

- **後方互換性**: `/fork` は削除されておらず、エイリアスとして継続して使用可能です。既存のメモや習慣に合わせて使い続けても問題ありません
- **命名の意図**: `/branch` という名前はGitのブランチに似た概念を表しており、「会話の分岐点を作る」という機能をより分かりやすく表現しています
- **セッション管理**: 分岐したセッションは独立して管理されます。`/sessions` コマンドで確認できます
