## 原文（日本語訳）

Prometheusメトリクスエンドポイント（`OTEL_METRICS_EXPORTER=prometheus`）が不正な `# UNIT` 行を出力していた問題を修正しました。

## 原文（英語）

Fixed the Prometheus metrics endpoint (`OTEL_METRICS_EXPORTER=prometheus`) emitting invalid `# UNIT` lines

## 概要

Claude Codeは `OTEL_METRICS_EXPORTER=prometheus` を設定することで、Prometheusがスクレイピングできる形式でメトリクスを直接公開できます。今回の修正では、このエンドポイントが出力する `# UNIT` メタデータ行がPrometheusのテキスト形式（exposition format）の仕様に沿わない不正な内容になっていた不具合を修正しました。これにより、Prometheus本体や `promtool` などの検証ツール、Grafana連携などでのパースエラーやスクレイピング失敗が解消されます。

## 基本的な使い方

この修正は挙動の自動修正であり、追加の設定は不要です。既存の設定のままアップデートするだけで、正しい形式のメトリクスが出力されるようになります。

```bash
export CLAUDE_CODE_ENABLE_TELEMETRY=1
export OTEL_METRICS_EXPORTER=prometheus
```

Claude Codeを起動すると、上記設定に基づきPrometheus形式のメトリクスがエンドポイント経由で公開されます。

## 実践例

### Prometheusでのスクレイピング設定を確認する

`prometheus.yml` に以下のようなスクレイプ設定を追加し、Claude Codeのメトリクスエンドポイントを監視対象に加えます。

```yaml
scrape_configs:
  - job_name: "claude-code"
    static_configs:
      - targets: ["localhost:9464"]
```

修正前は `# UNIT` 行の不正な記述によりスクレイピング自体やパースが失敗するケースがありましたが、修正後は正常に取り込まれるようになります。

### promtoolでメトリクス出力を検証する

アップグレード後に問題が解消されているかを手元で確認する場合は、`promtool` でエンドポイントの出力を検証できます。

```bash
curl -s http://localhost:9464/metrics | promtool check metrics
```

修正前のバージョンではここで `# UNIT` 行に関するフォーマットエラーが出ていましたが、修正後はエラーなく検証を通過します。

### Grafanaダッシュボードでの可視化

Prometheus経由で取り込んだClaude Codeのメトリクスは、Grafanaのダッシュボードでトークン使用量やコストの推移として可視化できます。パースエラーが解消されたことで、メトリクスの欠落なく継続的なモニタリングが可能になります。

## 注意点

- この修正はエンドポイントの出力フォーマットに関するバグ修正であり、メトリクスの項目や意味自体は変わりません
- `OTEL_METRICS_EXPORTER` には `otlp` / `prometheus` / `console` / `none` を指定できます
- Prometheus形式を利用する場合は `CLAUDE_CODE_ENABLE_TELEMETRY=1` の設定も必要です
- 古いバージョンのClaude Codeで取り込んだメトリクスに欠損があった場合、アップグレード後に再度スクレイピング設定を見直すことを推奨します
