## 原文（日本語に翻訳）

サブエージェントが実行中にプロンプト下のエージェントパネルが非表示になる問題を修正しました（2.1.122 でのリグレッション）。

## 原文（英語）

Fixed agent panel below the prompt being hidden when subagents are running (regression in 2.1.122)

## 概要

バージョン 2.1.122 で発生したリグレッション（後退バグ）が修正されました。サブエージェントが実行中の際、プロンプト入力欄の下に表示されるエージェントパネルが非表示になってしまう問題です。修正により、サブエージェントの動作状況をリアルタイムで確認できるパネルが常時表示されるようになります。

## 基本的な使い方

この修正は自動的に適用されます。サブエージェントを使用する際、エージェントパネルが正しく表示されるようになります。

```
# サブエージェントを使用するタスクを実行
> このリポジトリのすべてのテストを実行して結果をまとめてください

# エージェントパネル（修正後は正しく表示される）:
┌─ エージェント ────────────────────┐
│ [実行中] test-runner-agent         │
│ [待機中] reporter-agent            │
└───────────────────────────────────┘
```

## 実践例

### 並行サブエージェントのモニタリング

```
# 複数のサブエージェントが並行して動作するタスク
> 3つの異なるマイクロサービスのコードレビューを並行して実行してください

# 修正後: エージェントパネルで各エージェントの状況を確認可能
┌─ エージェント ────────────────────────┐
│ [実行中] service-a-reviewer          │
│ [実行中] service-b-reviewer          │
│ [待機中] service-c-reviewer          │
│ [完了]   orchestrator                │
└───────────────────────────────────────┘
```

### 長時間タスクでのエージェント追跡

```
# 時間のかかるタスクでのエージェント状況確認
> このコードベースを解析してドキュメントを生成してください

# エージェントパネルが常時表示され、進捗を把握できる
# 2.1.122〜2.1.128: パネルが消えてしまい状況不明
# 2.1.129: パネルが正常に表示され続ける
```

### バージョン確認方法

```bash
# 現在のバージョンを確認
claude --version

# 2.1.129 以降を使用していることを確認
# 2.1.122〜2.1.128 の場合はアップデートを推奨
```

## 注意点

- このバグは 2.1.122 で導入され、2.1.129 で修正されました。2.1.122〜2.1.128 を使用している場合は、アップデートすることを強く推奨します。
- エージェントパネルはサブエージェントが実行中の場合のみ表示されます。単一エージェントの場合は表示されません。
- パネルの表示形式はターミナルの幅や設定によって異なる場合があります。
