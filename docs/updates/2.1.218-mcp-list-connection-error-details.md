## 原文（日本語に翻訳）

MCPサーバーへの接続に失敗した際に、`claude mcp list` と `/mcp` にHTTPステータスとエラーテキストを表示するようにしました。また、MCP設定値に見えない先頭・末尾の空白が含まれている場合の警告も追加しました。

## 原文（英語）

Added HTTP status and error text to `claude mcp list` and `/mcp` when a server fails to connect, and a warning for MCP config values with hidden leading or trailing whitespace

## 概要

これまで、MCPサーバーへの接続に失敗した場合、`claude mcp list` や `/mcp` コマンドでは「接続失敗」といった簡潔な情報しか表示されず、原因の特定にはさらなる調査が必要でした。v2.1.218からは、接続失敗時にHTTPステータスコードとエラーメッセージの本文が表示されるようになり、認証エラーなのかサーバー側の問題なのかをその場で判断しやすくなりました。あわせて、URLやトークンなどのMCP設定値に、コピー＆ペースト時などに紛れ込みやすい見えない先頭・末尾の空白文字が含まれている場合に警告が表示されるようになり、気づきにくい設定ミスの早期発見に役立ちます。

## 基本的な使い方

登録済みのMCPサーバーの状態はCLIまたはインタラクティブモードのどちらからでも確認できます。

```bash
# CLIから確認
claude mcp list
```

```
# インタラクティブモード（Claude Code内）から確認
/mcp
```

サーバーへの接続に失敗している場合、以前は次のような簡素な表示でした。

```
weather-api: ✗ failed to connect
```

修正後は、HTTPステータスとエラーテキストが併記されます（イメージ）。

```
weather-api: ✗ failed to connect (HTTP 401 Unauthorized: "invalid API key")
```

## 実践例

### 認証エラーの原因特定

APIキーやトークンが誤っている場合、HTTPステータス401や403とともにエラーテキストが表示されるため、認証情報の問題だとすぐに分かります。

```bash
claude mcp list
```

```
secure-api: ✗ failed to connect (HTTP 401 Unauthorized: "invalid token")
```

トークンを再発行して設定を更新すれば解決できることが、詳細メッセージから判断できます。

### サーバー側の一時的な障害の切り分け

HTTPステータスが5xx系の場合は、設定側ではなくサーバー側に問題がある可能性が高いと判断できます。

```bash
/mcp
```

```
internal-tool: ✗ failed to connect (HTTP 503 Service Unavailable: "server temporarily overloaded")
```

この場合はしばらく時間をおいて再接続を試すなど、対応方針を切り分けやすくなります。

### 空白文字混入の警告に気づく

URLやヘッダーの値をコピー＆ペーストした際に、末尾に余分な空白やタブが紛れ込んでいることがあります。

```bash
claude mcp list
```

```
weather-api: ⚠ warning: config value for "url" has leading or trailing whitespace
```

このような警告が出た場合は、`.mcp.json` や `claude mcp add` で設定した値を再確認し、余分な空白を取り除くことで接続できるようになることがあります。

```json
{
  "mcpServers": {
    "weather-api": {
      "type": "http",
      "url": "https://api.weather.com/mcp "
    }
  }
}
```

## 注意点

- HTTPステータスとエラーテキストが表示されるのは、サーバーがHTTP/SSEなどのトランスポートで応答した場合が中心です。ネットワーク自体に到達できない場合はステータスコードが取得できないこともあります。
- 空白文字の警告は先頭・末尾の空白のみを対象としており、値の中間に含まれる問題までは検出しません。
- この改善はエラーの可視性を高めるものであり、接続そのものの成功率を上げる機能ではありません。原因の特定後は、設定の修正やサーバー側の対応が別途必要です。
- Claude Code v2.1.218以降で利用できます。
