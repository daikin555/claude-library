# Anthropic Consoleで認証：--consoleフラグによるAPIビリング対応

## 原文（日本語に翻訳）

`claude auth login` に `--console` フラグを追加 — Anthropic Console（APIビリング）認証に対応

## 原文（英語）

Added `--console` flag to `claude auth login` for Anthropic Console (API billing) authentication

## 概要

`claude auth login` コマンドに新しい `--console` フラグが追加されました。これにより、Anthropic ConsoleのAPIキーを使ったビリングベースの認証がよりスムーズに行えるようになります。Claude Proのサブスクリプションではなく、APIの従量課金で利用したいユーザー向けの機能です。

## 基本的な使い方

```bash
claude auth login --console
```

このコマンドを実行すると、Anthropic ConsoleのAPIキー認証フローが開始されます。

## 実践例

### 新規セットアップ時のAPI認証

```bash
# Anthropic Consoleアカウントでログイン
claude auth login --console

# 現在の認証状態を確認
claude auth status
```

### 複数アカウントの切り替え

```bash
# Consoleアカウント（APIビリング）でログイン
claude auth login --console

# 通常のAnthropicアカウントに切り替える場合
claude auth login
```

### CIやサーバー環境での利用

APIキーによる認証はヘッドレス環境に適しています。環境変数での設定も引き続き利用可能です：

```bash
# 環境変数によるAPIキー設定（CI/CD環境向け）
export ANTHROPIC_API_KEY=sk-ant-xxxxx
claude -p "タスクを実行してください"
```

## 注意点

- `--console` フラグはAnthropicのConsoleダッシュボード（console.anthropic.com）でAPIキーを作成・管理しているユーザー向けです
- APIビリングはトークン使用量に基づいた従量課金制のため、使用量の監視が重要です
- 通常の `claude auth login`（フラグなし）はClaude ProなどのサブスクリプションユーザーのAnthropicアカウント認証に使用します
- 既存のセッションがある場合、再ログインすると以前の認証情報が上書きされます
