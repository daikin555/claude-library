# 修正：Agentツールが自動モードで安全分類器のコンテキスト超過時に権限を要求するバグ

## 原文（日本語に翻訳）

自動モードで安全分類器のトランスクリプトがコンテキストウィンドウを超えた際にAgentツールが権限を要求していた問題を修正した。

## 原文（英語）

Fixed Agent tool prompting for permission in auto mode when the safety classifier's transcript exceeded its context window.

## 概要

Claude Codeの自動モード（`--auto-approve` 等）でAgentツールを使用中、安全分類器が処理するトランスクリプトがコンテキストウィンドウを超えた場合に、誤ってユーザーへの権限確認が求められていたバグが修正されました。長時間の自動化セッションや複雑なタスクで問題が起きやすく、自動処理が中断されるという不便を引き起こしていました。

## 基本的な使い方

### 修正後の自動モードでの動作

```bash
# 自動モードでのClaude Code実行
claude --auto-approve

# 修正前: 長いセッションで突然権限確認が出てブロックされていた
# 修正後: 安全分類器のコンテキスト超過があっても自動モードが継続される
```

### CI/CDでの影響

```yaml
# GitHub Actionsでの自動レビュー処理
- name: Automated Code Review
  run: |
    claude --auto-approve --print "コードをレビューしてください"
    # 長い会話履歴があっても権限確認でブロックされなくなった
```

## 注意点

- **安全機能は維持**: この修正は自動モードでの不正な権限要求を修正するもので、セキュリティの安全分類器自体は引き続き正常に機能します。
- **長時間セッションで特に効果的**: トランスクリプトが長くなる長時間の自動化セッションで特に恩恵を受けます。
