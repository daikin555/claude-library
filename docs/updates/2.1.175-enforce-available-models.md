## 原文（日本語に翻訳）

`enforceAvailableModels` 管理設定（managed setting）を追加しました。有効にすると、`availableModels` のallowlistがDefaultモデルにも適用されるようになります（許可されていないモデルに解決されるはずだったDefaultは、allowlistの先頭の許可モデルにフォールバックします）。また、ユーザー設定やプロジェクト設定で、管理された `availableModels` リストを拡張（広げる）できなくなります。

## 原文（英語）

Added `enforceAvailableModels` managed setting — when enabled, the `availableModels` allowlist also constrains the Default model (a Default that would resolve to a disallowed model now falls back to the first allowed model), and user or project settings can no longer widen a managed `availableModels` list

## 概要

`enforceAvailableModels` は、組織の管理者が強制する管理設定（managed settings / policy settings）として追加された新しいオプションです。これまで `availableModels` のallowlistはモデルピッカーに表示されるモデルを制限していましたが、「Default」（サブスクリプション階層に応じた実行時のデフォルトモデル）はallowlistの対象外で、許可外のモデルに解決されることがありました。この設定を有効にすると、Defaultもallowlistの制約を受け、さらにユーザー設定やプロジェクト設定で管理者が定めたリストを広げることができなくなります。

## 基本的な使い方

管理設定（managed settings）またはポリシー設定で、`enforceAvailableModels` を `true` にし、空でない `availableModels` リストと組み合わせて指定します。

```json
// 管理設定 (managed settings) の例
{
  "availableModels": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5-20251001"
  ],
  "enforceAvailableModels": true
}
```

この設定により、`/model` で「Default」を選んだ場合でも、Defaultが許可外のモデル（例: Opus）に解決されるときは、allowlistの先頭にある `claude-sonnet-4-6` にフォールバックします。

## 実践例

### Defaultモデルもallowlistの範囲に収める

```json
// コスト管理のためOpusを許可せず、Defaultも確実に制約したい
{
  "availableModels": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5-20251001"
  ],
  "enforceAvailableModels": true
}
```

`enforceAvailableModels` を有効にしないと、ユーザーが「Default」を選んだ際に許可外のモデルが使われる余地がありました。有効にすることで、Defaultも `availableModels` の範囲内に収まり、許可外モデルに解決される場合はリスト先頭のモデルへフォールバックします。

### ユーザー/プロジェクト設定による上書き（拡張）を防ぐ

```json
// 管理設定 (managed settings)
{
  "availableModels": ["claude-sonnet-4-6", "claude-haiku-4-5-20251001"],
  "enforceAvailableModels": true
}
```

```json
// ユーザー設定やプロジェクト設定 (.claude/settings.json) で
// 別のモデルを足そうとしても...
{
  "availableModels": ["claude-opus-4-8"]  // 管理リストを広げることはできない
}
```

`enforceAvailableModels` が有効な場合、ユーザー設定やプロジェクト設定で管理された `availableModels` リストを拡張（widen）することはできません。組織ポリシーが確実に優先されます。

## 注意点

- これは組織の管理者が強制する**管理設定（managed settings）/ ポリシー設定**です。個人のユーザー設定やプロジェクト設定で自由に有効化する用途とは異なります。
- 効果を発揮させるには、`enforceAvailableModels: true` と**空でない `availableModels` リスト**の両方を指定する必要があります。
- 有効時、Defaultが許可外モデルに解決される場合は `availableModels` の**先頭（first allowed model）**にフォールバックします。allowlistの並び順が実質的なデフォルトを決めることになります。
- 有効時は、ユーザー設定・プロジェクト設定で管理された `availableModels` リストを広げる（モデルを追加する）ことはできなくなります。
- この機能には Claude Code v2.1.175 以降が必要です。
