## 原文（日本語に翻訳）

headlessモードおよびSDKセッションにおいて、compaction（要約）後にfork-sessionの系譜（lineage）情報が失われる問題を修正しました

## 原文（英語）

Fixed fork-session lineage being lost after compaction in headless and SDK sessions

## 概要

セッションをフォーク（fork）すると、元の会話から分岐した新しいセッションが作られ、内部的には「どのセッションからフォークされたか」という系譜（lineage）情報が記録されます。以前は、headlessモードやClaude Agent SDK経由で利用しているセッションで、会話が長くなりコンパクション（古いメッセージを要約してコンテキストを節約する処理）が実行されると、このフォーク元セッションの系譜情報が失われてしまうことがありました。この修正により、compactionが行われた後もフォークの系譜情報が正しく保持されるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 通常の対話的なCLIセッションよりも、headlessモードやSDK経由でフォークとcompactionを組み合わせて使うワークフローに影響していた問題
- ユーザー側の追加操作は不要で、アップデートにより自動的に系譜情報が保持されるようになる
- セッションの系譜情報に依存する分析や監査ツールを利用している場合、修正後はより正確な履歴が得られるようになる
