## 原文（日本語に翻訳）

`claude-opus-4-8` のようなバージョン固有のIDを使って `availableModels` の許可リストを設定すると、`/model` ピッカーのOpusとSonnet 1M行が非表示になる問題を修正しました

## 原文（英語）

Fixed `availableModels` allowlists hiding the `/model` picker's Opus and Sonnet 1M rows when entries use version-specific IDs like `claude-opus-4-8`

## 概要

`availableModels` にバージョン固有のモデルID（例: `claude-opus-4-8`）を指定した場合、`/model` ピッカーでOpusやSonnetの1Mコンテキスト行（`claude-opus-4-8[1M]` など）が誤って非表示になる問題が修正されました。許可リストに含まれているモデルの1Mバリアントは正しく表示されるようになります。

## 基本的な使い方

```json
// .claude/settings.json
{
  "availableModels": [
    "claude-opus-4-8",      // これを設定すると...
    "claude-sonnet-4-6"
  ]
}
```

```
/model
# 修正前: 1Mコンテキスト行が非表示
# claude-opus-4-8          ← 表示
# claude-opus-4-8[1M]      ← 非表示だった（バグ）
# claude-sonnet-4-6        ← 表示
# claude-sonnet-4-6[1M]    ← 非表示だった（バグ）

# 修正後: 1Mコンテキスト行も正しく表示
# claude-opus-4-8          ← 表示
# claude-opus-4-8[1M]      ← 表示（修正後）
```

## 実践例

### 1Mコンテキストモードを利用したい場合

```json
// バージョン固有IDで設定しても1Mオプションが使えるようになった
{
  "availableModels": [
    "claude-opus-4-8",
    "claude-sonnet-4-6"
  ]
}
```

```
/model
# Opus 1Mを選択できるようになった
```

## 注意点

- この修正以前は `claude-opus-4-8` と `claude-opus-4-8[1M]` を別々に列挙する必要がありました
- 修正後は基本モデルIDを指定するだけで、そのモデルの1Mバリアントも自動的に有効になります
