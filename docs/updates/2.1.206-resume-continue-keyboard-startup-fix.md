## 原文（日本語訳）

`claude --resume` と `--continue` が起動時にキーボード入力に反応しない不具合を修正した。

## 原文（英語）

Fixed `claude --resume` and `--continue` not responding to keyboard input on startup

## 概要

`claude --resume`（セッション選択ピッカーから過去の会話を再開）や `claude --continue`（直前の会話を自動で再開）を実行した直後、キーボード入力が反応しない不具合がありました。起動直後にセッション一覧を矢印キーで選んだり、続けてすぐにメッセージを入力しようとしても操作を受け付けない状態になっていましたが、今回のアップデートでこの不具合が修正されました。

## 基本的な使い方

これまで通りのコマンドで問題なく利用できます。

```bash
claude --resume
```

```bash
claude --continue
```

短縮形も利用できます。

```bash
claude -r
```

```bash
claude -c
```

## 実践例

### 会話ピッカーからセッションを選んで即座に操作を続ける

`claude --resume` を実行するとセッション一覧が表示されますが、修正前は起動直後に矢印キーやEnterキーを押しても反応しないことがあり、少し待ってから操作し直す必要がありました。修正後は起動直後からキー操作が正しく反映されるため、待たずにセッションを選択できます。

```bash
claude --resume
```

### 直前のセッションを再開してすぐに指示を続ける

`claude --continue` に加えて、再開と同時にプロンプトを渡す使い方でも起動直後の反応性が改善されています。

```bash
claude -c "テストが通るか確認して"
```

### スクリプトや自動化から非対話的に再開する場合

`-p` と組み合わせた非対話実行でも起動シーケンスは共通の基盤を使っているため、今回の修正はインタラクティブなキー入力待ちが解消されるという形で対話的な利用シーンに直接効果があります。

```bash
claude -c -p "型エラーがないか確認して"
```

## 注意点

- 関連する不具合として、同じ2.1.206では「Windowsでセットアップ用のプロンプトが `claude --resume` の前に表示された際、agents view内でキーボード入力が無視される」不具合も別途修正されています。プラットフォーム固有の症状が残る場合はそちらも参照してください。
- 症状が再発する場合は、ターミナルエミュレータ側のキー入力設定や、Claude Codeのバージョンが2.1.206以降になっているかを確認してください。
- 公式ドキュメントには本項目（起動直後のキー入力不具合の詳細）についての明記が本記事作成時点では見当たらなかったため、changelogの記載内容をもとに記述しています。最新情報は公式ドキュメントを参照してください。
