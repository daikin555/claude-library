## 原文（日本語訳）

「Needs attention」または「Favorites」に表示されているアイテムが `/plugin` のInstalledタブで同じアイテムが2回表示される問題を修正しました。

## 原文（英語）

Fixed `/plugin` Installed tab showing the same item twice when it appears under Needs attention or Favorites

## 概要

Claude Codeの `/plugin` コマンドを開いた際、「Installed（インストール済み）」タブで一部のプラグインが2回表示される問題がありました。これは、そのプラグインが同時に「Needs attention（要注意）」または「Favorites（お気に入り）」セクションにも表示されている場合に発生していました。v2.1.116ではこの重複表示が修正され、各プラグインが一度だけ表示されるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。`/plugin` を開くと、Installedタブの表示が正しくなっています。

```bash
# プラグイン管理画面を開く
/plugin

# Installed タブで重複表示がなくなった
```

## 実践例

### お気に入りプラグインの確認

```bash
/plugin
# → Installed タブを開く
# → お気に入りに登録したプラグインが1回だけ表示される（以前は2回表示されていた）
```

### 要注意プラグインの管理

```bash
/plugin
# → Installed タブで "Needs attention" のプラグインが重複なく一覧表示される
# → アップデートや再設定が必要なプラグインを正確に確認できる
```

## 注意点

- この修正は表示上のバグ修正で、プラグインの動作自体には影響しません
- 「Needs attention」は更新が必要、設定に問題があるなどの状態を示します
- プラグインをお気に入りに追加するには、プラグインの詳細画面からスター（★）をクリックします
- 重複表示によってプラグインが2つインストールされていたわけではありません
