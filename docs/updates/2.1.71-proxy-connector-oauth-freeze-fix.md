## 原文（日本語に翻訳）

多数のclaude.aiプロキシコネクターが期限切れのOAuthトークンを同時に更新しようとする際に発生するスタートアップUIフリーズを修正しました。

## 原文（英語）

Fixed startup UI freeze when many claude.ai proxy connectors refresh an expired OAuth token simultaneously

## 概要

複数のclaude.aiプロキシコネクターが設定されている環境で、OAuthトークンの有効期限が切れた際に複数のコネクターが同時にトークンのリフレッシュを試みることで、スタートアップ時にUIがフリーズする問題が修正されました。特に多数のコネクターを使用している組織での環境で顕著に発生していました。修正後は、トークンのリフレッシュ処理が適切に調整され、起動時のフリーズが解消されました。

## 基本的な使い方

この修正はv2.1.71以降で自動的に適用されます。複数のclaude.aiプロキシコネクターを設定している場合でも、スムーズに起動できるようになります。

```bash
# 複数のコネクターが設定された環境でも正常に起動
claude

# OAuthトークンが期限切れであっても、フリーズなしに再認証が行われる
```

## 実践例

### 組織環境での複数コネクター使用

エンタープライズ環境で複数のclaude.aiプロキシコネクターを使用している場合。

```bash
# 多数のorg設定コネクターがある環境で起動
claude

# 以前: トークン期限切れ時に起動時UIがフリーズすることがあった
# 修正後: 全てのコネクターのトークン更新が適切に処理される
```

### 長期間使用後の再起動

OAuthトークンの有効期限が切れた状態から再起動する場合。

```bash
# 数日ぶりにClaudeを起動（トークンが期限切れの可能性がある）
claude

# スムーズに起動し、必要に応じてトークンが更新される
```

### CI/CD環境でのプロキシコネクター

自動化パイプラインで複数のコネクターを使用する場合。

```bash
# 自動化スクリプト内での起動
claude --print "コードをレビューしてください"

# トークン期限切れによるフリーズがなくなり、自動化が安定する
```

## 注意点

- この修正はclaude.aiプロキシコネクターを使用している環境に特化した修正です。
- 通常のClaude Code利用（プロキシコネクターなし）には影響しません。
- 関連する修正として、OAuthトークンリフレッシュ後のコネクター再接続失敗の修正（`2.1.71-claudeai-connector-reconnect-fix.md`）もv2.1.71で対応されています。
- 組織管理者はコネクターの設定を確認し、不要なコネクターを削除することで起動時の処理を軽減できます。
