## 原文（日本語に翻訳）

- /ideコマンドで拡張機能を自動インストールする際に「onInstall is not defined」でクラッシュする問題を修正しました

## 原文（英語）

- Fixed /ide crashing with "onInstall is not defined" when auto-installing the extension

## 概要

`/ide` コマンドを使用してIDE拡張機能を自動インストールしようとした際に、「onInstall is not defined」というエラーでクラッシュする問題がありました。バージョン2.1.73でこの問題が修正され、拡張機能の自動インストールが正常に完了するようになりました。

## 基本的な使い方

```bash
# Claude CodeセッションでIDE拡張機能をインストール
/ide

# 自動インストールが正常に実行される（v2.1.73以降）
```

## 実践例

### VSCodeとの連携設定

```bash
# Claude Codeを起動
claude

# IDE連携を設定
/ide
# → VSCode拡張機能が自動的にインストールされる
# → 以前は「onInstall is not defined」エラーでクラッシュしていた
```

### 拡張機能の再インストール

```bash
# 拡張機能に問題がある場合は再度/ideコマンドを実行
/ide

# 拡張機能が正常にインストール・更新される
```

### IDE連携の確認

```bash
# 拡張機能のインストール後、IDE連携が有効か確認
# VSCode内でClaude Codeのサイドバーが表示されることを確認する
```

## 注意点

- この修正はv2.1.73で適用されます。それ以前のバージョンでは、`/ide` コマンドの自動インストールでクラッシュが発生する可能性があります
- クラッシュが発生した場合でも、手動でVSCode拡張機能をマーケットプレイスからインストールすれば回避できていました
- `/ide` コマンドはVSCode、Cursor、Windsurf など複数のIDEをサポートしています
- 拡張機能のインストールにはIDEが起動している必要がある場合があります
