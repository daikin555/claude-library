## 原文（日本語に翻訳）

OAuthトークンの更新後（ログインから8時間後）にMaxプランユーザーがデフォルトでSonnetに設定される拡張機能の問題を修正しました。

## 原文（英語）

[VSCode] Fixed extension defaulting Max plan users to Sonnet after the OAuth token refreshes (8 hours after login)

## 概要

Claude.ai の Max プラン（claude.ai/max）ユーザーが VS Code の Claude Code 拡張機能を使用している場合、OAuthトークンが更新されるタイミング（ログインから約8時間後）に、利用可能なより高性能なモデルではなく Sonnet モデルがデフォルトとして選択されてしまう問題がありました。v2.1.86ではこの問題が修正され、トークン更新後も Max プランで利用可能なモデルが正しく維持されます。

## 基本的な使い方

Max プランのユーザーは、OAuthトークンの更新後も設定したモデルが正しく保持されます。

```
# VS CodeのClaude Codeパネルで
1. Claude.ai Maxプランでログイン
2. 希望のモデル（例: Claude Opus）を選択
3. 8時間後（OAuthトークン更新後）も選択したモデルが維持される
```

## 実践例

### ユースケース: 長時間の開発セッションでのモデル維持

```
# 朝にVS Codeを開いてMaxプランでログイン
# 好みのモデルを設定
/model → Claude Opus 4 を選択

# 8時間後（夕方）もOpusのまま作業を継続
# v2.1.86以前: トークン更新後に自動的にSonnetに変わってしまう
# v2.1.86以降: Opusのまま維持される
```

### ユースケース: 複数日にわたるプロジェクト作業

```
# VS Codeを長時間使い続ける開発者
# 毎日OAuthトークンが更新されても
# モデル設定が意図せず変更されることがなくなります
```

## 注意点

- この修正は VS Code の Claude Code 拡張機能のみに関するものです（ターミナルのclaude CLIには影響しません）
- OAuthトークンは通常8時間ごとに自動更新されます
- Max プランのサブスクリプションが有効であることが前提です
- モデルの利用可能状況はプランと地域によって異なります
