## 原文（日本語訳）

worktreeにgitリポジトリが存在しないバックグラウンドセッションが削除できない問題を修正しました。

## 原文（英語）

Fixed background sessions whose worktree has no git repository being undeletable

## 概要

何らかの理由でworktreeに紐づくgitリポジトリが存在しない状態（例えば手動でディレクトリを削除した場合や、リポジトリ自体が破棄された場合など）になったバックグラウンドセッションを、Claude Code上から削除しようとしてもエラーになり、削除できずに残り続けてしまう不具合がありました。この修正により、gitリポジトリが存在しない状態のセッションでも正常にクリーンアップ（削除）できるようになりました。

## 基本的な使い方

この修正はバグ修正のため、特別な操作は不要です。バックグラウンドセッションの削除は通常通り `claude agents`（Agent View）や関連コマンドから行えます。

```bash
# 実行中・完了済みのバックグラウンドセッションを一覧表示
claude agents
```

Agent View上で対象のセッションを選択し、削除操作を行うと、修正後はworktreeのgitリポジトリが失われている場合でも正しく削除が完了します。

## 実践例

### worktreeのgitリポジトリが失われたケース

例えば、以下のような操作でworktree配下のgit管理情報が壊れてしまった場合を考えます。

```bash
# worktree配下の .git を誤って削除してしまった、などのケース
rm -rf .claude/worktrees/old-task/.git
```

このような状態のバックグラウンドセッションが残っていても、修正前は削除操作自体が失敗し、Agent View上にゾンビセッションとして残り続けてしまうことがありました。修正後は、gitリポジトリの有無にかかわらずセッションの削除処理が正しく完了します。

### 不要なバックグラウンドセッションの定期整理

複数のバックグラウンドタスクを運用している場合、定期的に不要なセッションを整理することでAgent Viewを見やすく保てます。

```bash
claude agents
# 一覧から不要なセッションを選んで削除
```

## 注意点

- この不具合は、worktreeのディレクトリやgit管理情報が何らかの理由で破損・消失した特殊なケースで発生していました
- 通常の削除フロー自体に変更はなく、あくまで「削除できずに残ってしまう」異常系の修正です
- 削除できないセッションが残っている場合は、アップデート後に再度削除を試すことで解消される可能性があります
