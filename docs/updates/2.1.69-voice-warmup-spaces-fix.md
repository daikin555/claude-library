## 原文（日本語に翻訳）

音声入力がウォームアップ中に5つすべてのスペースを表示せず、最大2つに制限するよう修正（「長押し中…」ヒントと整合）

## 原文（英語）

Fixed Voice input showing all 5 spaces during warmup instead of capping at ~2 (aligning with the "keep holding…" hint)

## 概要

音声入力のウォームアップ中（マイクボタンを押し始めた直後）に、5つのスペースが表示されていましたが、「長押し中…」のヒントと整合させるために最大2つのスペースに制限されました。これにより、ウォームアップ状態とアクティブな音声入力状態の視覚的な区別が明確になります。

## 基本的な使い方

```
# 音声入力ボタンを押した直後のウォームアップ中
# 修正前: 5つのスペースドット（● ● ● ● ●）が表示
# 修正後: 最大2つのスペースドット（● ●）が表示
#         「長押し中…」のヒントと視覚的に整合
```

## 実践例

### 音声入力の開始フロー

```
1. マイクボタンを押す（または音声ショートカット）
   → ウォームアップ: ● ●  （修正後: 2つのみ）
   → 「長押し中…」ヒントが表示

2. 長押し続ける
   → 音声入力アクティブ: ● ● ● ● ●  （フル表示）
   → 話し始める

3. ボタンを離す
   → ディクテーション完了
```

### ウォームアップ状態の視覚的フィードバック

```
# 修正後のより直感的な状態表示:
押した瞬間: ● ●        ← ウォームアップ中（2スペース）
押し続ける: ● ● ● ● ● ← アクティブ（5スペース）
```

## 注意点

- この変更は視覚的なフィードバックの改善であり、音声入力の機能自体は変わりません
- ウォームアップ時間や動作も変更されていません
- 「長押し中…」のヒントとの整合性を改善するための変更です
