# /remote-controlのリトライが「connecting…」で止まって見える問題を修正

## 原文（日本語に翻訳）

`/remote-control` のリトライが「connecting…」で止まっているように見えていた問題を修正しました。各リトライの結果が表示されるようになりました。

## 原文（英語）

Fixed `/remote-control` retries appearing stuck on "connecting…" — each retry now shows its result

## 概要

`/remote-control` でリモート接続を試みる際、リトライが発生してもUI上では「connecting…」と表示されたままになり、処理が止まっているように見える問題が修正されました。今後は各リトライの結果（成功・失敗・次のリトライ）が画面に表示され、接続状況が分かりやすくなります。

## 基本的な使い方

```bash
# リモートコントロールセッションへの接続
/remote-control connect

# v2.1.126以前:
# connecting…  ← ずっとこのまま（リトライしていることが分からない）

# v2.1.126以降:
# Attempt 1/3: connecting…
# Attempt 1/3: failed (timeout) - retrying in 2s
# Attempt 2/3: connecting…
# Attempt 2/3: connected ✓
```

## 注意点

- 接続先のサーバーが利用不可の場合、リトライ後に最終的なエラーメッセージが表示されます
- リトライ回数と間隔は設定で変更できます
