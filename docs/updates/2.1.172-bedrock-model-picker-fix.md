## 原文（日本語に翻訳）

BedrockのモデルピッカーでBedrockが提供していないモデルが選択肢に表示され、選択するとセッションモデルが無音で切り替わり複数行に選択マーカーが付く問題を修正しました

## 原文（英語）

Fixed the `/model` picker on Bedrock offering models the provider doesn't serve — selecting one silently switched the session model and lit the selection marker on multiple rows

## 概要

Amazon Bedrock経由でClaude Codeを使用している場合、`/model` コマンドのピッカーにBedrockが実際には提供していないモデルが表示される問題がありました。さらに、そのモデルを選択するとエラーを表示せず無音でセッションモデルが変更され、UI上では複数の行に選択マーカーが付くという混乱を招く不具合も同時に修正されました。

## 基本的な使い方

```
# Bedrock経由でClaude Codeを使用中
/model

# 修正前: BedrockがサポートしないモデルIDも候補に表示され、
#         選択しても見かけ上何か変わったように見えるが実際には正しく動作しなかった
# 修正後: Bedrockが実際にサポートするモデルのみが候補に表示される
```

## 実践例

### Bedrock利用者のモデル切り替え

```bash
# Amazon Bedrock設定でClaudeを起動
export ANTHROPIC_BEDROCK_BASE_URL=https://bedrock-runtime.us-east-1.amazonaws.com
claude

/model
# 修正後: Bedrockでサポートされているモデルのみがリストアップされる
# 例: claude-sonnet-4-6、claude-haiku-4-5 など（Bedrockが提供するモデルに限定）
```

## 注意点

- BedrockではAnthropicが提供するすべてのモデルが利用できるわけではありません
- 利用可能なモデルはAWSリージョンによっても異なります
- `/status` コマンドで現在接続中のプロバイダーとモデルを確認できます
