## 原文（日本語に翻訳）

`description:` フロントマターフィールドがないプロジェクトスキルがClaudeの利用可能なスキル一覧に表示されない問題を修正

## 原文（英語）

Fixed project skills without a `description:` frontmatter field not appearing in Claude's available skills list

## 概要

`SKILL.md` ファイルのフロントマターに `description:` フィールドがない場合、そのスキルがClaudeの利用可能なスキル一覧に表示されていませんでした。この修正により、`description:` が省略されたスキルも一覧に表示されるようになります。スキルの説明は任意項目になりました。

## 基本的な使い方

```yaml
---
# description: がなくてもスキルが認識される（修正後）
name: my-skill
triggers:
  - "/my-skill"
---

## My Skill

このスキルの説明...
```

## 実践例

### description なしのシンプルなスキル定義

```yaml
---
name: format-code
---

## コードフォーマットスキル

コードを自動フォーマットします。以下の言語に対応:
- JavaScript / TypeScript
- Python
- Go
```

### 最小限のフロントマターでのスキル

```yaml
---
name: run-tests
---

## テスト実行スキル

プロジェクトのテストスイートを実行し、結果を報告します。
```

### description を追加した完全な例（推奨）

```yaml
---
name: code-review
description: "コードのレビューと改善提案を行うスキル"
---

## コードレビュースキル

詳細な説明...
```

## 注意点

- `description:` フィールドは任意になりましたが、スキル一覧での表示品質のために記述を推奨します
- `description:` がない場合、スキル名のみが一覧に表示されます
- 修正前にdescriptionなしで作成したスキルも、v2.1.69以降で自動的に一覧に表示されるようになります
