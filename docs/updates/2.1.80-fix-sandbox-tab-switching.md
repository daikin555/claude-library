## 原文（日本語に翻訳）

`/sandbox` のタブ切り替えがTabキーや矢印キーに反応しない問題を修正しました

## 原文（英語）

Fixed `/sandbox` tab switching not responding to Tab or arrow keys

## 概要

`/sandbox` コマンドで開くサンドボックス管理画面において、TabキーまたはTabキーを使ってタブを切り替えようとしても反応しない問題がありました。キーボード操作が受け付けられないため、マウスを使うか別の方法でタブを切り替える必要がある状況でした。この修正により、`/sandbox` 画面でTabキーと矢印キーによるタブ切り替えが正しく動作するようになります。

## 基本的な使い方

`/sandbox` コマンドを実行後、TabキーまたはTab/矢印キーでタブを切り替えられるようになりました。

```
# サンドボックス管理画面を開く
/sandbox

# キーボード操作
Tab     : 次のタブに移動
→       : 次のタブに移動（または項目間の移動）
←       : 前のタブに移動
↑↓     : タブ内の項目を選択
Enter   : 選択した操作を実行
```

## 実践例

### サンドボックス設定のキーボードナビゲーション

```
# /sandbox コマンドを実行
/sandbox

# 修正後のキーボード操作
→ または Tab を押す: 次のタブに移動
← を押す: 前のタブに移動

# 各タブでサンドボックスの設定を確認・変更
```

### マウスを使わないサンドボックス管理

```
# キーボードのみでサンドボックスを管理
1. /sandbox を実行
2. Tab/矢印キーでタブを切り替え
3. ↑↓ キーで項目を選択
4. Enter で設定を変更
5. q または Esc で閉じる
```

### サンドボックス設定の確認

```bash
# サンドボックスの現在の設定を確認
/sandbox

# タブを切り替えて各カテゴリの設定を確認
# - 許可されているコマンド
# - ブロックされているコマンド
# - ファイルシステムのアクセス設定
```

## 注意点

- **関連する修正**: v2.1.79で `/permissions` の矢印キーナビゲーション、v2.1.80でも `/permissions` の改善が行われています。キーボードナビゲーションの統一が進められています
- **Tabキーの用途**: 修正後、Tabキーはタブ切り替えとフォーカス移動の両方に使用される場合があります。コンテキストによって動作が異なることに注意してください
- **ターミナルの設定**: 一部のターミナルではTabキーが他の機能（補完など）に割り当てられている場合があります
- **サンドボックスの目的**: `/sandbox` はClaudeが実行できるコマンドやアクセスできるリソースを制限するためのセキュリティ機能です
