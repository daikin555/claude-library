## 原文（日本語に翻訳）

サーバー管理設定を変更し、無害な機能トグルやコストに関するトグルではsettings-approvalプロンプトが表示されないようにしました。

## 原文（英語）

Changed server-managed settings so benign feature and cost toggles no longer trigger the settings-approval prompt

## 概要

サーバー管理設定（server-managed settings）は、組織の管理者がAdmin Console等を通じて、ユーザーのClaude Code設定を一元的に配布・更新できる仕組みです。これまでは、配布される設定にわずかな変更が含まれているだけでも、内容の重大度に関わらず一律に「settings-approval」プロンプトが表示され、ユーザーが内容を確認して承認しないとClaude Codeを継続利用できませんでした。v2.1.218では、機能の有効・無効やコストに関する設定など、セキュリティ上リスクの低い「無害な」トグルの変更については、この承認プロンプトを表示せず自動的に適用されるようになりました。これにより、組織側は軽微な設定変更を配布しやすくなり、ユーザー側も不要な確認操作に煩わされにくくなります。

## 基本的な使い方

ユーザー側で明示的に行う操作はありません。組織の管理者がAdmin Console等でサーバー管理設定を更新した際の挙動が変わります。

```text
# 管理者が「新しいベータ機能を有効化」のような
# 無害な機能トグルを配布した場合

# 変更前: Claude Code起動時に承認プロンプトが表示され、
#         ユーザーが承認しないと利用を継続できない

# 変更後: プロンプトなしで自動的に設定が適用され、
#         ユーザーはそのまま作業を継続できる
```

## 実践例

### 新機能のロールアウト

```text
# 管理者が組織全体に新しい実験的機能を段階的に有効化する場合、
# 各ユーザーのターミナルで承認プロンプトが表示されなくなったため、
# 展開がスムーズになり、ユーザーの作業が中断されない
```

### コストに関するトグルの調整

```text
# 例: あるモデルのデフォルト効果レベル（effort）や
#     コスト関連のデフォルト値を組織側で調整する場合

# 無害なコストトグルとして扱われるため、
# ユーザー側に承認を求めることなく反映される
```

### 引き続き承認が必要な変更との違い

```text
# 一方で、シェルコマンドの許可設定や
# カスタム環境変数の注入など、
# セキュリティ上リスクの高い設定変更は
# これまで通りsettings-approvalプロンプトが表示され、
# ユーザーの明示的な承認が必要
```

## 注意点

- この変更が対象とするのは「無害」と判断される機能・コストトグルに限られます。permissions.denyやシェルコマンド許可、カスタム環境変数、監査ログのパス、テレメトリ設定などリスクの高い項目は引き続き承認プロンプトの対象です
- この機能は組織でサーバー管理設定を利用している場合に関係する変更であり、個人利用のみで組織管理下にない場合は直接的な影響はありません
- 承認プロンプトが表示されなくなったからといって設定が無効化されているわけではなく、無害な変更は静かに適用されている点に注意してください。現在の設定内容は `/config` や `claude doctor` 等で確認できます
