## 原文（日本語訳）

`/release-notes` が、閲覧した内容をモデルのコンテキストに追加してしまう不具合を修正した。これまで「Show all」を選択すると、変更履歴全体がそれ以降のすべてのリクエストに注入され続けていた。

## 原文（英語）

Fixed `/release-notes` adding the viewed notes to the model's context — "Show all" previously injected the entire changelog into every subsequent request

## 概要

`/release-notes` はセッション内でバージョンを選択しながらChangelogを閲覧できるインタラクティブなコマンドです。これまでは、一覧の中で「Show all」を選択すると、その時点で表示された変更履歴全文がそのままモデルのコンテキストに追加されてしまい、以降のすべてのリクエストにその内容が引き続き含まれるという不具合がありました。単に画面上でChangelogを確認したいだけなのに、意図せずコンテキスト消費とコストが増大してしまう問題です。今回の修正で、閲覧内容がモデルのコンテキストに残らなくなりました。

## 基本的な使い方

セッション内で `/release-notes` を実行し、確認したいバージョンを選択します。

```text
/release-notes
```

バージョン一覧から矢印キーで選択し、Enterで詳細を表示します。

## 実践例

### アップデート直後に変更点だけをさっと確認する

Claude Codeを新しいバージョンに更新した直後、何が変わったのかを確認したい場合に使います。

```text
/release-notes
# 更新後のバージョンを選択して内容を確認
```

修正後は、内容を閲覧してもその後のやり取りのコンテキストサイズには影響しません。

### 長時間セッションの途中でChangelogを参照する

長時間続いているセッションの途中で「この挙動は新しいバージョンで変わったのか？」を確認したくなることがあります。

```text
/release-notes
# 「Show all」で過去バージョンも含めて一覧表示して確認
```

修正前はここで「Show all」を使うと以降のリクエストすべてに変更履歴全文が含まれ続け、無駄なトークン消費とコスト増につながっていましたが、修正後はそのような持ち越しは発生しません。

## 注意点

- 修正前のバージョンで「Show all」を多用していた長時間セッションでは、コンテキストサイズが想定以上に大きくなっていた可能性があります。心当たりがある場合はv2.1.208以降へのアップデートを推奨します。
- あくまで「モデルのコンテキストに残り続ける」という表示後の挙動に関する修正であり、Changelogの内容自体や `/release-notes` の表示機能そのものは変わりません。
- 本記事作成時点では、`/release-notes` コマンド自体を個別に解説する公式ドキュメントページは見当たらなかったため、changelogの記載内容をもとに記述しています。最新情報は公式ドキュメントを参照してください。
