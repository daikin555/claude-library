## 原文（日本語に翻訳）

パーミッションが解決された後にRemote Controlセッションのステータスが「Requires Action」のままになる問題を修正しました。

## 原文（英語）

Fixed Remote Control session status staying stuck on "Requires Action" after a permission is resolved

## 概要

claude.aiのRemote Control機能を使用している際に、パーミッションの確認を行って承認・拒否の操作を完了しても、セッションのステータス表示が「Requires Action（アクション要求中）」のままになってしまうバグが修正されました。修正後は、パーミッション解決後にステータスが適切に更新されます。

## 基本的な使い方

Remote Controlセッションでパーミッションを承認または拒否した後、ステータスが自動的に更新されます：

1. Remote Controlセッションを開始
2. Claudeがツール使用のパーミッションを要求
3. ユーザーが承認または拒否
4. ステータスが「Requires Action」から「Running」（または「Waiting」）に自動更新

## 実践例

### ユースケース: claude.aiのRemote Controlでの承認フロー

claude.aiのWebインターフェースからRemote Controlセッションを操作する場合：

1. claude.aiでRemote Controlセッションを開始
2. Claudeがファイルへの書き込みを要求
3. 「Allow（許可）」をクリック
4. ステータスが「Requires Action」から自動的に変わる（修正前はスタックしていた）

### ユースケース: 自動化ワークフローでの管理

複数のパーミッション確認が連続して発生するワークフローで、各パーミッション解決後に正しくステータスが進行します。

## 注意点

- この修正はRemote Control機能を使用している場合にのみ関連します
- Remote Control機能はclaude.aiのプレミアムプランで利用可能です
- ステータスの更新はWebSocket接続経由でリアルタイムに行われます
