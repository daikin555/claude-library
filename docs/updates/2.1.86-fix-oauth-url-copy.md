## 原文（日本語に翻訳）

`c` ショートカットがOAuthログインURLの完全なURLではなく約20文字しかコピーしない問題を修正しました。

## 原文（英語）

Fixed the `c` shortcut copying only ~20 characters of the OAuth login URL instead of the full URL

## 概要

OAuthによるログインが必要な際に表示されるURL（認証用のOAuthリンク）をコピーするために `c` ショートカットを使用すると、URLの最初の約20文字しかクリップボードにコピーされないバグが修正されました。修正後は、完全なURLがコピーされるようになります。

## 基本的な使い方

MCP OAuthやclaude.aiへのログインが必要な際、`c` キーでURLをコピーできます：

1. ログインが必要なMCPサーバーに接続する
2. OAuthのURLが表示される
3. `c` キーを押してURLをコピー（修正後は完全なURLがコピーされる）
4. コピーしたURLをブラウザに貼り付けて認証を完了

## 実践例

### ユースケース: MCPサーバーへのOAuth認証

```
以下のURLでログインしてください：
https://auth.example.com/oauth/authorize?client_id=claude-code&scope=read%3Auser&state=...

[c] URLをコピー   [Enter] ブラウザで開く
```

以前は `c` を押すと `https://auth.exa` などの短い文字列しかコピーされませんでしたが、修正後は完全なURLがコピーされます。

### ユースケース: ヘッドレス環境でのOAuth

ブラウザが自動的に開けないヘッドレス環境では、`c` でURLをコピーして別のマシンのブラウザで開く操作が正常に機能します。

## 注意点

- この修正はOAuth認証フロー中の `c` ショートカットにのみ関連します
- OAuthのURLは通常非常に長く（200文字以上）、以前は大部分が切れていました
- 修正後は完全なURLがクリップボードにコピーされます
