## 原文（日本語に翻訳）

- VSCode: 統合ターミナルのスクロール速度がネイティブターミナルと一致しない問題を修正

## 原文（英語）

- VSCode: Fixed scroll speed in integrated terminals not matching native terminals

## 概要

VSCodeの統合ターミナル内でClaude Codeを使用する際、スクロール速度がネイティブ（OS標準）のターミナルと異なる問題が修正されました。以前はVSCode統合ターミナルでのスクロールがネイティブターミナルに比べて速すぎたり遅すぎたりする状態でしたが、この修正によりネイティブターミナルと同じ感覚でスクロールできるようになりました。

## 基本的な使い方

VSCodeの統合ターミナルでClaude Codeを使用する際、スクロール操作が自然に動作します:

```
# VSCode統合ターミナルでClaude Codeを起動
claude

# 長い出力結果のスクロールが
# ネイティブターミナルと同じ速度で動作する
```

## 実践例

### 長いコード出力の確認

大量のコード出力やログをスクロールして確認する際の操作感が改善されました:

```
claude> このプロジェクトのすべてのTODOコメントを見つけて
# 大量の検索結果が表示される
# → スクロールがネイティブターミナルと同じ速度で動作
```

### 差分の確認

ファイルの差分表示など、上下にスクロールしながら確認する作業が快適になりました:

```
claude> 最近の変更の差分を表示して
# 差分出力を上下にスクロール
# → 自然なスクロール速度で確認可能
```

## 注意点

- この修正はVSCode拡張版のClaude Codeにのみ影響します
- ネイティブターミナル（iTerm2、Terminal.appなど）での動作には変更ありません
- VSCodeのスクロール関連設定（`editor.mouseWheelScrollSensitivity`など）との相互作用がある場合があります
- トラックパッドとマウスホイールの両方で改善が適用されます
