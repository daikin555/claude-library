## 原文（日本語訳）

`name` frontmatterフィールドを持つプラグインスキルが、スラッシュコマンドのオートコンプリートでプラグインの接頭辞を失っていた問題を修正しました。

## 原文（英語）

Fixed plugin skills with a `name` frontmatter field losing their plugin prefix in slash-command autocomplete

## 概要

プラグインが提供するスキルは、通常 `/プラグイン名:スキル名` のような接頭辞付きの名前でスラッシュコマンドメニューに表示されます。しかし、`SKILL.md` のfrontmatterに明示的な `name` フィールドを指定していると、オートコンプリート上でこのプラグイン接頭辞が失われ、他のプラグインの同名スキルや通常のスキルと区別がつかなくなる不具合がありました。今回の修正により、`name` フィールドを指定していても正しくプラグイン接頭辞付きで表示されるようになります。

## 基本的な使い方

この修正は自動的に適用される挙動であり、追加の設定は不要です。プラグイン開発者は、これまで通り `SKILL.md` に `name` フィールドを指定して構いません。

```yaml
---
name: deploy
description: 本番環境へのデプロイを支援するスキル
---
```

修正前はこのスキルがオートコンプリートで `/deploy` のように接頭辞なしで表示されることがありましたが、修正後は `/my-plugin:deploy` のように、インストールされているプラグイン名を接頭辞として正しく表示されます。

## 実践例

### 複数プラグインで同名スキルが衝突するのを防ぐ

異なるプラグインが同じ `name: deploy` のスキルを持っていても、プラグイン接頭辞によってオートコンプリート上で区別できるようになります。

```
/infra-plugin:deploy   ... infra-pluginが提供するdeployスキル
/app-plugin:deploy     ... app-pluginが提供するdeployスキル
```

修正前は `name` フィールドの指定によりどちらも `/deploy` と表示され、意図しないスキルを呼び出してしまうリスクがありました。

### バージョン文字列に依存しない安定したスキル名を維持しつつ、出自を明確にする

`name` フィールドを指定する主な目的は、マーケットプレイス経由でインストールした際にディレクトリ名（バージョン文字列など）がそのままスキル名になってしまうのを防ぐことです。この修正により、`name` を指定しても以下のようにプラグイン名は失われず両立します。

```yaml
---
name: review
description: コードレビューを行うスキル
---
```

```
/code-review-plugin:review
```

### プラグイン開発時にオートコンプリートの表示を確認する

プラグインを開発・配布する際は、Claude Codeのセッションで `/` を入力し、意図した接頭辞付きの名前でスキルが表示されるかを確認する運用が有効です。

```
/ と入力してメニューを開く
→ /your-plugin-name:skill-name が期待通り表示されるか確認
```

## 注意点

- この修正はオートコンプリート表示上の問題であり、スキル自体の呼び出しロジックには影響しません
- プラグイン未使用の（プロジェクト/ユーザー直下の）スキルには接頭辞は付与されません
- `name` フィールドを省略した場合の挙動（ディレクトリ名がそのままスキル名になる点）自体はこの修正の対象外です
