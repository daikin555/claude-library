## 原文（日本語に翻訳）

大きな編集済みファイルの添付スニペット生成時にハングする問題を修正しました。

## 原文（英語）

Fixed a hang when generating attachment snippets for large edited files

## 概要

Claude Codeがファイルを編集する際、変更内容を添付スニペットとしてまとめる処理があります。大容量のファイルを編集したときにこの処理がハング（応答なし）状態になる不具合がありました。v2.1.84ではこの問題を修正し、大きなファイルでも安定して編集が完了するようになりました。

## 基本的な使い方

ファイル編集は通常通り行うことができます。修正後は大きなファイルでもハングが発生しなくなります。

```bash
# 大きなファイルをClaudeに編集させる例
claude "src/large-file.ts のリファクタリングをしてください"
```

Editツールがファイルを変更した後、添付スニペット生成が自動的に行われます。

## 実践例

### ユースケース: 数千行のファイルを編集する

大規模なソースファイルに対してClaudeが編集を行う場合：

```
# 例: 5000行のTypeScriptファイルにリファクタリングを依頼
Claude: src/components/LargeComponent.tsx を編集して...
```

v2.1.84以前ではこのような大きなファイルの編集後にハングが発生することがありましたが、修正後は正常に完了します。

### ユースケース: 複数の大きなファイルを連続編集する

```bash
# バッチ処理で複数ファイルを編集する場合も安定して動作します
claude "src/ ディレクトリ内のすべてのTypeScriptファイルを更新してください"
```

## 注意点

- この修正はファイルサイズに依存したハングを解消しますが、非常に大きなファイル（数MB以上）の処理には時間がかかる場合があります
- 編集処理中に応答がない場合は、Ctrl+C で中断して再試行することができます
