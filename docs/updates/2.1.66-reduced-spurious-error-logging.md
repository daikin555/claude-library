# 不要なエラーログの出力を削減 — ノイズの少ないクリーンなログ体験

## 原文（日本語に翻訳）

不要なエラーログの出力を削減しました。

## 原文（英語）

Reduced spurious error logging

## 概要

Claude Code v2.1.66で、実際には問題のない状況で誤って出力されていた「不要なエラーログ（spurious error logging）」が削減されました。これによりログ出力がよりクリーンになり、本当のエラーと誤検知を区別しやすくなります。デバッグ時のノイズが減少し、実際に対処が必要な問題を見つけやすくなります。

## 基本的な使い方

このアップデートはバックグラウンドで自動的に適用されます。特別な設定変更は不要です。v2.1.66以降にアップデートするだけで、不要なエラーログが表示されなくなります。

### バージョンの確認

```bash
claude --version
```

v2.1.66以上であれば、この修正が適用されています。

### ログの確認方法

```bash
# デバッグログを確認（より詳細な出力）
claude --debug

# ログファイルの場所（一般的な場所）
~/.claude/logs/
```

## 実践例

### バックグラウンドタスク実行時のログ改善

以前のバージョンでは、バックグラウンドエージェントが正常完了した際にも不要なエラーメッセージが出力されることがありました。v2.1.66からは、そのような誤検知エラーが表示されなくなります。

```
# 以前（不要なエラーが混在していた例）
✓ バックグラウンドタスク完了
Error: connection reset (不要な誤検知エラー)
✓ 結果を統合しました

# v2.1.66以降（クリーンな出力）
✓ バックグラウンドタスク完了
✓ 結果を統合しました
```

### デバッグ時の効率化

エラーログのノイズが減ることで、`--debug` モードでの調査がより効率的になります：

```bash
# --debugモードで実際の問題を特定しやすくなる
claude --debug

# 実際のエラーのみが表示され、原因特定が迅速に
```

## 注意点

- **本物のエラーは引き続き表示**: この変更は誤検知の削減であり、実際のエラーは引き続き正しく報告されます。
- **ログレベルの設定**: もし詳細なログが必要な場合は、`--debug` フラグを使用することで詳細情報を確認できます。
- **自動適用**: ユーザー側での設定変更は一切不要です。アップデートするだけで恩恵を受けられます。
