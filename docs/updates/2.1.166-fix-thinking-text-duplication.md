## 原文（日本語に翻訳）

ストリーミング中に Ctrl+O トランスクリプトビューで思考テキストが重複していたバグを修正しました。

## 原文（英語）

Fixed duplicated thinking text in the Ctrl+O transcript view while streaming

## 概要

`Ctrl+O` で開くトランスクリプトビューにおいて、Extended Thinking（拡張思考）が有効な場合にストリーミング中に思考テキストが重複して表示されるバグを修正しました。修正後は思考テキストが一度だけ正しく表示されます。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
```

トランスクリプトビューの表示確認：

```
# Claude Code 使用中に Ctrl+O を押してトランスクリプトを開く
# → 思考テキストが重複せず、正常に表示されることを確認
```

## 実践例

### トランスクリプトビューの使い方

```bash
# 思考機能が有効なモデルで Claude Code を起動
claude --model claude-sonnet-4-6

# 複雑な質問を入力してストリーミング中に
Ctrl+O  # トランスクリプトビューを開く
# → 修正後は思考テキストが一度だけ表示される
```

## 注意点

- この問題は Extended Thinking が有効な時にのみ発生していました
- Ctrl+O トランスクリプトビューは現在進行中の会話をテキスト形式で確認するための機能です
- 修正前の表示乱れにより読みにくかった思考プロセスが、修正後は正常に読めるようになります
