## 原文（日本語に翻訳）

マウストラッキングを完全にサポートしていないWindowsコンソールでのマウストラッキングを無効化しました

## 原文（英語）

Disable mouse tracking on Windows consoles that don't fully support it

## 概要

マウストラッキング機能を完全にサポートしていないWindowsのコンソール環境（従来のWindows Consoleなど）で、マウストラッキングを自動的に無効化するようになりました。これにより、非対応環境での誤動作や不要な入力エラーが解消されます。

## 基本的な使い方

この修正は自動的に適用されます。Windowsコンソールがマウストラッキングをサポートしているかどうかを自動検出し、非対応の場合は無効化します。

```
# 従来のWindowsコンソール（cmd.exeなど）でClaude Codeを使用
claude
# 修正後: マウストラッキングが自動的に無効化され、正常に動作する
```

## 実践例

### 対応コンソールの例

| コンソール | マウストラッキング |
|-----------|----------------|
| Windows Terminal | 対応（有効） |
| cmd.exe（従来版） | 非対応（自動無効化） |
| PowerShell 5.x | 環境依存（自動検出） |
| VS Code統合ターミナル | 対応（有効） |
| WSL2ターミナル | 対応（有効） |

## 注意点

- マウストラッキングが無効の環境では、マウスクリックによるカーソル移動が使えません
- キーボードナビゲーション（矢印キー、Tab等）は引き続き使用できます
- Windows Terminalや最新のPowerShellを使用することで、フルUI機能を利用できます
