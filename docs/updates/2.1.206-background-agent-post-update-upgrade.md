## 原文（日本語訳）

バックグラウンドエージェントは、Claude Codeのアップデート直後にバックグラウンドで新しいバージョンへアップグレードされるようになった。これまでのように、セッションにアタッチするタイミングで発生する遅いstale-session（古いセッション）アップグレードを待つ必要がなくなった。

## 原文（英語）

Background agents now upgrade to a new version in the background right after a Claude Code update, instead of paying a slow stale-session upgrade when you attach

## 概要

Claude Codeでは `/background`（`/bg`）や `claude --bg` によってセッションをバックグラウンドで実行できますが、Claude Code本体がアップデートされても、稼働中のバックグラウンドセッションはそれまで古いバージョンのまま動き続けていました。これまでは、そのセッションに再アタッチした瞬間に新バージョンへのアップグレード処理が走り、待たされることがありました。今回のアップデートにより、アップグレード処理はアタッチ時ではなく、Claude Code本体の更新直後にバックグラウンドで先んじて実行されるようになり、アタッチ時の待ち時間がなくなります。

## 基本的な使い方

ユーザー側で特別な操作は不要です。バックグラウンドセッションを普段どおり作成し、

```bash
claude --bg "flaky なテストを調査して修正して"
```

または

```text
/bg テストスイートを実行して失敗を修正して
```

のように利用していれば、Claude Code本体の自動アップデートが行われた際に、バックグラウンドで動いているセッションも裏側で順次新バージョンへ移行されます。

## 実践例

### 一晩中バックグラウンドで動かしていたエージェントに翌朝アタッチする

夜間に長時間タスクを `claude --bg` で走らせておき、翌朝 `claude agents` でアタッチするケースです。以前は、Claude Code自体が夜間に自動更新されていた場合、アタッチした瞬間にセッション側のアップグレード処理が走り、応答が返ってくるまで待たされることがありました。今回の変更後は、更新直後に裏側でアップグレード済みのため、アタッチしてすぐに操作を再開できます。

```bash
claude agents
```

### 複数のバックグラウンドセッションを並行運用している場合

複数の `--bg` セッションを同時に走らせているチームや個人の場合、Claude Code更新のタイミングで全セッションが一括して裏側でアップグレードされるため、どのセッションにアタッチしてもストレスなく作業を継続できます。

## 注意点

- この改善はあくまで「アップグレードが走るタイミング」を早める変更であり、バックグラウンドエージェントの機能自体が変わるわけではありません。
- 自動更新を無効化している環境（企業ポリシーなどでアップデートチャネルを固定している場合）では、そもそも新バージョンへの切り替えが発生しないため本改善の恩恵は限定的です。
