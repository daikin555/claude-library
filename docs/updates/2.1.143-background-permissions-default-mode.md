## 原文（日本語に翻訳）

- `claude agents` から起動したバックグラウンドセッションが、`settings.json` の `permissions.defaultMode` を尊重するようになりました（以前はautoモードに上書きされていました）

## 原文（英語）

- Background sessions launched from `claude agents` now honor `permissions.defaultMode` from settings.json (was previously overridden to auto mode)

## 概要

以前のバージョンでは、`claude agents` ビューから起動したバックグラウンドセッションのパーミッションモードが自動的に auto モードに上書きされていました。今回の修正により、`settings.json` の `permissions.defaultMode` 設定が正しく適用されるようになりました。チームや組織の標準パーミッション設定を全セッションに一貫して適用できます。

## 基本的な使い方

`.claude/settings.json` または `~/.claude/settings.json` で `permissions.defaultMode` を設定します：

```json
{
  "permissions": {
    "defaultMode": "default"
  }
}
```

`claude agents` から起動したバックグラウンドセッションに、この設定が自動的に適用されます。

## 実践例

### パーミッションモードの設定例

```json
// .claude/settings.json

// 通常のデフォルト（各操作で確認が必要）
{
  "permissions": {
    "defaultMode": "default"
  }
}

// autoモード（安全と判断された操作は自動承認）
{
  "permissions": {
    "defaultMode": "auto"
  }
}

// bypassモード（すべての操作を自動承認）
{
  "permissions": {
    "defaultMode": "bypassPermissions"
  }
}
```

### 利用可能なパーミッションモード

| モード | 説明 |
|--------|------|
| `default` | 各ツール操作でユーザー確認が必要 |
| `auto` | Claude Codeが安全と判断した操作を自動承認 |
| `bypassPermissions` | すべての操作を確認なしで実行（信頼できる環境のみ） |

### チーム設定でのデフォルトモード管理

```json
// .claude/settings.json（プロジェクト共通設定）
{
  "permissions": {
    "defaultMode": "auto",
    "allow": [
      "Bash(git *)",
      "Bash(npm test)",
      "Read(*)"
    ]
  }
}
```

これにより、`claude agents` から起動するすべてのバックグラウンドセッションに一貫したパーミッション設定が適用されます。

### セッション個別のオーバーライド

`claude agents` 起動時に `--permission-mode` フラグで個別にオーバーライドも可能です（v2.1.143 で追加）：

```bash
# settings.json の defaultMode を上書きして autoモードで起動
claude agents --permission-mode auto
```

## 注意点

- v2.1.143 以前は `claude agents` から起動したセッションが常に auto モードで動作していました。アップデート後は `settings.json` の設定が適用されるため、意図した動作を確認してください
- `bypassPermissions` モードは、完全に信頼できる環境でのみ使用してください
- マネージド設定（managed settings）による `defaultMode` はユーザー設定より優先されます
- `claude agents --permission-mode` フラグは `settings.json` の `permissions.defaultMode` より優先されます
