## 原文（日本語に翻訳）

フォーカスモードを改善しました：Claudeは最終メッセージのみをユーザーが見ることを認識しているため、より自己完結したサマリーを書くようになりました。

## 原文（英語）

Improved focus mode: Claude now writes more self-contained summaries since it knows you only see its final message

## 概要

フォーカスモードでは、ユーザーはClaudeの最終メッセージのみを受け取ります。このアップデートにより、Claudeはこのコンテキストを認識し、中間の思考プロセスや会話の流れを省いた、より自己完結型のサマリーを生成するようになりました。ユーザーは必要な情報がすべて含まれた完結した回答を受け取れます。

## 基本的な使い方

```bash
# フォーカスモードで実行（最終メッセージのみ表示）
claude --focus "このプロジェクトの問題点を分析して"

# Claudeは中間プロセスを省き、
# 完結したサマリーを直接提供する
```

## 実践例

### コードレビューでの自己完結型フィードバック

```bash
# フォーカスモードでコードレビューを依頼
claude --focus "以下のコードをレビューして問題点を教えて:
$(cat src/main.py)"

# 改善前: 「まず...次に...そして...最終的に...」という形式
# 改善後: 問題点のリストと修正方法が直接提供される
```

### プロジェクト状況レポートの生成

```bash
#!/bin/bash
# フォーカスモードでプロジェクトの週次レポートを生成

# git logからコミット情報を取得
recent_commits=$(git log --oneline --since="1 week ago")

# フォーカスモードで自己完結型レポートを生成
claude --focus "以下のコミット履歴から週次レポートを作成して:
$recent_commits

レポートには以下を含めること:
- 主な変更点
- 完了したタスク
- 注意が必要な問題"
```

### 技術文書の要約

```bash
# 長い技術文書をフォーカスモードで要約
# 背景知識なしでも理解できる自己完結型サマリーを生成

cat technical_spec.md | claude --focus \
  "この技術仕様書を、前提知識なしで理解できるように要約して。
   主要な決定事項と理由を含めること。"
```

### バグレポートの分析

```bash
# バグレポートをフォーカスモードで分析
# 原因と解決策を含む完結した回答を取得

claude --focus "以下のエラーログを分析して、原因と修正方法を教えて:
$(cat error.log | tail -50)"
```

## 注意点

- フォーカスモードでは中間の思考プロセスは表示されません
- Claudeが「最終メッセージだけが見える」という認識を持つことで、より凝縮された情報を提供します
- 長い作業が必要なタスクでは、途中経過が見えないため適切なタスク設計が重要です
- 複雑なマルチステップタスクよりも、明確な出力形式があるタスクに適しています
- 自己完結型サマリーは、後で参照する際にも文脈なしで理解できます
