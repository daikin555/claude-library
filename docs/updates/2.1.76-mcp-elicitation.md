# MCPエリシテーション対応：MCPサーバーがタスク途中で構造化入力を要求可能に

## 原文（日本語に翻訳）

MCPエリシテーションのサポートを追加 — MCPサーバーがタスク実行中にインタラクティブなダイアログ（フォームフィールドまたはブラウザURL）を通じて構造化された入力を要求できるようになった

## 原文（英語）

Added MCP elicitation support — MCP servers can now request structured input mid-task via an interactive dialog (form fields or browser URL)

## 概要

Claude Code v2.1.76で、MCP（Model Context Protocol）のエリシテーション機能がサポートされました。これにより、MCPサーバーがツール実行の途中でユーザーに対して構造化された入力を要求できるようになります。従来の自由形式のテキスト入力ではなく、JSONスキーマに基づいたフォームフィールドやブラウザURLを通じて、正確なデータ収集が可能です。

## 基本的な使い方

MCPエリシテーションは、MCPサーバー側から自動的にトリガーされます。サーバーが `elicitation/create` リクエストを送信すると、Claude Code上にインタラクティブなダイアログが表示されます。

エリシテーションには2つのモードがあります：

- **フォームモード（`form`）**: テキストフィールド、ドロップダウン、チェックボックスなどのフォーム要素が表示される
- **URLモード（`url`）**: ブラウザで認証URLを開く（OAuth認証フローなど）

ユーザーはダイアログに入力を行い、その結果がMCPサーバーに返されます。

## 実践例

### 認証情報の入力

MCPサーバーがAPIキーやクレデンシャルを要求するケースです：

```
# MCPサーバーからのエリシテーションリクエスト例
メッセージ: "APIに接続するための認証情報を入力してください"
フォームフィールド:
  - API Key (テキスト入力)
  - Region (ドロップダウン: us-east-1, ap-northeast-1, eu-west-1)
  - Use SSL (チェックボックス)
```

ユーザーにはClaude Code上でフォームが表示され、必要な情報を入力して送信できます。

### OAuth認証フロー

MCPサーバーがOAuth認証を必要とする場合のURLモード例：

```
# MCPサーバーからのURLモードエリシテーション
メッセージ: "GitHubアカウントの認証が必要です"
URL: https://github.com/login/oauth/authorize?client_id=...
```

ブラウザが開き、認証を完了するとMCPサーバーに結果が返されます。

### デプロイ設定の確認

MCPサーバーがデプロイ先の設定をユーザーに確認するケースです：

```json
{
  "message": "デプロイ設定を確認してください",
  "mode": "form",
  "requested_schema": {
    "type": "object",
    "properties": {
      "environment": {
        "type": "string",
        "enum": ["staging", "production"],
        "title": "デプロイ先環境"
      },
      "dry_run": {
        "type": "boolean",
        "title": "ドライランモード"
      },
      "version_tag": {
        "type": "string",
        "title": "バージョンタグ"
      }
    }
  }
}
```

## 注意点

- **MCPサーバー側の対応が必要**: エリシテーション機能を利用するには、MCPサーバー側がエリシテーションリクエストを送信するように実装されている必要がある
- **ユーザーの操作が必要**: エリシテーションが発生するとClaude Codeはユーザーの入力を待つため、完全な自動化フローでは注意が必要
- **フックによる自動応答**: `Elicitation` フックを使えば、エリシテーションにプログラム的に応答し、ダイアログをスキップすることも可能（別記事参照）
- **スキーマバリデーション**: フォームモードではJSONスキーマに基づくバリデーションが行われるため、不正な入力は送信前にキャッチされる
