## 原文（日本語に翻訳）

`#NNN` 参照を含むマークダウンリンクが、リンクされたURLではなく現在のリポジトリを誤って指していた問題を修正しました。

## 原文（英語）

Fixed markdown links containing `#NNN` references incorrectly pointing to the current repository instead of the linked URL

## 概要

マークダウン形式のリンクに `#123` などのGitHubのissue番号やPR番号の参照が含まれている場合、リンク先のURLではなく現在のリポジトリの該当番号のissue/PRに誤ってリンクされてしまう問題が修正されました。これによりClaudeが提示するリンクや、コード内のマークダウンリンクが正しいURLを指すようになります。

## 基本的な使い方

```markdown
<!-- 問題が発生していたリンクの例 -->
[修正のPR](https://github.com/other-org/other-repo/pull/42)

<!-- 修正前: 現在のリポジトリの #42 を指していた -->
<!-- 修正後: 正しく other-org/other-repo の PR #42 を指す -->
```

## 実践例

### 外部リポジトリのissueへのリンク

```markdown
# README.md や記事内での外部リンク

## 関連情報
- [バグ報告 #123](https://github.com/facebook/react/issues/123)
  <!-- 修正前: 現在のリポジトリの issue #123 にリンク -->
  <!-- 修正後: react リポジトリの issue #123 に正しくリンク -->

- [機能リクエスト #456](https://github.com/vercel/next.js/issues/456)
  <!-- 修正後: next.js リポジトリの正しいリンク -->
```

### Claude Codeの出力でのリンク

```bash
claude

> このバグに関連するissueを教えてください

# Claudeが返す例（修正後は正しいリンクを提示）:
# "関連する issue は [#1234](https://github.com/other-org/repo/issues/1234) です"
# → クリックすると正しい外部リポジトリのissueに飛ぶ
```

### PRのコメントやドキュメントでの使用

```markdown
## 変更履歴

- [upstream の修正 #789](https://github.com/expressjs/express/pull/789) を取り込みました
  <!-- 修正後: Express.js リポジトリの PR #789 に正しくリンク -->

- 詳細は [RFC #42](https://github.com/tc39/proposals/issues/42) を参照してください
  <!-- 修正後: tc39 リポジトリの正しいissueにリンク -->
```

## 注意点

- この問題はGitHubリポジトリ内で作業しているときに発生していました
- `#NNN` 形式のアンカーを含むURL（例: `https://example.com/page#section`）が影響を受けていました
- GitHubのAutolink参照（`#123` のみ）は引き続き現在のリポジトリを指します（これは正常な動作）
- 完全なURLを含むマークダウンリンク `[text](https://...)` の形式で修正が適用されます
- Claude Codeの出力内でリンクをクリックする際は、URLを確認してから移動することを推奨します
