## 原文（日本語に翻訳）

Agent view: ←キーで離脱したセッションは、マウスホバーや矢印キーで選択カーソルが移動した後も、視覚的にマークされたままになります

## 原文（英語）

Agent view: the session you pressed ← from stays visibly marked even after mouse hover or arrow keys move the selection

## 概要

Agent viewはClaude Codeで複数のバックグラウンドエージェントセッションを一覧・管理するための画面です。あるセッションにアタッチしている状態から `←`（左矢印キー）で離脱してagent viewの一覧に戻ると、これまではその後マウスホバーや矢印キーで選択カーソルを他の行に動かすと、直前まで自分がいたセッションがどれだったのか見た目上わからなくなっていました。v2.1.210では、`←` で離脱した直後のセッションが独立した「マーク」として視覚的に残り続けるようになり、選択カーソルを他の行に移動しても、元いたセッションを見失わずに再アタッチできます。

## 基本的な使い方

### 修正前の問題

```bash
claude

# あるバックグラウンドセッションにアタッチして作業
# ← キーで離脱し、agent viewの一覧に戻る

# 一覧内で矢印キーを動かしたり、マウスを他の行に
# ホバーしたりすると、選択ハイライトが移動してしまい
# 「さっきまでいたセッションはどれだったか」が
# 見た目上わからなくなる
```

### 修正後の動作

```bash
claude

# セッションAにアタッチ → ← で離脱
# → セッションAの行に「離脱元」を示すマークが付く

# 一覧内で ↑ / ↓ を押して選択カーソルをB、Cへ移動しても
# セッションAの行には引き続きマークが残る
# → マウスをB, Cの行にホバーしても同様にマークは消えない

# セッションAへ素早く戻りたい場合、マークを目印に
# 該当行へカーソルを合わせて → キーで再アタッチできる
```

## 実践例

### 複数セッションを行き来しながら状況を確認する

```bash
claude

# セッションA（リファクタリング中）にアタッチ
# 進捗を確認して ← で離脱

# agent view一覧で他のセッションB, Cの状態も
# ↑ / ↓ で確認して回る

# セッションAの行だけは「直前にいた場所」として
# マークされ続けているので、一通り確認し終えたあと
# 迷わずセッションAに戻れる
```

### マウス操作とキーボード操作が混在する場合

```bash
claude

# セッションDから ← で離脱
# マウスでセッションEの行にカーソルを合わせて詳細をチェック

# → セッションDの行のマークはホバーによって消えない
# → 選択ハイライト（現在カーソルがある行）と
#   離脱元マーク（セッションD）が別々に見分けられる
```

### 入力待ちのセッションを巡回してから元のセッションに戻る

```bash
claude

# フッターの ← 2 agents を見てagent viewへ移動
# セッションFにアタッチして質問に回答 → ← で離脱

# 続けてセッションGにアタッチして回答 → ← で離脱

# 一覧にはセッションFとGそれぞれの「最後に離脱した」
# マークではなく、直近に ← した1件が保持される仕様のため、
# 直前の操作対象を見失いにくい
```

## 注意点

- Agent viewの基本的なキー操作は、`↑` / `↓` で行移動、`→` でアタッチ、`←` でデタッチ（一覧に戻る）です。加えて `Shift+↑` / `Shift+↓` でセッションの並び替え、`Ctrl+T` でピン留め、`Ctrl+R` でリネームができます
- このマークはあくまで「直前に `←` で離脱した1セッション」を示すものであり、複数セッションを同時にマークし続けるものではありません
- アタッチ中の画面はターミナルの代替スクリーン（alternate screen）に入るため、通常のスクロールバックが効きません。空のプロンプトで `←` を押すか、ダイアログにフォーカスがある場合は `Ctrl+Z` で抜けられます
- スクリーンリーダーモードでの本マークの読み上げ対応状況については、[アクセシビリティ関連ページ](https://code.claude.com/docs/en/accessibility)の既知の制約もあわせて確認してください
