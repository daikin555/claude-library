## 原文（日本語訳）

`/doctor` の更新チェックを修正し、Homebrewでインストールした場合は設定ファイルのチャンネルではなく、そのcask（インストール元パッケージ）のチャンネルと比較するようにした

## 原文（英語）

Fixed `/doctor`'s update check to compare Homebrew installs against their cask's channel instead of the settings channel

## 概要

Claude CodeにはHomebrewの `claude-code` cask（安定版寄りの `stable` チャンネル相当）と、より新しいバージョンを追いかける `claude-code@latest` のような選択肢があります。これまで `/doctor` の更新チェックは、実際にインストールされているcaskの種類にかかわらず `settings.json` の `autoUpdaterChannel`（例: `latest`）を基準に最新バージョンかどうかを判定していたため、Homebrewでstable系caskを使っているユーザーに対して「更新が必要」といった誤った警告が出ることがありました。今回の修正で、Homebrewインストールについてはそのcask自身が追従しているチャンネルを基準に比較するようになり、誤検知が解消されました。

## 基本的な使い方

通常通り `/doctor`（またはエイリアスの `/checkup`）を実行するだけで、修正後の正しいロジックで更新状況が診断されます。

```text
/doctor
```

## 実践例

### Homebrewのstable caskを使っている場合

`brew install --cask claude-code` で導入し、設定ファイル上は `latest` チャンネルを指定しているようなケースです。以前はこの組み合わせで「最新版ではありません」という警告が不要に表示されることがありましたが、修正後はcaskが実際に追従しているチャンネル（stable相当）と比較されるため、警告が正確になります。

### `claude-code@latest` caskを使っている場合

より新しいリリースを追うために `claude-code@latest` caskを導入している場合も、そのcask自体のチャンネルを基準に比較が行われるため、`/doctor` の更新チェック結果がインストール実態と一致するようになります。

### CLIから直接診断したい場合

Claude Codeのセッションを開けない場合でも、シェルから同様の診断を実行できます。

```bash
claude doctor
```

## 注意点

- この修正はHomebrew経由でインストールした場合の更新チェックに限定される内容です。npmグローバルインストールやネイティブインストーラーなど他の導入方法には直接関係しません。
- `autoUpdaterChannel` の設定自体（`stable` / `latest` の切り替え）が不要になるわけではなく、Homebrew環境における比較対象の基準が是正されたという内容です。
