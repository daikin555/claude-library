## 原文（日本語に翻訳）

バックグラウンドのサブエージェント完了通知に経過時間を追加（例："Agent completed · 3h 2m 5s"）

## 原文（英語）

Added elapsed duration to background subagent completion notifications (e.g. "Agent completed · 3h 2m 5s")

## 概要

Claude Code v2.1.144で、バックグラウンドのサブエージェントが完了したときの通知に経過時間が表示されるようになりました。例えば「Agent completed · 3h 2m 5s」のような形式で、タスクが完了するまでにかかった時間を一目で確認できます。長時間かかるタスクのパフォーマンス把握や、今後の作業見積もりに役立てることができます。

## 基本的な使い方

バックグラウンドでサブエージェントを実行すると、完了時に以下のような通知が表示されます。

```
Agent completed · 3h 2m 5s
```

時間の表示形式は以下のようにタスクの所要時間に応じて変化します：

```
Agent completed · 45s          # 45秒で完了
Agent completed · 5m 30s       # 5分30秒で完了
Agent completed · 1h 15m 0s    # 1時間15分で完了
Agent completed · 3h 2m 5s     # 3時間2分5秒で完了
```

## 実践例

### 大規模なコード解析タスクの所要時間を把握する

大量のファイルを処理するタスクで、実際にかかった時間を記録・把握できます。

```bash
# バックグラウンドで大規模解析を依頼
claude --bg "プロジェクト全体のコードを解析してセキュリティ脆弱性レポートを作成して"

# タスク完了時に通知が表示される
# 例: "Agent completed · 12m 34s"
# → このプロジェクトの解析には約12分かかることが把握できる
```

### 定期タスクの実行時間を最適化する

CI/CDパイプラインなどで定期実行するタスクの所要時間を把握し、スケジュールを最適化できます。

```bash
# テスト実行タスクをバックグラウンドで実行
claude --bg "全テストを実行して失敗したテストの修正案を提案して"

# 完了通知: "Agent completed · 8m 20s"
# → テスト実行と分析に約8分かかることが分かる
# → CIパイプラインのタイムアウト設定の参考にできる
```

### 複数タスクの効率比較

同様のタスクを異なる方法で実行した場合の所要時間を比較し、より効率的なアプローチを選択できます。

```bash
# アプローチA: ファイル単位で処理
claude --bg "各ファイルを順番に解析してドキュメントを生成して"
# 完了: "Agent completed · 25m 10s"

# アプローチB: バッチ処理で一括処理
claude --bg "全ファイルをまとめて解析してドキュメントを一括生成して"
# 完了: "Agent completed · 8m 45s"

# → バッチ処理の方が約3倍速いことが判明
```

## 注意点

- 表示される経過時間はサブエージェントがタスクを開始してから完了するまでの実時間です
- 時間の表示形式は `Xh Ym Zs` 形式で、省略なく表示されます（秒は必ず表示されます）
- この機能はバックグラウンドサブエージェントが対象です（インタラクティブセッションには適用されません）
- 通知はセッション完了時に一度だけ表示されます
