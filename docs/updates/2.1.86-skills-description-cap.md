## 原文（日本語に翻訳）

コンテキスト使用量を削減するため、`/skills` 一覧でのスキル説明を250文字に制限しました。

## 原文（英語）

Skill descriptions in the `/skills` listing are now capped at 250 characters to reduce context usage

## 概要

Claude Codeの `/skills` コマンドでスキル一覧を表示する際、各スキルの説明文が非常に長くなることがありました。詳細な説明はスキルの理解には役立ちますが、コンテキストウィンドウの貴重なスペースを消費する問題もありました。v2.1.86からスキル説明は最大250文字に制限され、コンテキスト使用量が最適化されます。

## 基本的な使い方

`/skills` コマンドでスキル一覧を確認すると、各スキルの説明が最大250文字で表示されます。

```bash
# スキル一覧を表示
/skills

# 出力例
Available skills:
- my-skill: このスキルは○○の機能を提供します。使用方法は...（最大250文字で切り詰め）
- another-skill: 別の機能の説明...
```

## 実践例

### ユースケース: 多数のスキルが登録された環境での利用

```bash
# 多くのスキルが設定されているプロジェクトで
/skills
# → 各スキルの説明が250文字に制限されるため
#   一覧が簡潔に表示され、コンテキストを節約できます
```

### ユースケース: スキルのFrontmatter最適化

スキルを作成する場合、説明文を250文字以内に収めることで表示が切り詰められることなく全文が表示されます。

```yaml
---
title: "My Skill"
description: "このスキルは特定のタスクを自動化します。コードの生成、レビュー、テストの実行をサポートします。プロジェクト固有の規約に従って作業します。"
# ↑ 250文字以内に収めることで全文が表示されます
---
```

長い説明を記述する場合は、重要な情報を最初の250文字以内に含めるように設計することを推奨します。

## 注意点

- 250文字を超える説明は切り詰められ、`/skills` の一覧では全文は表示されません
- スキルファイル内の説明文自体は削除・変更されません（表示のみの制限）
- スキルの詳細を確認したい場合は、スキルファイルを直接参照してください
- スキル説明の最適化（250文字以内）はスキル作成時のベストプラクティスとして取り入れることを推奨します
