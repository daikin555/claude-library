## 原文（日本語に翻訳）

デフォルト入力デバイスが変更された後、長時間セッションで macOS の音声ディクテーションが無音をキャプチャする問題を修正

## 原文（英語）

Fixed voice dictation on macOS capturing silence in long-running sessions after the default input device changes

## 概要

macOS 環境で Claude Code の音声入力（ディクテーション）機能を使用している際、デフォルトのオーディオ入力デバイスが変更された後に音声が録音されなくなる（無音のみキャプチャされる）バグが修正されました。長時間のセッション中にヘッドセットの接続・切断や入力デバイスの切り替えを行った際に発生していた問題です。修正後はデバイスの変更を自動検出して音声キャプチャを継続します。

## 基本的な使い方

音声入力は Claude Code のショートカットキーで起動できます。

```bash
# Claude Code を起動
claude

# 音声入力のショートカット（デフォルト）
# macOS: Ctrl+Shift+V で音声入力を開始/停止
```

## 実践例

### ヘッドセット切り替え後の音声入力

長時間作業中にヘッドセットを交換した後でも、音声入力が正常に動作するようになりました。

```
修正前のフロー:
1. Claude Code を起動（内蔵マイクで音声入力）
2. ヘッドセットを接続（デフォルト入力デバイスが変わる）
3. ❌ 以降の音声入力が無音扱いになる
4. Claude Code の再起動が必要

修正後のフロー:
1. Claude Code を起動（内蔵マイクで音声入力）
2. ヘッドセットを接続（デフォルト入力デバイスが変わる）
3. ✅ ヘッドセットのマイクで引き続き音声入力が正常動作
```

### オンライン会議と並行して使用

Zoom や Google Meet などがオーディオデバイスを切り替えた後でも、Claude Code の音声入力が引き続き動作します。

```
1. Claude Code を音声入力モードで使用
2. ビデオ会議アプリが起動してオーディオデバイスを変更
3. 会議終了後もデバイスが切り替わったままの状態
4. ✅ 修正後: Claude Code が新しいデバイスを自動検出して音声入力継続
```

### 長時間コーディングセッションでの使用

数時間にわたるコーディングセッション中に複数回デバイスを切り替えた場合でも、再起動不要で音声入力を継続できます。

## 注意点

- macOS 固有の修正です（Linux・Windows には適用されません）
- 長時間セッション（数時間以上）で特に効果があります
- デバイス変更後に音声入力が無効になった場合は、Claude Code を再起動することで回避できます（修正前の対処法）
- この修正により再起動不要で継続使用が可能です
