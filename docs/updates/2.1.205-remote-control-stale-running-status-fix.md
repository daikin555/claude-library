## 原文（日本語訳）

Webおよびモバイルの Remote Control パネルで、バックグラウンドタスクが古い「実行中(Running)」ステータスのまま表示され続ける不具合を修正。メンバーシップ（接続構成）が変化するたびにタスクの完全な状態を転送するようにした。

## 原文（英語）

Fixed background tasks in the web and mobile Remote Control panels showing stale "Running" status by forwarding full task state on every membership change

## 概要

Remote Control は、手元のマシンで動いている Claude Code のセッションを claude.ai/code のWeb画面やiOS/Android向けのClaudeアプリから引き継いで操作できる機能です。セッションの処理自体はローカルマシン上で継続され、クラウドには移動しません。今回の修正では、バックグラウンドで実行中のタスクが実際には完了しているにもかかわらず、Web・モバイルのRemote Controlパネル上では「実行中」のステータスのまま表示され続けてしまう不具合が解消されました。原因は、新しいデバイスが接続するなどしてセッションの接続構成（membership）が変わったタイミングで、タスクの状態が正しく再送されていなかったことにあり、以後はメンバーシップ変更のたびにタスクの完全な状態を転送するよう修正されています。

## 基本的な使い方

Remote Controlのセッションを開始し、バックグラウンドタスク（例えばサブエージェントによる並列実行や `/bg` 的な長時間実行タスク）を走らせた状態で、別のデバイスからパネルを開いて状態を確認するという使い方が基本になります。

```text
# ローカルのターミナルでClaude Codeを起動し、Remote Controlを有効化
claude --remote-control
```

セッション開始後に表示されるQRコードやリンクから、スマートフォンのClaudeアプリやブラウザ版の claude.ai/code に接続すると、実行中のタスク一覧やステータスがパネルに表示されます。

## 実践例

### 外出先でバックグラウンドタスクの完了を確認する

自宅や職場のPCで大きめのリファクタリングやテスト実行をバックグラウンドタスクとして開始し、外出先からスマートフォンのClaudeアプリでRemote Controlパネルを開いて進捗を確認するケースです。今回の修正前は、タスクが実際にはローカル側で完了していても、モバイル側のパネルには「実行中」のまま古い状態が残ってしまうことがありました。修正後は、パネルを開いた（＝新しい接続がメンバーシップに加わった）タイミングで最新のタスク状態が転送されるため、完了しているタスクは正しく「完了」として表示されます。

```text
> 大きなテストスイートをバックグラウンドで実行して、終わったら教えて
```

### 複数デバイスを行き来しながら長時間タスクを監視する

PCのブラウザで開いていたRemote Controlパネルから、途中でタブレットやスマートフォンに接続を切り替えて同じセッションを監視するようなケースでも、この修正が効いてきます。接続デバイスが入れ替わる（メンバーシップが変化する）たびに、そのタイミングでの正確なタスク状態一式が新しい接続先に送られるようになったため、デバイスを切り替えた直後に古い「実行中」表示に惑わされることがなくなります。

```text
> 依存パッケージの一括アップデートとビルド確認をバックグラウンドで進めて
```

## 注意点

- この修正はRemote Controlの表示上のバグ修正であり、バックグラウンドタスクそのものの実行ロジックやスケジューリングを変更するものではありません。
- Remote Controlの利用にはClaude Code v2.1.51以降が必要で、Pro・Max・Team・Enterpriseプランで利用できます。Team・Enterpriseプランでは管理者が事前にRemote Controlを有効化する必要があります。
- Remote Control自体は処理をクラウドに移動させる機能ではなく、あくまでローカルで動作中のセッションへ他のデバイスから接続して操作・閲覧する仕組みです。ネットワークの切断やスリープなど、接続状況によっては状態の反映に多少のタイムラグが生じる可能性があります。
