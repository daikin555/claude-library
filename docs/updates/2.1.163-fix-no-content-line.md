## 原文（日本語に翻訳）

`/mcp` や `/plugins` などのパネルダイアログを閉じた後に、トランスクリプトに "(no content)" という余分な行が残る問題を修正。

## 原文（英語）

Fixed a stray "(no content)" line left in the transcript after closing panel dialogs like /mcp and /plugins

## 概要

`/mcp` や `/plugins` などのパネルダイアログを閉じた際に、会話のトランスクリプトに不要な `(no content)` という行が残るバグが修正されました。この不要なテキストがトランスクリプトを汚染し、会話履歴が見づらくなっていた問題が解消されます。

## 基本的な使い方

修正後は特別な操作は不要です。`/mcp` や `/plugins` などのパネルダイアログを開いて閉じても、トランスクリプトに `(no content)` が残らなくなります：

```
/mcp
# MCP ダイアログを開いて確認し、閉じる
# 修正前: "(no content)" がトランスクリプトに追加される
# 修正後: トランスクリプトが汚染されない
```

## 注意点

- この問題は UI の表示バグです。実際の会話内容やデータには影響しませんでした。
- 影響を受けるダイアログ: `/mcp`、`/plugins` などのパネル形式のダイアログ。
