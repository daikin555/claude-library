## 原文（日本語訳）

`/cd` にディレクトリパスの補完候補が追加され、`/add-dir` と同じ挙動になった。

## 原文（英語）

Added directory path suggestions to `/cd`, matching `/add-dir` behavior

## 概要

`/cd` はセッション中に作業ディレクトリを別のフォルダへ移動できるコマンドですが、これまでパス入力時の補完候補が表示されず、正確なパスを手入力する必要がありました。今回のアップデートで、`/add-dir` と同様にディレクトリパスの候補がインラインで表示されるようになり、タイプ量を減らしつつ入力ミスも防げるようになりました。

## 基本的な使い方

Claude Codeのセッション内で `/cd` に続けてパスの一部を入力すると、候補となるディレクトリが一覧表示されます。

```text
/cd ../
```

表示された候補から矢印キーなどで選択し、Tabキーで補完してEnterを押すと、そのディレクトリにセッションが移動します。

## 実践例

### 兄弟プロジェクトへの素早い移動

モノレポやワークスペース配下に複数のプロジェクトが並んでいる場合、パス全体を覚えていなくても入力補完だけで目的のディレクトリまでたどり着けます。

```text
/cd ../backend
```

候補一覧から `backend/` を選択するだけで、フルパスをタイプする必要がありません。

### 深い階層のディレクトリへの移動

ネストの深いディレクトリ構成でも、`/`区切りごとに候補が更新されるため、タイプミスによる「ディレクトリが見つからない」エラーを避けられます。

```text
/cd packages/
```

続けて入力すると `packages/` 配下のサブディレクトリ候補が表示され、目的の場所まで段階的に絞り込めます。

## 注意点

- 補完候補はカレントディレクトリからの相対パスを基準に表示されます。
- `/cd` はセッション自体を新しいフォルダへ移動する点が `/add-dir`（アクセス可能なディレクトリを追加するだけ）と異なるため、用途に応じて使い分けが必要です。
- 補完機能はあくまで入力補助であり、実際に存在しないパスを選択・入力した場合はこれまで通りエラーになります。
