## 原文（日本語に翻訳）

編集を行っていないプラン承認が「（ユーザーによって編集済み）」とラベル付けされ、古いスナップショットでプランファイルを上書きしてしまう問題を修正しました

## 原文（英語）

Fixed plan approvals without edits being labeled "(edited by user)" and overwriting the plan file with a stale snapshot

## 概要

Claude Code v2.1.210で修正された、プランモード（Plan Mode）に関するバグです。プランモードでは、Ctrl+Gなどでプランファイルをエディタで開いて編集し、フィードバックを与えることができますが、ユーザーが実際には何も編集せずにそのまま承認しただけの場合でも、以前は誤って「(edited by user)」というラベルが付き、さらに編集前の古いスナップショットの内容でプランファイルを上書きしてしまうことがありました。今回の修正で、実際に編集が行われた場合のみ正しく「編集済み」と判定されるようになり、無編集の承認でプランファイルが古い内容に巻き戻されることもなくなりました。

## 基本的な使い方

### 修正前の問題

```text
1. プランモードでClaudeがプランを提示
2. Ctrl+G でプランファイルをエディタで開く（内容は確認のみ、変更なし）
3. エディタを閉じてそのままプランを承認
4. 修正前:
   - プランが「(edited by user)」と誤ってラベル付けされる
   - さらに、プランファイルが編集前の古いスナップショットの
     内容で上書きされてしまう（実際の最新プラン内容と食い違う）
```

### 修正後の動作

```text
1. プランモードでClaudeがプランを提示
2. Ctrl+G でプランファイルを開くが、内容は変更しない
3. プランを承認
4. 修正後:
   - 実際に変更がなければ「編集済み」ラベルは付かない
   - プランファイルは最新の内容のまま保たれ、
     古いスナップショットで上書きされることもない
```

## 注意点

- この修正は Claude Code v2.1.210 で実装されました
- プランファイルの保存先は `plansDirectory` 設定でカスタマイズ可能です（v2.1系で追加された設定）
- 「編集済み」かどうかの判定は、実際にファイル内容がユーザーによって変更されたかどうかに基づくようになりました
- プランを意図的に編集してフィードバックを与えたい場合は、引き続きCtrl+Gでエディタを開いて内容を変更し保存することで、その変更内容がプランに反映されます
- 古いスナップショットによる上書きが発生していた場合、最悪プラン内容の一部が失われる可能性があったため、この修正はプランモードを日常的に使うユーザーにとって重要な安定性向上です
