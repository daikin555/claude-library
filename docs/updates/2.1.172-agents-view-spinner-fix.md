## 原文（日本語に翻訳）

ワーカーが返答した後もエージェントビューでセッションが「Working」状態のままビジースピナーが最大30秒表示され続ける問題を修正しました

## 原文（英語）

Fixed the agents view keeping a session under Working with a busy spinner for up to 30 seconds after the worker replied

## 概要

`claude agents` のエージェント管理画面において、バックグラウンドエージェントが処理を完了して返答した後も、ビジースピナーと「Working」状態が最大30秒間表示され続ける問題が修正されました。修正後は、エージェントが完了したタイミングで即座にステータスが更新されます。

## 基本的な使い方

この修正は自動的に適用されます。エージェントの完了状態がリアルタイムで反映されるようになります。

```bash
# エージェント管理画面を開く
claude agents

# 修正前: エージェント完了後も最大30秒間 "Working..." と表示
# 修正後: 完了と同時に正確なステータスに更新される
```

## 実践例

### バックグラウンドエージェントの監視

```
# エージェントを起動してステータスを確認
claude agents

# 各エージェントのステータス:
# [完了]  コードレビューエージェント  ← 即座に完了表示
# [実行中] テスト実行エージェント
# [完了]  ドキュメント生成エージェント ← 即座に完了表示
```

## 注意点

- この修正はUI表示の問題であり、実際の処理完了タイミングには影響しません
- エージェントの結果はステータス更新直後から参照できます
