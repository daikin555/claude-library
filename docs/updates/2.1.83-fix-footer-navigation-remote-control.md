## 原文（日本語に翻訳）

設定駆動モードでフッターナビゲーションが不可視のリモートコントロールピルを選択する問題を修正しました。

## 原文（英語）

Fixed footer navigation selecting an invisible Remote Control pill in config-driven mode

## 概要

設定ファイルによって Remote Control の UI 要素（ピル）が非表示になっている設定駆動モードにおいて、フッターのナビゲーション操作（Tab や矢印キーなど）が不可視の Remote Control ピルを選択してしまい、ナビゲーションがスタックしたように見える問題が修正されました。

## 基本的な使い方

フッターナビゲーションが可視の要素のみを対象にナビゲートするようになりました。

```bash
$ claude
# フッターナビゲーション（Tab キーなど）を使用
# 修正後: 非表示の Remote Control ピルをスキップして
# 可視の要素のみにフォーカスが当たる
```

## 実践例

### ユースケース: 設定駆動モードでのキーボードナビゲーション

Remote Control を使用しない設定でも、フッターのナビゲーションが正常に動作します。

```json
// settings.json で Remote Control を非表示にしている設定
{
  "remoteControl": { "enabled": false }
}
```

```bash
$ claude
# Tab でフッターをナビゲート
# 修正後: Remote Control ピル（非表示）をスキップして
# 他の可視要素（コンフィグ、ヘルプなど）にフォーカス
```

### ユースケース: スムーズなフッターナビゲーション

不可視要素でナビゲーションが詰まらなくなり、スムーズに操作できます。

```bash
# フッターナビゲーションの流れ（修正後）
Tab → [Config] → [Help] → [Exit]  # 不可視ピルはスキップ
# 修正前: Tab → [見えない Remote Control] → 詰まる
```

## 注意点

- 「ピル（pill）」は UI のタブや選択可能な UI コンポーネントを指します
- この問題は Remote Control 機能が設定によって非表示になっている場合のみ発生していました
- フッターナビゲーションは可視性チェックを含むよう改善されました
