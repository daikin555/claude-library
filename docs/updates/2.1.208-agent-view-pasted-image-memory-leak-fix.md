## 原文（日本語訳）

エージェントビューにおいて、peek返信を送信した後も、貼り付けた画像が画面のライフタイムの間ずっとメモリに保持され続けるメモリリークを修正した。

## 原文（英語）

Fixed a memory leak in the agent view where pasted images were retained for the screen's lifetime after sending peek replies

## 概要

[エージェントビュー](https://code.claude.com/docs/en/agent-view)（`claude agents`）は複数のバックグラウンドセッションを一画面で管理できる機能で、行を選択して `Space` キーを押すと開くpeekパネルから、画像を貼り付けて返信を送ることができます。これまでは、peekパネルから画像付きの返信を送信した後も、その画像データがエージェントビュー画面自体を閉じるまでメモリ上に保持され続けるメモリリークがありました。返信ごとに画像がスクリーンショットや図などのデータであることも多く、返信回数が増えるほどメモリ使用量が積み上がっていく問題でした。今回の修正で、返信送信後は不要になった画像データが適切に解放されるようになりました。

## 基本的な使い方

エージェントビューでバックグラウンドセッションに画像付きで返信する際の基本操作は変わりません。

```text
claude agents
# 行を選択して Space でpeekパネルを開く
# 画像を貼り付けてから Enter で返信を送信
```

v2.1.208以降では、返信送信後に画像データがメモリから解放されるようになりました。

## 実践例

### 多数のバックグラウンドセッションにスクリーンショット付きで頻繁に返信する運用

UIの確認待ちなど、複数のバックグラウンドセッションに対して頻繁にスクリーンショットを貼り付けて返信するような使い方をしている場合、修正前は返信を重ねるごとにエージェントビューのメモリ使用量が増加していました。

```text
claude agents
# セッションAをpeek → スクリーンショットを貼り付けて返信
# セッションBをpeek → 別のスクリーンショットを貼り付けて返信
# ...を繰り返す
```

修正後は、それぞれの返信送信後に画像データが解放されるため、返信回数を重ねてもメモリ使用量が積み上がりません。

### エージェントビューを長時間開きっぱなしにする運用

一日を通してエージェントビューを開いたまま、複数のバックグラウンドタスクを監視・返信し続けるような運用でも、メモリ使用量の増加を抑えられます。

```text
claude agents
# 画面を閉じずに、一日を通して複数セッションへpeek返信を繰り返す
```

## 注意点

- この修正は内部的なメモリ管理の改善であり、ユーザーが行う操作や画面上の見た目には変化はありません。
- 特に、画像付きのpeek返信を頻繁に行い、かつエージェントビューを長時間開いたままにする使い方をしている場合に影響が大きい不具合でした。
- 影響を受けていた場合は、Claude Codeを最新版（v2.1.208以降）に更新することで解消します。
