## 原文（日本語訳）

Bashコール内で作成されたPRの出力がインラインの3万文字制限を超えていた場合に、セッションとPRの紐付けが行われないことがあった不具合を修正。

## 原文（英語）

Fixed session-to-PR linking missing a PR created in a Bash call whose output exceeded the 30K inline limit

## 概要

Claude Codeには、`gh pr create` などでPRを作成すると、そのPRを作成したセッションIDを自動的に記録し、`claude --from-pr <PR番号>` や `/resume` のピッカーからPR番号・URLだけでそのセッションに戻れるようにする「セッション-PR紐付け」機能があります。今回の修正は、PR作成コマンドを実行したBashツール呼び出しの出力が3万文字（デフォルトのインライン出力上限）を超えていた場合に、出力の途中省略（ミドル・トランケーション）によってPR URLなどの手がかりがうまく検出できず、セッションとPRの紐付けが漏れてしまうことがあった不具合への対処です。

## 基本的な使い方

通常の利用フローでは、特別な操作は不要です。Claude Codeが `gh pr create` を実行してPRを作成すると、裏側で自動的にセッションIDとPRが紐付けられます。

```text
> このバグ修正をコミットしてPRを作成して
```

PR作成後は、PR番号だけを指定してそのセッションに戻ることができます。

```bash
claude --from-pr 123
```

もしくは `/resume` のピッカーにPRのURLをそのまま貼り付けて検索することでも、該当セッションを見つけられます。

## 実践例

### 出力の多いテストやビルドと同じBashコールでPRを作成する場合

`gh pr create` の前後に大量のログを出すコマンド（テストスイートの全件実行やビルドログの出力など）を同じBashツール呼び出しの中でまとめて実行すると、出力全体が3万文字を超え、Claude Codeは冒頭と末尾を残して中間部分を省略します。修正前はこの省略によってPR作成の痕跡が検出できず、セッションとPRが紐付かないことがありました。修正後は、こうしたケースでも作成されたPRとセッションが正しく紐付けられ、`claude --from-pr` で問題なく復帰できます。

```bash
npm test && npm run build && gh pr create --title "fix: ..." --body "..."
```

### 大規模な差分に対して `gh pr create --body` を使う場合

差分が大きいリポジトリで自動生成されたPR本文（変更ファイル一覧や詳細な説明）が長大になるケースでも、Bashの出力サイズが膨らみやすくなります。このような場合でも、修正後はPR作成コマンドの実行結果が正しく検出され、後から `/resume` のピッカーでPR URLを検索して該当セッションを呼び出せます。

```bash
gh pr create --title "feat: add large migration" --body "$(cat pr_body.md)"
```

## 注意点

- Bashツールの出力キャプチャには既定で3万文字（`BASH_MAX_OUTPUT_LENGTH` で調整可能）という上限があり、これを超える出力は先頭と末尾を残して中間が省略されます。今回の修正はこの省略処理とセッション-PR紐付けの検出ロジックの相互作用に起因する不具合であり、出力上限そのものが撤廃されたわけではありません。
- セッション-PR紐付けは、Claude Code自身が `gh pr create` などでPRを作成した場合にのみ機能します。ユーザーが手動で作成したPRにはセッション情報が紐付きません。
- セッション情報はローカルに保存されるため、`claude --from-pr` はPRを作成したマシンと異なる端末では利用できません。
