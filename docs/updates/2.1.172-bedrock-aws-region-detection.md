## 原文（日本語に翻訳）

Amazon Bedrockが、`AWS_REGION` が設定されていない場合に `~/.aws` の設定ファイルからAWSリージョンを読み取るようになりました。AWS SDKの優先順位に従っており、`/status` でリージョンの取得元を確認できます。

## 原文（英語）

Amazon Bedrock now reads the AWS region from `~/.aws` config files when `AWS_REGION` isn't set, matching AWS SDK precedence; `/status` shows where the region came from

## 概要

Amazon Bedrock経由でClaude Codeを使用する際、環境変数 `AWS_REGION` が未設定でも `~/.aws/config` や `~/.aws/credentials` から自動的にリージョンを検出するようになりました。AWS SDKが採用する標準的な優先順位に従うため、既存のAWS CLI設定をそのまま活用できます。

## 基本的な使い方

`~/.aws/config` が正しく設定されていれば、追加設定不要でリージョンが自動検出されます：

```ini
# ~/.aws/config の例
[default]
region = us-east-1

[profile my-project]
region = ap-northeast-1
```

リージョンの取得元を確認するには `/status` を実行：

```
/status
```

出力例：
```
AWS Region: ap-northeast-1 (from ~/.aws/config)
```

## 実践例

### プロファイルを使い分けるケース

```bash
# 特定のAWSプロファイルでClaude Codeを起動
AWS_PROFILE=my-project claude

# ~/.aws/config の [profile my-project] セクションのリージョンが使用される
```

### 複数環境での運用

```ini
# ~/.aws/config
[profile production]
region = us-east-1

[profile staging]
region = ap-northeast-1
```

```bash
# ステージング環境でClaudeを使う
AWS_PROFILE=staging claude

# 本番環境でClaudeを使う
AWS_PROFILE=production claude
```

### CI/CD環境での設定

```yaml
# GitHub Actions例
- name: Run Claude Code with Bedrock
  env:
    AWS_ROLE_ARN: arn:aws:iam::123456789:role/claude-role
    # AWS_REGION は不要 — ~/.aws/config から自動取得
  run: claude --bedrock "コードレビューを実施してください"
```

## 注意点

- **優先順位**: `AWS_REGION` 環境変数 > `~/.aws/config` の順で評価されます。明示的に `AWS_REGION` を設定した場合はそちらが優先されます
- **取得元の確認**: `/status` コマンドでリージョンがどこから取得されたかを確認できます
- **名前付きプロファイル**: `AWS_PROFILE` 環境変数で使用するプロファイルを指定できます
