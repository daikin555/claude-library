# 修正：ポリシー管理プラグインが最初にインストールしたプロジェクト以外で自動更新されないバグ

## 原文（日本語に翻訳）

ポリシー管理プラグインが最初にインストールしたプロジェクト以外のプロジェクトで実行した場合に自動更新されていなかった問題を修正した。

## 原文（英語）

Fixed policy-managed plugins never auto-updating when running from a different project than where they were first installed.

## 概要

組織のポリシーで管理されているプラグインが、最初にインストールされたプロジェクト以外のプロジェクトでClaude Codeを実行した場合に自動更新されないバグが修正されました。複数のプロジェクトを扱う開発環境で、ポリシー管理プラグインを常に最新の状態に保てるようになります。

## 基本的な使い方

### 修正後の正しい動作

```bash
# プロジェクトAでプラグインをインストール
cd /projects/project-a
claude
# → ポリシー管理プラグインがインストールされる

# プロジェクトBでClaude Codeを実行
cd /projects/project-b
claude
# 修正前: プラグインが自動更新されなかった
# 修正後: プラグインが正しく自動更新される
```

### ポリシー管理プラグインの確認

```bash
# インストール済みプラグインの確認
/plugins
# または
claude plugins list
```

## 注意点

- **v2.1.108で修正済み**: アップデート後は自動的に修正が適用されます。
- **ポリシー管理プラグイン**: 組織の管理者が設定したプラグイン（MCPサーバーなど）で、ユーザーが個別に無効化できないものが対象です。
- **個人プラグインは影響なし**: ユーザーが自分でインストールした個人プラグインは、この問題の影響を受けていませんでした。
