## 原文（日本語に翻訳）

影響を受ける設定でのサブエージェントの生成とセッションタイトル生成時に発生していた `thinking.disabled.display: Extra inputs are not permitted` 400 エラーを修正しました。

## 原文（英語）

Fixed `thinking.disabled.display: Extra inputs are not permitted` 400 errors on subagent spawns and session-title generation for affected configurations.

## 概要

特定の設定（`thinking.disabled`）を使用している環境でサブエージェントを起動したり、セッションタイトルを自動生成しようとしたときに 400 エラーが発生するバグが修正されました。このエラーにより、サブエージェントを使用するワークフローやセッションタイトルの自動生成が失敗していました。

## 基本的な使い方

このバグ修正により、以下の操作が正常に動作するようになりました：

```bash
# thinking が無効化された設定でサブエージェントを使用
claude
# Agent ツールや Task ツールを使ってサブエージェントを呼び出しても 400 エラーが発生しなくなった
```

## 実践例

### 影響を受けていたシナリオ

**修正前（エラーが発生していた）：**
```
設定: thinking.disabled = true の環境でサブエージェントを呼び出し
→ エラー: 400 Bad Request
  "thinking.disabled.display: Extra inputs are not permitted"
```

**修正後（正常に動作）：**
```
設定: thinking.disabled = true の環境でサブエージェントを呼び出し
→ サブエージェントが正常に起動し処理を実行
```

### セッションタイトル自動生成

```bash
# 新しいセッションを開始
claude
# セッションタイトルが自動的に生成される
# 以前は thinking.disabled 設定時にエラーが発生していた
```

## 注意点

- このバグは `thinking.disabled` 関連の設定を持つ環境でのみ発生していました
- Anthropic API の Extended Thinking 機能を無効化している場合や、特定のモデル設定を使用している場合に影響がありました
- バグ修正のため、ユーザー側での特別な対応は不要です
