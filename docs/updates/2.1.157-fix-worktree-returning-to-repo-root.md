## 原文（日本語に翻訳）

`--worktree` および `--worktree --tmux` が、現在リンクされたワークツリーではなく正規のリポジトリルートに戻ってしまう問題を修正しました。

## 原文（英語）

Fixed `--worktree` and `--worktree --tmux` returning to the canonical repo root instead of the current linked worktree

## 概要

`--worktree` オプションを使用してセッションを開始した場合、セッション終了後に元のリンクされたワークツリーのディレクトリではなく、正規のリポジトリルート（`.git` が存在するディレクトリ）に移動してしまうバグがありました。`--tmux` と組み合わせて使用した場合も同様の問題が発生していました。このバグはv2.1.157で修正されました。

## 影響を受けていた状況

- `git worktree add` で作成したリンクワークツリー内で `claude --worktree` を実行していた場合
- `claude --worktree --tmux` を使用してtmuxセッション内でワークツリー作業をしていた場合
- 複数のワークツリーを並行して使用し、各ワークツリーで独立した作業を行っていた場合

## 注意点

- v2.1.157 以降では自動的に修正されており、特別な操作は不要です
