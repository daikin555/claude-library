# /plugin Uninstallが「Enabled」と表示していたバグを修正

## 原文（日本語に翻訳）

`/plugin` Uninstallが「Uninstalled」の代わりに「Enabled」と報告していた問題を修正しました。

## 原文（英語）

Fixed `/plugin` Uninstall reporting "Enabled" instead of "Uninstalled"

## 概要

`/plugin` コマンドでプラグインをアンインストールした際に、成功メッセージとして「Uninstalled」と表示されるべきところ、誤って「Enabled」と表示されていたUIバグが修正されました。プラグインのアンインストール操作自体は正常に動作していましたが、ユーザーが混乱する原因になっていました。

## 基本的な使い方

```bash
# プラグインのアンインストール
/plugin uninstall my-plugin

# v2.1.126以前:
# ✓ Enabled   ← 誤ったメッセージ（実際はアンインストール済み）

# v2.1.126以降:
# ✓ Uninstalled   ← 正しいメッセージ
```

## 注意点

- アンインストールの動作自体は以前から正しく機能していました（表示のみのバグ）
- この修正はUIの表示修正であり、プラグイン管理の動作に変更はありません
