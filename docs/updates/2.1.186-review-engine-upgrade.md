# /review \<pr\> が /code-review medium と同じエンジンを使用するように変更

## 原文（日本語に翻訳）
`/review <pr>` が `/code-review medium` と同じレビューエンジンを使用するように変更されました。

## 原文（英語）
Changed `/review <pr>` to use the same review engine as `/code-review medium`

## 概要
PRレビューを行う `/review <pr>` コマンドが、ローカルコードレビューに使用される `/code-review medium` と同じエンジンで動作するようになりました。これにより、PRレビューの品質と一貫性が向上し、両コマンドで同等のレビュー精度が得られます。

## 基本的な使い方

### PRのレビュー
```
/review 123
```
または
```
/review https://github.com/owner/repo/pull/123
```

### ローカルコードのレビュー（同じエンジン）
```
/code-review medium
```

## 実践例

### PRのレビューを実行
```
/review 456
```
プルリクエスト #456 を `/code-review medium` と同等の品質でレビューします。バグ、セキュリティ問題、コードの品質に関する分析が行われます。

### 変更前との違い
この変更以前：
- `/review <pr>` は独自のシンプルなレビューロジックを使用
- `/code-review medium` はより高度なマルチエージェントレビューエンジンを使用

この変更以後：
- 両方とも同じレビューエンジンを使用
- PRレビューの品質が向上

### レビューレベルの選択
`/code-review` には複数のレベルがあります：
```
/code-review low     # 高信頼度の問題のみ（少数・精度重視）
/code-review medium  # バランスの取れたレビュー（/review <pr>と同等）
/code-review high    # 広範囲なカバレッジ（不確実な指摘も含む）
```

## 注意点
- この変更により `/review <pr>` の出力形式や詳細度が変わる場合があります
- レビューの品質向上により、処理時間が若干増加する可能性があります
- PRのURLまたは番号（リポジトリが設定されている場合）で指定できます
