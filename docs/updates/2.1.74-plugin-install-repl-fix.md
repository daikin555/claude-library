## 原文（日本語に翻訳）

ローカルソースを持つマーケットプレイスプラグインのREPL内での`/plugin install`が失敗していた問題を修正

## 原文（英語）

Fixed `/plugin install` failing inside the REPL for marketplace plugins with local sources

## 概要

Claude CodeのREPL（対話型セッション）内で`/plugin install`コマンドを実行した際に、ローカルソースを持つマーケットプレイスプラグインのインストールが失敗していたバグが修正されました。修正後は、REPL内からでもローカルソース付きのプラグインを正常にインストールできるようになります。

## 基本的な使い方

```bash
# REPL内でプラグインをインストール（修正後は正常動作）
/plugin install my-local-plugin
```

## 実践例

### REPL内でのプラグインインストール

```
# Claude Code REPL 内で
/plugin install

# マーケットプレイスからローカルソース付きプラグインを選択
# → 修正前: インストール失敗
# → 修正後: 正常にインストール完了
```

### ローカルソースプラグインの種類

ローカルソースを持つプラグインとは、ディスク上のファイルを参照するプラグインです：

```json
// プラグインの設定例（ローカルソース）
{
  "name": "my-plugin",
  "source": {
    "type": "local",
    "path": "./plugins/my-plugin"
  }
}
```

### プラグインのインストール後確認

```bash
# インストール済みプラグインの確認
/plugin list

# 特定プラグインの詳細確認
/plugin info my-plugin

# プラグインが正常に動作しているか確認
/my-plugin  # プラグイン提供のコマンドを実行
```

### REPL vs CLIでのインストール比較

```bash
# CLIからのインストール（従来通り動作）
claude plugin install my-plugin

# REPL内からのインストール（2.1.74で修正）
# Claude Code を起動して
/plugin install my-plugin
```

## 注意点

- この修正はローカルソースを持つマーケットプレイスプラグインに限定されます
- リモートURLのプラグインは影響を受けていませんでした
- プラグインのインストールにはネットワーク接続が必要な場合があります
- ローカルソースのパスが存在することを確認してからインストールしてください
