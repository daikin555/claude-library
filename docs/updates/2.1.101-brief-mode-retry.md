## 原文（日本語に翻訳）

ブリーフモードを改善しました：Claudeが構造化メッセージではなくプレーンテキストで応答した場合に、一度リトライするようになりました。

## 原文（英語）

Improved brief mode to retry once when Claude responds with plain text instead of a structured message

## 概要

ブリーフモード（`--brief` フラグ）は、Claudeの応答を簡潔な構造化形式で出力するモードです。このアップデートにより、Claudeが期待される構造化メッセージではなくプレーンテキストで応答した場合、自動的に一度リトライを行うようになりました。これにより、ブリーフモードの信頼性と安定性が向上します。

## 基本的な使い方

```bash
# ブリーフモードで実行
claude --brief "コードをレビューして"

# プレーンテキスト応答が返ってきた場合、自動的にリトライされる
# ユーザーは何も特別な操作をする必要はない
```

## 実践例

### CI/CDパイプラインでのブリーフモード活用

```bash
#!/bin/bash
# CI環境でブリーフモードを使用してコードレビューを自動化

# ブリーフモードは構造化出力を保証するため、
# スクリプトでの解析が容易になる
result=$(claude --brief --print "このPRの変更点を要約して")

# 応答が構造化されているため、jqなどで解析可能
echo "$result" | jq '.summary'
```

### バッチ処理でのブリーフモード使用

```bash
#!/bin/bash
# 複数ファイルをブリーフモードで処理

files=("app.js" "utils.js" "api.js")

for file in "${files[@]}"; do
  echo "処理中: $file"
  
  # リトライ機能により、各ファイルの処理が安定して実行される
  claude --brief --print "$(cat $file) のコードを分析して"
done
```

### 自動化スクリプトでの安定性確保

```bash
# ブリーフモードの出力を期待する自動化スクリプト
# リトライ機能により、予期しないプレーンテキスト応答を回避

claude --brief -p "テストを生成して" > test_output.json

# 出力が構造化されていることを確認
if python3 -c "import json; json.load(open('test_output.json'))"; then
  echo "構造化出力の取得に成功しました"
else
  echo "出力の解析に失敗しました"
fi
```

## 注意点

- リトライは1回のみ行われます。2回目も失敗した場合はそのまま結果が返されます
- リトライによりレスポンス時間が若干長くなる可能性があります
- ブリーフモードは `--brief` フラグまたは `-b` で有効化できます
- このリトライはユーザーには透過的に行われ、特別な設定は不要です
- 自動化スクリプトや CI 環境で特に有効な改善です
