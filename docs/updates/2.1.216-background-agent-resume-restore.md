## 原文（日本語訳）

レジュームしたバックグラウンドエージェントセッションがデフォルトエージェントに戻ってしまう問題を修正しました。エージェントのプロンプトとツール制限が正しく復元されるようになります。

## 原文（英語）

Fixed resumed background agent sessions reverting to the default agent: the agent's prompt and tool restrictions are now restored

## 概要

カスタムエージェント（サブエージェント定義）でバックグラウンドセッションを開始し、後からそのセッションをレジュームすると、これまではそのエージェント固有のシステムプロンプトやツール制限が失われ、デフォルトエージェントの挙動に戻ってしまう不具合がありました。この修正により、レジューム時にエージェントの定義（プロンプトとツール制限）が正しく復元され、元のエージェントとして継続動作するようになります。

## 基本的な使い方

この修正はバグ修正であり、追加の設定は不要です。カスタムエージェントでバックグラウンドセッションを起動し、後で再開するだけで、修正後の正しい挙動が得られます。

```bash
# カスタムエージェントでバックグラウンドセッションを開始
claude --agent code-reviewer -p "このPRをレビューして" &

# 後からそのセッションを一覧・再開
claude agents
claude --resume <session-id>
```

修正前は `--resume` 後にエージェント固有の制約（許可されたツールやプロンプト）が失われ、汎用のデフォルトエージェントとして動作してしまっていましたが、修正後は `code-reviewer` エージェントの定義がそのまま維持されます。

## 実践例

### ツール制限付きエージェントを使ったバックグラウンドタスク

`.claude/agents/read-only-auditor.md` のようにツール制限を持つエージェントを定義している場合を考えます。

```yaml
---
name: read-only-auditor
description: コードを読み取り専用で監査するエージェント
tools: Read, Grep, Glob
---

あなたはコードベースを読み取り専用で監査するエージェントです。
ファイルの編集や書き込みは絶対に行わないでください。
```

このエージェントでバックグラウンドセッションを開始し、途中で中断・再開しても、修正後は `tools: Read, Grep, Glob` という制限が維持されたまま処理が継続します。修正前は、このセッションをレジュームするとツール制限が外れ、Editツールなどが誤って使える状態に戻ってしまう可能性がありました。

### 複数エージェントを並行運用しているケース

`claude agents`（Agent View）で複数のバックグラウンドセッションを管理している場合、それぞれのセッションが異なるカスタムエージェントで動いていても、再開のたびに設定が正しく引き継がれるようになったため、セッションごとの役割分担を安心して長期間維持できます。

## 注意点

- 影響を受けるのはカスタムエージェント（`--agent` で指定したエージェントやサブエージェント委譲経由で開始されたセッション）を使ったバックグラウンドセッションのレジューム時のみです
- デフォルトエージェントで開始したセッションには影響しません
- ツール制限の復元はセキュリティ上重要な修正です。特に読み取り専用など制限付きエージェントを運用している場合はアップデートを推奨します
