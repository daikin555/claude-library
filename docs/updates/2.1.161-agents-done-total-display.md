# claude agentsの進捗表示: done/totalカウンターとpeek機能の改善

## 原文（日本語に翻訳）

`claude agents` の行に、作業がファンアウトされている場合に詳細の前に `done/total` が表示されるようになりました。peekには最も長く実行されているアイテムが表示されます。

## 原文（英語）

`claude agents` rows now show `done/total` before the detail when work is fanned out; peek shows the longest-running item

## 概要

Claude Code v2.1.161では、`claude agents` ビューで複数のサブエージェントが並列実行（ファンアウト）されている場合、各行に `完了数/合計数` の形式で進捗カウンターが表示されるようになりました。また、peekビューには最も長く実行中のアイテムが表示されるため、ボトルネックとなっているサブエージェントをすぐに特定できます。

## 基本的な使い方

`claude agents` コマンドでエージェントリストを表示すると、ファンアウト中のエージェントに自動的に進捗が表示されます。

```bash
# エージェントリストを表示
claude agents
```

表示例：
```
● my-workflow  3/10  Processing file_007.ts
  └─ (peek) Analyzing authentication module (running 4m 32s)
```

## 実践例

### 大規模コードベースの分析進捗確認

複数ファイルを並列で分析するワークフローを実行すると、`claude agents` でリアルタイムの進捗が確認できます。

```bash
# バックグラウンドでエージェントを起動
claude --bg "プロジェクト全体のセキュリティ脆弱性をスキャンしてください"

# 別ターミナルで進捗を確認
claude agents
# → ● security-scan  15/47  Scanning src/api/auth.ts
#     └─ (peek) Checking JWT validation (running 12m 05s)
```

### マルチファイルリファクタリングの追跡

```bash
claude --bg "すべてのReactコンポーネントをTypeScriptに移行してください"
claude agents
# → ● ts-migration  8/23  Converting Button.jsx
#     └─ (peek) Converting DataTable.jsx (running 8m 15s) ← ボトルネック特定
```

### 長時間実行中のボトルネック特定

peekに表示される最長実行アイテムを見ることで、全体の処理が遅い原因を即座に把握できます。該当のサブエージェントに注目すれば、処理の詰まりを早期に発見できます。

## 注意点

- `done/total` カウンターはファンアウト（並列実行）されているエージェントにのみ表示されます
- 最長実行中のアイテムはボトルネック検出に役立ちますが、複雑なタスクが割り当てられた場合など正常なケースでも長くなることがあります
- `claude agents` ビューはリアルタイムで更新されます
