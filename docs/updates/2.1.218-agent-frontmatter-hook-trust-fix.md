## 原文（日本語に翻訳）

未信頼のフォルダからエージェントfrontmatterのhooksが実行されてしまう問題を修正しました。現在は、フックが実行されるにはエージェントファイル自身が置かれているフォルダがワークスペース信頼を承認済みである必要があります

## 原文（英語）

Fixed agent frontmatter hooks running from untrusted folders: hooks now require the agent file's own folder to have accepted workspace trust

## 概要

Claude Codeのカスタムサブエージェントは、Markdownファイルのfrontmatterに`PreToolUse`・`PostToolUse`・`Stop`などのhooksを定義し、そのエージェント専用の処理をフックとして実行できます。以前は、このエージェントfrontmatterのhooksが、ワークスペース信頼（workspace trust）が承認されていないフォルダに置かれたエージェント定義であっても実行されてしまうことがあり、セキュリティ上の抜け穴になっていました。この修正により、エージェントfrontmatterに定義されたhooksが実行されるためには、そのエージェントファイル自身が置かれているフォルダに対してワークスペース信頼が承認されていることが必要になりました。

ワークスペース信頼とは、Claude Codeが未知のディレクトリでコードを自動実行する前にユーザーへ確認を求める仕組みで、信頼していないフォルダ由来のコード（hooksなど）が無条件に実行されるのを防ぐためのものです。

## 基本的な使い方

エージェント定義ファイル（例: `.claude/agents/my-agent.md`）のfrontmatterに`hooks`を定義する場合、そのファイルが置かれているフォルダがワークスペース信頼を承認済みである必要があります。

```markdown
---
name: my-custom-agent
description: カスタムエージェント
hooks:
  PreToolUse: |
    #!/bin/bash
    echo "Tool: $TOOL_NAME" >> /tmp/agent-tools.log
---

# エージェントのプロンプト内容
このエージェントは...
```

この`my-agent.md`が置かれているフォルダ（プロジェクトルートやサブディレクトリなど）がワークスペース信頼のダイアログで承認されていない場合、`PreToolUse`フックは実行されません。

## 実践例

### 信頼済みプロジェクト内でのエージェントhooks利用

通常どおりプロジェクトを開いてワークスペース信頼を承認していれば、以前と同様にエージェントのhooksは問題なく動作します。

```bash
cd my-trusted-project
claude
# ワークスペース信頼のダイアログで「信頼する」を選択
# → .claude/agents/ 配下のエージェントfrontmatter hooksが正常に実行される
```

### 未信頼のフォルダに置かれたエージェント定義

外部から取得したプラグインやサブモジュールなど、信頼を承認していないフォルダにエージェント定義が置かれている場合、そのエージェントのfrontmatter hooksは実行されなくなります。

```bash
# 未信頼のサブフォルダにエージェント定義がある場合
untrusted-folder/.claude/agents/suspicious-agent.md

# → suspicious-agent.md のfrontmatterに定義されたhooksは実行されない
#   （フォルダの信頼を承認するまでスキップされる）
```

### 複数フォルダにまたがるエージェント運用

モノレポなどで複数のサブフォルダにエージェントを配置している場合、フォルダごとに信頼状態が管理されるため、信頼済みフォルダのエージェントhooksのみが有効になります。

```bash
project-root/                # 信頼済み
  .claude/agents/trusted-agent.md   # hooksが実行される
vendor/third-party-tool/      # 未信頼
  .claude/agents/vendor-agent.md    # hooksは実行されない
```

## 注意点

- Claude Code v2.1.218で修正されたセキュリティ関連の変更
- 判定対象はエージェントファイル自身が置かれているフォルダのワークスペース信頼状態であり、Claude Codeを起動したカレントディレクトリの信頼状態とは別
- 信頼されていないフォルダのエージェントは、エージェント自体は利用できてもfrontmatterのhooks部分だけが実行されない（既存のグローバルフックには影響しない）
- 未信頼フォルダのフックを有効にするには、当該フォルダに対してワークスペース信頼を承認する必要がある
