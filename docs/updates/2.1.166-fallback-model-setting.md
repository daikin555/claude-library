## 原文（日本語に翻訳）

`fallbackModel` 設定を追加し、プライマリモデルが過負荷または利用不可の場合に順番に試される最大3つのフォールバックモデルを設定可能になりました。`--fallback-model` フラグがインタラクティブセッションにも適用されるようになりました。

## 原文（英語）

Added `fallbackModel` setting to configure up to three fallback models tried in order when the primary model is overloaded or unavailable; `--fallback-model` now also applies to interactive sessions

## 概要

Claude Code が API の過負荷や利用不可に直面した際に自動的に切り替えるフォールバックモデルを最大3つまで設定できるようになりました。これにより、プライマリモデルに問題が発生した場合でも作業を中断せずに継続できます。また `--fallback-model` フラグがインタラクティブセッションでも有効になりました。

## 基本的な使い方

`settings.json` に `fallbackModel` を設定します：

```json
{
  "model": "claude-opus-4-8",
  "fallbackModel": "claude-sonnet-4-6"
}
```

複数のフォールバックモデルを配列で指定（最大3つ）：

```json
{
  "model": "claude-opus-4-8",
  "fallbackModel": ["claude-sonnet-4-6", "claude-haiku-4-5-20251001"]
}
```

コマンドラインから指定する場合：

```bash
claude --model claude-opus-4-8 --fallback-model claude-sonnet-4-6
```

## 実践例

### 本番環境でのフォールバック設定

高可用性が求められる環境での設定例：

```json
{
  "model": "claude-opus-4-8",
  "fallbackModel": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5-20251001"
  ]
}
```

この設定では、Opus 4.8 が利用不可の場合に Sonnet 4.6 を試し、それも駄目なら Haiku 4.5 を使用します。

### インタラクティブセッションでのフォールバック

```bash
# インタラクティブセッション起動時にフォールバックモデルを指定
claude --model claude-opus-4-8 --fallback-model claude-sonnet-4-6

# セッション内でモデルが過負荷になった場合、自動的に Sonnet にフォールバック
```

### CI/CD パイプラインでの活用

```bash
# 非インタラクティブモードでも同様にフォールバックを設定
claude -p "コードをレビューしてください" \
  --model claude-opus-4-8 \
  --fallback-model claude-sonnet-4-6
```

## 注意点

- `fallbackModel` は単一の文字列または最大3つの文字列配列で指定します
- フォールバックは API の過負荷・利用不可エラー時のみ発動します（認証エラーやレートリミットでは発動しません）
- フォールバック先のモデルによって出力品質やコストが変わる場合があります
- 3P プロバイダー（Bedrock/Vertex）経由の場合、フォールバックの挙動が異なる場合があります
