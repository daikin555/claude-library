## 原文（日本語に翻訳）

`--model claude-opus-4-0` および `--model claude-opus-4-1` が現行バージョンではなく非推奨の Opus バージョンに解決されていた問題を修正しました

## 原文（英語）

Fixed `--model claude-opus-4-0` and `--model claude-opus-4-1` resolving to deprecated Opus versions instead of current

## 概要

`--model` フラグで `claude-opus-4-0` や `claude-opus-4-1` を指定した場合、最新の現行バージョンではなく非推奨（deprecated）の古いバージョンに解決されてしまうバグがありました。この修正により、これらのエイリアスが正しく現行の Opus モデルに解決されるようになります。

## 基本的な使い方

```bash
# 修正後は正しい現行バージョンが使用される
claude --model claude-opus-4-0 "このタスクを実行して"
claude --model claude-opus-4-1 "このタスクを実行して"
```

## 実践例

### モデルを明示的に指定してClaudeを起動

```bash
# Opus 4.0 を使用（修正後は現行バージョンに正しく解決）
claude --model claude-opus-4-0

# Opus 4.1 を使用
claude --model claude-opus-4-1

# 使用中のモデルを確認
> /model
# → 正しいモデル名が表示されるようになった
```

### 設定ファイルでモデルを指定する場合

```json
// ~/.claude/settings.json
{
  "model": "claude-opus-4-0"
}
```

```bash
# 設定ファイルで指定した場合も正しく解決される
claude
# → claude-opus-4-0 の現行バージョンが使用される
```

### 環境変数でモデルを指定する場合

```bash
# 環境変数を使用する場合も同様に修正済み
export ANTHROPIC_MODEL=claude-opus-4-1
claude "複雑な分析タスクをお願いします"
```

## 注意点

- モデルのエイリアス（`claude-opus-4-0` 等）は、常に Anthropic が推奨する現行バージョンに解決されます
- 特定の旧バージョンを明示的に使用したい場合は、完全なバージョン識別子を使用してください
- 非推奨モデルは将来的に API からも削除される予定です
- `/model` コマンドで現在使用中のモデルを確認できます
