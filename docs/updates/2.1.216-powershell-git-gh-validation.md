## 原文（日本語訳）

PowerShellツールにおける `git` および `gh` コマンドの引数検証を改善しました。

## 原文（英語）

Improved validation of `git` and `gh` command arguments in the PowerShell tool

## 概要

Windows環境でPowerShellツールを使用している場合、Claude Codeは `git` や `gh` コマンドの引数を解析し、パーミッション設定（許可・拒否ルール）と照合しています。今回のアップデートでは、この引数検証ロジックが強化され、これまで見逃されたり誤判定されたりしていた可能性のある引数パターンがより正確に扱われるようになりました。これにより、`git`・`gh` コマンドに対する許可/拒否ルールが意図通りに機能しやすくなります。

## 基本的な使い方

この修正は内部の検証ロジックの改善であり、ユーザー側での追加設定は不要です。v2.1.216 にアップデートするだけで、PowerShellツール経由の `git` / `gh` コマンド実行時に、より正確な引数検証が自動的に適用されます。

```powershell
# PowerShellツールを有効化している環境
git commit -m "message"
gh pr create --title "..." --body "..."
# → これらのコマンドの引数がより厳密に検証された上でパーミッションルールと照合される
```

## 実践例

### git/ghに対する許可・拒否ルールを設定している場合

`.claude/settings.json` で `git` や `gh` コマンドに対して許可/拒否ルールを設定している場合、今回の検証強化によってルールの適用精度が向上します。

```json
{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff*)",
      "Bash(gh pr view*)"
    ],
    "deny": [
      "Bash(git push --force*)",
      "Bash(gh repo delete*)"
    ]
  }
}
```

PowerShellツールを有効化している環境（`CLAUDE_CODE_USE_POWERSHELL_TOOL=1` など）でも、これらのルールが引数の内容に基づいて正しく評価されるようになりました。

### CI/自動化環境での安全性向上

Windows上のCIエージェントなどでClaude Codeを自動運用しているケースでは、`git` や `gh` の危険な引数の組み合わせ（例: リダイレクトやコマンド連結を含むもの）が誤って許可されるリスクが下がります。

```powershell
# 例: 危険な引数パターンを含むコマンド
git log; Remove-Item -Recurse -Force .

# 検証強化により、このようなパターンも適切にパーミッションチェックの対象となる
```

## 注意点

- この修正は主にWindows環境でPowerShellツールを使用しているユーザーに影響します
- Bashツールのみを使用している環境（macOS/Linuxなど）には直接的な影響はありません
- 既存の許可/拒否ルールの設定内容自体を変更する必要はありませんが、これまで想定通りに動作していなかったエッジケースが解消される可能性があります
- セキュリティ関連の修正のため、Windows環境でPowerShellツールを利用している場合は早めのアップデートを推奨します
