## 原文（日本語に翻訳）

会話に複数の画像が含まれている場合に「会話内の画像を処理できず、削除されました」というエラーが繰り返し表示される問題を修正しました

## 原文（英語）

Fixed a repeating "an image in the conversation could not be processed and was removed" error when the conversation contained multiple images

## 概要

複数の画像を含む会話において、「an image in the conversation could not be processed and was removed」というエラーメッセージが何度も繰り返し表示される不具合が修正されました。この問題はユーザー体験を著しく損なっていましたが、今後は同じエラーが重複して表示されることはありません。

## 基本的な使い方

この修正は自動的に適用されます。複数画像を含む会話でも、エラーが発生した場合は1回のみ通知されます。

```
# 修正前: 複数画像使用時に同じエラーが何度も表示される
claude
> [画像1を添付]
> [画像2を添付]
> [画像3を添付]
エラー: 画像を処理できませんでした
エラー: 画像を処理できませんでした  ← 繰り返し表示されていた
エラー: 画像を処理できませんでした  ← 繰り返し表示されていた

# 修正後: エラーは1回のみ表示
エラー: 画像を処理できませんでした（1回のみ）
```

## 実践例

### 複数スクリーンショットを使ったUIレビュー

```
# 複数画像を添付してUIレビューを依頼
claude
> [スクリーンショット1]
> [スクリーンショット2]
> [スクリーンショット3]
> これらの画面のデザインを比較してフィードバックをください
# 修正後: エラーがあっても適切な1回の通知のみ
```

## 注意点

- 画像処理エラー自体が解消されたわけではなく、重複表示の問題が修正されたものです
- 処理できない画像形式（非常に大きい画像、サポート外の形式等）は引き続きエラーになります
- サポートされている画像形式: PNG, JPEG, GIF, WebP
