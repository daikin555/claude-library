## 原文（日本語に翻訳）

クラッシュしたバックグラウンドタスクを再オープンした際、再起動の代わりに最大5秒間ブランク画面が表示される問題を修正

## 原文（英語）

Fixed reopening a crashed background task showing a blank screen for up to 5 seconds instead of its restart

## 概要

バックグラウンドタスクがクラッシュした後に再オープンしようとすると、最大5秒間ブランク（真っ白）な画面が表示されてからタスクが再起動するという UX の問題が修正されました。修正後は、クラッシュしたタスクを再オープンすると即座に再起動プロセスが始まります。ユーザーが「フリーズしているのか、処理中なのか」と迷う状況が解消されます。

## 基本的な使い方

```bash
# クラッシュしたバックグラウンドタスクの確認と再起動
claude agents

# エージェント一覧から再起動したいタスクを選択すると即座に再起動
# （修正前は最大5秒間のブランク画面があった）
```

## 実践例

### クラッシュしたタスクの即時復旧

```
修正前のフロー:
1. バックグラウンドタスクがクラッシュ
2. claude agents でタスクを選択
3. ❌ 最大5秒間ブランク画面（何も表示されない）
4. ようやくタスクが再起動

修正後のフロー:
1. バックグラウンドタスクがクラッシュ
2. claude agents でタスクを選択
3. ✅ 即座にタスクの再起動が始まる（ブランク画面なし）
```

### 大規模処理タスクの管理

```bash
# 大きなコードベースの分析など、長時間かかるバックグラウンドタスク
claude --background "プロジェクト全体のセキュリティ脆弱性をスキャンしてください"

# タスクが予期せず終了した場合
claude agents
# ✅ タスクを選択すると即座に再起動できる
```

### 複数タスクの管理

```bash
# 複数のバックグラウンドタスクを管理
claude agents
# クラッシュしたタスクを選択 → 即座に再起動
# 他のタスクは影響を受けない
```

## 注意点

- クラッシュの原因自体は修正されていません（クラッシュの発生自体を防ぐものではありません）
- 再起動後もクラッシュが繰り返す場合は、タスクの内容を確認してください
- バックグラウンドタスクのログは `claude agents` から確認できます
- タスクがクラッシュを繰り返す場合は、メモリ不足やタスクの複雑さが原因の可能性があります
