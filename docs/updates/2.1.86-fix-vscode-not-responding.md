## 原文（日本語に翻訳）

長時間の処理中に拡張機能が誤って「応答なし」を表示する問題を修正しました。

## 原文（英語）

[VSCode] Fixed extension incorrectly showing "Not responding" during long-running operations

## 概要

VS Code の Claude Code 拡張機能において、長時間かかる操作（大規模なコード分析、複数ファイルの処理、複雑なエージェントタスクなど）を実行中に、実際には正常に処理が続いているにもかかわらず「Not responding（応答なし）」のメッセージが表示される問題がありました。v2.1.86ではこの誤検出が修正され、長時間の処理中も正確な状態が表示されます。

## 基本的な使い方

VS Code 拡張機能を使用する際、長時間の操作でも誤った「応答なし」メッセージが表示されなくなります。

```
# VS CodeでClaude Codeを使用
1. VS Code を開く
2. Claude Code 拡張機能のパネルを表示
3. 長時間かかるタスクを依頼
   例: "このプロジェクト全体のコードをレビューしてください"
4. v2.1.86以降では処理中も「応答なし」は表示されません
```

## 実践例

### ユースケース: 大規模なコードベース分析

```
# VS CodeのClaude Codeパネルで
> このリポジトリ全体を分析して、セキュリティの問題を特定してください

# v2.1.84以前: 分析中に「Not responding」が表示されることがある
# v2.1.86以降: 正常に処理中として表示され続ける
```

### ユースケース: 複数ファイルのリファクタリング

```
# 複数ファイルにまたがるリファクタリングタスク
> src/ 以下のすべてのコンポーネントをReact Hooksに移行してください

# 長時間かかる処理でも「応答なし」と誤表示されなくなります
```

## 注意点

- VS Code の Claude Code 拡張機能（`vsix` パッケージ）を最新版に更新する必要があります
- 本当に応答がない（ハングしている）場合は、VS Code のリロードや拡張機能の再起動を試みてください
- 長時間の処理中は、拡張機能のステータスバーで進捗を確認できます
