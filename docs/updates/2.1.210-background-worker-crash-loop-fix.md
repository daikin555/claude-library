## 原文（日本語に翻訳）

クライアントがバックグラウンドサービスへの接続をリセットした際に、バックグラウンドワーカーがクラッシュループを起こす不具合を修正しました。

## 原文（英語）

Fixed background workers crash-looping when a client resets its connection to the background service

## 概要

Claude Codeは、`npm run dev` のような長時間実行コマンドやサブエージェントのタスクをバックグラウンドサービス上のワーカープロセスとして実行し、ターミナルクライアント（対話セッション）はそのワーカーに接続して進捗や出力を受け取ります。この修正前は、クライアント側の接続が予期せずリセットされた場合（ネットワーク切断、ターミナル強制終了、SSHセッション切れなど）に、バックグラウンドワーカー側がその状態をうまく処理できず、再起動と失敗を繰り返す「クラッシュループ」に陥ることがありました。v2.1.210では、クライアントの接続リセットを検知し正しくハンドリングするようになったため、バックグラウンドワーカーが安定して動作を継続、あるいは正常に終了できるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
# 長時間実行するdevサーバーをバックグラウンドで起動
claude
> npm run dev をバックグラウンドで実行して

# Ctrl+B でバックグラウンドに移動、あるいはClaudeが自動でバックグラウンド実行

# ここでネットワーク切断やターミナルの強制終了などが発生し、
# クライアント接続がリセットされる

# 修正前:
# バックグラウンドサービス上のワーカーが異常な状態を検知できず、
# 再起動 → 即エラー → 再起動 … を繰り返す（クラッシュループ）
# CPU使用率が高止まりし、他のバックグラウンドタスクにも影響
```

### 修正後の動作

```bash
claude
> npm run dev をバックグラウンドで実行して

# 接続リセットが発生しても:
# 1. バックグラウンドサービスが接続リセットを正しいイベントとして検知
# 2. ワーカープロセスは通常のライフサイクル通り継続 or 適切に終了
# 3. クラッシュループは発生しない

# セッションを再接続すると、タスクの状態を正しく取得できる
claude --resume
> バックグラウンドタスクの状態を確認して
```

## 実践例

### SSH経由でのリモート開発中の接続断

不安定なネットワーク環境からSSH越しにClaude Codeを使い、バックグラウンドでビルドやテストウォッチャーを走らせているケース。

```bash
# リモートサーバーにSSH接続してClaude Codeを起動
ssh dev-server
claude
> pnpm test --watch をバックグラウンドで実行して

# SSHセッションが不安定になり接続が切れる
# → 修正前はサーバー側でクラッシュループが発生し、
#   再接続後もバックグラウンドタスクが正しく機能しないことがあった
# → 修正後は接続が切れてもワーカーは安定動作を継続

# 再接続
ssh dev-server
claude --resume
> テストの実行結果を教えて
```

### ターミナルアプリの強制終了からの復帰

ターミナルアプリのクラッシュやOSのスリープ復帰などで接続が急に切れた場合。

```bash
claude
> 大規模なビルドタスクをバックグラウンドで開始して

# ターミナルアプリを強制終了（Cmd+Q / タスクマネージャーから終了など）

# 修正前: バックグラウンドサービス側でクラッシュループが発生することがあった
# 修正後: サービスは安定して稼働し続け、ビルドタスクも継続

# Claude Codeを再起動して同じセッションに戻る
claude --resume
> ビルドは終わった？
```

### 複数のバックグラウンドタスクを並行運用している場合の影響範囲

1つのクライアント接続リセットが、他の無関係なバックグラウンドタスクに悪影響を及ぼさないことを確認する。

```bash
claude
> devサーバー、テストウォッチャー、lintウォッチャーをそれぞれバックグラウンドで起動して

# 1つのクライアントセッションの接続がリセットされても、
# 修正後は他のバックグラウンドタスクのワーカーには影響しない
> 実行中のバックグラウンドタスク一覧を見せて
```

## 注意点

- この修正はClaude Code v2.1.210で導入されました
- バックグラウンドタスクは、セッションがアイドル状態のままOSからメモリ逼迫のシグナルを受けた場合など、一定条件下では引き続き終了されることがあります（本修正は「接続リセット時の異常なクラッシュループ」を対象とした修正であり、正常な終了条件を変更するものではありません）
- 不安定なネットワーク環境（SSH、リモート開発コンテナなど）でバックグラウンドタスクを多用する場合に特に効果があります
- 問題が疑われる場合は `--debug` フラグでバックグラウンドサービスとのやり取りのログを確認できます
