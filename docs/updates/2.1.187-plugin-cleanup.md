## 原文（日本語に翻訳）

`/plugin` を改善：最近使用していないプラグインを表示し、整理できるようにした。

## 原文（英語）

Improved `/plugin` to surface plugins you haven't used recently so you can clean them up.

## 概要

`/plugin` コマンドが更新され、最近使用していないプラグインをリストアップして表示するようになりました。長期間使用していないプラグインを簡単に特定して削除できるため、Claude Codeの環境を整理整頓しやすくなります。不要なプラグインを削除することで、パフォーマンスの向上やコンテキストの節約にもつながります。

## 基本的な使い方

```bash
/plugin
```

コマンドを実行すると、インストール済みプラグインの一覧が表示されます。その中に「最近使用していないプラグイン」のセクションが追加されており、整理の候補となるプラグインが確認できます。

## 実践例

### 最近使っていないプラグインを確認する

```bash
/plugin
```

表示例：
```
インストール済みプラグイン:
  ✓ github-integration    (最終使用: 2日前)
  ✓ eslint-helper         (最終使用: 1週間前)
  ✓ docker-tools          (最終使用: 3ヶ月前) ← 長期未使用
  ✓ aws-helper            (最終使用: 6ヶ月前) ← 長期未使用

長期間使用していないプラグイン:
  docker-tools, aws-helper
```

### 不要なプラグインを削除する

長期間使用していないと判断したプラグインをリストから削除できます：

```bash
/plugin
# → 不要なプラグインを選択して削除
```

または `claude mcp remove` コマンドで直接削除：

```bash
claude mcp remove docker-tools
```

### プラグイン環境の定期メンテナンス

月次のメンテナンスとして `/plugin` を実行し、不要なプラグインを整理することで、Claude Codeの起動時間を短縮できます：

```bash
/plugin
# 「長期間使用していないプラグイン」セクションを確認
# 不要なものを削除してシステムを軽量化
```

## 注意点

- 「最近使用していない」の基準は相対的なもので、使用頻度と経過時間に基づいています
- プラグインを削除しても再インストールは可能です（プラグインのソースが利用可能な場合）
- 定期的に必要なプラグインを少数に絞ることで、コンテキストウィンドウの効率的な利用につながります
- チームで共有のプラグイン設定を使用している場合は、削除前にチームメンバーへの確認を推奨します
