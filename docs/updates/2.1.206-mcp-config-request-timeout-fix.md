## 原文（日本語訳）

`--mcp-config` または `.mcp.json` で設定されたMCPサーバーが、サーバーごとの `request_timeout_ms` 設定を無視してしまい、新規セッションで長時間実行されるMCPツール呼び出しがデフォルトの60秒でタイムアウトしてしまう不具合を修正した。

## 原文（英語）

Fixed MCP servers configured via `--mcp-config` or `.mcp.json` ignoring a per-server `request_timeout_ms`, which caused long-running MCP tool calls to time out at the 60s default in fresh sessions

## 概要

これまで `--mcp-config` オプションや `.mcp.json` ファイル経由でMCPサーバーを設定する際、サーバーごとに指定した `request_timeout_ms` の値が正しく反映されず、新規セッション開始直後はデフォルトの60秒タイムアウトが適用されてしまう不具合がありました。今回の修正により、設定した `request_timeout_ms` が新規セッションでも正しく尊重されるようになり、時間のかかるMCPツール呼び出し（大規模なデータ処理や外部APIの遅い応答など）が意図せず途中で打ち切られることがなくなります。

## 基本的な使い方

`.mcp.json` でMCPサーバーごとにタイムアウト値をミリ秒単位で指定します。

```json
{
  "mcpServers": {
    "my-slow-server": {
      "command": "node",
      "args": ["./server.js"],
      "request_timeout_ms": 300000
    }
  }
}
```

`--mcp-config` フラグでも同様に、タイムアウトを含む設定ファイルを指定できます。

```bash
claude --mcp-config ./my-mcp-config.json
```

## 実践例

### 大規模データ処理を行うMCPサーバーの利用

データベースの全件スキャンや大量ファイルの解析など、数分単位で処理が完了するMCPツールを呼び出す場合、`request_timeout_ms` を長めに設定しておくことで、新規セッションでもタイムアウトによる強制終了を回避できます。

```json
{
  "mcpServers": {
    "data-analyzer": {
      "command": "python",
      "args": ["analyzer_server.py"],
      "request_timeout_ms": 600000
    }
  }
}
```

### 外部API連携で応答が遅いMCPサーバーの利用

レート制限や外部サービス側の処理遅延で応答に時間がかかるMCPサーバーを利用する際も、これまではセッション開始直後だけ60秒タイムアウトが適用されてしまうケースがありましたが、修正後は設定値がセッション開始時から一貫して適用されます。

```bash
claude --mcp-config ./external-api-mcp.json
```

## 注意点

- この修正はセッション開始直後（フレッシュセッション）で発生していた設定無視の問題に対するものです。既存のセッションで発生していた挙動とは異なる場合があります。
- `request_timeout_ms` はツール呼び出し1回あたりのハードなタイムアウト値であり、サーバー側からの進捗通知（progress notification）によって延長されるものではない点に注意してください。
- タイムアウト関連には他にも `MCP_TIMEOUT`（サーバー起動タイムアウト）や `MCP_TOOL_TIMEOUT` といった環境変数があり、混同しないようにしてください。
