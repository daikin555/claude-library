## 原文（日本語に翻訳）

`.claude/skills` に新しいプラグインをスキャフォールドするための `claude plugin init <name>` コマンドが追加されました。

## 原文（英語）

Added `claude plugin init <name>` to scaffold a new plugin in `.claude/skills`

## 概要

`claude plugin init <name>` コマンドにより、プラグイン開発に必要なディレクトリ構造とファイルを自動生成できるようになりました。手動でディレクトリやファイルを作成する手間が省け、公式フォーマットに沿ったプラグインを素早く開始できます。生成されたスキャフォールドはそのまま `.claude/skills/` に配置されるため、自動ロード機能と組み合わせて即座に利用できます。

## 基本的な使い方

```bash
# プロジェクトディレクトリで新しいプラグインをスキャフォールド
claude plugin init my-plugin

# グローバルにスキャフォールドする場合
claude plugin init my-plugin --global
```

実行すると `.claude/skills/my-plugin/` に以下の構造が生成されます。

```
.claude/
└── skills/
    └── my-plugin/
        ├── plugin.json        # プラグインメタデータ
        └── skills/
            └── example/
                └── SKILL.md   # サンプルスキル
```

## 実践例

### 新しいプロジェクト用プラグインを作成する

```bash
cd my-project

# スキャフォールドを生成
claude plugin init code-standards

# 生成されたファイルを確認
ls -la .claude/skills/code-standards/

# plugin.json を編集してメタデータを設定
# SKILL.md を編集してスキルの動作を定義
```

生成される `plugin.json` の例：

```json
{
  "name": "code-standards",
  "version": "1.0.0",
  "description": "プロジェクトのコーディング規約を強制するスキル集",
  "author": "Your Name"
}
```

### スキャフォールドをカスタマイズする

生成された `SKILL.md` の例：

```markdown
# Example Skill

トリガーコマンド: /code-standards:example

## 説明
このスキルの説明をここに書きます。

## 手順
1. ステップ 1
2. ステップ 2
3. ステップ 3

## ルール
- 守るべき制約や注意点
```

### 複数のスキルを持つプラグインを構築する

```bash
claude plugin init team-tools

# 追加のスキルディレクトリを作成
mkdir -p .claude/skills/team-tools/skills/daily-standup
mkdir -p .claude/skills/team-tools/skills/pr-review

# 各 SKILL.md を編集して機能を実装
```

これにより `/team-tools:daily-standup` や `/team-tools:pr-review` として呼び出せるスキルが作成されます。

## 注意点

- スキャフォールドされたプラグインは `.claude/skills/` に配置されるため、v2.1.157 の自動ロード機能により即座に利用可能になります。
- プラグイン名にはスペースや特殊文字を使わず、`kebab-case` を推奨します。
- スキルは 2,000 トークン以下に収めると最適なパフォーマンスが得られます。
- 既存のプラグインと名前が衝突しないよう、`claude plugin list` で確認してから作成することを推奨します。
- 生成後は `plugin.json` の `description` フィールドを必ず更新して、プラグインの用途を明確にしましょう。
