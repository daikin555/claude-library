## 原文（日本語に翻訳）

Ctrl+Vによる画像貼り付けがクリップボードから読み込まれている間、「Pasting…」フッターヒントを追加しました。

## 原文（英語）

Added a 'Pasting…' footer hint while a Ctrl+V image paste is being read from the clipboard

## 概要

Ctrl+Vで画像をClaude Codeに貼り付ける際、クリップボードからデータを読み込む処理が完了するまでの間、画面下部のフッターに「Pasting…」というヒントが表示されるようになりました。大きな画像や処理に時間がかかる場合でも、操作が受け付けられているかどうかが視覚的に確認できます。これにより、貼り付け処理中のユーザー体験が改善されます。

## 基本的な使い方

特別な設定は不要です。Ctrl+Vで画像を貼り付けるだけで自動的にフッターヒントが表示されます。

```
操作手順:
1. 画像をクリップボードにコピー（スクリーンショットや画像ファイルのコピーなど）
2. Claude Codeのプロンプトで Ctrl+V を押す
3. フッターに「Pasting…」が表示される（処理中）
4. 画像の読み込みが完了すると通常の状態に戻る
5. プロンプトに画像が添付された状態でメッセージを送信できる
```

## 実践例

### スクリーンショットの貼り付け

デスクトップやアプリケーションのスクリーンショットを素早くClaude Codeに共有できます。

```
使用手順:
1. PrtScn（Print Screen）またはCmd+Shift+4（macOS）でスクリーンショットを撮影
   → クリップボードに画像がコピーされる
2. Claude Codeのターミナルで Ctrl+V
   → フッターに「Pasting…」が表示される
3. 読み込み完了後、プロンプトに「このUIの問題点を教えてください」などと入力
4. Enterで送信
```

### 画像ファイルのコピー＆ペースト

ファイルマネージャーやブラウザからコピーした画像を直接貼り付けられます。

```
使用例:
- エラーダイアログのスクリーンショット → バグ報告に使用
- デザインモックアップ → UI実装の相談
- グラフや図表 → データ分析の依頼
- コードのスクリーンショット → OCRなしでそのまま解析を依頼
```

### iTerm2でのCmd+Vによる貼り付け

iTerm2ではClaude CodeはCtrl+VをCmd+Vにマッピングしています。

```
iTerm2での操作:
1. 画像をクリップボードにコピー
2. Claude Codeで Cmd+V を押す（iTerm2はCtrl+Vにマッピング済み）
3. 「Pasting…」ヒントが表示される
4. 画像が添付されたプロンプトが準備される
```

## 注意点

- 「Pasting…」ヒントはフッター（画面下部）に表示されます。フルスクリーン表示モードが有効な場合のみ表示されます。`CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` を設定している場合は表示されない可能性があります。
- 画像の貼り付けはターミナルがブラケットペースト（bracketed paste）をサポートしている必要があります。サポートしていないターミナルでは動作が異なる場合があります。
- 大きな画像ファイルは処理に時間がかかるため、「Pasting…」が表示される時間が長くなることがあります。
- テキストの貼り付けには「Pasting…」ヒントは表示されません（画像の貼り付け専用の表示です）。
