## 原文（日本語訳）

agents viewが改善され、ステータス列が64文字で切り詰められる代わりに、ターミナルの全幅を使用するようになった。

## 原文（英語）

Improved agents view: status column now uses full terminal width instead of truncating at 64 characters

## 概要

`agents` view（バックグラウンドで実行中・完了したエージェントセッションを一覧管理する画面）では、各セッションのステータス表示が従来64文字で強制的に切り詰められていました。今回の変更により、ステータス列がターミナルウィンドウの幅いっぱいまで表示されるようになり、長いステータスメッセージやタスク内容が省略されずに確認しやすくなりました。

## 基本的な使い方

Claude Codeのセッション内でagents viewを開きます。

```text
/agents
```

複数のバックグラウンドタスクを実行している状態でagents viewを開くと、各セッションのステータス列がターミナル幅に合わせて表示されます。

## 実践例

### 複数のバックグラウンドエージェントを並行管理する

複数の調査タスクやビルド確認をバックグラウンドで並列実行している際、agents viewを開いて各セッションの進捗を一目で確認できます。

```text
/agents
```

ワイドなターミナルウィンドウで作業している場合、以前は64文字で切れていた長いステータス文字列（例: 実行中のコマンドの詳細やタスクの説明）が、より多くの情報量で表示されるようになります。

### 狭いターミナルとワイドターミナルでの表示比較

ターミナルの横幅を広げてからagents viewを再度開くと、ステータス列の表示幅が自動的に追従して広がることを確認できます。逆に狭いターミナルでは、これまで通り読みやすい範囲で情報が調整されます。

```text
/agents
```

## 注意点

- 本改善は表示上の変更であり、エージェントの実行結果やステータスの内容そのものには影響しません。
- ターミナルの幅が狭い環境では、引き続き情報が一部省略される場合があります。
- 本改善はagents view（ダッシュボード）に関するものであり、個別セッションのトランスクリプト表示とは異なります。
