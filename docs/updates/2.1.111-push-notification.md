- プッシュ通知ツールを追加 — Remote Control と「Claude が判断したときにプッシュ」の設定が有効な場合、Claude がモバイルプッシュ通知を送信できます

## 原文（英語）

- Added push notification tool — Claude can send mobile push notifications when Remote Control and "Push when Claude decides" config are enabled
- `/context`, `/exit`, and `/reload-plugins` now work from Remote Control (mobile/web) clients

## 概要

Claude Code が Remote Control 経由でモバイルデバイスにプッシュ通知を送信できるようになりました。長時間のタスクが完了したときや、ユーザーの判断が必要なときに通知を受け取れます。また、Remote Control クライアント（モバイル/ウェブ）から `/context`、`/exit`、`/reload-plugins` コマンドが使えるようになりました。

## 基本的な使い方

### Remote Control の設定

1. Claude Code のモバイルアプリで Remote Control を有効化
2. 設定で「Push when Claude decides（Claude が判断したときにプッシュ）」を有効化

### Claude が通知を送るタイミング

- 長時間のビルドやテストが完了したとき
- ユーザーの入力や確認が必要なとき
- 重要なエラーが発生したとき

## 実践例

### 長時間タスクの完了通知

```bash
# Claude に長いタスクを依頼して離席
claude "全テストスイートを実行して、失敗したテストを修正して。完了したら通知して"
```

タスクが完了すると、設定したモバイルデバイスに通知が届きます。

### Remote Control からのコマンド実行

モバイルデバイスや Web ブラウザから Remote Control を使って:

```
/context    # 現在のコンテキスト使用量を確認
/exit       # セッションを終了
/reload-plugins  # プラグインを再読み込み
```

### CI/CD と組み合わせたワークフロー

```bash
# デプロイタスクを開始して通知を待つ
claude "本番環境にデプロイして、完了またはエラーが発生したらプッシュ通知を送って"
# → 外出中でもスマートフォンで結果を確認
```

## 注意点

- プッシュ通知には Remote Control の設定と「Push when Claude decides」の有効化が必要です
- 通知はネットワーク接続が必要です
- Remote Control は Claude Code の公式モバイルアプリまたはウェブクライアントで利用可能です
- `/exit` を Remote Control から実行するとセッションが終了するため注意が必要です
