## 原文（日本語に翻訳）

スキルおよびプラグインのfrontmatterにおけるブール値として、`true`/`false` に加えて `yes`/`no`、`on`/`off`、`1`/`0`（大文字小文字を区別しない）を受け付けるようにしました。

## 原文（英語）

Added `yes`/`no`/`on`/`off`/`1`/`0` (case-insensitive) as accepted values for skill and plugin frontmatter booleans, alongside `true`/`false`

## 概要

スキルやプラグインのMarkdownファイルのfrontmatterには、`background:` や `disabled:` のようにブール値（真偽値）を取るフィールドがあります。これまでは厳密にYAMLの `true`/`false` のみが有効な値として扱われ、それ以外の表記を使うと期待通りに解釈されないことがありました。v2.1.218以降、これらのフィールドは `yes`/`no`、`on`/`off`、`1`/`0` といった表記も受け付けるようになり、いずれも大文字・小文字を区別せず認識されます。これにより、スキルやプラグインの作者は好みの表記や既存の設定ファイルの慣習に合わせてブール値を記述できるようになりました。

## 基本的な使い方

```yaml
---
name: my-skill
description: サンプルスキル
context: fork
background: Yes
---
```

`background: true` の代わりに `background: Yes` と書いても、これまで通りバックグラウンド実行が有効になります。

## 実践例

### `on`/`off` での指定

```yaml
---
name: legacy-tool-skill
description: 旧ツールとの連携用スキル
disabled: off
---
```

`disabled: off` は `disabled: false` と同じ意味に解釈され、スキルは有効な状態になります。

### `1`/`0` での指定

```yaml
---
name: experimental-skill
description: 実験的な機能を試すスキル
context: fork
background: 1
---
```

`background: 1` は `background: true` と同じ意味に解釈され、このスキルはバックグラウンドで実行されます。

### プラグインのagent frontmatterでの利用

```yaml
---
name: my-plugin-agent
description: プラグイン提供のエージェント
isolation: worktree
background: no
---
```

`background: no` と書くことで `background: false` と同じ意味になり、フォアグラウンドで実行されます。大文字表記の `No` や `NO` でも同様に解釈されます。

## 注意点

- 既存の `true`/`false` 表記は引き続きそのまま利用できます。今回の変更はあくまで受け付ける値の種類が増えただけで、既存のfrontmatterファイルを書き換える必要はありません
- `yes`/`no`、`on`/`off`、`1`/`0` はすべて大文字・小文字を区別しません（例: `Yes`、`ON`、`On` はすべて有効です）
- YAML自体の仕様上、クオートなしの `yes`/`no`/`on`/`off` は環境によって解釈が揺れることがあるため、意図しない挙動を避けたい場合は明示的に `true`/`false` を使うのも一つの方法です
