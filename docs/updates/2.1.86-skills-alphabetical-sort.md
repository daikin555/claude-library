## 原文（日本語に翻訳）

スキル説明のコンテキスト使用量を削減するため、`/skills` リスト内のスキル説明を250文字に制限。また `/skills` メニューをアルファベット順にソートして検索しやすくした

## 原文（英語）

Skill descriptions in the `/skills` listing are now capped at 250 characters to reduce context usage. Changed `/skills` menu to sort alphabetically for easier scanning

## 概要

Claude Code v2.1.86より、`/skills`コマンドで表示されるスキル一覧に2つの改善が加わりました。スキル説明が250文字に制限されることでコンテキストウィンドウの消費が抑えられ、アルファベット順にソートされることで目的のスキルを素早く見つけられるようになりました。多数のスキルを使用するプロジェクトで特に効果的です。

## 基本的な使い方

```
# /skills コマンドでスキル一覧を表示
/skills

# v2.1.86以降の表示例（アルファベット順）:
# - claude-api: Build apps with the Claude API... (250文字まで)
# - commit: Create git commits with proper...
# - frontent-design: Create distinctive...
# - loop: Run a prompt on a recurring interval...
# - schedule: Create, update, list scheduled...
# - simplify: Review changed code for reuse...
# - update-config: Configure Claude Code harness...
```

## 実践例

### 大規模プロジェクトでのスキル管理

多数のカスタムスキルを持つプロジェクトでの使用例：

```
/skills

# アルファベット順で表示されるため目的のスキルを素早く発見
# analyze-code
# build-docker
# deploy-staging
# format-code
# generate-tests
# lint-check
# run-migrations
```

### スキルのコンテキスト使用量の比較

```
# v2.1.85以前: 長い説明のスキルがコンテキストを大量消費
# スキル説明が1000文字のスキルが10個ある場合:
# → 10,000文字分のコンテキストを消費

# v2.1.86以降: 各スキル説明が250文字に制限
# → 最大2,500文字のコンテキスト消費に抑制
# → 約75%のコンテキスト節約
```

### スキルファイルの最適化

スキルの説明（`description`フィールド）を250文字以内に収めておくことで、切り捨てを防げます：

```yaml
# .claude/skills/my-skill.md のfrontmatter例
---
name: my-skill
description: "このスキルはXXXを行います。YYYとZZZをサポートし、
  AAA形式でBBBを生成します（250文字以内に収める）"
---

# スキルの内容...
```

## 注意点

- 250文字を超えるスキル説明は自動的に切り捨てられます — スキルの機能自体には影響しません
- スキルの実際の内容（frontmatter以外）は制限を受けません
- アルファベット順ソートは英語表記のスキル名に基づきます
- 日本語スキル名は英語名の後にソートされます
