## 原文（日本語に翻訳）

アイドル状態のプロンプトでEsc Escを押してもリワインドメニューが開かなくなっていた問題（回帰バグ）を修正。バックグラウンドエージェントを停止するにはCtrl+CまたはCtrl+X Ctrl+Kを使用すること

## 原文（英語）

Fixed Esc Esc at an idle prompt not opening the rewind menu (regression); use Ctrl+C or Ctrl+X Ctrl+K to stop background agents

## 概要

Claude Codeで、何も実行していないアイドル状態のプロンプト入力欄に対して `Esc` キーを2回押す（Esc Esc）操作をしても、リワインドメニュー（会話やコード変更を過去のチェックポイントまで巻き戻すメニュー）が開かなくなっていた回帰バグが修正されました。あわせて、バックグラウンドで動作しているサブエージェントを停止する操作は Esc Esc ではなく `Ctrl+C` または `Ctrl+X Ctrl+K` を使うという役割分担が明確化されています。

## 基本的な使い方

リワインドメニューは `/rewind` コマンド、またはプロンプト入力欄が空の状態で `Esc` を2回押すことで開けます。

```bash
# Claude Code を起動
claude

# プロンプト入力欄が空の状態（アイドル状態）で
# Esc を2回押す → リワインドメニューが開く

# コマンドからも開ける
/rewind
```

入力欄に文字が入力されている場合は、Esc Esc は入力テキストのクリアに使われ、リワインドメニューは開きません（クリアされたテキストは入力履歴に保存されるため、`Up` キーで呼び戻せます）。

## 実践例

### コード変更を直前の状態に戻す

Claude Codeに編集を依頼した結果が意図したものと違った場合、アイドル状態でEsc Escを押してリワインドメニューを開き、対象のプロンプト時点まで戻せます。

```
1. claude を起動し、いくつかのプロンプトでファイルを編集させる
2. 結果が望ましくないことに気づく
3. プロンプト入力欄が空であることを確認（テキストが残っていればEscで一度クリア）
4. Esc を2回押す → リワインドメニューが開く
5. 「Restore code」「Restore conversation」「Restore code and conversation」
   などから戻したい範囲を選択
```

### 修正前後の挙動比較

```
[修正前（回帰バグあり）]
1. アイドル状態のプロンプトで Esc Esc を押す
2. ❌ リワインドメニューが開かない（無反応、または別の挙動）
3. /rewind コマンドを明示的に入力する必要があった

[修正後]
1. アイドル状態のプロンプトで Esc Esc を押す
2. ✅ リワインドメニューが正しく開く
3. /rewind と同じメニューにショートカットからアクセス可能
```

### バックグラウンドエージェントの停止とリワインドの使い分け

Esc Escはあくまで「リワインドメニューを開く」ためのショートカットであり、バックグラウンドで実行中のサブエージェントを止める操作ではありません。バックグラウンドエージェントの停止には以下を使用します。

```bash
# 現在の操作・処理全体を中断する
# Ctrl+C

# このセッションで動作中の全バックグラウンドサブエージェントを停止する
# （確認のため2回押す）
# Ctrl+X Ctrl+K
```

| 操作 | ショートカット | 用途 |
| --- | --- | --- |
| リワインドメニューを開く | Esc Esc（プロンプトが空の場合） | コード/会話を過去のチェックポイントに戻す |
| 現在の処理を中断 | Ctrl+C | 実行中の操作をキャンセル |
| バックグラウンドエージェントを全停止 | Ctrl+X Ctrl+K | バックグラウンドのサブエージェントを停止 |

### 長時間のバックグラウンド作業を整理する

複数のバックグラウンドサブエージェントを走らせて作業を進めている場合、想定外の変更をしてしまったサブタスクがあれば、まず `Ctrl+X Ctrl+K` でバックグラウンドエージェントを止めてから、`Esc Esc`（または `/rewind`）でコードを必要な時点まで巻き戻す、という組み合わせが有効です。

## 注意点

- リワインドメニューはプロンプト入力欄が**空のとき**のみ Esc Esc で開きます。文字が入力されている場合は入力テキストのクリアが優先されます。
- チェックポイントはBashコマンド（`rm`、`mv`、`cp` など）による変更を追跡しません。Claudeのファイル編集ツールによる直接の編集のみが対象です。
- チェックポイントは既定で30日後に自動的に削除されます（設定で変更可能）。
- バックグラウンドエージェントの停止は `Ctrl+C`（現在の操作の中断）または `Ctrl+X Ctrl+K`（バックグラウンドサブエージェントの全停止）で行い、Esc Escとは役割が異なります。
- Esc Esc や Ctrl+X Ctrl+K などのキー割り当ては `~/.claude/keybindings.json` でカスタマイズ可能です。ただし `Ctrl+C` はハードコードされた割り込み操作のため再割り当てできません。
