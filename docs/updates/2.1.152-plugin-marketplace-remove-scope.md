## 原文（日本語に翻訳）

`claude plugin marketplace remove` が `marketplace add`・`install`・`uninstall` との対称性のため `--scope user|project|local` を受け入れるようになりました。

## 原文（英語）

`claude plugin marketplace remove` now accepts `--scope user|project|local` for symmetry with `marketplace add`, `install`, and `uninstall`

## 概要

`claude plugin marketplace remove` コマンドに `--scope` オプションが追加されました。これにより、`marketplace add`・`install`・`uninstall` コマンドと同様に、削除対象のスコープ（`user`・`project`・`local`）を明示的に指定できます。スコープを指定することで、同名のマーケットプレイスが複数のスコープに登録されている場合でも、意図したスコープのものだけを削除できます。

## 基本的な使い方

```bash
# ユーザースコープのマーケットプレイスを削除
claude plugin marketplace remove --scope user <marketplace-name>

# プロジェクトスコープのマーケットプレイスを削除
claude plugin marketplace remove --scope project <marketplace-name>

# ローカルスコープのマーケットプレイスを削除
claude plugin marketplace remove --scope local <marketplace-name>
```

## 実践例

### スコープごとのマーケットプレイス管理

異なるスコープに登録されたマーケットプレイスを個別に管理する例です。

```bash
# プロジェクト固有のマーケットプレイスを追加
claude plugin marketplace add --scope project https://project-plugins.example.com

# プロジェクトスコープのマーケットプレイスのみ削除（ユーザースコープには影響なし）
claude plugin marketplace remove --scope project https://project-plugins.example.com
```

### チームプロジェクトでのマーケットプレイス管理

チームでプロジェクトスコープのマーケットプレイスを管理する例です。

```bash
# マーケットプレイス一覧を確認
claude plugin marketplace list

# プロジェクトスコープの特定マーケットプレイスを削除
claude plugin marketplace remove --scope project my-team-marketplace

# 削除後の確認
claude plugin marketplace list
```

### スコープの整理

不要になったスコープごとのマーケットプレイス登録を整理する例です。

```bash
# ユーザースコープのテスト用マーケットプレイスを削除
claude plugin marketplace remove --scope user test-marketplace

# ローカルスコープの一時的なマーケットプレイスを削除
claude plugin marketplace remove --scope local temp-marketplace
```

## 注意点

- `--scope` を省略した場合のデフォルト動作は従来通りです
- `user`・`project`・`local` のスコープは、それぞれ `~/.claude/`・`.claude/`（プロジェクト）・ローカル設定に対応します
- 同名のマーケットプレイスが複数のスコープに存在する場合、`--scope` を明示的に指定することを推奨します
- この変更は他のマーケットプレイスサブコマンド（`add`・`install`・`uninstall`）との一貫性を保つためのものです
