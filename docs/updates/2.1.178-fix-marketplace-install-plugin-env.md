## 原文（日本語に翻訳）

`CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` が新しいマーケットプレイスのインストール（クローン）を妨げる問題を修正しました。

## 原文（英語）

Fixed `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` preventing fresh marketplace installs from cloning

## 概要

`CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` 環境変数は、プラグインのインストール失敗時にマーケットプレイスデータを保持するための設定です。しかしこの環境変数が設定されている状態では、新規のマーケットプレイスプラグインのインストール（gitクローン）が正常に実行されない問題がありました。この修正により、フェイルセーフ設定が有効でも新規インストールが正常に動作するようになります。

## 基本的な使い方

`CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` は主に開発・デバッグ用の環境変数です:

```bash
# 設定方法（開発時のデバッグ用）
export CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1
claude

# 修正後: この設定があっても新規プラグインのインストールが正常に動作する
```

## 実践例

### プラグインのインストールが修正される場面

```bash
# フェイルセーフ設定を有効にしたままプラグインをインストール
export CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1

# 以前の動作
claude plugin install @company/my-plugin
# → エラー: Failed to clone marketplace repository
# → インストール失敗

# 修正後の動作
claude plugin install @company/my-plugin
# → Successfully cloned marketplace repository
# → プラグインがインストールされる
```

### 開発環境でのデバッグシナリオ

```bash
# プラグイン開発者向け: インストール失敗時のデータ保持
export CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1

# 複数のプラグインを順次インストール
claude plugin install plugin-a   # 成功
claude plugin install plugin-b   # 失敗しても既存データを保持（設定の意図通り）
claude plugin install plugin-c   # 修正前は失敗していたが、修正後は成功
```

### 環境変数の確認と管理

```bash
# 現在の設定確認
echo $CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE

# 設定を無効化
unset CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE

# 永続的に無効化するには .bashrc / .zshrc の設定を削除
```

## 注意点

- `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` は主にプラグイン開発者やデバッグ用の設定です。通常の使用では設定不要です
- この環境変数が設定されていない場合、この問題の影響を受けません
- 修正前にこの問題を回避するには、`unset CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE` してからインストールし、その後再設定する必要がありました
- 修正後はこの回避策は不要です
