# ターン実行時間の表示：/configメニューに「Show turn duration」トグルを追加

## 原文（日本語に翻訳）

`/config` メニューに「Show turn duration（ターン実行時間を表示）」トグルを追加

## 原文（英語）

Added "Show turn duration" toggle to the `/config` menu

## 概要

`/config` メニューに新しい「Show turn duration」トグルが追加されました。このオプションを有効にすると、Claudeがレスポンスを生成するのにかかった時間がターンごとに表示されます。パフォーマンスの把握や、複雑なタスクがどのくらい処理時間を要するかを確認するのに役立ちます。

## 基本的な使い方

```
/config
```

`/config` を実行して設定メニューを開き、「Show turn duration」の項目をトグルしてオン/オフを切り替えます。

## 実践例

### ターン実行時間の有効化

```
/config
```

メニューが開いたら、「Show turn duration」を探してEnterキーで有効化します。以降のレスポンスでは、各ターンの終わりに実行時間が表示されます：

```
[ターン実行時間: 3.2秒]
```

### パフォーマンス分析への活用

複雑なコードレビューや大規模なファイル処理を依頼する場合：

```
このリポジトリ全体のコードレビューをお願いします
（内部でファイル読み取り・分析を実施）

[ターン実行時間: 45.8秒]
```

実行時間を確認することで、タスクの複雑さとAPI処理時間の関係を把握できます。

### タスク最適化への活用

```
# 遅いリクエストを特定して最適化
大量のファイルを一度に処理 → 実行時間: 60秒

# プロンプトを分割して効率化
ファイルを分割して処理 → 各ターン実行時間: 15秒
```

## 注意点

- ターン実行時間にはAPIのネットワーク遅延、モデルの処理時間、ツール呼び出し時間が含まれます
- ネットワーク環境やサーバー負荷によって実行時間は変動します
- この情報はデバッグや最適化の参考として活用してください
- 設定は `/config` メニューからいつでもオフに戻せます
