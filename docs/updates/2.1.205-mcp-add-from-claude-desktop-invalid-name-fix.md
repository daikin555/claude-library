## 原文（日本語訳）

サーバー名にサポート対象外の文字が含まれている場合に `claude mcp add-from-claude-desktop` が処理途中でスタックしてしまう不具合を修正。無効な名前は今後レポートされるようになり、それ以外の残りのサーバーは引き続きインポートされる。

## 原文（英語）

Fixed `claude mcp add-from-claude-desktop` getting stuck when a server name contains unsupported characters; invalid names are now reported and remaining servers still import

## 概要

`claude mcp add-from-claude-desktop` は、Claude Desktopの設定ファイル（`claude_desktop_config.json`）に登録済みのMCPサーバーをClaude Codeへ一括インポートするコマンドです。macOSまたはWSL環境で利用でき、対話的なダイアログでインポート対象のサーバーを選択できます。従来はインポート対象のサーバー名にサポートされていない文字が含まれていると、コマンド全体が処理途中でスタックしてしまい、他の正常なサーバーまで一切インポートされないという問題がありました。今回の修正により、不正な名前を持つサーバーはエラーとして個別に報告されるようになり、それ以外の有効なサーバーは引き続き正常にインポートされるようになりました。

## 基本的な使い方

```bash
claude mcp add-from-claude-desktop
```

コマンドを実行すると、Claude Desktopに設定済みのMCPサーバー一覧が対話的なダイアログで表示されます。インポートしたいサーバーを選択して確定すると、Claude Code側の設定に追加されます。

インポート後は `claude mcp list` で正しく取り込まれたかどうかを確認できます。

```bash
claude mcp list
```

## 実践例

### 複数サーバーが登録された環境で一部だけ名前が不正な場合

Claude Desktopに5つのMCPサーバーを登録していて、そのうち1つのサーバー名にスペースや記号などサポート対象外の文字が含まれているケースを想定します。

```bash
claude mcp add-from-claude-desktop
```

修正前は、この1つの不正な名前が原因でコマンド全体が応答しなくなり、Ctrl+Cなどで強制終了するしかありませんでした。修正後は、不正な名前のサーバーだけがエラーとして報告され、残り4つの正常なサーバーは問題なくインポートされます。不正なサーバーについては、Claude Desktop側で名前を修正してから再度コマンドを実行することで個別に取り込めます。

### プロジェクト共有スコープでインポートする

チームでMCPサーバー設定を共有したい場合は、`--scope` フラグを使ってスコープを指定してインポートできます。

```bash
claude mcp add-from-claude-desktop --scope project
```

`project` スコープを指定するとプロジェクトルートの `.mcp.json` に設定が保存され、Gitでコミットしてチームメンバーと共有できます。この場合も、不正な名前を持つサーバーが混在していれば個別に報告され、有効なサーバーのみが `.mcp.json` に追加されます。

## 注意点

- このコマンドはmacOSまたはWSL環境でのみ利用可能で、Claude Desktopの標準的な設定ファイルの場所を前提としています。
- 既に同名のサーバーがClaude Code側に存在する場合は、`server_1` のように末尾に連番が付与されてインポートされます。
- どのような文字が「サポート対象外」と判定されるかについての詳細な仕様は、本記事作成時点では公式ドキュメントに明記が見つかりませんでした。不明な場合は、Claude Desktop側でサーバー名を英数字とハイフン・アンダースコアなどの基本的な文字に変更してから再インポートすることを推奨します。
- `--scope` には `local`（デフォルト、現在のプロジェクトのみ）・`project`（`.mcp.json` でチーム共有）・`user`（全プロジェクトで利用可能）の3種類があります。
