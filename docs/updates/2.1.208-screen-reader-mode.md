## 原文（日本語訳）

スクリーンリーダーモードを追加：スクリーンリーダー利用者向けのオプトイン式プレーンテキスト表示。`claude --ax-screen-reader` を実行するか、`CLAUDE_AX_SCREEN_READER=1` を設定するか、settingsに `"axScreenReader": true` を追加することで有効化できる。

## 原文（英語）

Added screen reader mode: opt-in plain-text rendering for screen reader users. Run `claude --ax-screen-reader`, set CLAUDE_AX_SCREEN_READER=1, or add "axScreenReader": true to settings.

## 概要

Claude CodeのCLIは通常、装飾的な罫線やアニメーションを含むリッチなターミナルUIで描画されます。しかしこの表示はスクリーンリーダーにとって読み上げづらいものでした。今回追加されたスクリーンリーダーモードは、装飾的な罫線やアニメーションを排したフラットなプレーンテキストで画面を描画するオプトイン機能で、スクリーンリーダー利用者がClaude Codeをより快適に使えるようにするものです。

## 基本的な使い方

CLIの起動時にフラグを付けるだけで有効化できます。

```bash
claude --ax-screen-reader
```

## 実践例

### 環境変数で常時有効化する

シェルの設定ファイル（`.bashrc` や `.zshrc` など）に環境変数を設定しておけば、毎回フラグを付けなくてもスクリーンリーダーモードが常に有効になります。

```bash
export CLAUDE_AX_SCREEN_READER=1
```

### settings.jsonで恒久的に設定する

チーム全体やマシン単位でスクリーンリーダーモードを既定にしたい場合は、`~/.claude/settings.json` などの設定ファイルに記述しておく方法もあります。

```json
{
  "axScreenReader": true
}
```

なお、優先順位は `--ax-screen-reader` フラグが最も高く、次に環境変数 `CLAUDE_AX_SCREEN_READER` が続きます。`CLAUDE_AX_SCREEN_READER=0` を設定すると、`axScreenReader` がsettingsで `true` になっていてもスクリーンリーダーモードを強制的に無効化できます。

## 注意点

- スクリーンリーダーモードは常にクラシックレンダラーで描画されるため、`tui` 設定（フルスクリーン描画モード）は無効時には効果を持ちません。ただし、Agent Viewからアタッチしたバックグラウンドセッションはこのモードでもフルスクリーンで描画されます。
- 本機能を利用するには Claude Code v2.1.181 以降が必要です（環境変数 `CLAUDE_AX_SCREEN_READER` の要件として公式ドキュメントに明記）。
- スクリーンリーダーモード有効時は、装飾用の記号を隠す、トランスクリプト上の記号を短いラベルとして読み上げる、ネストしたテーブルを「見出し: 値」の形式で読み上げるなど、読み上げ品質を高めるための改善が継続的に行われています。
