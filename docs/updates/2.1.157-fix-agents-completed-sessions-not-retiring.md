## 原文（日本語に翻訳）

アイドル状態のサブエージェントがまだパークされているか、バックグラウンドシェルをリークしていた場合に、`claude agents` の完了済みセッションがリタイアされない問題を修正しました

## 原文（英語）

Fixed `claude agents` completed sessions not retiring when an idle subagent was still parked or had leaked a backgrounded shell

## 概要

Claude Code v2.1.157 で修正されたバグです。`claude agents` でエージェントセッションが完了しても、アイドル状態のサブエージェントがまだ待機中（パーク状態）だったり、バックグラウンドシェルプロセスが残存していたりすると、完了済みセッションがリタイア（終了・クリーンアップ）されませんでした。修正後は、これらの残存プロセスが存在しても、完了したセッションは正しくリタイアされます。

## 影響を受けていた状況

- `claude agents` でエージェントが完了しても、セッション一覧に残り続ける場合
- サブエージェントが複数のタスクを実行し、一部がアイドル状態でパークされている場合
- エージェント実行中にバックグラウンドシェルコマンド（`&` 付きコマンドなど）が起動され、そのまま残存している場合
- 長時間実行されたエージェントセッションが終了後もリソースを占有し続ける場合

## 注意点

- v2.1.157 以降では自動的に修正されており、特別な操作は不要です
- 修正により、完了したセッションは残存するサブプロセスに関わらず正しくリタイアされます
- リタイアされたセッションのリソース（メモリ、プロセス）は適切に解放されます
- バックグラウンドシェルのリークは内部的に検出・クリーンアップされるようになりました
