## 原文（日本語に翻訳）

数秒間アイドル状態になった後に画面が一瞬空白になる問題を修正しました。

## 原文（英語）

Fixed screen flashing blank after being idle for a few seconds

## 概要

Claude Code を使用中に数秒間操作しないでいると、画面が一瞬白紙になるフラッシュ現象が発生する問題が修正されました。この問題は UI の描画処理に関わるタイミングの問題で、作業の妨げになっていましたが、本修正によって安定した表示が維持されるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。Claude Code を起動して使用するだけで、アイドル状態での画面フラッシュが発生しなくなります。

```bash
$ claude
# 数秒間何もしなくても画面がフラッシュしなくなった
# 安定した UI 表示が維持される
```

## 実践例

### ユースケース: 応答待ち中の表示安定性

Claude が長い回答を生成している間や、ユーザーが入力を考えている間など、しばらく待機している状況で特に効果があります。

```bash
$ claude
> 複雑なアーキテクチャの設計について相談したいです...
# Claude が回答を生成中に画面が安定して表示される
# 数秒待っても画面がフラッシュしない
```

### ユースケース: 長時間の作業セッション

複数のタスクを実行する長いセッションでも、タスク間の待機中に画面フラッシュが起きなくなります。

```bash
$ claude
> /edit ファイルの編集をお願いします
# 編集完了後、次のタスクを考えている間も画面が安定
> 次は...
```

## 注意点

- この問題はターミナルの描画タイミングに関連しており、特定のターミナルエミュレータで顕著でした
- フラッシュ現象が起きていた場合、目が疲れたり、集中が乱れたりする問題がありました
- 修正後は `Ctrl+L` による明示的な画面クリア以外では、意図しない画面の空白化は起きません
