## 原文（日本語に翻訳）

Issue/PR参照がクリッカブルリンクになるのは、`owner/repo#123` 形式で記述された場合のみになりました。`#123` のような単純な番号参照は自動リンク化されなくなりました。

## 原文（英語）

Changed issue/PR references to only become clickable links when written as `owner/repo#123` — bare `#123` is no longer auto-linked

## 概要

これまでチャット内で `#123` のような番号を書くと自動的にリンクとして認識されていましたが、今後は `anthropics/claude-code#123` のように `owner/repo#123` 形式で記述した場合のみクリッカブルリンクになります。これにより、番号（CSS ID、プルリクエスト番号以外の用途）が誤ってリンク化されることを防ぎます。

## 基本的な使い方

### 変更前（#123 がリンク化されていた）

```
バグ #123 を修正しました。
// ↑ #123 が自動的にリンクになっていた
```

### 変更後（owner/repo#123 形式のみリンク化）

```
anthropics/claude-code#123 のバグを修正しました。
// ↑ owner/repo#123 形式のみクリッカブルリンクになる

バグ #123 を修正しました。
// ↑ bare #123 はリンクにならない（プレーンテキスト）
```

## 実践例

### 正しいリンク形式

```markdown
# PRやIssueを参照する場合（クリッカブルリンクになる）
- anthropics/claude-code#1234 を参照してください
- facebook/react#28257 のPRと関連しています
- owner/repo#999 のIssueを確認してください
```

### リンク化されない例

```markdown
# 以下はリンクにならない
- バグ #456 を修正
- PR #789 をマージ
- Issue #100 を参照
// ↑ これらはプレーンテキストとして扱われる
```

### チャットでのPR/Issue共有

```
# 良い例: リポジトリ情報を含めた完全な参照
「anthropics/claude-code#2345 で報告された問題と同じです。
myorg/myproject#67 のPRで対応しています。」

# 注意が必要な例: 短縮形は使えない
「PR #67 で対応しています」→ リンクにならない
```

### CLAUDE.mdへの記述

```markdown
<!-- CLAUDE.md -->
# 関連リソース

このプロジェクトは myorg/myrepo#100 のIssueから始まりました。
詳細は myorg/myrepo#105 のPRを参照してください。
```

## 注意点

- この変更により、意図しない自動リンク化が減少します
- GitHubなどのリポジトリへのリンクを作成したい場合は、必ず `owner/repo#番号` 形式を使用してください
- 既存のドキュメントやコメントで `#番号` 形式を使っている場合、将来的にはリポジトリ情報を追加することを推奨します
- `owner/repo` の部分は実際のGitHubリポジトリのオーナー名とリポジトリ名に対応します
