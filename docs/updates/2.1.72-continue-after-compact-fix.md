## 原文（日本語に翻訳）

- `--compact`後に`--continue`で最新の位置から再開されない問題を修正

## 原文（英語）

- Fixed `--continue` not resuming from the most recent point after `--compact`

## 概要

`--compact`コマンドでコンテキストを圧縮した後に`--continue`（`-c`）フラグでセッションを再開すると、圧縮前の古い位置から再開されてしまうバグが修正されました。この修正により、コンテキスト圧縮後も正しく最新の位置からセッションを継続できるようになります。

## 基本的な使い方

```bash
# セッション中にコンテキストを圧縮
claude
> /compact

# セッションを終了して再開
claude --continue
# 修正後: --compact後の最新の状態から正しく再開される
```

## 実践例

### 長時間セッションの管理

コンテキストウィンドウが逼迫した際に`/compact`で圧縮してから作業を再開するワークフロー:

```bash
# 長時間の開発セッション
claude
> 大規模なリファクタリングを進めて
> ... (多くのやり取り)
> /compact  # コンテキストを圧縮して軽量化

# 一度ターミナルを閉じて、後で再開
claude -c
# --compact後の最新の会話状態から正しく再開
# 圧縮されたコンテキストのサマリーが保持されている
```

### パイプラインでの利用

スクリプトからClaude Codeを呼び出す場合にも正しく動作します:

```bash
# 最初のセッション
claude -p "大きなコードベースの分析を始めて"

# コンテキストが大きくなったら圧縮して継続
claude -c -p "/compact"
claude -c -p "分析の続きをお願い"
# 正しく最新の状態から再開
```

## 注意点

- `--compact`はコンテキストウィンドウの使用量を削減するための機能で、会話の要約を生成して古い詳細を除去します
- `--continue`（短縮形: `-c`）は直近のセッションを再開するフラグです
- この修正はセッション状態の管理に関するもので、圧縮されたコンテキストの内容自体には影響しません
- 複数のセッションがある場合、`--continue`は最も最近のセッションを再開します
