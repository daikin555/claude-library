# MCP OAuthフロークリーンアップのリスナーリーク修正

## 原文（日本語に翻訳）

MCP OAuthフロークリーンアップのリスナーリークを修正しました。

## 原文（英語）

Fixed listener leak in MCP OAuth flow cleanup

## 概要

Claude Code v2.1.63で、MCPサーバーのOAuth認証フローにおけるリスナーリークが修正されました。OAuth認証プロセス完了後、イベントリスナーが正しく解放されない問題があり、これが長時間の使用でメモリ消費量の増加につながる可能性がありました。

## 基本的な使い方

この修正はバックグラウンドで自動的に適用されます。ユーザー側での操作は不要です。

## 実践例

### 影響を受けるシナリオ

- OAuth認証が必要なMCPサーバー（GitHub、GitLab等）を使用している場合
- 認証の再実行やセッション中に複数回OAuthフローを実行する場合

### 修正後の改善

- MCP OAuth認証後のメモリ使用量が適切に解放される
- 長時間セッションでのメモリ消費量が安定

## 注意点

- v2.1.63にアップデートすることで自動的に修正が適用されます
- OAuth認証を使用するMCPサーバーを頻繁に使用するユーザーに特に効果があります
