## 原文（日本語に翻訳）

サンドボックスが設定で有効になっているときにWindowsで表示されていた「サンドボックスの依存関係が見つからない」という誤った起動時警告を修正しました。

## 原文（英語）

Fixed a spurious "sandbox dependencies missing" startup warning on Windows when sandbox was enabled in settings.

## 概要

Windows環境でClaude Codeのサンドボックス機能を設定ファイルで有効化していた場合、実際にはサンドボックスが正常に動作しているにもかかわらず、起動時に「sandbox dependencies missing（サンドボックスの依存関係が見つかりません）」という誤った警告が表示されていました。この修正により、誤警告が表示されなくなり、ユーザーの混乱を防げるようになりました。

## 基本的な使い方

### サンドボックスの設定方法

```json
// .claude/settings.json または ~/.claude/settings.json
{
  "sandbox": true
}
```

### 修正後の確認方法

修正後は、Windowsで上記設定をした状態でClaude Codeを起動しても、誤った警告は表示されなくなります：

```bash
# 修正前：誤警告が表示されることがあった
# WARNING: sandbox dependencies missing
# 修正後：この警告は表示されない（サンドボックスが正常な場合）
claude
```

## 実践例

### サンドボックス有効時の起動確認

```bash
# Claude Codeのバージョン確認
claude --version

# サンドボックスが有効な設定でClaude Codeを起動
# 修正後は不要な警告なしに起動する
claude "ファイル一覧を表示してください"
```

### Windows環境でのサンドボックス設定例

```json
// プロジェクトごとのサンドボックス設定
// .claude/settings.json
{
  "sandbox": true,
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)"
    ]
  }
}
```

### 設定の確認コマンド

```bash
# 現在の設定を確認（設定ファイルの場所）
# Windows: %APPDATA%\Claude\settings.json または プロジェクトの .claude/settings.json

# Claude Codeの設定を表示
claude config list
```

## 注意点

- **この修正はWindowsのみ対象**: macOSやLinux環境には影響しません
- **実際の依存関係不足は別問題**: 本当にサンドボックスの依存関係が不足している場合（例：Windowsのサンドボックス機能が無効化されている場合）は、引き続き警告が表示されます。これは正常な動作です
- **誤警告と正規警告の区別**: 修正前に表示されていた警告は誤検知でしたが、実際の依存関係エラーは正しく報告されます
- **アップデート方法**: 最新版へのアップデートは `npm update -g @anthropic-ai/claude-code` または Claude Codeの自動更新機能を使用してください
