## 原文（日本語に翻訳）

ターン終了後に `/bg` または `←←` で作成されたバックグラウンドセッションがエージェント一覧で永遠に「Working」と表示される問題を修正しました。

## 原文（英語）

Fixed background sessions created with `/bg` or `←←` after a turn finished showing "Working" forever in the agents list

## 概要

`/bg` コマンドまたは `←←`（左矢印キー2回）でバックグラウンドセッションを作成した際、タスクが完了しているにもかかわらずエージェント一覧（`claude agents`）で「Working」ステータスが永続的に表示される問題が修正されました。ターン終了後に作成されたバックグラウンドセッションの状態が正しく「Idle」または「Done」に更新されるようになります。

## 基本的な使い方

バックグラウンドセッションの作成方法:

```bash
# 方法1: /bg コマンドを使用
/bg

# 方法2: ←← (左矢印キー2回) で現在のセッションをバックグラウンドへ

# バックグラウンドセッションの一覧確認
claude agents
```

修正前は完了したセッションも「Working」と表示され続けましたが、修正後は正確なステータスが表示されます。

## 実践例

### バックグラウンドセッションのワークフロー

```bash
# 1. 長時間タスクを開始
claude
> 10000行のコードベースを分析して問題を報告してください

# 2. タスクが終了直後に /bg でバックグラウンドへ移動
/bg
# → 新しいセッションを開始できる

# 3. claude agents で状態確認
claude agents
# 修正前: session-1 [Working]  ← 完了しているのに
# 修正後: session-1 [Done]     ← 正確に完了と表示
```

### 複数バックグラウンドセッションの管理

```bash
claude agents

# 修正後の正確な表示
# ┌─────────────────────────────────┐
# │ Active Agents                   │
# ├─────────────────────────────────┤
# │ session-1 [Idle]  - 待機中     │
# │ session-2 [Done]  - 完了       │
# │ session-3 [Working] - 実行中  │
# └─────────────────────────────────┘
```

### ステータスに基づく次のアクション

```bash
# 完了セッションの結果を確認
claude agents
# → "Done" のセッションを選択してEnter
# → 結果を確認

# アイドル中のセッションに新しいタスクを送信
claude agents
# → "Idle" のセッションを選択してメッセージを送信
```

## 注意点

- この問題は「ターン終了後」に作成されたバックグラウンドセッションで特に発生していました
- 実行中のセッションに対してバックグラウンド化（Ctrl+B）する場合は別の修正（[subagent transcript fixes](./2.1.178-fix-subagent-transcript-messages.md)）が適用されています
- 「Working」が正確に表示されるようになったことで、不要なセッションの特定・終了がしやすくなりました
- セッションが本当に長時間「Working」の場合（ハング）は、`claude agents` でセッションを選択してCtrl+Cで終了できます
