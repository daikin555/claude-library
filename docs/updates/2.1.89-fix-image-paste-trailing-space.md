## 原文（日本語に翻訳）

画像の貼り付け時に末尾のスペースが挿入されなくなりました。

## 原文（英語）

Image paste no longer inserts a trailing space

## 概要

Claude Codeの入力フィールドに画像を貼り付けた際、不要な末尾のスペースが自動的に挿入されていました。このスペースが原因で、画像の後に続けてテキストを入力する際に余分な空白が入り込んだり、スペースを手動で削除する必要がありました。v2.1.89でこの不要なスペースの挿入が修正されました。

## 基本的な使い方

```bash
# Claude Code での画像貼り付け操作
# 1. クリップボードに画像をコピー（スクリーンショットなど）
# 2. Claude Code の入力フィールドで Ctrl+V (または Cmd+V on Mac)
# 3. 画像が貼り付けられる

# 修正前:
# [画像] _ ← 末尾にスペースが入っていた

# 修正後:
# [画像] ← スペースなし
```

## 実践例

### 画像を貼り付けてテキストを続ける

```
# スクリーンショットを貼り付けてコード修正を依頼する例

# 修正前の操作:
# 1. スクリーンショットを Cmd+V で貼り付け
# 2. [画像] が挿入されるが末尾にスペースが付く
# 3. スペースを手動で削除してからテキストを入力
# 4. "このエラーを修正してください"

# 修正後の操作:
# 1. スクリーンショットを Cmd+V で貼り付け
# 2. [画像] が挿入される（スペースなし）
# 3. そのままテキストを入力
# 4. "このエラーを修正してください"
```

### 複数の画像を連続して貼り付ける

```bash
# 複数の画像を貼り付ける場合
# 1枚目の画像を貼り付け → スペースなし
# 2枚目の画像を貼り付け → スペースなし

# テキストと画像を混在させる例:
# Before: [画像1] [テキスト] [画像2] ← 各画像後のスペースが自動挿入されない
# After:  [画像1] [テキスト] [画像2] ← 意図通りの配置

# 修正後: 手動でスペースや改行を追加する必要があります（意図的な場合）
```

### 画像入力を使ったワークフロー

```bash
# エラースクリーンショットを添付してデバッグを依頼する
# 1. エラー画面のスクリーンショットを撮る
# 2. Claude Code に貼り付け（Cmd+V）
# 3. "このエラーはどう修正しますか？" と入力
# 4. Enter で送信

# 修正後: ステップ3でバックスペースを押してスペースを削除する必要がなくなった
```

## 注意点

- 画像と次のテキストの間にスペースを入れたい場合は、手動でスペースキーを押してください
- この修正はすべてのプラットフォーム（macOS、Windows、Linux）に適用されます
- 画像の貼り付けに対応しているのはClipboardにビットマップデータが含まれる場合です
- ファイルパスとしての画像参照（`@path/to/image.png`）には影響しません
