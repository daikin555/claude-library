## 原文（日本語に翻訳）

Pro/Max/Team PremiumプランでSonnet 4.5を使用しているユーザーがSonnet 4.6に自動的に移行されるように変更

## 原文（英語）

Changed Sonnet 4.5 users on Pro/Max/Team Premium to be automatically migrated to Sonnet 4.6

## 概要

Claude CodeをPro、Max、またはTeam Premiumプランで利用しているユーザーが `claude-sonnet-4-5` モデルを使用している場合、自動的に最新の `claude-sonnet-4-6` へ移行されます。Sonnet 4.6は前バージョンと比較して全体的な性能が向上しており、特に複雑なコーディングタスクで改善が見られます。

## 変更前後の動作

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| 使用モデル | claude-sonnet-4-5 | claude-sonnet-4-6（自動） |
| 操作 | 不要 | 不要（自動で移行） |
| 設定 | そのまま | そのまま |

## Sonnet 4.6の改善点

- コーディングタスクにおける精度向上
- 長いコンテキストの理解力強化
- より自然な日本語対応

## 基本的な使い方

特に操作は不要です。次回Claude Code起動時から自動的にSonnet 4.6が使用されます。

```bash
# 何も変更しなくても自動的に4.6が使われる
claude
```

現在使用されているモデルを確認するには：

```
/model
```

## 実践例

### 現在のモデルを確認する

```
/model
# または
claude --print-model
```

### モデルを明示的に指定する場合

特定のモデルを使いたい場合は引き続き指定できます：

```bash
claude --model claude-sonnet-4-6
```

### APIを使っている場合

Claude Code SDK経由でAPIを直接使用している場合は、モデルIDを更新することを推奨します：

```python
import anthropic

client = anthropic.Anthropic()
response = client.messages.create(
    model="claude-sonnet-4-6",  # 4.5から4.6に更新
    max_tokens=1024,
    messages=[{"role": "user", "content": "コードをレビューして"}]
)
```

## 注意点

- 自動移行はPro/Max/Team Premiumプランのユーザーが対象です
- APIキーを使用しているユーザーは自動移行されません（手動でモデルを更新してください）
- Sonnet 4.5を引き続き使いたい場合は `--model claude-sonnet-4-5` で明示的に指定できます
