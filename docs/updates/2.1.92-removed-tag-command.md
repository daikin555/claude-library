## 原文（日本語に翻訳）

`/tag` コマンドを削除しました

## 原文（英語）

Removed `/tag` command

## 概要

`/tag` コマンドがClaude Codeから削除されました。このコマンドはセッションやコンテキストにタグを付けるために使用されていましたが、今回のアップデートで廃止されています。`/tag` を使用していたワークフローは代替手段に移行する必要があります。

## 基本的な使い方

`/tag` コマンドは使用できなくなりました。代替として以下の方法を検討してください：

```bash
# /tag の代わりにセッション管理を活用
# セッション名を明示的に指定して起動
claude --session-name "feature-auth-refactor"

# または /config でセッション設定を確認
/config
```

## 実践例

### セッション識別の代替方法

```bash
# 以前の方法（廃止）
# /tag feature-auth
# /tag sprint-42

# 新しい方法: セッション起動時に名前を付ける
claude --session-name "sprint-42-auth-feature"

# セッション一覧で識別
claude --list-sessions
```

### ノートや記録の代替

```bash
# タグの代わりにCLAUDE.mdやメモファイルを活用
# プロジェクトルートに作業メモを残す
cat >> .claude-notes.md << 'EOF'
## セッション記録 2026-04-04
- 認証機能のリファクタリング実施
- /tag の代わりにこのファイルで管理
EOF
```

## 注意点

- **破壊的変更**: `/tag` コマンドを使用しているスクリプトや自動化ワークフローは更新が必要です
- **移行先の検討**: タグの用途（セッション識別、作業記録など）に応じて、`--session-name` フラグやCLAUDE.mdファイルへの記録など適切な代替手段を選択してください
