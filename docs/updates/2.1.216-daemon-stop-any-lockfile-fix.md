## 原文（日本語訳）

`claude daemon stop --any` が、古いレガシーデーモンのロックファイルを経由して無関係なプロセスを終了させてしまう可能性がある不具合を修正しました。

## 原文（英語）

Fixed `claude daemon stop --any` potentially terminating an unrelated process via a stale legacy daemon lockfile

## 概要

`claude daemon stop --any` コマンドは、実行中のClaude Codeデーモンプロセスをロックファイルに記録されたPIDを頼りに特定して停止します。しかし、旧バージョンで作成された古い（レガシーな）ロックファイルが残っていた場合、そこに記録されたPIDが再利用され、Claude Codeとは無関係な別のプロセスを終了させてしまう可能性がありました。今回の修正により、ロックファイルの妥当性がより厳密にチェックされ、このような誤停止が発生しないようになりました。

## 基本的な使い方

この修正はバグ修正であり、ユーザー側での設定変更は不要です。`claude daemon stop --any` を通常通り実行すれば、内部的に安全性チェックが強化された状態でデーモンの停止処理が行われます。

```bash
# 実行中のClaude Codeデーモンを停止する
claude daemon stop --any
```

修正後は、ロックファイルの内容（PIDだけでなく付随するメタデータなど）を検証し、対象プロセスが本当にClaude Codeデーモンであることを確認したうえで終了処理が行われます。

## 実践例

### デーモンの状態を確認してから停止する

不安な場合は、停止前にデーモンの状態を確認する運用にすると安全です。

```bash
# デーモンの状態を確認
claude daemon status

# 問題なければ停止
claude daemon stop --any
```

### CI/CD環境でのクリーンアップ処理

CI環境でセッション終了時にデーモンをクリーンアップする際も、今回の修正により誤って他のプロセスを巻き込むリスクが低減されます。

```bash
#!/bin/bash
# ci-cleanup.sh
# ジョブ終了時にClaude Codeデーモンを安全に停止
claude daemon stop --any || echo "デーモンは既に停止しているか、対象が見つかりませんでした"
```

### 古いロックファイルが残っている場合の確認

アップグレード直後など、古いロックファイルの残存が疑われる場合は、ロックファイルの場所を確認しておくと安心です（環境によりパスは異なります）。

```bash
# 一般的なロックファイルの場所を確認（環境依存）
ls -la ~/.claude/ 2>/dev/null | grep -i lock
```

## 注意点

- この修正はデーモン管理内部のロジック改善であり、通常のユーザー操作（対話的なClaude Codeの利用）には影響しません。
- レガシーロックファイルが原因で問題が発生していた環境では、Claude Codeをアップデートすることで解消されます。
- `--any` オプションは、対象を特定せずに稼働中のデーモンを停止する際に使うオプションです。特定のデーモンのみを対象にしたい場合は、他のオプションの利用を検討してください。
