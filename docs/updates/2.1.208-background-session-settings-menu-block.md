## 原文（日本語訳）

`/install-github-app` と `/mcp` の設定メニューが、バックグラウンドセッションでは開かなくなった。

## 原文（英語）

`/install-github-app` and the `/mcp` settings menu no longer open in background sessions

## 概要

Claude Codeには、ユーザーが端末上で直接操作するフォアグラウンドセッションのほかに、エージェントチームや自動化パイプラインなどが裏側で実行するバックグラウンドセッションがあります。`/install-github-app`（GitHub連携のセットアップウィザード）や `/mcp` の設定メニューはユーザーの選択入力を前提とした対話型UIですが、これまではバックグラウンドセッションでも誤って起動してしまい、誰も応答できない入力待ち状態のままセッションが止まってしまうことがありました。今回の修正で、これらの対話型メニューはバックグラウンドセッションでは開かなくなりました。

## 基本的な使い方

通常どおり、対話的に操作しているフォアグラウンドのセッションでは、これまでと変わらず以下のコマンドで各メニューを開けます。

```text
/install-github-app
```

```text
/mcp
```

一方、エージェントチームのメンバーや自動化タスクなど、ユーザーの入力を受け付けられないバックグラウンドセッションでこれらのコマンドが実行されようとした場合には、対話メニューが開かれなくなりました。

## 実践例

### エージェントチームのメンバーがGitHub連携を誤って開始してしまうのを防ぐ

複数のClaude Codeインスタンスが協調して動く[エージェントチーム](https://code.claude.com/docs/en/agent-teams)構成では、各メンバーが与えられたタスクを自律的に処理します。タスク文脈に `/install-github-app` のような操作が紛れ込んでいても、バックグラウンドのメンバーセッションではメニューが起動しなくなったため、応答を待ち続けてタスク全体が停止するリスクが減りました。

### CI/自動化パイプラインでの安定動作

GitHub Actionsなどの自動化コンテキストからClaude Codeを実行するケースでは、対話的な設定変更を意図せず引き起こす余地がそもそも小さくなりました。GitHub連携のセットアップ自体は、対話的な端末から `/install-github-app` を実行して行う運用が引き続き推奨されます。

## 注意点

- バックグラウンドセッションでGitHub連携やMCPサーバーの設定を行いたい場合は、フォアグラウンド（対話的な）セッションで実行する必要があります。
- 本修正はメニューの起動制御に関するものであり、`/install-github-app` や `/mcp` コマンド自体の機能や設定方法が変わるわけではありません。
- バックグラウンドセッションの判定方法や、起動がブロックされた際にどのようなメッセージが表示されるかについての詳細な記載は、本記事作成時点の公式ドキュメントには見当たらず、changelogの記述をもとに記述しています。最新情報は公式ドキュメントを参照してください。
