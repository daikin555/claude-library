## 原文（日本語に翻訳）

メッセージピッカーからコンパクトを実行した後、`/context`がコンパクト前の古いトークン使用量を表示してしまう問題を修正しました

## 原文（英語）

Fixed `/context` reporting stale pre-compact token usage after compacting from the message picker

## 概要

`/context`コマンドは、現在の会話がコンテキストウィンドウをどれだけ使用しているかを表示する機能です。以前は、通常の`/compact`ではなく、メッセージピッカー（過去のメッセージを選択してそこから会話を巻き戻す機能）経由でコンパクトを実行した場合に、`/context`の表示が更新されず、コンパクト前の古いトークン使用量が表示され続けることがありました。修正後は、メッセージピッカー経由のコンパクトでも、`/context`が正しく最新のトークン使用量を反映するようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 通常の`/compact`フローでは元々問題なく動作していた
- メッセージピッカーからのコンパクト後に表示が古いままになる、比較的限定的なケースの修正
- 表示上の問題であり、実際のコンテキスト使用量やコンパクト処理自体には影響しない
