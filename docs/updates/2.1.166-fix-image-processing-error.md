## 原文（日本語に翻訳）

処理できない画像がセッションで送信された場合に繰り返し発生していた「image could not be processed」エラーと余分なトークン使用を修正しました。

## 原文（英語）

Fixed a recurring "image could not be processed" error and extra token usage when an unprocessable image was sent in a session

## 概要

セッション中に処理できない画像（破損ファイルや非対応フォーマットなど）を送信した場合、「image could not be processed」というエラーが繰り返し発生し、さらに余分なトークンが消費されるバグを修正しました。この修正により、画像処理エラーが適切に処理され、セッションの安定性とコスト効率が向上します。

## 基本的な使い方

この修正はバグフィックスであり、特別な設定変更は不要です。v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
```

## 実践例

### 修正前の問題

以前は処理できない画像を送信すると：
- 「image could not be processed」エラーが繰り返し表示される
- エラーが発生するたびに余分なトークンが消費される
- セッションが不安定になる

### 修正後の動作

v2.1.166 以降では：
- 処理できない画像は適切にスキップまたはエラー表示される
- エラーが繰り返されなくなる
- 余分なトークン消費がなくなる

## 注意点

- 処理できない画像形式（極端に大きい画像、破損ファイルなど）を使用する場合は、事前に適切な形式に変換することを推奨します
- 対応画像フォーマット：PNG、JPEG、GIF、WebP
