## 原文（日本語訳）

不可視のUnicode文字を含むコマンドに対するPowerShellツールの権限検証の不具合を修正しました。

## 原文（英語）

Fixed PowerShell tool permission validation of commands containing invisible Unicode characters

## 概要

PowerShellツールは、コマンドを実行する前に権限ルール（allow/deny/ask）と照合して安全性を検証します。今回の修正前は、ゼロ幅スペース（zero-width space）やゼロ幅接合子（ZWJ）、双方向テキスト制御文字（RTLオーバーライドなど）といった画面上には表示されない不可視のUnicode文字がコマンド文字列に紛れ込んでいる場合、権限検証ロジックがコマンドの内容を正しく解釈できず、本来プロンプトが必要なコマンドが誤って許可されたり、意図した許可ルールが正しく適用されなかったりする可能性がありました。これはセキュリティに関わる修正であり、v2.1.216以降ではこのような不可視文字を含むコマンドも正しく解析・検証されるようになりました。

## 基本的な使い方

この修正もユーザー側の設定変更は不要です。Claude Codeをv2.1.216以降にアップデートするだけで、PowerShellツールが以下のようなコマンドに対しても正しく権限検証を行うようになります。

- コピー&ペーストされたテキストやWebページから取り込んだコマンド文字列に、目に見えないUnicode制御文字が混入しているケース
- 一見無害に見えるが、不可視文字によって実際のコマンド内容が視覚的な表示とズレているケース

## 実践例

### PowerShellコマンドの許可ルールが不可視文字によって迂回されないことの確認

`.claude/settings.json` でPowerShellの許可ルールを厳格に運用している場合、修正後は不可視文字混入コマンドに対しても意図通りにプロンプトが表示されるかを確認しておくと安心です。

```json
{
  "permissions": {
    "allow": [
      "PowerShell(Get-ChildItem *)"
    ],
    "ask": [
      "PowerShell(Remove-Item *)"
    ]
  }
}
```

### 権限ルールの動作確認

不審なコマンド文字列を貼り付けて実行させた際に、期待通りに権限プロンプトが表示されるかどうかは `/permissions` で現在の設定を確認しつつ、実際にコマンドを試してみることで検証できます。

```
/permissions
```

### 最新版へのアップデート確認

セキュリティ関連の修正のため、バージョンが適用されているかをまず確認します。

```bash
claude --version
```

## 注意点

- この修正はセキュリティ強化を目的としたバグ修正であり、可能な限り速やかにv2.1.216以降へアップデートすることが推奨されます
- PowerShellツール自体は環境によってオプトイン機能です。有効化方法は `CLAUDE_CODE_USE_POWERSHELL_TOOL` 環境変数を参照してください
- 不可視文字関連の権限検証強化は、過去のリリースでも `&` によるバックグラウンドジョブの迂回対策や `-ErrorAction Break` のハング対策など、段階的に行われてきた一連のハードニング施策の一部です
