## 原文（日本語に翻訳）

起動時に claude.ai MCP 設定の取得を約 3 秒待機するという起動時のリグレッションを修正しました。

## 原文（英語）

Fixed a startup regression where Claude Code would wait ~3s for claude.ai MCP config fetch before proceeding

## 概要

以前のバージョンで導入されたリグレッションにより、Claude Code の起動時に claude.ai の MCP（Model Context Protocol）設定を取得するために約 3 秒間待機してしまう問題が発生していました。この修正では、MCP 設定の取得を非同期で行うか、タイムアウトを短縮することで、起動速度が改善されました。

## 基本的な使い方

修正は自動的に適用されます。起動時の待機時間が大幅に短縮されます。

```bash
$ claude
# 以前は ~3 秒の待機があったが、即座に起動するようになった
> すぐに作業を開始できる
```

## 実践例

### ユースケース: 頻繁に Claude Code を再起動するワークフロー

コーディング作業中に Claude Code を頻繁に起動・終了する場合、3 秒の削減は積み重なって大きな時間節約になります。

```bash
# 複数のプロジェクト間での起動も素早い
$ claude  # 即座に起動
> 作業完了
$ claude  # 再起動も素早い
```

### ユースケース: スクリプトでの連続起動

CI/CD やスクリプトで複数回 Claude Code を起動する場合、待機時間の削減が効果的です。

```bash
#!/bin/bash
for file in *.py; do
    claude --print "このファイルをレビューして: $file" < "$file"
    # 各起動が 3 秒速くなる
done
```

## 注意点

- この問題はリグレッション（以前のバージョンでは発生しなかった問題が新バージョンで再発したもの）として分類されます
- MCP 設定の取得自体は引き続き行われますが、起動をブロックしないように改善されました
- claude.ai の MCP コネクター（Slack、Gmail など）は引き続き正常に使用できます
