## 原文（日本語に翻訳）

推論プロファイルの検出がまだ処理中の間に、非US地域のAWS Bedrockで`/model`ピッカーが無効な`us.*`モデルIDを`settings.json`に保存し続ける問題を修正しました。

## 原文（英語）

Fixed `/model` picker on AWS Bedrock in non-US regions persisting invalid `us.*` model IDs to `settings.json` when inference profile discovery is still in-flight

## 概要

AWS Bedrockを使用し、米国以外のリージョン（ap-northeast-1など）で動作している場合、`/model`コマンドでモデルを選択した際に`us.*`形式の無効なモデルIDが`settings.json`に保存されてしまう問題が修正されました。推論プロファイルの検出処理が完了する前にモデル選択画面が表示されることが原因でした。

## 修正内容

推論プロファイルの検出が完了してから`/model`ピッカーに正しいリージョン対応のモデルIDを表示するようになりました。

```
Before: 推論プロファイル検出中 → /model → us.* のIDが表示・保存される（ap-northeast-1では無効）
After:  推論プロファイル検出完了後 → /model → 正しいリージョンのモデルIDが表示・保存される
```

## 注意点

- この修正はAWS Bedrock経由でClaude Codeを使用しており、かつ非USリージョン（ap-northeast-1、eu-west-1など）で動作している場合に関係します
- すでに無効な設定が保存されている場合は、`settings.json`の`model`フィールドを手動で修正するか、`/model`で再選択してください
