## 原文（日本語訳）

Agent view: Ctrl+Xで、リネームされたブランチのワークツリーを削除できるようになり、未push（未プッシュ）のコミットを破壊することがなくなり、ワークツリーを残す場合はセッション行も保持されるようになり、再利用されたワークツリー名は現在のベースにリセットされるようになった。

## 原文（英語）

Agent view: Ctrl+X now deletes renamed-branch worktrees, never destroys unpushed commits, keeps the session row when a worktree is kept, and reused worktree names reset to the current base

## 概要

Agent View（`claude agents`）では、バックグラウンドセッションごとに独立したgitワークツリーが自動的に作成されることがあり、セッションを削除する`Ctrl+X`（2秒以内に2回押す）でそのワークツリーも一緒に削除されます。今回のアップデートでは、この削除処理まわりの複数の不具合がまとめて修正され、ブランチ名を変更したワークツリーの削除、未pushコミットの保護、ワークツリーを保持した場合のUI表示、ワークツリー名を再利用する際のベース分岐点のリセットが改善されました。

## 基本的な使い方

Agent Viewでセッションを選択し、`Ctrl+X`を押すとまずセッションが停止し、2秒以内にもう一度`Ctrl+X`を押すとセッションと（条件を満たせば）そのワークツリーが削除されます。

```text
# Agent View上でセッションを選択した状態で
Ctrl+X       # 1回目: セッションを停止
Ctrl+X       # 2回目（2秒以内）: セッションとワークツリーを削除
```

## 実践例

### ブランチ名を途中でリネームしたセッションを片付ける

作業中に`git branch -m`などでブランチ名を変更したセッションは、以前はCtrl+Xで対応するワークツリーがうまく削除できないことがありました。今回の修正で、リネーム後のブランチ名からでも正しくワークツリーを特定して削除できるようになり、片付け漏れによるディスクの肥大化を防げます。

### 未pushのコミットが残ったセッションを誤って削除しそうになった場合

バックグラウンドセッションのワークツリーにコミット済みだがまだリモートにpushしていない変更がある状態でCtrl+Xを2回押しても、そのワークツリーは削除されずに保持されるようになりました。あわせて、ワークツリーが保持された場合はAgent View上のセッション行自体も残るため、「削除したはずなのに変更がどこにあるか分からない」という事態を避けられます。

### 同じ名前のワークツリーを別セッションで使い回す場合

過去に使ったワークツリー名を新しいセッションで再利用すると、以前はブランチの分岐元（ベース）が古いままになることがありました。今回の修正で、名前を再利用した際はベースが現在のブランチ状態にリセットされるようになり、意図しない古いベースからの作業を防げます。

## 注意点

- Ctrl+Xによる削除は「未push・未ロックのワークツリーで、コミットが残っていない」場合にのみ実際にワークツリー本体まで削除されます。未pushコミットが残っている場合や、別の実行中セッションがロックしているワークツリー、ユーザー自身が作成したワークツリーは保持されます。
- ワークツリーが保持される場合、Agent Viewのフッターに保持されたパスと理由が表示されます。
- 削除する前に、残しておきたい変更はコミット・pushしておくことが推奨されます。
