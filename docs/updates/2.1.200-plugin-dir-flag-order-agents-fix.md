## 原文（日本語に翻訳）

`claude agents --plugin-dir <dir>` において、`--plugin-dir` フラグが `agents` の後ろに置かれた場合にエージェントビュー上でそのプラグインのagentsとskillsが表示されない問題を修正しました。

## 原文（英語）

Fixed `claude agents --plugin-dir <dir>` not showing the plugin's agents and skills in the agent view when the flag is placed after `agents`

## 概要

`claude agents` サブコマンドは `--plugin-dir` を含め、トップレベルの `claude` コマンドと同様のフラグを受け付けます。しかし、`--plugin-dir` を `agents` サブコマンドの後ろに置いた場合（`claude agents --plugin-dir <dir>`）に限り、そのディレクトリから読み込まれるはずのプラグインが提供するagentsとskillsがエージェントビューに反映されない不具合がありました。この修正により、フラグの位置に関わらず `--plugin-dir` で指定したプラグインのagents・skillsが正しく表示されるようになりました。

## 基本的な使い方

```bash
# 修正前・修正後ともに動作していたパターン（フラグがagentsより前）
claude --plugin-dir ./my-plugin agents

# 修正前は動作しなかったが、修正後は正しく動作するパターン
claude agents --plugin-dir ./my-plugin
```

**修正前:** `claude agents --plugin-dir ./my-plugin` のように `agents` の後ろに `--plugin-dir` を置くと、プラグインは読み込まれるもののエージェントビュー上にそのプラグインのagentsやskillsが一覧に出てこなかった。

**修正後:** フラグの前後どちらの位置に置いても、`--plugin-dir` で指定したプラグインのagentsとskillsがエージェントビューに正しく表示される。

## 実践例

### ローカル開発中のプラグインをエージェントビューで確認する

プラグイン開発中に、動作確認のためにローカルディレクトリを直接読み込んでエージェントビューを開くワークフローで、コマンドの書き方（フラグの位置）に依存せず一貫した結果が得られるようになりました。

```bash
claude agents --plugin-dir ./dist/my-plugin
# プラグインが定義するカスタムagentsやskillsがエージェントビューの一覧に表示される
```

### 複数プラグインをまとめて検証する

複数の `--plugin-dir` を重ねて指定し、複数プラグインのagents/skillsが同時に一覧へ反映されるかを確認する場合にも、フラグの位置による表示漏れがなくなりました。

```bash
claude agents --plugin-dir ./plugin-a --plugin-dir ./plugin-b.zip
```

### CIやスクリプトから `claude agents --json` で確認する

自動化スクリプトで `claude agents --plugin-dir <dir> --json` のようにプラグイン読み込みと出力形式指定を組み合わせて呼び出す場合も、フラグの順序に起因する取りこぼしがなくなり、期待通りにプラグイン由来のagents/skillsが結果に含まれるようになります。

## 注意点

- `--plugin-dir` はディレクトリまたは `.zip` アーカイブを1つ指定するフラグで、複数プラグインを読み込みたい場合はフラグを繰り返し指定します。
- 読み込まれるのはそのセッション限りのローカルプラグインです。永続的にインストールしたプラグインとは別の仕組みです。
- この修正はエージェントビュー上の表示に関するものです。プラグイン自体の読み込みや実行結果には影響しません。
