## 原文（日本語訳）

ログアウトした状態で `/remote-control` を実行すると「Unknown command」と表示されてしまう不具合を修正した。今後はサインイン方法を案内するようになる。

## 原文（英語）

Fixed `/remote-control` showing "Unknown command" when logged out — it now explains how to sign in

## 概要

`/remote-control` はローカルのセッションを別のデバイスから続けられるようにするRemote Control機能を起動するコマンドですが、この機能を使うにはclaude.aiアカウントでのサインインが必須です。これまではログアウト状態で `/remote-control` を実行すると、あたかもコマンド自体が存在しないかのように「Unknown command」と表示されてしまい、ユーザーが何をすべきか分かりにくいという問題がありました。今回の修正で、ログアウト状態の場合はサインインが必要であることと、その手順（`/login` の実行）が案内されるようになりました。

## 基本的な使い方

サインイン済みの状態でセッション内から以下を実行するとRemote Controlが起動します。

```text
/remote-control
```

セッションURLが表示され、別のデバイス（スマートフォンなど）からそのセッションに接続できます。スペースキーでQRコードを表示することも可能です。

## 実践例

### ログアウト状態で誤って/remote-controlを実行した場合

サインインしていないことを忘れて `/remote-control` を実行してしまうケースです。

```text
/remote-control
```

修正前は「Unknown command」とだけ表示され、コマンドが存在しないのか、何か別の問題があるのか判断できませんでした。修正後は「サインインが必要です。`/login` を実行してください」といった趣旨の案内が表示され、次に取るべきアクションが明確になります。

### 案内に従ってサインインしてから再実行する

```text
/login
```

でclaude.aiアカウントにサインインした後、改めて以下を実行します。

```text
/remote-control
```

サインインが完了していれば、通常通りセッションURLやQRコードが表示され、別のデバイスからの接続が可能になります。

## 注意点

- Remote ControlはAPIキー認証には対応しておらず、claude.aiアカウントでの `/login` によるサインインが必須です。
- 組織でTrusted Devicesが有効になっている場合、このデバイスがまだ登録されていないとRemote Controlを利用できないことがあります。登録はサインイン時に自動的に行われ、別途の登録コマンドはありません。
- 初回起動時に「not enabled for your account」と表示される既知の問題がある場合は、`/logout` してから再度 `/login` することで解消することがあります。
