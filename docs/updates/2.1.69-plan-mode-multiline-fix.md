## 原文（日本語に翻訳）

プランモードのフィードバック入力がマルチラインテキスト入力に対応していなかった問題を修正しました（バックスラッシュ+Enter および Shift+Enter で改行が挿入されるようになります）

## 原文（英語）

Fixed plan mode feedback input not supporting multi-line text entry (backslash+Enter and Shift+Enter now insert newlines)

## 概要

Claude Code のプランモード（計画フェーズ）では、Claudeが提示した計画に対してフィードバックを入力できますが、そのフィードバック入力欄でマルチライン（複数行）テキストの入力ができないバグがありました。修正後はバックスラッシュ+Enter（`\` + Enter）および Shift+Enter で改行を挿入できるようになり、詳細なフィードバックが書きやすくなります。

## 基本的な使い方

```
プランモードのフィードバック入力で改行を挿入する方法:
- Shift+Enter: 改行を挿入
- バックスラッシュ+Enter（\ + Enter）: 改行を挿入
```

## 実践例

### プランモードでの詳細なフィードバック入力

```bash
# Claude Code を起動してプランモードで計画を確認
claude --plan "新しい認証システムを実装する"

# Claude が計画を提示した後、フィードバックを入力:
> このアプローチは良さそうですが、いくつか修正が必要です:[Shift+Enter]
> [Shift+Enter]
> 1. データベース設計の見直しが必要です[Shift+Enter]
> 2. セキュリティ要件を追加してください[Shift+Enter]
> 3. テスト計画も含めてください[Shift+Enter]
> [Shift+Enter]
> 詳しくは以下を参照してください...

# Enterで送信
[Enter]
```

### バックスラッシュ+Enter の使用

```bash
# バックスラッシュ+Enter でも改行可能
> ステップ1を変更してください\
> データベースは PostgreSQL を使用する\
> Redis もキャッシュ層として追加する
```

### 通常モードとプランモードの違い

```bash
# 通常の入力モード（以前から対応）
> 複数行のプロンプト[Shift+Enter]
> 続きの行

# プランモードのフィードバック（今回修正）
/plan
# → 計画が表示される
# → フィードバック入力欄でもマルチライン入力が使えるようになった
```

## 注意点

- プランモードは `--plan` フラグまたは `/plan` コマンドで使用できます
- 改行を挿入するには Shift+Enter または `\` + Enter を使います（Enter 単体では送信されます）
- マルチライン入力により、詳細なフィードバックや構造化されたコメントが書きやすくなりました
- フィードバック入力後、通常の Enter でフィードバックを送信します
