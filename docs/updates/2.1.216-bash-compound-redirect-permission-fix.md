## 原文（日本語訳）

`&&` リストや否定（negation）の中にリダイレクトを含む複合ステートメントに対するBashコマンドの権限チェックを修正しました。

## 原文（英語）

Fixed Bash command permission checking for compound statements with redirects inside `&&` lists or negations

## 概要

Claude Codeは、`&&` や `||`、`;` などの演算子で連結された複合Bashコマンドを解析し、各サブコマンドごとに個別の権限ルールと照合します。しかし、`&&` リストの中にリダイレクト（`>`, `>>` など）が含まれる場合や、`!` による否定を伴うステートメントでは、権限チェックのパース処理が正しく動作せず、意図した許可・拒否ルールが適用されないことがありました。今回の修正により、これらの複雑な構文を持つ複合コマンドでも各サブコマンドが正しく識別され、権限ルールが期待通りに適用されるようになりました。

## 基本的な使い方

この修正はバグ修正であり、ユーザー側の設定変更は不要です。以下のような複合コマンドを実行した際に、各サブコマンドが正しく権限ルールと照合されるようになります。

```bash
# && の中にリダイレクトを含むコマンド
mkdir -p ./out && echo "done" > ./out/log.txt

# 否定を伴うコマンド
! grep -q "TODO" file.txt && echo "no todo found"
```

## 実践例

### リダイレクトを含むビルドコマンドの許可設定

ビルド結果をログファイルにリダイレクトしつつ、後続コマンドを `&&` で連結する場合でも、各サブコマンドが正しく個別に権限チェックされます。

`.claude/settings.json`:
```json
{
  "permissions": {
    "allow": [
      "Bash(npm run build:*)",
      "Bash(echo:*)"
    ]
  }
}
```

```bash
npm run build && echo "build success" > build-status.txt
```

このとき、`npm run build` と `echo` のそれぞれが独立してルールと照合されるため、意図しない全体拒否や誤った全体許可が発生しにくくなります。

### 否定を使ったチェックコマンド

ファイルの存在確認やテスト失敗時の分岐など、`!` を使った条件式でも権限チェックが正しく機能します。

```bash
! test -f dist/output.js && npm run build > build.log
```

### 承認済みルールの確認

複合コマンドを一度承認した際に、Claude Codeがどのサブコマンドごとにルールを保存するか確認したい場合は、`/permissions` で現在のルール一覧を確認できます。

```
/permissions
```

## 注意点

- Claude Codeが認識するコマンド区切り文字は `&&`, `||`, `;`, `|`, `|&`, `&`, および改行です。
- ルールはサブコマンドごとに独立して評価されるため、`Bash(safe-cmd *)` を許可しても `safe-cmd && other-cmd` の `other-cmd` 部分まで自動的に許可されるわけではありません。
- 1つの複合コマンドに対して保存されるルールは最大5件までという制限があります。
- リダイレクト先のパスによっては、書き込み権限とは別にファイルアクセスの権限確認が発生する場合があります。
