## 原文（日本語に翻訳）

ツールの結果レンダラーがUI要素の代わりに数値のbigint値やプレーンテキストを返した場合にセッションがクラッシュする問題を修正しました

## 原文（英語）

Fixed a session crash when a tool's result renderer returned a numeric bigint value or plain text instead of a UI element

## 概要

Claude Code v2.1.210で修正された、ツール結果表示に関する安定性の問題です。カスタムツール（MCPツールやプラグインが提供するツール）の結果レンダラーが、本来期待されるUIコンポーネント（スタイル付きテキスト要素）ではなく、`bigint`型の数値やただの文字列を返してしまうケースがあり、これがそのままレンダリングパイプラインに渡ることでセッション全体がクラッシュしていました。今回の修正により、レンダラーが非UI値を返した場合でも安全にフォールバック処理され、クラッシュせずに表示が継続されるようになりました。

## 基本的な使い方

### 修正前の問題

```text
1. MCPサーバーやプラグインがカスタムツールを提供
2. ツール実行後、結果レンダラーが呼び出される
3. レンダラーの実装ミスなどにより、UI要素ではなく
   数値(bigint)やプレーンな文字列を直接returnしてしまう
4. Claude Codeのレンダリング層がUI要素以外を想定しておらず、
   例外が発生してセッションがクラッシュ
5. ユーザーは作業中のセッションを失う
```

### 修正後の動作

```text
1. カスタムツールの結果レンダラーが呼び出される
2. 戻り値がUI要素かどうかを検証
3. bigintやプレーンテキストなど、UI要素以外が返された場合は
   安全なフォールバック表示（テキストとしてそのまま整形して表示）に切り替え
4. セッションはクラッシュせず継続
```

## 実践例

### MCPツールのカスタムレンダラーを実装する場合

MCPサーバー側でツール結果の表示をカスタマイズする際、レンダラーの戻り値の型に注意が必要でした。

```typescript
// 修正前: bigintをそのままreturnするとクラッシュの原因になっていた
function renderToolResult(result: unknown) {
  if (typeof result === "bigint") {
    return result; // UI要素ではないためクラッシュしていた
  }
  return createUIElement(result);
}

// 修正後: このようなケースでもクラッシュせず、
// テキストとして安全にフォールバック表示される
```

### 巨大な数値を扱うツール（例: ファイルサイズ、タイムスタンプ）

`bigint`は64ビット整数などを正確に扱うために使われることがあります。

```bash
# 例: 巨大なinode番号やファイルサイズをbigintで返すツール
claude

> 大きなログファイルの統計情報を取得して

# 修正前: レンダラーがbigint値をそのまま返す実装だと
#          セッションがクラッシュしていた
# 修正後: 文字列化されて安全に表示される
```

### プラグイン開発時のデバッグ

プラグイン開発者が独自のツールレンダラーを実装している場合の確認例です。

```bash
claude --debug

> カスタムツールを実行して結果表示を確認

# --debug ログでレンダラーの戻り値の型不整合が
# 検出された場合の挙動を確認できる
```

## 注意点

- この修正は Claude Code v2.1.210 で実装されました
- 影響を受けるのは主にMCPサーバーやプラグインが提供する、独自の結果レンダラーを持つカスタムツール
- 標準搭載のツール（Bash、Read、Editなど）は元々UI要素を正しく返す実装になっているため、通常はこの問題の影響を受けません
- レンダラーが非UI値（bigint、string、numberなど）を返した場合、現在はクラッシュせずにフォールバック表示されますが、意図した見た目にならない可能性はあるため、ツール開発者は引き続き正しいUI要素を返す実装を推奨します
- セッションが不安定になった場合は `--debug` フラグでログを確認し、どのツールのレンダラーが問題を起こしていたかを特定できます
