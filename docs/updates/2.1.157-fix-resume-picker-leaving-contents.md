## 原文（日本語に翻訳）

フルスクリーンモードで終了した際に `--resume` セッションピッカーがターミナルにその内容を残す問題を修正しました

## 原文（英語）

Fixed the `--resume` session picker leaving its contents on the terminal after exiting in fullscreen mode

## 概要

Claude Code v2.1.157 で修正されたバグです。`claude --resume` でセッションピッカーを表示した際、フルスクリーンモードで終了（Esc やセッション選択後）すると、ピッカーの UI コンテンツがターミナル画面にそのまま残っていました。修正後は、セッションピッカーを閉じた際にターミナルの画面が正しくクリアされます。

## 影響を受けていた状況

- `claude --resume` を実行してセッションピッカーを表示した場合
- セッションを選択、または Esc でピッカーを閉じた後にターミナルにゴミが残る場合
- フルスクリーンモードでセッションピッカーが表示されている場合
- ピッカー終了後にターミナルの表示が乱れて操作しづらい場合

## 注意点

- v2.1.157 以降では自動的に修正されており、特別な操作は不要です
- 修正後は、セッションピッカーを閉じると画面が適切にクリアされ、ターミナルの通常状態に戻ります
- セッションを選択した場合も、Esc でキャンセルした場合も、どちらも正常にクリアされます
- 万が一、古いバージョンで画面が乱れた場合は `reset` コマンドでターミナルを初期化できます:
  ```bash
  reset
  # または
  tput reset
  ```
