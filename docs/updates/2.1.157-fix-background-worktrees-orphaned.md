## 原文（日本語に翻訳）

30日間のジョブ保持スイープ後に `.claude/worktrees/` 配下のバックグラウンドエージェントワークツリーが孤立する問題を修正しました

## 原文（英語）

Fixed background agent worktrees under `.claude/worktrees/` being orphaned after the 30-day job retention sweep

## 概要

Claude Code v2.1.157 で修正されたバグです。バックグラウンドエージェントが作成した Git ワークツリー（`.claude/worktrees/` 配下）が、30日間のジョブ保持クリーンアップ処理の後も削除されずに孤立したまま残っていました。修正後は、ジョブ保持クリーンアップが実行された際に、対応するワークツリーも正しく削除されます。

## 影響を受けていた状況

- バックグラウンドエージェントを使用してタスクを実行した場合
- 30日以上前のジョブが保持クリーンアップの対象になった場合
- `.claude/worktrees/` ディレクトリに古いワークツリーが蓄積されていた場合
- ディスクスペースが不要なワークツリーによって消費されていた場合
- `git worktree list` で不要なワークツリーが残っている場合

## 注意点

- v2.1.157 以降では自動的に修正されており、特別な操作は不要です
- 修正により、ジョブが30日後にクリーンアップされる際に対応するワークツリーも削除されます
- 既存の孤立ワークツリーを手動でクリーンアップするには、`git worktree remove` コマンドを使用してください
- 孤立したワークツリーのディレクトリは `.claude/worktrees/` 配下にあります
- 手動クリーンアップ例:
  ```bash
  # 孤立したワークツリーを確認
  git worktree list
  
  # 不要なワークツリーを削除
  git worktree remove .claude/worktrees/<worktree-name>
  
  # または強制削除
  git worktree remove --force .claude/worktrees/<worktree-name>
  ```
