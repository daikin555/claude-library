## 原文（日本語訳）

offset/limitを使って、極端に長い1行を含むファイルを読み込む際に発生していたメモリ膨張の問題を修正した。行全体をメモリに読み込む代わりに、明確なエラーを返すようになった。

## 原文（英語）

Fixed a memory blowup when reading files with extremely long single lines using offset/limit — the read now returns a clean error instead of loading the whole line

## 概要

Claude Codeの `Read` ツールは大きなファイルを扱う際に `offset` と `limit` を指定して部分的に読み込むことができますが、これまで1行の長さが極端（数十MB〜それ以上）なファイルに対してこの操作を行うと、行全体を丸ごとメモリに読み込もうとしてメモリ使用量が急激に膨張する問題がありました。今回の修正により、そのような巨大な1行を検出した場合は行全体を読み込まず、明確なエラーメッセージを返すようになりました。ログファイルやビルド生成物、minifyされたJSファイルなど、改行がほとんど無い巨大テキストファイルを扱う際の安定性が向上します。

## 基本的な使い方

これはユーザーが明示的に呼び出す機能ではなく、Claude Codeの内部動作の改善です。特別な操作は不要で、アップデート後は自動的に適用されます。

```text
Read ツールで offset/limit を指定してファイルを読み込む
→ 極端に長い1行に遭遇した場合、以前はメモリ膨張が発生
→ 修正後はクリーンなエラーを返し、セッションがクラッシュしにくくなる
```

## 実践例

### 巨大な1行のログファイルを部分的に調査する

ミニファイされたJSONログや、改行を含まないアプリケーションログなど、1行が非常に長いファイルに対して `offset`/`limit` で一部だけを読もうとしたケースです。

```text
Read(file_path: "/var/log/app/huge-single-line.log", offset: 1, limit: 10)
```

以前はこの操作がメモリを大量消費してプロセスを不安定にする可能性がありましたが、修正後は安全にエラーとして処理され、Claudeは代替手段（`grep`やバイトオフセットでの部分読み込みなど）を検討できます。

### CIやバックグラウンドタスクでの安定性向上

自動化されたワークフローやバックグラウンドで動くエージェントが、想定外に巨大な1行を含む生成ファイル（例: バンドルされたJSアセット）を読み込もうとした場合でも、プロセス全体がメモリ不足で落ちるリスクが減り、エラーとして安全に処理を継続できます。

```text
サブエージェントが dist/bundle.min.js を offset/limit 付きで調査
→ 修正前: メモリ使用量が急増しセッションが不安定化する可能性
→ 修正後: エラーメッセージが返り、処理を継続できる
```

## 注意点

- この修正はあくまで安全策であり、極端に長い1行のファイルそのものを読み込めるようになるわけではありません。該当ファイルは `grep` などの別ツールで部分的に確認するのが実用的です。
- ユーザーが直接設定するオプションはなく、Claude Code内部の挙動として自動的に適用されます。
- 通常の改行を含むテキストファイルの読み込みには影響しません。
