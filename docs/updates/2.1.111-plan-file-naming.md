## 原文（日本語に翻訳）

- プランファイルがランダムな単語のみではなく、プロンプトにちなんだ名前（例: `fix-auth-race-snug-otter.md`）で保存されるようになりました

## 原文（英語）

- Plan files are now named after your prompt (e.g. `fix-auth-race-snug-otter.md`) instead of purely random words

## 概要

Claude Code のプランファイルが、ユーザーのプロンプト内容を反映した名前で保存されるようになりました。以前は純粋にランダムな単語の組み合わせでしたが、今後はプロンプトのキーワードが名前に含まれるため、後から見つけやすくなります。

## 基本的な使い方

### プランファイルの生成

```
/plan 認証のレースコンディションを修正して
```

以前の出力例: `snug-otter-purple-moon.md`
現在の出力例: `fix-auth-race-snug-otter.md`

プロンプトの内容（`fix-auth-race`）がファイル名の先頭に含まれます。

### プランディレクトリの確認

```bash
ls ~/.claude/plans/
# fix-auth-race-snug-otter.md
# add-user-dashboard-bright-star.md
# refactor-api-layer-calm-fox.md
```

## 実践例

### 複数のプランを並行管理

```bash
# 認証修正のプラン
/plan 認証システムのバグを修正して
# → fix-auth-bug-swift-hawk.md

# 新機能のプラン
/plan ユーザーダッシュボードを追加して
# → add-user-dashboard-keen-wolf.md
```

名前からプランの内容が推測しやすくなります。

### 過去のプランを再利用

```bash
# プラン一覧を確認
ls ~/.claude/plans/

# 関連するプランを検索
ls ~/.claude/plans/ | grep "auth"
# fix-auth-race-snug-otter.md
# fix-auth-session-bright-moon.md
```

### /resume でプランを再開

```
/resume fix-auth-race-snug-otter.md
```

名前がわかりやすいため、目的のプランをすぐに特定できます。

## 注意点

- ファイル名はプロンプトのキーワード + ランダムな単語の組み合わせです（完全に一致するわけではありません）
- プランディレクトリは `plansDirectory` 設定で変更できます
- ファイル名の長さは適切に制限されます
