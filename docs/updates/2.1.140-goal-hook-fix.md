## 原文（日本語に翻訳）

`disableAllHooks`または`allowManagedHooksOnly`が設定されている場合に`/goal`がサイレントにハングする問題を修正 — 解決しないインジケーターの代わりに明確なメッセージが表示されるようになりました

## 原文（英語）

Fixed `/goal` silently hanging when `disableAllHooks` or `allowManagedHooksOnly` is set — now shows a clear message instead of an indicator that never resolves

## 概要

`disableAllHooks`または`allowManagedHooksOnly`が設定された環境で`/goal`コマンドを使用すると、処理インジケーターが消えずにサイレントにハングする問題が修正されました。現在は明確なエラーメッセージが表示され、状況を把握できるようになっています。

## 基本的な使い方

`/goal`コマンドは通常通り使用できます：

```
/goal テストが全パスする
```

`disableAllHooks`や`allowManagedHooksOnly`が設定されている場合は、ハングせずに明確なメッセージが表示されます。

## 影響を受ける設定

```json
// settings.json
{
  "disableAllHooks": true
  // または
  // "allowManagedHooksOnly": true
}
```

## 注意点

- この修正により、フック制限のある環境でも`/goal`の状態が明確になります
- `/goal`コマンド自体はフックを必要とするため、`disableAllHooks`設定下では使用に制限があります
- エンタープライズ管理環境（`allowManagedHooksOnly`設定）でも同様の改善が適用されます
