## 原文（日本語訳）

大規模な会話から生成されたバックグラウンドエージェントやフォークを含むセッションを再開する際のメモリ使用量を削減した。

## 原文（英語）

Reduced memory usage when resuming sessions with background agents or forks spawned from large conversations

## 概要

Claude CodeにはサブエージェントをCtrl+Bでバックグラウンド実行したり、会話を`fork`して別セッションとして走らせたりする機能があります。元になった会話が非常に長い場合、そこから派生したバックグラウンドエージェントやフォークのセッションを`--resume`や`/resume`で再開する際に、本来不要なメモリまで読み込んでしまうことがありました。今回のアップデートは、この再開処理におけるメモリ効率を改善する内部的な信頼性向上で、ユーザーが操作するコマンドや設定に変更はありません。

## 基本的な使い方

このアップデート自体は新しいコマンドやフラグを追加するものではなく、既存の再開系コマンドの内部動作が改善されたものです。

```bash
claude --resume <session-id>
```

上記のように、大きな会話から派生したセッションを再開しても、以前よりメモリ消費を抑えて動作します。

## 実践例

### 長時間の調査セッションから複数のバックグラウンドエージェントを分岐させていた場合

大規模なリファクタリングやバグ調査で会話が長くなった状態からCtrl+Bで複数のサブエージェントをバックグラウンド実行していると、あとからそれぞれのセッションを`/tasks`経由でアタッチ・再開する場面が増えます。従来はこの再開時にメモリ使用量が跳ね上がることがありましたが、今回の改善によって同様の操作を繰り返しても安定して動作しやすくなります。

### `fork`で分岐した長い会話を後日再開する場合

膨大なコンテキストを持つセッションから`fork`して並行作業用のセッションを作り、そのセッションを翌日以降に`claude --resume`で再開するようなワークフローでも、元の巨大な会話全体を毎回メモリに展開するオーバーヘッドが軽減されます。

## 注意点

- ユーザーが直接呼び出すコマンドやオプションに変更はなく、内部的なメモリ管理の最適化です。
- 効果を体感しやすいのは、非常に長い会話からバックグラウンドエージェントやフォークを多用しているプロジェクトです。
- 具体的な削減率など数値的な情報はchangelogに記載がなく、本記事作成時点では確認できませんでした。
