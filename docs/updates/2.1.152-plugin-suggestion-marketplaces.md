## 原文（日本語に翻訳）

`pluginSuggestionMarketplaces` マネージド設定を追加しました。管理者は、コンテキスト対応のヒント経由で提案可能な組織マーケットプレイスを許可リストに追加できます。

## 原文（英語）

Added `pluginSuggestionMarketplaces` managed setting: admins can allowlist org marketplaces whose plugins may be suggested via context-aware tips

## 概要

新しい `pluginSuggestionMarketplaces` マネージド設定により、管理者が組織のマーケットプレイスを許可リストに登録できます。登録されたマーケットプレイスのプラグインのみが、コンテキスト対応のヒントを通じてユーザーに提案されるようになります。これにより、組織が承認したプラグインだけがユーザーに推奨される環境を実現できます。

## 基本的な使い方

管理者は組織のマネージド設定ファイルに `pluginSuggestionMarketplaces` を追加します。

```json
{
  "pluginSuggestionMarketplaces": [
    "https://marketplace.example.com",
    "https://plugins.myorg.internal"
  ]
}
```

## 実践例

### 組織のプラグインマーケットプレイスの設定

組織が運営するプラグインマーケットプレイスを許可リストに登録する例です。

```json
{
  "pluginSuggestionMarketplaces": [
    "https://plugins.mycompany.com"
  ]
}
```

この設定により、`https://plugins.mycompany.com` に登録されたプラグインのみがユーザーへのヒントとして提案されます。

### 複数マーケットプレイスの許可リスト

複数の信頼できるマーケットプレイスを登録する例です。

```json
{
  "pluginSuggestionMarketplaces": [
    "https://marketplace.primary-vendor.com",
    "https://internal-plugins.myorg.local"
  ]
}
```

### 組織ポリシーによるプラグイン提案の制御

セキュリティポリシーに基づき、承認済みのプラグインのみを提案する設定の例です。

```json
{
  "pluginSuggestionMarketplaces": [],
  "disablePluginSuggestions": true
}
```

空の配列を指定することで、外部マーケットプレイスからのプラグイン提案を無効にできます。

## 注意点

- `pluginSuggestionMarketplaces` はマネージド設定のため、管理者権限が必要です（ユーザーレベルの settings.json では使用できません）
- リストに含まれていないマーケットプレイスのプラグインは、ヒント・提案として表示されません（手動でのインストール自体は制限されません）
- 組織のセキュリティポリシーに従って、信頼できるマーケットプレイスのみを登録してください
- コンテキスト対応のヒントは、ユーザーの作業内容に基づいて関連プラグインを提案する機能です
