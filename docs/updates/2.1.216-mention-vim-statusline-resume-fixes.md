## 原文（日本語訳）

ファイル変更を伴うフック実行後に `@メンション` が何も添付せずに失敗する問題、vimの `c` オペレーターやペーストのドットリピート（`.`）が効かない問題、レジューム時にステータスラインが2回実行される問題、レジュームピッカーが失敗時にハングする問題を修正しました。

## 原文（英語）

Fixed @-mentions silently attaching nothing after file-modifying hooks, vim dot-repeat of `c`-operators and paste, statusline running twice on resume, and resume-picker hangs on failure

## 概要

このアップデートでは、入力操作やセッション再開に関連する4つの独立した不具合がまとめて修正されました。ファイル変更フック実行後に `@` でファイルを参照しても何も添付されない問題、vimエディタモードで `c`（change）オペレーターやペースト操作をドットリピート（`.`キー）で繰り返せない問題、セッションレジューム時にステータスラインスクリプトが二重実行される問題、そしてレジュームピッカーが何らかの理由で失敗するとClaude Codeがハングしてしまう問題が解消されています。いずれも設定変更は不要で、アップデートするだけで自動的に改善されます。

## 基本的な使い方

これらはすべてバグ修正のため、ユーザー側の設定は不要です。アップデート後は以下の挙動が自動的に改善されます。

- `PostToolUse` などのファイル変更フックが動いた直後でも、`@ファイル名` を入力すると正しくファイルが会話コンテキストに添付されるようになります
- vimエディタモード（`editorMode: "vim"`）で `cw` や `p` などの操作を行った後、`.` キーで直前の変更を正しく繰り返せるようになります
- `claude --resume` でセッションを再開したとき、statusLineスクリプトが1回だけ実行されるようになります
- `claude --resume` のインタラクティブなセッション選択（ピッカー）で内部処理が失敗しても、ハングせずに適切にエラーが返るようになります

## 実践例

### フック実行後の @メンション添付を確認する

`PreToolUse`/`PostToolUse` フックでファイルを書き換えるようなワークフローを使っている場合、修正後は以下のように問題なく動作します。

```jsonc
// .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command", "command": "prettier --write \"$CLAUDE_FILE_PATH\"" }
        ]
      }
    ]
  }
}
```

このようなフォーマッター系フックの直後でも、続けて `@src/App.tsx` のようにファイルを参照すれば、これまで発生していた「添付が空になる」現象なく正しくコンテキストに読み込まれます。

### vimモードでのドットリピート

`~/.claude/settings.json` でvimモードを有効化している場合:

```json
{
  "editorMode": "vim"
}
```

NORMALモードで `cw新しいテキスト<Esc>` のように変更を行った後、別の単語にカーソルを移動して `.` を押すと、同じ変更が正しく繰り返されるようになりました。同様に `p`（ペースト）の後の `.` によるリピートも修正されています。

### resumeの動作確認

```bash
# 特定セッションを再開（statusLineは1回だけ実行される）
claude --resume <session-id>

# インタラクティブピッカーでセッションを選択
claude --resume
```

ピッカー表示中に内部エラーが発生した場合でも、CLIがフリーズせず通常通り終了・再試行できるようになりました。

## 注意点

- いずれもバグ修正であり、新しい設定項目は追加されていません
- vimモードの詳細な操作一覧は公式ドキュメントの「Vim editor mode」セクションを参照してください
- statusLineスクリプトが重い処理を行っている場合、二重実行バグの解消によりレジューム直後の応答が体感的に速くなる可能性があります
- 本修正はv2.1.216時点のものです。古いバージョンでは引き続き発生する可能性があるためアップデートを推奨します
