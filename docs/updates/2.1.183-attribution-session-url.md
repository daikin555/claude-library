## 原文（日本語に翻訳）

Web および Remote Control セッションでのコミットと PR から claude.ai セッションリンクを省略するための `attribution.sessionUrl` 設定を追加しました。

## 原文（英語）

Added `attribution.sessionUrl` setting to omit the claude.ai session link from commits and PRs in web and Remote Control sessions.

## 概要

Claude Code が Web または Remote Control セッション経由でコミットや PR を作成する際、デフォルトでは claude.ai のセッションリンクが帰属情報として付加されます。新しい `attribution.sessionUrl` 設定を使うことで、このリンクをコミットメッセージや PR 説明から省略できます。プライバシーの保護やコミット履歴の整理に役立ちます。

## 基本的な使い方

### settings.json での設定

プロジェクトの `.claude/settings.json` またはユーザー設定に追加します：

```json
{
  "attribution": {
    "sessionUrl": false
  }
}
```

### 設定の確認

```bash
# /config コマンドで確認・変更
/config attribution.sessionUrl=false
```

## 実践例

### セッションURLなしのコミット

設定前（デフォルト）のコミットメッセージ：
```
feat: add user authentication

🤖 Generated with Claude Code
Session: https://claude.ai/session/abc123...
Co-Authored-By: Claude <noreply@anthropic.com>
```

設定後（`attribution.sessionUrl: false`）のコミットメッセージ：
```
feat: add user authentication

🤖 Generated with Claude Code
Co-Authored-By: Claude <noreply@anthropic.com>
```

### プロジェクト単位での設定

```json
// .claude/settings.json
{
  "attribution": {
    "sessionUrl": false
  }
}
```

### グローバル設定での適用

```json
// ~/.claude/settings.json
{
  "attribution": {
    "sessionUrl": false
  }
}
```

### CI/CD での活用

```yaml
# GitHub Actions での例
- name: Run Claude Code
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  run: |
    claude --dangerously-skip-permissions "PRを作成してください"
  # セッションURLなしでコミット・PRが作成される（settings.jsonで設定済みの場合）
```

## 注意点

- この設定は **Web セッション** および **Remote Control セッション** でのみ有効です
- ローカルの通常セッションでは、この設定に関わらず元々セッションURLは付与されません
- `attribution` オブジェクト内の他のサブキー（将来追加される可能性あり）と組み合わせて使用できます
- プライベートリポジトリでも公開 GitHub リポジトリでも使用可能です
- セッションURLを省略してもコミットの `Co-Authored-By: Claude` は残ります（Claude の帰属は維持）
