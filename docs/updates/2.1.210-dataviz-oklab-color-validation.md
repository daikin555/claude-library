## 原文（日本語に翻訳）

バンドルされているdatavizスキルのチャート配色検証を、知覚的なOKLab色差と再調整された色覚多様性しきい値によって改善しました

## 原文（英語）

Improved the bundled dataviz skill's chart color validation with perceptual OKLab color difference and recalibrated color-blindness thresholds

## 概要

Claude Codeには、チャートやダッシュボードを作成する際に色選定のガイダンスを読み込む組み込みの `/dataviz` スキルがあります。このスキルには、配色が十分に区別可能かどうかをプログラム的にチェックするランナブルなカラーパレット検証ツールが含まれています。v2.1.210では、この検証ロジックが単純なRGB/Lab距離ではなく、より人間の知覚に近いOKLab色差モデルを使うように改善され、色覚多様性（色弱・色盲）判定のしきい値も実データに基づいて再調整されました。これにより、「数値上は違う色だが人の目にはほぼ同じに見える」誤検出や、逆に「見分けがつくのに警告される」誤検出が減ります。

## 基本的な使い方

`/dataviz` スキルはClaude Codeにあらかじめバンドルされており、チャートやダッシュボードを作成しようとすると自動的にコンテキストへ読み込まれます。明示的に呼び出すことも可能です。

```bash
claude

> ダッシュボード用に売上カテゴリ別の棒グラフを作りたい。dataviz skillを使って配色を検証して
```

Claudeはスキル内の検証ロジック（OKLab色差計算）を使い、提案した配色パレットが以下を満たすかチェックします。

- 各系列の色が知覚的に十分離れている（OKLab距離が閾値以上）
- 1型／2型色覚（P型・D型）や3型色覚（T型）でも系列を区別できる
- ライトモード・ダークモード双方で許容できるコントラストを保っている

## 実践例

### カラーパレットの検証をClaudeに依頼する

```bash
claude

> 次の5色パレットが色覚多様性の観点で問題ないか確認して:
> #1f77b4, #ff7f0e, #2ca02c, #d62728, #9467bd
```

Claudeはdatavizスキルのバリデーターを使って、各色ペアのOKLab色差をシミュレートされた色覚タイプ（P型・D型・T型）ごとに計算し、しきい値を下回るペアがあれば具体的にどの2色が問題かを指摘します。

### 既存のチャートライブラリの配色を再チェックしてもらう

```bash
claude

> src/charts/theme.ts で定義しているカテゴリカルカラー配列を読んで、
> dataviz skillの色差検証にかけてほしい。しきい値を割る組み合わせがあれば
> 代替色を提案して
```

再調整されたしきい値により、以前は「境界線上」で見逃されていた組み合わせや、逆に過剰に警告されていた組み合わせがより正確に判定されるようになります。

### ダッシュボード新規作成時の自動チェック

```bash
claude

> KPIダッシュボードを新規作成して。折れ線グラフ3本、棒グラフ2本、
> それぞれ異なる系列色を使う
```

Claudeはチャートを設計する過程でdatavizスキルを自動的に読み込み、選定した配色に対してOKLab色差チェックを内部的に実行してから提案するため、ユーザーが明示的に検証を依頼しなくても、色覚多様性に配慮した配色が選ばれやすくなります。

## 注意点

- `/dataviz` スキルはClaude Code v2.1.198で導入された組み込みスキルで、インストール作業は不要です
- 今回の改善はv2.1.210で、検証アルゴリズムを単純な色差計算からOKLab（知覚均等色空間）ベースに変更し、色覚多様性のしきい値を実測データに合わせて再調整したものです
- 検証はあくまでガイダンスであり、最終的な配色選定はClaudeの判断に委ねられます。重要なプロダクトのダッシュボードでは、実際のツール（例: シミュレーターや実機での確認）でも最終確認することを推奨します
- 色数が非常に多いパレット（8色以上など）では、OKLab距離だけで全ペアを完全に区別可能にするのは原理的に難しい場合があります。系列が多い場合は色だけでなく、線種やマーカー形状、ラベル直接表示なども併用するとよいでしょう
