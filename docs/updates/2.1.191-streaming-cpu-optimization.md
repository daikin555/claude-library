## 原文（日本語に翻訳）

テキスト更新を100msにまとめることで、ストリーミング応答中のCPU使用率を約37%削減

## 原文（英語）

Reduced CPU usage during streaming responses by ~37% by coalescing text updates to 100ms

## 概要

Claude Codeのストリーミング応答（AIが回答を逐次表示する際）のCPU使用率が約37%削減されました。テキスト更新を100ミリ秒単位にまとめること（コアレッシング）で実現しており、特に長い応答や低スペックマシンでの動作が改善されます。

## 基本的な使い方

この改善は自動的に適用されます。ユーザー側での設定変更は不要です。

```bash
# ストリーミング中（AIが応答を生成中）のCPU負荷が低減される
claude "長い説明を生成してください"

# → ストリーミング中のCPU使用率が従来比約37%減
```

## 実践例

### 長大な出力を伴うタスク

```bash
# 長いコードの生成や詳細な説明
claude "1000行のPythonスクリプトを書いてください"

# 改善前: ストリーミング中にCPUが高負荷
# 改善後: CPU使用率が約37%低減、他のアプリも快適に動作
```

### バックグラウンドでの実行

```bash
# バックグラウンドエージェントが長い出力を生成する場合
# CPU競合が減少するため、フォアグラウンドの作業が快適になる
claude "包括的なドキュメントを生成してください" &
```

### 低スペック環境での使用

```bash
# RAM/CPU が限られた環境（古いマシン、CI環境など）でのパフォーマンス向上
# ストリーミング中のCPU使用量が減るため、タイムアウトリスクが低減
```

## 技術的な詳細

**コアレッシング（Coalescing）とは：**

```
改善前（フレームごとに更新）:
t=0ms: "こんにちは"
t=10ms: "こんにちは、"
t=20ms: "こんにちは、今日は"
→ 100ms間に10回のUI更新 = CPU負荷大

改善後（100msにまとめて更新）:
t=0ms: 更新開始
t=100ms: "こんにちは、今日は..." （100ms分をまとめて表示）
→ 100ms間に1回のUI更新 = CPU負荷 約37%削減
```

表示の体感速度はほぼ変わらず（100msは人間にはリアルタイムと感じられる）、CPU効率が大幅に向上します。

## 注意点

- 100msのコアレッシングにより、ストリーミング中のテキスト表示が若干バッファされます。体感上の遅延はほとんどありませんが、非常に厳密なリアルタイム性が求められる用途では注意が必要です。
- CPU削減効果（37%）は平均値であり、応答の内容や長さによって変動します。
- この変更はUIのレンダリング最適化であり、AIの応答速度（トークン生成速度）には影響しません。
