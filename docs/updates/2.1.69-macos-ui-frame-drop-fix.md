## 原文（日本語に翻訳）

OAuthを使用するMCPサーバーを使用した際のmacOSのUIフレームドロップを修正（2.1.xからのリグレッション）

## 原文（英語）

Fixed UI frame drops on macOS when using MCP servers with OAuth (regression from 2.1.x)

## 概要

v2.1.x で導入されたリグレッションにより、OAuthを使用するMCPサーバー利用時にmacOSでUIのフレームドロップが発生する問題がありました。この修正により、OAuthベースのMCPサーバー（GitHub、Slack、Notionなど）を使用する際のUIの滑らかさが回復しました。

## 基本的な使い方

この修正は自動的に適用されます。v2.1.69以降では、OAuthを使用するMCPサーバーとの連携時もUIがスムーズに動作します。

```bash
# OAuthを使用するMCPサーバーを設定して使用する場合
claude
> GitHubのIssueを一覧表示してください

# 修正後: UIのフレームドロップなしに滑らかに動作
```

## 実践例

### GitHub MCPサーバーとの連携

```bash
# ~/.claude/claude_desktop_config.json または設定済みの場合
# GitHub OAuth MCPサーバーを使用した操作
> 最近のPRを一覧表示して、レビューが必要なものを教えてください

# 修正前: 処理中にUIがカクつく
# 修正後: スムーズなUI動作
```

### Slack MCPサーバーとの連携

```bash
# Slack OAuth MCPサーバーを使用
> 今日のdevチャンネルのメッセージを要約してください

# 修正後: フレームドロップなしにレスポンスが表示される
```

### 複数のOAuth MCPサーバーを同時使用

```bash
# 複数のOAuthサービスを組み合わせた場合も安定動作
> GitHubのIssueをSlackに投稿するワークフローを実行してください
```

## 注意点

- この問題はmacOS固有のリグレッションでした（v2.1.x で導入）
- OAuth認証を必要としないMCPサーバーはこの問題の影響を受けていませんでした
- 問題が解消されない場合は `claude --version` でバージョンを確認してください
