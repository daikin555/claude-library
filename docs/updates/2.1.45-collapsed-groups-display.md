## 原文（日本語に翻訳）

折りたたまれた読み取り・検索グループが、処理中のサマリー行の下に現在処理中のファイルまたは検索パターンを表示するように改善

## 原文（英語）

Improved collapsed read/search groups to show the current file or search pattern being processed beneath the summary line while active

## 概要

Claude Code 2.1.45 では、複数ファイルの読み取りや検索を行う際に表示される「折りたたみグループ」の UI が改善されました。処理中（アクティブ）の場合、グループのサマリー行の下に現在処理中のファイルパスや検索パターンがリアルタイムで表示されるようになります。これにより、Claude Code が今何を読んでいるか・何を検索しているかが一目でわかるようになり、長い処理でも状況を把握しやすくなります。

## 基本的な使い方

この改善は UI の表示に関するものです。特別な設定は不要で、Claude Code を 2.1.45 以上にアップデートするだけで自動的に適用されます。

## 実践例

### 複数ファイル読み取り時の表示

大規模なコードベースを分析する際、Claude Code は複数のファイルを順次読み込みます。

```
# 修正前の表示（どのファイルを読んでいるかわからない）
▼ Reading 15 files...

# 修正後の表示（現在処理中のファイルが見える）
▼ Reading 15 files...
  📄 src/components/UserProfile.tsx
```

### 検索実行中の表示

`grep` や `Glob` などの検索ツールを使っている場合：

```
# 修正前の表示
▼ Searching 8 patterns...

# 修正後の表示（現在の検索パターンが見える）
▼ Searching 8 patterns...
  🔍 **/*.test.ts
```

### 長い処理での進捗確認

大規模なリポジトリを分析するプロンプトを出した場合：

```
> このリポジトリ全体のアーキテクチャを分析してください

# Claude Code の表示例（2.1.45 以降）
▼ Reading 42 files...
  📄 src/core/index.ts        ← 今このファイルを読んでいる
```

処理が完了すると折りたたみグループに戻ります：

```
▶ Read 42 files              ← 処理完了後は折りたたまれる
```

## 注意点

- この表示はファイル読み取りや検索が「アクティブな処理中」の場合のみ表示されます。完了後は従来通りサマリー行のみ表示されます。
- 表示されるファイルパスや検索パターンはリアルタイムで更新されます。
- 処理が非常に高速な場合、表示が切り替わる前に完了することもあります。
- この UI 改善はコマンドラインの Claude Code でのみ有効です（SDK から使う場合は異なります）。
