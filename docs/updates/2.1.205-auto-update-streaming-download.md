## 原文（日本語訳）

自動アップデート時のバイナリダウンロードが、メモリ上へのバッファリングではなくディスクへのストリーミング方式に変更された。これにより、アップデータのピークメモリ使用量が約400MB削減される。

## 原文（英語）

Auto-update binary downloads now stream to disk instead of buffering in memory, cutting the updater's peak memory usage by roughly 400 MB

## 概要

Claude Codeはネイティブインストールの場合、起動時やセッション中に自動でバックグラウンド更新をチェックし、新しいバイナリをダウンロード・適用します。従来はダウンロードしたバイナリデータを一旦メモリ上にすべて読み込んでからディスクに書き込んでいたため、更新のたびに大きなメモリ消費が発生していました。今回の変更では、ダウンロードしたデータを逐次ディスクへストリーミング書き込みする方式に変わり、アップデータのピークメモリ使用量が約400MB削減されています。メモリ制約のある環境やコンテナ環境でのバックグラウンド更新の安定性向上につながる改善です。

## 基本的な使い方

自動アップデートは通常、ユーザーが意識せず動作します。ネイティブインストールでは、Claude Codeの起動時およびセッション中に定期的に更新チェックが行われ、新しいバイナリはバックグラウンドでダウンロード・適用され、次回起動時に反映されます。

直近のアップデート適用結果を確認したい場合は以下を実行します。

```text
claude doctor
```

今すぐ最新版を取得したい場合は、バックグラウンド更新を待たずに手動で更新できます。

```text
claude update
```

## 実践例

### メモリ制約のあるコンテナ・サンドボックス環境

CPUやメモリの割り当てが小さいコンテナ、あるいはメモリ上限が設定されたサンドボックス環境でClaude Codeを常駐させている場合、従来の「ダウンロードデータを丸ごとメモリに保持してからディスクへ書き出す」方式では、更新処理のタイミングでメモリ使用量が一時的に跳ね上がり、OOM（Out of Memory）によるプロセスキルのリスクがありました。ストリーミングダウンロード化によってピークメモリが約400MB下がったことで、こうした環境でもバックグラウンド更新が安定して完走しやすくなります。

```json
{
  "env": {
    "DISABLE_AUTOUPDATER": "1"
  }
}
```

なお、メモリ制約が特に厳しく自動更新自体を避けたい場合は、上記のように `settings.json` の `env` に `DISABLE_AUTOUPDATER` を設定してバックグラウンド更新を無効化し、必要なタイミングで `claude update` を手動実行する運用も選択できます。

### CI環境やヘッドレス実行での多数プロセス並列稼働

CIパイプラインなどでClaude Codeを多数のジョブ・コンテナで同時に起動する場合、各プロセスが自動更新チェックのタイミングで同時にバイナリダウンロードを行うと、ホスト全体のメモリ使用量が積み上がりやすい状況でした。1プロセスあたりのピークメモリが約400MB下がることで、同時実行数が多い環境でもホスト全体のメモリ圧迫が緩和されます。CI環境では、更新タイミングのばらつきによるビルドの再現性低下を避けるため、`autoUpdatesChannel` の設定や `DISABLE_AUTOUPDATER` を使ってバージョンを固定する運用と組み合わせるとより安定します。

## 注意点

- この改善はネイティブインストールの自動アップデート機構に関するものです。Homebrew・WinGet・Linuxパッケージマネージャー経由でインストールしている場合、既定では自動更新は行われず手動更新が必要です（`CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE=1` で自動更新化も可能）。
- 約400MBという削減幅はダウンロードするバイナリのサイズや実行環境に依存するため、実際の削減量は環境によって変動します。
- 自動更新を完全に止めたい場合は `DISABLE_AUTOUPDATER=1`、あるいは関連する非必須通信をまとめて無効化する `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` を利用できますが、これらは自動更新チェックを止めるだけで、`claude update` や `claude install` による手動更新は引き続き利用可能です。
- 更新結果や失敗の有無は `claude doctor` で確認できます。
