## 原文（日本語訳）

バックグラウンドセッション: クライアントが接続されていない場合、`/mcp` と `/install-github-app` がエージェントビューに「入力待ち（needs input）」のリクエストを表示するようになりました。

## 原文（英語）

Background sessions: `/mcp` and `/install-github-app` now park a "needs input" request in the agent view when no client is attached

## 概要

`/mcp` や `/install-github-app` はインタラクティブな設定フロー（認証やOAuthの承認など）を必要とするコマンドです。従来、ターミナルなどのクライアントが接続されていないバックグラウンドセッションでこれらのコマンドを実行すると、応答できずに処理が止まってしまう、あるいは単に拒否されるといった挙動でした。今回の変更により、クライアント未接続時にはリクエストがエージェントビュー上に「入力待ち」として保留され、後からクライアントを接続した際に対応できるようになりました。

## 基本的な使い方

この修正は自動的に適用される挙動改善のため、特別な設定は不要です。バックグラウンドで動作しているセッションに対して `/mcp` や `/install-github-app` を実行すると、クライアントが接続されていない場合は自動的にエージェントビュー上で「入力待ち」の状態として扱われます。

```
/mcp
```

```
/install-github-app
```

## 実践例

### バックグラウンドで動いているエージェントにMCPサーバーを追加したい場合

CLIやWebのagent viewからバックグラウンドセッションを開き、`/mcp` を実行してMCPサーバーの認証を行おうとすると、クライアントが未接続の間はリクエストが「入力待ち」としてagent viewに表示されます。ターミナルやWeb UIを開いてセッションに接続すると、保留されていた入力待ちリクエストに応答できます。

```
# バックグラウンドセッション側で実行
/mcp add my-server -- npx -y @my-org/mcp-server
```

### GitHub Appのインストールをバックグラウンドセッションから開始する場合

`/install-github-app` を実行した際、対話的な承認フローが必要になりますが、クライアントがまだ接続されていなければリクエストはagent view上に保留され、処理がハングしたり黙って失敗したりすることがなくなりました。

```
/install-github-app
```

後でエージェントビューを開くと、保留中の「入力待ち」リクエストとして表示されるため、そこから承認フローを続行できます。

## 注意点

- この挙動はバックグラウンドセッション（クライアントが接続されていない状態）にのみ適用されます。通常のインタラクティブセッションでは従来通り即座にプロンプトが表示されます。
- 「入力待ち」状態のリクエストは、クライアントが接続されるまで保留され続けます。長時間放置すると、意図せずセッションが応答待ちのまま残る点には注意してください。
- OAuth認証などブラウザでの操作が必要なフローは、引き続きブラウザ経由でのユーザー操作が必要です。
