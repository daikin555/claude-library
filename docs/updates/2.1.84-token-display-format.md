## 原文（日本語に翻訳）

100万以上のトークン数が「1512.6k」ではなく「1.5m」として表示されるようになりました。

## 原文（英語）

Token counts ≥1M now display as "1.5m" instead of "1512.6k"

## 概要

大きなコンテキストを使用する際のトークン数表示が改善されました。これまで100万トークン以上の場合は「1512.6k」のように表示されていましたが、「1.5m」のように百万単位の短縮形で表示されるようになりました。長いセッションや大きなファイルを扱う際のトークン数の把握が容易になります。

## 基本的な使い方

表示形式の変更は自動的に適用されます。特別な設定は不要です。

```
// 変更前
Context: 1512.6k tokens

// 変更後
Context: 1.5m tokens
```

## 実践例

### 大規模コードベースの分析時

大きなリポジトリ全体を読み込む場合など、コンテキストが100万トークンを超えることがあります。

```
# /stats コマンドでのトークン表示例（変更後）
Total tokens used: 2.3m
Cached tokens: 1.8m
New tokens: 0.5m
```

### 長時間セッションでのモニタリング

```
# セッション進行とともにトークン数が増加するケース
Opening context... 856.3k tokens
After file analysis... 1.2m tokens
After code generation... 1.8m tokens
```

### Claude.aiの最大コンテキスト活用

Claude 3.5/3.7シリーズの200kトークンや将来の大規模コンテキストモデルを使用する際、表示がより読みやすくなります。

```
コンテキスト使用量: 1.4m / 2.0m (70%)
```

## 注意点

- この変更は**表示のみ**です。実際のトークン計算や課金には影響しません
- 「m」は百万（million）を表します（「k」は千）
- 小数点1桁まで表示されます（例: 1.5m = 1,500,000トークン前後）
- 100万トークン未満の場合は引き続き「k」表示が使用されます
