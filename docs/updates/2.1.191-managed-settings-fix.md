## 原文（日本語に翻訳）

管理設定を修正：`forceRemoteSettingsRefresh` がMDMまたはファイルポリシー経由で設定された場合に有効になり、フェッチ時にプロキシが古い応答を返すことを防ぐ `Cache-Control: no-cache` を送信するようになりました

## 原文（英語）

Fixed managed settings: `forceRemoteSettingsRefresh` now takes effect when set via MDM or file policy, and the fetch sends `Cache-Control: no-cache` to prevent proxies from serving stale responses

## 概要

企業向けのマネージド設定（MDMや設定ファイルで展開される管理者設定）に関する重要な修正です。`forceRemoteSettingsRefresh` 設定がMDMやファイルポリシー経由で設定された場合に正しく機能するようになり、また設定取得時に `Cache-Control: no-cache` ヘッダーを送信してプロキシによるキャッシュ問題を防止します。

## 基本的な使い方

```json
// MDMまたは管理設定ファイル経由で展開
{
  "forceRemoteSettingsRefresh": true
}
```

改善後はこの設定がMDM経由で配布された場合でも正しく動作し、リモート設定が常に最新の状態で取得されます。

## 実践例

### 企業MDMでの設定展開

```xml
<!-- macOS MDM プロファイル（Jamf等）での設定例 -->
<dict>
  <key>com.anthropic.claude-code</key>
  <dict>
    <key>forceRemoteSettingsRefresh</key>
    <true/>
  </dict>
</dict>
```

```
# 改善前: MDM経由の forceRemoteSettingsRefresh が無視されることがあった
# 改善後: MDM設定が正しく適用され、リモート設定が強制的に更新される
```

### ファイルポリシー経由での設定

```json
// /etc/claude/managed-settings.json（Linux企業環境）
// または
// /Library/Application Support/Claude/managed-settings.json（macOS企業環境）
{
  "forceRemoteSettingsRefresh": true,
  "allowedTools": ["Read", "Edit", "Write"]
}
```

### プロキシ環境でのキャッシュ問題回避

```
# 企業プロキシ経由で設定を取得する場合
# 改善前: プロキシが古い設定をキャッシュして返すことがあった
# 改善後: Cache-Control: no-cache ヘッダーにより最新設定を確実に取得

# HTTP リクエストの改善点:
GET /settings HTTP/1.1
Host: settings.anthropic.com
Cache-Control: no-cache  ← 追加されたヘッダー
```

## 注意点

- この修正はエンタープライズ（組織管理）環境を対象としています。個人ユーザーには直接影響しません。
- `forceRemoteSettingsRefresh` を有効にすると、起動時にネットワークアクセスが必要になります。オフライン環境では注意が必要です。
- MDM設定の変更反映には、デバイスのMDMチェックイン（通常数分〜数時間）が必要です。
- 企業のプロキシがHTTPヘッダーを変更する場合、`Cache-Control: no-cache` が削除される可能性があります。
