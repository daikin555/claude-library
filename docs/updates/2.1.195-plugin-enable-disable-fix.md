## 原文（日本語に翻訳）

プラグインの `plugin.json` の `name` が marketplace エントリ名と異なる場合に `/plugin` の Enable/Disable が動作しない問題を修正

## 原文（英語）

Fixed `/plugin` Enable/Disable not working when a plugin's `plugin.json` `name` differs from its marketplace entry name

## 概要

プラグインの `plugin.json` ファイルに記載された `name` フィールドと、マーケットプレイス上のエントリ名が異なる場合に、`/plugin` コマンドの Enable/Disable が正しく機能しないバグが修正されました。プラグイン開発者が内部名と公開名を別々に管理している場合に発生していた問題です。修正後は、内部名・マーケットプレイス名・表示名のどれでも正しく識別されます。

## 基本的な使い方

```bash
# プラグインの有効化（marketplace 名または plugin.json の name どちらでも動作）
/plugin enable my-plugin

# プラグインの無効化
/plugin disable my-plugin

# インストール済みプラグインの確認
/plugin list
```

## 実践例

### 内部名と公開名が異なるプラグイン

```json
// plugin.json の例（プラグイン内部設定）
{
  "name": "my-internal-plugin-name",
  "displayName": "My Awesome Plugin"
}
```

Marketplace では "awesome-plugin" として登録されていても、`/plugin enable awesome-plugin` と `/plugin enable my-internal-plugin-name` のどちらでも正しく動作するようになりました。

### プラグイン管理のベストプラクティス

```bash
# インストール済みプラグインの詳細確認
/plugin list

# 問題が発生した場合の再インストール
/plugin disable problematic-plugin
/plugin enable problematic-plugin
```

### トラブルシューティング

```bash
# 修正前: このコマンドが失敗することがあった
/plugin enable brave-search  # marketplace 名
# エラー: "Plugin 'brave-search' not found" (内部名は "brave_search")

# 修正後: どちらの名前でも動作する
/plugin enable brave-search  # ✅ 動作する
/plugin enable brave_search  # ✅ 動作する
```

## 注意点

- プラグイン開発者向け: `plugin.json` の `name` と marketplace のエントリ名が異なっていても問題なく動作します
- ユーザー向け: Enable/Disable 操作が失敗していた場合、このバージョンで解消されます
- プラグインの識別には内部 ID も使用されるようになりました
- `/plugin list` で正確なプラグイン名を確認できます
