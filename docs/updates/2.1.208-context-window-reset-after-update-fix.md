## 原文（日本語訳）

CLIの自動アップデート後にコンテキストウィンドウ（および自動コンパクトのインジケーター）が一瞬200kにリセットされ、長いコンテキストのセッションを再開した際に誤って「コンテキスト使用率100%」と表示されてしまう不具合を修正した。

## 原文（英語）

Fixed the context window (and auto-compact indicator) briefly resetting to 200k after the CLI auto-updates, causing a false "100% context used" when resuming long-context sessions

## 概要

Claude Codeはモデルのコンテキストウィンドウ（デフォルトでは約200,000トークン）の使用状況を画面上に表示し、上限に近づくと自動コンパクトが働く仕組みになっています。これまでは、CLIが自動アップデートされた直後にこの表示が一瞬デフォルト値の200kへリセットされてしまい、すでに大きくコンテキストを消費している長時間セッションを再開した場合に、実際の使用率とは無関係に「コンテキスト使用率100%」という誤った表示が出ることがありました。今回の修正でこの表示の不整合が解消されています。

## 基本的な使い方

特別な操作は不要です。通常通りセッションを使い続けている間にCLIが自動アップデートされても、コンテキストウィンドウの表示は正しい値のまま維持されます。

```text
claude
# 通常のセッション作業中に自動アップデートが走っても
# コンテキスト使用率の表示は正しい値を保つ
```

## 実践例

### 長時間の調査セッションを再開する場合

大量のファイルを読み込んで調査を続けているような、コンテキストをすでに大きく消費しているセッションでも、アップデートをまたいだ直後に誤って「100%使用」と表示されて不要な`/compact`を促されることがなくなりました。

```text
claude --resume <session-id>
# アップデート直後でも実際の使用率が正しく表示される
```

### 自動コンパクトのタイミング判断

自動コンパクトはコンテキスト使用率がおおよそ上限に近づいた時点で発動しますが、表示がリセットされることで意図しないタイミングで警告が出たり、逆に本来必要な警告が見えづらくなったりする問題を避けられます。

```text
/context
# 自動アップデート直後でも正確なコンテキストウィンドウの内訳を確認できる
```

## 注意点

- この不具合は表示（インジケーター）上の問題であり、実際にAPIへ送信されるコンテキスト量やコンパクトの実行タイミングそのものが誤っていたわけではありません。
- コンテキストウィンドウの内訳は`/context`コマンドやコンテキストウィンドウのビジュアライザーで確認できます。
- 自動コンパクトはコンテキスト使用率が実効コンテキストウィンドウのおおよそ98%に達したタイミングで発動します。
