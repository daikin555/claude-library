## 原文（日本語に翻訳）

`claude agents` のディスパッチおよびリプライ入力でマルチラインプロンプトを入力する際にカーソルが最初の行の末尾に固定されるバグを修正しました。

## 原文（英語）

Fixed the cursor sticking at the end of the first line when typing a multiline prompt in the `claude agents` dispatch and reply inputs

## 概要

`claude agents` の dispatch（新しいタスクの送信）や reply（エージェントへの返信）の入力欄でマルチラインのプロンプトを入力する際に、カーソルが常に最初の行の末尾に固定されてしまい、正常にテキストを編集できないバグを修正しました。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
claude agents  # 修正後は複数行入力が正常に機能する
```

## 実践例

### claude agents での複数行入力

```bash
claude agents

# dispatch/reply 入力欄で複数行のプロンプトを入力
# 修正後は正常にカーソルが移動する：
"まずREADMEを読んでください。
 次に、依存関係を確認してください。
 最後に、テストを実行してください。"

# Shift+Enter で改行を追加しながらカーソルを自由に移動できる
```

### 修正前後の動作比較

**修正前：**
- 複数行テキストを入力すると、カーソルが最初の行末に戻ってしまう
- 2行目以降の編集が困難
- テキスト選択や移動が正しく機能しない

**修正後：**
- カーソルが入力位置に正しく追従する
- 複数行テキストを自由に編集可能
- 矢印キーでの行間移動が正常に動作

## 注意点

- この問題は `claude agents` の dispatch と reply 入力欄に固有のバグでした
- 通常の Claude Code プロンプト入力欄では問題は発生していませんでした
