## 原文（日本語に翻訳）

チームメートがターンを終了したときに、チームメートが開始したバックグラウンドタスクが強制終了される問題を修正しました。

## 原文（英語）

Fixed background tasks started by a teammate being killed when the teammate finishes a turn.

## 概要

tmux チームメート機能を使用する際、チームメートエージェントが長時間かかるバックグラウンドタスク（ビルド、テスト実行、ファイル監視など）を開始した後にターンを終了すると、そのバックグラウンドタスクも一緒に強制終了されるバグが修正されました。チームメートが開始したタスクは、チームメートのターン終了後も継続して実行されるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。チームメートがバックグラウンドタスクを開始する場合：

```bash
# チームメートが開始したバックグラウンドタスクの例
# npm run build &      → チームメートのターン終了後も継続
# pytest &             → チームメートのターン終了後も継続
# docker build . &     → チームメートのターン終了後も継続
```

## 実践例

### 影響を受けていたシナリオ

```
# マルチエージェントワークフロー
リーダーエージェント：「バックグラウンドでビルドを実行して、終わったら教えてください」

チームメートA：
1. `npm run build` をバックグラウンドで開始
2. 「ビルドを開始しました」とリーダーに報告
3. チームメートAのターン終了

修正前：
→ チームメートAのターン終了と同時に `npm run build` が強制終了
→ ビルドが完了しない

修正後：
→ チームメートAのターンが終了してもビルドは継続
→ ビルド完了後に通知が届く
```

### 典型的な活用パターン

```
# 長時間タスクをチームメートに委任するパターン
チームメートA: "統合テストを実行して" → バックグラウンドで開始 → ターン終了
チームメートB: "ドキュメントを生成して" → バックグラウンドで開始 → ターン終了
チームメートC: "コードのリントを実行して" → バックグラウンドで開始 → ターン終了

# 修正後: すべてのバックグラウンドタスクが並行して継続実行される
```

## 注意点

- この問題は **チームメート機能**（tmux を使ったマルチエージェント）に特有でした
- 単一エージェントでのバックグラウンドタスクには影響しませんでした
- チームメートが開始したタスクの監視は、バックグラウンドタスク通知機能を通じて行われます
- この修正により、チームメートを使った並行処理ワークフローの信頼性が大幅に向上します
