## 原文（日本語訳）

使用上限（spend limit）の調整プロンプトを改善し、使用上限の変更がサーバー側で拒否された場合にその理由を表示するようにしました。

## 原文（英語）

Improved the spend limit adjustment prompt to show the server's reason when a spend limit change is rejected

## 概要

Claude Codeでは、ワークスペースや個人アカウントの使用上限（spend limit）を調整できますが、組織のポリシーや権限の都合でサーバー側から変更が拒否されることがあります。これまではこの拒否が起きても具体的な理由が分からず、なぜ変更できないのか利用者が把握しづらい状態でした。今回の改善により、サーバーから返される拒否理由がそのままプロンプトに表示されるようになり、対処法を判断しやすくなりました。

## 基本的な使い方

使用上限の変更を試みると、拒否された場合は理由付きでメッセージが表示されます（イメージ）。

```bash
# 使用上限を変更しようとした場合
> 月間使用上限を $500 に変更しますか？

# 拒否された場合の表示例:
Spend limit change rejected: organization policy caps individual limits at $200/month.
Contact your workspace administrator to request a higher limit.
```

## 実践例

### 組織の上限ポリシーに阻まれた場合

拒否理由に組織のポリシー上限が明記されるため、管理者への問い合わせがスムーズになります。

```bash
# 例: 拒否メッセージに組織ポリシーの上限が明記される
Spend limit change rejected: exceeds organization-wide cap of $200/month.
```

このメッセージを管理者にそのまま共有することで、対応が早まります。

### 権限不足で変更できない場合

一般メンバーがワークスペース全体の上限を変更しようとした場合など、権限起因の拒否理由が表示されます。

```bash
# 例: 権限不足による拒否
Spend limit change rejected: your role does not have permission to modify workspace-level limits.
```

### Claude Consoleでの確認と調整

拒否理由を踏まえて、Claude Console側で正しい権限・上限設定を確認します。

```bash
# CLIでの変更が拒否された場合、Console側で状況を確認
# https://console.claude.com/ の Limits ページを参照
```

## 注意点

- 表示される理由文言はサーバー側の実装に依存するため、状況によって内容が異なります
- 個人プラン（Pro/Max）では `/usage-credits` など別のコマンド体系で月間上限を管理する場合があります
- 組織・チームプランでは、上限変更の権限が組織管理者やワークスペース管理者に限定されていることがあります
- 拒否理由が不明瞭な場合は、Claude Consoleの利用状況・上限ページで詳細を確認してください
