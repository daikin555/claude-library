## 原文（日本語訳）

スクロール中にVS Codeの統合ターミナルで散在する空白セルとComposerのUI要素が消える問題を修正しました。

## 原文（英語）

Fixed scattered blank cells and disappearing composer chrome in the VS Code integrated terminal during scrolling

## 概要

VS Code（および互換エディタ）の統合ターミナルでClaude Codeを使用する際、スクロール操作を行うと画面のあちこちに空白セルが散らばったり、入力欄（Composer）のUI要素（ボーダー、ラベルなどの「chrome」）が消えてしまう問題がありました。v2.1.116ではターミナルの描画処理が修正され、スクロール時でもUI要素が正しく保持されるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。VS Codeの統合ターミナルでClaude Codeを使用する際に効果があります。

```bash
# VS Codeの統合ターミナル（Ctrl+`）でClaude Codeを起動
claude

# スクロール操作中もUIが崩れなくなった
```

## 実践例

### 長い会話履歴のスクロール

```bash
# 長い会話の後にスクロールしても
# 空白セルやComposerの崩れが発生しなくなった
```

### コード生成後のスクロール

```bash
claude "このプロジェクトのコンポーネント構成を提案してください"
# → 長い提案内容が生成された後にスクロールしても表示が崩れない
```

## 注意点

- この修正はVS Codeの統合ターミナル、Cursor、Windsurfのような互換エディタでの統合ターミナルに適用されます
- 外部ターミナル（iTerm2、Ghosttyなど）では発生しない問題でした
- `/terminal-setup` を実行することで追加の最適化が適用されます
- 問題が引き続き発生する場合は `/doctor` で診断情報を確認してください
