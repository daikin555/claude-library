## 原文（日本語訳）

`--permission-prompt-tool` にMCPサーバーを指定した際、サーバーの接続処理が完了する前（コールドスタート時）に「MCP tool not found」というエラーでクラッシュしてしまう不具合を修正した。

## 原文（英語）

Fixed `--permission-prompt-tool` pointing at an MCP server crashing with "MCP tool not found" on cold start before the server finishes connecting

## 概要

`--permission-prompt-tool` は、Claude Codeがツール実行の許可をユーザーに確認する際に、指定したMCPツールを経由してその可否を判断させる仕組みです。これまでは、指定先がMCPサーバー提供のツールである場合、そのサーバーの接続処理が完了する前（コールドスタート時）に許可確認が発生すると、「MCP tool not found」というエラーでクラッシュしてしまう不具合がありました。今回の修正により、MCPサーバーの接続完了を待ってから許可確認を行うようになり、起動直後の一時的な不整合によるクラッシュが解消されています。

## 基本的な使い方

MCPサーバーが提供する権限確認用ツールを `--permission-prompt-tool` に指定して起動します。

```bash
claude --permission-prompt-tool mcp__my-permission-server__approve
```

## 実践例

### CI/非対話環境での自動承認フローの構築

CIパイプラインなど、対話的なプロンプトを表示できない環境で、MCPサーバー経由の許可判定ロジックを使って自動承認/拒否を行いたい場合に利用します。以前はプロセス起動直後にこの許可確認が走ると、MCPサーバーの接続確立前にツールが見つからずクラッシュすることがありましたが、修正後は接続完了を待つため安定して動作します。

```bash
claude --permission-prompt-tool mcp__ci-approval__check -p "デプロイスクリプトを更新して"
```

### 起動直後に即座に権限確認が必要なタスクの実行

セッション開始と同時に権限が必要なツール呼び出しを行うようなタスクでは、MCPサーバーの起動・接続に多少の時間がかかるため、これまで起動直後のタイミング次第でクラッシュが再現していました。修正後はこのタイミング依存のクラッシュが解消され、コールドスタートでも安定して許可確認が機能します。

```bash
claude --permission-prompt-tool mcp__my-permission-server__approve "設定ファイルを即座に書き換えて"
```

## 注意点

- `--permission-prompt-tool` によるMCP経由の許可確認は、応答が返るまでClaudeの実行がブロックされるため、承認ロジック側の処理は極力短時間で完了するように設計してください。
- 非対話モードでは、ユーザー操作を要求するようなツールに対して許可ツールが「allow」を返しても、「MCP tool requires user interaction; not supported via --permission-prompt-tool」として拒否に変換される仕様がある点に注意してください。
- 許可判定は、`settings.json` の `allowedTools`/`disallowedTools` や `--allowedTools`/`--disallowedTools` による静的ルールが優先され、それらに一致しない場合にのみ `--permission-prompt-tool` の動的判定が行われます。
