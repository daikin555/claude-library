## 原文（日本語に翻訳）

エージェントMarkdownファイルを変更し、プラグインの名前空間用に予約されている `:` を含むエージェント名を拒否するようにしました。

## 原文（英語）

Changed agent markdown files to reject agent names containing `:`, which is reserved for plugin namespacing

## 概要

カスタムサブエージェントは、frontmatterに `name` フィールドを持つMarkdownファイルとして定義します。Claude Codeでは、プラグインが提供するサブエージェントを `プラグイン名:エージェント名` のような形式で名前空間化して区別しており、コロン（`:`）はこの名前空間表記のために予約された記号です。これまでは、ユーザーが自分で定義するエージェントの `name` フィールドにコロンを含めることができてしまい、プラグインの名前空間表記と衝突・混同する可能性がありました。v2.1.218以降、エージェント定義自身の `name` にコロンを含めることはできなくなり、そのようなファイルはエラーとして拒否されるようになりました。

## 基本的な使い方

```yaml
---
name: code-reviewer
description: コードレビューを行うサブエージェント
---

このエージェントはコードレビューに特化しています。
```

上記のように、`name` はハイフン区切りのkebab-case（英数字とハイフンのみ）で指定します。

## 実践例

### 誤った例（拒否されるようになった名前）

```yaml
---
name: my-plugin:code-reviewer
description: コードレビューを行うサブエージェント
---
```

このように自分で定義するエージェントファイルの `name` にコロンを含めると、v2.1.218以降はエラーとなり読み込まれません。コロンはプラグインが提供するエージェントを名前空間化するためにClaude Code内部で使用される記号であり、ユーザー定義のエージェント名には使用できません。

### 正しい例（kebab-caseで命名）

```yaml
---
name: my-plugin-code-reviewer
description: コードレビューを行うサブエージェント
---
```

コロンの代わりにハイフンを使うことで、有効な名前として認識されます。

### プラグイン提供のエージェントとの見分け方

```bash
# プラグインをインストールすると、
# そのプラグインが提供するエージェントは
# "プラグイン名:エージェント名" の形式で一覧に表示される
> @
# 候補: my-plugin:code-reviewer （プラグイン提供）
#       my-code-reviewer         （自分で定義したエージェント）
```

## 注意点

- 既存のエージェントファイルの `name` にコロンを使っていた場合、v2.1.218へのアップデート後にそのファイルが読み込まれなくなる可能性があります。エラーが出た場合はハイフンなどコロン以外の文字に名前を変更してください
- エージェント名として使用できるのは基本的に小文字英数字とハイフン（kebab-case）です
- この制限はプラグインの名前空間表記（`プラグイン名:エージェント名`）との衝突を避けるためのものであり、プラグイン自身が提供するエージェント名にコロンが含まれることは想定内の正常な挙動です
