## 原文（日本語訳）

`claude-opus-4-8` における `/code-review` のfindings（指摘事項）の品質が、すべてのeffortレベルにわたって改善された。

## 原文（英語）

Improved `/code-review` findings quality on claude-opus-4-8 across all effort levels

## 概要

`/code-review` コマンドは、変更差分に対してcorrectnessバグやコードの重複・簡潔化・効率性などの観点から複数のエージェントを並列に走らせ、レビュー結果（findings）を報告する機能です。今回のアップデートでは、モデル `claude-opus-4-8` を使用した場合の指摘内容の精度・有用性が、`low`・`medium`・`high` などすべてのeffortレベルで底上げされました。誤検知の削減や、より実用的で信頼度の高い指摘が期待できます。

## 基本的な使い方

```text
/code-review
```

effortレベルを指定して実行することもできます。

```text
/code-review --effort high
```

## 実践例

### PR作成前のセルフレビュー

ローカルでの変更をコミットする前に、`claude-opus-4-8` をモデルとして使っているセッションで `/code-review` を実行し、バグや簡潔化の余地がないかを確認します。

```text
/code-review
```

effortレベルが低い場合でも、今回の改善により指摘の精度が高まっているため、素早く軽量なチェックとして使いやすくなっています。

### 高精度が必要なクリティカルな変更のレビュー

決済処理や認証まわりなど、慎重な確認が必要な変更では、effortレベルを上げて実行することで、より広いカバレッジの指摘を得られます。

```text
/code-review --effort xhigh
```

`claude-opus-4-8` を使うことで、以前よりも高いeffortレベルにおいても質の高いfindingsが得られるようになりました。

### 指摘結果をPRコメントとして投稿

レビュー結果をGitHub PR上のインラインコメントとして残したい場合は、`--comment` オプションを付与します。

```text
/code-review --comment
```

改善された指摘品質により、レビューコメントの信頼性向上が期待できます。

## 注意点

- 本改善はモデル `claude-opus-4-8` を使用している場合の効果であり、他モデル利用時の挙動を直接変更するものではありません。
- `/code-review` はテストやドキュメントのみの差分など、実行時の挙動を検証できない変更には向いていません。
- `--fix` オプションを併用すると、指摘内容をそのまま作業ツリーに適用できますが、適用前に差分を確認することが推奨されます。
