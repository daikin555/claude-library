## 原文（日本語訳）

バックグラウンドエージェントが自動アップデートによる再起動の最中である場合、`claude attach` がその再起動から復帰するのを待たずにエラーを出していた不具合を修正。

## 原文（英語）

Fixed `claude attach` erroring when a background agent was mid-upgrade restart instead of waiting for it to come back

## 概要

`claude attach` は、`claude --bg` などで起動したバックグラウンドセッション（バックグラウンドエージェント）に、シェルからセッションIDを指定して接続し直すためのコマンドです。Claude Codeはネイティブインストール環境で自動アップデートを行っており、更新が適用されるタイミングでバックグラウンドエージェントのプロセスが再起動されることがあります。今回の修正前は、ちょうどこの再起動中のタイミングで `claude attach` を実行するとエラーになってしまい、再接続に失敗していました。修正後は、プロセスが再起動から復帰するまで `claude attach` 側が待機するようになり、アップデートに伴う一時的な再起動と接続タイミングが重なっても正常にセッションへ復帰できます。

## 基本的な使い方

バックグラウンドで起動したセッションのIDを指定して接続します。

```bash
# バックグラウンドでタスクを実行するセッションを起動
claude --bg "tests/auth の flaky なテストを調査して"

# 表示されたセッションIDを使って後から接続
claude attach 7c5dcf5d
```

接続すると、離れている間に何が起きたかの簡単な要約が表示され、以降は通常のセッションと同様にコマンドやキーボードショートカットがそのまま使えます。空のプロンプトで `←` を押すか `/exit` を実行すると、再びデタッチしてagent view（`claude agents`）に戻れます。

## 実践例

### アップデート直後のタイミングでバックグラウンドセッションに再接続する

長時間実行中のバックグラウンドエージェントがある状態で、Claude Codeの自動アップデートが走り再起動が発生した直後に `claude attach` を実行しても、以前は「エラーになって接続できない」ことがありました。この修正により、再起動が完了してエージェントのプロセスが復帰するまで内部的に待ってから接続するため、ユーザー側は特に何もせずそのまま `claude attach <id>` を再実行すれば復帰できます。

```bash
# 自動アップデートによる再起動が発生していても
claude attach 7c5dcf5d
# → 以前: 即座にエラー
# → 修正後: プロセスの復帰を待ってから正常に接続
```

### 複数のバックグラウンドエージェントを長期間運用する

夜間バッチ処理やCIに近い形で複数のバックグラウンドエージェントを長時間走らせている場合、Claude Codeの自動アップデートは定期的にバックグラウンドで発生し得ます。`claude agents` のagent view から一覧を確認し、必要なセッションだけ `claude attach` で個別に接続する運用を続けていても、アップデートによる再起動のタイミングと接続操作が重なることで接続エラーが起きなくなったため、監視・介入のワークフローが安定します。

```bash
# 実行中のバックグラウンドセッション一覧を確認
claude agents

# 特定のセッションに接続（アップデート直後でも安定して接続できる）
claude attach <session-id>
```

## 注意点

- この修正はあくまで「再起動中に接続しようとした際の待機・復帰処理」に関するものであり、自動アップデート自体の挙動（アップデートの有無やタイミング）を変更するものではありません。
- 自動アップデートの挙動は `settings.json` の `autoUpdatesChannel`（`latest`/`stable` の切り替え）や、環境変数 `DISABLE_AUTOUPDATER` で制御できます。アップデートを完全に無効化している環境では、そもそも本不具合の状況（再起動中のattach）自体が発生しません。
- Homebrew/WinGetなど一部のインストール方法では `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE` を有効にした場合にバックグラウンドでアップグレードコマンドが走り、成功時に再起動を促されることがあります。このようなケースでも同様に、再起動と `claude attach` のタイミングが重なる可能性があります。
- 本修正はv2.1.198以降で統一された「agent viewから開いた場合とシェルの `claude attach <id>` から開いた場合の挙動が同じである」という前提のもとで、両方の経路に適用されます。
