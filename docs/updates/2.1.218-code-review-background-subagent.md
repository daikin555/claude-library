## 原文（日本語に翻訳）

`/code-review` をバックグラウンドサブエージェントとして実行するように変更しました。これにより、レビュー作業が会話を占有しなくなり、また、それより前に実行されたスラッシュコマンドによる変更をレビュー対象として引き続き扱えるようになります。

## 原文（英語）

Changed `/code-review` to run as a background subagent, so review work no longer fills your conversation and keeps stacked slash commands as its review target

## 概要

`/code-review` はこれまで、メインの会話の中で直接レビュー処理を実行していたため、レビューの過程（差分の読み込みや検討過程など）が会話のトランスクリプトを埋めてしまい、他の作業がしづらくなることがありました。v2.1.218からは `/code-review` がバックグラウンドサブエージェントとして実行されるようになり、レビュー作業はメインの会話の外で進行し、結果だけが会話に返されます。また、「stacked slash commands（積み重ねられたスラッシュコマンド）」、つまり同じ会話の中でそれ以前に実行した `/code-review` などのスラッシュコマンドが加えた変更を、引き続きレビュー対象として正しく扱えるようになっています。

## 基本的な使い方

```bash
/code-review
```

これまでと同じコマンドで呼び出せますが、実行後の見え方が変わります。

```
# 変更前: レビューの実行過程がそのまま会話に表示され、
#          トランスクリプトが長くなる

# 変更後: レビューはバックグラウンドのサブエージェントとして進行し、
#          会話は埋まらない。完了するとレビュー結果のサマリーが
#          会話に返される
```

## 実践例

### 通常の差分レビュー

現在のブランチの変更点をレビューする、最も基本的な使い方です。

```bash
/code-review
```

レビューがバックグラウンドで実行されている間も、メインの会話で別の指示を出したり作業を続けたりできます。完了すると、レビューコメントが会話に反映されます。

### 複数のスラッシュコマンドを連続して実行した後のレビュー

会話の中で `/code-review` より前に別のスラッシュコマンド（例えば自動修正系のコマンドなど）を実行し、その変更を含めてレビューしたい場合です。

```bash
# 1. 何らかのスラッシュコマンドでコードを変更
/some-fix-command

# 2. その変更を含めてレビュー
/code-review
```

「stacked slash commands」の扱いにより、`/code-review` は直前のスラッシュコマンドが加えた変更を正しくレビュー対象として認識します。

### バックグラウンドで進行中に他の作業を続ける

レビューが完了するのを待たずに、別の質問や別のタスクを続けられます。

```bash
/code-review
# レビューはバックグラウンドで進行

# レビューの完了を待たずに、別の作業を続行できる
このファイルのリファクタリングを手伝って
```

## 注意点

- レビュー結果は完了時に会話へ報告されるため、レビューの途中経過を逐次確認したい場合は Agent View などバックグラウンドタスクの管理画面から状況を確認できます。
- 大規模な差分や複数PRにまたがる深いレビューを行いたい場合は、クラウド上で実行される `/code-review ultra`（旧 `/ultrareview`）の利用も検討してください。
- レビュー対象は引き続き現在のブランチとベースブランチとの差分が基本です。想定と異なるベースと比較された場合は、明示的にベースを指定してください。
