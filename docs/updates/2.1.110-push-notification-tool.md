## 原文（日本語に翻訳）

プッシュ通知ツールを追加 — Remote Controlと「Push when Claude decides」設定が有効な場合、ClaudeはモバイルプッシュNotificationを送信できるようになりました。

## 原文（英語）

Added push notification tool — Claude can send mobile push notifications when Remote Control and "Push when Claude decides" config are enabled.

## 概要

Claude Code v2.1.110では、Claudeが自律的にモバイルプッシュ通知を送信できる新しいツールが追加されました。Remote Control機能と「Push when Claude decides」設定の両方が有効になっている場合、Claudeは長時間タスクの完了や重要なイベント発生時に、ユーザーのモバイルデバイスへ通知を送ることができます。これにより、PCの画面を常に監視しなくても、重要な処理の完了を即座に把握できるようになります。

## 基本的な使い方

### 前提条件の確認

プッシュ通知ツールを利用するには以下の設定が必要です：

1. **Remote Controlの有効化** — claude.aiでRemote Control機能を有効にする
2. **「Push when Claude decides」設定の有効化** — 通知設定でClaudeの判断による通知を許可する
3. **モバイルアプリのインストール** — スマートフォンにClaude公式アプリをインストール済みであること

### 設定確認コマンド

```bash
# Remote Control が有効かどうか確認
claude config get remoteControl

# 現在の通知設定を確認
claude config get pushNotifications
```

## 実践例

### ユースケース1: 長時間タスク完了の通知

大規模なビルドやテスト実行など、時間のかかる処理が完了したときにClaudeが自動で通知を送ります。

```
ユーザー: 全テストスイートを実行して、完了したら通知して。

Claude: テストスイートの実行を開始します。
（数分後）
[push_notification ツール実行]
通知内容: 「全テストスイートが完了しました。結果: 142件成功、3件失敗」
```

### ユースケース2: 重要なエラー検出時の通知

ログ監視中に重大なエラーが検出された場合、Claudeが判断してプッシュ通知を送ります。

```
ユーザー: production ログを監視して、CRITICALエラーが出たら知らせて。

Claude: ログ監視を開始します。
（エラー検出時）
[push_notification ツール実行]
通知内容: 「CRITICAL: データベース接続エラーが検出されました（エラーコード: DB-5001）」
```

### ユースケース3: デプロイ完了通知

CI/CDパイプラインの実行やデプロイが完了したときの通知。

```bash
# Claude に依頼するプロンプト例
claude "本番環境へのデプロイを実行して、完了またはエラー発生時にプッシュ通知を送って"
```

Claudeはデプロイの進行を監視し、完了または失敗を検出した時点でモバイルに通知を送ります。

### ユースケース4: バックグラウンド処理の進捗報告

```
ユーザー: 大量のCSVデータを処理して、25%ごとに進捗をプッシュ通知して。

Claude: CSVデータ処理を開始します。
[25%完了時] 通知: 「処理進捗: 25% 完了（10,000件/40,000件）」
[50%完了時] 通知: 「処理進捗: 50% 完了（20,000件/40,000件）」
[75%完了時] 通知: 「処理進捗: 75% 完了（30,000件/40,000件）」
[完了時]    通知: 「CSVデータ処理完了（40,000件すべて処理済み）」
```

## 注意点

- **Remote Controlが必須** — この機能はRemote Controlが有効なセッションでのみ動作します。ローカルCLIのみの使用では通知は送信されません。
- **「Push when Claude decides」の設定** — Claudeが自律的に通知を送るには、この設定を明示的に有効にする必要があります。設定が無効の場合、ユーザーが明示的に要求した場合のみ通知が送られます。
- **通知頻度に注意** — 過度に頻繁な通知はユーザー体験を損ないます。重要なイベントのみ通知するようClaudeに指示することを推奨します。
- **バッテリーへの影響** — プッシュ通知の受信にはアプリのバックグラウンド実行が必要なため、モバイルのバッテリー消費に若干影響する場合があります。
- **モバイルアプリの通知許可** — OSレベルでClaude アプリの通知許可が有効になっていることを確認してください。
