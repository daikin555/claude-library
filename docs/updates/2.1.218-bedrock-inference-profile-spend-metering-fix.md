## 原文（日本語に翻訳）

ゲートウェイの支出計測（spend metering）で、Bedrockのapplication-inference-profile ARNやその他の設定でマッピングされたアップストリームモデルIDが、設定済みモデルの正しい料金で計算されるよう修正しました

## 原文（英語）

Fixed gateway spend metering to price Bedrock application-inference-profile ARNs and other config-mapped upstream model IDs at the configured model's rates

## 概要

AWS Bedrockのapplication-inference-profile ARNや、設定ファイルでカスタムにマッピングされたアップストリームのモデルIDを使用している場合、ゲートウェイの支出（spend）計測処理がこれらのIDを設定済みモデルの料金に正しく紐付けられず、支出額が過小または過大に計算されることがありました。修正後は、こうした設定でマッピングされたモデルIDに対しても、対応する設定済みモデルの正しい料金レートが適用され、支出計測が正確になります。

## 注意点

- Claude Code v2.1.218で修正
- Bedrockのapplication-inference-profile ARN、および設定でマッピングされたその他のアップストリームモデルIDを利用しているゲートウェイ経由の利用が対象
- 支出上限（spend limit）を運用しているチームは、これまで記録されていた支出額が実際より不正確だった可能性がある点に注意
- 修正後は設定済みモデルの正しいレートで支出が計測される
