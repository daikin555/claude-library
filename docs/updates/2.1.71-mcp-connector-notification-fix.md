## 原文（日本語に翻訳）

claude.ai MCPコネクターのスタートアップ通知が、以前に接続したコネクターのみではなく、全ての組織設定コネクターに対して表示されていた問題を修正しました。

## 原文（英語）

Fixed claude.ai MCP connector startup notifications appearing for every org-configured connector instead of only previously connected ones

## 概要

claude.aiのMCPコネクターが起動時に通知を表示する際、以前に実際に接続していたコネクターのみに通知を限定すべきところ、組織で設定されている全てのコネクターに対して通知が表示されていた問題が修正されました。これにより、特に多数のコネクターが組織設定されている環境では、起動時に大量の通知が表示されてしまっていました。修正後は実際に以前接続したコネクターのみに通知が表示されます。

## 基本的な使い方

この修正はv2.1.71以降で自動的に適用されます。起動時のMCPコネクター通知が適切にフィルタリングされます。

```bash
# Claude Codeの起動
claude

# 修正前: 組織で設定されている全コネクターの通知が表示される
# 修正後: 以前に接続していたコネクターのみ通知が表示される
```

## 実践例

### 大規模な組織環境での起動

多数のorg設定コネクターがある環境でのClaude Code起動。

```bash
# 100個のコネクターが組織で設定されている環境
claude

# 修正前: 100個全てのコネクター通知が表示される
# 修正後: 以前接続していた（例: 3個の）コネクターのみ通知される
# → 起動時のノイズが大幅に削減される
```

### 新規ユーザーの初回起動

組織のコネクターが設定されているが、まだ接続したことがないユーザーの起動。

```bash
# 新規ユーザーがClaude Codeを起動
claude

# 修正前: 組織の全コネクター通知が表示される
# 修正後: 以前に接続したコネクターがないため、通知は表示されない
```

### 特定コネクターのみ使用するワークフロー

組織で設定されているコネクターの一部のみを使用する場合。

```bash
# 3つのコネクターのみ以前に接続したことがある
claude

# 修正後: 以前接続した3つのコネクターのみ通知される
# 残りの未使用コネクターは通知されない
```

## 注意点

- この修正は組織（org）設定のclaude.ai MCPコネクターを使用している環境に特化した修正です。
- 個人設定のコネクターには影響しません。
- 関連する修正として、未認証コネクターのスタートアップ通知ノイズ削除（`2.1.71-startup-notification-noise-removed.md`）もv2.1.71で対応されています。
- コネクターの接続履歴はClaude Codeが管理しており、手動でリセットすることはできません。
