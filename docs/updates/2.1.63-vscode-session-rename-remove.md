# VSCode：セッションリストにリネームと削除アクションを追加

## 原文（日本語に翻訳）

VSCode：セッションリストにセッションのリネームと削除アクションが追加されました。

## 原文（英語）

VSCode: Added session rename and remove actions to the sessions list

## 概要

Claude Code VSCode拡張機能のv2.1.63から、セッションリストでセッションのリネームと削除が可能になりました。これにより、長期的なプロジェクトでの複数セッション管理が容易になります。意味のある名前をつけることで、過去の会話に素早くアクセスできます。

## 基本的な使い方

### セッションのリネーム

1. VSCodeのClaude Code拡張機能パネルを開く
2. セッションリストで対象のセッションを右クリックまたはホバー
3. 「Rename（リネーム）」を選択
4. 新しい名前を入力して確定

### セッションの削除

1. セッションリストで対象のセッションを右クリックまたはホバー
2. 「Remove（削除）」を選択
3. 確認ダイアログで削除を確定

## 実践例

### プロジェクト別セッションの整理

```
セッションリストの例（リネーム前）：
- セッション 2026-02-20 14:30
- セッション 2026-02-21 09:15
- セッション 2026-02-28 10:00

リネーム後：
- auth-feature-implementation
- api-refactoring
- bug-fix-payment-flow
```

### 古いセッションの削除

```
# 完了したプロジェクトや不要になったセッションを削除
# セッションリストをクリーンに保つ

1. 完了したセッション "old-feature-x" を右クリック
2. "Remove" を選択
3. セッション履歴から削除される
```

### チーム開発でのセッション管理

```
# 複数の機能に同時に取り組む際のセッション管理
セッション名の例:
- feature/user-authentication
- feature/payment-integration
- bugfix/memory-leak-fix
- review/pr-123
```

## 注意点

- この機能はVSCode拡張機能専用です。CLI版のClaude Codeには適用されません
- セッションを削除すると、その会話履歴は永続的に削除されます
- リネームはUIでの表示名の変更であり、内部IDには影響しません
