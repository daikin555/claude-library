## 原文（日本語に翻訳）

モデルの応答テキストを含む `claude_code.assistant_response` OpenTelemetryログイベントを追加。`OTEL_LOG_ASSISTANT_RESPONSES=1` が設定されていない限り内容は削除される。この変数が未設定の場合は `OTEL_LOG_USER_PROMPTS` に従うため、すでにプロンプト内容をログしているデプロイメントはアップグレード時にレスポンス内容も受け取り始める。プロンプトのみのログを維持するには `OTEL_LOG_ASSISTANT_RESPONSES=0` を設定すること。

## 原文（英語）

Added `claude_code.assistant_response` OpenTelemetry log event containing the model's response text. Redacted unless `OTEL_LOG_ASSISTANT_RESPONSES=1`; when that var is unset it follows `OTEL_LOG_USER_PROMPTS`, so deployments that already log prompt content will start receiving response content on upgrade — set `OTEL_LOG_ASSISTANT_RESPONSES=0` to keep prompts-only.

## 概要

Claude Code のOpenTelemetryサポートに新しいログイベント `claude_code.assistant_response` が追加されました。これによりモデルの応答テキストをオブザーバビリティシステムで記録できます。デフォルトでは内容が削除（redact）されており、明示的に有効化が必要です。すでにユーザープロンプトをログしている環境では、アップグレード後に応答内容も記録され始める点に注意が必要です。

## 基本的な使い方

### アシスタントレスポンスのログを有効化する

```bash
# アシスタントレスポンスのログを有効化
export OTEL_LOG_ASSISTANT_RESPONSES=1

# または .env ファイルに追記
echo "OTEL_LOG_ASSISTANT_RESPONSES=1" >> .env
```

### プロンプトのみのログを維持する（重要）

```bash
# プロンプトをログしている環境でレスポンスをログしない場合
export OTEL_LOG_ASSISTANT_RESPONSES=0
```

## 実践例

### 基本的なOpenTelemetry設定

```bash
# OpenTelemetryエンドポイントを設定してレスポンスログを有効化
export OTEL_EXPORTER_OTLP_ENDPOINT="https://otel-collector.example.com"
export OTEL_LOG_USER_PROMPTS=1
export OTEL_LOG_ASSISTANT_RESPONSES=1
```

### Datadog / Grafanaでの活用例

```yaml
# docker-compose.yml での設定例
services:
  claude-code:
    environment:
      - OTEL_EXPORTER_OTLP_ENDPOINT=http://otel-collector:4317
      - OTEL_SERVICE_NAME=claude-code
      - OTEL_LOG_USER_PROMPTS=1
      - OTEL_LOG_ASSISTANT_RESPONSES=1
```

### ログイベントの構造

`claude_code.assistant_response` イベントには以下の情報が含まれます:

```json
{
  "name": "claude_code.assistant_response",
  "attributes": {
    "response.text": "アシスタントの応答テキスト（OTEL_LOG_ASSISTANT_RESPONSES=1 の場合）",
    "session.id": "セッションID",
    "model": "claude-opus-4-8"
  }
}
```

### 環境変数の動作マトリクス

| `OTEL_LOG_USER_PROMPTS` | `OTEL_LOG_ASSISTANT_RESPONSES` | 動作 |
|---|---|---|
| 未設定 / 0 | 未設定 | どちらもログしない |
| 1 | 未設定 | 両方ログ（フォールバック動作） |
| 1 | 0 | プロンプトのみログ |
| 1 | 1 | 両方ログ |
| 未設定 / 0 | 1 | レスポンスのみログ |

## 注意点

- **重要**: `OTEL_LOG_USER_PROMPTS=1` を設定していた環境はアップグレード後に自動的にレスポンスもログされ始めます。プロンプトのみのログを維持したい場合は `OTEL_LOG_ASSISTANT_RESPONSES=0` を明示的に設定してください
- アシスタントレスポンスには機密情報が含まれる可能性があります。ログの保存・転送先のセキュリティを確認してください
- GDPR等の規制への準拠が必要な環境では、法務チームに確認の上で設定してください
