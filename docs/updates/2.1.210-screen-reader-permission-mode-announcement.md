## 原文（日本語に翻訳）

スクリーンリーダーモードで、Shift+Tabによってモードを切り替えた際に、権限モードの変更が音声で読み上げられるようになりました

## 原文（英語）

Screen reader mode now announces permission mode changes aloud when cycling modes with Shift+Tab

## 概要

Claude Codeにはスクリーンリーダー（VoiceOverやNVDAなど）向けに、ターミナルUIを平文の1行ずつのテキストへ置き換える「スクリーンリーダーモード」（`--ax-screen-reader` など）があります。これまでこのモードでは、Shift+Tabで権限モード（通常モード／自動編集許可／プランモード／すべて許可など）を切り替えても、その変更が画面上の表示でしか分からず、スクリーンリーダー利用者は現在どのモードにいるのか把握しづらいという課題がありました。v2.1.210では、Shift+Tabでモードを切り替えるたびに、切り替わった先のモード名が音声で読み上げられるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
claude --ax-screen-reader

# Shift+Tab を押してモードを切り替える
# → 画面上の表示（フッターのモード名）は変わるが、
#   スクリーンリーダーはそれを読み上げない
# → 現在どの権限モードなのか、視覚的に確認する以外に
#   知る手段がなかった
```

### 修正後の動作

```bash
claude --ax-screen-reader

# [Screen Reader Mode: on via flag]

> ここでファイルを編集して

# Shift+Tab を押す
# → スクリーンリーダーが「Permission mode: Accept Edits」
#   のように、切り替わった先のモード名を読み上げる

# もう一度 Shift+Tab
# → 「Permission mode: Plan」のように読み上げが続く
```

## 実践例

### 環境変数でスクリーンリーダーモードを有効にして作業する

```bash
export CLAUDE_AX_SCREEN_READER=1
claude

# セッション開始時に案内行が表示される
# [Screen Reader Mode: on via env]

> このリポジトリのテストを直して
# Shift+Tab で「プランモード」に切り替え、内容を確認してから
# 再度 Shift+Tab で「自動編集許可モード」に切り替えて実行させる
# 各切り替えのたびにモード名が読み上げられるため、
# 画面を注視しなくても現在のモードを把握できる
```

### 設定ファイルで常時有効化する

```json
{
  "axScreenReader": true
}
```

VS Codeの統合ターミナルを含め、そのマシン上のすべてのセッションでスクリーンリーダーモードが有効になります。

### 権限プロンプトと組み合わせて使う

```bash
claude --ax-screen-reader

> package.json を更新して依存関係を上げて

# Permission Required: の行が読み上げ対象としてラベル付けされる
# Shift+Tab で「すべて許可（bypass permissions）」モードに
# 切り替えれば、その旨も音声で通知されるため、
# 誤って意図しない権限モードのまま作業を続けるリスクが減る
```

## 注意点

- スクリーンリーダーモードの利用にはClaude Code v2.1.181以降が必要です
- 有効化方法は3通りあり、優先順位は「`--ax-screen-reader` フラグ > `CLAUDE_AX_SCREEN_READER` 環境変数 > `axScreenReader` 設定」の順です
- スクリーンリーダーモードは自動では有効になりません（スクリーンリーダーの実行を検知して自動オンにする機能はまだありません）。手動での有効化が必要です
- これまで「プランモードへの突入のようなモード変更はまだ読み上げられない」という既知の制約がドキュメントに明記されていましたが、今回の変更はその一部（Shift+Tabによる権限モードの巡回）を解消するものです。すべてのモード変更が網羅的に読み上げられるわけではない可能性がある点に留意してください
- スクリーンリーダーモードは非対話モード（`-p` フラグ）の挙動には影響しません。非対話モードはもともと平文出力です
- 問題が見つかった場合は、使用しているスクリーンリーダー・OS・ターミナルアプリを明記のうえ、[Claude Codeのissueトラッカー](https://github.com/anthropics/claude-code/issues)に報告してください
