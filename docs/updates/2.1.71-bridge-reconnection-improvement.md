## 原文（日本語に翻訳）

ブリッジセッションの再接続を、最大10分待つ代わりに、ラップトップのスリープ解除後数秒以内に完了するよう改善しました。

## 原文（英語）

Improved bridge session reconnection to complete within seconds after laptop wake from sleep, instead of waiting up to 10 minutes

## 概要

ラップトップがスリープから復帰した際のブリッジセッションの再接続時間が大幅に改善されました。以前は再接続に最大10分もの時間がかかることがあり、その間Claude Codeが使用できない状態が続いていました。v2.1.71以降では、スリープから復帰した後、数秒以内にブリッジセッションが再接続されるようになりました。

## 基本的な使い方

この改善はv2.1.71以降で自動的に適用されます。ラップトップのスリープ/復帰後のClaude Codeの応答性が大幅に向上します。

```bash
# スリープ前にClaude Codeを使用
claude

# （ラップトップをスリープ）
# （スリープから復帰）

# 修正後: 数秒以内にブリッジセッションが再接続される
> 作業を続けてください
# 以前: 最大10分待つ必要があった
```

## 実践例

### ラップトップを閉じて再度開く場合

外出中にラップトップを閉じ、後で再度開いてセッションを再開する場合。

```bash
# 朝にClaudeセッションを開始
claude
> このプロジェクトのアーキテクチャを検討してください

# 昼休みにラップトップを閉じる（スリープ）

# 午後に再開
# 修正後: 数秒でセッションが復帰し、作業を続けられる
> 先ほどの提案を実装してください
```

### 会議中のスリープからの復帰

会議でラップトップを閉じた後、すぐに作業を再開する場合。

```bash
# 会議前にClaudeでコードレビューを進めていた
claude
> このPRをレビューしてください

# （会議中: ラップトップスリープ）
# （会議終了: ラップトップ復帰）

# 修正後: 数秒で再接続され、すぐに作業を再開できる
# 修正前: 最大10分待たないとClaudeが応答しなかった
```

### 複数のデバイスを使用する開発者

デスクトップとラップトップを行き来する場合の接続信頼性向上。

```bash
# 在宅勤務でのラップトップ使用
claude

# オフィスに移動（ラップトップをスリープ、その後復帰）

# 到着後すぐに作業を再開
> 続きを進めてください
# 数秒以内に再接続完了
```

## 注意点

- この改善はv2.1.71以降で自動的に適用されます。
- ブリッジセッションはClaude Codeが使用する内部の接続メカニズムです。
- ネットワーク環境によっては、再接続に少し時間がかかる場合があります。
- 再接続中は一時的にClaude Codeが応答しない場合がありますが、以前より大幅に短い時間で解消されます。
