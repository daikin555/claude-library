## 原文（日本語に翻訳）

貼り付けられた改行をCtrl+J（LF）としてエンコードするターミナルにおいて、複数行のペーストが改行の代わりに `j` の文字を含んだ1行に潰れてしまう問題を修正しました。

## 原文（英語）

Fixed multi-line paste collapsing into one line with `j` in place of newlines in terminals that encode pasted newlines as Ctrl+J

## 概要

一部のターミナルは、ブラケットペースト（bracketed paste）でテキストが貼り付けられた際、行区切りの改行をCtrl+J（制御コードとしてのLF）として送信する実装になっています。これまでのClaude Codeはこの入力を正しく改行として扱えず、Ctrl+Jを文字どおりの「`j`」という1文字として入力欄に挿入してしまっていました。その結果、複数行のテキストを貼り付けると、本来複数行であるべき内容が、改行の代わりに `j` が挿入された状態の1行に潰れてしまい、貼り付けた内容が意味をなさなくなっていました。v2.1.218ではこの誤変換を修正し、こうしたターミナルでの貼り付け時にも改行が正しく改行として保持されるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 影響を受けていたのは、ペースト時の改行をCtrl+J（LF）としてエンコードする特定のターミナル実装を使用している場合です。すべてのターミナルで発生していた問題ではありません。
- 症状としては、複数行のコードやテキストを貼り付けた際に、改行が消えて代わりに `j` の文字が挿入され、1行の壊れたテキストになる形で現れていました。
- この修正はペースト処理に関するものであり、通常のキー入力としての `Ctrl+J` や `j` の入力には影響しません。
