## 原文（日本語訳）

CLI 2.1.203以降で、Cowork VMモードのローカルエージェントセッションが「Not logged in・/login を実行してください」というエラーで起動に失敗する不具合を修正した。

## 原文（英語）

Fixed Cowork VM-mode local-agent sessions failing to start with "Not logged in · Please run /login" on CLI 2.1.203+

## 概要

Cowork（Claude Desktopアプリの「Cowork」タブ）は、Dispatchや長時間の自律的なエージェントタスクを実行するための機能です。「ローカル（VM）モード」で動作するセッションでは、会話や操作を担う「エージェントループ」自体はユーザーのデバイス上でネイティブに動作しつつ、シェルコマンドやコード実行はハイパーバイザーで隔離されたLinux VM内で行われる、という2層構成になっています。今回のリリースでは、CLI 2.1.203以降の環境でこのVMモードのローカルエージェントセッションを開始しようとすると、実際にはログイン済みであるにもかかわらず「Not logged in · Please run /login」というエラーが表示され、セッションがまったく起動できなくなる不具合が修正されました。VM側のエージェントプロセスに認証状態（ログインセッション情報）が正しく引き継がれていなかったことが原因とみられ、2.1.205ではこの引き継ぎ処理が修正されています。

## 基本的な使い方

Cowork自体の基本的な使い方は、Claude Desktopアプリを開き、「Cowork」タブからタスクを開始するだけです。

```text
1. Claude Desktopアプリを起動し、サインインする
2. 「Cowork」タブを開く
3. 実行したいタスクを入力して送信する
```

VMモード（ローカル実行）を使うローカルエージェントセッションでは、エージェントループはユーザーのデバイス上で動作し、コード実行やシェルコマンドはデバイス上に用意された隔離VM内で実行されます。今回の修正により、2.1.203/2.1.204で発生していた起動失敗が解消され、通常どおりログイン状態のままセッションを開始できるようになりました。

## 実践例

### アップデート後にローカルCoworkセッションが正常に起動するか確認する

2.1.203や2.1.204で「Not logged in · Please run /login」エラーに遭遇していた場合は、まずCLIおよびDesktopアプリを2.1.205以降にアップデートしてください。

```text
1. Claude Desktopアプリを最新版に更新する（自動更新、または手動で再起動して更新を確認）
2. Cowork タブでVMモードのローカルエージェントセッションを新規に開始する
3. エラーが表示されず、タスクが正常に開始されることを確認する
```

もし更新後も同様のエラーが出る場合は、念のため一度サインアウトしてから `/login` で再ログインし、再度セッションを開始してみるとよいでしょう。

### 既存のCoworkセッションが影響を受けていた場合の対処

不具合発生中に起動に失敗したセッションが残っている場合、アップデート後に該当セッションを再度開き直すことで正常に再開できることが期待されます。

```text
1. 2.1.205以降にアップデートする
2. 失敗していたローカルエージェントセッションを開き直す
3. 正常に起動しない場合は、いったんセッションを破棄し、新規セッションとして再作成する
```

## 注意点

- この不具合はCLI 2.1.203以降で発生していたものであり、影響を受けるのは「VMモード」で動作するローカルエージェントセッション（エージェントループはデバイス上、コード実行は隔離VM上という構成）です。Anthropicのサーバー上で完結する通常のCoworkセッションについては、公式情報の範囲では明確な言及が見つかりませんでした。
- Cowork自体はPro/Max/Team/Enterpriseなどの有料プランで利用可能な機能です。無料プランでは利用できない場合があります。
- 認証状態の引き継ぎに関する内部実装の詳細（VMへどのようにログインセッションを渡しているかなど）は、本記事作成時点で公開されている公式ドキュメントには記載が見つかりませんでした。上記の説明はchangelogの記述とCoworkのアーキテクチャに関する一般的な公開情報をもとにした推測を含みます。
- アップデート後も同様のエラーが解消しない場合は、Claudeのヘルプセンターやサポートへの問い合わせを検討してください。
