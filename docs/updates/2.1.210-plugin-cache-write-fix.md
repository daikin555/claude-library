## 原文（日本語に翻訳）

プラグインキャッシュへの書き込みが失敗した際に一時ファイルが残ってしまう問題と、Windowsおよびネットワークファイルシステム上でロックされたファイルのリネームが失敗する問題を修正しました。

## 原文（英語）

Fixed plugin cache writes leaving temp files behind on failure and failing on locked-file renames on Windows and network filesystems

## 概要

Claude Codeはマーケットプレイスからインストールしたプラグインを、セキュリティと検証のためにローカルのプラグインキャッシュ（`~/.claude/plugins/cache`）にコピーして利用します。この修正前は、キャッシュへの書き込みが何らかの理由で失敗した場合に不要な一時ファイルが残ってしまったり、Windowsやネットワークドライブ（SMB共有など）上でファイルが他プロセスにロックされている場合にリネーム処理が失敗し、プラグインの更新やインストールがエラーになることがありました。v2.1.210では書き込み処理とリネーム処理の堅牢性が改善され、失敗時に一時ファイルを残さず、ロックされたファイルに対してもリトライ等でリネームが成功しやすくなりました。

## 基本的な使い方

### 修正前の問題

```bash
# プラグインを更新
> /plugin update my-plugin

# ネットワークドライブやアンチウイルスソフトなどにより
# 一時的にファイルがロックされている状況で発生しやすい

# 修正前:
# - キャッシュ書き込みが失敗
# - ~/.claude/plugins/cache/ に *.tmp や不完全なファイルが残存
# - 再実行してもリネームに失敗し続けることがある
ls ~/.claude/plugins/cache/my-plugin/
# my-plugin.tmp123456  ← ゴミファイルが残る
```

### 修正後の動作

```bash
> /plugin update my-plugin

# 修正後:
# - 書き込みが失敗した場合、一時ファイルは自動的にクリーンアップされる
# - Windows/ネットワークファイルシステムでのロックされたリネームも
#   適切にリトライ・処理され、成功しやすくなる
ls ~/.claude/plugins/cache/my-plugin/
# （不完全な一時ファイルは残らない）
```

## 実践例

### Windows環境でのプラグイン運用

WindowsではウイルススキャナやOneDrive同期がファイルを一時的にロックすることがあり、プラグインのキャッシュ更新に失敗しやすい環境でした。

```powershell
# Windows PowerShellでのプラグインインストール
> claude
> /plugin marketplace add my-org/plugins
> /plugin install my-plugin@my-org

# 修正前: ウイルススキャナのロックと競合し、
#         "rename failed: file is locked" のようなエラーが発生することがあった
# 修正後: ロック解除を待ってリトライするため、成功率が向上
```

### ネットワークファイルシステム上での`.claude`ディレクトリ運用

社内共有ドライブやNFS上にホームディレクトリを置いている開発チームでの利用。

```bash
# ホームディレクトリがNFSマウントされている環境
echo $HOME
# /mnt/nfs-home/username

> /plugin update --all

# 修正前: NFS特有のリネーム挙動（他クライアントによる
#         ファイルロック）で失敗することがあった
# 修正後: リネーム失敗時のフォールバック処理が改善され、
#         キャッシュの整合性が保たれやすくなった
```

### キャッシュの手動クリーンアップとの併用

修正後もキャッシュに問題が疑われる場合は、手動でキャッシュをクリアして再インストールできます。

```bash
# プラグインキャッシュを完全にクリア
rm -rf ~/.claude/plugins/cache

# Claude Codeを再起動してプラグインを再取得
claude
> /plugin marketplace update
> /plugin install my-plugin@my-org
```

## 注意点

- この修正はClaude Code v2.1.210で導入されました
- プラグインキャッシュは通常 `~/.claude/plugins/cache` に配置され、バージョンごとに個別ディレクトリとして管理されます
- 今回の修正はキャッシュへの「書き込み」処理に関するものであり、プラグイン自体のコード変更をユーザーに反映させるには、`plugin.json` のバージョン番号を上げる必要がある点は変わりません
- Windows特有の問題として、ウイルス対策ソフトやクラウド同期ツール（OneDriveなど）によるファイルロックは引き続き発生し得るため、問題が続く場合はそれらの一時停止を試すことも有効です
- 問題が解消しない場合は、`rm -rf ~/.claude/plugins/cache`（Windowsでは対応するキャッシュディレクトリの削除）でキャッシュを初期化し、再インストールすることで回避できます
