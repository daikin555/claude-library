## 原文（日本語に翻訳）

`availableModels` の制限がサブエージェントのモデルオーバーライド、エージェントディスパッチのモデルピッカー、アドバイザーモデルに適用されない問題を修正しました

## 原文（英語）

Fixed `availableModels` restrictions not being applied to subagent model overrides, the agent dispatch model picker, and the advisor model

## 概要

設定ファイルの `availableModels` で使用可能なモデルを制限しても、その制限がサブエージェントのモデル指定、`claude agents` のディスパッチ画面のモデルピッカー、アドバイザーモデルには適用されていませんでした。この修正により、`availableModels` 設定がすべての場面で一貫して適用されるようになります。

## 基本的な使い方

```json
// .claude/settings.json
{
  "availableModels": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5-20251001"
  ]
}
```

修正後: 上記設定により、以下のすべてで制限が適用されます：
- `/model` コマンドでのモデル選択
- サブエージェント起動時のモデル指定 (`opts.model`)
- `claude agents` ディスパッチのモデルピッカー
- アドバイザーモデルの選択

## 実践例

### 組織でのモデル利用管理

```json
// 組織ポリシー: コスト管理のためHaikuとSonnetのみ許可
{
  "availableModels": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5-20251001"
  ]
}
```

```javascript
// ワークフロースクリプト内でOpusを指定しようとしても...
const result = await agent("タスク実行", {
  model: "claude-opus-4-8"  // availableModelsに含まれない
})
// 修正後: 制限に基づいてエラーまたは許可モデルへフォールバック
```

## 注意点

- 組織管理者が `availableModels` を設定している場合、ワークフロースクリプト内のモデル指定も制限を受けます
- この修正はセキュリティと組織ポリシーの一貫した適用のために重要です
- 自分のセッションで `availableModels` を設定する場合も同様に全エージェントに適用されます
