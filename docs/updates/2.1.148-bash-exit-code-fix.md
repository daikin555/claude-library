## 原文（日本語に翻訳）

Bashツールが一部のユーザーに対してすべてのコマンドで終了コード127を返していた問題を修正しました（2.1.147で発生したリグレッション）。

## 原文（英語）

> Fixed the Bash tool returning exit code 127 on every command for some users (a regression introduced in 2.1.147)

## 概要

バージョン2.1.147で導入されたリグレッションにより、一部のユーザー環境でBashツールが実行するすべてのコマンドが終了コード127（「command not found」）を返す問題が発生していました。2.1.148でこの問題が修正されました。影響を受けていたユーザーは、アップデートにより通常のコマンド実行が再び正常に動作するようになります。

## 影響を受けていた症状

```bash
# 正常なコマンドのはずが...
$ ls
# 終了コード: 127 (command not found)

$ git status
# 終了コード: 127 (command not found)

$ npm install
# 終了コード: 127 (command not found)
```

コマンド自体は存在するにもかかわらず、すべてのBashコマンドが「コマンドが見つからない」エラーを返す状態でした。

## 対処方法

2.1.147を使用していて影響を受けている場合は、2.1.148以降へアップデートしてください。

```bash
# Claude Codeをアップデート
claude update

# バージョン確認
claude --version
```

## 注意点

- このリグレッションは2.1.147で発生し、2.1.148で修正されました
- 影響は一部のユーザー環境に限定されており、すべてのユーザーが影響を受けていたわけではありません
- 2.1.147を使用していてBashコマンドが正常に動作していた場合は、このリグレッションの影響を受けていなかった可能性があります
