## 原文（日本語訳）

リクエストが進行中の状態でサーバーがHTTP/2接続をGOAWAYでクローズした際に、supervisedセッションおよびバックグラウンドセッションがクラッシュする不具合を修正した。

## 原文（英語）

Fixed supervised and background sessions crashing when a server closed an HTTP/2 connection with a GOAWAY while requests were in flight

## 概要

Claude CodeはAPIサーバーとの通信にHTTP/2を利用していますが、サーバー側の都合（負荷分散やコネクションの入れ替えなど）で接続を閉じる際に送られる「GOAWAYフレーム」を、リクエストが飛行中（in flight）のタイミングで受信すると、supervisedモードやバックグラウンドで動作しているセッションがクラッシュしてしまう不具合がありました。今回のアップデートでこの接続クローズ処理が修正され、GOAWAYを受け取っても安全に扱われるようになっています。

## 基本的な使い方

ユーザー側で特別な対応は不要です。通常通りバックグラウンドセッションやsupervisedセッションを利用しているだけで、この修正の恩恵を受けられます。

```text
claude agents
# バックグラウンドで長時間動作させていても、
# サーバー側のコネクションクローズでクラッシュしなくなった
```

## 実践例

### 長時間実行されるバックグラウンドタスク

長時間にわたってAPIリクエストをやり取りするバックグラウンドセッションでは、サーバー側のコネクションのローテーションによってGOAWAYが送られてくる可能性が相対的に高くなります。今回の修正により、こうした接続の入れ替わりが発生してもセッションが落ちずに継続できます。

```text
claude agents
# 数時間単位で動き続けるバックグラウンドエージェントでも安定して稼働
```

### エンタープライズプロキシ経由での利用

社内プロキシやロードバランサーを経由してAPIにアクセスしている環境では、途中の中継機器がコネクションを再構成する際にGOAWAYが発生しやすくなります。そうした構成下でのsupervisedセッションの安定性が向上しています。

```text
export HTTPS_PROXY="https://proxy.example.com:8443"
claude
# プロキシ経由の接続でもGOAWAY受信時にクラッシュしない
```

## 注意点

- この修正はネットワーク層の内部的な安定性向上であり、ユーザーが設定を変更する必要はありません。
- HTTP/2のGOAWAYはサーバー側が正常な運用の一環として送信するものであり、エラーではありません。以前はこれを適切にハンドリングできていなかった点が問題でした。
