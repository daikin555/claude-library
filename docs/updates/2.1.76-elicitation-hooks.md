# ElicitationおよびElicitationResultフック：エリシテーション応答のインターセプトとオーバーライド

## 原文（日本語に翻訳）

エリシテーション応答がMCPサーバーに返される前にインターセプトしてオーバーライドするための新しい `Elicitation` および `ElicitationResult` フックを追加

## 原文（英語）

Added new `Elicitation` and `ElicitationResult` hooks to intercept and override responses before they're sent back

## 概要

Claude Code v2.1.76で、MCPエリシテーションに対応する2つの新しいフックイベントが追加されました。`Elicitation` フックはMCPサーバーが入力を要求した際にダイアログ表示前にインターセプトでき、`ElicitationResult` フックはユーザーが応答した後にMCPサーバーへ返す前にその内容を変更またはブロックできます。これにより、エリシテーションの自動応答やセキュリティポリシーの適用が可能になります。

## 基本的な使い方

`~/.claude/settings.json` またはプロジェクトの `.claude/settings.json` にフック設定を追加します：

```json
{
  "hooks": {
    "Elicitation": [
      {
        "matcher": "my-mcp-server",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/elicitation-handler.sh"
          }
        ]
      }
    ],
    "ElicitationResult": [
      {
        "matcher": "my-mcp-server",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/result-handler.sh"
          }
        ]
      }
    ]
  }
}
```

`matcher` にはMCPサーバー名を指定します。空文字列を指定するとすべてのMCPサーバーに対して発火します。

## 実践例

### Elicitationフックで自動応答する

MCPサーバーからの入力要求に対して、ダイアログを表示せずにプログラム的に応答するスクリプト例：

```bash
#!/bin/bash
# auto-respond-elicitation.sh
# 標準入力からフックの入力JSONを読み取る
INPUT=$(cat)

SERVER_NAME=$(echo "$INPUT" | jq -r '.mcp_server_name')
MODE=$(echo "$INPUT" | jq -r '.mode')

# 特定のサーバーに対して自動的に認証情報を返す
if [ "$SERVER_NAME" = "database-server" ] && [ "$MODE" = "form" ]; then
  cat <<'EOF'
{
  "hookSpecificOutput": {
    "hookEventName": "Elicitation",
    "action": "accept",
    "content": {
      "username": "deploy-user",
      "database": "production"
    }
  }
}
EOF
  exit 0
fi

# それ以外はデフォルトのダイアログ表示に任せる
exit 0
```

### ElicitationResultフックでセキュリティポリシーを適用する

ユーザーの応答を検証し、ポリシーに違反する場合はブロックするスクリプト例：

```bash
#!/bin/bash
# validate-elicitation-result.sh
INPUT=$(cat)

ACTION=$(echo "$INPUT" | jq -r '.action')
CONTENT=$(echo "$INPUT" | jq -r '.content')
SERVER_NAME=$(echo "$INPUT" | jq -r '.mcp_server_name')

# 信頼されていないMCPサーバーへの応答をブロック
TRUSTED_SERVERS=("internal-api" "deploy-server" "database-server")
IS_TRUSTED=false
for server in "${TRUSTED_SERVERS[@]}"; do
  if [ "$SERVER_NAME" = "$server" ]; then
    IS_TRUSTED=true
    break
  fi
done

if [ "$IS_TRUSTED" = false ]; then
  echo "信頼されていないMCPサーバー '$SERVER_NAME' へのエリシテーション応答をブロックしました" >&2
  exit 2  # exit code 2でブロック（actionがdeclineに変更される）
fi

exit 0
```

### ElicitationResultフックで応答内容を書き換える

ユーザーの入力を修正してからMCPサーバーに返すスクリプト例：

```bash
#!/bin/bash
# modify-elicitation-result.sh
INPUT=$(cat)

ACTION=$(echo "$INPUT" | jq -r '.action')

# acceptの場合のみ内容を変更
if [ "$ACTION" = "accept" ]; then
  # 環境名を正規化する（例：prod → production）
  ENVIRONMENT=$(echo "$INPUT" | jq -r '.content.environment // empty')
  if [ "$ENVIRONMENT" = "prod" ]; then
    cat <<'EOF'
{
  "hookSpecificOutput": {
    "hookEventName": "ElicitationResult",
    "action": "accept",
    "content": {
      "environment": "production"
    }
  }
}
EOF
    exit 0
  fi
fi

exit 0
```

## 注意点

- **matcherはMCPサーバー名でマッチ**: `matcher` フィールドにはMCPサーバー名を指定する。空文字列ですべてのサーバーにマッチ
- **Elicitationフックの出力**: `hookSpecificOutput` で `action`（`accept` / `decline` / `cancel`）と `content`（フォーム値）を指定して自動応答できる
- **ElicitationResultフックのexit code 2**: 終了コード2で応答をブロックし、actionが自動的に `decline` に変更される。stderrの内容がユーザーに表示される
- **フック種別**: `type: "command"` のフックのみがサポートされている
- **入力フィールド**: 両フックとも `mcp_server_name`、`action`、`mode`、`elicitation_id`、`content` などのフィールドを受け取る
