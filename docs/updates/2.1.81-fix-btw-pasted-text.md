# 修正：アクティブな応答中に/btwを使うとペーストしたテキストが含まれない問題

## 原文（日本語に翻訳）

アクティブな応答中に使用した場合に `/btw` がペーストしたテキストを含まない問題を修正。

## 原文（英語）

Fixed `/btw` not including pasted text when used during an active response

## 概要

Claude が応答を生成している最中に `/btw` コマンドでメモや補足情報を入力し、テキストをペーストした場合、そのペーストしたテキストが `/btw` メッセージに含まれない問題が修正されました。これにより、応答中にペーストした内容も正しくコンテキストとして伝えられるようになります。

## 基本的な使い方

`/btw` は Claude が応答中に追加のコンテキストを非同期で伝えるためのコマンドです：

```
# Claude が応答を生成している間に
/btw ここに追加情報をペーストして伝えられる（修正後は正しく機能）

# 例：エラーログをペーストする
/btw [Ctrl+V でエラーログをペースト]
```

## 実践例

### ### 応答中に追加のエラーログを提供

```
# Claude が解析中に新しいエラーが発生した場合
/btw 今こんなエラーも出てきました：
TypeError: Cannot read property 'map' of undefined
  at Component.render (App.js:42)
```

### ### 応答中に関連ファイルの内容を共有

```
# Claude が回答を生成中に参考になりそうなファイルをペースト
/btw 参考までに設定ファイルの内容です：[ペースト]
```

## 注意点

- `/btw` は Claude の応答を中断しない。次の応答ターンで考慮される
- 長い内容をペーストする際は、ファイルを直接読ませる方が確実な場合もある
- アクティブな応答がない場合でも `/btw` は通常どおり機能する
