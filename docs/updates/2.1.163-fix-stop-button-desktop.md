## 原文（日本語に翻訳）

デスクトップアプリで、基盤となるプロセスがすでに終了していた場合に、バックグラウンドタスクチップの Stop ボタンをクリックしてもチップがクリアされない問題を修正。

## 原文（英語）

Fixed clicking Stop on a background-task chip in the desktop app not clearing the chip when the underlying process was already gone

## 概要

Claude Code デスクトップアプリのバックグラウンドタスクチップに表示される Stop ボタンをクリックした際、対象のプロセスがすでに終了していた場合にチップが残り続けるバグが修正されました。Stop ボタンをクリックすると、プロセスの状態に関わらずチップが正しくクリアされるようになります。

## 基本的な使い方

修正後は特別な操作は不要です。デスクトップアプリのバックグラウンドタスクチップで Stop ボタンをクリックすると、チップが適切に消えるようになります。

## 注意点

- この問題はデスクトップアプリの UI のみに関するバグです。CLI (`claude`) の動作には影響していませんでした。
- プロセスがすでに終了していてもチップが残る UI の不整合が解消されました。
