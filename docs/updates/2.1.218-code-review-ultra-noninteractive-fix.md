## 原文（日本語に翻訳）

非対話型セッションで`/code-review ultra`が黙ってローカルレビューを実行してしまう問題を修正しました。現在は正しくクラウドレビューを起動します

## 原文（英語）

Fixed `/code-review ultra` silently running a local review in non-interactive sessions — it now launches the cloud review

## 概要

`/code-review`にはローカルで完結する通常レビューと、`ultra`オプションを付けてクラウド上でより深いレビューを実行する2種類の動作があります。以前は、CIパイプラインやヘッドレス実行など非対話型セッションで`claude`を使って`/code-review ultra`を実行すると、何のエラーも表示されないままローカルレビューにフォールバックしてしまい、意図したクラウドレビューが実行されていないことに気づきにくい状態でした。修正後は、非対話型セッションでも`ultra`指定時には正しくクラウドレビューが起動するようになりました。

## 基本的な使い方

```bash
# 対話型セッションでのクラウドレビュー起動（従来どおり）
/code-review ultra
```

## 実践例

### CI/ヘッドレス環境で明示的にクラウドレビューを実行する

```bash
# 非対話型（ヘッドレス）モードでclaudeを起動し、
# code-review ultraを実行するケース
claude -p "/code-review ultra" --output-format json

# 修正前: 黙ってローカルレビューが実行され、
#         クラウドレビューが実行されていないことに気づきにくい
# 修正後: 意図どおりクラウドレビューが起動する
```

### 通常のローカルレビューと使い分ける

```bash
# ローカルのみで完結する軽量なレビュー
/code-review

# クラウド上のより深いレビュー
/code-review ultra
```

## 注意点

- Claude Code v2.1.218で修正
- 影響を受けるのは主にCIやスクリプトからの非対話型（headless）実行
- 対話型セッションでは以前から正しくクラウドレビューが起動していた
- クラウドレビューにはネットワーク接続や関連する権限・認証設定が必要になる場合がある
