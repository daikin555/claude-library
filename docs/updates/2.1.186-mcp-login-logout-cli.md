# CLIからMCPサーバーに認証：claude mcp login/logout コマンド

## 原文（日本語に翻訳）
`claude mcp login <name>` と `claude mcp logout <name>` を追加し、インタラクティブな `/mcp` メニューを開かずにCLIからMCPサーバーの認証ができるようになりました。SSH経由で完了するための `--no-browser` stdin リダイレクトもサポートしています。

## 原文（英語）
Added `claude mcp login <name>` and `claude mcp logout <name>` to authenticate MCP servers from the CLI without opening the interactive `/mcp` menu, with `--no-browser` stdin redirect support for completing over SSH

## 概要
MCPサーバーへの認証が、インタラクティブな `/mcp` メニューを使わずに直接CLIコマンドで実行できるようになりました。特にSSH経由でのリモート操作時に有用で、`--no-browser` フラグを使用することでブラウザが起動できない環境でも認証を完了できます。スクリプトや自動化ワークフローへの組み込みも容易になりました。

## 基本的な使い方

### MCPサーバーにログイン
```bash
claude mcp login <サーバー名>
```

### MCPサーバーからログアウト
```bash
claude mcp logout <サーバー名>
```

### SSH環境でのブラウザなし認証
```bash
claude mcp login <サーバー名> --no-browser
```

## 実践例

### 通常のMCPサーバー認証
```bash
# 設定済みのMCPサーバー「github」に認証
claude mcp login github

# ブラウザが自動で開き、OAuthフローで認証完了
# 次回以降の接続で認証済み状態が維持される
```

### SSHリモートセッションでの認証
```bash
# SSH経由でリモートサーバーに接続中の場合
# --no-browser フラグでstdin経由の認証に切り替え
claude mcp login github --no-browser

# 端末に認証URLが表示される
# ローカルPCのブラウザでそのURLを開いて認証
# 表示される認証コードを端末に貼り付けて完了
```

### 認証状態の確認と再認証
```bash
# 設定済みMCPサーバーの一覧を確認
claude mcp list

# 既存の認証情報をクリアして再認証
claude mcp logout github
claude mcp login github
```

### CI/CDパイプラインでの活用
```bash
# 環境変数から認証情報を利用する場合
# --no-browser でブラウザなし認証フローを使用
claude mcp login my-server --no-browser
```

## 注意点
- 認証情報はローカルに安全に保存されます
- `--no-browser` フラグはSSH環境やヘッドレス環境（GUIのないサーバー）での使用に適しています
- 認証が必要なMCPサーバー（OAuthやトークン認証を使用するもの）のみに適用されます
- 従来の `/mcp` メニューからの認証も引き続き使用できます
- ログアウトしても、MCPサーバー自体の設定（`claude mcp list` で表示される設定）は削除されません
