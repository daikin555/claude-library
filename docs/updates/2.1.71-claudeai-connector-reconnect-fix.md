## 原文（日本語に翻訳）

OAuthトークンの更新後にclaude.aiコネクターが再接続できない問題を修正しました。

## 原文（英語）

Fixed claude.ai connectors failing to reconnect after OAuth token refresh

## 概要

claude.aiコネクターがOAuthトークンのリフレッシュ後に接続を再確立できず、利用不能な状態になってしまうバグが修正されました。トークンが更新された後、コネクターが新しいトークンを使用して再接続するための処理が正しく動作していませんでした。修正後はトークンの更新が完了した後、コネクターが自動的に新しいトークンで再接続するようになります。

## 基本的な使い方

この修正はv2.1.71以降で自動的に適用されます。OAuthトークンが更新された後も、claude.aiコネクターが自動的に再接続されます。

```bash
# claude.aiコネクターを使用したセッション
claude

# OAuthトークンが期限切れになっても、自動的に更新・再接続される
> MCPツールを使用してタスクを実行してください

# 以前: トークン更新後にコネクターが切断され、エラーが発生することがあった
# 修正後: シームレスに再接続される
```

## 実践例

### 長時間セッション中のトークン自動更新

OAuthトークンの有効期限が切れるような長時間のセッションでの利用。

```bash
# 長時間のセッションを開始
claude

# 数時間の作業中にトークンが期限切れになる
> 引き続きclaude.aiのMCPツールを使用してください

# 以前: トークン更新後にコネクターエラーが発生し、手動での再起動が必要だった
# 修正後: 自動的にトークンが更新され、シームレスに継続できる
```

### 複数の組織コネクターの使用

複数のclaude.ai組織コネクターが設定されている環境での利用。

```bash
# 複数のコネクターが設定された環境
claude

# いずれかのコネクターのトークンが更新された場合も
# 全てのコネクターが正常に再接続される
```

### 自動化タスクでのコネクター使用

非インタラクティブモードでclaude.aiコネクターを使用する場合。

```bash
# バッチ処理での使用
claude --print "MCPツールを使用してデータを処理してください"

# トークン更新が必要な場合も自動的に処理される
```

## 注意点

- この修正はclaude.aiコネクターを使用している環境に特化した修正です。
- トークンの更新には短時間の処理が必要で、その間コネクターが一時的に利用不能になる場合があります。
- 関連する修正として、プロキシコネクターのOAuthフリーズ修正（`2.1.71-proxy-connector-oauth-freeze-fix.md`）もv2.1.71で対応されています。
- セキュリティのため、トークンの有効期限は定期的に確認することを推奨します。
