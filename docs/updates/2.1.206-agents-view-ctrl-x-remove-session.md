## 原文（日本語訳）

agents viewの変更点: Ctrl+Xが完了済みセッションを永続的に削除するようになり、セッションが二重に表示されることもなくなった。削除されたバックグラウンドジョブは削除されたままになる。

## 原文（英語）

Changed agents view: Ctrl+X now permanently removes a completed session, and sessions no longer render twice; deleted background jobs stay deleted

## 概要

`agents` view上で完了済みセッションに対して `Ctrl+X` を押すと、以前は一覧上から一時的に消えたように見えても内部的には残り続けたり、同じセッションが二重に表示されたりすることがありました。今回の変更で `Ctrl+X` は完了済みセッションを名実ともに永続的に削除する操作となり、セッションの二重描画バグと、削除したはずのバックグラウンドジョブが再度一覧に現れてしまう不具合が解消されました。

## 基本的な使い方

agents viewを開き、削除したい完了済みセッションにカーソルを合わせて `Ctrl+X` を押します。

```text
/agents
```

```text
（一覧内で完了セッションを選択した状態で）
Ctrl+X
```

これにより、そのセッションは一覧から永続的に削除されます。

## 実践例

### 完了した調査タスクを一覧から整理する

複数のバックグラウンドタスクを実行し終えた後、agents viewを開いて不要になった完了セッションを一つずつ `Ctrl+X` で削除し、一覧を見やすく整理します。

```text
/agents
```

以前のバージョンでは削除してもセッションが再度表示されたり二重に見えたりすることがありましたが、今回の修正により削除した項目は確実に一覧から消えたままになります。

### グループヘッダーでの一括削除

同じグループ（例: 同一ブランチやタスクに紐づくセッション群）のヘッダー上で `Ctrl+X` を押すと、確認の上でそのグループに属するセッションをまとめて削除できます。

```text
/agents
```

大量のバックグラウンドジョブを溜め込んでしまった場合の整理に有効です。

## 注意点

- `Ctrl+X` による削除は完了済みセッションが対象であり、実行中のセッションに対する挙動（停止など）とは区別されます。
- 削除は永続的な操作であるため、誤って必要なセッションのログを削除しないよう注意してください。
- 本修正により解消されたのは表示上の二重描画バグと削除後の再出現バグであり、セッション自体の実行結果には影響しません。
