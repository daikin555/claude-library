## 原文（日本語に翻訳）

`/plugin` アンインストールダイアログを修正：`n` キーを押すとプラグインのデータディレクトリを保持しながら正しくアンインストールするようになりました。

## 原文（英語）

Fixed `/plugin` uninstall dialog: pressing `n` now correctly uninstalls the plugin while preserving its data directory

## 概要

`/plugin uninstall` を実行した際に表示される確認ダイアログで、「データディレクトリも削除しますか？」という質問に対して `n`（いいえ）と答えた場合、プラグインのデータを残したままアンインストールすることが意図された動作ですが、実際にはアンインストールが行われないバグが修正されました。修正後は `n` を押すと、データディレクトリを保持しながらプラグインが正しくアンインストールされます。

## 基本的な使い方

プラグインをアンインストールする際の正しい動作：

```
/plugin uninstall my-plugin

プラグイン "my-plugin" をアンインストールしますか？ [Y/n]: Y
データディレクトリも削除しますか？ [y/N]: n
← 「n」を押すとデータを保持してアンインストール（修正後の動作）
```

## 実践例

### ユースケース: プラグインの設定を保持してアンインストールする

プラグインの設定データ（認証情報、カスタム設定など）を残しつつアンインストールする場合：

```
/plugin uninstall my-data-plugin
データディレクトリも削除しますか？ [y/N]: n

# プラグインはアンインストールされたが、データは保持される
# 後で再インストールすると設定が引き継がれる
```

### ユースケース: プラグインの再インストール準備

プラグインを一時的に無効化して再インストールしたい場合に、データを保持したままアンインストールできます：

```
/plugin uninstall plugin-to-update
データも削除しますか？ [y/N]: n

# アップデート版をインストール
/plugin install plugin-to-update
# 以前のデータが引き継がれる
```

## 注意点

- 修正前は `n` を押してもアンインストールが行われず、プラグインが残ったままでした
- `y` を押すとデータディレクトリも含めて完全に削除されます（この動作は変わりません）
- プラグインのデータディレクトリは通常 `~/.claude/plugins/plugin-name/data/` などに保存されています
