## 原文（日本語に翻訳）

スケジュールタスク（`/loop`、`CronCreate`）が実行された際に、トランスクリプトにタイムスタンプマーカーが追加されるようになりました。

## 原文（英語）

Added timestamp markers in transcripts when scheduled tasks (`/loop`, `CronCreate`) fire

## 概要

`/loop` コマンドや `CronCreate` ツールを使ったスケジュールタスクが実行されると、トランスクリプトにタイムスタンプマーカーが記録されるようになりました。これにより、長時間実行するタスクのログを見返す際に、各実行がいつ行われたかを正確に把握できます。定期実行のデバッグや監査に役立ちます。

## 基本的な使い方

`/loop` コマンドで定期タスクを設定すると、トランスクリプトに自動的にタイムスタンプが記録されます：

```
/loop 30m 現在のシステム状態を確認してレポートしてください
```

実行されるたびに、トランスクリプト内に以下のようなマーカーが挿入されます：

```
[2026-03-27T10:30:00Z] ── Scheduled task fired ──
```

## 実践例

### ユースケース: 定期的なシステム監視タスク

```
/loop 1h サーバーのディスク使用量、メモリ使用量、エラーログを確認してサマリーを作成してください
```

タイムスタンプマーカーにより、各実行時刻が明確になります：

```
[2026-03-27T10:00:00Z] ── Scheduled task fired ──
（タスク実行結果）
[2026-03-27T11:00:00Z] ── Scheduled task fired ──
（タスク実行結果）
```

### ユースケース: CronCreateを使った定期実行

Claudeに以下のように依頼することでcronタスクを作成できます：

```
毎朝9時にプロジェクトのGitHubのissue一覧を確認して優先順位をまとめてください
```

Claudeが `CronCreate` ツールを使用してタスクを登録し、実行のたびにタイムスタンプが記録されます。

## 注意点

- タイムスタンプはUTC形式で記録されます
- タイムスタンプマーカーはトランスクリプトファイルに保存されるため、後から参照可能です
- `/loop` の実行間隔は `5m`、`30m`、`1h` などの形式で指定できます
