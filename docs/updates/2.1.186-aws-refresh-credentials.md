# /login に「Claude Platform on AWS - 認証情報を更新」オプションを追加

## 原文（日本語に翻訳）
`awsAuthRefresh` が設定されている場合、`/login` に「Claude Platform on AWS - 認証情報を更新」オプションを追加しました。

## 原文（英語）
Added "Claude Platform on AWS - refresh credentials" option to `/login` when `awsAuthRefresh` is configured

## 概要
AWS Bedrock経由でClaude Codeを使用している環境で、`awsAuthRefresh` 設定が有効な場合に `/login` コマンドから直接AWS認証情報を更新できるようになりました。AWSの一時認証情報（STS）が期限切れになった場合などに、セッションを中断せずに再認証できます。

## 基本的な使い方

### awsAuthRefreshの設定
```json
// ~/.claude/settings.json
{
  "awsAuthRefresh": true
}
```

### /login からの認証情報更新
```
/login
# → 「Claude Platform on AWS - 認証情報を更新」オプションが表示される
# → 選択するとAWS認証情報の更新フローが開始される
```

## 実践例

### AWS Bedrock環境での認証情報管理
```json
// ~/.claude/settings.json
{
  "awsAuthRefresh": true,
  "awsRegion": "us-east-1"
}
```

設定後、認証情報が期限切れになった場合：
```
/login
# → 「Claude Platform on AWS - refresh credentials」を選択
# → 認証情報が更新される
# → セッションを再起動せずに続行可能
```

### 定期的な認証情報更新（STS一時認証情報）
AWS STSの一時認証情報を使用している場合、有効期限が切れる前に：
```
/login
# → AWS認証情報更新オプションを選択
# → 新しい一時認証情報を取得
```

## 注意点
- このオプションは `awsAuthRefresh` 設定が有効な場合にのみ表示されます
- AWS BedrockまたはAWS上のClaude Platformを使用している場合に適用されます
- 通常のAnthropic API経由のユーザーには表示されません
- 認証情報の更新にはAWS CLIまたは適切なIAM権限が必要な場合があります
