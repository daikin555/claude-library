## 原文（日本語に翻訳）

[VSCode] ターミナルを認識した加速機能を使用して統合ターミナルのスクロールホイールの応答性を改善

## 原文（英語）

[VSCode] Improved scroll wheel responsiveness in the integrated terminal with terminal-aware acceleration

## 概要

VS Codeの統合ターミナルでClaude Codeを使用する際のスクロールホイールの応答性が改善されました。「ターミナルを認識した加速（terminal-aware acceleration）」が導入され、スクロール量が内容に応じて適切に調整されます。長い会話履歴や大量の出力をスクロールする際の操作性が向上し、よりスムーズな閲覧体験が得られます。

## 基本的な使い方

この改善はユーザー操作不要で自動的に適用されます。VS CodeのClaude Code拡張機能を最新版に更新するだけです。

```
# VS Code 内でスクロール
# マウスホイール / トラックパッドを使ってスムーズにスクロール
```

## 実践例

### スクロール操作の改善点

修正後のスクロール動作の特徴：

```
# 短い距離のスクロール（細かい調整）
ホイール1ノッチ → 数行スクロール（精度が向上）

# 速いスクロール（大量コンテンツの閲覧）
ホイールを素早く回す → 加速してスムーズにスクロール
```

### VS Code でのスクロール設定

スクロール動作をカスタマイズ：

```json
// VS Code settings.json
{
  // ターミナルのスクロール速度調整
  "terminal.integrated.mouseWheelScrollSensitivity": 1.5,

  // スムーズスクロールを有効化
  "terminal.integrated.smoothScrolling": true,

  // スクロール行数の設定
  "terminal.integrated.scrollback": 10000
}
```

### 長い出力の効率的な閲覧

```
# Claude Code で大量の出力を生成するタスクを実行
"すべてのテストを実行してください"

# 長い出力をスムーズにスクロールして確認
# 改善後はレスポンスが向上し、スクロール操作が快適に
```

### キーボードショートカットとの組み合わせ

```
# スクロール操作
Ctrl+↑ / Ctrl+↓     → 1行スクロール
Shift+PageUp/PageDown → ページスクロール
マウスホイール         → ターミナル認識加速スクロール（今回改善）

# 特定の位置にジャンプ
Ctrl+Home  → 先頭へ
Ctrl+End   → 末尾へ
```

### トラックパッドでの使用

MacBook等のトラックパッドでも改善を享受できます：

```
# 2本指でスワイプ → スムーズなスクロール
# 3本指でスワイプ → VS Codeのデフォルト動作

# より快適なトラックパッド設定
"terminal.integrated.mouseWheelScrollSensitivity": 2.0
```

## 注意点

- この改善はVS Code統合ターミナルのみに適用されます（外部ターミナルには影響なし）
- スクロール感度はVS Codeの`terminal.integrated.mouseWheelScrollSensitivity`設定で調整できます
- macOSのシステム設定でスクロール速度を変更している場合は、VS Codeの設定と組み合わせて調整してください
- 高DPIディスプレイ（Retinaなど）でもスクロールの精度が向上しています
