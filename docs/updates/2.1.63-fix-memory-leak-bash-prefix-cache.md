# Bashコマンドプレフィックスキャッシュのメモリリーク修正

## 原文（日本語に翻訳）

Bashコマンドプレフィックスキャッシュのメモリリークを修正しました。

## 原文（英語）

Fixed memory leak in bash command prefix cache

## 概要

Claude Code v2.1.63で、Bashコマンドのプレフィックス解析に使用するキャッシュのメモリリークが修正されました。このキャッシュはコマンドの種類（読み取り専用か書き込みを伴うかなど）を識別するために使われており、長時間のセッションでメモリが蓄積する問題がありました。

## 基本的な使い方

この修正はバックグラウンドで自動的に適用されます。ユーザー側での操作は不要です。

## 実践例

### 影響を受けるシナリオ

- 多数のBashコマンドを実行する長時間セッション
- CI/CD環境でのClaude Code自動化スクリプト
- 繰り返し多くのコマンドを実行する開発ワークフロー

### 修正後の改善

- 長時間セッションでのメモリ使用量が安定
- 特に大量のBashコマンドを実行する場合のパフォーマンスが向上

## 注意点

- v2.1.63にアップデートすることで自動的に修正が適用されます
- Bashツールを頻繁に使用する長時間セッションで特に効果があります
