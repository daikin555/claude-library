## 原文（日本語に翻訳）

左矢印キーによって会話が取り消しできないまま破棄されてしまう問題を修正しました。編集直後に左矢印キーを押すと、現在は確認を求めるようになり、また、エージェントビューでEscキーを押すと、バックグラウンドに送っていた元の会話に戻るようになりました。

## 原文（英語）

Fixed the left arrow key discarding the conversation with no undo: presses right after editing now ask to confirm, and Esc in the agent view returns to the conversation it backgrounded

## 概要

以前のClaude Codeでは、入力欄でテキストを編集した直後に左矢印キーを押すと、意図せず現在の会話がバックグラウンドに送られたり破棄されたりすることがあり、しかもこれを元に戻す手段（アンドゥ）がありませんでした。誤操作で作業中の会話を失ってしまうリスクがある、危険な挙動でした。v2.1.218では、編集直後の左矢印キー押下時には確認ダイアログを表示するように変更され、誤って会話を失う事故を防げるようになりました。あわせて、エージェントビュー（バックグラウンドで実行中のエージェントを表示する画面）でEscキーを押した際に、それがバックグラウンドに送った元の会話へ正しく戻るようになりました。

## 基本的な使い方

入力欄でテキストを編集した直後に左矢印キーを押すと、確認が表示されます。

```
1. 入力欄でメッセージを編集する
2. 編集直後に左矢印キーを押す
3. 「会話をバックグラウンドに送りますか？」といった確認が表示される
4. 確認して初めて会話がバックグラウンドに移動する（誤操作では破棄されない）
```

エージェントビューでは、Escキーで元の会話に戻れます。

```
1. 会話をバックグラウンドに送り、エージェントビューでその進行状況を確認する
2. Escキーを押す
3. その会話がバックグラウンドに送られた元の会話へ戻る（会話は失われない）
```

## 実践例

### 誤って左矢印キーを押してしまった場合

長い入力を編集していて、うっかりカーソル移動のつもりで左矢印キーを押してしまうケースです。

```
# 修正前:
# メッセージ編集直後に左矢印キーを押す
# → 確認なしに会話がバックグラウンド送り/破棄される
# → 元に戻せない

# 修正後:
# 同じ操作をしても、確認ダイアログが表示される
# → 「はい」を選ばない限り会話は維持される
```

### エージェントビューから元の会話に戻る

複数の会話をバックグラウンドで並行して進めている場合、エージェントビューで個々の進行状況を確認できます。

```
# エージェントビューである会話の詳細を開いている状態で
Esc を押す

# 修正前: どこに戻るか分かりにくい、あるいは元の会話に正しく戻らないことがあった
# 修正後: バックグラウンドに送った元の会話へ確実に戻る
```

## 注意点

- 確認ダイアログが出るのは「編集した直後」に左矢印キーを押した場合が対象です。通常のカーソル移動としての左矢印キー操作全般に確認が入るわけではありません。
- この修正はClaude Code v2.1.218以降で有効です。
- バックグラウンドへの会話送りや復元の全体的な仕組み自体は変わっておらず、あくまで誤操作防止と復帰導線の改善です。
