## 原文（日本語訳）

`/doctor` がClaudeの応答中でも、現在のターンの終了を待たずに開けるようになりました。

## 原文（英語）

`/doctor` can now be opened while Claude is responding, without waiting for the current turn to finish

## 概要

`/doctor` は環境設定や接続状態を診断するコマンドですが、従来はClaudeが応答中（ツール実行中・テキスト生成中）には使用できませんでした。v2.1.116では、応答の途中でも `/doctor` を開けるようになりました。長時間のタスク実行中に接続問題が疑われる場合や、エラーの兆候を察知したときに、タスクを中断することなく即座に診断情報を確認できます。

## 基本的な使い方

```bash
# Claudeが応答中でも /doctor を実行可能
# 別のメッセージとして送信するだけでOK
/doctor
```

## 実践例

### 長時間タスク中の接続確認

大規模なコードベースの分析や長時間エージェントタスクの実行中に、接続状態を確認できます。

```bash
# タスクを開始
claude "このリポジトリ全体のセキュリティ脆弱性を調査してください"

# 処理が長引いて接続を確認したいとき、応答を待たずに実行
/doctor
# → API接続状態、設定、環境情報が表示される
```

### エラー発生時の即時診断

ツール実行中にエラーが発生したとき、すぐに /doctor で環境を確認できます。

```bash
# 何かおかしいと感じたらすぐに診断
/doctor
# → Claudeは作業を継続しつつ、診断情報が表示される
```

## 注意点

- `/doctor` を開いてもClaude Codeの処理は中断されません
- 診断情報は現在の処理状態のスナップショットを示します
- MCP接続状態、API設定、認証情報などが確認できます
- 問題が検出された場合は、診断結果に基づいて適切な対処を行ってください
