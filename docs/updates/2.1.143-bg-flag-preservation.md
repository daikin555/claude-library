## 原文（日本語に翻訳）

- `/bg` が `--mcp-config`、`--settings`、`--add-dir`、`--plugin-dir`、`--strict-mcp-config` を維持するようになりました。バックグラウンドセッションが再起動後もMCPサーバーと設定を維持します
- `/bg` と `←`（デタッチ）が `--fallback-model` を維持するようになりました。バックグラウンドワーカーが過負荷時にフォールバックモデルにデグレードし、ハードフェイルしなくなります
- `/bg` と `←`（デタッチ）が `--allow-dangerously-skip-permissions` を維持するようになりました。フォークされたワーカーがパーミッションサイクル内でバイパスを利用できます
- `claude --bg --dangerously-skip-permissions` がリタイア→復帰をまたいで維持されるようになりました

## 原文（英語）

- `/bg` now preserves `--mcp-config`, `--settings`, `--add-dir`, `--plugin-dir`, and `--strict-mcp-config`, so backgrounded sessions keep their MCP servers and settings across respawn
- `/bg` and `←`-detach now preserve `--fallback-model`, so backgrounded workers degrade to the fallback model on overload instead of hard-failing
- `/bg` and `←`-detach now preserve `--allow-dangerously-skip-permissions`, so the forked worker keeps bypass-permissions available in its Shift+Tab cycle
- `claude --bg --dangerously-skip-permissions` now persists across retire→wake

## 概要

`/bg` コマンドおよび `←` キーによるデタッチ操作が、元のセッションで設定した各種フラグを正しく引き継ぐようになりました。MCP設定・カスタム設定ファイル・フォールバックモデル・パーミッションバイパス設定が、バックグラウンドセッションの再起動（respawn）やリタイア後の復帰（retire→wake）を経ても維持されます。

## 基本的な使い方

```bash
# MCP設定とカスタム設定を指定してClaude Codeを起動
claude --mcp-config ./mcp-servers.json --settings ./team-settings.json

# 会話中に /bg でバックグラウンドに移行
/bg

# → バックグラウンドセッションが再起動しても
#   --mcp-config と --settings が維持される
```

## 実践例

### MCPサーバー設定を維持したバックグラウンド移行

```bash
# 複数のMCPサーバーを指定して起動
claude --mcp-config config/mcp.json --strict-mcp-config

# 長時間タスクをバックグラウンドに移行
/bg "データベースのマイグレーションスクリプトをすべて更新してください"

# セッションが再起動しても config/mcp.json の設定が保持される
# → MCPサーバーとの接続が維持される
```

### フォールバックモデルを設定して高可用性を確保

```bash
# プライマリモデルが過負荷の場合のフォールバックを設定
claude --model claude-opus-4-7 --fallback-model claude-sonnet-4-6

# バックグラウンドに移行
/bg "テストスイート全体を実行して失敗したテストを修正してください"

# → Opusが過負荷になった場合、Sonnetにデグレードして継続
# → ハードフェイルではなく、処理が継続される
```

### ← キーによるデタッチでパーミッションバイパスを維持

```bash
# パーミッションバイパスを許可して起動
claude --allow-dangerously-skip-permissions

# ← キーでバックグラウンドにデタッチ
# → デタッチ後もShift+Tabのサイクルにbypassモードが含まれる
```

### リタイア→復帰をまたいだパーミッション維持

```bash
# 危険なパーミッションスキップを有効にして起動
claude --bg --dangerously-skip-permissions "大規模な自動化タスク"

# セッションがリタイアして復帰しても dangerously-skip-permissions が維持される
```

### 維持されるフラグ一覧

| フラグ | 維持タイミング |
|--------|----------------|
| `--mcp-config` | `/bg` 移行後・respawn後 |
| `--settings` | `/bg` 移行後・respawn後 |
| `--add-dir` | `/bg` 移行後・respawn後 |
| `--plugin-dir` | `/bg` 移行後・respawn後 |
| `--strict-mcp-config` | `/bg` 移行後・respawn後 |
| `--fallback-model` | `/bg`・`←`デタッチ後 |
| `--allow-dangerously-skip-permissions` | `/bg`・`←`デタッチ後 |
| `--dangerously-skip-permissions` | retire→wake後 |

## 注意点

- 以前のバージョンでは `/bg` でバックグラウンドに移行すると、これらの設定が失われる問題がありました。このバージョンで修正されています
- `--dangerously-skip-permissions` および `--allow-dangerously-skip-permissions` は信頼できる環境でのみ使用してください
- フォールバックモデルは過負荷時のみ使用され、通常時はプライマリモデルが使われます
- `--strict-mcp-config` を指定している場合、列挙されていないMCPサーバーへのアクセスは引き続き禁止されます
