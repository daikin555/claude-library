## 原文（日本語に翻訳）

EnterpriseプランのOAuthユーザーだけでなく、TeamプランのOAuthユーザーに対してもポリシー制限（リモートコントロール制限など）の取得を追加

## 原文（英語）

Added policy limit fetching (e.g., remote control restrictions) for Team plan OAuth users, not just Enterprise

## 概要

これまでEnterpriseプランのOAuthユーザーにのみ適用されていたポリシー制限の取得が、Teamプランユーザーでも機能するようになりました。リモートコントロール制限などの組織ポリシーが、Teamプランを利用している組織でも正しく反映されます。

## 対象ユーザー

- **対象**: Claude.aiのTeamプランでOAuth認証を使用している組織
- **以前の動作**: Enterpriseプランのみポリシー制限が取得・適用されていた
- **変更後の動作**: TeamプランでもOAuth経由でポリシーが正しく取得・適用される

## 影響を受けるポリシー

- リモートコントロールの使用制限
- その他の組織レベルの制限設定

## 実践例

### Teamプランでのリモートコントロール制限

Teamプランの管理者がclaude.ai/adminコンソールでリモートコントロールを制限している場合、メンバーがClaude Codeで `/remote-control` を使おうとすると、適切にブロックされるようになります。

### 組織ポリシーの動作確認

```bash
# Teamプランユーザーでログイン
claude

# /remote-control が制限されている場合
/remote-control
# → 「リモートコントロールは組織のポリシーにより無効化されています」などのメッセージが表示される
```

## 注意点

- この変更はTeamプランでOAuth認証を使用している場合に適用されます
- APIキーで認証している場合は従来通りの動作です
- ポリシー設定はclaude.ai管理コンソールで行います
