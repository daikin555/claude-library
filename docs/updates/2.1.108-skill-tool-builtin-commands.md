# モデルが組み込みスラッシュコマンドをSkillツールで自動発見・実行

## 原文（日本語に翻訳）

モデルがSkillツールを通じて `/init`・`/review`・`/security-review` などの組み込みスラッシュコマンドを発見して実行できるようになった。

## 原文（英語）

The model can now discover and invoke built-in slash commands like `/init`, `/review`, and `/security-review` via the Skill tool.

## 概要

Claude Codeのモデルが、Skillツールを介して組み込みのスラッシュコマンド（`/init`、`/review`、`/security-review` など）を自律的に発見・実行できるようになりました。これまでユーザーが手動でスラッシュコマンドを呼び出す必要があった場面で、Claudeが文脈に応じて適切なコマンドを自動的に選択・実行できるようになります。複雑なタスクの自動化やエージェントワークフローの質が大幅に向上します。

## 基本的な使い方

### Claudeが自動でコマンドを選択する例

```
# ユーザーのプロンプト
「このプロジェクトの初期設定をしてください」

# Claudeが自動的に /init を発見して実行
# → CLAUDE.md の生成、プロジェクト構造の分析が自動で行われる
```

```
# ユーザーのプロンプト
「このPRをセキュリティの観点からレビューしてください」

# Claudeが /security-review を発見して実行
# → セキュリティレビューが自動で開始される
```

### ユーザーが明示的にスキル実行を依頼する例

```
「コードのレビューをお願いします」
→ Claudeが /review スキルを発見して実行

「プロジェクトのCLAUDE.mdを作成してください」
→ Claudeが /init スキルを発見して実行
```

## 実践例

### エージェントワークフローでの活用

```
# 複合的なタスクの自動処理
「新しいAPIエンドポイントを追加した後、コードレビューとセキュリティチェックを行ってください」

# Claudeが自律的に：
# 1. コードを追加（EditツールやBashツールを使用）
# 2. /review スキルを発見して実行（コードレビュー）
# 3. /security-review スキルを発見して実行（セキュリティチェック）
```

### 新規プロジェクトのセットアップ自動化

```
「このリポジトリをClaudeで使えるように初期設定してください」

# Claudeが実行するステップ：
# 1. プロジェクト構造の分析
# 2. /init スキルを自動呼び出し → CLAUDE.md の生成
# 3. 重要なファイルとディレクトリの説明を追加
```

### コードレビュープロセスの自動化

```bash
# スクリプトやCIでClaude Codeを使用する場合
claude --print "mainブランチとの差分をレビューして、問題点をリストアップしてください"

# Claudeが適切なスキル（/review）を発見・実行して
# 差分分析とレビュー結果を出力する
```

### 対応している組み込みスラッシュコマンド

| コマンド | 機能 |
|---------|------|
| `/init` | プロジェクト初期設定・CLAUDE.md生成 |
| `/review` | コードレビューの実行 |
| `/security-review` | セキュリティレビューの実行 |
| その他の組み込みスキル | 文脈に応じて自動発見 |

## 注意点

- **ユーザー定義スキルとの違い**: 組み込みスラッシュコマンドはClaude Codeに標準搭載されているもので、カスタムスキル（`.claude/skills/` に配置するもの）とは別物です。
- **自動実行の透明性**: Claudeがスキルを呼び出す際は、どのスキルを使用しているかがUIに表示されます。
- **権限設定との連携**: 自動モード（`--auto-approve` 等）でも、Skillツール経由のコマンド実行は適切な権限チェックが行われます。
- **エージェントモードでの活用**: サブエージェント（Agentツール）使用時にも、組み込みスラッシュコマンドの自動発見・実行が機能します。
