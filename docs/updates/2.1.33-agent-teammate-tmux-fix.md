## 原文（日本語に翻訳）

tmux環境でのエージェントチームメイトセッションを修正しました。

## 原文（英語）

Fixed agent teammate sessions in tmux

## 概要

tmux（terminal multiplexer）環境でエージェントチームメイト機能を使用する際に発生していた問題が修正されました。これにより、tmuxユーザーもエージェントチーム機能を安定して利用できるようになります。

## 基本的な使い方

tmux内でエージェントチーム機能を有効にして使用できます。

```bash
# tmuxセッション内で
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude
```

## 実践例

### tmux環境でのエージェントチーム起動

tmuxウィンドウ内でエージェントチームを使用する場合。

```bash
# tmuxセッションを開始
tmux new -s development

# エージェントチームを有効化
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

# エージェントチームが正常に動作
# 修正前：セッションの初期化エラー
# 修正後：正常に起動
```

### 複数ペインでの並行作業

tmuxの複数ペインでClaude Codeセッションを実行する場合。

```bash
# tmuxで画面分割
tmux split-window -h

# 左ペイン：メインセッション
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

# 右ペイン：別の作業
# エージェントチームメイトが正常に通信
```

### tmuxセッションの永続化

tmuxセッションをデタッチして再アタッチする場合。

```bash
# セッション開始
tmux new -s claude-work
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

# デタッチ（Ctrl+B → D）
# 後で再アタッチ
tmux attach -t claude-work

# エージェントチーム状態が保持される
```

## 注意点

- エージェントチーム機能は研究プレビュー段階です
- `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 環境変数が必要です
- tmux環境でも通常の端末と同様に動作するようになりました
- tmux内でのキーボードショートカットは、tmuxのプレフィックスキーと競合しないよう注意してください
- この修正により、tmuxユーザーもエージェントチーム機能を利用できるようになりました
