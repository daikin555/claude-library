## 原文（日本語に翻訳）

MEMORY.mdインデックスを読み込み上限超過の状態にしてしまうメモリ書き込みは、これまでのサイレントな切り捨てではなく、明示的なエラーを発生させるようになりました

## 原文（英語）

Memory writes that leave a MEMORY.md index over its read limit now produce an explicit error instead of silent truncation

## 概要

Claude Codeのauto memory機能は、プロジェクトごとに `~/.claude/projects/<project>/memory/MEMORY.md` をインデックスファイルとして持ち、セッション開始時に「先頭200行、または先頭25KBのどちらか早い方まで」を読み込みます。これまでは、Claudeがメモリを書き込んだ結果MEMORY.mdがこの上限を超えてしまっても何のエラーも出ず、次回セッション以降は超過分がサイレントに読み込まれない（Claudeがその存在を忘れる）という挙動でした。v2.1.210では、書き込み操作の結果としてMEMORY.mdが上限を超える場合、その場で明示的なエラーが発生するようになり、問題に気づかないまま記憶が失われる事態を防ぎます。

## 基本的な使い方

### 修正前の問題

```bash
claude

> APIのリトライ戦略についてもっと詳しく覚えておいて

# Claudeが MEMORY.md に追記
# → 200行 / 25KBを超過してしまう
# → 書き込み自体はエラーなく成功したように見える
# → 次回セッション開始時、超過した行はロードされない
# → Claudeは以前覚えたはずの内容を「忘れている」
# → ユーザーはそれに気づかず、同じ説明を繰り返すことになる
```

### 修正後の動作

```bash
claude

> APIのリトライ戦略についてもっと詳しく覚えておいて

# MEMORY.md への追記がインデックスの読み込み上限（200行 / 25KB）を
# 超える場合、書き込み操作自体が明示的なエラーで失敗する

# tool error:
# MEMORY.md index exceeds read limit (200 lines / 25KB).
# Move detailed content into a topic file and keep MEMORY.md
# as a concise index.

# Claudeはエラーを受けて、詳細を debugging.md のような
# トピックファイルへ移動し、MEMORY.md 側は要点のみの
# インデックスとして書き直す
```

## 実践例

### トピックファイルへの分割を促されるケース

```bash
claude

> このプロジェクトのビルド手順とデバッグでハマったポイントを
> 全部メモリに残しておいて

# MEMORY.md への直接追記が上限超過でエラーになった場合、
# Claudeは自動的に以下のような構成へ分割する

~/.claude/projects/<project>/memory/
├── MEMORY.md          # 「ビルド手順は build.md 参照」等の短い索引のみ
├── build.md           # ビルド手順の詳細
└── debugging.md       # デバッグでハマったポイントの詳細
```

### `/memory` コマンドで現状を確認する

```bash
claude

> /memory

# MEMORY.md と関連するトピックファイルの一覧が表示される
# エディタで開いて、内容が肥大化していないか直接確認できる
```

### 手動でMEMORY.mdを整理する

```bash
# エラーが出た場合、MEMORY.mdを直接編集して
# 詳細な記述をトピックファイルへ移す

$EDITOR ~/.claude/projects/<project>/memory/MEMORY.md

# 索引としては簡潔に:
# - ビルド: build.md参照
# - API規約: api-conventions.md参照
# - デバッグ: debugging.md参照
```

### auto memoryを一時的に無効化して様子を見る

```bash
claude

> /memory
# auto memoryトグルをオフにする

# もしくは settings.json で
```

```json
{
  "autoMemoryEnabled": false
}
```

## 注意点

- MEMORY.mdの読み込み上限は「先頭200行、または先頭25KBのどちらか早い方まで」で、これは今回の修正でも変わっていません。変わったのは、上限を超える書き込みが発生した際の挙動（サイレント切り捨て → 明示的エラー）です
- この上限が適用されるのはMEMORY.md（インデックスファイル）のみです。CLAUDE.mdファイルは長さに関わらず全文が読み込まれます（ただし短いファイルほど遵守率は高くなります）
- `debugging.md` や `api-conventions.md` のようなトピックファイルはセッション開始時には読み込まれず、Claudeが必要になった時点で通常のファイル読み取りツールでオンデマンドに読みます。詳細情報はトピックファイル側に逃がすのが正しい設計です
- auto memoryはマシンローカルです。同一Gitリポジトリの全ワークツリー・サブディレクトリで1つのメモリディレクトリを共有しますが、別マシンやクラウド環境とは共有されません
- `/memory` コマンドでいつでもMEMORY.mdや関連ファイルを一覧・編集・削除できます。すべてプレーンなMarkdownです
