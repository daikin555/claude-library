## 原文（日本語訳）

コマンド実行中に、Claudeを起動した元のディレクトリが削除・ロック・アンマウントされた場合にWindowsでクラッシュしていた問題を修正。

## 原文（英語）

Fixed a Windows crash when the directory Claude was launched from is deleted, locked, or unmounted while a command is running

## 概要

Windows版のClaude Codeでは、CLIを起動した際のカレントディレクトリ（起動元ディレクトリ）をプロセスがハンドルとして保持し続ける挙動があり、コマンド実行中にそのディレクトリが削除されたり、外部ドライブの取り外し等でアンマウントされたり、他プロセスにロックされたりすると、Claude Code自体がクラッシュしてしまう問題がありました。本バージョンでは、このようなファイルシステム側の変化が起きてもプロセスが異常終了しないように修正されています。ネットワークドライブやリムーバブルストレージ上で作業しているユーザーにとって特に影響のあった不具合です。

## 基本的な使い方

この修正はユーザーが明示的に操作する機能ではなく、内部的な安定性の改善です。特別な設定や操作は不要で、Windows版のClaude Codeを最新バージョンに更新するだけで恩恵を受けられます。

```powershell
winget upgrade Anthropic.ClaudeCode
```

アップデート後は、これまでクラッシュの原因になっていた「起動元ディレクトリの消失」に対して、Claude Codeがより堅牢に動作するようになります。

### 動作確認の例

たとえば以下のように、Claude Codeをあるディレクトリで起動した状態のまま、そのディレクトリを別プロセスからリネームまたは削除しても、以前のように即座にクラッシュすることはなくなります（起動元ディレクトリ以外の場所にいる場合の挙動を含め、引き続きコマンドの実行結果自体はディレクトリの状態に依存します）。

```powershell
# PowerShellでClaude Codeを起動
cd C:\projects\my-app
claude

# 別のターミナルから起動元ディレクトリを削除しても
# Claude Code本体が異常終了しなくなった
Remove-Item -Recurse -Force C:\projects\my-app
```

## 実践例

### ネットワークドライブ上のプロジェクトで作業している場合

社内共有ドライブ（例: `Z:\shared\project`）をマウントした状態でClaude Codeを起動し、長時間のタスクを実行中にVPNが切断されたりネットワークドライブが一時的に切断されたりするケースがあります。従来はこの瞬間にWindows版Claude Codeがクラッシュし、実行中の作業内容が失われることがありましたが、この修正により、起動元ディレクトリへのアクセスが失われてもプロセスが落ちにくくなりました。

```powershell
# Zドライブ（ネットワーク共有）上でClaude Codeを起動
cd Z:\shared\project
claude
# ネットワーク切断が発生してもClaude Code本体は継続動作しやすくなる
```

### 外部ストレージ（USBドライブ）上で作業している場合

USBメモリや外付けSSD上のプロジェクトディレクトリでClaude Codeを起動したまま、作業中にドライブを物理的に取り外してしまう、あるいはEnterprise環境のリムーバブルディスクポリシーによってアンマウントされる、といった状況が想定されます。以前はこの操作が原因でWindowsアプリ全体がクラッシュすることがありましたが、修正後はこうしたストレージの喪失に対する耐性が向上しています。

```powershell
# 外付けドライブ E: 上のプロジェクトで起動
cd E:\portable-projects\demo
claude
# ドライブ取り外し時もクラッシュしにくくなった
```

## 注意点

- この修正はあくまで「Claude Code本体がクラッシュしなくなる」ことを保証するものであり、起動元ディレクトリが失われた状態でファイル操作系のツール（Read/Write/Editなど）を実行した場合、そのコマンド自体はディレクトリが存在しないことによるエラーになる可能性があります。
- Windows特有の問題であり、macOS/Linux版のClaude Codeでは元々この種のクラッシュは報告されていません。
- 根本的な回避策としては、作業中は起動元ディレクトリを外部ドライブの取り外しやネットワーク切断の影響を受けにくいローカルディスク上に置くことが引き続き推奨されます。
- Windows版はネイティブインストーラー（`irm https://claude.ai/install.ps1 | iex` またはWinGet経由）での利用が推奨されており、WinGet経由の場合は自動更新されないため、`winget upgrade Anthropic.ClaudeCode` で定期的に最新化することが必要です。
