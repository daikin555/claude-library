# /clear コマンドがキャッシュされたスキルをリセットしない問題を修正

## 原文（日本語に翻訳）

`/clear` がキャッシュされたスキルをリセットしない問題を修正しました。新しい会話で古いスキルのコンテンツが残り続ける可能性がありました。

## 原文（英語）

Fixed `/clear` not resetting cached skills, which could cause stale skill content to persist in the new conversation

## 概要

Claude Code v2.1.63で、`/clear` コマンドを実行してもキャッシュされたスキルがリセットされない問題が修正されました。この問題により、`/clear` 後の新しい会話でも古いスキルの内容が残り続け、スキルを更新した場合でも古い内容が使用されることがありました。修正後は `/clear` 実行時にスキルキャッシュも適切にリセットされます。

## 基本的な使い方

### /clear コマンドの使用

```
/clear
```

修正後は、新しい会話を開始する際に最新のスキル定義が読み込まれます。

## 実践例

### スキルを更新してから新しい会話を開始する

```
# スキルファイルを更新
.claude/skills/my-skill.md を編集

# 会話をクリア
/clear

# 修正前：古いスキル内容でキャッシュされたまま
# 修正後：新しいスキル内容が正しく読み込まれる

# 新しい会話でスキルを使用
/my-skill
```

### 開発中のスキルのテスト

```
# スキル開発中の反復テストワークフロー
1. スキルを更新
2. /clear で会話をリセット（スキルキャッシュも適切にクリア）
3. 更新後のスキルをテスト
4. 改善して繰り返す
```

## 注意点

- v2.1.63にアップデートすることで自動的に修正が適用されます
- カスタムスキルを頻繁に更新・テストするユーザーに特に効果があります
- スキルのホットリロード機能（セッション中の自動更新）とは別の修正です
