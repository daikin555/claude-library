## 原文（日本語に翻訳）

スキルやコマンド内で対応する引数がない `$1`/`$2` のような位置指定プレースホルダーが、無言で削除されてしまう不具合を修正しました。現在はそのまま（verbatim）保持されます。

## 原文（英語）

Fixed unmatched `$1`/`$2` positional placeholders in skills and commands being silently stripped; they are now preserved verbatim

## 概要

Claude Codeのカスタムスラッシュコマンドやスキルでは、`$1`、`$2`、`$ARGUMENTS` のような位置指定プレースホルダーを使って、呼び出し時に渡された引数をテンプレートに埋め込むことができます。今回の修正前は、呼び出し時に対応する引数が渡されなかった `$1` や `$2` が、テンプレート展開時に黙って空文字列に置き換えられ（実質的に消えて）しまうバグがありました。v2.1.210では、対応する引数がない位置指定プレースホルダーはそのままの文字列として保持されるようになり、意図しない情報欠落やプロンプトの意味変化を防げるようになりました。

## 基本的な使い方

### 修正前の問題

```markdown
<!-- .claude/commands/review.md -->
---
description: "PRをレビューする"
---
$1 のPRを $2 の観点でレビューして。
```

```bash
# 引数を1つしか渡さない場合
> /review 123

# 修正前:
# "$2" が silently strip され、以下のようなテンプレートになってしまう
# " 123 のPRを  の観点でレビューして。"
# → "$2の観点で"という指示が消え、Claudeに意図が伝わらない
```

### 修正後の動作

```bash
> /review 123

# 修正後:
# 対応する引数がない $2 はそのまま文字列として残る
# "123 のPRを $2 の観点でレビューして。"
# → プレースホルダーが未展開のまま残るため、
#   引数不足に気づきやすく、意図しない省略も起きない
```

## 実践例

### 複数引数を取るスキルでの引数チェック

引数が足りない場合に `$2` がそのまま残ることを利用して、Claude自身が「引数が不足している」と判断しやすくなります。

```markdown
<!-- .claude/skills/migrate-component/SKILL.md -->
---
name: migrate-component
description: "コンポーネントを別フレームワークに移行する"
---
$1 コンポーネントを $2 から $3 に移行して。
テストも一緒に更新すること。
```

```bash
> /migrate-component SearchBar React

# 修正後の展開結果:
# "SearchBar コンポーネントを React から $3 に移行して。テストも一緒に更新すること。"
# → $3 が未展開のまま残るため、Claudeが
#   「移行先フレームワークが指定されていません」と気づき、
#   ユーザーに確認を返すことができる
```

### オプション引数を持つコマンドの安全な設計

一部の引数が省略可能なコマンドで、省略時にプレースホルダー文字列自体をデフォルト値の判定材料として使う。

```markdown
<!-- .claude/commands/deploy.md -->
---
description: "指定環境にデプロイする"
---
$1 環境（未指定時は staging 扱い）にデプロイして。
追加オプション: $2
```

```bash
> /deploy production

# 修正後:
# "production 環境（未指定時は staging 扱い）にデプロイして。
#  追加オプション: $2"
# → Claudeは "$2" という未展開の文字列を見て
#   「追加オプションは特に指定されていない」と正しく解釈できる
```

### 既存テンプレートの互換性確認

既存のコマンド・スキルを棚卸しし、未使用の位置指定プレースホルダーが意図通りに扱われるか確認する。

```bash
# .claude/commands 配下のテンプレートを確認
> .claude/commands/ 配下のコマンドで $1〜$9 を使っているものを一覧にして

# アップグレード後、各コマンドを実際に引数不足で呼び出してみて、
# プレースホルダーが黙って消えず、テンプレート内に残ることを確認する
> /my-command arg1
```

## 注意点

- この修正はClaude Code v2.1.210で導入されました
- 修正前の挙動（未指定プレースホルダーの無言削除）に依存したテンプレート設計をしていた場合、v2.1.210以降では出力に `$2` のような未展開文字列がそのまま含まれるようになるため、既存のコマンド・スキルの出力が変わる可能性があります
- 引数の過不足を防ぐには、`$ARGUMENTS` を使って全引数をまとめて受け取り、Claude側でパースさせる設計や、コマンド定義内で必須引数を明記しておく方法が推奨されます
- `$1`、`$2` などの位置指定プレースホルダーと `$ARGUMENTS`（全引数まとめて）は挙動が異なるため、用途に応じて使い分けてください
- スキルとカスタムコマンドの両方でこの修正が適用されます
