## 原文（日本語に翻訳）

APIキー・Bedrock・VertexユーザーでHTTP 429レート制限エラーが読みやすいメッセージの代わりに生のJSONダンプで表示される問題を修正しました。

## 原文（英語）

Fixed 429 rate-limit errors showing a raw JSON dump instead of a clean message for API-key, Bedrock, and Vertex users

## 概要

APIキー認証、AWS Bedrock、Google Vertex AI経由でClaude Codeを使用している際にレート制限（HTTP 429）エラーが発生した場合、JSON形式の生のエラーレスポンスが表示されていた問題が修正されました。現在は「レート制限に達しました。しばらく待ってからリトライしてください」といった人が読みやすいメッセージが表示されるようになりました。

## 修正内容

```
Before: {"error":{"type":"rate_limit_error","message":"Rate limit exceeded",...}}
After:  レート制限に達しました。少し待ってから再試行してください。
```

## 注意点

- この修正はAPIキー認証、AWS Bedrock、Google Vertex AIを使用しているユーザーに適用されます
- claude.ai経由（oauth認証）のユーザーは以前から適切なメッセージが表示されていました
