## 原文（日本語に翻訳）

`/remote-control` および `claude remote-control` にオプションの名前引数を追加（`/remote-control My Project` または `--name "My Project"`）。claude.ai/codeに表示されるカスタムセッションタイトルを設定できます。

## 原文（英語）

Added optional name argument to `/remote-control` and `claude remote-control` (`/remote-control My Project` or `--name "My Project"`) to set a custom session title visible in claude.ai/code

## 概要

リモートコントロールセッションにカスタム名前を付けられるようになりました。複数のリモートセッションを管理する場合に、claude.ai/codeのセッション一覧から目的のセッションを素早く識別できます。

## 基本的な使い方

### スラッシュコマンドで名前を指定

```
/remote-control My Project
```

```
/remote-control バックエンドAPI開発
```

### CLIから名前を指定

```bash
claude remote-control --name "My Project"
claude remote-control --name "バックエンドAPI開発"
```

## 実践例

### 複数プロジェクトのリモートセッション管理

複数のプロジェクトを並行して開発している場合：

```bash
# プロジェクトAのディレクトリで
cd ~/projects/frontend
claude remote-control --name "フロントエンド開発"

# プロジェクトBのディレクトリで
cd ~/projects/backend
claude remote-control --name "APIサーバー開発"

# データ分析タスク用
cd ~/analysis
claude remote-control --name "Q1データ分析"
```

これにより、claude.ai/code上でセッション一覧を見たときに、それぞれの目的が一目でわかります。

### チームでの利用

チームメンバーがリモートコントロールを使う場合、セッション名にユーザー名や作業内容を入れると管理しやすくなります：

```bash
claude remote-control --name "田中 - 認証機能実装"
```

### 既存セッションに接続してから名前を変更

```
/remote-control 新しいセッション名
```

## 注意点

- セッション名はclaude.ai/codeのUIに表示されます
- 名前にスペースが含まれる場合は引用符で囲むか、スラッシュコマンド形式（`/remote-control 名前`）を使用してください
- 名前を指定しない場合は従来通りの動作（デフォルト名）になります
