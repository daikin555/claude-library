## 原文（日本語に翻訳）

展開したサブエージェントトランスクリプトビュー（Ctrl+O）でツール呼び出しの下に余分な空行が表示される問題を修正

## 原文（英語）

Fixed extra blank lines appearing below each tool call in the expanded subagent transcript view (Ctrl+O)

## 概要

`Ctrl+O` でサブエージェントのトランスクリプトを展開して表示した際、各ツール呼び出しの下に不要な空行が挿入されていました。この修正により、トランスクリプトビューがすっきりと表示され、内容の読みやすさが向上します。

## 基本的な使い方

```
# サブエージェントのトランスクリプトを展開表示
Ctrl+O

# 修正前: 各ツール呼び出しの後に余分な空行が表示される
# 修正後: 余分な空行がなくなり、整理された表示になる
```

## 実践例

### サブエージェントのトランスクリプト確認

```
# マルチエージェントタスク実行後
> 複数のファイルを並行して処理してください

# Ctrl+O でトランスクリプトを確認
# 修正前:
#   [Tool: read_file] config.json
#                                    ← 余分な空行
#   [Tool: read_file] package.json
#                                    ← 余分な空行
#   [Tool: write_file] output.json
#                                    ← 余分な空行
#
# 修正後:
#   [Tool: read_file] config.json
#   [Tool: read_file] package.json
#   [Tool: write_file] output.json
```

### 長いトランスクリプトのスクロール

```
# 多数のツール呼び出しがある場合
# 修正後は余分な空行がなく、より多くの内容を一度に確認できる
Ctrl+O
# → コンパクトな表示でトランスクリプトを効率的に確認
```

## 注意点

- この修正は表示の改善であり、機能上の変化はありません
- `Ctrl+O` はサブエージェントを使用するタスクで最も効果的です
- 通常の会話ビューには影響しません
