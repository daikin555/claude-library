## 原文（日本語に翻訳）

コマンド候補の例をHaikuを呼び出す代わりに決定論的に生成するよう変更

## 原文（英語）

Changed example command suggestions to be generated deterministically instead of calling Haiku

## 概要

Claude Codeが表示するコマンド候補（例：`/help` などのコマンドの使用例）は、以前はClaude Haiku APIを呼び出して動的に生成していました。この変更により、候補は決定論的なアルゴリズムで生成されるようになりました。APIコストの削減、表示の高速化、ネットワーク接続不要での候補表示が実現します。

## 基本的な使い方

この変更は自動的に適用されます。コマンド候補が以前より素早く表示され、一貫した内容が表示されます。

```bash
# コマンドを入力する際
/

# 修正後: 候補がHaiku APIを待たずに即座に表示される
# → オフライン環境でも候補が表示される
```

## 実践例

### コマンド候補の素早い表示

```bash
claude
# スラッシュを入力
/

# 修正前: Haiku APIの応答を待ってから候補を表示
# 修正後: 即座に決定論的な候補を表示
# /help, /clear, /config, /context, ...
```

### ネットワーク制約のある環境での使用

```bash
# 不安定なネットワーク環境
claude
/

# 修正後: API呼び出しなしで候補が表示される
# → ネットワークが遅い環境でもスムーズ
```

### コスト効率の向上

```bash
# 頻繁に候補を確認する場合
# 修正前: 各候補表示でHaiku APIを呼び出し → コスト発生
# 修正後: 決定論的生成 → APIコストなし

# 1日に100回候補を確認しても追加コストなし
```

## 注意点

- 候補の内容は決定論的になるため、毎回同じ候補が表示されます（以前はHaikuによって動的に変化していました）
- コマンドの機能自体は変わりません
- この変更によりAPIコストが削減されますが、コマンドの実行結果には影響しません
- より一貫した候補が表示されることで、学習しやすくなります
