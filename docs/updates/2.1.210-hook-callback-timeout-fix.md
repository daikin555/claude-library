## 原文（日本語に翻訳）

フックコールバックのタイムアウトがモデルに対してユーザーの拒否として誤って報告され、無人（unattended）セッションが停止して待機してしまう問題を修正しました

## 原文（英語）

Fixed a hook callback timeout being misreported to the model as a user rejection, which made unattended sessions stop and wait

## 概要

Claude Code v2.1.210で修正された、フック（hooks）機構に関する重要なバグです。`PreToolUse`などのフックが応答を返すまでにタイムアウトしてしまった場合、以前はこれが「ユーザーがツール実行を拒否した」というシグナルとしてモデルに伝わってしまい、モデルはユーザーからの追加指示を待つ状態に入っていました。CI/CDや自動化パイプラインなど、人がその場にいない「無人セッション」ではこの誤動作によりセッションが永久に停止してしまう問題がありました。今回の修正で、フックのタイムアウトは正しく「タイムアウト」として扱われ、モデルが不要な待機状態に陥らないようになりました。

## 基本的な使い方

### 修正前の問題

```json
// settings.json - 実行に時間がかかるフックの例
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "slow-validation-script.sh"
          }
        ]
      }
    ]
  }
}
```

```text
1. slow-validation-script.sh の実行に時間がかかりすぎる
2. フックのタイムアウトが発生
3. 修正前: モデルには「ユーザーがツール実行を拒否した」と伝わる
4. モデルは追加のユーザー入力を待つ状態になり、無人セッションでは
   永久に停止してしまう
```

### 修正後の動作

```text
1. フックがタイムアウト
2. 修正後: 「タイムアウトが発生した」ことが正しくモデルに伝わる
3. モデルはタイムアウトとして状況を認識し、
   適切に処理を継続または報告できる
4. 無人セッションが不要に停止・待機することがなくなる
```

## 実践例

### CI環境での自動フック実行

CIパイプラインなど、人手を介さずにClaude Codeを動かすユースケースで特に重要な修正です。

```bash
# ヘッドレスモードでの実行例
claude -p "テストを実行して失敗を修正して" \
  --output-format stream-json

# 修正前: PreToolUseフックが重い処理（lint等）でタイムアウトすると
#         「ユーザー拒否」と誤認識され、セッションがハングしていた
# 修正後: タイムアウトとして扱われ、処理が継続される
```

### タイムアウトしやすい重いフックの設定

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "heavy-static-analysis.sh",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
```

```text
# 5秒でタイムアウトする設定
# 修正前: タイムアウト時に「ユーザーが拒否した」という誤情報がモデルに渡る
# 修正後: タイムアウトである旨が正しく伝わり、モデルが適切に次の一手を判断できる
```

### スケジュール実行タスクとの組み合わせ

`/loop` やcronによる定期実行タスクなど、完全に無人で動くセッションでの影響を確認します。

```bash
# 定期実行されるセッションでフックがタイムアウトするケース
claude -p "定期メンテナンスタスクを実行" --continue

# 修正前: フックタイムアウトのたびに「ユーザー拒否」扱いとなり、
#         セッションが無限に応答待ちになり自動化が破綻していた
# 修正後: 正しくタイムアウトとして扱われ、自動化が継続する
```

## 注意点

- この修正は Claude Code v2.1.210 で実装されました
- 影響が大きいのは、人間が対話的に操作していない「無人（unattended）セッション」（CI、バッチ処理、スケジュール実行など）
- フックの`timeout`設定は引き続き有効なので、重い処理を行うフックには適切なタイムアウト値を設定することが推奨されます
- タイムアウトが頻発する場合は、フックスクリプト自体の処理速度を見直すことも検討してください
- `--debug` フラグを使うことで、フックの実行状況やタイムアウトの発生を確認できます
