## 原文（日本語に翻訳）

macOS でデーモンが接続中に終了した後に孤立した `claude --bg-pty-host` プロセスが CPU 100% でスピンするバグを修正しました。

## 原文（英語）

Fixed orphaned `claude --bg-pty-host` processes spinning at 100% CPU after the daemon dies while connected on macOS

## 概要

macOS 環境でバックグラウンド PTY ホストのデーモンが異常終了した後、`claude --bg-pty-host` プロセスが孤立して CPU を 100% 消費し続けるバグを修正しました。この問題によりバッテリー消耗や Mac の過熱が発生していましたが、修正後は孤立プロセスが適切に終了します。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
```

## 実践例

### 修正前の問題の確認方法

以前の問題が発生していた場合、アクティビティモニタで確認できました：

```bash
# ターミナルで確認
ps aux | grep "bg-pty-host"
# 複数の claude --bg-pty-host プロセスが高CPU使用率で表示されていた
```

### 修正後の確認

```bash
# v2.1.166 以降では孤立プロセスが自動的にクリーンアップされる
ps aux | grep "bg-pty-host"
# → 実行中のセッションがある場合のみプロセスが表示される
```

### 既存の孤立プロセスをクリーンアップ（アップデート前の残留プロセス）

```bash
# 残留している孤立プロセスを手動で終了
pkill -f "claude --bg-pty-host"
```

## 注意点

- この修正は macOS 環境に限定されます
- Linux・Windows 環境には影響しません
- v2.1.166 より前のバージョンを使用していた場合、既存の孤立プロセスが残っている可能性があります。その場合は手動で終了してください
