## 原文（日本語）
`.claude/agents/`ディレクトリ内の非エージェントMarkdownファイルに対する不要な警告を修正しました。

## 原文（英語）
Fixed spurious warnings for non-agent markdown files in `.claude/agents/` directory

## 概要
`.claude/agents/`ディレクトリ内にエージェント定義ではない通常のMarkdownファイル（READMEやドキュメントなど）を配置した際に、誤って警告が表示されていた問題が修正されました。これにより、エージェントディレクトリ内でドキュメントファイルを自由に管理できるようになりました。

## 基本的な使い方

```bash
# .claude/agents/ディレクトリにREADMEを配置
mkdir -p .claude/agents
echo "# エージェント一覧" > .claude/agents/README.md

# 修正前: 警告が表示される
# 修正後: 警告は表示されない（正常動作）
```

## 実践例

### エージェントディレクトリのドキュメント化
```bash
# プロジェクトのエージェント設定ディレクトリ
.claude/
└── agents/
    ├── README.md              # エージェント一覧の説明
    ├── code-reviewer.md       # エージェント定義
    ├── test-runner.md         # エージェント定義
    └── CONTRIBUTING.md        # 新規エージェント追加ガイド

# 修正前: README.mdとCONTRIBUTING.mdに対して警告が表示される
# 修正後: エージェント定義ファイルのみが処理され、ドキュメントは無視される
```

```bash
├── templates/
│   └── template.md            # エージェント作成テンプレート

│   ├── api-developer.md       # エージェント定義

└── frontend/
    ├── README.md              # フロントエンドエージェントの説明
    └── ui-developer.md        # エージェント定義

# 修正により、各README.mdに警告が表示されなくなる
# エージェント定義ファイルのみが正しく認識される
```

### エージェントの使用例ドキュメント
```bash
# エージェントごとの使用例を追加
.claude/agents/

└── test-runner-guide.md       # 詳細ガイド（非エージェントファイル）

# 修正により、*-examples.mdや*-guide.mdファイルに警告が出ない
# ドキュメントを自由に配置できる
```

### 開発環境と本番環境の分離

```bash
└── production/
    ├── README.md              # 本番環境用の説明

```

## 注意点

- この修正はMarkdownファイルの処理のみに影響し、エージェント機能自体は変更されません
- エージェント定義ファイルには引き続き適切なフォーマットが必要です
- `.claude/agents/`ディレクトリ内のファイル構造は自由に整理できます
- エージェント定義ファイルとドキュメントファイルを明確に区別するため、命名規則を設けることを推奨します
