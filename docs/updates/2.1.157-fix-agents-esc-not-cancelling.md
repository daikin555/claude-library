## 原文（日本語に翻訳）

`claude agents` で低速な「opening...」中にEscキーを押してもキャンセルされず、リストが操作不能になる問題を修正しました

## 原文（英語）

Fixed `claude agents` pressing Esc not cancelling a slow "opening…", leaving the list unresponsive

## 概要

Claude Code v2.1.157 で修正されたバグです。`claude agents` のセッションリストでエージェントセッションを開こうとした際、「opening...」の表示が出て応答が遅い場合に Esc キーを押してもキャンセルされず、リストが操作不能な状態になっていました。修正後は、「opening...」中に Esc キーを押すと正しくキャンセルされ、リストの操作が再開できます。

## 影響を受けていた状況

- `claude agents` でセッションリストを表示している場合
- エージェントセッションを開こうとして「opening...」の表示が長時間続いている場合
- ネットワーク遅延やリソース不足でセッションの起動に時間がかかっている場合
- Esc キーを押してもリストが固まったまま応答しない場合
- キャンセルできずに強制終了（Ctrl+C）に頼らなければならない場合

## 注意点

- v2.1.157 以降では自動的に修正されており、特別な操作は不要です
- 「opening...」中に Esc キーを押すと、開こうとしていた操作がキャンセルされます
- キャンセル後はセッションリストが正常に操作可能な状態に戻ります
- 接続自体が遅い場合はタイムアウト後に自動的にキャンセルされることもあります
