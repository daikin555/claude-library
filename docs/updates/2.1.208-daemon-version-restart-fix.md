## 原文（日本語訳）

バックグラウンドセッションにおいて、古いバージョンのデーモンが、新しいバージョンから生成されたワーカープロセスを古いバイナリへ黙って再起動してしまう問題を修正した。

## 原文（英語）

Background sessions: an older daemon no longer silently restarts workers spawned by a newer version onto the older binary

## 概要

Claude Codeのバックグラウンドセッションは、ユーザーごとの「スーパーバイザー」プロセスがワーカープロセスをまとめて管理する仕組みで動いています。Claude Code本体が更新されると、スーパーバイザーはディスク上の新しいバイナリを検知して新バージョンへ再起動しますが、以前はこのタイミング次第で、まだ古いバージョンのままのスーパーバイザーが、実際にはより新しいバージョンで起動していたワーカー（バックグラウンドセッション）を古いバイナリ上へ黙って巻き戻してしまうことがありました。今回の修正で、スーパーバイザーは常に「より新しいバージョンへ」しかワーカーを移行させないようになりました。

## 基本的な使い方

この修正自体はバックグラウンドの整合性を担保する内部的なバグ修正で、直接叩く新しいコマンドはありません。スーパーバイザーの状態は次のコマンドで確認できます。

```bash
claude daemon status
```

実行中のスーパーバイザーのバージョンやワーカー数、`claude`コマンド自体のバージョンとの差異などが表示されます。

## 実践例

### Claude Codeをアップデートした直後にバックグラウンドセッションを使い続ける場合

自動更新でClaude Codeのバイナリが新しくなった直後は、スーパーバイザーが新バージョンへ再起動するまでの間、複数バージョンが混在するタイミングが生じます。以前はこの過渡期に、更新前の古いスーパーバイザーが新しいバージョンで動いていたワーカーを誤って古いバイナリへ再起動させてしまうことがありましたが、修正後はこうした「バージョンの巻き戻り」が起きなくなりました。

### `claude daemon status`でバージョンの不整合を診断する

スーパーバイザーとCLIのバージョンが食い違っている場合、`claude daemon status`はその旨を警告し、`claude daemon stop --any`で新しいバージョンへ切り替えるよう促します。今回の修正と合わせて、開発者はワーカーが意図せず古いバイナリに戻っていないか、より安心して確認できるようになりました。

```bash
claude daemon status
# 出力例（イメージ）: running supervisor version differs from invoked `claude` version
claude daemon stop --any
```

## 注意点

- スーパーバイザーによる自動再起動は「新しいバージョンへの移行」のみを行い、古いバージョンへ戻すことはありません。作業中・入力待ち・ターミナルがアタッチされているセッションは中断されず、次にプロセスが再起動するタイミングで新バージョンに移行します。
- スーパーバイザーが実行中のバイナリより古いバージョンの場合、そのスーパーバイザーはワーカーに手を出さず、より新しいスーパーバイザーが引き継ぐまでそのまま新バージョンで動き続けます（今回の修正で保証された挙動）。
- 手動でバージョンを揃えたい場合は`claude daemon stop --any`でスーパーバイザーを停止し、次回起動時に最新バイナリから再度起動させることができます。
