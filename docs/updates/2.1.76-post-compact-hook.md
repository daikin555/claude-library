# PostCompactフック：コンパクション完了後に処理を実行

## 原文（日本語に翻訳）

コンパクション完了後に発火する `PostCompact` フックを追加

## 原文（英語）

Added `PostCompact` hook that fires after compaction completes

## 概要

Claude Code v2.1.76で、コンパクション（会話の圧縮）が完了した後に発火する `PostCompact` フックが追加されました。長時間のセッションでコンテキストウィンドウがいっぱいになると自動的にコンパクションが実行されますが、その際にプロジェクトルールや作業コンテキストが失われることがあります。このフックを使えば、コンパクション後にCLAUDE.mdの再読み込みを促したり、外部状態を更新したり、要約をログに記録するなどのフォローアップ処理を自動実行できます。

## 基本的な使い方

`~/.claude/settings.json` またはプロジェクトの `.claude/settings.json` にフックを設定します：

```json
{
  "hooks": {
    "PostCompact": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/post-compact-handler.sh"
          }
        ]
      }
    ]
  }
}
```

`matcher` には `manual`（`/compact` コマンド実行時）または `auto`（自動コンパクション時）を指定できます。空文字列で両方にマッチします。

## 実践例

### コンパクション要約をログファイルに保存

コンパクションの要約を記録して、後から参照できるようにする：

```bash
#!/bin/bash
# log-compaction.sh
INPUT=$(cat)

SESSION_ID=$(echo "$INPUT" | jq -r '.session_id')
TRIGGER=$(echo "$INPUT" | jq -r '.trigger')
SUMMARY=$(echo "$INPUT" | jq -r '.compact_summary')
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

LOG_DIR="$HOME/.claude/compaction-logs"
mkdir -p "$LOG_DIR"

cat >> "$LOG_DIR/compaction.log" <<EOF
---
timestamp: $TIMESTAMP
session_id: $SESSION_ID
trigger: $TRIGGER
summary: |
$(echo "$SUMMARY" | sed 's/^/  /')
EOF
```

### CLAUDE.mdの再読み込みリマインダー

コンパクション後にCLAUDE.mdの内容を標準出力に書き出し、Claude Codeに再認識させる：

```bash
#!/bin/bash
# remind-project-rules.sh
INPUT=$(cat)

CWD=$(echo "$INPUT" | jq -r '.cwd')

# プロジェクトのCLAUDE.mdが存在すれば内容を表示
if [ -f "$CWD/CLAUDE.md" ]; then
  echo "=== コンパクション後のリマインダー ==="
  echo "プロジェクトルール (CLAUDE.md) を再確認してください："
  cat "$CWD/CLAUDE.md"
  echo "=== リマインダー終了 ==="
fi
```

### 手動コンパクション時のみ通知

`/compact` コマンドを手動で実行した場合にのみ通知を送る：

```json
{
  "hooks": {
    "PostCompact": [
      {
        "matcher": "manual",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"コンパクションが完了しました\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

### 外部ツールとの連携

コンパクション情報を外部のプロジェクト管理ツールやSlackに通知する：

```bash
#!/bin/bash
# notify-compaction.sh
INPUT=$(cat)

TRIGGER=$(echo "$INPUT" | jq -r '.trigger')
SUMMARY=$(echo "$INPUT" | jq -r '.compact_summary' | head -c 500)

# 自動コンパクションの場合のみ通知
if [ "$TRIGGER" = "auto" ]; then
  curl -s -X POST "$SLACK_WEBHOOK_URL" \
    -H "Content-Type: application/json" \
    -d "{\"text\": \"Claude Codeで自動コンパクションが発生しました。\\n要約: ${SUMMARY}\"}"
fi
```

## 注意点

- **決定制御なし**: PostCompactフックはコンパクションの結果を変更できない。フォローアップ処理のみ実行可能
- **`type: "command"` のみサポート**: PostCompactフックではコマンドタイプのフックのみ使用できる
- **入力フィールド**: `trigger`（`manual` または `auto`）と `compact_summary`（コンパクションの要約テキスト）が提供される
- **コンテキスト喪失への対策**: コンパクションは会話の60〜70%のコンテキストを解放するが、その過程でプロジェクトルールなどの情報が薄まる可能性がある。PostCompactフックを活用して重要な情報を再注入することが推奨される
- **自動コンパクションの頻度**: v2.1.76ではサーキットブレーカーが追加され、自動コンパクションのリトライは最大3回までに制限されている
