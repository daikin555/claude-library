## 原文（日本語に翻訳）

Webhookペイロードや転送されたPRコメントなど、人間が直接入力したものではない入力に対して `ultracode` キーワードのオプトインが発火してしまう不具合を修正しました

## 原文（英語）

Fixed the `ultracode` keyword opt-in firing on non-human-originated input such as webhook payloads and relayed PR comments

## 概要

`ultracode` はプロンプト中にキーワードとして含めることで、そのタスクだけxhighの推論エフォートとDynamic Workflowsによる自動オーケストレーションを有効化する機能です。しかし、これまではCIやボット連携などで自動転送されるWebhookペイロードやPRコメントの本文にたまたま「ultracode」という文字列が含まれていた場合、人間が意図的に入力したわけではないのに `ultracode` モードが誤って有効化されてしまう問題がありました。Claude Code v2.1.210では、このオプトインの発火条件が人間が直接入力したプロンプトに限定されるよう修正されました。

## 基本的な使い方

### 修正前の問題

```bash
# GitHub Actions などから転送されたPRコメントの本文に
# たまたま "ultracode" という単語が含まれていた場合
#
# 例: PRコメント
# "この変更はultracodeライブラリのバージョンアップに関連しています"

claude -p "$RELAYED_PR_COMMENT"

# 修正前: コメント中の "ultracode" という文字列に反応し、
#         意図せずxhighエフォート + Dynamic Workflowsが有効化されてしまう
#         → 想定より大幅に時間とトークンを消費する
```

### 修正後の動作

```bash
claude -p "$RELAYED_PR_COMMENT"

# 修正後: Webhookペイロードや転送されたPRコメントなど、
#         人間が直接タイプしたのではない入力については
#         ultracodeキーワードのオプトインは発火しない
# → 通常のエフォート設定でタスクが実行される
```

## 実践例

### CI連携でのPRレビュー自動化

```yaml
# .github/workflows/pr-review.yml
- name: Run Claude Code review
  run: |
    claude -p "$(gh pr view ${{ github.event.pull_request.number }} --json body -q .body)"
```

```bash
# PR本文に "ultracode" という単語が偶然含まれていても、
# 修正後はこの自動化フローで誤ってultracodeモードが
# 有効化されることはない
```

### Webhook経由でのタスクディスパッチ

```bash
# 外部システムからのWebhookペイロードをそのままプロンプトとして渡す構成
curl -X POST https://internal-bot/dispatch \
  -d '{"prompt": "issue本文には ultracode という単語が含まれる場合がある"}'

# 修正後: このような非対話的・非人間由来の入力では
#         ultracodeのキーワードマッチングはスキップされる
```

### 人間が意図的に使う場合との違い

```bash
claude

> このリファクタリングタスクを ultracode で進めて

# こちらは対話セッションでユーザーが直接入力しているため、
# 修正後も引き続き正しくultracodeモード
# （xhighエフォート + Dynamic Workflows）が有効化される
```

## 注意点

- この修正はClaude Code v2.1.210（2026年7月15日リリース）で行われました
- `ultracode` はセッション全体に適用する `/effort ultracode` という設定と、単一タスクのみに適用するプロンプトキーワードとしての使い方の2種類があります
- ultracodeの利用にはClaude Code v2.1.154以降が必要で、有償プラン、Anthropic API、Bedrock、Vertex AI、Microsoft Foundryのいずれかで利用可能です
- 本修正により、CI/CDパイプラインやボットが転送する外部由来のテキスト（PRコメント、Issue本文、Webhookペイロードなど）にキーワードが偶然含まれていても、意図しないコスト増や実行時間の増加は発生しなくなりました
- 自動化フローで明示的に `ultracode` を有効化したい場合は、`--effort ultracode` のようなCLIフラグや `/effort` コマンドなど、意図が明確な手段を使うことが推奨されます
