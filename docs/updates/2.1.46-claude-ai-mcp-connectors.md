## 原文（日本語に翻訳）

Claude Code で claude.ai MCP コネクターの使用をサポートするようになりました。

## 原文（英語）

Added support for using claude.ai MCP connectors in Claude Code.

## 概要

Claude Code v2.1.46 から、claude.ai のウェブインターフェースで設定した MCP（Model Context Protocol）コネクターを Claude Code でも利用できるようになりました。これにより、claude.ai のコネクター設定を一元管理しつつ、ターミナルの Claude Code でも同じツールや外部サービスへのアクセスが可能になります。設定の重複なく、ウェブと CLI で同一の MCP サーバーを活用できます。

## 基本的な使い方

Claude Code で claude.ai の MCP コネクターを使うには、claude.ai アカウントでログインしている必要があります。

```bash
# 1. Claude Code を claude.ai アカウントでログインして起動
claude

# 2. claude.ai で設定した MCP コネクターが自動的に利用可能になる
# 追加の設定は不要
```

claude.ai でのコネクター設定方法：
1. [claude.ai/settings/connectors](https://claude.ai/settings/connectors) にアクセス
2. 「コネクターを追加」をクリック
3. MCP サーバーの URL を入力して設定

## 実践例

### GitHub コネクターを Claude Code で活用する

claude.ai に GitHub MCP コネクターを設定している場合、Claude Code でそのまま使えます：

```
# claude.ai での設定（事前に行っておく）
設定 > コネクター > GitHub コネクターを追加

# Claude Code での利用（自動的に利用可能）
> GitHub のリポジトリ一覧を表示して
> PR #123 の内容を確認して
> main ブランチの最新コミットを教えて
```

### Slack コネクターとの連携

```
# claude.ai での設定
設定 > コネクター > Slack コネクターを追加

# Claude Code での利用
> #dev-channel に「デプロイ完了」と投稿して
> 今日の #general チャンネルのメッセージをまとめて
```

### カスタム MCP コネクターの設定

Pro/Max プランでは独自の MCP サーバーを接続できます：

```
# claude.ai の設定画面
設定 > コネクター > カスタムコネクターを追加
→ MCP サーバーの URL を入力（例: https://your-mcp-server.example.com）

# Claude Code でカスタムツールが使用可能になる
> /your-custom-tool を使ってデータを取得して
```

### Team/Enterprise 環境での利用

```
# Team/Enterprise プランでは管理者がコネクターを設定
# チームメンバー全員が同じコネクターを Claude Code で利用可能

管理者: claude.ai 管理コンソール > コネクター設定
メンバー: Claude Code で自動的に利用可能
```

## 注意点

- **アカウントログインが必須**：この機能は claude.ai アカウントでログインした状態でのみ利用できます。API キーのみの認証では使用できません。
- **プランによる制限**：カスタムコネクターの追加は Pro/Max プランが必要です。Team/Enterprise プランでは管理者のみが設定できます。
- **セキュリティ考慮**：外部サービスに接続する MCP コネクターはそのサービスへのアクセス権を Claude Code に付与します。信頼できるサービスのみ接続してください。
- **claude.ai との同期**：コネクターは claude.ai 側で設定・管理します。Claude Code 単独でコネクターを追加する機能は提供されていません。
- **表示確認**：Claude Code 内で利用可能なコネクターは、claude.ai から来ているものとして識別子が表示されます。
