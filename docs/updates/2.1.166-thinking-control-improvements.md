## 原文（日本語に翻訳）

`MAX_THINKING_TOKENS=0`、`--thinking disabled`、およびモデルごとの思考トグルが、Claude API 経由でデフォルトで思考を行うモデルに対して思考を無効化できるようになりました（3P プロバイダーは変更なし）。

## 原文（英語）

`MAX_THINKING_TOKENS=0`, `--thinking disabled`, and the per-model thinking toggle now disable thinking on models that think by default via the Claude API (3P providers unchanged)

## 概要

Claude API 経由でデフォルトで Extended Thinking（拡張思考）を行うモデル（Claude 3.7 Sonnet など）に対して、思考機能を完全に無効化できるようになりました。`MAX_THINKING_TOKENS=0` 環境変数、`--thinking disabled` フラグ、および設定画面のモデルごとの思考トグルが、これらのモデルでも有効に機能します。

## 基本的な使い方

環境変数で思考を無効化：

```bash
MAX_THINKING_TOKENS=0 claude "コードをレビューしてください"
```

コマンドラインフラグで無効化：

```bash
claude --thinking disabled "簡単な質問に答えてください"
```

## 実践例

### コスト削減のため思考を無効化

シンプルなタスクでは思考プロセスが不要な場合があります：

```bash
# 単純なファイル操作タスクでは思考を無効化してコストを削減
claude --thinking disabled "このファイルの行数を数えてください"

# 複雑な推論が必要な場合は思考を有効にする（デフォルト）
claude "このアルゴリズムの時間複雑度を分析してください"
```

### CI/CD パイプラインでの高速化

思考プロセスを省略することで応答速度が向上します：

```bash
# CI環境変数として設定
export MAX_THINKING_TOKENS=0

# パイプラインでの使用（思考なしで高速動作）
claude -p "テストが通過しているか確認してください"
```

### 特定のモデルで思考を制御

```bash
# Claude API でデフォルト思考モデルを使いつつ思考を無効化
claude --model claude-sonnet-4-6 --thinking disabled "要約してください"
```

### 設定ファイルでの制御

`settings.json` でモデルごとの思考設定を管理：

```json
{
  "thinkingEnabled": false
}
```

## 注意点

- この変更は **Claude API 直接接続時のみ** 有効です。Amazon Bedrock や Google Vertex AI などの 3P プロバイダー経由では挙動が変わりません
- 思考を無効化すると、複雑な推論問題での精度が低下する場合があります
- `MAX_THINKING_TOKENS=0` は思考を完全に無効化しますが、正の値を設定すると上限を設けます
- 思考が有効なモデルで `--thinking disabled` を使用すると API コストが削減される場合があります
