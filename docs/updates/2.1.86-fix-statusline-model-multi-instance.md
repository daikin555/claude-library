## 原文（日本語に翻訳）

複数のClaude Codeインスタンスを実行中に、一方で `/model` を使用すると別のインスタンスのステータスラインが誤ったモデルを表示する問題を修正しました。

## 原文（英語）

Fixed statusline showing another session's model when running multiple Claude Code instances and using `/model` in one of them

## 概要

複数のClaude Codeインスタンス（ウィンドウ/タブ）を同時に開いている状態で、一方のインスタンスで `/model` コマンドを使ってモデルを切り替えると、他のインスタンスのステータスバーに誤ったモデル名が表示されてしまうバグが修正されました。各インスタンスが独立してモデル状態を管理するようになります。

## 基本的な使い方

複数のClaude Codeウィンドウを開いている状態でモデルを切り替える：

```
# ウィンドウ1でモデルを変更
/model claude-opus-4-5

# ウィンドウ2のステータスラインは変化しない（以前は変わってしまっていた）
```

## 実践例

### ユースケース: 異なるモデルを使い分けて並行作業する

複数のClaude Codeインスタンスで異なるモデルを使用する場合：

```
# ウィンドウ1: 高速な作業にsonnetを使用
/model claude-sonnet-4-5

# ウィンドウ2: 複雑な分析にopusを使用
/model claude-opus-4-5

# 各ウィンドウのステータスラインが正しいモデルを表示するようになった
```

### ユースケース: 複数プロジェクトを並行して開発する

複数のプロジェクトを別々のClaude Codeインスタンスで管理する場合、各インスタンスのモデル設定が独立して維持されます。

## 注意点

- この問題は複数のClaude Codeインスタンスを同時に実行している場合にのみ発生していました
- ステータスラインの表示が誤っていただけで、実際に使用されるモデルは正しかった可能性があります
- 修正後は各インスタンスが独立した状態管理を行います
