## 原文（日本語に翻訳）

`/skills`メニューで推定トークン数によるソートをサポートしました。`t`キーを押してトグルできます。

## 原文（英語）

/skills menu now supports sorting by estimated token count — press t to toggle.

## 概要

Claude Code v2.1.111では、`/skills`メニューに推定トークン数によるソート機能が追加されました。`t`キーを押すことでトークン数順と通常の順序を切り替えることができ、コンテキスト消費量を考慮したスキルの選択が容易になります。

## 基本的な使い方

### スキルメニューを開く

```
# スキル一覧を表示
/skills
```

### トークン数でソートする

```
# /skillsメニュー内でtキーを押す
t
```

### ソートのトグル

```
# 初回: tキー押下 → トークン数の多い順にソート
t

# 再度tキー押下 → 元の順序（デフォルト順）に戻る
t
```

## 実践例

### コンテキスト節約を意識したスキル選択

```
/skills

# tキーでトークン数ソートを有効化
t

# 表示例：
# スキル一覧（トークン数順）:
#   skill-name-a    ~500 tokens
#   skill-name-b    ~1,200 tokens
#   skill-name-c    ~3,500 tokens
#   skill-name-d    ~8,000 tokens
```

### 大量のスキルが登録されている環境での管理

```
# 多数のプラグインがある環境で、軽量スキルを素早く見つける
/skills
t  # トークン数昇順でソート → 軽量スキルが上位に来る
```

### スキルの最適化判断に活用する

```
# トークン数を確認してスキルの削減・統合を検討
/skills
t

# 高トークン消費スキルを特定
# → スキルのSKILL.mdを見直して最適化するかどうか判断
```

### セッションコンテキストの管理

```
# コンテキストが逼迫している場面でスキルを選択する場合
/context  # 現在のコンテキスト使用量を確認

/skills
t  # トークン数の少ないスキルを選択することでコンテキスト消費を抑える
```

## 注意点

- 表示されるトークン数は**推定値**です。実際の消費量はスキルの実行内容によって異なります
- `t`キーはソートをトグルするため、押すたびにトークン数順と元の順序が切り替わります
- トークン数ソートはスキルのSKILL.mdファイルのサイズや内容に基づいて推定されます
- コンテキストウィンドウが限られている場合、トークン数の少ないスキルを選択することでより多くの作業をセッション内で完結できます
- このソート機能は表示のみの変更であり、スキルの実際の動作には影響しません
