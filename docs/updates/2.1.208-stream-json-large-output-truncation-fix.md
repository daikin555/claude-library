## 原文（日本語訳）

`claude -p`から大きなレスポンスをパイプした際に、stream-json／JSON出力が途中で切れてしまい、resultメッセージが欠落する不具合を修正した。

## 原文（英語）

Fixed truncated stream-json/JSON output and missing result message when piping large responses from `claude -p`

## 概要

`claude -p`（headlessモード）で`--output-format json`や`--output-format stream-json`を使い、出力を別コマンドへパイプして処理するケースはCI/CDやスクリプトから非常によく使われます。しかしこれまでは、レスポンスのサイズが大きい場合に出力の最終行が途中で切れてしまい、本来最後に届くはずの`result`メッセージ（最終的なテキスト・コスト・セッションメタデータを含む）が丸ごと欠落することがありました。公式ドキュメントにも「v2.1.208より前は、大きなレスポンスをパイプすると最終行が切り詰められ、resultメッセージが欠落することがあった」と明記されており、今回のアップデートでこの問題が修正されています。

## 基本的な使い方

`claude -p`に`--output-format json`または`--output-format stream-json`を指定し、出力を`jq`などにパイプして利用します。

```bash
claude -p "Summarize this project" --output-format json | jq -r '.result'
```

## 実践例

### CIパイプラインで大きな差分をレビューさせる

PRの差分など出力が大きくなりがちな用途で`--output-format json`をパイプ処理する場合、以前はレスポンスが途中で切れて`result`フィールドが取得できないことがありましたが、修正後は最後まで確実に受け取れます。

```bash
gh pr diff "$1" | claude -p \
  --append-system-prompt "You are a security engineer. Review for vulnerabilities." \
  --output-format json | jq -r '.result'
```

### stream-jsonでリアルタイム出力を処理しつつ最終resultを確実に取得する

`--output-format stream-json`と`--include-partial-messages`を組み合わせてトークンをリアルタイム表示する用途でも、長い応答の最終行が欠けずに`result`メッセージまで届くようになりました。

```bash
claude -p "Explain recursion in detail" \
  --output-format stream-json --verbose --include-partial-messages | \
  jq -rj 'select(.type == "stream_event" and .event.delta.type? == "text_delta") | .event.delta.text'
```

## 注意点

- stream-jsonはNDJSON（改行区切りJSON）形式で、1行が1つのJSONオブジェクトになっているため、`jq`などで処理する際は行単位でパースする必要があります。
- ストリームの最終行は必ず`result`メッセージになり、最終的な応答テキスト・コスト・セッションメタデータが含まれます。
- `claude -p`は標準入力（stdin）についても10MBの上限があり、超過するとエラーで終了します。大きな入力を扱う場合はファイルに書き出してパスを渡す方法が推奨されます。
