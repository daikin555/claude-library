# /copy コマンドに「常にフルレスポンスをコピー」オプションを追加

## 原文（日本語に翻訳）

`/copy` ピッカーに「常にフルレスポンスをコピー」オプションが追加されました。選択すると、以降の `/copy` コマンドはコードブロックピッカーをスキップして、フルレスポンスを直接コピーします。

## 原文（英語）

Added "Always copy full response" option to the `/copy` picker. When selected, future `/copy` commands will skip the code block picker and copy the full response directly.

## 概要

Claude Code v2.1.63から、`/copy` コマンドのピッカーに「Always copy full response（常にフルレスポンスをコピー）」オプションが追加されました。このオプションを選択すると、以降の `/copy` 実行時にコードブロック選択ピッカーが表示されず、Claudeの回答全体が自動的にクリップボードにコピーされます。コードだけでなく説明文も含めてコピーしたい場合や、毎回選択操作を行いたくない場合に便利です。

## 基本的な使い方

### /copy コマンドの使用

```
/copy
```

ピッカーが表示され、以下の選択肢が表示されます：
- 特定のコードブロックを選択してコピー
- **Always copy full response（常にフルレスポンスをコピー）** ← 新機能

### 設定の有効化

```
/copy
→ "Always copy full response" を選択
→ 以降の /copy は自動的にフルレスポンスをコピー
```

## 実践例

### ドキュメント作成時にフルレスポンスを使用

```
# Claudeにドキュメントを生成させ、全文をコピー
ユーザー: このAPIのドキュメントを書いてください

Claude: [説明文 + コードサンプル + 注意事項を含む回答]

/copy
→ "Always copy full response" を選択
→ 説明文・コード・注意事項すべてがコピーされる
```

### コードレビューコメントをそのままコピー

```
ユーザー: このコードをレビューしてください

Claude: [詳細なレビューコメントとコード修正案]

/copy
→ フルレスポンスがコピーされ、GitHub PRにそのまま貼り付け可能
```

### 設定をリセットする場合

```
# 再度 /copy を実行するとピッカーが表示される
# 特定コードブロックの選択に戻すことができる
```

## 注意点

- 「Always copy full response」を設定すると、セッション内で以降の `/copy` コマンドはすべてフルレスポンスをコピーします
- コードブロックのみをコピーしたい場合は、再度 `/copy` を実行してピッカーから選択してください
- コピーされる内容はMarkdown形式です
