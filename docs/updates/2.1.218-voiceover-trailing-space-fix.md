## 原文（日本語に翻訳）

`--ax-screen-reader`モードにおいて、入力欄の末尾でスペースを入力した際にVoiceOverが「改行」と読み上げてしまい、実際に入力されたスペースがエコーされない問題を修正しました

## 原文（英語）

Fixed VoiceOver reading "new line" instead of echoing the typed space at the end of the input in `--ax-screen-reader` mode

## 概要

Claude Code v2.1.218で修正された、アクセシビリティ機能に関するバグです。macOSのVoiceOverと組み合わせて使用する`--ax-screen-reader`モードでは、入力行の末尾でスペースキーを押すと、本来なら入力したスペース文字がそのまま読み上げられるべきところ、誤って「new line（改行）」と読み上げられてしまい、ユーザーが実際に何を入力したのか混乱する原因になっていました。修正後は、末尾のスペースが正しくスペースとして読み上げられるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 影響範囲は`--ax-screen-reader`フラグ使用時、かつ入力行の末尾にスペースを入力した場合に限られる
- VoiceOverなどスクリーンリーダーを利用するユーザーの入力確認精度が向上
- 修正には特別な設定変更は不要
