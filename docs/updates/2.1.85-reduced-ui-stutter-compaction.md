## 原文（日本語に翻訳）

大規模セッションでコンパクションが発生した際のUIのスタッター（ちらつき/引っかかり）を軽減しました。

## 原文（英語）

Reduced UI stutter when compaction triggers on large sessions

## 概要

大規模な会話セッションでコンテキストウィンドウが自動コンパクション（圧縮）をトリガーした際に、UIが一時的に固まったり、スタッター（ちらつきや引っかかり）が生じる問題が改善されました。コンパクション処理がよりスムーズに行われるようになり、ユーザーの操作が中断されにくくなります。

## 基本的な使い方

この改善はバックグラウンドで適用されます。長い会話セッションで自動コンパクションが発生した際でも、UIの操作感が向上します。

コンパクションが発生するタイミング：
- コンテキストウィンドウが満杯に近づいたとき
- `/compact` コマンドを明示的に実行したとき

## 実践例

### ユースケース: 長時間の開発セッション

数時間にわたるセッションで、自動コンパクションが発生してもUIが引っかからずに継続して作業できます。

### ユースケース: 大きなコードベースの分析

多くのファイルを読み込んで分析した後、コンテキストが満杯になってコンパクションが発生しても、UIの応答性が維持されます。

## 注意点

- この改善はパフォーマンスの最適化であり、機能的な変更はありません
- コンパクション中はClaude Codeが応答をサマリーに置き換えるため、一部の会話履歴が要約されます
- 大規模セッション（多くのメッセージやコードを含む）で最も効果が顕著です
