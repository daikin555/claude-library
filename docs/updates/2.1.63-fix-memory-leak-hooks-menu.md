# Hooks設定メニュー操作時のメモリリーク修正

## 原文（日本語に翻訳）

Hooks設定メニューを操作する際のメモリリークを修正しました。

## 原文（英語）

Fixed memory leak when navigating hooks configuration menu

## 概要

Claude Code v2.1.63で、`/hooks` コマンドで開くHooks設定メニューを操作する際に発生するメモリリークが修正されました。メニューをナビゲートするたびにメモリが蓄積する問題があり、特にhooks設定を頻繁に変更するユーザーへの影響がありました。

## 基本的な使い方

この修正はバックグラウンドで自動的に適用されます。ユーザー側での操作は不要です。

## 実践例

### 影響を受けるシナリオ

```
# /hooks メニューを開いて設定を変更する操作
/hooks
→ 設定メニューをスクロール・選択
→ 修正前：ナビゲートのたびにメモリ蓄積
→ 修正後：適切なメモリ解放
```

## 注意点

- v2.1.63にアップデートすることで自動的に修正が適用されます
- Hooks設定を頻繁に変更するユーザーや、長時間のセッションでHooksをカスタマイズするユーザーに特に効果があります
