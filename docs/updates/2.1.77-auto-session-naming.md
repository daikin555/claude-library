## 原文（日本語に翻訳）

プランを承認すると、セッションがプランの内容から自動的に命名されるようになりました

## 原文（英語）

Sessions are now auto-named from plan content when you accept a plan

## 概要

Claude Codeのプランモードでプランを承認すると、そのプランの内容を元にセッションが自動的に命名されるようになりました。これにより、複数のセッションを管理する際に、どのセッションが何のタスクかを一目で把握できるようになります。手動でセッション名を付ける手間が省け、作業の整理が容易になります。

## 基本的な使い方

特別な操作は不要です。プランモードでプランを作成・承認するだけで、自動的にセッション名が設定されます。

```bash
# プランモードを有効にして作業を開始
/plan

# Claudeがプランを提示し、承認すると自動的にセッションが命名される
# 例: "ECサイトのカート機能実装" のようなセッション名が自動生成
```

## 実践例

### 複数の開発タスクを並行管理

```bash
# セッション1: ユーザー認証機能の実装
/plan
"ユーザー認証機能（ログイン・ログアウト・パスワードリセット）を実装してください"
# → プラン承認後、セッション名が自動設定される
# 例: "ユーザー認証機能の実装"

# セッション2: 決済システムの統合
/plan
"Stripeを使った決済システムを統合してください"
# → セッション名: "Stripe決済システム統合"
```

### セッション一覧での確認

```bash
# セッション一覧を表示して命名されたセッションを確認
/sessions

# 自動命名されたセッションが識別しやすい名前で表示される
# - "ユーザー認証機能の実装" (2時間前)
# - "Stripe決済システム統合" (昨日)
# - "データベースマイグレーション" (3日前)
```

### 長期プロジェクトでのセッション管理

大きなプロジェクトでは複数のサブタスクをそれぞれ別セッションで管理することがあります。

```bash
# 各機能ごとにセッションを作成
/plan
"REST APIのエンドポイント設計と実装"
# セッション名: "REST APIエンドポイント設計と実装"

# 別セッションで別機能
/plan
"フロントエンドのコンポーネントライブラリ構築"
# セッション名: "フロントエンドコンポーネントライブラリ構築"
```

## 注意点

- **プラン承認のタイミング**: 命名はプランを「承認」した時点で行われます。プランを却下した場合は命名されません
- **命名の上書き**: 自動命名後に手動でセッション名を変更することも引き続き可能です
- **日本語対応**: プランが日本語で書かれている場合、日本語のセッション名が生成されます
- **既存セッションへの影響**: この機能は新しくプランを承認したセッションにのみ適用され、既存の名前なしセッションは変更されません
