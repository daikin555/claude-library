## 原文（日本語に翻訳）

`AskUserQuestion` のダイアログが、デフォルトでは自動継続しないように変更されました。アイドルタイムアウトを使いたい場合は `/config` から任意で有効にできます。

## 原文（英語）

Changed `AskUserQuestion` dialogs to no longer auto-continue by default; opt into an idle timeout via `/config`

## 概要

`AskUserQuestion` はClaudeがユーザーに選択肢を提示して回答を求めるツールです。従来は一定時間ユーザーからの応答がないと、回答なしのまま自動的に処理を継続する挙動がありましたが、これは特にレビューや承認が必要な場面で意図しない継続につながるとの指摘がありました。今回の変更で、この自動継続はデフォルトで無効化され、Claudeはユーザーが回答するまで待機するようになります。従来のように一定時間で自動継続させたい場合は、`/config` から明示的にアイドルタイムアウトを有効にする必要があります。

## 基本的な使い方

デフォルトの動作を変更する必要はありません。`AskUserQuestion` が表示された質問は、ユーザーが回答するまでそのまま待機します。

自動継続（アイドルタイムアウト）を使いたい場合は、Claude Codeのセッション内で `/config` を実行し、`AskUserQuestion` の待機やアイドルタイムアウトに関する項目を探して有効化します。

```
claude
> /config
```

`/config` 画面内の該当項目からタイムアウト関連の設定をオンにすると、以前のように無応答が続いた場合に自動継続する挙動へ切り替えられます。

## 実践例

### コードレビューや承認フローでの利用

破壊的な操作（本番環境へのデプロイ、大量ファイルの削除など）の可否をAskUserQuestionで確認させている場合、離席中に自動継続されると意図しない実行につながる恐れがありました。デフォルト化された「自動継続しない」挙動により、明示的な回答を待つまでClaudeは待機し続けるため、安全性が高まります。

### 長時間の対話型セッションを安全に離席する

複数の選択肢を提示された状態でPCから離れても、無回答のまま処理が進んでしまう心配がなくなりました。戻ってきてから落ち着いて回答を選べます。

### 自動化・CI的な用途で従来の自動継続を再現したい場合

無人実行に近い形でセッションを流したいケースなど、あえて従来の「一定時間で自動継続」という挙動が必要な場合は、`/config` からアイドルタイムアウトを明示的に有効化することで、以前と同様の動作に戻せます。

## 注意点

- デフォルト挙動の変更であり、既存のワークフローが「回答なしで進んでいた」場合は、今後は回答するまで処理が止まる点に注意してください。
- 自動化パイプラインなどで無応答時の自動継続に依存していた場合は、`/config` でアイドルタイムアウトを明示的に有効化しないと処理が止まったままになる可能性があります。
- `/config` のメニュー項目名や設定粒度は今後のバージョンで変わる可能性があるため、実際の画面表示を確認してください。
