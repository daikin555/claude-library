## 原文（日本語に翻訳）

`claude attach` がセッション遷移中に「job not found」や「agent is still starting」といったエラーで失敗することがある不具合を修正しました。attachはデーモンが安定するのを待つようになり、attach処理が遅延している間に行われたターミナルのリサイズは、処理完了後にまとめて反映されるようになりました

## 原文（英語）

Fixed `claude attach` sometimes failing with "job not found" or "agent is still starting" errors during session transitions — attach now waits for the daemon to settle, and terminal resizes during a slow attach are applied once it completes

## 概要

`claude attach` は、バックグラウンドで動作しているエージェントセッションに端末をアタッチ（接続）するためのコマンドです。しかし、セッションが起動処理からバックグラウンド実行へ遷移する過渡期にattachを実行すると、バックグラウンドデーモン側の状態がまだ安定していないために「job not found」（ジョブが見つからない）や「agent is still starting」（エージェントはまだ起動中）といったエラーで失敗することがありました。Claude Code v2.1.210では、attachがデーモンの状態が落ち着くまで待機してから接続を試みるようになり、さらに接続が遅延している間にターミナルサイズが変更された場合も、attach完了後にそのリサイズが正しく反映されるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
# バックグラウンドエージェントを起動した直後にattachを試みる
claude --bg
# → セッションIDが返る（例: session-abc123）

claude attach session-abc123
# 起動処理がまだ完了しきっていないタイミングでは
# 以下のようなエラーで失敗することがあった
#
# Error: job not found
# あるいは
# Error: agent is still starting
```

### 修正後の動作

```bash
claude --bg
# → セッションIDが返る（例: session-abc123）

claude attach session-abc123
# 修正後: attachはバックグラウンドデーモンの状態が
# 安定するまで内部的に待機してから接続する
# → 「job not found」「agent is still starting」といった
#   一時的なエラーで失敗することがなくなった
```

## 実践例

### /background で切り離した直後にattachし直す

```bash
claude
> 長いタスクを実行中...
> /background
# セッションがバックグラウンドエージェントとして切り離される

# すぐに再アタッチしたい場合
claude attach session-abc123

# 修正前: セッション遷移直後だとエラーになることがあった
# 修正後: デーモンが安定するのを待ってから正しくアタッチされる
```

### 複数セッションを起動して順にattachする運用

```bash
claude --bg  # session-A
claude --bg  # session-B
claude --bg  # session-C

# すべて起動直後に立て続けにattachする
for s in session-A session-B session-C; do
  claude attach "$s"
done

# 修正後: 各セッションの起動タイミングにばらつきがあっても、
# attachはそれぞれ安定を待ってから接続するため
# エラーで失敗する頻度が大幅に減少する
```

### attach待機中にターミナルウィンドウをリサイズする

```bash
claude attach session-abc123
# attachの接続処理に多少時間がかかっている間に
# ターミナルウィンドウの幅・高さを変更した場合

# 修正前: リサイズが正しく反映されず、
#         表示崩れが起きることがあった
# 修正後: attach完了時点で、待機中に発生した
#         リサイズがまとめて反映され、
#         正しいサイズで描画が再開される
```

## 注意点

- この修正はClaude Code v2.1.210（2026年7月15日リリース）で行われました
- `claude attach <session>` は起動中のバックグラウンドエージェントセッションに端末を再接続するためのコマンドです
- バックグラウンドで実行中のセッション一覧は `/tasks` や `claude agents` から確認できます
- セッションが停止（stopped/idle）している場合と起動中（starting）の場合とでattachの挙動が異なる点には引き続き注意してください
- 本修正はタイミング依存の不具合の修正であり、通常の利用フローや既存のコマンド体系に変更はありません
- 大規模なバックグラウンドタスクを多数並行運用している場合、セッション遷移中のattach失敗が減ることで、自動化スクリプトのリトライ処理を簡略化できる可能性があります
