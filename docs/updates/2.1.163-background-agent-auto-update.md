## 原文（日本語に翻訳）

バックグラウンドエージェントセッションが Claude Code の新バージョンへのアップデートをバックグラウンドで行うようになりました。アップデート後にセッションを開いても、コールドリスタートを待つ必要がなくなります。

## 原文（英語）

Background agent sessions now update to a new Claude Code version in the background, so opening a session after an update no longer waits on a cold restart

## 概要

これまで Claude Code のアップデート後にバックグラウンドエージェントセッションを開くと、コールドリスタート（初回起動の初期化処理）が発生して待ち時間が生じていました。今後はアップデートがバックグラウンドで処理されるため、セッションをすぐに開けるようになります。

## 基本的な使い方

特別な操作は不要です。Claude Code がアップデートされた後にバックグラウンドエージェントセッションを開くと、自動的にバックグラウンドでアップデートが適用され、すぐに作業を再開できます。

```bash
# アップデート後もセッションを即座に開ける
claude agents
```

## 実践例

### 長時間動作するバックグラウンドエージェントのワークフロー

1. バックグラウンドエージェントでタスクを開始する
2. Claude Code のアップデートが自動的に行われる
3. `claude agents` でセッションを開くと、コールドリスタートなしですぐに接続できる

従来はアップデート後の初回接続時に数秒から十数秒の待ち時間が発生していましたが、この改善によりシームレスなセッション継続が可能になります。

## 注意点

- バックグラウンドでのアップデートは自動的に行われます。ユーザーが意識して操作する必要はありません。
- アップデート中のセッションは中断されません。バックグラウンドで更新が完了した後、次回の接続時から新バージョンが有効になります。
