# ブリッジポーリングループのリスナーリーク修正

## 原文（日本語に翻訳）

ブリッジポーリングループのリスナーリークを修正しました。

## 原文（英語）

Fixed listener leak in bridge polling loop

## 概要

Claude Code v2.1.63で、ブリッジポーリングループにおけるリスナーリークが修正されました。このバグはClaude CodeがIDEや外部ツールとの通信に使用するブリッジ機能において、イベントリスナーが適切に解放されない問題でした。長時間のセッションでメモリ使用量が増加する原因となっていた可能性があります。

## 基本的な使い方

この修正はバックグラウンドで自動的に適用されます。ユーザー側での操作は不要です。

## 実践例

### 影響を受けるシナリオ

- VS CodeやJetBrains IDEなどとClaude Codeを統合して長時間使用する場合
- Claude CodeのIDE統合機能を使用した長期セッション

### 修正後の改善

- 長時間セッションでのメモリ使用量が安定
- IDE統合の動作がより安定

## 注意点

- v2.1.63にアップデートすることで自動的に修正が適用されます
- この修正は特にIDE統合（VSCode、JetBrains等）を使用するユーザーに恩恵があります
