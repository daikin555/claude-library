# 修正：管理設定がないTeam/Enterprise組織で--channelsがバイパスされる問題

## 原文（日本語に翻訳）

他に管理設定が構成されていない Team/Enterprise 組織での `--channels` バイパスを修正。

## 原文（英語）

Fixed `--channels` bypass for Team/Enterprise orgs with no other managed settings configured

## 概要

Team または Enterprise プランの組織で、他の管理設定が何も構成されていない場合、`--channels` フラグが意図せずバイパスされてしまうバグが修正されました。これにより、`--channels` を使った権限リレー機能が Team/Enterprise 環境でも正しく動作するようになります。

## 基本的な使い方

修正後は、Team/Enterprise 組織での `--channels` が正しく機能します：

```bash
# Team/Enterprise 環境で channels を利用
claude --channels https://your-channel-server.example.com
```

管理設定（managed settings）が他に何も設定されていない状態でも、チャンネルサーバーへの接続が維持されます。

## 注意点

- この修正は `--channels` 権限リレー機能（v2.1.81 で同時追加）と組み合わせて使用することを想定
- Team/Enterprise 管理者がチャンネルサーバーを設定していない場合は引き続き利用不可
- 個人プランでは影響なし
