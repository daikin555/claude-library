## 原文（日本語に翻訳）

終了したプロセスの子プロセスが出力パイプを保持し続ける場合に、Windows で PowerShell コマンド検証が時間予算を大幅に超えてハングすることがあったバグを修正しました。

## 原文（英語）

Fixed PowerShell command validation occasionally hanging far past its time budget on Windows when a killed process's children held its output pipes

## 概要

Windows 環境で PowerShell コマンドの検証（安全性チェック）が規定の時間を大幅に超えてフリーズするバグを修正しました。終了したプロセスの子プロセスが出力パイプを保持し続ける場合に発生していた問題で、この修正により Windows での Claude Code の応答性が向上します。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```powershell
# Windows の PowerShell でアップデート
npm update -g @anthropic-ai/claude-code
```

## 実践例

### 修正前の問題

Windows の PowerShell ターミナルで以下の状況が発生していました：
- コマンドを入力後、Claude が応答するまで異常に長い待機時間が発生
- バックグラウンドプロセスを含むコマンドの実行後にハングが起きやすかった
- タスクマネージャーで確認すると、子プロセスがパイプを保持したまま残っていた

### 修正後の動作

v2.1.166 以降では：
- PowerShell コマンド検証が適切なタイムアウト内に完了
- ハングが解消され、Claude Code がスムーズに動作
- 子プロセスが適切にクリーンアップされる

```powershell
# 修正後はこのようなコマンドもスムーズに処理される
claude "このPowerShellスクリプトを実行してください"
```

## 注意点

- この修正は Windows 環境の PowerShell に限定されます
- macOS・Linux 環境のシェルには影響しません
- `cmd.exe` を使用している場合は影響しません（PowerShell 固有の問題でした）
