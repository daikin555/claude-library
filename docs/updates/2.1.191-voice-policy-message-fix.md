## 原文（日本語に翻訳）

`/voice` が組織のポリシーにより無効化されている場合に汎用的な「利用不可」メッセージを表示していた問題を修正 — 制限内容を具体的に説明するようになりました

## 原文（英語）

Fixed `/voice` showing a generic "not available" message when disabled by an organization's policy — it now explains the restriction

## 概要

組織の管理者が `/voice`（音声入力機能）を無効化している場合、以前は原因不明の汎用的なエラーメッセージが表示されていました。この修正により、組織のポリシーによって制限されている旨が明確に表示され、ユーザーが管理者に問い合わせるべき状況を正確に把握できるようになりました。

## 基本的な使い方

```bash
# /voice を実行
/voice

# 改善前のメッセージ（原因不明）
# ❌ 音声入力は利用できません

# 改善後のメッセージ（原因が明確）
# ℹ️ 音声入力はお使いの組織のポリシーにより無効化されています
#    管理者にお問い合わせください
```

## 実践例

### 企業環境でのトラブルシューティング

```
# 状況: 企業のAnthropic管理コンソールで音声機能が無効化されている

# ユーザーが /voice を試みると
/voice
> 音声入力はお使いの組織のポリシーにより無効化されています

# 対応: ITまたはAIツール管理者に連絡して
# ポリシーの変更を依頼するか、代替手段を確認する
```

### 管理者向けの確認

```
# Anthropic管理コンソールで確認すべき設定:
# - 組織レベルの機能制限
# - 音声入力の有効/無効フラグ
# - ユーザーグループ別の権限設定
```

## 注意点

- このメッセージが表示された場合、個人レベルの設定では解決できません。組織の管理者に連絡する必要があります。
- 個人のClaudeアカウント（組織アカウントでない場合）では、組織ポリシーによる制限はありません。
- 音声機能の有効化は組織の契約プランや地域によっても制限される場合があります。
