## 原文（日本語に翻訳）

`C:\Users\unicorn` のように `\u` から始まるセグメントを含むWindowsパスが、ツールの入力においてCJK文字に化けてしまい、該当ファイルにアクセスできなくなる問題を修正しました。

## 原文（英語）

Fixed Windows paths with `\u`-prefixed segments (like `C:\Users\unicorn`) being corrupted into CJK characters in tool inputs, which made those files inaccessible

## 概要

Windowsのパス区切り文字はバックスラッシュ（`\`）であるため、`C:\Users\unicorn` のようなパスには `\U`（`Users`）や `\u`（`unicorn` など小文字始まりのフォルダ名）といった「バックスラッシュ + u」の並びが含まれることがあります。この並びが、JavaScript/JSONにおけるUnicodeエスケープシーケンス（`\uXXXX`）として誤って解釈されてしまい、パス文字列がCJK（中国語・日本語・韓国語）文字などの意図しない文字に化けて壊れてしまうバグがありました。この結果、該当するパスのファイルにClaude Codeのツール（Read、Editなど）からアクセスできなくなっていました。v2.1.218ではこの誤変換を修正し、`\u` を含むWindowsパスが正しくツールに渡されるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 影響を受けていたのは主に、フォルダ名やユーザー名が `u` で始まる（`Users`、`unicorn` など）Windows環境のパスです。
- 症状としては、該当パスのファイルを読み書きしようとするとエラーになったり、実際には存在しない文字化けしたパスにアクセスしようとして失敗したりする形で現れていました。
- macOSやLinuxなど、バックスラッシュをパス区切り文字として使わない環境ではこの問題は発生しません。
