## 原文（日本語に翻訳）

`/plugin` の閲覧・発見ペインにプラグインの最終更新日時を表示するようになりました

## 原文（英語）

The `/plugin` browse and discover panes now show when a plugin was last updated

## 概要

Claude Code v2.1.144で、`/plugin` コマンドの閲覧（browse）および発見（discover）ペインにプラグインの最終更新日時が表示されるようになりました。これにより、プラグインが積極的にメンテナンスされているかどうかをインストール前に確認できます。最新のClaude Codeバージョンとの互換性や信頼性を判断する重要な指標として活用できます。

## 基本的な使い方

`/plugin` コマンドを実行すると、閲覧・発見画面の各プラグインに最終更新日時が表示されます。

```
/plugin
```

表示例：

```
プラグイン一覧:

  github-tools
  GitHubのIssue・PRを操作するツール群
  最終更新: 3日前

  database-helper
  データベーススキーマ管理を支援するプラグイン
  最終更新: 2週間前

  legacy-formatter
  コードフォーマッタープラグイン
  最終更新: 8ヶ月前
```

## 実践例

### アクティブにメンテナンスされているプラグインを選ぶ

複数の類似プラグインから選ぶ際、最終更新日時でアクティブなものを選択できます。

```
/plugin discover

検索: "test runner"

結果:
  test-runner-pro
  高機能なテスト実行支援プラグイン
  最終更新: 1週間前         ← 最近更新されている

  simple-test
  シンプルなテスト支援プラグイン
  最終更新: 14ヶ月前        ← 更新が止まっている可能性

# → test-runner-proを選択する判断材料になる
```

### インストール済みプラグインのメンテナンス状況を確認する

閲覧（browse）ペインで既存プラグインの更新状況を定期的にチェックできます。

```
/plugin browse

インストール済みプラグイン:

  my-workflow-helper
  最終更新: 2日前    ← 活発にメンテナンス中

  old-tool
  最終更新: 11ヶ月前 ← 代替プラグインを検討する時期かもしれない
```

### Claude Codeのメジャーアップデート後にプラグインの互換性を判断する

Claude Codeがメジャーアップデートされた後、プラグインが新バージョンに対応しているか最終更新日で判断できます。

```
# Claude Code 2.1.x 系列のリリース後
/plugin browse

  api-connector
  最終更新: 昨日       ← 新バージョンへの対応済みの可能性が高い

  data-processor
  最終更新: 6ヶ月前    ← 新バージョン非対応の可能性あり、注意が必要
```

## 注意点

- 最終更新日時はプラグインの作者がリポジトリを更新した日時を基準としています
- 更新日が古くても安定して動作するプラグインもあります。更新日時はあくまで参考情報です
- この変更は閲覧（browse）と発見（discover）の両方のペインに適用されます
- 表示形式は相対時間（「3日前」「2週間前」など）で表示されます
