## 原文（日本語に翻訳）

バックグラウンドエージェントやワークフローがまだ実行中の場合、レスポンス後タイマーが「N 個のバックグラウンドエージェント/ワークフローの終了を待機中」と表示するようになりました。結果が処理されると累積時間が報告されます。

## 原文（英語）

The post-response timer now shows "Waiting for N background agents/workflows to finish" when backgrounded agents or workflows are still running, and reports the cumulative time once their results are processed

## 概要

メインの応答が完了した後もバックグラウンドでエージェントやワークフローが実行されている場合、プロンプト後のタイマーに「N 個のバックグラウンドエージェント/ワークフローの終了を待機中」というメッセージが表示されるようになりました。バックグラウンド処理が完了すると、累積処理時間が報告されます。これによりバックグラウンド処理の状況が把握しやすくなります。

## 基本的な使い方

```
# バックグラウンドエージェントを含むタスクを実行
→ メインの応答後に以下が表示される：
  "Waiting for 3 background agents to finish..."

# バックグラウンド処理完了後
→ "Completed in 12.3s (including 3 background agents)"
```

## 実践例

### 並列エージェントタスクの完了確認

複数のエージェントを並列実行した後、すべての完了を確認する例です。

```
# 複数エージェントを起動するタスク
"コードベース全体をレビューして問題を報告してください"

→ メイン応答（コーディネーション情報）が表示
→ "Waiting for 5 background agents to finish..."

# しばらく後
→ "All background agents completed. Total: 45.2s"
→ 各エージェントの結果が統合されて表示
```

### ワークフロー完了の待機

長時間のワークフロー実行中の状況を把握する例です。

```
# ワークフローを起動
/run-tests

→ 初期応答後
→ "Waiting for 1 background workflow to finish..."
→ テスト完了後に累積時間とともに結果が表示
```

## 注意点

- このタイマー表示はバックグラウンドエージェント・ワークフローがある場合にのみ表示されます
- 累積時間はバックグラウンドエージェントの実行時間を含む全体の処理時間を示します
- バックグラウンドエージェントの結果が処理された後に次のプロンプトを入力できます
