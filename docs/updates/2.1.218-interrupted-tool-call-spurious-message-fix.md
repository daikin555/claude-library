## 原文（日本語に翻訳）

ツール呼び出しが中断された後に表示される誤った「[Request interrupted by user]」メッセージと、応答の途中でツールが中断された際にトランスクリプトに残る対応のない`tool_use`ブロックを修正しました

## 原文（英語）

Fixed spurious "[Request interrupted by user]" messages after interrupted tool calls, and an unpaired `tool_use` block left in the transcript when a tool aborted mid-response

## 概要

Claude Code v2.1.218で修正された、ツール呼び出し中断時のトランスクリプト表示に関するバグです。以前は、実行中のツール呼び出しを中断（Escキーなど）した際に、実際には該当しない場合でも「[Request interrupted by user]」という紛らわしいメッセージが表示されることがありました。また、応答生成の途中でツールが中断されると、対応する`tool_result`が存在しない`tool_use`ブロックがトランスクリプトに残ってしまい、会話履歴の整合性が崩れることがありました。修正後は、これらの表示が正しく制御され、トランスクリプトが常に正確でクリーンな状態に保たれるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- ツール実行中に中断操作を行った場合の表示・記録の正確性が向上
- トランスクリプトの整合性が保たれるため、セッション再開時などの不具合も減少すると見込まれる
- ユーザー側で特別な設定変更は不要
