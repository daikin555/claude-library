## 原文（日本語に翻訳）

`/feedback`が利用不可な際に、スラッシュメニューから消えるのではなく、理由を説明するようになりました。

## 原文（英語）

`/feedback` now explains why it's unavailable instead of disappearing from the slash menu

## 概要

これまで`/feedback`コマンドが利用できない状況（ネットワーク未接続時や特定の環境設定など）では、スラッシュコマンドメニューから完全に消えていました。ユーザーはなぜコマンドが使えないのか分からず混乱することがありました。今回の改善により、`/feedback`が利用不可の場合でもメニューに表示され、なぜ使えないのかの理由が説明されます。

## 基本的な使い方

`/feedback`コマンドはClaude Codeへのフィードバックを送るために使用します：

```
> /feedback
```

スラッシュコマンドメニューを開くには、入力欄で`/`を入力します：

```
> /
# メニューが表示され、/feedbackを含むすべてのコマンドが一覧表示される
```

## 実践例

### 利用可能な場合

```
> /feedback
# フィードバック入力フォームが表示される
```

### 利用不可の場合（修正後）

```
> /feedback
# 以前: コマンドがメニューに表示されない（何が起きているか不明）
# 修正後: 「フィードバック機能は現在利用できません: [理由]」のような説明が表示される
```

## 注意点

- フィードバックの送信にはインターネット接続が必要です
- 企業プロキシや特定のネットワーク環境では利用できない場合があります
- フィードバック機能が利用できない場合は、[GitHubイシュー](https://github.com/anthropics/claude-code/issues)からも報告できます
- この改善はユーザーエクスペリエンスのみに関わるもので、機能の追加・削除はありません
