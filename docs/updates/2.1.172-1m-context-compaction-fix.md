## 原文（日本語に翻訳）

使用クレジットなしで1Mコンテキストを使用しているセッションが永久にスタックする問題を修正しました。セッションが標準コンテキスト制限内に収まるよう自動コンパクションが行われるようになりました。

## 原文（英語）

Fixed sessions using 1M context without usage credits getting permanently stuck — the session now automatically compacts back under the standard context limit

## 概要

1Mコンテキストモードを利用中、使用クレジットが不足した場合にセッションが完全に応答しなくなる（永久スタック）問題が修正されました。今後は自動コンパクションが実行され、標準のコンテキスト制限内に収まるよう調整されてセッションが継続できるようになります。

## 基本的な使い方

この修正は自動的に適用されます。1Mコンテキストセッションがクレジット不足で制限に達した場合：

**修正前の挙動**: セッションが完全にスタックし、操作不能になる
**修正後の挙動**: 自動コンパクションが実行され、セッションが継続できる状態に戻る

## 実践例

### 長時間の作業セッションでの影響

```
# 長大なコンテキストを持つセッションで作業中
claude
> 非常に大きなコードベースの分析を継続中...
> コンテキストが1M近くに達した場合でも...
> 自動コンパクションにより作業が継続できる
```

### コンパクションの確認

```
/status
# コンテキスト使用量と自動コンパクション状況を確認できる
```

## 注意点

- コンパクションが実行されると、会話の初期部分が要約されて圧縮されます
- 圧縮された部分の詳細は参照できなくなる場合があります
- 1Mコンテキストを継続利用するには適切な使用クレジットが必要です
- `/compact` コマンドで手動コンパクションも実行できます
