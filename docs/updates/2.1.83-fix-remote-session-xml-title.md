## 原文（日本語に翻訳）

バックグラウンドエージェントが最初のユーザーメッセージの前に完了した場合に、リモートコントロールのセッションタイトルに生の XML が表示される問題を修正しました。

## 原文（英語）

Fixed Remote Control session titles showing raw XML when a background agent completed before the first user message

## 概要

Remote Control セッションでバックグラウンドエージェントが最初のユーザーメッセージが送信される前に処理を完了した場合、セッションタイトルに生の XML データ（`<title>...</title>` など）がそのまま表示されてしまう問題が修正されました。これにより、セッション一覧が正しいタイトルで表示されるようになります。

## 基本的な使い方

この修正は自動的に適用されます。セッションタイトルが正しく表示されるようになりました。

```
# Remote Control セッション一覧（修正後）
[ コード分析タスク         ] ← 正しいタイトル
[ データ処理セッション     ] ← 正しいタイトル

# 修正前の誤表示例
[ <title>コード分析タスク</title>タスク ] ← 生の XML が混入
```

## 実践例

### ユースケース: 素早く完了するエージェントタスク

設定確認や短いファイル読み取りなど、素早く完了するエージェントタスクでも正しいタイトルが表示されます。

```bash
# Remote Control でのバックグラウンドエージェント起動
$ claude --channels
> 設定ファイルを確認してサマリーを作成してください
# エージェントが素早く完了しても
# セッションタイトルに生の XML が表示されない
```

### ユースケース: セッション履歴の管理

セッション履歴を後から参照する際、クリーンなタイトルで管理できます。

```
セッション履歴（修正後）:
2026-03-20 設定ファイル確認  ← クリーンなタイトル
2026-03-19 コードレビュー
2026-03-18 デプロイ作業
```

## 注意点

- この問題はバックグラウンドエージェントが最初のユーザーメッセージ「前」に完了した特殊なタイミングでのみ発生していました
- 生の XML が表示されていた場合は、次のセッションではクリーンなタイトルに修正されます
- セッションタイトルの生成ロジック自体は変更なく、表示のサニタイズが修正されました
