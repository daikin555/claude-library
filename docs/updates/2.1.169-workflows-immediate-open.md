# /workflows コマンド：ターン実行中でも即時表示に改善

## 原文（日本語に翻訳）

`/workflows` コマンドが、ターンの進行中でも即座に開くようになりました

## 原文（英語）

`/workflows` now opens immediately even while a turn is in progress

## 概要

`/workflows` コマンドが、Claudeがレスポンスを生成している最中でも即座に開けるようになりました。これまではターンの完了を待つ必要がありましたが、今後は実行中のワークフローの進捗をリアルタイムで確認できます。長時間実行するワークフローを監視しながら、次のアクションを計画する際に特に便利です。

## 基本的な使い方

### ワークフロー一覧を開く

Claude Codeのセッション中、いつでも以下のコマンドを実行できます（ターン実行中でも可）：

```
/workflows
```

### ワークフロー実行中の確認

マルチエージェントワークフローを起動した後、進捗を確認するために即座に `/workflows` を開けます。

## 実践例

### 長時間ワークフローの進捗監視

大規模なコードレビューや分析ワークフローを開始した直後に進捗を確認する：

```
# ワークフローを起動
> 全コンポーネントのセキュリティ監査を実行してください

# Claudeがワークフローを開始したらすぐに進捗確認（レスポンス生成中でも）
/workflows
```

ターンが完了するのを待たずに、並行して何個のサブエージェントが動いているかをリアルタイムで把握できます。

### 複数ワークフローの並行管理

```
# 1つ目のワークフロー起動
> バックエンドのテストスイートを実行してください

# 即座に /workflows で確認しながら次の指示を考える
/workflows

# 必要に応じて2つ目のワークフローを追加指示
```

### ワークフローのエラー早期発見

エラーが発生しているワークフローを素早く検知するために、処理中でも `/workflows` を確認できます。

## 注意点

- `/workflows` はワークフロー（`Workflow` ツールを使用した処理）の一覧を表示します
- バックグラウンドエージェント（`claude agents`）とは別のビューです
- ワークフローのステータス、フェーズ、完了したサブエージェント数などが確認できます
- 表示後に `q` または `Esc` でコマンドラインに戻れます
