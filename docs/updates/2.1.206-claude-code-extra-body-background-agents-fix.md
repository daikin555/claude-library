## 原文（日本語訳）

`claude agents` や `--bg` によるバックグラウンドワーカーで `CLAUDE_CODE_EXTRA_BODY` が黙って無視されていた不具合を修正した。シェルでエクスポートされたオーバーライドは、ディスパッチ元のセッションに追従するようになった。

## 原文（英語）

Fixed `CLAUDE_CODE_EXTRA_BODY` being silently ignored by `claude agents` / `--bg` background workers; the shell-exported override now follows the dispatching session

## 概要

`CLAUDE_CODE_EXTRA_BODY` は、APIリクエストボディに任意のJSONをマージできる環境変数で、プロバイダー固有のパラメータ渡しなどに使われます。これまで `claude agents` コマンドや `--bg` フラグで起動するバックグラウンドワーカーでは、シェルでエクスポートしたこの環境変数が反映されず、静かに無視されてしまう不具合がありました。今回の修正で、ディスパッチ元セッションのシェル環境で設定した `CLAUDE_CODE_EXTRA_BODY` が、バックグラウンドワーカーにも正しく引き継がれるようになりました。

## 基本的な使い方

シェルで `CLAUDE_CODE_EXTRA_BODY` をエクスポートしてからバックグラウンドエージェントを起動します。

```bash
export CLAUDE_CODE_EXTRA_BODY='{"output_config": {"effort": "high"}}'
claude --bg "長時間かかるリファクタリングタスクを実行して"
```

## 実践例

### バックグラウンドワーカーでプロバイダー固有パラメータを維持したい場合

特定のプロバイダー向けの追加パラメータ（例: 出力設定やベータ機能フラグ）を常に付与したい場合、これまでは `--bg` 実行時にその設定が失われていましたが、修正後はディスパッチ元セッションの設定がそのままバックグラウンドワーカーに引き継がれます。

```bash
export CLAUDE_CODE_EXTRA_BODY='{"metadata": {"team": "platform"}}'
claude --bg "テストスイート全体を実行して失敗箇所を修正して"
```

### `claude agents` で複数エージェントを並列実行する場合

複数のバックグラウンドエージェントを `claude agents` 経由でディスパッチする際も、シェルにエクスポートした `CLAUDE_CODE_EXTRA_BODY` が各ワーカーセッションに一貫して適用されるようになったため、エージェントごとに設定をコピーし直す必要がありません。

```bash
export CLAUDE_CODE_EXTRA_BODY='{"output_config": {"effort": "medium"}}'
claude agents run my-routine
```

## 注意点

- `CLAUDE_CODE_EXTRA_BODY` はAPIリクエストボディにJSONをマージする強力な機能のため、指定するJSONの内容によっては意図しない挙動を招く可能性があります。設定内容は慎重に確認してください。
- 本修正はシェルで直接エクスポートされたオーバーライドが「ディスパッチ元セッションに追従する」というものであり、設定ファイル経由の指定方法など詳細な適用範囲は公式ドキュメントで今後明記される可能性があります。
- バックグラウンド実行（`--bg`）や `claude agents` の挙動は環境ごとに異なる場合があるため、実際に反映されているかは出力やログで確認することを推奨します。
