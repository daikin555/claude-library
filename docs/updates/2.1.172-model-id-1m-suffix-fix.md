## 原文（日本語に翻訳）

`ANTHROPIC_DEFAULT_OPUS_MODEL` にすでに1Mサフィックスが含まれている場合、モデルIDに1Mコンテキストサフィックスが二重に付く（例: `[1M][1m]`）問題を修正しました

## 原文（英語）

Fixed model IDs getting a doubled 1M-context suffix (e.g. `[1M][1m]`) when `ANTHROPIC_DEFAULT_OPUS_MODEL` already includes one

## 概要

環境変数 `ANTHROPIC_DEFAULT_OPUS_MODEL` に1Mコンテキストを示すサフィックスを含むモデルID（例: `claude-opus-4-8[1m]`）を設定した場合、Claude Codeがさらにサフィックスを追加して `claude-opus-4-8[1m][1M]` のような二重サフィックスになる問題が修正されました。

## 基本的な使い方

```bash
# 修正前: この設定が問題を引き起こしていた
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-8[1m]"
# 内部でさらにサフィックスが付加され "claude-opus-4-8[1m][1M]" になっていた

# 修正後: 設定したモデルIDがそのまま正しく使用される
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-8[1m]"
# "claude-opus-4-8[1m]" として正しく処理される
```

## 実践例

### 環境変数でのモデル設定

```bash
# .env または shell設定ファイル
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-8[1m]"

# Claude Codeを起動
claude
/status
# 修正後: モデルが "claude-opus-4-8[1m]" と正しく表示される
# 修正前: "claude-opus-4-8[1m][1M]" と二重サフィックスで表示されていた
```

## 注意点

- `ANTHROPIC_DEFAULT_OPUS_MODEL` 環境変数は、Opusモデルをデフォルトとして使用する際のカスタマイズに使用します
- 1Mコンテキストを使用する場合は `[1m]` または `[1M]` のいずれか一方のみをモデルIDに付加してください
- 大文字小文字は区別されないため `[1m]` と `[1M]` は同じ意味です
