# Opus 4.6がMaxおよびTeamサブスクライバーでデフォルトmediumエフォートに変更

## 原文（日本語に翻訳）

Opus 4.6はMaxおよびTeamサブスクライバーに対してデフォルトでmediumエフォートを使用するようになりました。Mediumエフォートはほとんどのタスクで十分に機能します。速度と精度のバランスが取れたスイートスポットです。`/model`コマンドでいつでも変更できます。

## 原文（英語）

Opus 4.6 now defaults to medium effort for Max and Team subscribers. Medium effort works well for most tasks — it's the sweet spot between speed and thoroughness. You can change this anytime with `/model`

## 概要

Claude Code v2.1.68より、MaxおよびTeamサブスクライバー向けにOpus 4.6のデフォルトエフォートレベルがmediumに変更されました。Opus 4.6のアダプティブ思考機能により、タスクの複雑さに応じて推論の深さが自動調整され、コストと速度のバランスが最適化されます。多くの日常的なタスクにはmediumエフォートで十分であり、必要に応じていつでも変更可能です。

## 基本的な使い方

### エフォートレベルの確認と変更

`/model` コマンドでモデルとエフォートレベルを確認・変更できます。

```
/model
```

モデル選択画面が表示されたら、左右の矢印キーでエフォートレベルを調整します。

### 起動時にエフォートレベルを指定

```bash
# Claude Codeを起動（デフォルトのmediumエフォートが適用される）
claude

# 明示的にhighエフォートで起動
claude --model claude-opus-4-6
# → /modelで起動後にエフォートを変更
```

## 実践例

### エフォートレベルの使い分け

#### シンプルなタスク（mediumエフォートで十分）

```
# コードのフォーマット、コメント追加、簡単なバグ修正など
「このファイルのインデントを修正してください」
「このコードにJSDocコメントを追加してください」
```

Mediumエフォートは速度と精度のバランスが良く、ほとんどの日常タスクで適切な品質を発揮します。

#### 複雑なタスク（highエフォートを推奨）

```
# アーキテクチャ設計、難しいデバッグ、パフォーマンス最適化など
「このシステム全体のスケーラビリティを分析して改善案を提案してください」
「このメモリリークの根本原因を調査してください」
```

highエフォートでは、より深い推論と分析が行われます。

#### 最高難度のタスク（maxエフォートまたはultrathink）

```
# 最高レベルの推論が必要なタスク
「このアルゴリズムのO(n²)をO(n log n)に最適化する方法を考えてください」
```

### エフォートレベルの対応表

| エフォートレベル | 特徴 | 適したタスク |
|---|---|---|
| `low` | 最速・最低コスト | 定型的な作業、簡単な質問 |
| `medium`（デフォルト） | 速度と精度のバランス | ほとんどの日常タスク |
| `high` | 高品質な出力 | 複雑なコーディング、分析 |
| `max` | 最高の性能 | 最難関の問題 |

## 注意点

- **デフォルト変更の影響**: 以前はhighエフォートがデフォルトでしたが、v2.1.68からmediumに変更されました。既存のワークフローで品質が変化する場合は、`/model`でhighに切り替えてください。
- **コスト最適化**: mediumエフォートはhighよりも高速でコスト効率が高く、大多数のタスクに対して十分な品質を提供します。
- **いつでも変更可能**: セッション中に `/model` コマンドを使えば、エフォートレベルをリアルタイムで変更できます。
- **Pro利用者への注意**: Proプランユーザーは追加利用を有効にした場合のみOpusモデルを使用できます。
