## 原文（日本語訳）

`/reload-plugins` およびバックグラウンドプラグイン自動更新が、すでに追加済みのマーケットプレイスから欠落しているプラグインの依存関係を自動インストールするようになりました。

## 原文（英語）

`/reload-plugins` and background plugin auto-update now auto-install missing plugin dependencies from marketplaces you've already added

## 概要

プラグインシステムが強化され、`/reload-plugins` コマンドとバックグラウンドの自動更新処理が、欠落している依存関係（他のプラグインが前提とするプラグイン）を自動的にインストールするようになりました。すでにマーケットプレイスを追加済みの環境であれば、手動で依存プラグインを探してインストールする手間が省けます。

## 基本的な使い方

```bash
# プラグインを手動でリロード（欠落依存関係が自動インストールされる）
/reload-plugins
```

バックグラウンド自動更新は定期的に実行されるため、通常は特別な操作は不要です。

## 実践例

### 依存関係のあるプラグインのインストール後

ある機能プラグインが別のプラグイン（ユーティリティプラグインなど）を必要とする場合、従来は手動でその依存プラグインもインストールする必要がありました。

```bash
# プラグインを追加した後
/reload-plugins
# → 依存関係として必要なプラグインが自動的にインストールされる
# → "Installing missing dependency: [plugin-name]" などのメッセージが表示される
```

### バックグラウンド自動更新の活用

Claude Code を起動したままにしておくと、バックグラウンドで定期的にプラグインの更新チェックが行われます。

```bash
# 特別な操作不要 — バックグラウンドで自動的に処理される
# 欠落している依存関係がある場合は自動インストールされる
```

### マーケットプレイスを複数追加している環境

```bash
# /plugin コマンドでマーケットプレイスを確認
/plugin
# → 追加済みのマーケットプレイスから依存関係を解決してインストール
```

## 注意点

- 自動インストールは**すでに追加済みのマーケットプレイス**からのみ行われます
- 未追加のマーケットプレイスの依存関係は自動インストールされません
- ネットワーク接続が必要です
- 自動インストールされたプラグインは `/plugin` の Installed タブで確認できます
- インストールに失敗した場合は、手動でマーケットプレイスを確認してください
