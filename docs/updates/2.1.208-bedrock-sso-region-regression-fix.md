## 原文（日本語訳）

`sso_region` がBedrockのリージョンと異なるAWS SSOプロファイルで、Bedrock認証が「Session token not found or invalid」エラーで失敗する不具合を修正した（2.1.207で発生したリグレッション）。

## 原文（英語）

Fixed Bedrock auth failing with "Session token not found or invalid" for AWS SSO profiles whose sso_region differs from the Bedrock region (2.1.207 regression)

## 概要

Claude CodeをAmazon Bedrock経由で利用する際、AWS SSOプロファイルの `sso_region`（IAM Identity Centerが動作するリージョン）と、実際にBedrockを呼び出すリージョンが異なる構成は珍しくありません。2.1.207で、Bedrockのリージョンがプロファイルの `sso_region` を上書きしてしまうリグレッションが混入し、IAM Identity Centerインスタンスが別リージョンにあるプロファイルで `Session token not found or invalid` という認証エラーが発生していました。2.1.208でこの不具合が修正され、Claude Codeは再びプロファイルの `sso_region` が指すIAM Identity Centerリージョンにロール認証情報を要求するようになりました。

## 基本的な使い方

AWS SSOプロファイルでログインしてからBedrock経由でClaude Codeを起動します。

```bash
aws sso login --profile=my-bedrock-profile
export AWS_PROFILE=my-bedrock-profile
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION=us-east-1
claude
```

Claude CodeはAWS SDKの既定の認証情報チェーンを利用してAWSクレデンシャルを解決します。

## 実践例

### IAM Identity CenterとBedrockのリージョンが異なる構成

組織のIAM Identity Centerが `us-east-1` にあり、Bedrockで使いたいモデルは `ap-northeast-1` にデプロイされている、といった構成はよくあります。`~/.aws/config` のプロファイルは次のように書けます。

```ini
[profile my-bedrock-profile]
sso_start_url = https://my-sso-portal.awsapps.com/start
sso_region = us-east-1
sso_account_id = 123456789012
sso_role_name = BedrockUser
region = ap-northeast-1
```

2.1.207ではこの構成でBedrock呼び出しのたびに `Session token not found or invalid` エラーになっていましたが、2.1.208では正しく `us-east-1` のIAM Identity Centerに認証情報がリクエストされ、Bedrock自体の呼び出しは `AWS_REGION`（未設定ならプロファイルの `region`、それもなければ `us-east-1`）で解決されたリージョンに送られます。

### 組織全体でBedrockをロールアウトしている場合

複数チームがそれぞれ異なるBedrockリージョンのモデルを使いつつ、SSOポータル自体は1つのリージョンに固定されているような全社展開でも、この修正により各プロファイルの `sso_region` がそのまま尊重されるため、リージョンをまたぐ構成でも認証エラーに悩まされにくくなります。

## 注意点

- 本修正は2.1.207で発生したリグレッションの修正です。2.1.206以前からアップグレードせずに使っていた場合は、そもそもこの問題は発生していません。
- `sso_region` はIAM Identity Centerへの認証にのみ使われるリージョンで、Bedrock自体を呼び出すリージョンとは別物です。Bedrock呼び出し先のリージョンは引き続き `AWS_REGION` → `AWS_DEFAULT_REGION` → プロファイルの `region` → `us-east-1` の優先順で解決されます。
- `/status` を実行すると、実際に解決されたBedrockのリージョンとその情報源を確認できます。
