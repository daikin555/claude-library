## 原文（日本語訳）

agent viewを改善。既存のプルリクエストを編集・マージ・コメント・pushしたセッションも、`claude agents` 上でそのPRへのリンクを表示するようになった。

## 原文（英語）

Improved agent view: sessions that edit, merge, comment on, or push to an existing PR now link it in `claude agents`

## 概要

`claude agents` で開くagent view（エージェントビュー）では、各行の右端に `#1234` のようなラベルが表示され、対応するプルリクエストへのハイパーリンクとして機能します。これまでは、バックグラウンドセッションがworktree内で変更を完結させ、自らPRを新規オープンした場合にのみこのラベルが表示されていました。今回の改善により、セッションが**既存のPR**（そのセッション自身が開いたものではなく、すでに存在するPR）に対して編集・マージ・コメント・pushといった操作を行った場合にも、同様にそのPRへのリンクが行に表示されるようになりました。これにより、レビュー対応やPRへの追加修正など、新規PR作成を伴わないタスクの進捗もagent view上で一目で追えるようになります。

## 基本的な使い方

`claude agents` を実行してagent viewを開き、既存PRに対する作業をバックグラウンドセッションに依頼します。

```text
claude agents
```

```text
> address review comments on PR #1234 and push the fixes
```

セッションが対象のPRに対してコメントの反映やpushを行うと、その行の右端に `#1234` のラベルが表示され、対応するプルリクエストの色（レビュー待ち・変更依頼あり・マージ可能など、ステータスに応じた色）付きリンクとしてジャンプできるようになります。

## 実践例

### 既存PRへのレビュー対応をバックグラウンドで進める

レビュアーからの指摘が入ったPRへの追従修正を、他の作業と並行してバックグラウンドセッションに任せることができます。

```text
claude --agent code-reviewer --bg "address review comments on PR 1234"
```

セッションがそのPRブランチへコミットをpushすると、agent view上の該当行に `#1234` ラベルが表示され、「Ready for review」グループに上がってくるため、対応が完了したタイミングを見逃しません。

### 複数PRにまたがるコメント・マージ作業をまとめて追跡する

複数の既存PRに対してコメント投稿やマージ作業を依頼した場合、それぞれのセッション行に対応するPR番号のリンクが表示されます。同一セッションが複数PRに関与した場合は `3 PRs` のようにまとめて表示され、`Space` キーで開くpeekパネルからすべてのPRを確認できます。

```text
> merge PR #2048 once CI passes, then leave a summary comment on PR #2050
```

また、`claude agents` のダイアログや検索入力欄に `#<番号>` またはPRのURLを入力してフィルタすると、そのPRに関与しているセッションだけを絞り込んで表示できます。

```text
#2048
```

## 注意点

- PR番号ラベルの色はPRのステータス（レビュー待ち、変更依頼あり、マージ可能など）によって変わります。対応が完了して番号が緑色になったタイミングがレビュー・マージの目安になります。
- ラベルはフォローアップの指示を送った後も表示が維持されるため、追加のやり取りをしている間も対象PRを見失うことはありません。
- この機能はターミナルがハイパーリンクをサポートしている場合にリンクとして機能します。非対応のターミナルではPR番号のテキスト表示のみとなります。
- worktreeで変更を分離しているバックグラウンドセッションは、条件を満たせば自らコミット・push・ドラフトPRのオープンまで行いますが、`main`/`master` への直接pushやforce-push、マージは行いません。既存PRへの操作範囲も同様に安全側に倒された制限が適用されます。
- 組織の管理者はagent view機能自体を管理設定で無効化できるため、環境によってはこの表示が見られない場合があります。
