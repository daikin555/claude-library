## 原文（日本語訳）

[VSCode] 英語やコードと混在した際に、右から左に書く言語（アラビア語、ヘブライ語、ペルシア語）のテキストが誤った順序で表示される問題を修正しました。

## 原文（英語）

[VSCode] Fixed right-to-left text (Arabic, Hebrew, Persian) rendering in the wrong order when mixed with English or code

## 概要

VS Code拡張機能のClaude Codeパネルで、アラビア語・ヘブライ語・ペルシア語などRTL（右から左に書く）言語のテキストが、英語の単語やコードスニペットと混在した場合に文字の並び順が崩れて表示される不具合がありました。今回の修正により、RTLテキストとLTR（左から右に書く）テキストが混在する文章でも、双方向テキスト（bidi）として正しい順序で表示されるようになりました。

## 基本的な使い方

この修正は表示バグの修正であり、ユーザー側の設定変更は不要です。VS Code拡張機能のチャットパネルやdiffビューでRTL言語を含むテキストを表示・入力すると、自動的に正しい順序でレンダリングされます。

```
# 例: アラビア語混じりのプロンプトをそのまま入力
هذا الملف يحتوي على دالة parseConfig() التي تقرأ config.json
```

## 実践例

### アラビア語のコメントを含むコードをレビューしてもらう

アラビア語のコメントと英語の関数名・コードが混在するファイルをClaudeにレビューさせた場合、VS Code拡張機能の応答表示でコメントの文字順序が崩れず正しく表示されます。

```javascript
// هذه الدالة تتحقق من صحة user.email قبل الحفظ
function validateEmail(user) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(user.email);
}
```

### ヘブライ語やペルシア語のドキュメントをClaudeに要約させる

ヘブライ語やペルシア語で書かれた仕様書に英語の技術用語（API名、関数名など）が混在していても、VS Code拡張機能のチャットパネル上で要約結果が正しい表示順序でレンダリングされます。

```
# 例: ヘブライ語混じりのプロンプト
תסביר לי מה עושה הפונקציה fetchUserData() בקובץ api.ts
```

### RTL言語話者が多い開発チームでのコードレビュー運用

多言語チームで、コミットメッセージやIssueの説明にRTL言語とコードスニペットが混在するケースでも、VS Code拡張機能上での表示が崩れなくなったため、レビューコメントのやり取りがしやすくなります。

## 注意点

- この修正はVS Code拡張機能に限定されたものです。ターミナル版のClaude Codeやその他のIDE統合（JetBrains等）でのRTL表示については別途確認が必要です。
- 双方向テキストの表示はエディタ・フォント・OSの設定にも依存するため、環境によっては完全な解消に加えてエディタ側の設定確認が必要な場合があります。
