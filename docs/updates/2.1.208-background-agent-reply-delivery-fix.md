## 原文（日本語訳）

バックグラウンドエージェントに入力した返信が、送信に失敗した際に失われてしまう不具合を修正した。テキストは保存されるようになり、セッションが再開したときに自動的に配信される。

## 原文（英語）

Fixed replies typed to a background agent being lost when delivery fails — the text is now saved and delivered when the session restarts

## 概要

Claude Codeの「Agent View（`claude agents`）」では、バックグラウンドで動いているセッションに対してPeekパネルなどから直接返信を送ることができます。しかしこれまでは、バックグラウンドサービスに一時的に接続できないなどの理由で送信が失敗すると、せっかく入力した返信内容がそのまま消えてしまう問題がありました。今回の修正により、送信に失敗した返信は破棄されずに保存され、対象セッションのプロセスが次に起動したタイミングでそのままプロンプトとして届けられるようになりました。

## 基本的な使い方

Agent View上で対象セッションの行を選び、`Space`キーでPeekパネルを開いてテキストを入力し、`Enter`で送信します。

```text
claude agents
# 対象セッションを選択して Space → 返信を入力 → Enter
```

送信が成功すればそのまま通常通りセッションに届きますが、失敗した場合も今回の修正のおかげで内容は保存されます。

## 実践例

### 一時的なネットワーク不調時の返信

バックグラウンドサービスに瞬断が起きたタイミングで質問に答えようとした場合でも、返信テキストを失う心配がなくなりました。

```text
Peekパネルで返信を入力 → Enter
→ 送信失敗時: 「reply was saved」というメッセージが表示される
→ セッションのプロセスが再起動すると、保存された返信が次のプロンプトとして自動的に投入される
```

### アイドル状態から復帰するセッションへの指示

長時間アイドルになっていたセッション（プロセスが一旦停止した状態）に対して先回りで返信しておいても、プロセスが再起動した際に確実にその内容が届きます。

```text
claude agents
# アイドル中のセッションに対して返信を入力して送信
# → 保存され、次回アタッチ／プロセス再起動時に配信される
```

## 注意点

- `!`で始まるBashコマンド形式の返信は保存対象外です。保存された文字列がそのまま次回プロンプトとしてセッションに渡されるため、Bashコマンドとして実行されると意図しない動作になる可能性があるための仕様です。
- 保存されるのはあくまで「送信自体が失敗したケース」であり、正常に配信された返信の動作は従来と変わりません。
- この機能はAgent View（`claude agents`）およびバックグラウンドセッション全般に関わる、per-userのスーパーバイザープロセス（デーモン）を前提とした挙動です。
