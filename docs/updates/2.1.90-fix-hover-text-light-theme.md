## 原文（日本語に翻訳）

ライトターミナルテーマでクリック展開のホバーテキストがほぼ不可視になる問題を修正

## 原文（英語）

Fixed click-to-expand hover text being nearly invisible on light terminal themes

## 概要

ライトカラーテーマのターミナル環境でClaude Codeを使用している際に、コンテンツを展開するためのクリック可能なホバーテキストが背景色とほぼ同一の色になってしまい、視認不可能になるUIバグが修正されました。これはアクセシビリティの問題でもあり、特にライトテーマを好むユーザーの使い勝手を大きく損なっていました。v2.1.90 以降では、ライトテーマでもホバーテキストが適切に表示されます。

## 基本的な使い方

修正内容はUIバグフィックスのため、v2.1.90 にアップデートするだけで問題が解消されます。ライトテーマを使用している場合に恩恵を受けられます：

```bash
# ターミナルのライトテーマ設定はターミナルアプリ側で行う
# Claude Codeはターミナルの色設定を自動的に検出します
claude
```

## 実践例

### ライトテーマターミナルでの使用

macOS のターミナル.appやiTerm2のライトテーマ、VSCode の統合ターミナルのライトテーマなどで：

```bash
# iTerm2などでライトテーマを設定してClaude Codeを起動
claude

# v2.1.90以降: ホバーテキストが正しく表示される
# 例: 折り畳まれたコードブロックの「クリックして展開」テキストが見える
```

### アクセシビリティ設定との組み合わせ

高コントラストのライトテーマを使用している場合も正常に動作するようになりました：

```bash
# 高コントラストライトテーマの設定例（VSCode）
# ターミナルの color.foreground を dark に設定
# Claude Codeがテーマを検出して適切な色を使用
```

## 注意点

- このバグはライトカラーテーマのターミナルを使用しているユーザーにのみ影響していました
- ダークテーマのユーザーには影響がありませんでした
- v2.1.90 以降では、ライト/ダーク両方のテーマで適切な色コントラストが保たれます
- ターミナルのカラー設定によっては、まだ表示が最適でない場合があります
