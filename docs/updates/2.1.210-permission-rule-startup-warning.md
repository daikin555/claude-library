## 原文（日本語に翻訳）

`Write(path)`、`NotebookEdit(path)`、`Glob(path)` 権限ルールに対する起動時警告を追加しました — 代わりに `Edit(path)` または `Read(path)` を使用してください

## 原文（英語）

Added a startup warning for `Write(path)`, `NotebookEdit(path)`, and `Glob(path)` permission rules — use `Edit(path)` or `Read(path)` instead

## 概要

Claude Code v2.1.210では、`settings.json` の `permissions` 設定に `Write(path)`・`NotebookEdit(path)`・`Glob(path)` のようにパスを指定したルールが定義されている場合、Claude Code起動時に警告が表示されるようになりました。これらのツールはパス引数によるファイル単位のルール評価が意図通りに機能しないため、ファイルパスを対象にアクセス制御したい場合は `Edit(path)` または `Read(path)` を使うことが推奨されています。ユーザーが気づかないまま「効いていないつもりのルール」を運用してしまう事故を防ぐための変更です。

## 基本的な使い方

### 修正前の問題

```json
// .claude/settings.json
{
  "permissions": {
    "deny": [
      "Write(./secrets/**)",
      "Glob(./secrets/**)"
    ]
  }
}
```

```bash
claude
# 警告なしで起動
# → ユーザーは "secrets配下は保護されている" と誤認しがちだが、
#   Write/Glob はパス単位のマッチングが期待通りに効かないケースがある
```

### 修正後の動作

```bash
claude

# 起動時に警告が表示される:
# ⚠ Warning: Permission rule "Write(./secrets/**)" uses a tool
#   that does not support path-scoped matching as expected.
#   Consider using Edit(./secrets/**) or Read(./secrets/**) instead.
```

## 実践例

### Edit(path) への書き換え

```json
// 修正前
{
  "permissions": {
    "deny": ["Write(./config/production.yaml)"]
  }
}
```

```json
// 修正後（推奨）
{
  "permissions": {
    "deny": ["Edit(./config/production.yaml)"]
  }
}
```

`Edit` ルールはファイル編集を行う組み込みツール全般に適用され、`Write` ツールによる新規ファイル作成も含めてブロックする効果を持ちます。

### Read(path) を使った探索系ツールの制御

```json
// 修正前
{
  "permissions": {
    "deny": ["Glob(./.env.*)"]
  }
}
```

```json
// 修正後（推奨）
{
  "permissions": {
    "deny": ["Read(./.env.*)"]
  }
}
```

`Read` ルールは `Grep` や `Glob` など、ファイルを読み取る組み込みツール全般に対してベストエフォートで適用されるため、`.env` 系ファイルへのアクセスを一括で防ぎたい場合に有効です。

### NotebookEditのルールをEditに統一する

```json
// 修正前
{
  "permissions": {
    "ask": ["NotebookEdit(./notebooks/**)"]
  }
}
```

```json
// 修正後（推奨）
{
  "permissions": {
    "ask": ["Edit(./notebooks/**)"]
  }
}
```

Jupyter Notebookの編集も `Edit` ルールの対象に含まれるため、個別に `NotebookEdit(path)` を指定する必要はありません。

## 注意点

- この警告はClaude Code v2.1.210（2026年7月15日リリース）で追加されました
- 警告が出ても既存のルール自体が即座に無効化されるわけではありません。あくまで「意図通りに動作しない可能性がある」という注意喚起です
- `Read` の deny ルールは同じパスに対する `Edit` ツール（新規ファイル作成を含む）もブロックしますが、`Write` と `NotebookEdit` は対象外なので、あらゆるツールからパスを保護したい場合は別途 `Edit` deny ルールを追加する必要があります
- 権限ルールは `deny` → `ask` → `allow` の順に評価され、deny ルールは配列内の順序やどの設定ファイルにあるかに関わらず allow ルールより常に優先されます
- グローバル設定（`~/.claude/settings.json`）、プロジェクト設定（`.claude/settings.json`）、ローカル設定（`.claude/settings.local.json`）のいずれで定義したルールでも警告の対象になります
