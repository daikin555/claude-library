# バックグラウンドエージェント実行後にEscとCtrl+Cが反応しない問題を修正

## 原文（日本語に翻訳）
メインターンの終了後にバックグラウンドエージェントが実行中の間、EscキーとCtrl+Cが反応しない問題を修正しました。

## 原文（英語）
Fixed Esc and Ctrl+C not responding while background agents are still running after the main turn ends

## 概要
メインターンが終了した後もバックグラウンドエージェントが実行を続けている際、EscキーやCtrl+Cによる操作が反応しなくなる問題がありました。この修正により、バックグラウンドエージェントが実行中でも、キーボード操作が正常に機能するようになりました。

## 基本的な使い方

この修正は自動的に適用されます。

### 影響を受けていたシナリオ
1. ワークフローやエージェントを起動してバックグラウンド実行
2. メインターンが完了
3. バックグラウンドエージェントがまだ実行中
4. （以前）EscやCtrl+Cを押しても反応がなかった

### 修正後
バックグラウンドエージェント実行中でも：
- Esc: 正常に動作（ダイアログを閉じる、操作をキャンセルするなど）
- Ctrl+C: 正常に動作（現在の操作を中断するなど）

## 注意点
- この修正により、長時間実行するバックグラウンドエージェントがある場合でもインタラクティブな操作が可能になります
- Ctrl+C はセッション全体を終了させる可能性があります。バックグラウンドエージェントを継続させたい場合は注意してください
