## 原文（日本語に翻訳）

組織のデフォルトモデルのサポートを追加 — 管理者が組織コンソールで設定し、ユーザー自身がモデルを選択していない場合に `/model` 上で「Org default」（または「Role default」）として表示されるようになった。

## 原文（英語）

Added support for organization default models — admins set it in the org console; it shows as "Org default" (or "Role default") in `/model` when you haven't picked one yourself

## 概要

Claude Codeに組織全体のデフォルトモデルを設定できる機能が追加されました。組織の管理者（OwnerまたはPrimary Owner）が管理コンソールの「Organization settings > Models」からデフォルトモデルを設定すると、個別にモデルを選択していないメンバー全員にそのモデルが自動適用されます。また、特定のロール（役割）ごとに異なるデフォルトモデルを設定することも可能です。

## 基本的な使い方

管理者は組織コンソールからデフォルトモデルを設定します。一般ユーザー側では特別な操作は不要で、`/model` を実行するとどのデフォルトが適用されているかを確認できます。

```
> /model
```

自分でモデルを選択していない場合、メニュー上に以下のように表示されます。

```
Org default (claude-sonnet-5)
```

ロールごとにデフォルトが設定されている場合は次のように表示されます。

```
Role default (claude-opus-4-6)
```

## 実践例

### 組織全体で統一モデルを強制したい場合

管理者がコンソールで組織全体のデフォルトを1つに設定すると、明示的にモデルを選択していないメンバー全員がそのモデルを使うようになります。コスト管理やモデル挙動の統一に有効です。

```
組織コンソール: Organization settings > Models > Default model
→ claude-sonnet-5 を選択して保存
```

### 部署・役割ごとに異なるモデルを使い分けたい場合

カスタムロールを作成し、ロールごとに異なるデフォルトモデルを割り当てることで、たとえば「リサーチチームはOpusをデフォルトに、サポートチームはSonnetをデフォルトに」といった運用が可能です。

```
組織コンソール: Organization settings > Roles > Research Team
→ Default model: claude-opus-4-6
```

複数グループに所属するメンバーがいて異なるロールデフォルトが競合する場合は、モデルファミリー（Haiku < Sonnet < Opus）とリリース日に基づき、より高性能なモデルが優先的にデフォルトとして適用されます。

### メンバー側で自分のモデルを明示的に選びたい場合

組織やロールのデフォルトより自分の好みを優先したい場合は、通常どおり `/model` で選択すれば、その選択がOrg defaultより優先されます。

```
> /model
# メニューから希望のモデルを選択
```

## 注意点

- 組織デフォルトを設定できるのはOwnerまたはPrimary Owner権限を持つ管理者のみです
- ロールのデフォルトモデルは、そのロールに割り当てられたメンバーに対して組織全体のデフォルトより優先されます
- 複数のロールに所属し異なるデフォルトが設定されている場合、モデルファミリー→リリース日の順でより高性能なモデルが採用されます
- ユーザー自身が `/model` で明示的にモデルを選択している場合は、組織・ロールのデフォルト設定よりそのユーザーの選択が優先されます
