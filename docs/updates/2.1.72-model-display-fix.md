## 原文（日本語に翻訳）

- Claudeが動作中に`/model`の結果が表示されない問題を修正

## 原文（英語）

- Fixed `/model` not displaying results when run while Claude is working

## 概要

Claudeがタスクを処理中（応答生成中やツール実行中）に`/model`コマンドを実行すると、結果が画面に表示されない問題が修正されました。`/model`はCurrentに使用中のモデルの確認やモデルの切り替えに使うコマンドですが、Claude動作中に実行するとUIの更新が正しく行われていませんでした。修正後は、Claudeの動作中でも`/model`の結果が正しく表示されます。

## 基本的な使い方

Claudeがタスクを実行中でも、`/model`コマンドの結果が表示されるようになりました:

```
claude
> 大きなプロジェクトを分析して
# Claudeが処理中...

> /model
# 修正後: 現在使用中のモデル情報が正しく表示される
# 以前は何も表示されないか、表示が遅れていた
```

## 実践例

### 処理中のモデル確認

長時間かかるタスクの実行中にどのモデルが使われているか確認する:

```
claude
> プロジェクト全体のコードレビューをして
# 処理が進行中...

> /model
# 現在のモデル（例: claude-sonnet-4-20250514）が表示される
```

### 処理中のモデル切り替え

処理中でもモデルを確認して、必要に応じて次のリクエストに備えてモデルを変更できます:

```
claude
> テストケースを生成して
# 処理中...

> /model
# 現在のモデル一覧と選択中のモデルが表示される
# 必要に応じてモデルを切り替え可能
```

## 注意点

- `/model`コマンドは現在のセッションで使用中のモデルの表示と切り替えに使用します
- 処理中にモデルを切り替えた場合、次のリクエストから新しいモデルが適用されます（現在処理中のリクエストには影響しません）
- この修正はUIの描画タイミングに関するもので、モデルの動作自体には影響しません
