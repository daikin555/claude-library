## 原文（日本語訳）

停止していたバックグラウンドエージェントにアタッチすると、セッションが起動している間も空白の「Session is starting」画面ではなく、そのトランスクリプトが即座に表示されるようになった。

## 原文（英語）

Attaching to a stopped background agent now shows its transcript immediately while the session warms up, instead of a blank "Session is starting" screen

## 概要

Agent View（`claude agents`）やシェルからの`claude attach`で、プロセスが停止しているバックグラウンドセッションにアタッチすると、これまではプロセスが起動し直すまで何も情報のない「Session is starting」という画面が表示されていました。今回の修正で、プロセス再起動中でも直前までの会話（トランスクリプト）の最後の画面分がまず表示され、準備が整い次第ライブのセッション表示に切り替わるようになりました。

## 基本的な使い方

停止しているバックグラウンドセッションにアタッチするには、Agent Viewから選択するか、シェルから`claude attach`を使います。

```bash
claude attach <session-id>
```

これまでは起動が完了するまで空白の待機画面でしたが、今回のアップデート以降はアタッチした瞬間から直前のトランスクリプトが表示され、プロセスの準備が整うと自動的にライブ表示へ切り替わります。

## 実践例

### 長時間放置していたバックグラウンドセッションの状況を素早く思い出す

しばらく他の作業をしていて、停止状態になったバックグラウンドエージェントに久しぶりにアタッチする場合、以前は起動が終わるまで何も見えず「何をしていたセッションだったか」を思い出せませんでした。今回の変更により、アタッチした瞬間に直近のやり取りが表示されるため、待ち時間を無駄にせず状況を把握できます。

### Agent Viewで複数セッションを切り替えながら素早く内容を確認する

Agent Viewで複数のバックグラウンドセッションを行き来しながら進捗を確認するようなワークフローでは、アタッチのたびに空白画面を待たされると確認のテンポが崩れます。プロセスの起動を待つ間も直前のトランスクリプトが表示され続けるため、内容確認とセッション切り替えをスムーズに行えます。

## 注意点

- 表示されるのはあくまで「直前までのトランスクリプトの最後の画面分」であり、プロセス再起動中に新しい応答が生成されているわけではありません。
- プロセスの準備が整うと、表示は自動的にライブのセッション画面に置き換わります。
- この改善はUI/UXの表示に関するものであり、セッションの再開ロジックや処理結果そのものを変更するものではありません。
