# 修正：複数の同時セッションで繰り返されていたOAuth再認証の問題

## 原文（日本語に翻訳）

1つのセッションがOAuthトークンをリフレッシュした際に、複数の同時Claude Codeセッションで繰り返し再認証が要求される問題を修正。

## 原文（英語）

Fixed multiple concurrent Claude Code sessions requiring repeated re-authentication when one session refreshes its OAuth token

## 概要

複数のターミナルタブやウィンドウでClaude Codeを同時に使用していた際、1つのセッションがOAuthトークンをリフレッシュすると、他のセッションが繰り返し再認証を求めてくる問題が修正されました。この修正により、複数セッションを並行して使用する際の認証フローが改善され、作業の中断が減ります。

## 基本的な使い方

修正後は、複数のClaude Codeセッションを同時に起動しても、認証が共有され各セッションで再ログインを求められなくなります。

```bash
# ターミナル1
claude  # 通常通り起動・認証

# ターミナル2（同時起動）
claude  # 別セッションだが再認証不要
```

## 実践例

### ### 複数プロジェクトの同時作業

異なるプロジェクトを複数のターミナルで同時に作業：

```bash
# ターミナル1: フロントエンドプロジェクト
cd ~/projects/frontend
claude

# ターミナル2: バックエンドプロジェクト（別ウィンドウ）
cd ~/projects/backend
claude

# どちらのセッションも、もう一方がトークンリフレッシュしても再認証不要
```

### ### tmux での複数ペイン作業

tmux で複数のClaude Codeセッションを管理：

```bash
# tmux セッション作成
tmux new-session -d -s work

# 複数ペインを作成してそれぞれでClaude Codeを起動
tmux split-window -h
tmux send-keys -t work:0.0 'claude' Enter
tmux send-keys -t work:0.1 'claude' Enter
```

## 注意点

- この修正はOAuth認証（claude.ai アカウントでのログイン）を使用している場合に適用される
- `ANTHROPIC_API_KEY` を使用している場合はこの問題の影響を受けない
- トークンのリフレッシュは自動的に行われ、ユーザーの操作は不要
