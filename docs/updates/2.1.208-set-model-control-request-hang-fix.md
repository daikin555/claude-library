## 原文（日本語訳）

`control_request` に文字列でない `set_model` ペイロードが含まれている場合、ヘッドレスのstream-jsonセッションが永久にハングしてしまう不具合を修正した。CLIはエラーレスポンスを返すようになった。

## 原文（英語）

Fixed headless stream-json sessions hanging permanently when a `control_request` carried a non-string `set_model` payload; the CLI now answers with an error response

## 概要

Claude Code Agent SDKは、CLIサブプロセスとJSON Lines形式で通信しており、`control_request`/`control_response` という制御メッセージを使ってモデル切り替えなどの操作を行います。`set_model` はそのうちの一つで、セッション中に使用するモデルを変更するための制御リクエストです。これまでは `set_model` のペイロードに文字列以外の値（数値やオブジェクトなど）が渡されると、CLI側の処理がどのレスポンスも返さないまま停止してしまい、SDKクライアントが該当リクエストを永久に待ち続けるハングが発生していました。修正後は、CLIが不正な形式を検知して明示的なエラーレスポンスを返すため、SDK側はハングせずにエラーとして処理を継続できます。

## 基本的な使い方

SDK/ヘッドレスモードでモデルを切り替えるには、`control_request` として `set_model` を送信します。

```json
{"type":"control_request","request_id":"1","request":{"subtype":"set_model","model":"claude-sonnet-5"}}
```

対応するCLIからの応答は次のような `control_response` になります。

```json
{"type":"control_response","response":{"request_id":"1","subtype":"success"}}
```

## 実践例

### Python SDKからモデルを動的に切り替える

長時間実行するエージェントで、タスクの複雑さに応じてモデルを切り替えたい場合に使います。

```python
from claude_agent_sdk import ClaudeSDKClient

async with ClaudeSDKClient() as client:
    await client.set_model("claude-opus-4-8")
    async for message in client.receive_response():
        print(message)
```

内部的にはこの呼び出しが `set_model` の `control_request` に変換されます。呼び出し元のコードが誤って数値やbooleanなど文字列以外の値を渡してしまった場合でも、修正後はCLIから明確なエラーが返るため、原因を特定しやすくなりました。

### 自前のSDKラッパーで不正な値を送ってしまうケース

独自にstream-jsonプロトコルを実装しているラッパーで、設定値のパース処理にバグがあり、モデル名の代わりに数値IDやnullを送ってしまうことがあります。

```bash
echo '{"type":"control_request","request_id":"2","request":{"subtype":"set_model","model":123}}' | \
  claude --input-format stream-json --output-format stream-json --verbose
```

修正前はこのリクエストの後、セッション全体が応答不能になっていましたが、修正後はエラーの `control_response` が返り、クライアント側でリトライやログ出力などの処理を続行できます。

## 注意点

- 文字列型の正しい `set_model` ペイロードには影響しません。
- この修正はヘッドレスモード（`claude -p`）およびAgent SDK経由の利用に関係し、通常のインタラクティブセッションでの `/model` コマンドには直接関係しません。
- SDKクライアントを独自実装している場合は、`set_model` に渡す値を必ず文字列（モデルエイリアスまたはフルモデルID）にする必要があります。
