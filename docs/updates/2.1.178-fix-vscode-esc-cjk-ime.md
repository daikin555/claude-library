## 原文（日本語に翻訳）

【VSCode】CJK IME候補ウィンドウをEscキーで閉じると実行中のClaudeタスクがキャンセルされる問題を修正しました。

## 原文（英語）

[VSCode] Fixed pressing Esc to dismiss a CJK IME candidate window canceling the running Claude task

## 概要

VS Code拡張機能でClaude Codeを使用中、日本語・中国語・韓国語（CJK）のIME入力時にEscキーで候補ウィンドウを閉じると、実行中のClaudeタスクまでキャンセルされてしまう問題が修正されました。日本語ユーザーにとって特に影響が大きかった問題で、修正によりIMEの操作がClaudeタスクに影響しなくなります。

## 基本的な使い方

この修正はVS Code拡張機能を使用中に自動的に適用されます。

**影響する環境:**
- VS Code + Claude Code拡張機能
- 日本語IME（Google日本語入力、MS IME等）
- 中国語IME（Pinyin等）
- 韓国語IME

**修正後の動作:**
- IME候補ウィンドウ表示中のEsc → 候補ウィンドウのみ閉じる
- Claudeタスクは継続して実行される

## 実践例

### 日本語入力中のシナリオ

修正前の問題のある動作:
```
1. Claudeに長い日本語のタスクを送信（実行中）
2. 入力欄に追加のメッセージを日本語で入力しようとする
3. IMEで変換候補ウィンドウが表示される
4. 意図しない変換候補をEscで閉じる
5. → Claudeの実行中タスクがキャンセルされてしまう！
```

修正後の正常な動作:
```
1. Claudeに長い日本語のタスクを送信（実行中）
2. 入力欄に追加のメッセージを日本語で入力しようとする
3. IMEで変換候補ウィンドウが表示される
4. Escで候補ウィンドウを閉じる
5. → 候補ウィンドウのみ閉じる。Claudeタスクは継続！
```

### 影響を受けるIMEの種類

- **日本語**: Google日本語入力（Mozc）、MS IME、ATOK
- **中国語**: Pinyin（ピン音）、Zhuyin（注音）
- **韓国語**: Hangul（ハングル）

### VS Code拡張機能のアップデート確認

```
VS Code拡張機能を最新版に更新:
1. VS Code → 拡張機能パネルを開く（Ctrl+Shift+X）
2. "Claude" または "Claude Code" を検索
3. "更新" ボタンが表示される場合はクリック
4. または自動更新を有効化
```

## 注意点

- この修正はVS Code拡張機能のみに関係します。ターミナルでのClaude Code CLIには影響しません
- JetBrains IDEの拡張機能には別途対応が必要な場合があります
- ESCキーを使ってClaudeタスクを明示的にキャンセルしたい場合は、IMEが非アクティブな状態（英語モード）でEscを押してください
- IMEの候補ウィンドウが表示されていない状態でのEscは引き続きタスクキャンセルに使用できます
