## 原文（日本語に翻訳）

バックグラウンドエージェントを改善: 起動結果でClaudeに「応答を終了してください」と指示しなくなった。エージェント実行中も他のタスクを継続して作業できる。

## 原文（英語）

Improved background agents: the launch result no longer instructs Claude to "end your response" — it keeps working on other tasks while the agent runs

## 概要

バックグラウンドエージェントを起動した後、Claude が「応答を終了する」ように指示されていた動作が改善されました。バックグラウンドエージェントが実行中でも、Claude は並行して他のタスクを継続できるようになります。これにより、長時間のバックグラウンドタスク（テスト実行、ビルド、デプロイなど）を待たずに、同じセッションで他の作業を続けられます。

## 基本的な使い方

バックグラウンドエージェントを起動する従来の方法は変わりません。改善点は動作の継続性です:

```
# バックグラウンドエージェントを起動する例
テストを実行して結果を教えて（バックグラウンドで）
```

**改善前**: エージェント起動後、Claude は「応答を終了」して入力待ち状態になる  
**改善後**: エージェント起動後も Claude は他のタスクを続けて実行できる

## 実践例

### 並行作業のシナリオ

```
ユーザー: テストスイート全体を実行しながら、
         フロントエンドのコードレビューもやってほしい

Claude: テストをバックグラウンドで起動します。
       [バックグラウンドエージェント起動]
       
       テストを実行しながら、フロントエンドのコードレビューを始めます。
       src/components/Button.tsx を確認します...
       [コードレビューを継続]
```

### 長時間タスクとの並行作業

```
ユーザー: 本番デプロイを開始して、その間に
         リリースノートを作成してほしい

Claude: デプロイをバックグラウンドで開始します。
       [デプロイエージェント起動]
       
       デプロイ実行中にリリースノートを作成します。
       [リリースノート作成を継続]
```

### バックグラウンドエージェントの確認

```bash
# バックグラウンドで実行中のエージェントを確認
# （サイドパネルまたは専用コマンドで確認可能）
```

キーボードショートカット `←←` でバックグラウンドセッションに移行できます。

## 注意点

- バックグラウンドエージェントと並行して作業する場合、リソース（APIレート制限など）を共有します
- エージェントの結果が必要なタスクは、エージェント完了後に続けて実行してください
- バックグラウンドセッションへの移動は `←←`（左矢印2回）で行えます
- 並行タスクが多すぎる場合、コンテキスト管理に注意してください
