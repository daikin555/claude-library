## 原文（日本語訳）

worktreeで隔離されたサブエージェントが `git -C`、`--git-dir`、または `GIT_DIR`/`GIT_WORK_TREE` を使って共有チェックアウトにgit操作をリダイレクトしてしまう問題を修正しました。

## 原文（英語）

Fixed worktree-isolated subagents redirecting git into the shared checkout via `git -C`, `--git-dir`, or `GIT_DIR`/`GIT_WORK_TREE`

## 概要

Claude Codeの `--worktree` 機能は、サブエージェントが独立したgit worktree内で作業することで、他のセッションのファイルに影響を与えないようにする隔離の仕組みです。しかし、これまでは `git -C <path>` オプションや `--git-dir` フラグ、あるいは `GIT_DIR`/`GIT_WORK_TREE` 環境変数を使うことで、サブエージェントが意図的または偶発的に隔離されたworktreeの外、つまり共有のメインチェックアウトに対してgitコマンドを実行できてしまう抜け穴がありました。この修正により、これらの経路での「隔離破り」が防止され、worktree隔離の安全性が強化されています。

## 基本的な使い方

この修正はバグ修正・セキュリティ強化であり、特別な設定は不要です。`--worktree` オプションでセッションを開始するだけで、修正後の安全な挙動が自動的に適用されます。

```bash
# 隔離されたworktreeでセッションを開始
claude --worktree feature-x
```

## 実践例

### 従来発生し得ていたリスクのあるパターン

修正前は、worktree内で動作しているはずのサブエージェントが、以下のようなコマンドを実行することで、意図せずメインの共有チェックアウトを変更してしまう可能性がありました。

```bash
# worktree内にいるはずが、-C でメインチェックアウトを直接操作
git -C /path/to/main-checkout commit -am "unintended change"

# --git-dir / --work-tree を使った迂回
git --git-dir=/path/to/main-checkout/.git --work-tree=/path/to/main-checkout status

# 環境変数経由での迂回
GIT_DIR=/path/to/main-checkout/.git GIT_WORK_TREE=/path/to/main-checkout git add .
```

修正後は、こうした経路を通じて隔離されたworktreeの外側（共有チェックアウト）に対してgit操作が行われることがブロックされます。

### 並行して複数のworktreeセッションを運用する場合

複数の `--worktree` セッションを並行稼働させている環境（例えばagent viewで複数のバックグラウンドタスクを管理している場合）でも、各セッションが自分のworktreeの外に影響を及ぼさないことが保証されるため、安心して並列開発を進められます。

```bash
claude --worktree task-a &
claude --worktree task-b &
claude agents  # 実行中のworktreeセッションを一覧表示
```

## 注意点

- 本修正はworktree隔離の安全性に関わるセキュリティ関連の修正です。worktree機能を使っている場合はアップデートを推奨します
- worktreeはデフォルトでリポジトリルート配下の `.claude/worktrees/<name>/` に作成され、`worktree-<name>` という新しいブランチ上で動作します
- 通常の（`--worktree` を使わない）セッションには影響しません
