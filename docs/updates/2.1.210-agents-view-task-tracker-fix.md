## 原文（日本語に翻訳）

←キーを押してAgent Viewを開き、その後セッションに戻った際にタスクトラッカーが失われてしまう不具合を修正しました。

## 原文（英語）

Fixed pressing ← to open the agents view dropping the task tracker when returning to the session

## 概要

Claude Codeでは、セッション中に `←` キーを押すとAgent View（複数セッションを一覧・管理する画面）を開いて他のセッションを確認したり操作したりできます。タスクトラッカー（`Ctrl+T` で開閉できる、多段階の作業の完了・進行中・キュー状況を表示するオーバーレイ）は、セッション内での進行状況を追跡する重要なUIです。この修正前は、`←` キーでAgent Viewに移動し、その後元のセッションに戻ってくると、それまで表示・追跡されていたタスクトラッカーの状態が失われてしまう不具合がありました。v2.1.210では、Agent Viewへの出入りに関わらずタスクトラッカーの状態が正しく保持されるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
claude
> 複数ファイルにまたがるリファクタリングをして

# Claudeがタスクをタスクトラッカーに登録して進行
# Ctrl+T でタスクトラッカーを表示
# [x] ファイルA修正
# [ ] ファイルB修正
# [ ] テスト更新

# ここで ← キーを押してAgent Viewに移動
# 他のセッションを確認後、→ または Enter で元のセッションに戻る

# 修正前:
# タスクトラッカーの状態が消えてしまい、
# 進行中だったタスクリストが再表示されない
# Ctrl+T を押しても空、あるいは不完全な状態になる
```

### 修正後の動作

```bash
claude
> 複数ファイルにまたがるリファクタリングをして

# Ctrl+T でタスクトラッカーを表示
# [x] ファイルA修正
# [ ] ファイルB修正
# [ ] テスト更新

# ← キーでAgent Viewに移動 → 他セッションを確認 → 元セッションに復帰

# 修正後:
# タスクトラッカーの状態がそのまま保持されている
# Ctrl+T を押すと、離脱前と同じ進行状況が表示される
# [x] ファイルA修正
# [ ] ファイルB修正
# [ ] テスト更新
```

## 実践例

### 複数セッションを行き来しながら長時間タスクを監視

Agent Viewで別セッションの状態を確認しつつ、元のセッションの進行状況も見失わずに追いたい場合。

```bash
# セッションA: 大規模マイグレーション実行中
claude
> データベースマイグレーションスクリプトを段階的に実行して

# タスクトラッカーで進行状況を確認
# Ctrl+T
# [x] スキーマ変更を適用
# [ ] データ移行バッチを実行
# [ ] 整合性チェック

# 他のバックグラウンドセッションの状況が気になったので ← でAgent Viewへ
# 別セッションをpeek（Spaceキー）して状況を確認
# → または Enter で元のセッションAに復帰

# 修正後: タスクトラッカーの進行状況がそのまま維持されている
Ctrl+T
```

### レビュー中に一時的に他セッションへ切り替え

コードレビューの途中で別のディスパッチ済みセッションの結果を確認しに行くケース。

```bash
claude
> このPRの変更点をレビューして問題点をタスクリストにまとめて

# タスクトラッカーにレビュー観点がリスト化される
Ctrl+T
# [ ] セキュリティ観点のチェック
# [ ] パフォーマンス観点のチェック
# [ ] テストカバレッジの確認

# ← でAgent Viewを開き、別セッションの完了通知を確認
# 元セッションに戻る

# 修正後: レビュータスクのリストが引き続き表示される
Ctrl+T
```

### 頻繁なセッション切り替えを伴うマルチタスク運用

Agent Viewを操作画面のハブとして使い、複数セッションを頻繁に行き来する運用スタイルでの安定性確認。

```bash
claude agents
# 複数セッションが一覧表示される

# セッションBにアタッチ（→ または Enter）
> 長時間タスクを開始して進捗をタスクトラッカーに反映して

# ← でAgent Viewに戻る → 別のセッションCを確認 → セッションBに再アタッチ
# を何度か繰り返す

# 修正後: 何度出入りしてもセッションBのタスクトラッカーの状態は保持される
```

## 注意点

- この修正はClaude Code v2.1.210で導入されました
- タスクトラッカーは `Ctrl+T` で表示/非表示を切り替えられるオーバーレイで、セッション内の多段階作業の進行状況（完了・進行中・キュー）を追跡します
- Agent Viewへの出入りは `←`（開く）、`→` または `Enter`（選択したセッションにアタッチ）、空のプロンプトで `←`（デタッチ）などのキー操作で行います
- 本修正はタスクトラッカーの「表示状態の保持」に関するものであり、タスク自体の実行結果やセッションの会話履歴には影響しません
- 頻繁にAgent Viewとセッションを行き来する運用（複数エージェントの並行監視など）で特に効果が実感しやすい修正です
