## 原文（日本語訳）

Claude Desktopのペイン名変更に先立ち、「Claude Browser」というMCPサーバー名を（「Claude Preview」と合わせて）予約した。これにより、ユーザーが設定するMCPサーバーはこの2つの名前のいずれでも登録できなくなった。

## 原文（英語）

Reserved the "Claude Browser" MCP server name (alongside "Claude Preview") ahead of the Claude Desktop pane rename; user-configured MCP servers can no longer register under either name

## 概要

Claude Desktopでは、今後ブラウザ機能を表示するペインの名称が「Claude Browser」に変更される予定です。この変更に備え、Claude Codeは「Claude Browser」という名前を、既存の「Claude Preview」と同様にシステム予約名として扱うようになりました。結果として、ユーザーが独自にMCPサーバーを設定する際、この2つの名前をサーバー名として使うことができなくなります。これは内部機能の名称とユーザー設定のMCPサーバー名が衝突するのを防ぐための措置です。

## 基本的な使い方

MCPサーバーを追加する際、`.mcp.json` やCLIの `claude mcp add` コマンドなどでサーバー名を指定しますが、以下の2つの名前は予約済みのため使用できません。

- `Claude Browser`
- `Claude Preview`

```bash
# 以下のようにこれらの名前でMCPサーバーを追加しようとすると失敗する（または登録が拒否される）
claude mcp add "Claude Browser" -- node my-mcp-server.js
```

通常のMCPサーバー設定手順自体（`claude mcp add`、`.mcp.json` への追記、`/mcp` コマンドでの確認など）に変更はなく、影響を受けるのはサーバー名として上記2つの予約語を使っていた場合のみです。

## 実践例

### 既存のMCPサーバー名が予約名と衝突している場合

もし既に `Claude Preview` や `Claude Browser` という名前でMCPサーバーを設定していた場合は、他の名前にリネームする必要があります。

```bash
# 設定済みのMCPサーバー一覧を確認
claude mcp list

# 予約名を使っていたサーバーを一度削除し、別名で登録し直す
claude mcp remove "Claude Preview"
claude mcp add "my-preview-tool" -- node my-mcp-server.js
```

### これから新規にMCPサーバーを設定する場合

新しくMCPサーバーを構築・設定する際は、命名段階で `Claude Browser` や `Claude Preview` といった名前を避け、サーバーの機能が分かる固有の名前を付けておくと、将来的な予約語追加によるトラブルを避けやすくなります。

```bash
claude mcp add "internal-web-browser" -- node ./servers/browser-tool.js
```

## 注意点

- 予約名は「Claude Browser」と「Claude Preview」の2つで、いずれもユーザー設定のMCPサーバー名として登録できません。
- これはClaude Desktop側でブラウザ関連のペイン名称を「Claude Browser」に変更する準備として行われた予約であり、Claude Code / Claude Desktopの内部機能名との衝突を避けるための措置です。
- 既存の設定でこれらの名前を使っていた場合、アップデート後に登録が拒否される、または動作しなくなる可能性があるため、別の名前へのリネームが必要です。
