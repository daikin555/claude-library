## 原文（日本語に翻訳）

トランスクリプト検索入力欄にマウスイベントがリテラルの「mouse」テキストとして挿入される問題を修正しました。

## 原文（英語）

Fixed mouse events inserting literal "mouse" text into transcript search input.

## 概要

会話履歴（トランスクリプト）の検索入力欄でマウスを操作した際、マウスイベントがテキストとして解釈されて「mouse」という文字列が入力欄に挿入されてしまうUIバグが修正されました。検索機能を使おうとした際に意図しない文字が入力される問題が解消されます。

## 基本的な使い方

トランスクリプト検索を使う場合、修正後は検索入力欄をクリックしてもマウスイベントが「mouse」テキストとして入力されません。

```
# トランスクリプト検索の使い方
Ctrl+F または / キー → 検索入力欄が表示される
テキストを入力して検索
```

## 実践例

### ユースケース: 過去の会話を検索

トランスクリプトから特定のコードや応答を検索する場合：

```
修正前の問題:
1. 検索入力欄を開く
2. マウスでクリックまたはスクロール
3. 「mouse」という文字が入力欄に挿入される
4. 検索したいキーワードに「mouse」が混入する

修正後の動作:
1. 検索入力欄を開く
2. マウスでクリックまたはスクロール → 「mouse」は挿入されない
3. 検索したいキーワードをそのまま入力できる
```

### ユースケース: 検索とコピーの組み合わせ

```
1. 検索欄で「エラーメッセージ」を検索
2. マウスで検索結果を選択
3. 修正後: マウスのクリック操作が「mouse」テキストに変換されない
4. 正常にコピー&ペーストが機能する
```

## 注意点

- この問題はトランスクリプト検索入力欄に固有のバグでした
- 通常のメッセージ入力欄では発生しない問題でした
- マウスイベントの処理がターミナルのマウスレポート機能と干渉していた可能性があります
