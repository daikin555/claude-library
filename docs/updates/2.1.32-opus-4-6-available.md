## 原文（日本語訳）
Claude Opus 4.6が利用可能になりました！

## 原文（英語）
Claude Opus 4.6 is now available!

## 概要
Claude Code で最新の Claude Opus 4.6 モデルが利用可能になりました。Opus 4.6 は Anthropic の最新フロンティアモデルで、より高度な推論能力と性能を提供します。

## 基本的な使い方

Claude Code を起動する際に、`--model opus` オプションを指定することで Opus 4.6 を使用できます。

```bash
claude --model opus
```

特定のタスクで Opus モデルを使用したい場合：

```bash
claude --model opus "複雑なコードのリファクタリングを手伝って"
```

## 実践例

### 複雑なアーキテクチャ設計

Opus 4.6 の高度な推論能力を活かして、複雑なシステム設計を依頼できます。

```bash
claude --model opus
> マイクロサービスアーキテクチャで、認証・決済・在庫管理を統合したECシステムの設計を提案してください
```

### 高度なコード解析

大規模なコードベースの分析や、複雑なバグの原因特定に活用できます。

```bash
claude --model opus
> このプロジェクト全体のパフォーマンスボトルネックを特定し、最適化案を提示してください
```

### 難易度の高いリファクタリング

レガシーコードの大規模なリファクタリングなど、高度な判断が必要なタスクに最適です。

```bash
claude --model opus
> この10年前のコードベースを、現代的なアーキテクチャパターンに移行する計画を立ててください
```

### サブエージェントでの選択的利用

特定のサブエージェントのみ Opus を使用することも可能です。

```bash
claude
> Task ツールを使って、Opus モデルで複雑な設計計画を立ててください
```

Task ツールの `model` パラメータで `"opus"` を指定することで、そのサブエージェントのみ Opus 4.6 を使用できます。

## 注意点

- Opus モデルは Sonnet や Haiku と比較してコストが高くなります
- 処理時間も比較的長くなる傾向があります
- 単純なタスクには Sonnet や Haiku の方が効率的な場合があります
- デフォルトのモデルは Sonnet 4.5 のままです
- API キーの使用制限（Rate Limit）に注意してください
