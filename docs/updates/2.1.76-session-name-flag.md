# -n / --name フラグ：起動時にセッション表示名を設定

## 原文（日本語に翻訳）

起動時にセッションの表示名を設定するための `-n` / `--name <name>` CLIフラグを追加

## 原文（英語）

Added `-n` / `--name <name>` CLI flag to set a display name for the session at startup

## 概要

Claude Code v2.1.76で、セッション起動時に表示名を設定できる `-n` / `--name` CLIフラグが追加されました。これは起動後に `/session rename` を実行するのと同等ですが、コマンドライン引数として指定できるため、スクリプトやターミナルマルチプレクサ（tmux、Zellijなど）から起動する際にインタラクティブな操作なしでセッション名を付与できます。

## 基本的な使い方

```bash
# セッション名を指定して起動
claude -n my-feature

# ロングオプションでも指定可能
claude --name auth-refactor
```

起動後、プロンプトバーにセッション名が表示されます。名前を付けたセッションは `--resume` で簡単に復帰できます：

```bash
# 名前付きセッションを再開
claude --resume auth-refactor
```

## 実践例

### tmuxで複数セッションを管理する

tmuxのペインごとに異なるタスクを実行する際に、セッション名で識別しやすくなります：

```bash
# tmuxペイン1: フロントエンド作業
tmux send-keys -t 0 "claude -n frontend-auth" Enter

# tmuxペイン2: バックエンドAPI作業
tmux send-keys -t 1 "claude -n backend-api" Enter

# tmuxペイン3: テスト作成
tmux send-keys -t 2 "claude -n test-coverage" Enter
```

### スクリプトからの自動起動

CI/CDパイプラインやシェルスクリプトからClaude Codeを起動する際にセッション名を設定：

```bash
#!/bin/bash
# タスクごとにClaude Codeセッションを起動するスクリプト
TASK_NAME="deploy-$(date +%Y%m%d-%H%M%S)"

claude -n "$TASK_NAME" -p "現在のブランチの変更をレビューしてください"
```

### worktreeフラグとの組み合わせ

worktreeとセッション名を同時に指定して、並列開発のセッションを管理：

```bash
# 機能開発用のworktreeを名前付きセッションで起動
claude --worktree feature-auth -n "auth-feature-dev"

# バグ修正用のworktreeを名前付きセッションで起動
claude --worktree bugfix-123 -n "bugfix-login-crash"
```

### /resume でのセッション復帰

名前を付けたセッションは、後から簡単に復帰できます：

```bash
# コマンドラインから名前で復帰
claude --resume frontend-auth

# セッション内から /resume で復帰
/resume frontend-auth
```

## 注意点

- **セッション名はプロンプトバーに表示される**: 起動時に指定した名前がプロンプトバーに表示され、セッションの識別に役立つ
- **`/rename` との違い**: `-n` フラグは起動時に一度だけ名前を設定するが、セッション中は `/rename` でいつでも名前を変更できる
- **セッションの復帰に便利**: 名前を付けておくと `claude --resume <name>` で素早く復帰できる。名前を付けていないセッションは初回のプロンプト内容で識別する必要がある
- **ユニークな名前を推奨**: 同じ名前のセッションが複数ある場合、`--resume` ではピッカーが表示される
