## 原文（日本語に翻訳）

現在のセッションでプラグインの `.zip` アーカイブをURLから取得する `--plugin-url <url>` フラグを追加しました。

## 原文（英語）

Added `--plugin-url <url>` flag to fetch a plugin `.zip` archive from a URL for the current session

## 概要

`--plugin-url` フラグを使用することで、ローカルにインストールされていないプラグインをURLから直接取得し、現在のセッションで利用することができます。GitHub ReleaseやCDNなどにホストされたプラグインを即座に試すことができ、インストール不要での一時利用に最適です。

## 基本的な使い方

```bash
claude --plugin-url https://example.com/my-plugin.zip
```

URLに `.zip` アーカイブ形式のプラグインを指定することで、そのセッション内でプラグインが有効化されます。

## 実践例

### GitHub Releaseからプラグインを取得する

```bash
claude --plugin-url https://github.com/your-org/your-plugin/releases/download/v1.0.0/plugin.zip
```

GitHubのリリースページに公開されたプラグインを直接指定して使用できます。

### CDNホスティングのプラグインを利用する

```bash
claude --plugin-url https://cdn.example.com/plugins/analyzer-v2.3.zip --print "このプロジェクトを分析してください"
```

`--print` フラグと組み合わせることで、URLプラグインを使った非インタラクティブな処理も可能です。

### 複数のフラグと組み合わせる

```bash
claude --plugin-url https://example.com/plugin.zip --model claude-opus-4-5
```

他のCLIフラグと併用して、モデル指定とプラグインの一時利用を同時に行えます。

## 注意点

- プラグインはセッション限定で有効化されます。永続的なインストールには `claude plugin install` コマンドを使用してください。
- URLは公開アクセス可能である必要があります。認証が必要なURLには対応していない場合があります。
- ダウンロードするプラグインのセキュリティには十分注意してください。信頼できるソースからのみ取得することを推奨します。
- `.zip` 形式のアーカイブのみサポートされています。
