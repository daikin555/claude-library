## 原文（日本語訳）

ターミナルのリサイズや大量の出力バーストにより、以前の会話履歴が繰り返し表示されるインラインモードのスクロールバック重複問題を修正しました。

## 原文（英語）

Fixed scrollback duplication in inline mode where resizing the terminal or large output bursts would repeat earlier conversation history

## 概要

Claude Codeのインラインモード（フルスクリーンモードではなく、通常のターミナルに組み込まれた状態）で使用中に、ターミナルウィンドウのサイズ変更や大量の出力が一度に行われた際、以前の会話履歴が重複して表示される問題がありました。v2.1.116ではスクロールバックの描画ロジックが修正され、このような重複表示が発生しなくなりました。

## 基本的な使い方

この修正は自動的に適用されます。インラインモードでClaude Codeを使用する際に効果があります。

```bash
# インラインモードでClaude Codeを使用
claude

# ターミナルウィンドウをリサイズしても会話履歴が重複しなくなった
```

## 実践例

### ターミナルウィンドウのリサイズ後

```bash
# 作業中にターミナルウィンドウのサイズを変更する
# → 以前の会話履歴が重複して表示される問題が解消された
# → クリーンな表示のままスクロールできる
```

### 大量の出力を含むタスクの実行後

```bash
claude "このプロジェクトの全ファイルを一覧表示して内容を分析してください"
# → 大量の出力が生成されても会話履歴の重複なし
```

## 注意点

- この問題はインラインモード（`--no-fullscreen` または環境によってはデフォルト）に固有のものでした
- フルスクリーンモード（`/focus` など）では発生しない問題でした
- ターミナルエミュレーターの種類に関わらず、インラインモードで発生していました
