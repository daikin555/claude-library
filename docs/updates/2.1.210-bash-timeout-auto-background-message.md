## 原文（日本語に翻訳）

コマンドがタイムアウトに達し自動的にバックグラウンド化された際のBash/PowerShellツールのメッセージを改善し、モデルがハング状態と明示的なバックグラウンド実行要求とを区別できるようにしました。

## 原文（英語）

Improved the Bash/PowerShell tool message when a command hits its timeout and is auto-backgrounded, so the model can distinguish a hang from an explicit background request

## 概要

Bash/PowerShellツールでコマンドが指定したタイムアウト（デフォルト2分、最大10分）に達すると、`run_in_background: true` を明示していなくても自動的にバックグラウンドタスクへ移行する仕組みがあります。しかし従来は、この「タイムアウトによる自動バックグラウンド化」と「モデル自身が最初から `run_in_background: true` を指定していたケース」のツール結果メッセージがほぼ同じで、Claudeがそのコマンドが単に長時間実行を意図したものなのか、実際にハングしている可能性があるのかを区別しにくいという問題がありました。v2.1.210では、タイムアウトによる自動バックグラウンド化の際に、その旨を明示したメッセージが返されるようになり、Claudeがハングの可能性を踏まえた適切な判断（様子見・出力確認・タスクの停止など）を行いやすくなりました。

## 基本的な使い方

### 修正前の問題

```bash
# 例: 応答が返ってこないコマンドを実行させてしまうケース
claude

> "sleep 999999" のような固まりそうなコマンドを実行して

# タイムアウト（デフォルト120000ms）に達すると
# 自動的にバックグラウンドタスクへ移行するが、
# 修正前はメッセージが以下のように
# 「意図的にバックグラウンド化された」場合と
# ほぼ見分けがつかなかった:
#
# "Command running in background (shell id: bash_1)"
```

### 修正後の動作

```bash
# 修正後は、タイムアウトによる自動バックグラウンド化の場合、
# その理由が明示されたメッセージが返る:
#
# "Command timed out after 120000ms and was automatically
#  moved to background (shell id: bash_1). This may indicate
#  the command is hanging rather than intentionally
#  long-running. Check output or consider killing the task."
#
# Claudeはこのメッセージを見て、
# 「単なる長時間ジョブ」なのか「ハングの疑いがある」のかを
# 区別した上で、出力確認や停止判断を行える
```

## 実践例

### ビルドコマンドが想定より長くかかるケース

```bash
claude

> npm run build を実行して

# ビルドが2分（デフォルトタイムアウト）を超えた場合、
# 自動的にバックグラウンド化され、
# タイムアウト起因である旨が明示される

# Claudeはこれを受けて:
# 1. しばらく待ってから /tasks や出力確認で進捗を見る
# 2. 明らかに固まっている兆候（出力が全く増えない等）があれば
#    タスクの停止を提案する
# といった、より適切な次の行動を選べるようになった
```

### 開発サーバーの意図的なバックグラウンド起動との違い

```bash
claude

> npm run dev をバックグラウンドで起動して

# この場合、Claudeは最初から run_in_background: true を
# 指定して実行するため、
# 「タイムアウトによる自動バックグラウンド化」の
# メッセージは表示されない
#
# → 「意図した長時間プロセス」と
#    「タイムアウトで移行した可能性のあるハング」が
#    メッセージ上ではっきり区別できる
```

### タイムアウト値のカスタマイズと組み合わせる

```bash
# 環境変数でデフォルト・最大タイムアウトを調整できる
export BASH_DEFAULT_TIMEOUT_MS=60000
export BASH_MAX_TIMEOUT_MS=300000

claude

> データベースのマイグレーションスクリプトを実行して

# 60秒でタイムアウトし自動バックグラウンド化された場合も、
# 改善されたメッセージにより
# Claudeが「異常なハングの可能性」を認識した上で
# 適切なフォローアップ（ログ確認・停止提案）を行いやすくなる
```

## 注意点

- この修正はClaude Code v2.1.210で実装されました
- タイムアウトのデフォルト値は120000ms（2分）、最大値は600000ms（10分）で、`BASH_DEFAULT_TIMEOUT_MS` / `BASH_MAX_TIMEOUT_MS` 環境変数で調整できます
- 自動バックグラウンド化された既存タスクは `/tasks`（バックグラウンドタスク一覧）から確認・停止できます
- この改善はメッセージの明確化が主眼であり、タイムアウトの挙動そのもの（自動的にバックグラウンドへ移行する仕組み）自体に変更はありません
- モデルがメッセージを踏まえてどう振る舞うかは会話の文脈にも依存するため、明確化されたからといって常に最適な判断が保証されるわけではありません。長時間かかることが分かっているコマンドは、最初から `run_in_background: true` を使うか、タイムアウト値を調整することが引き続き推奨されます
