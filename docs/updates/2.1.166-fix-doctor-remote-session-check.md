## 原文（日本語に翻訳）

リモートセッション内で `/doctor` を実行した際に「Not inside a remote session（リモートセッション外）」という矛盾したチェック失敗が表示されるバグを修正しました。

## 原文（英語）

Fixed `/doctor` showing a contradictory failed "Not inside a remote session" check when run inside a remote session

## 概要

`/doctor` コマンド（診断ツール）をリモートセッション内で実行した際に、「リモートセッション外にいない」という矛盾した失敗メッセージが表示されるバグを修正しました。修正後は、リモートセッション内で実行された場合に正しい診断結果が表示されます。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
# リモートセッション内で /doctor を実行
/doctor
# → 修正後は矛盾したエラーなく正しい診断結果が表示される
```

## 実践例

### /doctor の使い方

`/doctor` は Claude Code の設定・接続・環境を診断するコマンドです：

```bash
# 通常のセッションで実行
/doctor

# リモートセッション内で実行（修正後は正常動作）
# claude remote または claude agents から接続したセッション内で
/doctor
```

### 修正前後の比較

**修正前（リモートセッション内）：**
```
✗ Not inside a remote session  ← 矛盾したエラー（実際はリモートセッション内）
✓ API connection OK
...
```

**修正後（リモートセッション内）：**
```
✓ Inside a remote session      ← 正しい診断結果
✓ API connection OK
...
```

## 注意点

- この修正はリモートセッション環境での診断精度を改善するものです
- `/doctor` が表示する他のチェック項目への影響はありません
