## 原文（日本語に翻訳）

エンタープライズ向け: `managed-mcp.json` と一緒にclaude.aiクラウドMCPコネクターをロードするための `allowAllClaudeAiMcps` マネージド設定を追加しました。

## 原文（英語）

> Enterprise: added the `allowAllClaudeAiMcps` managed setting to load claude.ai cloud MCP connectors alongside `managed-mcp.json`

## 概要

エンタープライズ向けの新しいマネージド設定 `allowAllClaudeAiMcps` が追加されました。この設定を有効にすると、`managed-mcp.json` で定義したサーバーに加え、claude.aiのクラウドMCPコネクターを一括でロードできます。IT管理者が組織全体のMCP設定を一元管理しながら、claude.aiのクラウドコネクターも併用できるようになります。

## 基本的な使い方

エンタープライズのマネージド設定ファイルに `allowAllClaudeAiMcps` を追加します。

**macOS**: `/Library/Application Support/ClaudeCode/managed-settings.json`
**Linux**: `/etc/claude-code/managed-settings.json`

```json
{
  "allowAllClaudeAiMcps": true,
  "mcpServers": {
    "custom-internal-tool": {
      "command": "npx",
      "args": ["-y", "@company/internal-mcp-server"]
    }
  }
}
```

## 実践例

### `managed-mcp.json` とclaude.ai MCPを併用する

社内カスタムMCPサーバーを `managed-mcp.json` で管理しつつ、claude.aiのクラウドコネクター（GitHubやJiraなど）もユーザーが利用できるようにする構成です。

**`/Library/Application Support/ClaudeCode/managed-mcp.json`**:
```json
{
  "mcpServers": {
    "internal-db": {
      "command": "/usr/local/bin/internal-db-mcp",
      "args": ["--read-only"]
    },
    "internal-docs": {
      "command": "node",
      "args": ["/opt/company/docs-mcp/index.js"]
    }
  }
}
```

**`/Library/Application Support/ClaudeCode/managed-settings.json`**:
```json
{
  "allowAllClaudeAiMcps": true
}
```

この設定により：
- 社内の `internal-db`・`internal-docs` サーバーが管理者定義で利用可能
- claude.aiのクラウドMCPコネクターも同時に利用可能

### ユーザーへの影響

`allowAllClaudeAiMcps` が有効な場合、エンドユーザーはclaude.aiにサインインすることでクラウドコネクターに自動アクセスできます。追加の設定操作は不要です。

## 注意点

- この設定はシステムレベルのファイルへの書き込み権限が必要なため、管理者権限が必要です
- `allowedMcpServers` / `deniedMcpServers` のフィルタリング設定はclaude.aiコネクターにも適用されます
- エンタープライズプランのみ対象です
