## 原文（日本語に翻訳）

OAuthトークンがClaude Codeのログインとは異なるアカウントのものである場合に、ChromeのClaudeが接続に失敗する問題を修正しました。

## 原文（英語）

Fixed Claude in Chrome silently failing to connect when the OAuth token belongs to a different account than the Claude Code login

## 概要

ChromeでClaude（claude.ai）を使用しながらClaude Code CLIと連携する際、Chromeにログインしているアカウントと Claude Code CLIのログインアカウントが異なる場合に、接続がサイレントに失敗する問題が修正されました。以前はエラーメッセージなしに接続が失敗していましたが、今後は適切なエラーメッセージが表示されます。

## 基本的な使い方

Chrome連携（Claude in Chrome）機能の使用時に自動的に適用される修正です。

**影響するシナリオ:**
- 複数のGoogleアカウントをChromeで使用している
- 個人アカウントでclaude.aiにアクセスし、仕事用アカウントでClaude Code CLIにログインしている

## 実践例

### 問題が発生していた状況

```
状況:
- Chrome: personal@example.com でclaude.aiにログイン
- Claude Code CLI: work@company.com でログイン

以前の動作:
- /teleport または Claude in Chrome を起動
- → サイレントに失敗（エラーなし）
- → ユーザーは原因がわからず困惑

改善後の動作:
- → "Account mismatch: Chrome (personal@example.com) and
     Claude Code (work@company.com) are different accounts.
     Please use the same account for both." などのメッセージが表示
```

### 対処法

アカウントを一致させるための手順:

```bash
# オプション1: Claude Code CLIのアカウントを変更
claude auth logout
claude auth login  # Chromeと同じアカウントでログイン

# オプション2: Chromeのclaude.aiアカウントを変更
# Chromeで claude.ai にアクセス → アカウント切り替え
```

### 複数アカウント環境での推奨設定

```bash
# 作業ごとにアカウントを切り替える場合
# profile オプション（または設定）で管理

# 現在のアカウントを確認
claude auth status
```

## 注意点

- この問題はClaude in Chrome機能（`/teleport` 等）使用時にのみ発生していました
- 通常のClaude Code CLI使用（API経由）には影響しません
- 修正後は適切なエラーメッセージが表示されるようになり、問題の原因が一目でわかります
- 複数アカウントを管理するチームでは、どのアカウントを使用するか明示的に決めておくことを推奨します
