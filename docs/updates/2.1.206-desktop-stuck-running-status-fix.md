## 原文（日本語訳）

ターンの途中でスラッシュコマンドが送信された後、デスクトップセッションが「実行中（running）」の表示のまま固まってしまう不具合を修正した

## 原文（英語）

Fixed desktop sessions getting stuck showing "running" after a slash command was sent mid-turn

## 概要

Claude Codeのデスクトップアプリでは、Claudeが応答を生成している最中（ターンの途中）に別のスラッシュコマンドを送信すると、セッションのステータス表示が「実行中」のまま更新されなくなり、実際には処理が終わっているにもかかわらず、画面上は実行中のままに見えてしまう不具合がありました。今回のアップデートでこの状態管理の不具合が修正され、ターン中にスラッシュコマンドを送っても、ステータス表示が正しいタイミングで更新されるようになりました。

## 基本的な使い方

特別な操作は不要です。デスクトップアプリでいつも通りセッションを使い、Claudeが応答中でも気になった時点でスラッシュコマンド（例: `/compact` や `/clear` など）を送信すれば問題ありません。

```text
/compact
```

## 実践例

### 長い応答の途中でコンテキストを整理したい場合

Claudeが長いタスクを実行している最中に、コンテキストが膨らんできたので `/compact` を割り込みで送りたいようなケースです。

```text
/compact
```

修正前は、このように応答途中でスラッシュコマンドを送ると、コマンド自体は処理されてもセッションのステータス表示が「実行中」のまま残ってしまい、実際に操作可能かどうか分かりづらい状態になることがありました。修正後は処理完了後にステータス表示が正しく更新されます。

### 応答中に別のセッション設定を変更したい場合

処理が実行中の間に `/model` などで設定を変更したい場合も同様の状況が起こり得ました。

```text
/model
```

修正後は、ターンの途中でコマンドを挟んでも、デスクトップアプリのステータス表示（実行中/待機中など）が処理の実態と一致するようになり、セッションが「固まったように見える」ことでユーザーが混乱するのを防げます。

## 注意点

- 本不具合はデスクトップアプリ固有のステータス表示の問題であり、ターミナル（CLI）版のClaude Codeには影響していませんでした。
- 表示が「実行中」のまま更新されない場合でも、内部的な処理自体は継続・完了していたケースが多く、実害としては表示上の混乱が主な問題でした。
