## 原文（日本語訳）

トランスクリプトモード（Ctrl+O）のフッターヒントが、104桁未満の幅のターミナルで折り返されてしまう不具合を修正しました。

## 原文（英語）

Fixed the transcript-mode (Ctrl+O) footer hint wrapping on terminals narrower than 104 columns

## 概要

`Ctrl+O` で切り替える[トランスクリプトモード](https://code.claude.com/docs/en/fullscreen#search-and-review-the-conversation)では、画面下部に `/` で検索、`n`/`N` で次・前の一致へ移動、`g`/`G` で先頭・末尾へジャンプ、といった操作ヒントがフッターとして表示されます。今回の修正前は、幅が104桁未満の比較的狭いターミナルでこのフッターのヒント行が折り返され、レイアウトが崩れてしまうことがありました。v2.1.216ではこの折り返しが修正され、狭いターミナル幅でもフッターが崩れずに表示されるようになりました。

## 基本的な使い方

見た目の修正であり、特別な設定は不要です。Claude Codeをv2.1.216以降にアップデートし、トランスクリプトモードを開くだけで改善された表示を確認できます。

```
Ctrl+O
```

## 実践例

### 狭いターミナルでのトランスクリプトモード表示確認

ターミナル幅を104桁未満に狭めた状態で `Ctrl+O` を押し、フッターのヒント（検索・ジャンプ・スクロールのキー案内）が折り返さずに収まっているかを確認します。

```
# ターミナル幅を104桁未満にリサイズしてから
Ctrl+O
```

### トランスクリプト内検索の利用

トランスクリプトモードでは `/` キーで会話内検索ができ、`n`/`N` で一致箇所を前後に移動できます。狭いターミナルでもこれらの操作案内が正しく表示されるようになりました。

```
Ctrl+O
/
検索したいキーワード
Enter
```

### 通常表示に戻す

トランスクリプトモードから戻るには `Ctrl+O`、`Esc`、または `q` を押します。

```
Esc
```

## 注意点

- 104桁は目安の閾値であり、実際の折り返し発生条件はターミナルフォントや使用言語（全角文字の有無など）によっても変わる可能性があります
- フルスクリーンレンダリングはリサーチプレビュー機能のため、他の端末幅・高さに関する表示崩れが見つかった場合は `/feedback` で報告することが推奨されます
- この修正はフッターの表示レイアウトに関するものであり、トランスクリプトモードの検索・ナビゲーション機能自体（`/`、`n`/`N`、`{`/`}` など）に変更はありません
