# TaskCreate の信頼性向上：不正入力の自動修復とエラーメッセージ改善

## 原文（日本語に翻訳）

`TaskCreate` の信頼性を改善：不正な入力を自動修復し、未ロードのツールのバリデーションエラーにスキーマを含めるように

## 原文（英語）

Improved `TaskCreate` reliability: malformed inputs are repaired automatically and validation errors for unloaded tools include the schema

## 概要

`TaskCreate` ツールの信頼性が向上しました。モデルが不正な形式の入力を渡した場合でも自動的に修復を試み、タスク作成の失敗を減らします。また、未ロードのツールに対するバリデーションエラーが発生した際に、エラーメッセージにスキーマ情報が含まれるようになり、ワークフローのデバッグが容易になります。複雑なマルチエージェントワークフローでのタスク管理の安定性が向上します。

## 基本的な使い方

`TaskCreate` はワークフロースクリプト内で使用されるツールです。モデルがタスク作成を試みる際に、これまでより堅牢に動作します。

### 以前の問題（修正済み）

以前は、モデルが若干不正なパラメータ形式でタスクを作成しようとした場合、バリデーションエラーで失敗することがありました。

### 改善後の動作

不正な入力が検出されると、自動的に修復を試みてタスク作成を完了します。修復が不可能な場合は、より詳細なエラーメッセージ（スキーマ情報付き）が返されます。

## 実践例

### ワークフロースクリプトでのタスク作成

ワークフロースクリプト内での `TaskCreate` の典型的な使用例：

```javascript
// ワークフロースクリプト例
export const meta = {
  name: 'code-review-workflow',
  description: 'コードレビューを並行実行するワークフロー',
}

const files = ['src/auth.ts', 'src/api.ts', 'src/utils.ts']

const results = await pipeline(
  files,
  file => agent(`${file} のセキュリティレビューをしてください`, {
    label: `review:${file}`,
    phase: 'Review',
  })
)
```

このような場合に `TaskCreate` の内部呼び出しが安定して動作するようになりました。

### エラーメッセージの改善

未ロードのツールに対してバリデーションエラーが発生した際、以前：

```
Error: Validation failed for tool 'ToolSearch'
```

改善後：

```
Error: Validation failed for tool 'ToolSearch'
Schema: { "type": "object", "properties": { "query": { "type": "string" }, ... } }
```

スキーマ情報が含まれることで、モデルが正しいパラメータを渡せるよう修正しやすくなります。

### 複雑なマルチエージェントワークフローでの安定動作

大量のタスクを並行作成するワークフロー：

```javascript
const items = Array.from({ length: 50 }, (_, i) => `item-${i}`)

const results = await parallel(
  items.map(item => () =>
    agent(`${item} を処理してください`, { phase: 'Process' })
  )
)
```

以前はタスク作成の信頼性問題で一部のタスクが失敗することがありましたが、今回の改善でより安定して動作します。

## 注意点

- この改善はマルチエージェントワークフロー（`Workflow` ツール使用時）に特に恩恵があります
- 自動修復は軽微な形式の問題に対して機能します。根本的に不正なパラメータには修復が適用されません
- スキーマ情報付きのエラーメッセージは、ワークフロー開発者がデバッグする際の手助けになります
- `TaskCreate` は主にモデルとワークフローランタイムが内部的に使用するツールです
