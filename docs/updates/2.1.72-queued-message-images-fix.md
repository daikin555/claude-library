## 原文（日本語に翻訳）

- キューに入れたメッセージに添付画像が表示されない問題、および↑キーでキューメッセージを編集する際に画像が失われる問題を修正

## 原文（英語）

- Fixed queued messages not showing attached images, and images being lost when pressing ↑ to edit a queued message

## 概要

Claudeが処理中にキューに入れたメッセージに画像を添付した場合、2つの問題がありました。1つ目は、キューに入れたメッセージに添付された画像がUI上に表示されない問題です。2つ目は、↑キーを押してキューメッセージを編集しようとすると、添付していた画像が消えてしまう問題です。この修正により、キューメッセージの画像添付が正しく動作するようになりました。

## 基本的な使い方

Claudeの応答待ち中にメッセージをキューに入れる際、画像を添付できます:

```
claude> [処理中...]

# 処理中に次のメッセージを画像付きで入力（キューに追加される）
# → 画像が正しく表示される
# → ↑キーで編集しても画像が保持される
```

## 実践例

### スクリーンショット付きのバグ報告

処理待ち中にスクリーンショットを添付したメッセージをキューに入れる:

```
claude> 前のタスクを処理中...

# キューメッセージ: [スクリーンショット添付]
# 「このスクリーンショットのUIバグを修正して」
# → 画像が正しくキューメッセージに表示される
```

### キューメッセージの編集

キューに入れたメッセージを↑キーで編集する際に画像が保持されます:

```
claude> [処理中...]

# メッセージを入力して画像を添付
# → ↑キーで編集画面を開く
# → 画像が維持されたままテキストを修正可能
# → 修正後もキューに画像付きで保持される
```

## 注意点

- この修正はキューに入れたメッセージ（Claudeが処理中に入力したメッセージ）に限定されます
- 通常のメッセージ送信時の画像添付には影響ありません
- 画像の添付方法（ペースト、ドラッグ&ドロップなど）に関係なく修正が適用されます
- 複数の画像を添付している場合もすべて正しく保持されます
