# Opus 4.6のコンテキストウィンドウが1Mトークンに：Max・Team・Enterpriseプランでデフォルト適用

## 原文（日本語に翻訳）

Max、Team、EnterpriseプランでOpus 4.6の1Mコンテキストウィンドウをデフォルトで有効化（以前は追加使用量が必要でした）

## 原文（英語）

Added 1M context window for Opus 4.6 by default for Max, Team, and Enterprise plans (previously required extra usage)

## 概要

Claude Code v2.1.75から、Max・Team・EnterpriseプランのユーザーはOpus 4.6を使用する際、追加コストなしで100万トークンのコンテキストウィンドウがデフォルトで利用できるようになりました。以前は`/extra-usage`を有効にする必要がありましたが、今後は自動的に適用されます。これにより、大規模なコードベースや長大なドキュメントを丸ごとコンテキストに含めて作業することが可能になります。

## 基本的な使い方

特別な設定は不要です。Max・Team・EnterpriseプランでOpus 4.6を使用するだけで、1Mコンテキストウィンドウが自動的に有効になります。

```bash
# Opus 4.6をデフォルトモデルとして使用（最大1Mトークンのコンテキスト）
claude --model claude-opus-4-6

# または /model コマンドでセッション中に切り替え
/model claude-opus-4-6
```

1Mコンテキストウィンドウを無効にしたい場合は環境変数で制御できます：

```bash
export CLAUDE_CODE_DISABLE_1M_CONTEXT=true
claude
```

## 実践例

### 大規模コードベースを丸ごとコンテキストに含める

```bash
# プロジェクト全体（数万行のコード）を一度に分析
claude --model claude-opus-4-6

# Claude Code内でプロジェクト全体の構造を把握させる
> このリポジトリ全体のアーキテクチャを説明してください。
> src/以下のすべてのファイルを確認してから、リファクタリング計画を立ててください。
```

### 長大なドキュメントや仕様書の処理

```bash
# 大量のAPIドキュメントや仕様書を一度に読み込んで処理
> この1000ページの仕様書に基づいて、実装計画を作成してください。
> （複数の大きなファイルを @ファイル名 で添付）
```

### 長期的なエージェントタスク

1Mコンテキストウィンドウにより、コンテキストコンパクションが起きにくくなり、長時間にわたるエージェントタスクが途切れなく実行できます：

```bash
# 複雑なマルチステップタスクをコンテキスト切れなしに実行
> この大規模なマイグレーション作業を最初から最後まで実行してください。
> 途中でコンテキストが失われないよう、100以上のファイルを処理してください。
```

### ProプランユーザーがオプトインでOpus 1Mを使用

ProプランユーザーはデフォルトではOpus 4.6の1Mコンテキストウィンドウが使えませんが、オプトインは可能です：

```bash
/extra-usage
# → Opus 4.6の1Mコンテキストウィンドウが有効になる（追加コスト発生の可能性あり）
```

## 注意点

- **対象プラン**: Max・Team・Enterprise プランのみデフォルト有効。Proプランユーザーは `/extra-usage` でオプトイン可能
- **価格**: 追加コストなし。1Mトークン全体にわたって通常の料金体系（Opus 4.6は入力$5/出力$25 per 1Mトークン）が適用されます
- **出力トークン**: Opus 4.6の最大出力は128Kトークン（以前の64Kから倍増）
- **エフォートレベル**: MaxおよびTeamプランでは、Opus 4.6はデフォルトで「medium」エフォートで動作します
- **無効化**: `CLAUDE_CODE_DISABLE_1M_CONTEXT=true` 環境変数で1Mコンテキストウィンドウを無効にできます
- **ベンチマーク**: Opus 4.6はMRCR v2で78.3%のスコアを達成しており、1Mトークン全体にわたって高い精度を維持します
