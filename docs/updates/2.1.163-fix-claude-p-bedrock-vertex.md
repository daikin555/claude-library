## 原文（日本語に翻訳）

`CI=true` かつ Anthropic API キーが未設定の場合に、Bedrock/Vertex/Foundry で `claude -p` が "ANTHROPIC_API_KEY required" エラーで失敗する問題を修正。

## 原文（英語）

Fixed `claude -p` failing with "ANTHROPIC_API_KEY required" on Bedrock/Vertex/Foundry when `CI=true` and no Anthropic API key is set

## 概要

Amazon Bedrock、Google Vertex AI、または Azure Foundry 経由で Claude Code を使用する環境で、`CI=true` が設定されているときに `claude -p` が `ANTHROPIC_API_KEY required` エラーで失敗するバグが修正されました。Bedrock/Vertex/Foundry を使用する場合、Anthropic API キーは不要ですが、CI 環境で誤ってこのチェックがトリガーされていました。

## 基本的な使い方

Bedrock/Vertex/Foundry を使用する CI/CD 環境での `claude -p` が正常に動作するようになりました：

```bash
# Bedrock 環境（CI=true）での使用
export CI=true
echo "コードをレビュー" | claude -p --provider bedrock
# → 以前: "ANTHROPIC_API_KEY required" エラー
# → 修正後: 正常に動作
```

## 実践例

### GitHub Actions で Bedrock を使用する場合

```yaml
# .github/workflows/code-review.yml
- name: Claude Code レビュー
  env:
    CI: "true"
    AWS_REGION: us-east-1
    # ANTHROPIC_API_KEY は不要（Bedrock 使用）
  run: |
    git diff HEAD~1 | claude -p "このPRをレビューしてください"
```

修正前はこの設定で `ANTHROPIC_API_KEY required` エラーが発生していましたが、修正後は正常に動作します。

### Vertex AI を使用する CI パイプライン

```bash
# CI=true の環境（Jenkins、GitLab CI 等）
export CI=true
export ANTHROPIC_VERTEX_PROJECT_ID="my-gcp-project"
export CLOUD_ML_REGION="us-east5"

echo "セキュリティチェック" | claude -p "このコードに脆弱性がないか確認してください"
```

## 注意点

- この修正は Bedrock、Vertex AI、Azure Foundry を使用する場合に適用されます。
- Anthropic API（api.anthropic.com）を直接使用する場合は、引き続き `ANTHROPIC_API_KEY` が必要です。
- `CI=true` の環境変数は GitHub Actions、GitLab CI など多くの CI システムで自動的に設定されます。
