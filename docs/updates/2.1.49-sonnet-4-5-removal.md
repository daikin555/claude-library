## 原文（日本語に翻訳）

1Mコンテキストを持つSonnet 4.5がMaxプランから削除され、現在1Mコンテキストを持つフロンティアのSonnet 4.6モデルに置き換えられます。`/model` で切り替えてください。

## 原文（英語）

Sonnet 4.5 with 1M context is being removed from the Max plan in favor of our frontier Sonnet 4.6 model, which now has 1M context. Please switch in /model.

## 概要

Claude Codeのmaxプランにおいて、1Mトークンコンテキストウィンドウを持つSonnet 4.5が廃止され、同じく1Mトークンコンテキストを持つ最新のSonnet 4.6に統合されます。Sonnet 4.6はフロンティアモデルとして性能が向上しており、1Mコンテキストウィンドウを引き続き利用できます。`/model` コマンドでモデルを切り替えてください。

## 基本的な使い方

```bash
claude

# 現在のモデルを確認
/model

# Sonnet 4.6に切り替え
/model claude-sonnet-4-6

# または /model コマンドでモデルピッカーを表示
# リストからSonnet 4.6を選択
```

## 実践例

### ユースケース1: モデルの確認と移行

現在Sonnet 4.5を使用している場合の移行手順。

```bash
claude

# 現在のモデルを確認
/model
# → 現在: claude-sonnet-4-5 (または類似の名前)

# Sonnet 4.6に切り替え
/model claude-sonnet-4-6
# → モデルが切り替わり、1Mコンテキストも引き続き利用可能

# 変更を確認
/model
# → 現在: claude-sonnet-4-6 ✓
```

### ユースケース2: 設定ファイルでのモデル指定

プロジェクトの設定でモデルを指定している場合の更新。

```json
// .claude/settings.json
{
  // "model": "claude-sonnet-4-5"  ← 古い設定
  "model": "claude-sonnet-4-6"     // ← 新しい設定に変更
}
```

```bash
# 設定を更新後、Claude Codeを再起動
claude
# → Sonnet 4.6が自動的に使用される
```

### ユースケース3: エージェント定義のモデル更新

エージェント定義でSonnet 4.5を使用している場合。

```yaml
# .claude/agents/my-agent.md を更新
---
name: my-agent
description: "高性能タスク用エージェント"
# model: claude-sonnet-4-5  ← 古い設定
model: claude-sonnet-4-6    # ← 新しい設定に更新
---

エージェントの指示内容...
```

## 注意点

- Sonnet 4.5の1Mコンテキスト版はMaxプランから削除されます。移行期限については公式アナウンスを確認してください。
- Sonnet 4.6は1Mコンテキストウィンドウをサポートしており、大規模なコードベースの処理も引き続き可能です。
- Sonnet 4.6はSonnet 4.5と比較してパフォーマンスが向上しています。既存のワークフローはそのまま利用できます。
- Claude Codeのデフォルトモデルは `/model` コマンドまたは設定ファイルで変更できます。
- 詳細な移行スケジュールはAnthropicの公式アナウンスを参照してください。
