## 原文（日本語に翻訳）

`/ultrareview`のエラーフィードバックを改善し、不正な引数を渡した場合にClaudeがそれを同じまま再送するのではなく、正しい引数に修正できるようにしました

## 原文（英語）

Improved `/ultrareview` error feedback so Claude can correct an invalid argument instead of retrying it unchanged

## 概要

`/ultrareview`はクラウド上でより深いコードレビューを実行するコマンドで、PR番号やブランチ名といった引数を指定できます。以前は、不正な引数（存在しないPR番号や誤った形式のブランチ指定など）を渡した際に返されるエラーメッセージの情報が十分でなく、Claudeがそのエラー内容から正しい引数を推測できずに、同じ不正な引数のままコマンドを再試行してしまうことがありました。この修正により、エラーフィードバックがより具体的になり、Claudeがエラー内容を理解して引数を自己修正したうえで再実行できるようになりました。

## 基本的な使い方

`/ultrareview`に不正な引数を渡した場合の挙動が変わるだけで、ユーザーが新たに行う操作はありません。通常どおりコマンドを実行するだけです。

```bash
/ultrareview #9999   # 存在しないPR番号など
```

## 実践例

### 存在しないPR番号を指定した場合の自動修正フロー

```bash
# 誤ってPR番号を取り違えて指定
/ultrareview #9999

# 実行結果（改善後）:
# 1. サーバーから「PR #9999 が見つかりません」といった具体的なエラーが返る
# 2. Claudeがエラー内容を読み取り、現在のブランチに紐づく正しいPR番号
#    （例: #123）を特定
# 3. Claudeが引数を修正して /ultrareview #123 を再実行し、レビューが成功する
```

修正前は、エラーメッセージの情報が乏しかったため、Claudeが同じ`#9999`のまま再試行を繰り返し、結果的にレビューが実行できないことがありました。

### ブランチ名の指定ミスからの回復

```bash
# 存在しないブランチ名を指定
/ultrareview branch:feature-typo

# 改善後のエラーフィードバックには、利用可能なブランチ候補などの
# 手がかりが含まれるため、Claudeが正しいブランチ名
# （例: feature-typo-fix）に修正して再実行できる
```

## 注意点

- Claude Code v2.1.218で改善
- あくまでエラーメッセージの情報量とClaudeの自己修正能力に関する改善であり、`/ultrareview`自体の権限やレビュー対象範囲を変更するものではない
- Claudeが自動修正できるのは、エラー内容から正しい値を合理的に推測できる場合に限られる。手がかりが全くない場合は、引き続きユーザーへの確認が必要になることがある
