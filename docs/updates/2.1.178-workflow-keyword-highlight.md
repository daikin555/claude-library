## 原文（日本語に翻訳）

ワークフロープロンプトのキーワードが紫のシマーハイライトを使用するように変更され、「run a workflow」や「workflow:」などの明示的なフレーズでのみトリガーされるようになりました。単に「workflow」という単語に言及しただけではトリガーされません。

## 原文（英語）

Changed the workflow prompt keyword to use a purple shimmer highlight and trigger only on explicit phrases like "run a workflow" or "workflow:", not on any mention of the word

## 概要

ワークフロー機能のトリガーキーワードが改善されました。以前は「workflow」という単語を含む文章でワークフローが誤って起動する場合がありましたが、今回の変更で「run a workflow」「workflow:」などの明示的なフレーズが必要になりました。また、ワークフローキーワードが紫のシマー（光沢）ハイライトで表示されるようになり、視覚的に区別しやすくなりました。

## 基本的な使い方

ワークフローを起動するには明示的なフレーズを使用します:

```
# ワークフローが起動するフレーズ
run a workflow to deploy the app
workflow: deploy-production
use a workflow for this task

# ワークフローが起動しないフレーズ（以前は起動していた）
explain how the workflow works
the CI/CD workflow is broken
review the GitHub Actions workflow
```

## 実践例

### 明示的なワークフロー起動

```
# ワークフローを実行
run a workflow to audit all files for security issues

# 特定のワークフローを指定
workflow: security-audit
```

上記のフレーズが紫のシマーでハイライトされ、ワークフローが起動します。

### 誤起動が防止されるケース

```
# コードレビューの会話（以前は誤起動の可能性があった）
「このGitHub Actionsのworkflowファイルを確認してください」

# 技術的な説明
「このシステムはどのようなworkflowで動作しますか？」
```

これらのメッセージではワークフローは起動せず、通常の会話として処理されます。

### ワークフローの命名規則

```
# workflow: の後にワークフロー名を指定
workflow: deploy-staging
workflow: run-tests
workflow: generate-reports
```

## 注意点

- トリガーフレーズは英語での入力が基本です。日本語でワークフローを起動する場合は「run a workflow」「workflow:」のような英語フレーズを含めてください
- 紫のシマーハイライトはキーワードが認識されたことの視覚フィードバックです
- ワークフロー機能はマルチエージェントオーケストレーションを行うため、多くのトークンを消費します。意図しない起動を防ぐこの変更は重要です
- `Workflow` ツールを使用するカスタムスキルやエージェントには影響しません
