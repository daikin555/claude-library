# [VSCode] /remote-controlコマンド：ブラウザやスマートフォンからセッションを継続

## 原文（日本語に翻訳）

[VSCode] `/remote-control` を追加 — セッションをclaude.ai/codeにブリッジして、ブラウザやスマートフォンから継続できます

## 原文（英語）

[VSCode] Added `/remote-control` — bridge your session to claude.ai/code to continue from a browser or phone

## 概要

VS Code拡張機能に新しい `/remote-control` コマンドが追加されました。このコマンドを実行すると、現在進行中のVS CodeセッションをClaude.ai/codeにブリッジし、ブラウザやスマートフォンから同じセッションを継続できるようになります。デスクトップを離れた後も作業を続けたい場合や、モバイルデバイスからレビューや指示を送りたい場合に非常に便利な機能です。

## 基本的な使い方

VS CodeのClaude Codeチャット欄で：

```
/remote-control
```

コマンドを実行すると、claude.ai/codeへのリンクが生成されます。そのリンクをブラウザやスマートフォンで開くと、現在のセッションに接続できます。

## 実践例

### デスクトップからモバイルへの引き継ぎ

1. VS CodeでClaude Codeを使って作業中
2. 外出が必要になった場合：
   ```
   /remote-control
   ```
3. 生成されたURLをスマートフォンで開く
4. スマートフォンのブラウザからセッションを継続

### レビューや承認作業のモバイル対応

```
/remote-control
```

実行後、生成されたURLをチームメンバーに共有することで、コードレビューや承認作業を別のデバイスから行うことも可能です。

### 長時間タスクの監視

大規模なリファクタリングや分析タスクを開始した後：

```
# VS Codeで重いタスクを開始してから
/remote-control
```

生成されたURLでモバイルから進捗を確認しながら、必要に応じて追加の指示を送ることができます。

## 注意点

- この機能はVS Code拡張機能専用です。CLIのClaude Codeでは使用できません
- ブリッジセッションにはclaude.ai/codeへのアクセスが必要です
- セキュリティのため、生成されたリンクを不特定多数と共有する場合は注意してください。リンクはあなたのセッションに直接アクセスできます
- ネットワーク接続が切れた場合、セッションが途切れる可能性があります
- モバイルでの操作はブラウザインターフェース経由となるため、VS Codeの一部機能は使用できません
