## 原文（日本語訳）

セッションが数分間アイドル状態のままになった後、Claude Code on the webが同じ質問を再度尋ね、ユーザーが行った回答を破棄してしまう問題を修正しました。

## 原文（英語）

Fixed Claude Code on the web re-asking the same question and dropping your answer after the session sat idle for a few minutes

## 概要

Claude Code on the web（ブラウザ版のClaude Code）では、セッションが数分間操作されずアイドル状態になった後に、既に回答済みの質問（AskUserQuestionなど）が再度表示され、その際にユーザーがすでに入力していた回答が失われてしまう不具合がありました。アイドル状態からの復帰時にセッション状態と表示側の同期がずれることが原因と見られ、今回の修正でこの再質問・回答消失の問題が解消されています。

## 基本的な使い方

この修正はClaude Code on the webの内部的なセッション同期処理に対するものであり、ユーザー側の追加設定は不要です。ブラウザでセッションを開いたまま数分間離席した後に戻ってきても、既に回答した質問が再表示されたり、入力した回答が消えたりすることがなくなります。

## 実践例

### 離席を挟む長時間タスクでの利用

Claude Code on the webでタスクを実行させている間に、質問への回答後にしばらく離席するケースは珍しくありません。修正後は、数分程度セッションがアイドル状態になっても、戻ってきたときに同じ質問が再表示されて回答が失われるといった事態を避けられます。

### モバイル・タブ切り替えを伴う運用

ブラウザのタブを切り替えたり、モバイル端末で他のアプリに切り替えたりして一時的にセッションから離れる使い方をしている場合も、この修正によりセッションの応答性が向上します。

### 回答が反映されているかの確認

万が一、アイドル状態からの復帰後に挙動に違和感がある場合は、セッションを開き直す（ページを再読み込みする）ことで最新のセッション状態を取得できます。

```
# ブラウザでClaude Code on the webのセッションページを再読み込みする
```

## 注意点

- この修正はClaude Code on the web（ブラウザ版）に固有のものであり、CLI版のセッションアイドル動作には直接関係しません
- アイドル時間が数分を大きく超える場合や、ネットワーク切断を伴う場合は、別途セッションの再接続が必要になることがあります
- 修正後も、意図的に質問への回答をやり直したい場合は、通常通り新しい回答を送信すれば上書きされます
