## 原文（日本語に翻訳）

Auto モードはオプトイン同意が不要になりました。

## 原文（英語）

Auto mode no longer requires opt-in consent

## 概要

Claude Code の Auto モード（`--permission-mode auto`）が、オプトイン同意なしに直接利用できるようになりました。Auto モードは別の安全分類器を使って低リスクの操作を自動承認し、高リスクな操作はブロックまたは確認を求めます。この変更によりオンボーディングが簡略化され、自動権限処理をすぐに活用できます。

## 基本的な使い方

```bash
# Auto モードで Claude Code を起動
claude --permission-mode auto

# セッション中に Shift+Tab でモードを切り替えることも可能
# （Default → Auto → Plan の順にサイクル）
```

## 実践例

### 自動化タスクへの活用

スクリプトや CI/CD パイプラインで低リスクの操作を自動承認しながら実行する例です。

```bash
# Auto モードで非インタラクティブ実行
claude --permission-mode auto --print "テストを実行してエラーを修正してください"
```

### 開発ワークフローでの効率化

繰り返しの確認ダイアログを減らし、開発に集中する例です。

```bash
# Auto モードで起動（同意プロンプトなし）
claude --permission-mode auto

# → 読み取り・軽微な変更などは自動承認される
# → 破壊的な操作（rm -rf など）は確認または拒否される
```

### Shift+Tab によるモード切り替え

セッション中に Auto モードへ切り替える例です。

```
# プロンプト入力中に Shift+Tab を押してモードを切り替え
# Default → Auto → Plan → Default ... の順にサイクル
```

## 注意点

- Auto モードでは、低リスクと判断されたツール操作が自動承認されます。高リスクな操作（ファイル削除、システムコマンドなど）は依然として確認または拒否されます
- 同一セッション内で 3 回連続または合計 20 回ブロックされると、手動確認モードにフォールバックします
- 古い `--enable-auto-mode` フラグは廃止されています（`--permission-mode auto` を使用してください）
- Auto モードの安全性は別の Sonnet 4.6 分類器によって担保されています
