## 原文（日本語に翻訳）

API がネストされた `cache_creation` 内訳経由でのみキャッシュ書き込みを報告する場合に、`cache_creation_input_tokens` がトランスクリプトおよび結果の使用量で 0 として報告される問題を修正しました。

## 原文（英語）

Fixed `cache_creation_input_tokens` reporting as 0 in transcript and result usage when the API reports cache writes only via the nested `cache_creation` breakdown

## 概要

Claude API がキャッシュ書き込みのトークン数をネストされた `cache_creation` フィールド経由でのみ提供するケースにおいて、トランスクリプトや結果の使用量レポートで `cache_creation_input_tokens` が 0 として表示されていた問題が修正されました。v2.1.152 以降は、キャッシュ作成トークン数が正確に集計・表示されます。

## 影響を受けていた状況

- プロンプトキャッシュが有効で、キャッシュ書き込みが発生している場合
- `/usage` コマンドでキャッシュ使用量を確認している場合
- API のレスポンスでネストされた `cache_creation` フィールドが使用されているモデルとのやり取り

## 注意点

- v2.1.152 以降では自動的に修正されており、特別な操作は不要です
- キャッシュ使用量の集計精度が向上するため、`/usage` での表示が変わる場合があります
