## 原文（日本語訳）

空のURLで設定されたMCPサーバーは、`/mcp` でこれまでの設定エラーではなく「未設定（not configured）」と表示されるようになった。

## 原文（英語）

MCP servers configured with an empty URL now show as "not configured" in `/mcp` instead of a config error

## 概要

HTTPやSSEなど、リモートトランスポートでMCPサーバーを設定する際、`url` フィールドが空文字のまま残ってしまうケースがあります。テンプレートを流用した設定ファイルや、環境ごとに値を後から埋める運用などで発生しやすい状態です。これまでこの状態は他の設定ミスと同様に「設定エラー」として扱われ、原因を切り分けにくいメッセージが表示されていましたが、今回の修正で `/mcp` 上に「未設定」というより明確なステータスが表示されるようになりました。

## 基本的な使い方

`.mcp.json` などでMCPサーバーの `url` を空文字のままにしていると、`/mcp` を実行したときにそのサーバーが「未設定」として一覧に表示されます。

```json
{
  "mcpServers": {
    "internal-tool": {
      "type": "http",
      "url": ""
    }
  }
}
```

```text
/mcp
```

## 実践例

### プレースホルダーとしてURL未定のサーバー定義を残しておく場合

プロジェクトのテンプレートや共通の `.mcp.json` に、まだ接続先URLが決まっていない社内MCPサーバーの定義だけ先に用意しておきたいことがあります。このとき `url` を空文字のままにしておいても、`/mcp` では設定エラーではなく「未設定」と表示されるため、他の本当に壊れている設定と混同せずに済みます。

```json
{
  "mcpServers": {
    "future-analytics-server": {
      "type": "http",
      "url": ""
    }
  }
}
```

### チーム共有の設定ファイルで環境ごとにURLを後から注入する運用

CI/CDやオンボーディング手順の中で、`.mcp.json` に空の `url` をデフォルト値として置いておき、各メンバーや各環境が `claude mcp add` などで実際の値を設定する運用があります。この場合も「未設定」というステータスにより、まだ設定が完了していないだけであることが `/mcp` 上で一目で分かります。

```bash
claude mcp add --transport http internal-tool https://internal.example.com/mcp
```

## 注意点

- 「未設定」と表示されるサーバーは接続されないため、そのサーバーが提供するツールは利用できません。
- 実際に使うには `claude mcp add` で正しいURLを設定するか、`.mcp.json` を直接編集して `url` を埋める必要があります。
- `url` はあるが `type` フィールドがない場合は別のエラーとして扱われ、`MCP server "<name>" has a "url" but no "type"; add "type": "http" (or "sse" / "ws") to this entry` のようなメッセージが表示されます。空URLのケースとは区別されるので注意してください。
