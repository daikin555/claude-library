## 原文（日本語に翻訳）

`set-clipboard on` の tmux 内で `claude agents` のコピーオンセレクトがシステムクリップボードに届かない問題（2.1.153 のリグレッション）を修正しました

## 原文（英語）

Fixed copy-on-select in `claude agents` not reaching the system clipboard inside tmux with `set-clipboard on` (regression in 2.1.153)

## 概要

Claude Code v2.1.157 で修正されたバグです。v2.1.153 で導入されたリグレッション（後退バグ）により、tmux 環境で `set-clipboard on` を設定している場合に、`claude agents` のセッションリスト上でテキストを選択してもシステムクリップボードにコピーされなくなっていました。修正後は、tmux の `set-clipboard on` 設定でもコピーオンセレクトが正常に機能します。

## 影響を受けていた状況

- tmux セッション内で `claude agents` を使用している場合
- tmux の設定ファイル（`~/.tmux.conf`）に `set-clipboard on` が設定されている場合
- `claude agents` のリスト上でテキストを選択してもクリップボードに何もコピーされない場合
- v2.1.153 以降にアップデートしてからこの問題が発生するようになった場合

## 注意点

- v2.1.157 以降では自動的に修正されており、特別な操作は不要です
- これは v2.1.153 で導入されたリグレッションであり、v2.1.157 で修正されました
- tmux の `set-clipboard on` はターミナルエミュレーターのクリップボード統合を有効にする設定です
- tmux のクリップボード設定の参考:
  ```bash
  # ~/.tmux.conf での設定例
  set-option -g set-clipboard on
  ```
- macOS 環境では `reattach-to-user-namespace` ラッパーが引き続き必要な場合があります
- Wayland 環境では `wl-clipboard` パッケージが必要な場合があります
