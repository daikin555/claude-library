## 原文（日本語に翻訳）

フィードバックサーベイがターン完了直後に1桁の返答をセッション評価として取得してしまう問題を修正

## 原文（英語）

Fixed the feedback survey capturing a single-digit reply as a session rating immediately after a turn completes

## 概要

フィードバックアンケートが表示されている状態でターンが完了した直後に1〜9の数字キーを入力すると、その数字がセッション評価として誤って送信される問題がありました。ユーザーが数字を別の目的（コマンド番号の選択など）で入力した場合でも、フィードバックとして扱われていました。この修正により、意図せずフィードバックが送信される問題が解消されました。

## 基本的な使い方

この修正は自動的に適用されます。フィードバックサーベイが表示された際は、意図的に評価を選択する操作のみが認識されるようになりました。

```
# フィードバックサーベイの表示例
このセッションを評価してください (1-5):
1 - 非常に満足
2 - 満足
3 - 普通
4 - 不満
5 - 非常に不満

# 修正後：明示的な選択操作のみが評価として認識される
```

## 実践例

### フィードバックの意図的な送信

```
# ターン完了後にフィードバックサーベイが表示された場合
# 数字キーを押しても自動送信されない
# 明示的にEnterキーで確定する必要がある
```

### ターン完了後の通常操作

```bash
# ターン完了後に次のコマンドを入力する際
# 数字から始まるコマンドや返答が誤って評価として送信されなくなった
claude "ファイル1の内容を確認してください"
# → ターン完了
# → フィードバックサーベイ表示
# → 次のプロンプトで "1番目の提案を実装してください" と入力しても誤送信されない
```

## 注意点

- フィードバックはClaude Codeの品質向上に活用されます
- 意図的にフィードバックを送る際は、表示されたサーベイのプロンプトに従って操作してください
- フィードバックサーベイをスキップしたい場合はEscキーで閉じることができます
