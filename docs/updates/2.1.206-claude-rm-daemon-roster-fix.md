## 原文（日本語訳）

`claude rm` で削除したジョブがdaemonの名簿（roster）に残ってしまい、`claude agents` にその行が再表示されてしまう不具合を修正した。

## 原文（英語）

Fixed `claude rm` leaving the removed job in the daemon roster, causing the row to reappear in `claude agents`

## 概要

`claude rm` はバックグラウンドエージェントのジョブを削除するためのコマンドですが、これまでは削除処理がdaemon側で管理している名簿（roster）情報に正しく反映されないケースがありました。その結果、削除したはずのジョブが `claude agents` の一覧に再び表示されてしまう不具合が発生していました。今回の修正により、`claude rm` を実行した際にdaemonの名簿からも該当ジョブの情報が確実に取り除かれるようになり、削除後に一覧へ再出現することがなくなりました。

## 基本的な使い方

バックグラウンドエージェントのジョブを削除する基本的な流れは以下の通りです。

```bash
# 実行中・完了済みのバックグラウンドエージェントを一覧表示
claude agents

# 不要になったジョブをIDや名前を指定して削除
claude rm <job-id>
```

修正後は、削除したジョブが `claude agents` の一覧から確実に消え、再表示されることはありません。

## 実践例

### 完了したバックグラウンドジョブの整理

複数のバックグラウンドエージェントを走らせた後、不要になったジョブをまとめて片付けたい場合に使います。

```bash
claude agents
# → 一覧からIDを確認
claude rm job-abc123
claude agents
# → 修正前は削除したはずのjob-abc123が再表示されることがあったが、修正後は表示されない
```

### CI/自動化スクリプトでのクリーンアップ

自動化パイプラインの中で、処理が終わったジョブを都度削除して名簿をきれいに保ちたい場合にも有効です。

```bash
for job in $(claude agents --json | jq -r '.[] | select(.status=="done") | .id'); do
  claude rm "$job"
done
```

以前はdaemonの名簿にゴミが残り、後続の `claude agents` 実行で古いジョブが亡霊のように再表示されるおそれがありましたが、この修正でスクリプトの前提（削除したら二度と出てこない）が保証されるようになりました。

## 注意点

- この不具合はdaemon側の名簿管理に起因するものであり、`claude rm` 自体がジョブ実体を削除できていなかったわけではありません。表示上の再出現が主な症状でした。
- 既存の環境で名簿が既に壊れている場合は、Claude Codeをアップデートしたうえで `claude agents` を再確認してください。改善しない場合は `/doctor` や `/status` で環境診断を行うことも検討してください。
- バックグラウンドエージェントやdaemonまわりは他にも同時期に複数の関連修正（ビルド新旧判定や名簿破損によるorphanクリーンアップ停止など）が入っているため、症状が似ていても原因が異なる場合があります。
