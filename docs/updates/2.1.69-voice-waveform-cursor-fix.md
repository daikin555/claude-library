## 原文（日本語に翻訳）

入力途中でディクテーションをすると音声波形カーソルが最初のサフィックス文字を隠す問題を修正

## 原文（英語）

Fixed Voice waveform cursor covering the first suffix letter when dictating mid-input

## 概要

テキストの途中にカーソルを置いてディクテーション（音声入力）を開始すると、音声波形のアニメーションカーソルがカーソル後の最初の文字を隠してしまう問題がありました。この修正により、ディクテーション中もカーソル後の文字が正しく表示されます。

## 基本的な使い方

```
# テキストの途中でディクテーションを開始する場合
"This is a test [カーソル位置] message"

# 修正前: 音声波形が「m」を隠してしまう
"This is a test [波形]essage"

# 修正後: 文字が隠されずに表示される
"This is a test [波形] message"
```

## 実践例

### 文章の途中への挿入ディクテーション

```
# 既存の文章に追記する場合
"フロントエンド [カーソル] を実装してください"
            ↑ ここで音声入力を開始

# 修正後: カーソル後の文字「を」が正しく表示される
"フロントエンド [音声波形] を実装してください"
```

### 長い入力の編集

```
# 長いプロンプトを編集する場合
"このコードをリファクタリング [カーソル] して、
 テストも追加してください"

# 修正後: 波形アニメーション中も全文字が見える
```

## 注意点

- この修正は音声入力（ボイス機能）を使用する場合に適用されます
- 音声波形の表示位置の調整によりカーソル後の文字が正しく表示されます
- この問題は特に日本語など全角文字を含むテキストで目立っていた場合があります
