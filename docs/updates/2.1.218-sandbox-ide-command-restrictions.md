## 原文（日本語に翻訳）

IDEとのやり取りに関するサンドボックスのコマンド制限を改善しました。

## 原文（英語）

Improved sandbox command restrictions for IDE interactions

## 概要

Claude Codeのサンドボックス機能は、Bashツールなどが実行するコマンドに対してファイルシステムアクセスやネットワーク送信を制限し、意図しない操作やプロンプトインジェクションによる被害を抑えるための仕組みです。VS CodeなどのIDE拡張機能と連携している場合、Claude CodeはIDEと通信するための特定のコマンドやインターフェースを利用しますが、これらのIDE連携コマンドに対するサンドボックスの制限が甘いと、サンドボックス内から意図しない形でIDE側の機能や情報にアクセスできてしまう可能性がありました。v2.1.218では、こうしたIDE連携時のコマンドに対するサンドボックスの制限がより厳格かつ適切に見直され、サンドボックスの分離境界を保ちながら安全にIDE連携機能を利用できるようになりました。

## 基本的な使い方

サンドボックス機能はデフォルトで有効な環境では自動的に適用されるため、ユーザー側で特別な操作は不要です。現在のサンドボックス設定は `/sandbox` コマンドで確認できます。

```bash
claude
> /sandbox
```

IDEと連携した状態（VS Code拡張機能などから起動した場合）でも、サンドボックス化されたBashコマンドの実行は引き続き通常通り行われます。

## 実践例

### VS Code拡張機能経由での開発

```bash
# VS Code内蔵ターミナルまたは拡張機能からClaude Codeを起動
claude

> 現在開いているファイルのエラーを修正して

# IDEとの連携（開いているファイルの取得、診断情報の共有など）に
# 使われるコマンド・インターフェースに対しても、
# サンドボックスの分離境界が適切に適用される
```

### サンドボックス設定の確認

```bash
claude
> /sandbox

# サンドボックスの現在の状態（有効/無効、
# ファイルシステム/ネットワークの制限内容）を確認し、
# IDE連携時も想定通りの制限が効いているか確かめられる
```

## 注意点

- この変更はClaude Code v2.1.218で実装された内部的な改善であり、通常の利用フローに大きな変更はありません
- changelog上の記載は簡潔であり、具体的にどのIDE連携コマンドがどう変わったかの詳細は公開されていません。IDE連携時のサンドボックス動作に違いを感じた場合は、公式ドキュメントや最新のリリースノートを確認してください
- サンドボックス機能自体を無効化する `dangerouslyDisableSandbox` オプションなどを使っている場合は、この改善の対象外となる可能性があります
- IDE連携で問題が発生した場合は、`/sandbox` コマンドで現在の設定を確認し、必要に応じて `.claude/settings.json` のサンドボックス関連設定を見直してください
