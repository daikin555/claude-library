## 原文（日本語訳）

ログインの有効期限が切れた際に、すべてのモデルで「選択したモデルに問題があります」という誤解を招くエラーが表示され、`/login` の実行を促すメッセージが出ない不具合を修正した。

## 原文（英語）

Fixed an expired login failing every model with a misleading "There's an issue with the selected model" error instead of prompting to run `/login`

## 概要

Claude Codeでは認証トークンの有効期限が切れると本来は再ログインが必要ですが、これまでは根本原因（ログイン期限切れ）とは無関係な「There's an issue with the selected model」（選択したモデルに問題があります）という表示になってしまい、あたかもモデル選択やアクセス権限の問題であるかのように見えてしまう不具合がありました。今回の修正により、ログイン期限切れが原因の場合は正しく検知され、`/login` の実行を促す案内が表示されるようになりました。

## 基本的な使い方

エラーメッセージが表示されたら、案内に従って再ログインするだけです。

```text
/login
```

すでに `/status` で現在有効な認証情報を確認することもできます。

```text
/status
```

## 実践例

### 長時間セッションを開きっぱなしにしていて認証が切れた場合

PCをスリープさせたまま数日Claude Codeのセッションを開きっぱなしにしていたようなケースでは、裏側でOAuthトークンの有効期限が切れることがあります。修正前はどのモデルを選んでも「選択したモデルに問題があります。存在しないか、アクセス権がない可能性があります。`/model` で別のモデルを選んでください」という表示になり、モデルを変えても解決しないため混乱の原因になっていました。修正後は、この状況で `/login` の実行を促すメッセージが表示されるため、原因がすぐに分かります。

```text
/login
```

### プラン変更後にセッションを再ログインしていない場合

Web上でプランをアップグレード・ダウングレードした直後は、既存セッションのトークンが古いプラン情報を保持したままになっていることがあります。この場合も同様のモデルエラーが出ることがあるため、`/logout` してから `/login` し直すことで解消できます。

```text
/logout
```

```text
/login
```

## 注意点

- 本修正はエラーメッセージの表示・案内内容に関する不具合修正であり、認証の仕組み自体を変更するものではありません。
- 「There's an issue with the selected model」自体はログイン期限切れ以外の原因（モデルIDの設定ミス、プランに含まれないモデルの選択など）でも表示されるエラーです。原因の切り分けについては公式のエラーリファレンスを参照してください。
- v2.1.200より前に保存されたモデル設定が残っている場合、この修正だけでは直らないことがあります。設定箇所を優先順位に沿って確認する必要があります。
