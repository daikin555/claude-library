## 原文（日本語に翻訳）

サードパーティデプロイメント（Bedrock、Vertex、Foundry、または `ANTHROPIC_BASE_URL` ゲートウェイ）では、ファーストパーティの Anthropic サービスを指すスピナーのTipsが表示されなくなりました。

## 原文（英語）

Third-party deployments (Bedrock, Vertex, Foundry, or `ANTHROPIC_BASE_URL` gateway) no longer see spinner tips pointing at first-party Anthropic surfaces

## 概要

AWS Bedrock、Google Vertex AI、Azure Foundry、またはカスタムゲートウキー（`ANTHROPIC_BASE_URL`）を利用しているユーザーに対して、Anthropic の直接サービス（claude.ai など）へのリンクや案内を含むスピナーTipsが表示されなくなりました。これにより、サードパーティ経由でClaude Codeを利用するユーザーに対して、適切でない誘導が行われる問題が解消されます。

## 基本的な使い方

この変更は自動的に適用され、特別な設定は不要です。AWS Bedrock、Google Vertex AI、Azure Foundry、またはカスタム `ANTHROPIC_BASE_URL` を設定して利用している場合、スピナー表示中のTipsがファーストパーティのAnthropicサービスを案内しなくなります。

```bash
# AWS Bedrock 経由での利用
export AWS_DEFAULT_REGION=us-east-1
claude --model anthropic.claude-opus-4-5-20251101-v1:0

# Google Vertex AI 経由での利用
export ANTHROPIC_VERTEX_PROJECT_ID=your-project
claude --model claude-opus-4-5@20251101

# カスタムゲートウェイ経由での利用
export ANTHROPIC_BASE_URL=https://your-gateway.example.com
claude
```

## 実践例

### AWS Bedrock 環境での影響

```bash
# Bedrock 経由でClaude Codeを起動
export AWS_DEFAULT_REGION=us-east-1
export ANTHROPIC_AWS_REGION=us-east-1
claude

# 処理中のスピナーTipsは企業・Bedrockの文脈に適したもののみ表示される
# "claude.ai でサインアップ" などのTipsは表示されない
```

### 企業内ゲートウェイ環境での影響

```bash
# 企業内ゲートウェイ経由
export ANTHROPIC_BASE_URL=https://ai-gateway.company.internal
claude

# 社内ユーザーに対して外部サービスへの誘導が行われない
```

### Google Vertex AI 環境での影響

```bash
# Vertex AI 経由
export ANTHROPIC_VERTEX_PROJECT_ID=my-gcp-project
export CLOUD_ML_REGION=us-central1
claude

# Vertex AI ユーザーに適切なUXが提供される
```

## 注意点

- この変更はUXの改善であり、機能的な影響はありません。
- Anthropic の直接APIを利用している場合（`api.anthropic.com`）は引き続きファーストパーティのTipsが表示されます。
- サードパーティ経由でのTipsがどのような内容になるかは、将来的にプロバイダーごとにカスタマイズされる可能性があります。
