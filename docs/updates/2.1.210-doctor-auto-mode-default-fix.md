## 原文（日本語に翻訳）

`/doctor`がBedrock、Vertex、Foundry環境において、オートモードのデフォルト化提案をスキップしてしまう問題を修正しました。これらの環境ではオートモードはもはやオプトインが不要になっています

## 原文（英語）

Fixed `/doctor` skipping its auto-mode-default proposal on Bedrock, Vertex, and Foundry, where auto mode no longer needs an opt-in

## 概要

Claude Code v2.1.210で修正された、`/doctor`コマンドに関するバグです。`/doctor`は設定や環境の健全性をチェックし、必要に応じて改善提案（プロポーザル）を行いますが、その一つに「MCPツール検索のオートモードをデフォルトで有効にする」提案があります。Amazon Bedrock、Google Vertex AI、Microsoft Foundry経由でClaude Codeを利用している環境では、既にオートモードがオプトイン不要（デフォルトで利用可能）になっているにもかかわらず、以前は`/doctor`がこの提案自体をスキップしてしまい、ユーザーがオートモードの状態を適切に把握・設定できないことがありました。今回の修正で、これらのプラットフォームでも正しく提案が表示されるようになりました。

## 基本的な使い方

### 修正前の問題

```bash
# Amazon Bedrock経由でClaude Codeを利用している環境
export CLAUDE_CODE_USE_BEDROCK=1
claude

> /doctor
```

```text
修正前:
- Bedrock/Vertex/Foundry環境では、MCPツール検索の
  オートモードに関する提案がスキップされていた
- 本来はこれらの環境ではオプトイン不要でオートモードが
  使えるにもかかわらず、/doctor上ではその状態確認や
  提案が行われなかった
```

### 修正後の動作

```bash
export CLAUDE_CODE_USE_BEDROCK=1
claude

> /doctor
```

```text
修正後:
- Bedrock/Vertex/Foundry環境でも/doctorが正しく
  オートモードの状態を診断
- オプトイン不要である旨を踏まえた適切な提案・確認が行われる
```

## 実践例

### Bedrock環境での設定確認

```bash
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION=us-east-1
claude

> /doctor

# 修正後: MCPツール検索のオートモードに関する項目が
# 正しく診断結果に含まれる
```

### Vertex AI環境での設定確認

```bash
export CLAUDE_CODE_USE_VERTEX=1
export ANTHROPIC_VERTEX_PROJECT_ID=my-project
claude

> /doctor

# 修正後: Vertex環境でもオートモードのデフォルト提案が
# 正しく表示され、スキップされない
```

### Microsoft Foundry環境でのCI診断

無人環境やCIパイプラインで `claude doctor` を使い、設定状態をチェックする例です。

```bash
export CLAUDE_CODE_USE_FOUNDRY=1
claude doctor

# read-onlyの診断結果として、
# オートモードがオプトイン不要である状況が
# 正しく反映されるようになった
```

## 注意点

- この修正は Claude Code v2.1.210 で実装されました
- 対象となるのは Amazon Bedrock、Google Vertex AI、Microsoft Foundry 経由でClaude Codeを利用している環境
- これらのプラットフォームでは、MCPツール検索のオートモードは既にデフォルトで有効（オプトイン不要）になっています
- `/doctor` はインタラクティブセッション内で確認・修正提案を行いますが、`claude doctor` （CLIから直接実行）は読み取り専用の診断結果のみを表示します
- 設定周りに不安がある場合は `/context`、`/hooks`、`/mcp` など関連コマンドと合わせて確認すると、実際に読み込まれている設定の全体像を把握しやすくなります
