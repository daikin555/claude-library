## 原文（日本語訳）

ユーザーの回答が「待ってほしい」「先に説明してほしい」という内容であっても、AskUserQuestionがClaudeに対して「続行してよい」と伝えてしまう問題を修正しました。自由記述の回答には、現在は中立的な文言が使われるようになりました。

## 原文（英語）

Fixed AskUserQuestion telling Claude to continue even when your answer asked it to wait or explain first — free-text answers now get neutral wording

## 概要

AskUserQuestionツールでは、選択肢に加えて自由記述（free-text）で回答できます。これまでは、ユーザーが自由記述欄に「ちょっと待って」「先にこの部分を説明して」といった、続行を保留する趣旨の回答を書いても、Claude側には画一的に「続行してよい」という文言が渡されてしまい、ユーザーの意図に反してタスクが進んでしまうことがありました。今回の修正により、自由記述の回答は中立的な文言でClaudeに伝えられるようになり、ユーザーが待機や説明を求めている場合にそれを無視して先に進んでしまう挙動が解消されます。

## 基本的な使い方

この修正はAskUserQuestionツールの内部的な回答の伝え方に関するものであり、ユーザー側の操作方法は変わりません。質問に対して自由記述で回答する際、その内容がそのまま尊重されやすくなります。

```
Claude: どちらの実装方針を採用しますか？
  1) シンプルな同期処理
  2) 非同期キューを使う構成
  > その他（自由記述）: ちょっと待って、まず既存コードの構成を先に見せて
```

修正前は、この回答があっても内部的に「続行してよい」という文言が付与されることがありましたが、修正後はユーザーの発言内容に沿った中立的な文言が使われ、Claudeが先走って実装を進めることが減ります。

## 実践例

### 説明を先に求めるケース

自由記述で「先に理由を説明してから進めて」と回答した場合、これまでは説明を省略して次のアクションに進んでしまうことがありましたが、修正後は回答の趣旨がそのままClaudeに伝わりやすくなります。

```
> その他（自由記述）: 先になぜその方針がいいのか説明してから実装を進めて
```

### 一時停止を求めるケース

作業の一時停止や確認を求める回答も、以前は「続行OK」という誤ったニュアンスに変換されがちでしたが、修正後は保留の意図が保たれます。

```
> その他（自由記述）: 少し待って。この変更がテストに影響しないか先に確認したい
```

### 通常の自由記述による指示

続行を前提とした通常の自由記述回答（新しい要件の追記など）は、これまで通り自然にClaudeへ伝わります。

```
> その他（自由記述）: それに加えて、ログ出力もJSON形式にしてほしい
```

## 注意点

- この修正はAskUserQuestionの自由記述回答の解釈に関するものであり、選択肢（multiple choice）を選ぶ通常の回答フローには影響しません
- 回答の意図をより明確にしたい場合は、「待って」「まず説明して」のように明示的な指示を含めると効果的です
- Agent SDKでカスタムのUIを実装している場合も、この修正はAskUserQuestionの標準的な回答処理パスに対して適用されます
