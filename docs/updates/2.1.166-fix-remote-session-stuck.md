## 原文（日本語に翻訳）

ワーカー登録中の起動時に短時間のバックエンド障害が発生した場合にリモートセッションが永続的にスタックするバグを修正しました。

## 原文（英語）

Fixed remote sessions becoming permanently stuck when a brief backend disruption occurred during worker registration at startup

## 概要

リモートセッションの起動中（ワーカー登録フェーズ）に一時的なバックエンド障害が発生すると、セッションが永続的にスタックして回復不能になるバグを修正しました。この修正により、起動時の一時的なネットワーク障害やバックエンド問題からセッションが適切に回復できるようになります。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
```

## 実践例

### 修正前の問題

以前は以下の状況でセッションがスタックしていました：
- リモートセッション起動直後に一時的なネットワーク障害が発生
- バックエンドサーバーへの一時的なアクセス問題
- ワーカー登録フェーズでのタイムアウト

### 修正後の動作

v2.1.166 以降では：
- 起動時の一時的な障害から自動的に回復
- スタックしたセッションが適切にエラー表示または再接続
- セッションを手動で終了・再起動する必要がなくなる

## 注意点

- 依然として長時間のバックエンド障害では接続が失敗する場合があります
- リモートセッションが応答しない場合は、`claude agents` で状態を確認してください
