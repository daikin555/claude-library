## 原文（日本語に翻訳）

アイドル時のCPU使用率を削減しました: `/goal` のステータスチップがアイドル時に5Hzでターミナルを再レンダリングしなくなり、サブエージェントが並列実行中のUIの再レンダリング回数も削減されました

## 原文（英語）

Reduced idle CPU usage: `/goal` status chip no longer re-renders the terminal at 5 Hz while idle, and fewer UI re-renders while subagents run in parallel

## 概要

Claude Codeのアイドル時（処理待機中）のCPU使用率が大幅に削減されました。`/goal` コマンドのステータス表示が毎秒5回ターミナルを再描画していた無駄な処理を排除し、並列サブエージェント実行中のUI再描画も最適化されました。特にノートPCのバッテリー消費の改善に貢献します。

## 基本的な使い方

この改善は自動的に適用されます。Claude Codeの使用中、特にアイドル状態やサブエージェント実行中のCPU負荷が軽減されます。

```
/goal コードの品質を維持しながら機能を追加する

# 修正後: ゴール設定後のアイドル時に不要なCPU消費が発生しない
# 修正前: 5Hzでターミナルが再レンダリングされてCPUを無駄に消費していた
```

## 実践例

### バッテリー駆動での長時間作業

```bash
# ノートPCでの長時間作業時
claude
/goal このプロジェクトの全テストをパスさせる

# 並列サブエージェントを起動
# 修正後: サブエージェント実行中のUI再描画が最適化され、
#         バッテリー消費が抑制される
```

### サーバーでの高負荷環境

```bash
# CI/CDサーバーなど複数Claude Codeインスタンスが動作する環境
# 修正後: 各インスタンスのアイドル時CPU消費が削減され、
#         サーバーリソースの効率的な利用が可能
```

## 注意点

- この最適化はパフォーマンス改善であり、機能への影響はありません
- `/goal` コマンドの動作・表示自体は変わりません
- 特にサブエージェントを多用するワークフローで効果が顕著です
