## 原文（日本語訳）

ファストモードに対応したモデルに戻した後もファストモードがオフのままになってしまう不具合を修正。設定でファストモードが有効になっている場合、モデルを戻すと自動的に復元されるようになった。

## 原文（英語）

Fixed fast mode staying off after switching back to a model that supports it — it now restores automatically when enabled in settings

## 概要

[ファストモード](https://code.claude.com/docs/en/fast-mode)はClaude Opusを高速な設定で動かす機能で、Sonnetなど対応していないモデルに切り替えると自動的にオフになります。これまで、ファストモードが自動的にオフになった後にOpusなど対応モデルへ戻しても、ファストモードは再度オンにならず、`/fast` を手動で実行し直す必要がありました。今回の修正により、保存されているファストモードの設定（ユーザーの既定設定）がオンであれば、対応モデルに戻した時点でファストモードが自動的に復元されるようになりました。

## 基本的な使い方

ファストモードが有効な状態でSonnetなど非対応モデルに切り替え、その後Opusへ戻すだけで、ファストモードが自動的に再度オンになります。

```text
/fast          # ファストモードON（Opusに切り替わる）
/model sonnet  # 非対応モデルに切り替え → ファストモードは自動的にOFF
/model opus    # 対応モデルに戻す → 2.1.208以降はファストモードが自動的にONへ復元される
```

## 実践例

### コスト重視のモデルと高速なOpusを行き来する作業

普段はコストを抑えるためにSonnetで作業し、複雑な処理や素早い反復が必要な場面だけOpus＋ファストモードに切り替えるような使い方をしている場合、以前はOpusに戻すたびに `/fast` を打ち直す手間がありました。修正後は、一度ファストモードをオンにしておけば、モデルを行き来してもOpusに戻った瞬間に自動でファストモードが復元されます。

```text
/model opus
/fast          # ファストモードON
# ... 高速なやり取りで実装
/model sonnet  # コスト重視の作業に切り替え（ファストモードは自動OFF）
# ... レビューや軽微な修正
/model opus    # 再びOpusへ → ファストモードが自動的にONに戻る
```

### セッションごとにファストモードを明示的にオプトインさせたい場合

組織で `fastModePerSessionOptIn` を `true` に設定している場合、この自動復元は発生しません。モデルを行き来した場合でも、そのセッションで `/fast` を明示的に実行しない限りファストモードはオンになりません。コスト管理を優先する組織ではこの設定と組み合わせて運用することもできます。

```json
{
  "fastModePerSessionOptIn": true
}
```

## 注意点

- 自動復元されるのは、保存されているファストモードの既定設定（新しいセッションの開始時に使われる設定）がオンの場合のみです。`fastModePerSessionOptIn` が有効な環境では、セッション開始時点の設定に関わらずモデル切り替えによる自動復元は行われません。
- ファストモードが対応していないモデル（Sonnet、Haikuなど）に切り替えている間は、引き続きファストモードはオフのままです。
- ファストモードが再度有効化されると、「Fast mode ON」という確認メッセージとプロンプト脇の `↯` アイコンが表示されます。
- 本修正はClaude Code v2.1.208で導入されました。v2.1.208より前のバージョンでは、対応モデルに戻した後も `/fast` を手動で実行するまでファストモードはオフのままでした。
