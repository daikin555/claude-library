# 修正：claude agents で Ctrl+V による画像ペーストが機能しない問題

## 原文（日本語に翻訳）

ディスパッチ入力欄とセッション返信ボックスで Ctrl+V による画像ペーストが何もしない `claude agents` の問題を修正しました。画像なしでペーストすると、ヒントが表示されるようになりました。

## 原文（英語）

Fixed `claude agents` Ctrl+V image paste doing nothing in the dispatch input and the session reply box; pasting with no image now shows a hint

## 概要

`claude agents` の入力欄でクリップボードの画像を Ctrl+V でペーストしようとしても、何も起きない問題がありました。v2.1.162 でこの問題が修正され、クリップボードに画像がある場合は正しくペーストされるようになりました。また、クリップボードに画像がない状態でペースト操作を行うと、画像のコピー方法を示すヒントが表示されます。

## 基本的な使い方

```bash
# claude agents でセッションを開く
claude agents

# セッションに画像を添付する方法:
# 1. スクリーンショットを撮る (macOS: Cmd+Shift+4 など)
# 2. claude agents の入力欄で Ctrl+V（または Cmd+V）
# 3. 画像がペーストされてメッセージに添付される
```

## 実践例

### スクリーンショットをエージェントに送る

```bash
# 1. エラー画面のスクリーンショットを撮る
# macOS: Cmd+Shift+4 → クリップボードにコピー (Ctrl+コントロールキー付きで直接クリップボードへ)
# Linux: gnome-screenshot --clipboard や scrot -s

# 2. claude agents でセッションを開く
claude agents

# 3. 入力欄で Ctrl+V（修正後は動作する）
# → 画像が添付される
```

### 画像なしでペーストした場合のヒント表示

```
# クリップボードに画像がない状態で Ctrl+V を押すと:
# ヒント: 画像を添付するには、まず画像をクリップボードにコピーしてからペーストしてください。
# スクリーンショット: macOS は Cmd+Ctrl+Shift+4、Linux は gnome-screenshot --clipboard
```

### バグレポートに画像を添付する

```bash
# 1. バグ再現の画面をキャプチャ
# 2. claude agents でタスクを開く
claude agents attach <session-id>

# 3. 返信入力欄に Ctrl+V で画像ペースト
# 4. テキストで説明を追加
# 5. Enter で送信
```

## 注意点

- この修正はディスパッチ入力欄（新規セッション開始時）とセッション返信ボックスの両方に適用されます
- ターミナルの種類によっては、画像のクリップボードサポートが異なる場合があります
- iTerm2 や Kitty などの画像対応ターミナルで最も効果的に機能します
- 画像のサイズが大きすぎる場合はエラーが発生することがあります（その場合はリサイズして再試行）
