## 原文（日本語に翻訳）

セッションを再起動せずにスキルディレクトリを再スキャンする `/reload-skills` コマンドを追加しました。

## 原文（英語）

Added `/reload-skills` command to re-scan skill directories without restarting the session

## 概要

`/reload-skills` コマンドを使うことで、Claude Code のセッションを再起動することなく、スキルディレクトリ（`~/.claude/skills` および `.claude/skills`）を手動で再スキャンできます。スキルの追加・変更・削除を行った後、その変更をすぐに現在のセッションに反映させたい場合に便利です。

## 基本的な使い方

```bash
# スキルの変更後、現在のセッションで再スキャンを実行
/reload-skills
```

## 実践例

### 新しいスキルを追加した後の即時利用

スキルファイルを新規作成した後、`/reload-skills` を実行することで、そのスキルを即座に利用できます。

```bash
# 1. 新しいスキルを作成
cat > ~/.claude/skills/deploy/SKILL.md <<'EOF'
---
name: deploy
description: アプリケーションのデプロイ手順
---
# デプロイスキル
デプロイ処理を実行します。
EOF

# 2. 現在のセッションでスキルを再スキャン
/reload-skills

# 3. 即座に新しいスキルが利用可能になる
/deploy
```

### スキルの内容を修正した後の反映

既存のスキルを更新した後、再スキャンして最新の内容を反映させます。

```bash
# 1. 既存スキルを編集
vim ~/.claude/skills/my-skill/SKILL.md

# 2. セッションを維持したまま再スキャン
/reload-skills

# 3. 更新されたスキルで作業を継続
/my-skill
```

### git pull 後の最新スキルの取得

チームでスキルを共有している場合、`git pull` 後に `/reload-skills` で最新スキルを取得できます。

```bash
# 1. 最新のスキルを取得
git pull origin main

# 2. 現在のセッションに最新スキルを反映
/reload-skills

# 3. チームが追加したスキルが利用可能に
```

## 注意点

- `/reload-skills` はセッションを維持したまま実行されるため、会話履歴や進行中の作業は失われません
- 自動ホットリロード（v2.1.0で導入）が有効な環境では、ファイル変更時に自動的にリロードされますが、このコマンドを使って手動でも実行できます
- スキルファイルに構文エラーがある場合は、リロード後にエラーが通知されます
- `SessionStart` フックの `reloadSkills: true` 設定とは異なり、このコマンドはユーザーが任意のタイミングで実行するものです
