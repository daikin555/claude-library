## 原文（日本語訳）

`SendMessage` で再開したバックグラウンドエージェントが、エージェント一覧で「失敗（failed）」または「完了（completed）」のまま表示され続ける不具合を修正した。

## 原文（英語）

Fixed background agents staying shown as "failed" or "completed" in the agent list after being resumed with `SendMessage`

## 概要

Claude Codeのバックグラウンドエージェントは、一度完了（または失敗）した後でも `SendMessage` を使ってagentIdを指定し、同じエージェントを文脈を保持したまま再開できます。しかし今回の修正前は、この方法でエージェントを再開しても、`claude agents` などで開くエージェント一覧（agent view）上の表示ステータスが「完了」や「失敗」のまま更新されず、実際には裏で処理が再開・実行中であるにもかかわらず一覧上は止まっているように見えてしまう不具合がありました。今回のアップデートで、再開後のエージェントのステータス表示が実際の状態（作業中など）に正しく追従するようになりました。

## 基本的な使い方

すでに完了・失敗したバックグラウンドエージェントに対して、`SendMessage` ツールでagentId（またはエージェント名）を指定し、追加の指示を送ることでエージェントを再開できます。

```text
SendMessage({
  to: "agent-123", // 完了/失敗済みのエージェントID
  message: "先ほどの調査に加えて、テストカバレッジの状況も確認してください"
})
```

この修正により、再開直後からエージェント一覧（`claude agents` コマンドや、セッション内の左矢印から開くagent view）上のステータス表示が正しく「作業中」等に切り替わり、処理が完了すると再度「完了」に更新されるようになります。

## 実践例

### 完了済みエージェントに追加調査を依頼する

一度レポートを返して「完了」表示になった調査系のバックグラウンドエージェントに対し、後から追加の観点を依頼したいケースです。

```text
# 1. 事前にバックグラウンドで起動した調査エージェントが完了済み
# 2. 追加で見てほしい点が出てきたので再開する
SendMessage({
  to: "investigate-auth-flow",
  message: "認証まわりの調査に加えて、セッションのタイムアウト処理も対象に含めて再調査してください"
})
```

修正前は、ここで再開してもエージェント一覧上は「完了」のままで、実際に動いているのか判断がつきませんでした。修正後は一覧のステータスが「作業中」に切り替わるため、他のエージェントと並行して進捗を安心して確認できます。

### 失敗したエージェントを修正指示付きで再実行する

何らかのエラーで「失敗」表示になったバックグラウンドエージェントに対し、原因を伝えて再試行させるケースです。

```text
SendMessage({
  to: "migrate-db-schema",
  message: "権限エラーで失敗したようです。マイグレーション用のDB接続文字列を修正したので、同じ処理を再実行してください"
})
```

修正前はこの再開後も一覧上は「失敗」のバッジが残り続け、実際に再実行されているのか、それとも失敗したまま放置されているのかが紛らわしい状態でした。今回の修正でこの表示の不整合が解消されています。

## 注意点

- この修正はあくまで**エージェント一覧上の表示ステータス**に関するもので、`SendMessage` によるエージェント再開機能自体の挙動（文脈を保持したまま処理を再開する点）を変更するものではありません。
- `SendMessage` によるエージェント再開・エージェントチーム関連の機能は環境によって有効化条件が異なる場合があります（例: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` などの設定が必要になるケースが報告されています）。手元の環境で `SendMessage` が利用できない場合は、Claude Codeのバージョンや関連設定を確認してください。
- 再開ハンドル（agentIdからエージェントを見つけて起こすための紐付け情報）はセッション状態に依存するため、メインの会話がコンテキスト要約を挟んだ場合や、セッションをまたいで長時間経過した場合には、再開が期待通りに機能しないことがある点にも留意してください。
