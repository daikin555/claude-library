## 原文（日本語訳）

ターミナルの縦サイズが短い場合にモーダル検索ダイアログが画面をオーバーフローし、検索ボックスとキーボードヒントが非表示になる問題を修正しました。

## 原文（英語）

Fixed modal search dialogs overflowing the screen at short terminal heights, hiding the search box and keyboard hints

## 概要

Claude Codeの検索機能（`/` キーで開く検索ダイアログなど）を使用する際、ターミナルウィンドウの縦サイズが小さい場合（例：狭いターミナルペインや小さいウィンドウ）に、検索ダイアログが画面の端を超えてはみ出し、検索ボックスやキーボードショートカットのヒントが見えなくなる問題がありました。v2.1.116ではダイアログのレイアウトが修正され、ターミナルの縦サイズに応じて適切にダイアログが収まるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。小さいターミナルウィンドウでも検索ダイアログが正しく表示されます。

```bash
# ターミナルの縦サイズが小さい環境でも
claude

# 検索ダイアログを開いたときに検索ボックスが表示される
# Ctrl+F または /search
```

## 実践例

### 分割ターミナルペインでの使用

```bash
# VS Code や tmux の分割ターミナルで縦サイズが制限されている場合
# 検索ダイアログを開いても正しく表示されるようになった
Ctrl+F  # または /search コマンド
# → 検索ボックスとキーボードヒントが適切に表示される
```

### モバイルデバイスやSmall Displayでの使用

```bash
# 縦サイズが少ない環境（例：20行以下）でも
# ダイアログが画面に収まって表示される
```

## 注意点

- この問題はターミナルの縦サイズ（行数）が少ない場合に発生していました
- 横サイズ（列数）には影響しません
- 修正後も、極端に小さいターミナル（10行以下）では表示が制限される場合があります
- `/terminal-setup` でターミナルの最適な設定を行うことをお勧めします
