## 原文（日本語に翻訳）

Claude Code の更新後に再接続した際、`claude agents` のバックグラウンドセッションが実行中のバックグラウンドタスクを失う問題を修正。

## 原文（英語）

Fixed background sessions in `claude agents` losing their running background tasks when reattached after a Claude Code update

## 概要

Claude Code がアップデートされた後にバックグラウンドエージェントセッションに再接続すると、実行中のバックグラウンドタスクが失われるバグが修正されました。アップデートをまたいでもバックグラウンドタスクが継続し、再接続後に正しく引き継がれるようになります。

## 基本的な使い方

修正後は特別な操作は不要です。Claude Code のアップデート後に `claude agents` でバックグラウンドセッションに再接続しても、実行中のタスクが維持されます。

```bash
# バックグラウンドエージェントを起動
claude agents

# Claude Code がアップデートされた後でも再接続時にタスクが引き継がれる
claude agents
```

## 実践例

### 長時間実行タスク中にアップデートが発生した場合

1. バックグラウンドエージェントで長時間タスクを実行中
2. Claude Code のアップデートが自動適用される
3. `claude agents` で再接続
4. 修正前：実行中のタスクが失われていた
5. 修正後：タスクが継続して実行されており、進捗を確認できる

## 注意点

- この問題は Claude Code のアップデートが発生したタイミングにのみ起きていました。通常の使用では影響はありません。
- バックグラウンドセッションを使用している場合、アップデートをまたいでタスクの継続性が保証されるようになりました。
