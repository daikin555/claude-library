## 原文（日本語に翻訳）

VSCodeにプラン用のフルMarkdownドキュメントビューが追加され、コメントを追加してフィードバックを提供する機能もサポートされました。

## 原文（英語）

[VSCode] Added full markdown document view for plans in VS Code, with support for adding comments to provide feedback

## 概要

VS Code 2.1.70より、Claude CodeのプランがフルのMarkdownドキュメントとして表示できるようになりました。プランの内容を見やすいMarkdownフォーマットで確認しながら、各セクションにコメントを追加してClaude Codeへのフィードバックを直接提供できます。これによりプランのレビューと修正が大幅に効率化されます。

## 基本的な使い方

1. Claude Codeでプランモードを開始する（`/plan` コマンドまたはプランモードへの切り替え）
2. Claudeがプランを生成したら、VS Code上でMarkdownドキュメントビューで表示される
3. プランの各セクションを確認し、コメントを追加してフィードバックを入力する
4. フィードバックを元にClaudeがプランを修正する

## 実践例

### プランのレビューとフィードバック

大規模なリファクタリングや機能実装のプランをレビューする場合：

```
// プランモードの開始
/plan

// Claudeが提案するプラン例（Markdownビューで表示）:
## 実装計画
1. データモデルの設計
2. APIエンドポイントの実装
3. フロントエンドコンポーネントの更新
4. テストの作成

// コメントでフィードバック:
// → 「手順2の前にエラーハンドリングの設計が必要です」
// → 「手順4はユニットテストとE2Eテストに分けてください」
```

### 複雑なプランの把握

複数ファイルにまたがる変更計画をMarkdownで確認：

```markdown
# リファクタリングプラン（Markdownビューの例）

## 変更対象ファイル
- `src/auth/login.ts` - 認証ロジックの分離
- `src/api/users.ts` - ユーザーAPIの統一
- `tests/auth.test.ts` - テストケースの追加

## 実行順序
1. 認証モジュールを独立したサービスに分離
2. 既存APIとの互換性を確認
3. テストを実行して動作を検証

<!-- コメント例: 「手順1の前にバックアップを取るステップを追加してください」 -->
```

### チームレビューへの活用

プランをMarkdownとして共有し、チームメンバーとのレビューに活用：

```
// プランをMarkdownドキュメントとして確認し
// コメントを通じてチームの意見をClaude Codeに伝える

コメント例:
- 「セキュリティレビューが必要なステップはどれですか？」
- 「この変更は本番環境への影響がありますか？」
```

## 注意点

- この機能はVS Code拡張機能版のClaude Codeで利用可能です
- フルMarkdownドキュメントビューはプランモード時に自動的に表示されます
- コメントはプランの内容に直接フィードバックとして反映されます
- プランの修正はコメントを元にClaude Codeが自動的に提案します
