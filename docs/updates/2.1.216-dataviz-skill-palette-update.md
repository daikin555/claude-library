## 原文（日本語訳）

バンドル済みのdataviz（データ可視化）スキルを更新しました。デフォルトのチャートパレットの並び順を変更し、4系列チャートに対して直接ラベルを推奨していた誤ったガイダンスを修正しました。

## 原文（英語）

Updated the bundled dataviz skill: reordered the default chart palette and fixed guidance that suggested direct labels for four-series charts

## 概要

Claude Codeにはグラフやダッシュボードなどのデータ可視化を作成する際に自動的に参照される「dataviz」スキルがバンドルされています。今回の更新では、デフォルトのカテゴリカルカラーパレットの色の並び順が調整され、より見やすい配色の優先順位になりました。また、系列数が4つあるチャートに対して「直接ラベル（凡例を使わずデータ点近くにラベルを直書きする手法）」を推奨していた誤ったガイダンスが修正され、系列数が多い場合により適切な凡例・配色の使い分けが案内されるようになりました。

## 基本的な使い方

この修正はスキルの内部ガイダンスの更新であり、ユーザー側での設定変更は不要です。チャートやグラフの作成をClaudeに依頼すると、更新後のdatavizスキルが自動的に適用されます。

```
売上データを月別・カテゴリ別の棒グラフにして
```

このような依頼をすると、Claudeは更新されたパレット順序とガイダンスに基づいてチャートを生成します。

## 実践例

### 4系列の折れ線グラフを作成する場合

以前は4系列のチャートに対して直接ラベル（各線の端にラベルを直接添える手法）を推奨するガイダンスが誤って適用されるケースがありました。修正後は、系列数に応じて凡例とパレットを適切に使い分けるようになります。

```
# Claudeへの依頼例
月別のCPU/メモリ/ディスク/ネットワークの4系列の使用率を折れ線グラフで可視化して
```

### デフォルトパレットを使った棒グラフ・積み上げグラフの作成

パレットの並び順が更新されたことで、カテゴリ数が少ないチャート（2〜3系列程度）でも、より識別しやすい色の組み合わせがデフォルトで先頭から使われるようになります。

```
# Claudeへの依頼例
地域別（東京/大阪/福岡）の四半期売上を積み上げ棒グラフで表示して
```

### 既存のダッシュボードコードを更新後のガイダンスに合わせて見直す

すでにdataviz スキルを使って生成済みのチャートコードがある場合、Claudeに再生成や見直しを依頼することで、更新後のパレット順序・ラベル付けガイダンスを反映できます。

```
このダッシュボードのグラフの配色とラベルを、最新のガイドラインに沿って見直して
```

## 注意点

- パレットの並び順が変わったため、以前生成したチャートと新しく生成するチャートで、同じ系列でも色の対応がずれる可能性があります。既存のダッシュボードとの一貫性が必要な場合は明示的に色を指定してください。
- 4系列以上のチャートでは、直接ラベルよりも凡例を使う方が可読性が高いとされる点に変わりはありません。今回の修正はガイダンスの誤りを正したものです。
- dataviz スキルは `disableBundledSkills` 設定で無効化することも可能ですが、無効化した場合はこれらの改善も適用されません。
