## 原文（日本語訳）
サジェスチョンを受け入れた際に Tab 補完が入力フィールドを更新しなかった問題を修正しました

## 原文（英語）
Fixed tab completion not updating the input field when accepting a suggestion

## 概要
Tab キーでファイルパスやコマンドの補完候補を受け入れた際に、入力フィールドが正しく更新されなかった問題が修正されました。これにより、補完機能がスムーズに動作するようになります。

## 修正された問題

以前は、Tab キーで補完候補を受け入れても、以下の問題が発生していました：

- 入力フィールドに補完結果が反映されない
- 表示と実際の入力内容が不一致
- 複数回 Tab を押す必要がある
- 補完後の編集が困難

## Tab 補完の使い方

### ファイルパス補完

```bash
claude
> @src/co[Tab]
# → @src/components/
```

### スラッシュコマンド補完

```bash
claude
> /com[Tab]
# → /compact
```

### シェルコマンド補完（bash モード）

```bash
claude
> ! git che[Tab]
# → ! git checkout
```

## 注意点

- **即座の反映**: Tab キーを押すと、入力フィールドが即座に更新されます
- **継続編集**: 補完後、そのまま入力を続けることができます
- **複数候補**: 複数の候補がある場合、Tab を複数回押すことで循環します
- **Enter との違い**: Tab は補完のみ、Enter は補完して送信します
- **プロンプト編集**: 補完後もプロンプトを編集できます
- **エスケープ**: Esc キーで補完をキャンセルできます
