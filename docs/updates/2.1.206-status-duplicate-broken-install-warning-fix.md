## 原文（日本語訳）

`/status` が同じ「インストール破損」警告を2回表示してしまう不具合を修正した。

## 原文（英語）

Fixed `/status` listing the same broken-install warning twice

## 概要

`/status` はClaude Codeの認証方法やインストール状態などを確認できるコマンドで、インストールに問題がある場合には警告メッセージを表示します。これまでは、この「インストールが壊れている」旨の警告が同一内容のまま2回重複して表示されてしまう不具合がありました。今回の修正により、`/status` の出力で同じ警告が重複して並ぶことがなくなり、診断結果がすっきりと見やすくなりました。

## 基本的な使い方

セッション内で以下を実行するだけです。

```text
/status
```

現在の認証方法（`/login` によるアカウント認証かAPIキーか）や、インストールの状態、警告・エラーの有無が一覧表示されます。

## 実践例

### インストールに問題がある環境での状態確認

npmのグローバルインストールとローカルインストールが混在しているなど、インストールが壊れている（broken install）と判定される環境で `/status` を実行するケースです。

```text
/status
```

修正前は同じ「インストールが壊れています」という警告が出力の中に2回表示され、問題が2つあるのか、単なる表示バグなのか判断しづらい状態でした。修正後は警告が1回だけ表示されるため、他に本当に別の警告がないかを正しく確認できます。

### /doctor（/checkup）と組み合わせた原因調査

`/status` で警告に気づいた後、詳しい原因と修正方法を知りたい場合は `/doctor`（エイリアス `/checkup`）を続けて実行します。

```text
/status
/doctor
```

`/status` で問題の存在を簡潔に把握し、`/doctor` で詳細な診断と自動修正の提案を受け取るという流れが、重複表示のノイズなく行えるようになりました。

## 注意点

- この修正は表示上の重複を解消するものであり、警告の原因そのもの（インストールの破損状態）を自動的に修正するものではありません。修正には `/doctor` や `claude doctor` の実行が別途必要です。
- 起動時に表示されていた「claude command missing or broken」といった警告は、現在は起動時ではなく `/doctor` や `/status` の中で確認する形に整理されています。
- 症状が再発する場合は、Claude Codeのバージョンが2.1.206以降になっているか確認してください。
