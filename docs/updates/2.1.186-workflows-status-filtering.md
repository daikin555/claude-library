# /workflowsエージェント詳細ビューにfキーでステータスフィルタリングを追加

## 原文（日本語に翻訳）
`/workflows` エージェント詳細ビューにステータスフィルタリング（`f` キーを押す）を追加しました。

## 原文（英語）
Added status filtering (press `f`) to the `/workflows` agent detail view

## 概要
`/workflows` コマンドで表示されるエージェント詳細ビューに、ステータスによるフィルタリング機能が追加されました。`f` キーを押すことで、実行中・完了・エラーなどのステータスでエージェントを絞り込んで表示できます。大規模なワークフローで多数のエージェントを管理する際の視認性が大幅に向上します。

## 基本的な使い方

```
/workflows
```
ワークフロービューを開いた後、`f` キーを押してステータスフィルターを切り替えます。

| キー操作 | 機能 |
|---------|------|
| `f` | ステータスフィルターを切り替え |

## 実践例

### 実行中エージェントのみを確認する
```
/workflows
# ビュー内で 'f' キーを押す
# → 実行中のエージェントのみが表示される
```

### 大規模ワークフローの進捗管理
100個以上のサブエージェントが並行動作するワークフローを監視する際：
```
/workflows
# 'f' キーでフィルタリング
# → 完了済みエージェントを非表示にして実行中エージェントに集中
# → エラーが発生したエージェントを素早く特定
```

### ワークフロー完了の確認
```
/workflows
# 'f' キーで「完了」ステータスに絞り込み
# → すべてのエージェントが完了しているか確認
```

## 注意点
- フィルタリングはビューの表示のみを変更します。エージェントの実行には影響しません
- `f` キーで複数のフィルター状態（すべて表示→実行中のみ→完了のみ→エラーのみ、など）を順番に切り替えられます
- フィルター状態はビューを閉じると初期化されます

## 関連情報
- ワークフロー実行: `Workflow` ツールを使用したマルチエージェントオーケストレーション
- `/workflows` コマンドでリアルタイムの進捗監視
