## 原文（日本語に翻訳）

Markdownの出力で、GFMタスクリストチェックボックス（`- [ ] todo` / `- [x] done`）がプレーンな箇条書きではなく、チェックボックスとしてレンダリングされるようになりました。

## 原文（英語）

> Markdown output now renders GFM task list checkboxes (`- [ ] todo` / `- [x] done`) instead of plain bullets

## 概要

GitHub Flavored Markdown（GFM）のタスクリスト構文がClaude Codeの出力で正しくレンダリングされるようになりました。`- [ ]` や `- [x]` の記法が単なる箇条書きではなく、視覚的なチェックボックスとして表示されるため、タスク管理やTODOリストの可読性が向上します。

## 基本的な使い方

Claudeにタスクリストの作成を依頼すると、チェックボックス形式でレンダリングされます。

```markdown
- [x] 要件定義
- [x] データベース設計
- [ ] APIの実装
- [ ] フロントエンド開発
- [ ] テスト
- [ ] デプロイ
```

上記は視覚的に以下のように表示されます：
- ☑ 要件定義
- ☑ データベース設計
- ☐ APIの実装
- ☐ フロントエンド開発
- ☐ テスト
- ☐ デプロイ

## 実践例

### プロジェクトの進捗管理

```
Claudeへ: このスプリントのタスクをチェックボックス形式でまとめてください
```

Claudeが以下のような出力を返し、チェックボックスとして表示されます。

```markdown
## スプリント1 タスク

### バックエンド
- [x] ユーザー認証API
- [x] データベースマイグレーション
- [ ] ファイルアップロード機能
- [ ] キャッシュ実装

### フロントエンド
- [ ] ログイン画面
- [ ] ダッシュボード
- [ ] レスポンシブ対応
```

### コードレビューチェックリスト

```
Claudeへ: このコードのレビューチェックリストを作成してください
```

```markdown
## コードレビューチェックリスト

- [x] 命名規則が適切
- [x] エラーハンドリングが実装されている
- [ ] ユニットテストが追加されている
- [ ] ドキュメントが更新されている
- [ ] セキュリティ上の問題がない
```

## 注意点

- GFMタスクリスト構文（`- [ ]` および `- [x]`）を使用している場合にのみ適用されます
- Claude Code内のターミナル出力でのレンダリング改善であり、ファイルへの書き出し内容には影響しません
- チェックボックスの状態（完了/未完了）はレンダリング時のみで、インタラクティブな変更はできません
