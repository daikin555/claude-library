## 原文（日本語訳）

/extra-usage を有効化した直後に /fast がすぐに利用できなかった問題を修正しました。

## 原文（英語）

Fixed an issue where /fast was not immediately available after enabling /extra-usage

## 概要

このアップデートでは、`/extra-usage` を有効化した直後に `/fast` コマンドがすぐに使えなかった不具合が修正されました。これにより、追加使用量を有効にしてから高速モードを使い始めるまでの待ち時間がなくなり、よりスムーズなワークフローが実現されます。

## 基本的な使い方

修正前は以下のような手順が必要でしたが、修正後は待ち時間なしで即座に利用可能になります。

```bash
# 1. 追加使用量を有効化
/extra-usage

# 2. 以前: ここで /fast が使えるまで待つ必要があった
# 修正後: すぐに /fast が利用可能！
/fast
```

## 実践例

### ケース1: 大量のコード生成タスクの開始時

プロジェクトの初期セットアップや大規模なリファクタリングを開始する際、追加使用量を有効化してすぐに高速モードで作業を開始できます。

```bash
# セッション開始時に両方を設定
/extra-usage
/fast

# すぐに大量のコード生成タスクを開始
"新しいAPIエンドポイントを10個作成して、それぞれにテストを追加してください"
```

### ケース2: 緊急のバグ修正

緊急性の高いバグ修正で、追加使用量を使いながら高速な応答が必要な場合、待ち時間なしで作業を開始できます。

```bash
/extra-usage
/fast

# すぐにデバッグ作業を開始
"本番環境でエラーが発生しています。ログを確認して修正してください"
```

### ケース3: 集中作業セッション

長時間の集中作業セッションで、最初から最適な設定で開始できます。

```bash
# セッション開始時に設定を完了
/extra-usage
/fast

# すぐに作業開始
"この機能の実装を完成させましょう"
```

## 注意点

- この修正は内部的な処理の改善であり、コマンドの使い方自体は変わりません
- `/extra-usage` と `/fast` の両方を使用する場合でも、それぞれのコマンドは個別に実行する必要があります
- `/extra-usage` は追加の使用量枠を有効化し、`/fast` は高速モードを有効化する別々の機能です
