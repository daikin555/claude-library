# 起動時の通知が整理：重要度別グループ化と情報の集約

## 原文（日本語に翻訳）

起動時の通知が静かになりました。通知は重要度別にグループ化され、セッション情報とお知らせが 1 回の起動につき 1 行で共有されるようになりました。

## 原文（英語）

Quieter startup: notices group by severity, and session info and announcements share a single line per launch

## 概要

Claude Code の起動時に表示される通知メッセージが大幅に整理されました。以前は複数の通知が個別に表示されていましたが、v2.1.162 からは重要度別にグループ化され、セッション情報やお知らせは起動ごとに 1 行にまとめて表示されます。起動時の情報ノイズが減り、作業に集中しやすくなります。

## 基本的な使い方

```bash
# Claude Code を起動
claude

# 修正前（複数行の通知）:
# Claude in Chrome が有効です
# マーケットプレースがインストールされました
# モデルが自動更新されました
# セッション ID: abc123
# お知らせ: 新機能が利用可能です

# 修正後（整理された表示）:
# [重要] セッション abc123 | 新機能が利用可能
```

## 実践例

### 起動時の通知を確認する

```bash
# クリーンな起動体験
claude

# 通知は重要度順に表示:
# [エラー]   重大な問題（即時対応が必要）
# [警告]     注意が必要な事項
# [情報]     一般的な通知（まとめて表示）
```

### CI/CD での起動ノイズ削減

```bash
# CI環境でのログが整理される
claude --print "タスクを実行して" 2>&1 | grep -v "^\[情報\]"
# 重要なメッセージのみを抽出しやすくなった
```

### スクリプトでの起動メッセージ処理

```bash
#!/bin/bash
# 起動メッセージが整理されたため、パースしやすくなった
OUTPUT=$(claude --print "コードを確認して")

# 重要度別のメッセージを識別しやすい
echo "$OUTPUT" | grep "\[エラー\]" && echo "エラーが発生しました"
```

## 注意点

- 重要度の高い通知（エラー・警告）は引き続き明確に表示されます
- 情報レベルの通知は集約されるため、個別の通知が見えなくなる場合があります
- セッション情報とお知らせが 1 行にまとまるため、ログの行数が減ります
- この変更は見た目の改善であり、通知の内容自体は変わっていません
