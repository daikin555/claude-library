## 原文（日本語）
VSCode: リモートセッションのサポートが追加され、OAuthユーザーがclaude.aiからセッションを参照・再開できるようになりました。

## 原文（英語）
VSCode: Added support for remote sessions, allowing OAuth users to browse and resume sessions from claude.ai

## 概要
Visual Studio Code拡張機能に、リモートセッション機能が追加されました。OAuthでログインしているユーザーは、claude.ai上で実行されているセッションをVSCode内から直接参照し、再開できるようになります。これにより、ブラウザとVSCodeの間でシームレスに作業を切り替えることができます。

## 基本的な使い方

VSCode拡張機能でOAuth認証を完了している場合、リモートセッションが自動的に利用可能になります。

```
1. VSCodeでClaude Code拡張機能を開く
2. セッションピッカーを開く（通常はコマンドパレットから）
3. claude.aiで実行中のリモートセッションが一覧に表示される
4. 選択してセッションを再開
```

## 実践例

### ブラウザで開始した作業をVSCodeで継続

claude.aiのWebインターフェースで開始したコーディングタスクを、VSCodeで継続する例：

**シナリオ:** 通勤中にスマートフォンやタブレットでclaude.aiを使って設計やプロトタイピングを行い、オフィスや自宅のVSCodeで本格的な実装を継続する。

```
1. スマートフォンでclaude.aiにアクセス
2. 「新しいAPIエンドポイントを設計」というタスクを開始
3. オフィスに到着後、VSCodeを起動
4. セッションピッカーでリモートセッションを選択
5. スマートフォンでの会話履歴と設計内容が表示される
6. VSCodeのエディタで実装を開始
```

### チームメンバーとのコラボレーション

claude.aiで共有されたセッションをVSCodeで確認・作業：

**シナリオ:** チームメンバーがclaude.aiで作成したプロトタイプやコード例を、VSCodeで確認・改善する。

```
1. チームメンバーがclaude.aiでセッションを作成・共有
2. 自分のVSCodeでセッションピッカーを開く
3. 共有されたリモートセッションを選択
4. コードをローカル環境で検証・改善
```

### 複数デバイス間での作業継続

デスクトップ、ラップトップ、タブレットなど、複数のデバイス間で作業を移動：

**シナリオ:** デスクトップで開始した作業を、ラップトップで移動中に継続し、再びデスクトップで仕上げる。

```
1. デスクトップのVSCodeでセッションを開始
2. 外出先でラップトップを開く
3. ラップトップのVSCodeでリモートセッションを選択
4. 作業を継続
5. 帰宅後、デスクトップで同じセッションを再開
```

### ブラウザでの調査とVSCodeでの実装を並行

ブラウザ（claude.ai）で技術調査を行いながら、VSCodeで実装を進める：

**シナリオ:** ブラウザのclaude.aiで新しいライブラリやフレームワークについて質問し、その回答を参照しながらVSCodeで実装。

```
1. ブラウザのclaude.aiで「Next.js 14のServer Actionsの使い方」を質問
2. VSCodeでリモートセッションを開く
3. ブラウザでの会話履歴を参照しながら、VSCodeでコードを書く
4. 必要に応じてブラウザに戻って追加質問
```

### 緊急時のトラブルシューティング

出先で発生した緊急のバグを、手元のデバイスで素早く対処：

**シナリオ:** 本番環境でバグが発生し、手元にあるラップトップで緊急対応する必要がある。

```
1. スマートフォンでclaude.aiにアクセスし、エラーログを分析
2. ラップトップのVSCodeを起動
3. リモートセッションを開いて、分析結果を確認
4. VSCodeで修正コードを作成・デプロイ
```

### デモやプレゼンテーションの準備

プレゼン用のコードサンプルをブラウザで作成し、VSCodeで整形・検証：

**シナリオ:** 会議中にブラウザで簡単なデモコードを作成し、後でVSCodeで整形・テストする。

```
1. 会議中にclaude.aiで簡単なデモコードを作成
2. 会議後、VSCodeでリモートセッションを開く
3. コードを整形し、テストを追加
4. プレゼン資料に埋め込む
```

## 注意点

- リモートセッション機能を使用するには、OAuthでログインしている必要があります
- APIキー認証では、リモートセッション機能は利用できません
- claude.aiとVSCodeの間でセッションが同期されるため、ネットワーク接続が必要です
- セッションの同期には若干の遅延が発生する場合があります
- 無料プランと有料プランで、利用可能なリモートセッション数に制限がある場合があります
- ローカルで行った変更は、セッションを保存するまでclaude.aiには反映されません
- 複数のデバイスで同じセッションを同時に編集する場合、競合が発生する可能性があります
- セッションのプライバシーとセキュリティに注意し、機密情報を含むセッションの共有には慎重になってください
