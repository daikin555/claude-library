## 原文（日本語に翻訳）

UIコンポーネントがスタイル付きテキスト要素の外にコンテンツを返した際に、描画されたテキストの断片がクラッシュテレメトリに漏れてしまう不具合を修正しました

## 原文（英語）

Fixed a rendered text fragment leaking into crash telemetry when a UI component returned content outside a styled text element

## 概要

Claude CodeのターミナルUIはコンポーネントベースで構築されており、テキストは通常「スタイル付きテキスト要素」の内側にラップされて描画されます。しかし一部のUIコンポーネントが、スタイル付きテキスト要素の外側に直接テキストコンテンツを返してしまうケースがあり、その際にクラッシュが発生すると描画中のテキスト断片がそのままクラッシュテレメトリ（Sentry等のエラーレポート）に含まれてしまう不具合がありました。Claude Code v2.1.210ではこの不具合が修正され、意図しないテキストがクラッシュレポートに混入しないようになりました。

## 基本的な使い方

### 修正前の問題

```bash
# 特定のUIコンポーネントが不正な構造でテキストを返し、
# 描画エラーが発生した場合

claude

> （UIレンダリング時に内部エラーが発生するケース）

# 修正前:
# クラッシュレポートに、画面に表示されていたテキストの
# 断片（ファイル内容の一部やコマンド出力など）が
# 意図せず含まれてしまう可能性があった
```

### 修正後の動作

```bash
claude

# 修正後:
# UIコンポーネントの描画エラーが発生しても、
# クラッシュテレメトリに含まれるのはエラーのスタック
# トレースや診断情報のみで、描画中だった
# テキストコンテンツの断片は含まれない
```

## 実践例

### エラーレポートの内容確認

```bash
# エラーレポートを無効化している場合は影響を受けない
export DISABLE_ERROR_REPORTING=1
claude

# エラーレポートを有効にしている場合でも、
# 修正後はテキスト断片の混入経路自体が塞がれている
```

### 機密情報を含む可能性のある画面での安全性向上

```bash
claude

> .env ファイルの内容を確認して問題点を指摘して

# 万が一この操作中にUI描画エラーが起きても、
# 修正後は画面に表示されていた機密情報らしきテキストが
# クラッシュテレメトリに漏れる経路が塞がれている
```

### 企業環境でのテレメトリポリシーとの併用

```bash
# 非必須トラフィックをすべて無効化する設定
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
claude

# この設定に加えて今回の修正により、
# 万が一テレメトリが有効な環境でも
# 描画テキストの漏えいリスクがさらに低減される
```

## 注意点

- この修正はClaude Code v2.1.210（2026年7月15日リリース）で行われました
- Claude Codeはユーザーの端末からAnthropic（およびエラーロギング用のSentry）に対して、レイテンシや信頼性などの運用メトリクスを送信することがありますが、通常はコードやファイルパスは含まれません
- `DISABLE_ERROR_REPORTING` 環境変数を設定することで、エラーレポート自体を無効化できます
- `DISABLE_TELEMETRY` でテレメトリ全体を、`CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` でアンケートなどを含む非必須通信すべてを無効化できます
- Amazon Bedrock、Google Cloud、Microsoft Foundry、AWS上のClaude Platformを利用している場合、エラーレポート・テレメトリ・バグレポートはデフォルトで無効になっています
- 本修正はUIレンダリング層のバグ修正であり、ユーザー側で追加の設定変更を行う必要はありません
