# 修正：リモートコントロールセッションが最初のプロンプトから導出される代わりに汎用タイトルを表示する問題

## 原文（日本語に翻訳）

リモートコントロールセッションが最初のプロンプトから導出する代わりに汎用タイトルを表示していた問題を修正。

## 原文（英語）

Fixed Remote Control sessions showing a generic title instead of deriving from the first prompt

## 概要

Claude.ai のリモートコントロール機能を使って開始したセッションが、最初のプロンプト内容からタイトルを生成せず、「Claude Code」などの汎用タイトルが表示されていた問題が修正されました。修正後は、通常のセッションと同様に最初のプロンプトに基づいたセッションタイトルが表示されます。

## 基本的な使い方

修正後は、リモートコントロールで開始したセッションでも意味のあるタイトルが表示されます：

```
# リモートコントロールセッション開始
# 最初のプロンプト: "Reactコンポーネントをリファクタリングしてください"

# 以前（汎用タイトル）
セッション: "Claude Code"

# 修正後（プロンプトから導出）
セッション: "Reactコンポーネントのリファクタリング"
```

## 実践例

### ### claude.aiからのリモートセッション管理

複数のリモートコントロールセッションを管理する場合：

```
セッション一覧（修正後）:
- Reactコンポーネントのリファクタリング
- APIエンドポイントの設計
- データベースのパフォーマンス最適化
```

各セッションの内容が一目でわかるようになります。

## 注意点

- この修正は Claude.ai のリモートコントロール機能を使用している場合に効果的
- セッションタイトルは最初のプロンプトの後に自動生成される
- `/rename` コマンドで手動でタイトルを変更することも可能（関連修正も v2.1.81 で実施）
