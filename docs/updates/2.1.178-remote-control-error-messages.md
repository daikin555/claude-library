## 原文（日本語に翻訳）

Remote Controlのエラーメッセージを改善しました: 接続失敗時にフッターに永続的な赤い「/rc failed」インジケータが表示されるようになりました。「まだ有効になっていない」エラーは、ゲート制限・チェック失敗・古いエンタイトルメント・組織ポリシーのいずれが原因かを説明するようになりました。

## 原文（英語）

Improved Remote Control error messages: connection failures now show a persistent red "/rc failed" indicator in the footer, and the "not yet enabled" error now explains whether it's a gate, a check failure, stale entitlement, or org policy

## 概要

Remote Control（`/rc`）機能の接続エラー時のUXが大幅に改善されました。接続失敗時はフッターに赤い「/rc failed」インジケータが常時表示されるようになり、見落としを防ぎます。また「まだ有効になっていない」というエラーが発生した場合、その原因（ゲート制限・チェック失敗・古いエンタイトルメント・組織ポリシー）が具体的に説明されるようになりました。

## 基本的な使い方

Remote Controlを使用するには `/rc` コマンドを実行します:

```bash
/rc
```

接続失敗時のフッター表示（改善後）:
```
[Session info] ... | /rc failed ←（赤色で常時表示）
```

## 実践例

### エラーメッセージの種類と対処法

**ゲート制限の場合:**
```
Remote Control is not yet enabled: feature is gated for your account.
Contact support to request access.
```
→ Anthropicサポートへのアクセス申請が必要です。

**チェック失敗の場合:**
```
Remote Control is not yet enabled: pre-flight check failed.
Ensure your network configuration allows WebSocket connections.
```
→ ネットワーク設定（ファイアウォール、プロキシ）を確認してください。

**古いエンタイトルメントの場合:**
```
Remote Control is not yet enabled: entitlement is stale.
Try signing out and signing back in to refresh your credentials.
```
→ `claude auth logout` → `claude auth login` で認証情報を更新してください。

**組織ポリシーの場合:**
```
Remote Control is not yet enabled: org policy disables this feature.
Contact your organization administrator.
```
→ 組織の管理者にポリシー変更を依頼してください。

### トラブルシューティングフロー

```bash
# 1. /rc を試みる
/rc

# 2. フッターの赤いインジケータを確認
# 3. エラーメッセージの原因を読む
# 4. 原因に応じた対処を実施:

# 認証更新
claude auth logout && claude auth login

# または /doctor で全体確認
/doctor
```

## 注意点

- Remote Controlはすべてのユーザー・プランで利用可能なわけではありません
- 「/rc failed」インジケータはセッションが終了するまで表示され続けます。接続が回復すると消えます
- エラーメッセージはより詳細になりましたが、解決策が組織ポリシーやゲートに起因する場合はユーザー側での対処は限られます
- プロキシ環境での使用時はWebSocket接続が通過できることを確認してください
