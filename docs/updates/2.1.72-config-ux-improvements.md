## 原文（日本語に翻訳）

`/config` を改善しました。Escキーで変更をキャンセル、Enterキーで保存・閉じる、Spaceキーで設定のトグルが可能になりました。

## 原文（英語）

Improved `/config` — Escape now cancels changes, Enter saves and closes, Space toggles settings

## 概要

`/config` コマンドの操作性が向上しました。これまで直感的でなかったキーボード操作が、一般的なUIの慣習に合わせて改善されています。Escキーで変更を取り消し、Enterキーで設定を保存して閉じる、Spaceキーでオン/オフのトグル設定を切り替えるという、シンプルでわかりやすい操作体系になりました。

## 基本的な使い方

```
/config
```

`/config` を実行すると設定画面が開きます。

| キー | 動作 |
|------|------|
| `Space` | チェックボックス/トグル設定のオン・オフ切り替え |
| `Enter` | 変更を保存して設定画面を閉じる |
| `Esc` | 変更をキャンセルして設定画面を閉じる（変更は保存されない） |

## 実践例

### 設定のトグル操作

```
/config
# → 設定一覧が表示される
# → 矢印キーで項目を選択
# → Space キーで有効/無効を切り替え
# → Enter で保存・閉じる
```

### 変更のキャンセル

```
/config
# → 設定を変更したが気が変わった場合
# → Esc キーを押す
# → 変更は保存されずに設定画面が閉じる
```

### 複数設定の一括変更

```
/config
# → 項目1を Space でトグル
# → 矢印キーで項目2に移動
# → Space でトグル
# → Enter で全変更を一括保存
```

## 設定できる主な項目

`/config` で変更できる代表的な設定には以下のものがあります。

- 応答言語の設定（`language`）
- ファイルパーミッション
- VSCode統合設定
- デバッグモード
- その他のユーザー設定

## 注意点

- `Esc` を押した場合、セッション内で行ったすべての未保存の変更が破棄されます
- `Enter` を押すと設定が `settings.json` に永続化されます
- `Space` はトグル型の設定（チェックボックス形式）にのみ有効です。テキスト入力フィールドでは通常のスペース文字の入力となります
- この変更により、以前の操作方法（特定のキーで保存するなど）が変わっています。古い操作に慣れている場合は注意してください
