## 原文（日本語訳）

エージェントのタスクリストが更新されている間の入力応答性を改善した。タスクの更新でUI全体が再描画されなくなった。

## 原文（英語）

Improved input responsiveness while agent task lists update — task updates no longer re-render the entire UI

## 概要

Claude Codeはタスクを分解して進捗を管理する際にタスクリスト（TODOリストのような表示）を更新しますが、これまではタスクの更新のたびにターミナルUI全体が再描画されており、頻繁にタスクが更新される場面ではキー入力への反応が遅く感じられることがありました。今回の改善により、タスクリストの更新がUI全体ではなく該当箇所のみの再描画で済むようになり、更新が頻発する状況でも入力の応答性が維持されるようになりました。

## 基本的な使い方

これはユーザーが明示的に呼び出す機能ではなく、内部的なUI描画の改善です。特別な操作は不要で、アップデート後は自動的に適用されます。

```text
Claudeがタスクリストを更新しながら作業を進めている最中に、
ユーザーがキーボードで入力（次の指示のタイピングやEscでの中断など）を行う
→ 修正後は入力がラグなく反映される
```

## 実践例

### 多数のサブタスクを持つ長時間タスクの実行中に指示を追記する

大規模なリファクタリングや複数ファイルにまたがる実装など、タスクリストが頻繁に更新されるような長時間の作業をClaudeに任せている最中でも、途中で気づいた追加指示や修正依頼をストレスなく入力できます。以前はタスク更新のたびにUI全体が再描画され、タイピング中の文字が遅延したり取りこぼされたりすることがありました。

### Escキーによる作業中断がスムーズになる

タスクリストが目まぐるしく更新されている状況で、方向性の誤りに気づいてすぐに `Esc` で処理を止めたい場合にも、UI全体の再描画による遅延がなくなったことで、キー入力への反応がより即時に行われるようになります。

## 注意点

- ユーザー側での設定変更は不要です。
- 効果が体感しやすいのは、タスクリストの更新頻度が高い（サブタスク数が多い、進捗が細かく変わる）セッションです。
- タスクリスト自体の機能や表示内容が変わるわけではなく、あくまで描画パフォーマンスの改善です。
