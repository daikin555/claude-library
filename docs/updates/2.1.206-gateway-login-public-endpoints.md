## 原文（日本語訳）

Gateway: `/login` が、Anthropicが運用するパブリックなgatewayエンドポイントに対応するようになった。

## 原文（英語）

Gateway: `/login` now supports Anthropic-operated public gateway endpoints

## 概要

Claude CodeのGateway機能は、組織が自前でLLMゲートウェイをデプロイしてポリシー適用や認証情報の一元管理を行える仕組みです。従来は自社ホストのgatewayを前提とした`/login`フローが中心でしたが、今回のアップデートでAnthropicが運用するパブリックなgatewayエンドポイントに対しても `/login` から直接サインインできるようになり、自前でインフラを構築しなくてもgateway経由のポリシー適用を利用しやすくなりました。

## 基本的な使い方

Gatewayのエンドポイントが設定された環境でClaude Codeを起動し、通常通りログインコマンドを実行します。

```text
/login
```

環境がAnthropic運用のパブリックgatewayエンドポイントを指すよう構成されている場合、ログインフローがgatewayを認識し、そのポリシーに沿った認証・設定適用が行われます。

## 実践例

### 自前インフラを持たない組織でのポリシー適用

自社でgatewayコンテナやPostgreSQLデータベースを運用するリソースがない小規模なチームでも、Anthropic運用のパブリックgatewayエンドポイントを利用することで、管理ポリシーの適用や利用状況の可視化といったgatewayの恩恵を受けられます。

```text
/login
```

エンドポイント設定（例: `ANTHROPIC_BASE_URL` など環境変数やクライアント設定）をAnthropic運用のgatewayに向けておけば、追加のホスティング作業なしにログインが完了します。

### 既存の自前ゲートウェイからの移行検証

これまで自社ホストのgatewayを使っていたチームが、Anthropic運用のパブリックエンドポイントへの移行を検証する際にも、同じ `/login` コマンドで挙動を比較できます。

```text
/login
```

gateway側でBedrockやGoogle Cloudなど複数プロバイダーへのフェイルオーバーを設定していた場合は、パブリックエンドポイント利用時の挙動差も併せて確認しておくとよいでしょう。

## 注意点

- 「Anthropic-operated public gateway endpoints」は、組織が自前でホストするgatewayとは異なり、Anthropicが提供・運用するエンドポイントを指します。利用可否や対象範囲は組織の契約・設定に依存します。
- gatewayはインフラ次第で推論トラフィックのルーティング先（Claude API、Amazon Bedrock、Google Cloudなど）が異なるため、ログイン後の挙動はgateway側の設定に依存します。
- 既存の自前gateway運用からの切り替えを行う場合は、認証情報やポリシー適用範囲が変わる可能性があるため、事前に組織の管理者へ確認することを推奨します。
