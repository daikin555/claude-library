## 原文（日本語に翻訳）

`/autocompact`、`/context`、`/exit`、`/reload-plugins` が Remote Control（モバイル/ウェブ）クライアントから使えるようになった

## 原文（英語）

`/autocompact`, `/context`, `/exit`, and `/reload-plugins` now work from Remote Control (mobile/web) clients

## 概要

Remote Control 機能（モバイルアプリやウェブブラウザからローカルの Claude Code セッションを操作する機能）で使えるコマンドが拡充されました。`/autocompact`、`/context`、`/exit`、`/reload-plugins` の4つのコマンドが Remote Control クライアントから実行できるようになり、外出先でもコンテキスト管理やセッション制御がより柔軟に行えます。

## 基本的な使い方

```bash
# ローカルで Remote Control サーバーを起動
/remote-control
# または
claude remote-control

# モバイルアプリ / claude.ai/code から接続後、以下のコマンドが使用可能：
/autocompact    # コンテキスト自動圧縮のオン/オフを切り替え
/context        # 現在のコンテキスト使用状況を確認
/exit           # セッションを終了
/reload-plugins # プラグインをリロード
```

## 実践例

### モバイルからコンテキストを管理する

外出先のスマートフォンから、ローカルで動作中のセッションのコンテキストを確認・管理：

```bash
# [ローカル PC でセッション開始]
claude
/remote-control

# [スマートフォンの Claude アプリから接続]
# コンテキスト使用量を確認
/context

# コンテキストが多くなっていたら圧縮を有効化
/autocompact on
```

### リモートからプラグインを更新する

ローカル PC に戻らずにプラグインの更新を反映させる：

```bash
# [PC のターミナルで作業中にプラグインを更新]
# → プラグインファイルを更新

# [モバイル / ウェブブラウザから]
/reload-plugins
# → 最新のプラグイン設定が反映される
```

### リモートからセッションをクリーンに終了する

作業が完了したらリモートからセッションを終了：

```bash
# [モバイルから作業完了後]
/exit
# → ローカルの Claude Code セッションが終了する
```

### Remote Control で使えるコマンド一覧

| コマンド | 説明 |
|---------|------|
| `/autocompact` | コンテキスト自動圧縮の切り替え |
| `/context` | コンテキスト使用量の表示 |
| `/exit` | セッションの終了 |
| `/reload-plugins` | プラグインの再読み込み |
| その他のコマンド | 通常のプロンプト送信など |

## 注意点

- Remote Control 機能は Pro、Max、Team、Enterprise プランで利用可能です
- ローカルの Claude Code（CLI または VS Code 拡張）が実行中である必要があります
- `/exit` を実行するとローカルのセッションも終了します。重要な作業が進行中の場合は注意してください
- プラグインの更新を反映するには、まずローカルでプラグインファイルを更新してから `/reload-plugins` を実行してください
