## 原文（日本語訳）
`ToolSearch` の結果を会話内インライン表示から簡潔な通知として表示するように変更しました

## 原文（英語）
Changed `ToolSearch` results to appear as a brief notification instead of inline in the conversation

## 概要
MCP ツールを検索する `ToolSearch` の結果表示が、会話を中断しない簡潔な通知形式に変更されました。これにより、会話の流れがよりスムーズになります。

## 基本的な使い方

`ToolSearch` は Claude が必要な MCP ツールを検索する際に自動的に使用されます。

## 実践例

### 変更前の動作

```bash
claude
> Slack に通知を送ってください

[ToolSearch results]
Found tools:
- mcp__slack__post_message
- mcp__slack__create_channel
...

Using tool: mcp__slack__post_message
```

会話内に検索結果が長々と表示され、読みづらい状態でした。

### 変更後の動作

```bash
claude
> Slack に通知を送ってください

ℹ Found Slack tools  # 簡潔な通知

✓ Message posted to #general
```

検索結果は控えめな通知として表示され、会話の流れを妨げません。

### 複数ツールの検索

```bash
claude
> GitHub の issue を確認して、Slack に報告してください

ℹ Found GitHub tools
ℹ Found Slack tools

✓ Checked 3 issues
✓ Posted summary to #dev-team
```

## 注意点

- **会話の流れ**: 検索結果が会話を中断しなくなり、より自然な対話が可能です
- **視認性**: 重要な情報（実際のツール実行結果）に集中できます
- **デバッグ**: 詳細なツール検索ログが必要な場合は、`--debug` モードを使用してください
- **MCP 設定**: MCP ツールの自動検索は `/config` で設定できます
