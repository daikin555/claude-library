## 原文（日本語に翻訳）

トランスクリプトビューのナビゲーションを追加: キーボードショートカット表示用`?`、ユーザープロンプト間ジャンプ用`{`/`}`、ショートカットパネル切り替え用`v`

## 原文（英語）

Added transcript view navigation: `?` for keyboard shortcuts, `{`/`}` to jump between user prompts, `v` to toggle shortcut panel

## 概要

トランスクリプトビュー（会話履歴ビュー）にキーボードショートカットが追加され、長い会話履歴を効率よくナビゲートできるようになりました。`?`キーでショートカット一覧を表示し、`{`/`}`キーでユーザーのプロンプト間を素早くジャンプできます。

## 基本的な使い方

トランスクリプトビュー内で使えるキーボードショートカット：

| キー | 動作 |
|-----|------|
| `?` | キーボードショートカット一覧を表示 |
| `{` | 前のユーザープロンプトにジャンプ |
| `}` | 次のユーザープロンプトにジャンプ |
| `v` | ショートカットパネルの表示/非表示を切り替え |

## 実践例

### 長い会話履歴の効率的なナビゲーション

```
# トランスクリプトビューを開く（Ctrl+R などで開く）
# → 長い会話履歴が表示される

# { キーで前のユーザープロンプトにジャンプ
# } キーで次のユーザープロンプトにジャンプ
# → 会話のターンをすばやく移動できる

# ? キーでショートカット一覧を確認
# v キーでショートカットパネルを常時表示
```

### 特定の会話部分を素早く探す

複数ターンにわたる作業セッションで過去の指示を見直す際、`{`/`}`キーを使うことで各ユーザーターン間を素早く移動できます。スクロールよりも効率的に目的の箇所を見つけられます。

## 注意点

- これらのショートカットはトランスクリプトビュー内でのみ有効です
- バージョン2.1.139では、マウスクリック後にショートカットが効かなくなるバグも同時に修正されました
- Bash履歴モードの上矢印キーによる履歴ナビゲーションも同バージョンで改善されています
