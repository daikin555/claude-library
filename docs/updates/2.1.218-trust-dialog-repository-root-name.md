## 原文（日本語に翻訳）

信頼ダイアログを改善し、信頼付与の対象となるリポジトリルートの名前を明示するようにしました。

## 原文（英語）

Improved trust dialogs to name the repository root the grant covers

## 概要

Claude Codeでは、新しいフォルダやリポジトリを開いた際に、そのディレクトリを「信頼する」かどうかを確認する信頼ダイアログ（trust dialog）が表示されます。これまでのダイアログでは、信頼を付与する対象がどのディレクトリ（リポジトリルート）なのかが明確に示されていないことがあり、特にモノレポやネストしたリポジトリ構成では、ユーザーがどの範囲に対して信頼を許可しようとしているのか分かりにくい場合がありました。v2.1.218では、信頼ダイアログに信頼付与の対象となるリポジトリルートの名前が明示されるようになり、ユーザーは自分が何を信頼しようとしているのかをより正確に把握した上で判断できるようになりました。

## 基本的な使い方

新しいプロジェクトディレクトリでClaude Codeを起動すると、信頼ダイアログが表示されます。

```bash
cd ~/projects/my-app
claude
```

```
このフォルダを信頼しますか？

  リポジトリルート: ~/projects/my-app

  Claude Codeはこのディレクトリ内のファイルを読み書きしたり、
  コマンドを実行したりできるようになります。

  > はい、信頼する
    いいえ
```

以前は対象パスの表示が不明瞭な場合がありましたが、現在は信頼付与の対象となるリポジトリルートのパス名が明示的に表示されます。

## 実践例

### モノレポのサブディレクトリで作業する場合

```bash
cd ~/monorepo/packages/frontend
claude

# リポジトリルートが ~/monorepo であっても、
# 現在の作業ディレクトリが ~/monorepo/packages/frontend
# であることがダイアログ上で明確に示され、
# 信頼範囲がリポジトリ全体なのか
# サブディレクトリなのかを誤解しにくくなる
```

### 複数のネストしたGitリポジトリがある場合

```bash
cd ~/work/outer-repo/vendor/inner-repo
claude

# vendor/inner-repo が独立したGitリポジトリである場合、
# ダイアログにはどちらのリポジトリルートを
# 信頼対象としているかが名前付きで表示されるため、
# 意図しないリポジトリへの信頼付与を防ぎやすくなる
```

### CI/自動化環境で信頼済みディレクトリを事前設定する場合

```bash
# 対話的な信頼ダイアログを避けたい場合は、
# 事前に信頼済みディレクトリとして登録しておく
claude --add-dir ~/projects/my-app

# または settings.json 等で信頼済みパスを管理し、
# どのリポジトリルートを信頼しているかを
# チームで明確に共有できる
```

## 注意点

- この改善はClaude Code v2.1.218で実装されたUI上の変更であり、信頼の仕組み自体（一度信頼したディレクトリは再確認なしで利用できる、など）に変更はありません
- モノレポやネストしたリポジトリ構成では、どのリポジトリルート単位で信頼が管理されるかを正しく理解しておくことが重要です。ダイアログに表示されるリポジトリルート名を必ず確認してから信頼を許可してください
- 信頼していないディレクトリでは、ファイルの読み書きやコマンド実行が制限されるため、意図した範囲を信頼しているかをこのダイアログで確認する習慣をつけると安全です
