## 原文（日本語に翻訳）

- `.git`サフィックスなしのマーケットプレイスgit URLのサポートを追加（Azure DevOps、AWS CodeCommit）

## 原文（英語）

- Added support for marketplace git URLs without `.git` suffix (Azure DevOps, AWS CodeCommit)

## 概要

Claude Codeのプラグインマーケットプレイスで、`.git`サフィックスが付かないgitリポジトリURLをサポートするようになりました。Azure DevOpsやAWS CodeCommitなどのプラットフォームではリポジトリURLに`.git`が含まれないことがあり、これまではマーケットプレイスの追加に失敗していました。この修正により、これらのプラットフォームでホストされたプラグインマーケットプレイスを正しく追加・利用できるようになります。

## 基本的な使い方

`.git`サフィックスなしのURLでマーケットプレイスを追加できます:

```bash
# Azure DevOpsのリポジトリ
/plugin marketplace add https://dev.azure.com/org/project/_git/marketplace-repo

# AWS CodeCommitのリポジトリ
/plugin marketplace add https://git-codecommit.us-east-1.amazonaws.com/v1/repos/marketplace-repo

# GitHubの場合（従来通り.git付きでもなしでも動作）
/plugin marketplace add https://github.com/org/marketplace-repo
```

## 実践例

### Azure DevOpsでのマーケットプレイス利用

企業でAzure DevOpsを使用している場合:

```bash
# Azure DevOpsのリポジトリをマーケットプレイスとして追加
/plugin marketplace add https://dev.azure.com/mycompany/tools/_git/claude-plugins

# プラグインを検索・インストール
/plugin install my-plugin@claude-plugins
```

### AWS CodeCommitでのマーケットプレイス利用

AWSインフラを使用している場合:

```bash
# CodeCommitのリポジトリをマーケットプレイスとして追加
/plugin marketplace add https://git-codecommit.ap-northeast-1.amazonaws.com/v1/repos/claude-marketplace

# プラグインの一覧を確認
/plugin marketplace list
```

### settings.jsonでの設定

チーム全体で共有するマーケットプレイスを設定ファイルで管理:

```json
{
  "extraKnownMarketplaces": [
    "https://dev.azure.com/mycompany/tools/_git/claude-plugins"
  ]
}
```

## 注意点

- この変更はURLの解析処理のみに関するもので、gitのクローン操作自体には影響しません
- Azure DevOpsやAWS CodeCommitへのアクセスには適切な認証情報の設定が必要です
- プライベートリポジトリの場合、gitの認証設定（SSH鍵やクレデンシャルヘルパー）が事前に必要です
- GitHubやGitLabなど`.git`サフィックス付きURLのプラットフォームでは従来通り動作します
