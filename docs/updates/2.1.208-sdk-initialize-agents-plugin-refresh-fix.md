## 原文（日本語訳）

クライアントがアタッチする前にプラグインのリフレッシュが実行された場合、SDKセッションが `initialize` リクエストで定義されたエージェントを失ってしまう不具合を修正した。

## 原文（英語）

Fixed SDK sessions losing agents defined via the initialize request when a plugin refresh ran before the client attached

## 概要

Agent SDKでは、`agents` オプションなどを通じてセッション開始時（`initialize` リクエスト）にプログラム的にカスタムエージェント（サブエージェント）を定義できます。一方で[プラグイン](https://code.claude.com/docs/en/agent-sdk/plugins)からもエージェントを追加できます。これまでは、クライアントがセッションにアタッチするより前にプラグインのリフレッシュ処理が実行されてしまうタイミングの競合状態（レースコンディション）があり、その場合に `initialize` リクエストで定義したはずのエージェントが失われてしまう不具合がありました。今回の修正で、プラグインのリフレッシュのタイミングにかかわらず、`initialize` で定義したエージェントが確実に保持されるようになりました。

## 基本的な使い方

`query()` の `options.agents` にカスタムエージェントを定義してSDKセッションを開始します。

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const message of query({
  prompt: "このコードをレビューして",
  options: {
    agents: {
      "code-reviewer": {
        description: "コードレビューに特化したエージェント",
        prompt: "あなたはコードレビュー専門のエージェントです。",
      },
    },
  },
})) {
  // "code-reviewer" エージェントが利用可能な状態でセッションが開始される
}
```

## 実践例

### 独自ホストアプリでプログラム定義エージェントとプラグインを併用する

自社のツールやIDE拡張などでAgent SDKをホストしている場合、プログラムで定義したエージェントと、プラグイン由来のエージェントの両方を同時に使うことがあります。

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const message of query({
  prompt: "リリースノートを作成して",
  options: {
    agents: {
      "release-writer": {
        description: "リリースノート作成専門エージェント",
        prompt: "変更内容を要約してリリースノートを書きます。",
      },
    },
    plugins: [{ type: "local", path: "./my-plugin" }],
  },
})) {
  if (message.type === "system" && message.subtype === "init") {
    // 修正前は、プラグインのリフレッシュタイミング次第で
    // "release-writer" が message.agents から欠落することがあった
    console.log("Agents:", message.agents);
  }
}
```

### 起動直後にクライアントのアタッチが遅れがちな環境

サンドボックス環境やコンテナ起動直後など、クライアントのアタッチにわずかな遅延が生じやすい状況では、プラグインのリフレッシュが先に走ることでレースコンディションが顕在化しやすくなります。v2.1.208以降ではこうした環境でも `initialize` リクエストで定義したエージェントが確実に利用可能になります。

## 注意点

- この不具合はタイミング依存のレースコンディションであり、常に再現するわけではなく、プラグインのリフレッシュとクライアントのアタッチの順序によって発生有無が変わります。
- 影響を受けるのは主にAgent SDK経由でプラグインとプログラム定義エージェントを併用しているホストアプリケーションです。
- 影響を受けていた場合は、Claude Code / Agent SDKを最新版（v2.1.208以降）に更新することで解消します。
