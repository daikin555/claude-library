## 原文（日本語訳）

セッション途中でOAuthトークンが失効またはローテーションされた後、autoモードが「HTTP 401」という分類器（classifier）のエラーを理由にコマンドを拒否してしまう問題を修正しました。

## 原文（英語）

Fixed auto mode denying commands with "HTTP 401" classifier errors after the OAuth token expired or rotated mid-session

## 概要

Claude Codeの「autoモード」では、各ツール呼び出しの安全性をモデルベースの分類器（classifier）が事前に評価し、危険と判断されたものだけをブロックします。この分類器はAPIリクエストを内部的に行いますが、セッションの途中でOAuthトークンが失効・ローテーションされると分類器のリクエストが「HTTP 401（認証エラー）」で失敗することがありました。これまでは、この認証エラー自体が「危険なコマンド」と誤解釈され、本来問題のないコマンドまで拒否されてしまう不具合がありました。今回の修正により、分類器の認証エラーが原因でコマンドが不当にブロックされることがなくなります。

## 基本的な使い方

この修正はautoモードの内部動作に対するものであり、追加の設定は不要です。修正後は、長時間セッションの途中でOAuthトークンが更新されても、分類器のエラーによってコマンドが誤って拒否されることなく、autoモードでの作業を継続できます。

## 実践例

### 長時間のautoモードセッションでの継続性向上

長時間にわたってautoモードでタスクを実行していると、その間にOAuthトークンの有効期限切れやローテーションが発生することがあります。これまでは、そのタイミングで実行しようとしたコマンドが分類器のHTTP 401エラーのために拒否されることがありましたが、修正後はトークンが更新された後も正常に分類が継続され、安全なコマンドは引き続き自動実行されます。

```bash
# autoモードで長時間タスクを実行する例
claude --permission-mode auto
```

### 分類器エラーとコマンド拒否の切り分け

もしautoモード中にコマンドが不自然に拒否される場合は、認証まわりの問題かどうかを切り分けるために再ログインを試すのも有効です。

```bash
# Claude Codeの認証状態を再設定する
claude /login
```

### セッションの再開で解消するケース

セッション途中でトークンの不整合が疑われる場合、セッションを再開することで新しいトークンが取得され、分類器が正しく動作するようになることがあります。

```bash
claude --resume
```

## 注意点

- この修正はautoモード（`--permission-mode auto` など、モデルベースの分類器を使う権限モード）を使用している場合にのみ関係します
- 分類器はセッションのモデル設定に関わらず内部的に動作するため、トークンの失効タイミングはユーザーの操作とは独立して発生することがあります
- 認証エラー以外の理由でコマンドが拒否される場合は、分類器が実際にリスクを検出している可能性があるため、内容を確認してください
