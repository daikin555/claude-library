# APIリトライのカウントダウンが「0s」で止まる問題を修正

## 原文（日本語に翻訳）

APIリトライのカウントダウンが試行間で「0s」のままになる問題を修正しました。

## 原文（英語）

Fixed API retry countdown sticking at "0s" instead of counting down between attempts

## 概要

APIレート制限やエラーが発生してClaude Codeがリトライ待ちの状態になった際、カウントダウンタイマーが「0s」で止まってしまい実際にカウントダウンが動いていないように見える問題が修正されました。今後は残り時間が正確にカウントダウン表示されるため、リトライまでの待ち時間が分かりやすくなります。

## 基本的な使い方

```bash
# APIレート制限などでリトライが発生した場合
claude "長いタスクを実行して"

# v2.1.126以前:
# ⟳ Retrying in 0s...  ← ずっと0sのまま

# v2.1.126以降:
# ⟳ Retrying in 30s...
# ⟳ Retrying in 29s...
# ⟳ Retrying in 28s...
# → 正確にカウントダウンが表示される
```

## 注意点

- この修正はUIの表示改善です。リトライの動作自体は変更されていません
- カウントダウンが表示されている間は、Claude Codeがリトライ待機中です
- リトライ回数に上限がある場合、上限に達するとエラーメッセージが表示されます
