## 原文（日本語に翻訳）

長時間実行されるツール呼び出しがフリーズしているように見えないよう、折りたたまれたツールサマリー行にライブの経過時間カウンターを追加しました

## 原文（英語）

Added a live elapsed-time counter to the collapsed tool summary line so long-running tool calls visibly tick instead of looking stuck

## 概要

Claude Code v2.1.210では、Bashコマンドの実行やWeb検索など時間のかかるツール呼び出しが画面上で折りたたまれている（collapsed）際に、経過時間がリアルタイムでカウントアップ表示されるようになりました。これまでは折りたたみ表示中は経過時間が更新されず、処理がフリーズしているのか実行中なのか判別しづらいという問題がありました。この修正により、ユーザーは一目でツールが「まだ動いている」ことを確認できます。

## 基本的な使い方

### 修正前の問題

```bash
claude

> 大きなリポジトリ全体でテストを実行して

# 折りたたまれたサマリー行:
# ⏺ Bash(npm test)…
#
# 数十秒経過しても表示が変化しない
# → 本当に動いているのか、ハングしているのか分からない
```

### 修正後の動作

```bash
claude

> 大きなリポジトリ全体でテストを実行して

# 折りたたまれたサマリー行:
# ⏺ Bash(npm test)… (3s)
# ⏺ Bash(npm test)… (7s)
# ⏺ Bash(npm test)… (12s)
#
# 経過秒数が1秒ごとにインクリメントされて表示される
# → 実行中であることが視覚的に明確
```

## 実践例

### 長時間実行のビルドコマンド

```bash
claude

> プロジェクト全体をビルドして

# ⏺ Bash(npm run build)… (45s)
# ビルドが長引いても、カウンターが動いていれば
# プロセスが生きていることが分かる
```

### バックグラウンドタスクとの併用

```bash
claude

> 開発サーバーをバックグラウンドで起動して、
> その間にテストカバレッジを確認して

# バックグラウンドタスクとは別に、
# フォアグラウンドで実行中のツール呼び出しにも
# 経過時間カウンターが表示される
```

### 複数ツールが連続実行される場面

```bash
claude

> リポジトリ内のすべてのTypeScriptファイルを型チェックして

# ⏺ Bash(tsc --noEmit)… (2s)
# ⏺ Bash(tsc --noEmit)… (8s)
# ⏺ Bash(tsc --noEmit)… (15s)
# 完了後:
# ⏺ Bash(tsc --noEmit) (16.2s)
#
# 完了時には最終的な所要時間が固定表示される
```

## 注意点

- この機能はClaude Code v2.1.210（2026年7月15日リリース）で追加されました
- カウンターは折りたたまれたサマリー行にのみ表示され、展開表示（expanded）時の挙動には影響しません
- 経過時間はUI描画上のカウントであり、実際のツール実行時間の正確な計測値とは若干のずれが生じる場合があります
- スクリーンリーダーモードなど、他のアクセシビリティ機能との相互作用については今後のリリースで調整される可能性があります
- 非常に短時間で完了するツール呼び出しではカウンターが表示される前に完了するため、視認できないことがあります
