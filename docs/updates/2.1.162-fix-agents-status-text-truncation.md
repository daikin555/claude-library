# 修正：claude agents のライブステータステキストが広いターミナルで切り捨てられる問題

## 原文（日本語に翻訳）

広いターミナルでライブステータステキスト（ツール引数、返答、プロンプト、実行出力）が 60〜120 カラムで切り捨てられていた `claude agents` の問題を修正しました。ステータスの詳細はターミナルの全幅を使用するようになりました。

## 原文（英語）

Fixed `claude agents` cutting live status text (tool args, replies, prompts, exec output) at 60–120 columns on wide terminals; the status detail now uses the full terminal width

## 概要

`claude agents` コマンドでバックグラウンドセッションを監視している際、ツール引数・返答・プロンプト・実行出力などのライブステータステキストが、ターミナルの実際の幅に関わらず 60〜120 カラムで強制的に切り捨てられていました。v2.1.162 から、ステータス詳細がターミナルの全幅を活用して表示されます。これにより、広いターミナル（4K モニターなど）でも情報が途切れずに表示されます。

## 基本的な使い方

```bash
# バックグラウンドエージェントの状態を監視
claude agents

# 修正後: ターミナル幅に合わせてフル表示
# 例（200カラムのターミナル）:
# [実行中] タスク: ファイルの解析中 - Reading: /home/user/project/src/components/UserDashboard.tsx (line 1-250)
# ↑ 以前は途中で切れていたが、修正後はフル表示
```

## 実践例

### 広いターミナルでのエージェント監視

```bash
# ターミナルを広くして詳細なステータスを確認
# 以前: "Running: Analyzing file... args: {file: '/home/user/project/src/comp..."
# 修正後: "Running: Analyzing file... args: {file: '/home/user/project/src/components/UserDashboard.tsx', startLine: 1, endLine: 250}"
claude agents
```

### tmux での分割画面監視

```bash
# 広い tmux ペインでエージェントを監視
# ターミナル幅を最大限活用した詳細ステータス表示
tmux split-window -h
claude agents
```

### ログ出力との比較

```bash
# ステータスの完全な内容を確認（修正後）
claude agents --json | jq '.[] | .status'
# JSON モードも合わせて確認することで詳細が把握できる
```

## 注意点

- この修正はターミナル幅が広い場合（通常 120 カラム以上）に特に効果があります
- ターミナルのリサイズに動的に対応します
- 非常に長いツール引数や出力も途切れなく表示されるようになります
- 一部のターミナルエミュレーターでは折り返し設定が必要な場合があります
