## 原文（日本語に翻訳）

`/voice` の切り替え後に古い認証チェックを解消するために `/login` が必要になっていたボイスモードのバグを修正しました。

## 原文（英語）

Fixed voice mode requiring `/login` to clear a stale auth check after toggling `/voice`

## 概要

`/voice` コマンドでボイスモードを切り替えた後に、古い認証状態が残り `/login` を実行しなければならなかったバグを修正しました。この修正により、`/voice` トグル後に余分なログイン操作が不要になり、ボイスモードがスムーズに切り替えられるようになります。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
# ボイスモードの切り替え（修正後は /login が不要）
/voice
```

## 実践例

### 修正前の問題

以前は `/voice` を切り替えた後に以下の問題が発生していました：

```
/voice          # ボイスモードを有効化
/voice          # ボイスモードを無効化
# → 認証エラーが発生し、/login が必要になっていた
/login          # 不要なはずのログインが必要だった
```

### 修正後の動作

```
/voice          # ボイスモードを有効化
/voice          # ボイスモードを無効化
# → そのまま通常の操作を継続可能（/login 不要）
```

## 注意点

- ボイスモードは Claude.ai のサブスクリプションが必要な機能です
- 正規の認証エラー（セッション期限切れなど）では引き続き `/login` が必要です
