## 原文（日本語に翻訳）

大きなthinking予算を使用中にコンテキスト超過エラーが発生した際、同一の失敗確定リクエストを再送し続けるリトライループを修正しました。また、`Ctrl+B`でのバックグラウンド化にも、他の経路と同じバックグラウンドシェルの上限が適用されるようになりました

## 原文（英語）

Fixed a retry loop that re-sent identical doomed requests after a context-overflow error with a large thinking budget; `Ctrl+B` backgrounding now applies the same background-shell caps as other paths

## 概要

大きな拡張thinking予算（thinking budget）を使用している際にコンテキストウィンドウ超過エラーが発生すると、以前はClaude Codeが全く同じ内容のリクエストを何度も再送してしまい、毎回同じ理由で失敗するリトライループに陥ることがありました。この修正により、コンテキスト超過が原因で失敗することが分かっているリクエストをそのまま再送しなくなり、無駄なリトライが発生しなくなりました。あわせて、`Ctrl+B`でタスクをバックグラウンド化する際にも、他の方法でバックグラウンド化した場合と同じ同時実行数・リソースの上限が一貫して適用されるようになりました。

## 注意点

- Claude Code v2.1.218で修正
- 大きなthinking予算を設定している場合に発生しやすかった問題であり、ユーザー側の追加設定は不要
- `Ctrl+B`によるバックグラウンド化も、他のバックグラウンドシェル実行経路と同じ上限（同時実行数などのキャップ）で管理されるようになり、挙動が一貫した
- コンテキスト超過そのものを回避する機能ではなく、超過が確定した際の無駄なリトライを止めるための修正
