## 原文（日本語に翻訳）

`claude agents`: リストに URL を入力すると、最初のプロンプトにその URL が含まれていたセッションにフィルタリングされるようになりました。

## 原文（英語）

`claude agents`: typing a URL into the list now filters to the session whose first prompt contained it

## 概要

`claude agents` の一覧画面で URL を入力すると、その URL を最初のプロンプトに含んでいたセッションに絞り込めるようになりました。多くのバックグラウンドエージェントセッションを管理している場合に、特定の URL に関連したタスクを素早く見つけるのに役立ちます。

## 基本的な使い方

```bash
# エージェント一覧を開く
claude agents
```

一覧画面で URL を入力すると、その URL を含む最初のプロンプトを持つセッションに絞り込まれます。

## 実践例

### 複数の Web 関連タスクを管理

複数の URL に関するタスクをバックグラウンドで実行した後、特定のものを検索する：

```bash
# バックグラウンドでタスクを作成
claude --bg "https://github.com/anthropics/claude-code のREADMEを日本語に要約して"
claude --bg "https://docs.anthropic.com の最新ドキュメントを確認して"

# agents一覧でフィルタリング
claude agents
# → "github.com" と入力すると最初のセッションに絞り込まれる
```

### 特定プロジェクトのタスクを検索

```bash
# プロジェクトのリポジトリURLでフィルタリング
claude agents
# → "my-project" を含むURLを入力して関連セッションを特定
```

## 注意点

- フィルタリングは**最初のプロンプト**に URL が含まれているセッションを対象とします
- 途中のプロンプトに URL が含まれているだけのセッションはフィルタリングされません
- 部分一致でフィルタリングされます（完全一致は不要）
