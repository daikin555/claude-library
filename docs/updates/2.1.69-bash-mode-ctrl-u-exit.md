## 原文（日本語に翻訳）

- 空のBashプロンプト（`!`）でCtrl+Uを押すとBashモードを終了できるように追加（`escape`および`backspace`と同等の動作）
- インタビュー質問でのオプション選択に数値キーパッドのサポートを追加（以前はQWERTYの数字行のみ対応）

## 原文（英語）

- Added Ctrl+U on an empty bash prompt (`!`) to exit bash mode, matching `escape` and `backspace`
- Added numeric keypad support for selecting options in Claude's interview questions (previously only the number row above QWERTY worked)

## 概要

Claude CodeのBashモードにおけるキーボード操作が改善されました。空のBashプロンプトでCtrl+Uを押すことでBashモードを素早く終了できるようになりました。また、Claudeが質問を提示する際、テンキー（右側の数字キー）でも選択肢を選べるようになりました。

## 基本的な使い方

### Ctrl+UでBashモードを終了する

```
# Bashモードに入る
!
# 空の状態でCtrl+U → Bashモードを終了
```

Bashモードを終了する方法が統一されました：
- `Escape` キー
- `Backspace` キー（入力が空の場合）
- `Ctrl+U`（新規追加）

### テンキーでの選択

Claudeが複数の選択肢を提示する場面（例：ファイルの上書き確認など）で、テンキーの数字キーを使って選択できます：

```
どのアプローチを使いますか？
1. 新しいファイルを作成する
2. 既存のファイルを更新する
3. バックアップしてから更新する

> [テンキーの 1, 2, 3 で選択可能]
```

## 実践例

### 長いBashコマンドを入力してから中断する場合

```bash
# !でBashモードに入り、コマンドを入力し始める
! git log --oneline --graph --all
# 途中でやめたくなったら Ctrl+U で入力をクリアしてモードを抜ける
```

### リモートサーバー接続時（テンキーが便利）

外付けテンキーやラップトップの数字ロックキーを使った場合でも、Claudeの選択肢入力が正しく機能するようになりました。

## 注意点

- Ctrl+Uは**入力が空の場合のみ**Bashモードを終了します。入力中にCtrl+Uを押すと行全体が削除されます（標準的なターミナルの動作）
- テンキーのサポートはインタビュー形式の選択肢（数字選択）のみです
