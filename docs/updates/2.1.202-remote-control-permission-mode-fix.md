## 原文（日本語訳）

`/remote-control` セッションにおいて、モバイルアプリおよびWebアプリで誤ったパーミッションモードが表示される問題を修正しました。

## 原文（英語）

Fixed `/remote-control` sessions showing the wrong permission mode in the mobile and web apps

## 概要

本項目はバグ修正です。これまで `/remote-control` を使ってローカルのClaude Codeセッションをモバイルアプリやclaude.ai/code（Webアプリ）から操作する際、実際にローカルで適用されているパーミッションモード（Manual / Accept Edits / Plan など）とは異なるモードがモバイル・Web側の画面に表示されることがありました。今回の修正により、リモート側のUIに表示されるモードとローカルセッションで実際に有効なモードが一致するようになり、ユーザーが誤った権限状態を前提に操作してしまうリスクが解消されました。

## 基本的な使い方

`/remote-control`（または `/rc`）は、ローカルで実行中のClaude Codeセッションをスマートフォンやブラウザから継続操作できるようにする機能です。

```bash
# 既存のインタラクティブセッション内から実行
/remote-control

# セッション名を指定して開始することも可能
/remote-control My Project
```

```bash
# CLIから直接リモートコントロール付きでセッションを開始する場合
claude --remote-control

# サーバーモードで複数セッションを待ち受ける場合
claude remote-control
```

接続後は、claude.ai/code またはClaudeモバイルアプリ（iOS/Android）のセッション一覧から対象セッションを開き、プロンプト欄の横にあるモードのドロップダウンから権限モードを確認・切り替えできます。リモートコントロールのセッションでは、選択できるモードは「Ask permissions（都度確認）」「Auto accept edits（編集を自動承認）」「Plan mode（プランモード）」の3種類で、Auto modeやBypass permissionsはリモートからは選択できません。

## 実践例

### 外出先からモバイルアプリでコーディングを続ける

自宅のPCで作業を開始し、外出先からモバイルアプリでセッションを引き継ぐケースです。

```bash
# PC側で開始
cd my-project
claude --remote-control "My Project"
```

モバイルアプリの「Code」タブからセッションを開くと、PC側で設定したパーミッションモードがそのままモバイル画面のモード表示に反映されます。今回の修正前は、実際にはManual（都度確認）モードで動作しているのに、モバイル画面上ではAccept Edits等の別モードが表示されてしまうことがあり、ユーザーが「編集は自動承認されるはず」と誤解して確認せずに離席してしまう、といった混乱が起きる可能性がありました。修正後は表示と実態が一致するため、安心して権限状態を確認できます。

### 起動時にパーミッションモードを指定してリモートセッションを開始する

サーバーモードでリモートコントロールセッションを起動する際、開始時点のパーミッションモードを明示的に指定できます。

```bash
claude remote-control --permission-mode acceptEdits
```

この場合、Web/モバイル側のモードドロップダウンには「Auto accept edits」が表示されるべきですが、修正前はここに古い情報やデフォルトのモードが表示されてしまうケースがありました。修正後は、起動時に指定したモードが正しくリモートクライアントのUIに反映されます。セッション中に `Shift+Tab` 相当の操作（Web/モバイルではドロップダウン選択）でモードを切り替えた場合も、変更が即座にリモート画面へ同期されるようになりました。

## 注意点

- この修正はUI表示に関するバグ修正であり、実際にツール呼び出しを承認・拒否する権限ロジック自体を変更するものではありません。ただし、表示が誤っていると意図しない操作の承認・拒否につながる可能性があるため、重要な修正です。
- リモートコントロールセッションで選択できるモードは「Ask permissions」「Auto accept edits」「Plan mode」の3つに限られ、Auto modeやBypass permissionsモードはモバイル・Webからは利用できません（ローカルCLI側で該当フラグを付けて起動している場合でも同様です）。
- パーミッションモードの切り替えはチャット欄でClaudeに指示する方法ではなく、UI上のモードドロップダウン（Web/モバイル）やCLIの `Shift+Tab`（ローカル端末）で行う必要があります。
- Remote Controlはリサーチプレビュー機能であり、Team/Enterpriseプランではデフォルトで無効になっている場合があります。管理者がAdmin設定でRemote Controlを有効化する必要があります。
- 本修正を利用するには、修正が含まれるバージョン（v2.1.202以降）へのアップデートが必要です。`claude --version` でバージョンを確認してください。
