## 原文（日本語に翻訳）

パーティション対応AWSリージョンおよびプロキシ限定ネットワークにおいて、assume-roleプロファイルに対するBedrockセットアップウィザードのプロファイル検証が失敗する問題を修正しました

## 原文（英語）

Fixed the Bedrock setup wizard failing profile verification for assume-role profiles in partitioned AWS regions and on proxy-only networks

## 概要

`claude setup`のBedrock設定ウィザードにおいて、AWS GovCloudや中国リージョンなどのパーティション対応AWSリージョン、またはアウトバウンド通信がプロキシ経由に限定されたネットワークで、assume-role（クロスアカウントのロールスイッチ）を使用するAWSプロファイルのプロファイル検証がエラーになる問題がありました。この修正により、これらの環境でもプロファイル検証が正しく成功するようになりました。エンタープライズ環境やAWS Bedrockを利用するユーザーに関連する修正です。

## 注意点

- Claude Code v2.1.218で修正されました
- 主にAWS GovCloud・中国リージョンなどのパーティションリージョン、およびプロキシ経由のみでアウトバウンド通信が可能なネットワーク上でassume-roleプロファイルを使用しているエンタープライズ/Bedrockユーザーに影響します
- 通常のAWSリージョンや直接インターネット接続が可能な環境では、この問題の影響を受けていませんでした
