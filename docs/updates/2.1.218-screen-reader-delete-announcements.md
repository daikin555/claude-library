## 原文（日本語に翻訳）

`--ax-screen-reader` モードにおいて、単語削除・行削除（`Option+Delete`、`Ctrl+W`、`Cmd+Backspace`、`Ctrl+U`、`Ctrl+K`）の際に削除されたテキストをスクリーンリーダーで読み上げる機能を追加しました。

## 原文（英語）

Added screen-reader announcements of deleted text for word and line deletions (`Option+Delete`, `Ctrl+W`, `Cmd+Backspace`, `Ctrl+U`, `Ctrl+K`) in `--ax-screen-reader` mode

## 概要

`--ax-screen-reader` モードは、スクリーンリーダー利用者向けにプレーンテキストで画面を描画するオプトイン機能です。これまでは、単語単位や行単位でテキストを削除するキー操作を行っても、何が削除されたのかがスクリーンリーダーに読み上げられず、入力中のプロンプトがどう変化したのか把握しづらいという課題がありました。v2.1.218からは、こうした削除操作を行った際に削除されたテキストの内容がアナウンスされるようになり、VoiceOverやNVDA、JAWSなどのスクリーンリーダーを使うユーザーが、編集内容を音声で正確に把握できるようになります。

## 基本的な使い方

`--ax-screen-reader` モードを有効にした状態でClaude Codeを起動します。

```bash
claude --ax-screen-reader
```

入力欄でカーソル付近のテキストを削除するキー操作（例: `Ctrl+W` でカーソル左の単語を削除）を行うと、削除された単語がスクリーンリーダーによって読み上げられます。

```
入力中: "このファイルを読んで修正して"
Ctrl+W を押す
→ "修正して" が削除される
→ スクリーンリーダーが削除された内容（例: "修正して"）をアナウンス
```

## 実践例

### 単語単位の削除（Option+Delete / Ctrl+W / Cmd+Backspace）

カーソルの左側にある1単語を削除する操作です。いずれも同じ「単語削除」として扱われ、削除された単語が読み上げられます。

```
macOS:   Option+Delete または Cmd+Backspace
Linux/Windows端末など: Ctrl+W

例:
入力: "src/index.ts を編集して"
Ctrl+W を押す → "編集して" が削除され、削除内容がアナウンスされる
```

### カーソルから行頭までの削除（Ctrl+U）

カーソル位置から行の先頭までを削除するキー操作です。誤って長い文章を消してしまった場合でも、削除された範囲を音声で確認できます。

```
入力: "この関数のリファクタリングをお願いします"
（カーソルを "リファクタリング" の直前に置いた状態で）
Ctrl+U を押す
→ "この関数の" が削除され、削除内容がアナウンスされる
```

### カーソルから行末までの削除（Ctrl+K）

カーソル位置から行の末尾までを削除するキー操作です。長いプロンプトの後半部分だけを消したい場合に使います。

```
入力: "テストを追加してからコミットしてください"
（カーソルを "から" の直後に置いた状態で）
Ctrl+K を押す
→ "コミットしてください" が削除され、削除内容がアナウンスされる
```

## 注意点

- この機能は `--ax-screen-reader` フラグ、`CLAUDE_AX_SCREEN_READER=1` 環境変数、または settings の `"axScreenReader": true` のいずれかでスクリーンリーダーモードを有効にしている場合にのみ動作します。
- 通常モード（スクリーンリーダーモード無効時）では、この削除アナウンスは行われません。
- キーの割り当てはOSやターミナル、キーボード設定によって多少異なる場合があります（例: macOSでは `Option+Delete` と `Cmd+Backspace` がいずれも単語削除として動作）。
- 本機能はClaude Code v2.1.218で追加されました。
