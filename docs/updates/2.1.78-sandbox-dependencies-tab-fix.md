## 原文（日本語に翻訳）

`/sandbox` の「Dependencies」タブがmacOSでmacOS固有の情報の代わりにLinuxの前提条件を表示していた問題を修正しました。

## 原文（英語）

Fixed `/sandbox` Dependencies tab showing Linux prerequisites on macOS instead of macOS-specific info.

## 概要

`/sandbox` コマンドのDependenciesタブを開くと、macOS環境にもかかわらずLinux向けの依存関係情報（apt-getなどのパッケージ管理コマンド）が表示されるUIバグが修正されました。修正後はmacOS環境ではHomebrew等のmacOS固有の情報が正しく表示されます。

## 基本的な使い方

`/sandbox` コマンドでサンドボックスの設定を確認する際、Dependenciesタブに正確な環境別情報が表示されます。

```
/sandbox
```

コマンド実行後、Dependenciesタブを開くと、現在のOSに合わせた依存関係のインストール方法が表示されます。

## 実践例

### ユースケース: macOS環境でのサンドボックス依存関係確認

修正前（誤った表示）：
```
# Linuxの依存関係（macOSに表示されていた）
sudo apt-get install -y bubblewrap
```

修正後（正しい表示）：
```
# macOSの依存関係
brew install ...
```

### ユースケース: サンドボックスのセットアップ支援

新しいMacにClaude Codeをセットアップする際、`/sandbox` のDependenciesタブが正しいHomebrew コマンドを提示するため、スムーズにサンドボックスを有効化できます。

```bash
# /sandbox の Dependenciesタブに表示される正しいmacOS用コマンドに従ってインストール
brew install ...

# インストール後にサンドボックスを有効化
# settings.json で
# "sandbox": { "enabled": true }
```

## 注意点

- この修正はUIの表示に関するものです。サンドボックスの動作自体には影響しません
- macOS環境でのサンドボックスの前提条件がより明確になりました
- Linuxでの`/sandbox`表示は以前から正しく動作していました
