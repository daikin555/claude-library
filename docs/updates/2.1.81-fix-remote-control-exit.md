# 修正：リモートコントロールの/exitがセッションを確実にアーカイブしない問題

## 原文（日本語に翻訳）

リモートコントロール `/exit` がセッションを確実にアーカイブしない問題を修正。

## 原文（英語）

Fixed Remote Control `/exit` not reliably archiving the session

## 概要

リモートコントロールセッションで `/exit` コマンドを実行した際、セッションのアーカイブが確実に行われないことがある問題が修正されました。以前は `/exit` 後もセッションがアクティブなまま残ったり、アーカイブの失敗によりセッション管理が煩雑になることがありました。修正後は `/exit` が確実にセッションをアーカイブします。

## 基本的な使い方

```bash
# リモートコントロールセッションを終了・アーカイブ
/exit

# 修正後は確実にアーカイブされる
```

## 注意点

- リモートコントロール以外の通常セッションの `/exit` は以前から正常に動作していた
- アーカイブされたセッションは Claude.ai のセッション履歴から参照できる
- 重要な作業は `/exit` 前に保存を確認することを推奨
