## 原文（日本語）
既にインストール済みのプラグインに対して、誤解を招く「Space to Toggle」ヒントが表示されていた問題を修正しました。

## 原文（英語）
Fixed plugin browse showing misleading "Space to Toggle" hint for already-installed plugins

## 概要
Claude Code v2.1.41では、プラグイン一覧画面(`/plugin browse`)で既にインストール済みのプラグインに対して、「Spaceキーでトグル」という誤ったヒントが表示されていた問題が修正されました。これにより、インストール済みプラグインと未インストールプラグインの操作が明確に区別されるようになりました。

## 問題の詳細

### 修正前の動作
インストール済みのプラグインにも「Space to Toggle」というヒントが表示されていましたが、実際にはSpaceキーを押しても何も起こりませんでした。

```text
# 修正前の表示例
Installed Plugins:
✓ document-skills        [Space to Toggle]  ← 誤解を招く表示
✓ python-toolkit         [Space to Toggle]  ← 既にインストール済みなのに

Available Plugins:
○ rust-tools             [Space to Install]
○ go-helper              [Space to Install]
```

### 修正後の動作
インストール済みプラグインには、適切な操作ヒントが表示されます。

```text
# 修正後の表示例
Installed Plugins:
✓ document-skills        [Installed]
✓ python-toolkit         [Installed]

Available Plugins:
○ rust-tools             [Space to Install]
○ go-helper              [Space to Install]
```

## 基本的な使い方

### プラグイン一覧の表示
```bash
claude
> /plugin browse
```

修正後は、インストール状態が明確に表示されます。

### プラグインのインストール
未インストールのプラグインに対してのみ、Spaceキーでインストールできます。

```bash
# プラグイン一覧を表示
> /plugin browse

# 未インストールプラグインを選択（矢印キー）
○ rust-tools             [Space to Install]

# Spaceキーでインストール
[Space]
✓ rust-tools             [Installed]
```

## 実践例

### 初めてのプラグインインストール
プラグインシステムを初めて使用する場合、明確な表示により混乱を避けられます。

```bash
> /plugin browse

# インストール済みプラグインと未インストールプラグインが明確
Installed Plugins:
✓ document-skills        [Installed]

Available Plugins:
○ terraform-engineer     [Space to Install]  ← Spaceで操作可能
○ sql-pro                [Space to Install]  ← Spaceで操作可能

# 未インストールプラグインを選択してSpace
```

### インストール済みプラグインの確認
既にインストールしたプラグインを確認する場合です。

```bash
> /plugin browse

Installed Plugins:
✓ document-skills        [Installed]  ← 操作ヒントが適切
✓ python-pro             [Installed]
✓ cloud-architect        [Installed]

# これらのプラグインはSpaceキー操作の対象外と明確
```

### プラグインの管理
複数のプラグインをインストール・管理する場合、状態が一目でわかります。

```bash
> /plugin browse

Installed Plugins (5):
✓ document-skills        [Installed]
✓ python-pro             [Installed]
✓ terraform-engineer     [Installed]
✓ sql-pro                [Installed]
✓ github-actions-expert  [Installed]

Available Plugins (3):
○ network-engineer       [Space to Install]
○ security-engineer      [Space to Install]
○ devops-engineer        [Space to Install]

# インストール済み/未インストールが明確に区別される
```

### プラグインアンインストール時の操作
アンインストールは別のコマンドで行う必要があることが明確になります。

```bash
# インストール済みプラグインはSpace操作不可
> /plugin browse
✓ document-skills        [Installed]

# アンインストールは専用コマンドを使用
> /plugin uninstall document-skills
```

## この修正の利点

### ユーザー体験の向上
- 操作可能な項目と不可能な項目が明確
- 誤った期待を持たせない
- 初心者にも分かりやすいUI

### 混乱の防止
- 「Spaceキーを押したのに何も起こらない」という問題の解消
- インストール状態が一目で分かる
- 適切な操作方法への誘導

### プラグイン管理の効率化
- 現在の状態を素早く把握
- 必要なアクションが明確
- ワークフローの改善

## プラグイン管理の関連コマンド

### プラグイン一覧表示
```bash
> /plugin browse
```

### プラグインインストール
```bash
# UIから（修正後は未インストールプラグインのみSpace操作可能）
> /plugin browse
[Space on uninstalled plugin]

# コマンドラインから
> /plugin install <plugin-name>
```

### プラグインアンインストール
```bash
> /plugin uninstall <plugin-name>
```

### インストール済みプラグイン確認
```bash
> /plugin list
```

## 注意点
- この修正は表示とユーザーインターフェースの問題であり、プラグインの機能自体には影響しません
- インストール済みプラグインをアンインストールする場合は、`/plugin uninstall`コマンドを使用してください
- プラグインのインストール状態は`~/.claude/plugins/`ディレクトリで管理されています
