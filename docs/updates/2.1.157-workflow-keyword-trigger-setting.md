## 原文（日本語に翻訳）

プロンプト内の「workflow」という単語がダイナミックワークフローをトリガーしないようにするための「ワークフローキーワードトリガー」設定が `/config` に追加されました。

## 原文（英語）

Added a "Workflow keyword trigger" setting in /config to stop the word "workflow" in a prompt from triggering a dynamic workflow

## 概要

`/config` コマンドに新しい「ワークフローキーワードトリガー」設定が追加されました。この設定を無効にすることで、プロンプト内に「workflow」という単語が含まれていても、ダイナミックワークフローが自動的にトリガーされないようにできます。ワークフロー機能を使わずに「workflow」という単語を含む会話を行いたい場合に役立ちます。

## 基本的な使い方

設定を変更するには `/config` コマンドを実行し、「ワークフローキーワードトリガー」設定を見つけてオン/オフを切り替えます。

```
/config
```

設定項目「Workflow keyword trigger」を無効にすると、プロンプトに "workflow" という単語を含めてもダイナミックワークフローが起動しなくなります。

## 注意点

- この設定はダイナミックワークフローのキーワードトリガーのみを制御します
- ワークフロー機能自体が無効になるわけではありません（他の方法でワークフローを起動することは可能です）
- デフォルトでは従来通り「workflow」キーワードでトリガーが有効です
- v2.1.157 以降で利用可能です
