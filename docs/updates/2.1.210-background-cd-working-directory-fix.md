## 原文（日本語に翻訳）

コマンドがバックグラウンドに移動された後、Claudeが`cd`が反映されたと誤って想定してしまう問題を修正しました。ツールの結果には、作業ディレクトリが変更されていないことが明示されるようになりました

## 原文（英語）

Fixed Claude assuming a `cd` took effect after its command was moved to the background; the tool result now states the working directory is unchanged

## 概要

Claude Code v2.1.210で修正された、バックグラウンドタスク機能に関するバグです。`cd`を含むコマンドを実行中に、そのコマンドが（Ctrl+Bや`run_in_background`によって）バックグラウンドに移動された場合、以前はClaudeが「`cd`が実行され作業ディレクトリが変わった」と誤って前提を置いてしまい、以降の操作が想定と異なるディレクトリで行われる可能性がありました。今回の修正により、バックグラウンドに移動されたコマンドの結果には「作業ディレクトリは変更されていない」ことが明示され、Claudeが誤った前提を持たないようになりました。

## 基本的な使い方

### 修正前の問題

```bash
# Claudeが実行するコマンド例
cd my-project && npm run dev
```

```text
1. Claudeが上記コマンドを実行
2. 長時間実行されるため、ユーザーがCtrl+Bでバックグラウンドに移動
   （またはClaude自身がrun_in_background: trueで開始）
3. 修正前: Claudeは「cd my-project が実行され、
   カレントディレクトリが my-project に変わった」と誤って想定
4. 以降のツール呼び出し（例: 相対パスでのファイル読み書き）が
   意図しないディレクトリを基準に実行されてしまう
```

### 修正後の動作

```bash
cd my-project && npm run dev
```

```text
1. Claudeがコマンドを実行
2. バックグラウンドに移動される
3. 修正後: ツール結果に「作業ディレクトリは変更されていません」
   という情報が明示される
4. Claudeはセッションの作業ディレクトリが元のままであると
   正しく認識し、以降の操作を正しいパスで実行する
```

## 実践例

### 開発サーバーの起動をバックグラウンド化する場合

```bash
claude

> cd frontend && npm run dev をバックグラウンドで実行して

# 修正前: cd frontend が反映されたとClaudeが誤解し、
#         その後の `package.json を編集して` のような指示が
#         frontend/package.json ではなくルートのpackage.jsonを
#         対象にしてしまう、といった不整合が起きうる
# 修正後: 「作業ディレクトリは変更されていません」と結果に明示され、
#         Claudeはルートディレクトリ基準で以降の操作を続ける
```

### Ctrl+Bで手動バックグラウンド化するケース

```bash
claude

> cd services/api && ./run-migrations.sh --watch

# ユーザーが Ctrl+B を押してバックグラウンドに移動
# 修正後:
# ツール結果: "コマンドはバックグラウンドで実行中です。
#              作業ディレクトリは変更されていません（元のまま）。"
```

### 複数のバックグラウンドタスクを管理する場合

`/tasks`でバックグラウンドタスクの一覧を確認しつつ、作業ディレクトリの認識がずれないかを確認する例です。

```bash
claude

> cd logs && tail -f app.log をバックグラウンドで開始して
> 現在の作業ディレクトリで ls して

# 修正後: 2つ目の指示は cd logs 前の元のディレクトリを対象に
#         実行される（バックグラウンド化前提のcdは反映されないため）
```

## 注意点

- この修正は Claude Code v2.1.210 で実装されました
- `cd && <長時間実行コマンド>` のような形で作業ディレクトリを変更しつつ長時間コマンドを実行する場合、そのコマンドがバックグラウンドに回った時点で `cd` の効果はセッションの認識上「反映されていない」ものとして扱われます
- 本当に作業ディレクトリを変更したい場合は、バックグラウンド実行とは別に、フォアグラウンドで完結する `cd` コマンドを単独で実行するか、以降のコマンドで都度絶対パス・相対パスを明示するのが確実です
- バックグラウンドタスクの一覧・停止は `/tasks` コマンドで確認・管理できます
- `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1` を設定するとバックグラウンドタスク機能自体を無効化できます
