## 原文（日本語に翻訳）

未認証の組織登録claude.aiコネクターのスタートアップ通知ノイズを削除しました。

## 原文（英語）

Removed startup notification noise for unauthenticated org-registered claude.ai connectors

## 概要

組織（org）によって登録されているclaude.aiコネクターのうち、ユーザーがまだ認証していないものに関する不要なスタートアップ通知が削除されました。これらの通知はユーザーにとって有益な情報ではなく、単なるノイズとなっていたため、v2.1.71では表示されなくなります。必要な通知（認証済みコネクターの状態など）は引き続き表示されます。

## 基本的な使い方

この改善はv2.1.71以降で自動的に適用されます。Claude Codeの起動時の通知がより整理された状態になります。

```bash
# Claude Codeを起動
claude

# 修正前: 組織で設定されているが未認証のコネクターごとに通知が表示される
# 修正後: 未認証のコネクターに関する不要な通知が表示されない
# → よりクリーンな起動体験
```

## 実践例

### 大規模な組織環境での起動体験改善

多数の組織設定コネクターがある大企業環境での利用。

```bash
# 50個の組織コネクターが設定されているが、そのうち48個は未認証の状態
claude

# 修正前: 起動時に未認証コネクターに関する48個の通知が表示される
# 修正後: 不要な通知が表示されず、スムーズに起動
```

### 新規参加メンバーの初回起動

組織に新しく参加したメンバーがClaude Codeを初めて起動する場合。

```bash
# 新メンバーが初めてClaude Codeを起動
# 組織の全コネクターに対してまだ認証していない状態

claude

# 修正前: 全ての未認証コネクターの通知が溢れる
# 修正後: クリーンに起動し、必要に応じて個別に認証できる
```

### 複数プロジェクトでの日常的な起動

毎日複数回Claude Codeを起動する開発者の利用。

```bash
# 朝の作業開始時
claude

# 修正後: 必要な情報のみが表示され、
# 認証済みコネクターの状態は引き続き通知される
# 未認証コネクターのノイズは表示されない
```

## 注意点

- この改善はv2.1.71以降で自動的に適用されます。
- 認証済みのコネクターに関する通知は引き続き表示されます。
- コネクターを認証したい場合は、個別に認証操作を行ってください。
- 関連する修正として、MCPコネクター通知の修正（`2.1.71-mcp-connector-notification-fix.md`）もv2.1.71で対応されています。
- 通知の設定は組織管理者が制御する場合があります。
