## 原文（日本語訳）

`CLAUDE_CODE_MAX_OUTPUT_TOKENS`など一部の環境変数が、科学的記数法の値（例：`1e6`）を指定した際にサイレントに仮数部のみを使ってしまう不具合を修正した（`1e6`が`1`として扱われていた）。

## 原文（英語）

Fixed `CLAUDE_CODE_MAX_OUTPUT_TOKENS` and similar env vars silently using the mantissa of scientific-notation values (`1e6` became `1`)

## 概要

Claude Codeでは`CLAUDE_CODE_MAX_OUTPUT_TOKENS`のように、数値で挙動を制御する環境変数が数多く用意されています。これまでは、こうした数値環境変数に`1e6`のような科学的記数法（指数表記）で値を指定すると、パース処理が指数部を無視して仮数部（この例では`1`）だけを数値として採用してしまい、エラーも警告も出ないままユーザーの意図とまったく異なる小さな値が設定されてしまう不具合がありました。今回のアップデートでこのパース処理が修正され、科学的記数法の値も正しく解釈されるようになっています。

## 基本的な使い方

環境変数には引き続き通常の整数表記を使うのが最も確実です。

```bash
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=64000
claude
```

## 実践例

### 大規模リファクタリング時に出力上限を引き上げる

大きな出力を許可するために科学的記数法で値を指定しても、以前は意図せず`1`（トークン）として扱われてしまい、実質的に出力がほぼ切り詰められる状態になっていました。修正後は指定した値どおりに解釈されます。

```bash
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=64000  # 通常はこのように整数で指定するのが安全
claude -p "Refactor the payment module across all files"
```

### スクリプトやCI設定で数値をプログラム生成する場合

言語やツールによっては大きな数値がデフォルトで科学的記数法の文字列（例: `6e4`）として出力されることがあります。そうした値をそのまま環境変数に渡しても、修正後は仮数部だけが誤って採用されることなく正しくパースされます。

```bash
TOKENS=$(python3 -c "print(f'{64000:.0e}')")  # => "6e+04" のような文字列になり得る例
export CLAUDE_CODE_MAX_OUTPUT_TOKENS="$TOKENS"
claude -p "Run the full test suite and summarize failures"
```

## 注意点

- 意図しない小さな値が採用されてもエラーにはならず「サイレントに」誤動作するため、以前のバージョンでは気づきにくい不具合でした。数値を指定したのに出力が異常に短い場合は、CLIのバージョンを確認することをおすすめします。
- 可読性や意図の明確さの観点からは、環境変数には科学的記数法よりも通常の整数表記（例: `64000`）を使う方が無難です。
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS`は、拡張思考（extended thinking）の予算が出力上限を超えてエラーになる場合に、`MAX_THINKING_TOKENS`を下げる代わりに引き上げる用途でも使われます。
