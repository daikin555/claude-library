# ホスト管理デプロイメントでBedrock/Vertex/Foundry利用時の分析無効化を修正

## 原文（日本語に翻訳）

ホスト管理デプロイメント（`CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST`）がBedrock/Vertex/Foundryで分析を自動無効化しなくなりました。

## 原文（英語）

Host-managed deployments (`CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST`) no longer auto-disable analytics on Bedrock/Vertex/Foundry

## 概要

`CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST` を設定したホスト管理デプロイメントでAWS Bedrock、GCP Vertex AI、またはAzure Foundryを使用している場合、これまでAnthropicへの分析データ送信が自動的に無効化されていました。今回の変更により、この自動無効化が廃止されました。エンタープライズ環境でホスト管理を行っている管理者は、分析設定を明示的に制御できるようになります。

## 基本的な使い方

```bash
# ホスト管理デプロイメントの設定例
export CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST=true

# Bedrockを使用
export ANTHROPIC_BASE_URL=https://bedrock.us-east-1.amazonaws.com/anthropic

# 分析を明示的に無効化したい場合
# settings.jsonで設定
```

```json
{
  "analytics": {
    "enabled": false
  }
}
```

## 実践例

### エンタープライズ設定での分析制御

```bash
# マネージド設定ファイル（/etc/claude-code/managed-settings.json）
# システム管理者が制御する設定
{
  "CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST": true,
  "analytics": {
    "enabled": true  // 明示的に有効/無効を設定
  }
}
```

### Bedrock環境での設定

```bash
# AWS Bedrock + ホスト管理デプロイメント
export CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST=true
export AWS_REGION=us-east-1

# 以前は分析が自動的に無効化されていたが、
# v2.1.126以降は明示的な設定が必要
claude
```

## 注意点

- この変更により、以前は自動的に無効化されていた分析が、更新後に有効になる可能性があります
- 分析を無効化したい場合は、明示的に設定を変更してください
- プライバシーポリシーや組織のデータガバナンス要件を確認してから設定を決定してください
- `CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST` を使用している場合、ホスト側の管理設定を優先させることができます
