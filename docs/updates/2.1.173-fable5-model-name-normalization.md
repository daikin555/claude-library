## 原文（日本語に翻訳）

Fable 5のモデル名に`[1m]`サフィックスが付いている場合に正規化されなかった問題を修正しました。Fable 5はデフォルトで100万トークンのコンテキストを含むため、サフィックスは自動的に除去されるようになりました。

## 原文（英語）

Fixed Fable 5 model names with a `[1m]` suffix not being normalized — Fable 5 includes 1M context by default, so the suffix is now stripped automatically.

## 概要

Claude Fable 5モデルは、デフォルトで100万トークン（1M）のコンテキストウィンドウを持ちます。以前は、モデル名に`[1m]`サフィックスを付けて指定した場合（例：`claude-fable-5[1m]`）、モデル名が正しく正規化されずにエラーや予期しない動作が発生することがありました。この修正により、`[1m]`サフィックスは自動的に除去され、正しいモデル名として扱われるようになりました。

## 基本的な使い方

### 修正前の問題のある指定方法

```bash
# [1m]サフィックスを付けると正規化されなかった（修正前）
claude --model claude-fable-5[1m] "こんにちは"
```

### 修正後の動作

```bash
# サフィックス付きでもサフィックスなしでも同じように動作する（修正後）
claude --model claude-fable-5[1m] "こんにちは"  # [1m]は自動除去される
claude --model claude-fable-5 "こんにちは"       # こちらと同等

# settings.jsonでの指定
# "model": "claude-fable-5[1m]" も "claude-fable-5" として正しく処理される
```

## 実践例

### APIキーを使用した設定ファイルでの指定

```json
// .claude/settings.json
{
  "model": "claude-fable-5"
}
```

`[1m]`サフィックスを付けて設定していた場合も、自動的に正規化されます：

```json
// この設定も正しく動作するようになった（修正後）
{
  "model": "claude-fable-5[1m]"
}
```

### プログラムからのモデル指定

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

// [1m]サフィックスが自動除去されて正しく処理される
const message = await client.messages.create({
  model: "claude-fable-5", // 推奨：サフィックスなし
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello!" }],
});
```

### 長いコンテキストを活用した使い方

Fable 5は1Mトークンのコンテキストをデフォルトで持つため、大規模なドキュメントの処理も可能です：

```bash
# 大きなコードベース全体を分析
find ./src -name "*.ts" | xargs cat | claude --model claude-fable-5 \
  "このコードベースのアーキテクチャを分析して、改善点を提案してください"
```

## 注意点

- **Fable 5のデフォルトコンテキスト**: Fable 5は追加設定なしで1Mトークンのコンテキストを持ちます。`[1m]`サフィックスは冗長であり、今後は不要です
- **後方互換性**: 既存の設定やスクリプトで`[1m]`サフィックスを使用している場合も、自動除去により引き続き正しく動作します
- **モデルIDの統一**: 今後は`claude-fable-5`をモデルIDとして使用することを推奨します
- **他のモデルへの影響**: この修正はFable 5のみに適用されます。他のモデルには影響しません
