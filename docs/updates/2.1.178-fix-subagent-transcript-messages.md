## 原文（日本語に翻訳）

複数のサブエージェント問題を修正しました: サブエージェントのトランスクリプト表示にツール結果とライブ進捗が表示されるようになりました。サブエージェントのターン終了中に送信されたメッセージが失われなくなりました。また実行中のサブエージェントをバックグラウンド化（Ctrl+B）しても最初からやり直しにならなくなりました。

## 原文（英語）

Fixed several subagent issues: viewing a subagent's transcript now shows tool results and live progress, messages sent while it finishes its turn are no longer dropped, and backgrounding a running subagent (ctrl+b) no longer restarts it from scratch

## 概要

サブエージェント（`claude agents` で管理されるエージェント）に関する複数の問題が一度に修正されました。トランスクリプト表示の改善、メッセージ消失の修正、Ctrl+Bでのバックグラウンド化の安全性向上の3点が含まれます。これによりマルチエージェントワークフローの信頼性と使いやすさが大幅に向上します。

## 基本的な使い方

サブエージェントの管理は `claude agents` コマンドまたはセッション内のエージェントパネルで行います:

```bash
# エージェント一覧の確認
claude agents

# エージェントのトランスクリプト表示（改善済み）
# → ツール結果とライブ進捗も表示されるようになった
```

## 実践例

### トランスクリプト表示の改善

以前の表示（問題あり）:
```
[エージェント transcript]
User: ファイルを検索してください
Assistant: ファイルを検索します...
  （ツール呼び出し結果が表示されなかった）
  見つかりました。
```

改善後の表示:
```
[エージェント transcript]
User: ファイルを検索してください
Assistant: ファイルを検索します...
  🔧 Glob("**/*.ts") → 42 files found
  🔧 Read("src/main.ts") → 150 lines
  📊 処理中... (3/42 files)
  見つかりました。42件のTypeScriptファイルが存在します。
```

### メッセージ消失の修正

以前の動作（問題あり）:
```bash
# サブエージェントが処理中（ターン終了直前）に追加メッセージを送信
→ メッセージが消失（ドロップ）していた
```

改善後の動作:
```bash
# サブエージェントのターン終了直前にメッセージを送信しても
→ 安全にキューに入り、次のターンで処理される
```

### Ctrl+Bでのバックグラウンド化

以前の動作（問題あり）:
```
1. サブエージェントが長時間タスクを実行中
2. Ctrl+B でバックグラウンドへ移動
3. 後で確認すると → エージェントが最初からやり直しになっていた
```

改善後の動作:
```
1. サブエージェントが長時間タスクを実行中
2. Ctrl+B でバックグラウンドへ移動
3. 他の作業を行う
4. claude agents でエージェントを確認
5. → エージェントは中断された場所から継続している
```

## 注意点

- これらの修正はすべてサブエージェント機能（エージェントSDK、Agentツール）を使用する際に影響します
- Ctrl+Bのバックグラウンド化は「一時停止」ではなく「バックグラウンドへの移動」です。エージェントは引き続き実行されます
- `claude agents` で実行中のエージェント一覧を確認し、Ctrl+Bでフォアグラウンドに戻せます
- 長時間のマルチエージェントワークフローの信頼性が向上します
