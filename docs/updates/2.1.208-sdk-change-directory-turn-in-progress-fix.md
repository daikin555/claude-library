## 原文（日本語訳）

SDKホスト（Claude Desktopなど）において、バックグラウンドタスクが実行中のアイドル状態のセッションで、「ディレクトリ変更（Change directory）」が「A turn is in progress」というエラーで失敗する不具合を修正した。

## 原文（英語）

Fixed "Change directory" in SDK hosts (e.g. Claude Desktop) failing with "A turn is in progress" on idle sessions that have a running background task

## 概要

Claude Desktopのように、Agent SDK上に構築されたアプリケーション（SDKホスト）では、セッションの作業ディレクトリを別のフォルダに切り替える「ディレクトリ変更」機能が提供されています。これまで、ユーザーからの応答待ちで実際にはアイドル状態のセッションであっても、バックグラウンドタスク（例: バックグラウンドで実行中のBashコマンドなど）が動いていると、内部の「ターン」の状態管理が誤って「ターン実行中」と判定してしまい、ディレクトリ変更のリクエストが「A turn is in progress」というエラーで拒否されていました。今回の修正で、この誤判定が解消され、バックグラウンドタスクを抱えたアイドルセッションでも正常にディレクトリを変更できるようになりました。

## 基本的な使い方

Claude Desktopなど、Agent SDKをベースとしたホストアプリケーションでセッションを開いている状態で、ディレクトリ変更UI（またはSDK経由のディレクトリ変更リクエスト）を使って作業フォルダを切り替えます。

```text
# Claude Desktopのセッション設定画面などから
# 「Change directory」を選択し、新しい作業フォルダを指定する
```

修正前は、セッションがバックグラウンドタスクを抱えている状態でこの操作を行うと「A turn is in progress」エラーになり切り替えができませんでしたが、v2.1.208以降では問題なく切り替えられます。

## 実践例

### Claude Desktopでプロジェクトを切り替える

開発サーバーの起動など、バックグラウンドタスクを走らせたままのセッションで、別プロジェクトに作業ディレクトリを切り替えたい場合に有効です。

```text
1. Claude Desktopでセッションを開き、バックグラウンドでdev serverなどを実行させたままにする
2. セッションはアイドル状態（あなたの入力待ち）
3. 「Change directory」で別のプロジェクトフォルダへ切り替える
```

修正前はここで失敗していましたが、修正後はバックグラウンドタスクの有無に関わらず切り替えが成功します。

### 独自に構築したSDKホストアプリケーションでの利用

Agent SDKを使って自社ツールやカスタムクライアントを構築している場合、同様に「アイドルだがバックグラウンドタスクを持つセッション」に対してディレクトリ変更のコントロールリクエストを送るケースがあります。

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

// バックグラウンドタスクを持つセッションに対して
// ディレクトリ変更のコントロールリクエストを送信する場合も、
// v2.1.208以降は "A turn is in progress" エラーが発生しなくなる
```

## 注意点

- この修正はSDKホスト（Claude DesktopなどAgent SDK経由のクライアント）に関するものであり、CLIの `/cd` コマンド自体の挙動とは別の問題です。
- 問題が発生する条件は「セッションがアイドルだが、バックグラウンドタスクが実行中」という特定の状態に限られていました。
- 影響を受けていた場合は、Claude Codeおよび関連するSDKを最新版（v2.1.208以降）に更新することで解消します。
