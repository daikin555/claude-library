## 原文（日本語訳）

LSPプラグインに対する誤った「未使用プラグイン」の通知と、偏った未使用テレメトリを修正した

## 原文（英語）

Fixed false "disused plugin" tips and skewed disuse telemetry for LSP plugins

## 概要

Claude CodeにはインストールされたプラグインがしばらくEllipsisの間使われていない場合に「このプラグインは未使用です」といったヒントを表示する仕組みがありますが、LSP（Language Server Protocol）サーバーを提供するタイプのプラグインについては、実際には裏側で診断情報の提供やコードジャンプ（定義へ移動など）への応答といった形でアクティブに動作しているにもかかわらず、その利用が正しく検知されず「未使用」と誤判定されるケースがありました。今回の修正により、LSPプラグインの実際の利用状況が正しく検知され、誤ったヒント表示や、社内で収集される未使用状況テレメトリの偏りが解消されました。

## 基本的な使い方

この修正はClaude Code内部の検知ロジックに関するものであり、ユーザー側で特別な操作は不要です。LSP機能を提供するプラグイン（例: 各種言語サーバーを組み込むプラグイン）をインストールして通常通り利用するだけで、修正後の挙動が適用されます。

```text
/plugin
```

上記のようにプラグイン管理コマンドでインストール済みプラグインの一覧・状態を確認できます。

## 実践例

### 言語サーバー系プラグインを常用している場合

TypeScriptやPython向けのLSPプラグインを導入していて、エディタ的な補完・定義ジャンプ・診断表示のためにバックグラウンドで動作させているようなケースです。以前はコマンドとして明示的に呼び出す操作が少ないと「未使用プラグイン」として警告が出ることがありましたが、修正後は診断やナビゲーション応答も「利用」としてカウントされるため、実際に活用しているプラグインが誤って未使用扱いされることがなくなります。

### 複数プラグインを整理する場合

多数のプラグインを導入している環境で、本当に使われていないプラグインだけを整理したい場合に有用です。今回の修正でLSPプラグインの利用状況テレメトリの精度が上がったため、Anthropic側の統計や、将来的にClaude Codeが提示するかもしれない「本当に未使用のプラグイン」の判定がより信頼できるものになります。

## 注意点

- この修正はLSPプラグインの利用検知ロジックの内部改善であり、ユーザーが手動で設定を変更する必要はありません。
- 非LSP型のプラグイン（スラッシュコマンドやフック中心のプラグインなど）の未使用判定ロジックには直接関係しません。
