## 原文（日本語に翻訳）

Unicode サポートのないターミナルでのタスクリストにおけるバックグラウンドエージェント行間の空行表示を修正しました。

## 原文（英語）

Fixed blank lines appearing between background agent rows in the task list on terminals without Unicode support

## 概要

Unicode をサポートしていない旧来のターミナルエミュレーターで `claude agents` のタスクリストを表示した際に、各エージェント行の間に余分な空行が表示されていたバグを修正しました。修正後は Unicode 非対応ターミナルでも一覧が正しくコンパクトに表示されます。

## 基本的な使い方

この修正はバグフィックスであり、v2.1.166 以降へのアップデートで自動的に適用されます。

```bash
claude update
claude agents  # Unicode非サポートターミナルでも正常表示
```

## 実践例

### 影響を受けていたターミナルの例

Unicode をサポートしていない、または制限のあるターミナル：
- Windows の cmd.exe（レガシー設定）
- 一部の SSH クライアント（文字コード設定によっては）
- 企業環境の旧式ターミナルエミュレーター

### 修正前後の表示比較

**修正前（Unicode 非サポートターミナル）：**
```
Agent 1: Running...

Agent 2: Waiting...

Agent 3: Completed
```

**修正後：**
```
Agent 1: Running...
Agent 2: Waiting...
Agent 3: Completed
```

## 注意点

- Unicode をサポートする現代的なターミナル（iTerm2、Windows Terminal など）では問題は発生していませんでした
- この修正は表示の問題であり、機能への影響はありませんでした
