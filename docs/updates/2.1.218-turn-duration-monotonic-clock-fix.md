## 原文（日本語に翻訳）

システムクロックが調整された後にターンの所要時間がまれに負の値や不正確な値になる問題を、モノトニッククロックでターンの計測を行うことで修正しました

## 原文（英語）

Fixed rare negative or incorrect turn duration measurements after a system clock adjustment by timing turns with a monotonic clock

## 概要

ターン（1回の応答）の所要時間はこれまで壁時計時刻（wall-clock time）で計測されていたため、NTP同期やサマータイム切り替え、手動での時刻変更、VMのサスペンド/レジュームなどでターンの途中にシステム時刻が調整されると、負の値や意味をなさない所要時間が計測されることがありました。この修正により、システム時刻の変化の影響を受けないモノトニッククロックで計測するようになり、常に正確な所要時間が記録されます。

## 注意点

- Claude Code v2.1.218で修正されました
- 影響を受けるのは主にNTP同期やクロック変更が発生しやすい環境（クラウドVM、サスペンド/レジュームを行う端末など）です
- 計測値を利用するテレメトリやOpenTelemetry連携の精度が向上します
