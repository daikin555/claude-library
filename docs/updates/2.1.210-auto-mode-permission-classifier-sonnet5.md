## 原文（日本語に翻訳）

auto modeを改善しました。外部セッションにおいて、権限分類器がデフォルトでSonnet 5を使用するようになり、セッションの最初のリクエストで検証された上でそのセッション中は固定されるようになりました。

## 原文（英語）

Improved auto mode: the permission classifier now defaults to Sonnet 5 for external sessions, validated on the session's first request and pinned for the session

## 概要

auto modeは、人間による許可/拒否のプロンプトの代わりに、ツール呼び出しごとにML分類器（別インスタンスのClaude）がリスクを評価する仕組みです。分類器はスコープ逸脱・信頼できないインフラへのアクセス・プロンプトインジェクションの疑いなどを評価し、危険と判断した場合はブロックします。v2.1.210では、SDKなど外部から起動されるセッション（external session）における分類器のデフォルトモデルがSonnet 5になりました。さらに、セッション開始時の最初のリクエストでそのモデルが利用可能かどうかを検証し、以降はセッション中一貫して同じモデルに固定（pin）されるようになり、セッション途中でモデルの挙動が変わってしまうことによる分類のブレを防いでいます。

## 基本的な使い方

### auto modeの有効化

```bash
# CLIでauto modeを有効にして起動
claude --permission-mode auto

# または対話中に切り替え
claude
> /permission-mode auto
```

### 外部セッションでの挙動（SDK経由）

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

const session = query({
  prompt: "リポジトリのlintエラーを修正して",
  options: {
    permissionMode: "auto",
  },
});

// セッションの最初のリクエストで、
// 権限分類器に使うモデル（デフォルト: Sonnet 5）が
// 利用可能かどうか検証される
// → 検証OKであれば、そのセッションを通じて
//   Sonnet 5が分類器モデルとして固定される
```

## 実践例

### CI/CDパイプラインでのヘッドレス実行

```bash
# CI環境でauto modeを使い、人手の介在なしにタスクを実行するケース
claude -p "テストが通るまで修正を繰り返して" --permission-mode auto

# 外部セッションとして起動されるため、
# 権限分類器はデフォルトでSonnet 5が使われ、
# パイプライン実行中を通じて一貫したモデルで
# リスク評価が行われる
```

### 長時間稼働するSDKセッションでの一貫性

```typescript
// 数時間にわたる長時間セッションでも、
// 分類器モデルはセッション開始時に検証された
// Sonnet 5に固定されたまま変わらない
const session = query({
  prompt: "大規模リファクタリングを段階的に進めて",
  options: { permissionMode: "auto" },
});

// → セッション途中でモデルの可用性が変動しても、
//   最初に検証・固定されたモデルが使われ続けるため、
//   分類基準がセッション内で一貫する
```

### 分類器がリスクを検知しブロックするケース

```bash
claude --permission-mode auto

> リポジトリ内のすべてのファイルを削除して、リモートに強制pushして

# 権限分類器（Sonnet 5）がスコープ逸脱・破壊的操作と判断し、
# このアクションをブロック
# → 3回連続、または合計20回ブロックされると
#   auto modeは一時停止し、通常の確認プロンプトに戻る
```

## 注意点

- この改善はClaude Code v2.1.210で実装されました
- 「外部セッション（external session）」とは、Claude Agent SDKなどを通じて外部から起動されるセッションを指し、通常の対話的なCLIセッションと区別されます
- 分類器モデルはセッションの最初のリクエスト時に一度だけ検証され、以降はそのセッション中固定されます。そのため、セッション途中でモデルの利用可否状況が変化しても、分類器のモデルが動的に切り替わることはありません
- auto modeの分類器は「スコープ逸脱」「信頼できないインフラへのアクセス」「プロンプトインジェクションの疑い」の3つのリスクカテゴリを評価し、2段階（高速なyes/noフィルタ→必要な場合のみ詳細な推論）で判定します
- 分類器が3回連続、または合計20回ブロックすると、auto modeは自動的に一時停止し、通常の許可プロンプトに戻ります。プロンプトを承認するとauto modeが再開します
- 分類器はセッション開始時に設定された作業ディレクトリとリモートを信頼済みとして扱い、セッション中に `git remote add` や `git remote set-url` で追加・変更されたリモートは信頼されません
