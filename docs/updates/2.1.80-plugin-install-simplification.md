# プラグインのインストールヒントを単一の/plugin installコマンドに簡略化

## 原文（日本語に翻訳）

プラグインのインストールヒントを、2ステップのフローではなく単一の `/plugin install` コマンドを使用するように簡略化

## 原文（英語）

Simplified plugin install tips to use a single `/plugin install` command instead of a two-step flow

## 概要

Claude Code v2.1.80から、プラグインのインストールヒントが改善されました。以前は2ステップのプロセスが必要でしたが、単一の `/plugin install` コマンドで完結するようになりました。プラグインの推薦メッセージがよりシンプルで分かりやすくなり、即座にインストールを開始できます。

## 基本的な使い方

```
/plugin install <plugin-name>
```

プラグインの推薦ヒントが表示された場合、表示されたコマンドをそのままコピー＆ペーストして実行できます。

## 実践例

### 推薦ヒントからのインストール

以前（2ステップのフロー）：
```
💡 terraform-assistantプラグインが利用可能です
1. /plugin marketplace を開く
2. terraform-assistantを検索してインストール
```

現在（単一コマンド）：
```
💡 terraform-assistantプラグインが利用可能です
インストール: /plugin install terraform-assistant
```

### よく使うプラグインのインストール例

```bash
# Gitワークフロープラグイン
/plugin install git-workflow

# テストランナープラグイン
/plugin install test-runner

# コードレビュープラグイン
/plugin install code-reviewer
```

### インストール済みプラグインの管理

```bash
# インストール済みプラグイン一覧
/plugin list

# プラグインを無効化
/plugin disable plugin-name

# プラグインをアンインストール
/plugin uninstall plugin-name
```

## 注意点

- `/plugin install` コマンドは必要なすべての設定を自動的に行います
- インストール後、プラグインのスキルやコマンドは即座に利用可能になります
- プロジェクト固有のプラグインを追加する場合は、プロジェクトディレクトリ内で実行してください
- settings.jsonの `enabledPlugins` も自動的に更新されます
