## 原文（日本語に翻訳）

- プラグイン依存関係の強制管理を追加：`claude plugin disable` は、別の有効なプラグインが対象に依存している場合に拒否するようになり（コピー可能な無効化チェーンのヒント付き）、`claude plugin enable` はトランジティブな依存関係を強制的に有効化するようになりました

## 原文（英語）

- Added plugin dependency enforcement: `claude plugin disable` now refuses when another enabled plugin depends on the target (with a copy-pasteable disable-chain hint), and `claude plugin enable` force-enables transitive dependencies

## 概要

Claude Codeのプラグイン管理において、依存関係の整合性を保つ仕組みが追加されました。`claude plugin disable` コマンドは、他の有効なプラグインが依存しているプラグインを無効化しようとした場合に警告を出し、操作を拒否します。逆に `claude plugin enable` は、必要な依存プラグインを自動的に有効化します。

## 基本的な使い方

```bash
# プラグインを無効化する（依存関係がある場合は拒否される）
claude plugin disable my-plugin

# 依存関係があって拒否された場合の出力例：
# Error: 'plugin-b' depends on 'my-plugin'. To disable both, run:
#   claude plugin disable plugin-b && claude plugin disable my-plugin

# プラグインを有効化する（依存プラグインも自動で有効化）
claude plugin enable my-plugin
# → my-plugin が依存するプラグインが未有効の場合、自動的に有効化される
```

## 実践例

### 依存チェーンを持つプラグインを無効化する

```bash
# 例：deployment-tools が code-formatter に依存している場合
claude plugin disable code-formatter
# Error: 'deployment-tools' depends on 'code-formatter'.
# To disable the dependency chain, run:
#   claude plugin disable deployment-tools && claude plugin disable code-formatter

# ヒントをそのままコピー・ペーストして実行
claude plugin disable deployment-tools && claude plugin disable code-formatter
```

### 依存関係ごとプラグインを有効化する

```bash
# advanced-deploy が base-tools と code-formatter に依存している場合
claude plugin enable advanced-deploy
# → base-tools と code-formatter も自動で有効化される
```

### プラグインの依存関係を確認する

```bash
# 現在有効なプラグインと依存関係の一覧を表示
claude plugin list
```

## 注意点

- 依存関係のあるプラグインを無効化するには、ヒントに従って依存元のプラグインを先に無効化する必要があります
- `claude plugin enable` による自動有効化はトランジティブ（推移的）に適用されます。つまりA→B→Cの依存がある場合、Aを有効化するとBとCも自動で有効化されます
- マネージド設定（managed settings）によって強制有効化されているプラグインは、ローカル設定から無効化できません
- プロジェクト設定（`.claude/settings.json`）はユーザー設定より優先されます
