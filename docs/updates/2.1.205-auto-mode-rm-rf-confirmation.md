## 原文（日本語訳）

auto modeを改善し、コンテキストから解決できない変数を含む `rm -rf` を実行する前に確認を求めるようにした。

## 原文（英語）

Improved auto mode to ask before running `rm -rf` on a variable it can't resolve from context

## 概要

auto modeは、通常の権限確認プロンプトの代わりにClaudeの別インスタンス（分類器モデル）が会話の文脈を読み取り、各ツール呼び出しをリアルタイムで許可・拒否・エスカレーションする権限モードです。今回の改善では、`rm -rf $TARGET_DIR` のようにコマンド内に変数が含まれていて、その変数が実際に何を指すのか会話のコンテキストから確定できない場合に、分類器が安全側に倒れて自動承認せず、ユーザーに確認を求めるようになりました。変数の中身が展開されるまで対象パスが不明なままファイル削除が実行されてしまう事故を防ぐための安全対策強化です。

## 基本的な使い方

auto modeはPro以上のプランで、対応モデル（Claude Opus 4.6以降、またはSonnet 4.6）を使用している場合に選択できる権限モードです。Claude Codeのセッション内で権限モードを切り替えて使用します。

```text
# 権限モード切り替え（Shift+Tabで循環、またはコマンドで指定）
/permissions
# もしくは Shift+Tab で auto mode を選択
```

auto modeが有効な状態で、Claudeに任意のシェル操作を含むタスクを依頼すると、通常のファイル操作は自動承認されつつ、危険度が高い、または意図がコンテキストから確定できない操作についてはその都度確認が入るようになります。

## 実践例

### 未展開の変数を含む削除コマンドが実行されそうになるケース

Claudeが生成したスクリプトの中で、削除対象のパスが変数として渡され、その変数の値が会話履歴やこれまでのコマンド実行結果から一意に特定できない場合です。

```bash
#!/bin/bash
# Claudeが生成したクリーンアップスクリプトの一部
TARGET_DIR="${BUILD_ARTIFACT_DIR:-}"
rm -rf "$TARGET_DIR"
```

このような場合、`BUILD_ARTIFACT_DIR` が未設定だと `rm -rf ""` や `rm -rf` の対象が意図せず広がる恐れがあります。改善後のauto modeは、この変数が指す実際のパスを会話コンテキストから確認できないと判断すると、自動承認せずにユーザーへ確認を求めます。

### 複数ステップのタスクで一時ディレクトリ名が動的に決まるケース

一時ファイルのクリーンアップなど、直前の処理結果によってディレクトリ名が動的に決まるタスクをClaudeに任せている場合です。

```text
> 一時ビルドディレクトリを作成してテストを実行し、終わったら削除して
```

このようなタスクでは、Claudeが `mktemp` などで生成した変数名を使って `rm -rf "$TMP_DIR"` を実行しようとすることがあります。改善前は、直前の会話履歴からある程度パスを推測できれば自動承認されるケースがありましたが、改善後は変数の指す先が確実に特定できない限り、実行前に一度確認が挟まるようになり、意図しないディレクトリの削除リスクが下がります。

## 注意点

- auto modeでは元々、`rm -rf /` や `rm -rf ~` のようなファイルシステムのルートやホームディレクトリを対象とする削除は、bypassPermissionsモードであっても常に確認が入る「サーキットブレーカー」として扱われています。今回の改善はそれに加えて、変数を介した間接的な削除対象についても安全側の判断を強化するものです。
- auto modeの分類器は固定のコマンド許可・拒否リストではなく、会話の文脈から「ユーザーがこの具体的な操作を意図して許可したかどうか」を都度判断する仕組みです。そのため、同じ `rm -rf` コマンドでも文脈次第で確認が入るかどうかが変わります。
- auto modeの利用には、プランに応じた条件（Team/Enterpriseでは管理者による有効化、対応モデルの使用など）があります。利用しているプランやモデルによっては、この権限モード自体が選択肢に表示されない場合があります。
- フックが設定されている場合、auto modeやbypassPermissionsモードであってもフック側で操作をブロックできる点は変わりません。
