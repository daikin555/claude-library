## 原文（日本語に翻訳）

Windows TerminalおよびVS Code上のWSL2でのマウスホイールスクロールを修正（2.1.172のリグレッション）

## 原文（英語）

Fixed mouse-wheel scrolling in WSL2 under Windows Terminal and VS Code (regression in 2.1.172)

## 概要

バージョン2.1.172で発生したリグレッションにより、WSL2環境のWindows TerminalおよびVS Code統合ターミナルでマウスホイールによるスクロールが機能しなくなっていました。この修正により、WSL2ユーザーはマウスホイールを使って通常通りスクロールできるようになりました。

## 基本的な使い方

この修正はv2.1.179以降に自動適用されます。WSL2環境でClaude Codeを使用しているWindowsユーザーは、アップデート後すぐにスクロールが正常動作します。

```bash
# WSL2環境でのClaude Code起動（通常通り）
claude

# マウスホイールでの操作が正常に動作する
# - 会話履歴のスクロール
# - ファイル差分のスクロール
# - ツール出力のスクロール
```

## 実践例

### Windows Terminal での使用

```bash
# Windows Terminal でWSL2を起動
# wsl または Ubuntu などのプロファイルを選択
claude

# 長い応答やコードブロックをマウスホイールでスクロール可能
```

### VS Code 統合ターミナルでの使用

```bash
# VS Code のターミナルパネルからWSL2を使用
# ターミナル → 新しいターミナル（WSLを選択）
claude

# エディタと同様にマウスホイールでスクロール可能
```

## 注意点

- この問題はWSL2環境限定のリグレッションで、バージョン2.1.172から2.1.178まで影響を受けていました
- ネイティブLinuxやmacOS環境には影響なし
- キーボードショートカット（Page Up/Page Down、矢印キー）は問題の影響を受けていませんでした
