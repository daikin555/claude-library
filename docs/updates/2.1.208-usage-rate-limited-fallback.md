## 原文（日本語訳）

usageエンドポイントがレート制限されている場合、`/usage` はエラー画面の代わりに、「as of（時点）」の注記付きで直近の既知の使用状況バーを表示するようになった。

## 原文（英語）

`/usage` now shows your last-known usage bars with an "as of" note when the usage endpoint is rate-limited, instead of an error screen

## 概要

`/usage` はPro・Max・Team・Enterpriseプランのプラン上限に対する使用状況をバーで表示するコマンドですが、そのデータ取得元であるusageエンドポイント自体がレート制限されることがあります。これまではその場合エラー画面が表示されるだけでしたが、今回のアップデートで、このマシン上で過去60分以内に読み込まれた使用状況バーがあれば、それをいつ時点のデータかを示す注記とともに表示するよう改善されました。

## 基本的な使い方

```text
/usage
```

usageエンドポイントがレート制限中で、かつ過去60分以内に読み込まれたデータがある場合は、直近のバーが「直近の使用状況を表示しています」といった趣旨の注記とともに表示されます。`r` キーで再試行でき、成功すればバーは最新データに置き換わります。

## 実践例

### 短時間に何度も/usageを確認していてレート制限に当たった場合

セッション中に何度も `/usage` を開いて残量を確認していると、usageエンドポイント自体がレート制限されることがあります。この場合でも直近に読み込んだバーがそのまま表示され続けるため、エラー画面だけが表示されて状況が全く分からなくなることを避けられます。

```text
/usage
```

表示されたバーが「〇〇時点」の情報であることが注記され、`r` を押すことで最新データへの更新を試みられます。

### セッション開始直後にレート制限へ当たった場合

そのセッションでまだ一度も使用状況を読み込んでいない状態でレート制限にかかった場合は、表示できる直近データがないため、引き続き「usageエンドポイントがレート制限されている」旨のメッセージと再試行ショートカットが表示されます。2.1.208より前のバージョンでは、この状況は常にバーなしのエラー画面になっていました。

## 注意点

- 直近のバーが表示されるのは、そのマシン上で過去60分以内に使用状況を読み込んだことがある場合のみです。
- `r` キーで再試行すると、成功時に直近のバーが最新データに置き換わります。
- Session block（トークン使用量とコスト見積もり）はAPI利用者向けの情報であり、Max/Proなどのサブスクリプション利用者はプランの使用状況バーの方が課金に直結します。
- VS Code拡張機能のAccount & usageダイアログでも同様の内訳が確認できます（Claude Code v2.1.174以降が必要）。
