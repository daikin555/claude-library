## 原文（日本語に翻訳）

プランモードを改善しました：ユーザーの組織またはAuth設定がWeb上のClaude Codeに到達できない場合、「Ultraplanで精練する」オプションを非表示にするようになりました。

## 原文（英語）

Improved plan mode to hide the "Refine with Ultraplan" option when the user's org or auth setup can't reach Claude Code on the web

## 概要

プランモードには「Ultraplanで精練する」というオプションがあり、Web上のClaude Codeを使ってプランをさらに洗練させる機能です。このアップデートにより、ユーザーの組織設定や認証設定がWeb上のClaude Codeへの接続を許可しない場合、このオプションが自動的に非表示になります。利用できない機能を表示することによる混乱を防ぎます。

## 基本的な使い方

```bash
# プランモードで作業を開始
claude --plan "新しいAPIエンドポイントを設計して"

# Web Claude Codeにアクセス可能な場合:
# [1] プランを承認する
# [2] プランを修正する
# [3] Ultraplanで精練する  ← 表示される
# [4] キャンセル

# Web Claude Codeにアクセスできない場合:
# [1] プランを承認する
# [2] プランを修正する
# [4] キャンセル  ← Ultraplanオプションは非表示
```

## 実践例

### 企業環境でのプランモード

```bash
# 企業ネットワーク（インターネットアクセス制限あり）での使用
claude --plan "マイクロサービスアーキテクチャを設計して"

# 外部サービスへのアクセスが制限されている場合、
# UIには利用可能なオプションのみが表示される
# → "Ultraplanで精練する" は表示されない

# 利用可能なオプションで作業を継続
# プランを確認・承認してから実装を開始
```

### ネットワーク設定の確認

```bash
# Web Claude Codeへの接続確認
curl -s https://claude.ai/health || echo "Web Claude Codeへの接続不可"

# 接続できない場合は通常のプランモードを使用
# Ultraplanオプションは自動的に非表示になる

# プランモードでの通常の作業
claude --plan "データベーススキーマを最適化して"
```

### Bedrock/Vertex環境でのプランモード

```bash
# AWS Bedrockを使用した環境
export CLAUDE_PROVIDER=bedrock
export AWS_REGION=ap-northeast-1

# Bedrock経由での使用では、Web Claude Codeに接続できないため
# Ultraplanオプションは自動的に非表示になる
claude --plan "Lambda関数のアーキテクチャを設計して"

# 表示されるオプション（Bedrock環境）:
# [1] プランを承認する
# [2] プランを修正する
# [3] キャンセル
```

### カスタム認証環境での動作

```bash
# カスタムAPIエンドポイントを使用している場合
export ANTHROPIC_BASE_URL=https://api.custom-claude.company.com

# Web Claude Codeにアクセスできないカスタム環境では
# Ultraplanオプションが自動的に非表示になる

claude --plan "プロジェクト計画を作成して"
```

## 注意点

- この変更は純粋にUIの改善であり、機能の削除ではありません
- Ultraplanオプションは、利用可能な環境では引き続き表示されます
- 企業のファイアウォールやプロキシ設定が接続に影響する場合があります
- AWS Bedrock、Google Vertex等のサードパーティプロバイダー経由での使用時は、通常このオプションが非表示になります
- オプションが非表示になっても、プランモードの他の機能は正常に動作します
