## 原文（日本語に翻訳）

`ToolSearch`呼び出しのたびにREPLに「❯ Tool loaded.」が表示される問題を修正しました。

## 原文（英語）

Fixed "❯ Tool loaded." appearing in the REPL after every `ToolSearch` call

## 概要

`ToolSearch`ツールが呼び出されるたびにREPL（対話型インターフェース）に「❯ Tool loaded.」というメッセージが不必要に表示されていた問題が修正されました。ToolSearchはツールを動的に読み込む際に内部的に使用されますが、そのたびに表示されるメッセージが会話の流れを妨げるノイズとなっていました。修正後はこの不要なメッセージが表示されなくなり、クリーンな対話体験が提供されます。

## 基本的な使い方

この修正はv2.1.71以降で自動的に適用されます。ToolSearchが内部的に呼び出される際に不要なメッセージが表示されなくなります。

```bash
# Claude Codeでの通常の会話
> 何かツールを使って作業をしてください

# 修正前: ToolSearchが呼び出されるたびに「❯ Tool loaded.」が表示されていた
# 修正後: 不要なメッセージが表示されず、スムーズな対話が可能
```

## 実践例

### 複数ツールを使用する複雑なタスク

複数のツールが動的に読み込まれる複雑な作業。

```bash
# 複数のツールを使用するタスク
> このプロジェクトのセキュリティ監査を実施し、Linuxコマンドとコード解析ツールを活用してください

# 修正前: ツールが読み込まれるたびに「❯ Tool loaded.」が挿入され、出力が読みにくくなっていた
# 修正後: ツールの読み込みがバックグラウンドで静かに処理される
```

### MCPサーバーとの連携作業

MCPサーバーを介した外部ツールの動的読み込み。

```bash
# MCPツールを使用する作業
> データベースに接続して、ユーザーテーブルのスキーマを確認してください

# 修正後: MCPツールが静かに読み込まれ、関連するレスポンスのみ表示される
```

### 長時間にわたる自動化タスク

多数のツールを使用する自動化された作業フロー。

```bash
# 自動化タスク
> コードベースを解析し、テストを実行し、結果をレポートにまとめてください

# 修正後: ToolSearch呼び出しに伴うノイズなく、スムーズに進行する
```

## 注意点

- この修正はv2.1.71以降で自動的に適用されます。
- 「❯ Tool loaded.」メッセージはデバッグや開発時には有用な場合もありましたが、通常使用では不要なノイズでした。
- この修正はUI上のノイズを削除するもので、実際のツールの読み込み動作には影響しません。
- ToolSearchはツールを動的に提供するための内部機能であり、この修正でその存在がより透過的になります。
