## 原文（日本語に翻訳）

対象ディレクトリがすでに作業ディレクトリである場合の `/add-dir` メッセージを改善した。

## 原文（英語）

Improved `/add-dir` message when the directory is already a working directory

## 概要

`/add-dir` コマンドで既に作業ディレクトリとして追加済みのディレクトリを指定した場合のメッセージが改善されました。従来は不明瞭なエラーや無応答だったケースで、「すでに作業ディレクトリです」という明確なフィードバックが表示されるようになります。マルチディレクトリ作業時のUXが向上します。

## 基本的な使い方

`/add-dir` コマンドはセッションに追加の作業ディレクトリを加えるコマンドです:

```bash
# 新しいディレクトリを作業ディレクトリとして追加
/add-dir /path/to/other/project

# すでに追加済みのディレクトリを指定した場合（改善後）
/add-dir /path/to/already/added/project
# → "このディレクトリはすでに作業ディレクトリです" という明確なメッセージが表示される
```

## 実践例

### 複数プロジェクトの同時作業

```bash
# フロントエンドとバックエンドを同時に作業する場合
/add-dir /home/user/projects/backend

# セッションで作業できるディレクトリ:
# - /home/user/projects/frontend（初期ディレクトリ）
# - /home/user/projects/backend（追加したディレクトリ）
```

### 重複追加時の改善されたフィードバック

```bash
# 既に追加済みのディレクトリを再度追加しようとする
/add-dir /home/user/projects/backend

# 改善前: エラーや無応答
# 改善後: 明確なメッセージ
# → "'/home/user/projects/backend' はすでに作業ディレクトリとして追加されています。"
```

## 注意点

- `/add-dir` で追加できるディレクトリは、セキュリティ上の理由からアクセス権が必要です
- 追加したディレクトリはセッション終了まで有効です
- 同じプロジェクトの別ディレクトリを追加する場合は、まず `/add-dir` で別ディレクトリを指定する必要があります
