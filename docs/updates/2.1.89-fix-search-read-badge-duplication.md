## 原文（日本語に翻訳）

並列ツールを大量使用する際に、ターミナルのスクロールバックで折りたたまれた検索・読み込みグループのバッジが重複していた問題を修正しました。

## 原文（英語）

Fixed collapsed search/read group badges duplicating in terminal scrollback during heavy parallel tool use

## 概要

多数のツール（特にファイル検索や読み込み）を並列実行する際に、ターミナルのスクロールバックを確認すると、折りたたまれたツールグループのバッジ（「3ファイルを読み込み」などの表示）が重複して表示されるUIの問題がありました。実際の処理内容には影響ありませんでしたが、出力が読みにくくなっていました。v2.1.89で表示の重複が修正されました。

## 基本的な使い方

```bash
# 並列ツール使用例（自動的に折りたたまれて表示される）
claude "src/ディレクトリ内のすべてのTypeScriptファイルを読んでリファクタリングしてください"

# 修正前: スクロールバックで以下のようにバッジが重複していた
# ▶ Read 3 files  ← 重複
# ▶ Read 3 files  ← 重複
# ▶ Read 3 files  ← 重複

# 修正後: 一度だけ正しく表示される
# ▶ Read 3 files
```

## 実践例

### 並列読み込みが発生するシナリオ

```bash
# プロジェクト全体を分析するタスク
claude "このプロジェクトのアーキテクチャを分析してください"
# → 内部で多数のファイルを並列に読み込む
# → 修正後: ターミナルにバッジが重複せず表示される

# 複数のファイルを検索するタスク
claude "TypeScriptの型エラーをすべて修正してください"
# → 並列でファイルを検索・読み込む
# → 修正後: スクロールバックが整理された形で表示される
```

## 注意点

- この問題は表示のみの問題であり、実際の処理結果や機能には影響していませんでした
- 並列ツール使用時に特に発生していた問題です（シングルツール実行では発生しない）
- ターミナルのスクロールバック機能を多用する場合に視認性が向上します
- iTerm2、Terminal.app、tmux などすべてのターミナルエミュレータで修正が適用されます
