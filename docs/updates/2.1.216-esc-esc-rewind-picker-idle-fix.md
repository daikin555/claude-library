## 原文（日本語訳）

バックグラウンドタスクを伴う長時間実行セッションにおいて、アイドル状態のプロンプトでEsc-Escを押してもリワインドピッカーが開かない不具合を修正しました。

## 原文（英語）

Fixed Esc-Esc at an idle prompt not opening the rewind picker in long-running sessions with background tasks

## 概要

Claude Codeでは、プロンプト入力が空の状態でEscキーを2回押すと「リワインドピッカー」が開き、過去のチェックポイントに会話やコードを戻せます。しかし、バックグラウンドタスクが動いている長時間実行セッションでは、プロンプトがアイドル状態（入力待ち）であるにもかかわらずEsc-Escがリワインドピッカーを開かないケースがありました。今回の修正により、バックグラウンドタスクの有無にかかわらず、アイドル時のEsc-Escで確実にリワインドピッカーが開くようになりました。

## 基本的な使い方

この修正により、特別な設定なしに以下の操作が正しく機能するようになります。

```
# プロンプト入力欄が空の状態で
Esc → Esc
```

これにより、これまでの各プロンプト送信時点のチェックポイント一覧が表示され、任意の時点に「コードと会話を復元」「会話のみ復元」「コードのみ復元」などを選択できます。

## 実践例

### バックグラウンドでリサーチタスクを実行しながら過去の状態に戻す

長時間のバックグラウンドタスク（例: 大規模なコードベース調査）を走らせつつ、メインスレッドで作業を進めている際に、直前の変更を取り消したい場合の操作です。

```
# バックグラウンドでサブエージェントを実行中でも
# メインの入力欄が空であれば、Esc-Escでリワインドピッカーが開く
Esc → Esc
→ 過去のチェックポイントを選択
→ 「コードと会話を復元」を選択
```

### `/rewind` コマンドでの代替操作

キーボードショートカットがうまく機能しない環境や確実性を重視する場合は、スラッシュコマンドでも同様の操作が可能です。

```
/rewind
```

### 長時間セッションでの定期的なチェックポイント確認

長時間にわたる開発セッションでは、定期的にリワインドピッカーを開いて現在の到達点を確認しておくと、問題が起きたときにすぐ復旧できます。

```
# 作業の節目でリワインドピッカーを開いて確認
Esc → Esc
```

## 注意点

- リワインドはClaude Codeのファイル編集ツールによる変更のみを追跡します。外部のBashコマンドによる変更は対象外です。
- プロンプト入力欄にテキストが入力されている状態でEsc-Escを押すと、リワインドピッカーではなく入力内容のクリアが行われます。
- チェックポイントはセッション内でのプロンプト送信ごとに自動的に作成されます。
