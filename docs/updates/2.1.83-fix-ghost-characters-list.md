## 原文（日本語に翻訳）

高さ制限のあるリストでアイテムがオーバーフローした際にゴーストキャラクターが表示される問題を修正しました。

## 原文（英語）

Fixed ghost characters appearing in height-constrained lists when items overflow

## 概要

スラッシュコマンドの候補リストや設定メニューなど、高さに制限のあるリスト UI において、表示領域からはみ出すアイテムがある場合に余分な文字（ゴーストキャラクター）が表示される問題が修正されました。これにより、候補リストがクリーンに表示されるようになります。

## 基本的な使い方

スラッシュコマンドの候補やメニューが多数ある場合でも、クリーンに表示されるようになりました。

```bash
$ claude
> /           ← スラッシュコマンド候補を表示
  /clear
  /edit
  /exit
  /help
  /...        ← スクロール
# 修正後: はみ出したアイテムによるゴースト文字が表示されない
```

## 実践例

### ユースケース: プラグインが多い環境でのコマンド候補

プラグインによって多数のコマンドが追加されている場合、候補リストが長くなりますが、正しく表示されます。

```bash
$ claude
> /
# 多数のコマンドがある場合でも
# リストの境界付近にゴースト文字が現れない
# スムーズにスクロールできる
```

### ユースケース: 設定メニューでの操作

`/config` メニューなど、設定項目が多い場合も正しく表示されます。

```bash
$ claude
> /config
# 設定項目が多数あっても
# オーバーフロー部分にゴースト文字が出ない
```

## 注意点

- 「ゴーストキャラクター」は削除されていない余分な文字/ピクセルで、UI の描画バグです
- この問題は高さが制限されているすべてのリスト UI に影響していました
- 修正は描画ロジックの改善で、スクロール動作には変更はありません
