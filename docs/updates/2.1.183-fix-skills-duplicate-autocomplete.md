## 原文（日本語に翻訳）

複数のプラグインが有効になっているときに、ユーザーレベルのスキルがスラッシュコマンドのオートコンプリートに複数回表示される問題を修正しました。

## 原文（英語）

Fixed user-level skills appearing multiple times in slash-command autocomplete when multiple plugins are enabled.

## 概要

複数の Claude Code プラグインを有効にしている環境で、ユーザーレベルで定義されたスキルがスラッシュコマンドのオートコンプリートリストに重複して表示されるバグが修正されました。同じスキルが 2〜3 回表示されることで、コマンドの選択が煩雑になっていた問題が解消されました。

## 基本的な使い方

この修正は自動的に適用されます：

```bash
# 複数のプラグインが有効な環境で
/  # スラッシュを入力するとオートコンプリートが表示される

# 修正前: /my-skill が 2〜3 回表示される
# /my-skill
# /my-skill  ← 重複
# /my-skill  ← 重複

# 修正後: /my-skill が 1 回だけ正しく表示される
# /my-skill
```

## 実践例

### 影響を受けていた環境

```json
// .claude/settings.json での複数プラグイン設定例
{
  "plugins": [
    "~/plugins/my-plugin-a",
    "~/plugins/my-plugin-b"
  ]
}
```

上記のように複数プラグインを設定している場合、ユーザーレベルスキル（`~/.claude/skills/` 配下のスキル）が重複して表示されていました。

### ユーザーレベルスキルの確認

```bash
# ユーザーレベルスキルの場所
ls ~/.claude/skills/
# my-workflow.md
# code-review.md
# ...

# これらのスキルが複数プラグイン環境で重複表示される問題が修正された
```

## 注意点

- この問題は **ユーザーレベルのスキル**（`~/.claude/skills/` や `~/.claude/.claude/skills/` 配下）に限定されていました
- プロジェクトレベルのスキル（`.claude/skills/` 配下）は影響を受けていませんでした
- **複数のプラグインを有効にしている場合**にのみ発生していた問題です
- プラグインなし、または 1 つのみの場合は問題は発生しませんでした
