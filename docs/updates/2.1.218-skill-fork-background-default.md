## 原文（日本語に翻訳）

`context: fork` を指定したスキルをデフォルトでバックグラウンド実行するようにしました。スキルごとに `background: false` を指定することでオプトアウトできます。

## 原文（英語）

Changed skills with `context: fork` to run in the background by default; opt out per skill with `background: false`

## 概要

Skillのfrontmatterには `context: fork` というオプションがあり、これを指定するとスキルの実行が独立したサブエージェントのコンテキストに切り出され（フォークされ）、メインの会話履歴とは分離した状態で処理が行われます。これまでは `context: fork` を指定したスキルもフォアグラウンドで実行され、完了するまでメインの会話がブロックされていました。v2.1.218以降、`context: fork` を指定したスキルはデフォルトでバックグラウンド実行されるようになり、実行中でもメインの会話を継続できるようになりました。フォアグラウンドでの実行を維持したいスキル作者は、frontmatterに `background: false` を追加することでこの挙動をオプトアウトできます。

## 基本的な使い方

```yaml
---
name: deep-analysis
description: コードベース全体を分析して詳細なレポートを作成する
context: fork
---

このスキルはリポジトリ全体を調査し、アーキテクチャに関する
詳細なレポートを作成します。
```

`context: fork` のみを指定した場合、v2.1.218以降はこのスキルはバックグラウンドで実行され、実行中もユーザーは他の指示をメイン会話に入力し続けられます。

## 実践例

### バックグラウンド実行をそのまま活用する

```yaml
---
name: repo-deep-dive
description: リポジトリ全体の依存関係とアーキテクチャを調査する
context: fork
agent: Explore
---
```

```bash
claude
> repo-deep-diveスキルを実行して

# スキルはバックグラウンドで実行が開始され、
# 完了を待たずにメイン会話で別の作業を続けられる
> 実行中に別のバグ調査を進めて
```

### `background: false` でフォアグラウンド実行を維持する

対話的な確認が必要で、完了を待ってから次の指示を出したいスキルの場合は、明示的にフォアグラウンドを指定します。

```yaml
---
name: interactive-migration
description: マイグレーション計画をユーザーと対話しながら作成する
context: fork
background: false
---
```

```bash
claude
> interactive-migrationスキルを実行して

# background: false が指定されているため、
# 完了するまでメイン会話がスキルの実行結果を待つ
```

### 進捗確認しながらのバックグラウンドタスク管理

```bash
claude
> 大規模リファクタリングの影響範囲を調べるスキルを実行して

# context: fork かつデフォルトのバックグラウンド実行のため、
# 実行状況はタスク一覧から確認できる
> 実行中のタスク一覧を見せて
```

## 注意点

- この変更が適用されるのは `context: fork` を指定したスキルのみです。`context: fork` を指定していない通常のスキルの実行方式（フォアグラウンド）には影響しません
- バックグラウンド実行になったことで、スキルの完了を前提とした後続の指示を続けて出す場合は、完了を待つか進捗を確認してから指示する必要があります
- 対話的なやり取りや、結果を即座にメイン会話へ反映させたいスキルでは `background: false` を指定してフォアグラウンド実行を維持することを検討してください
