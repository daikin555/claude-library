## 原文（日本語訳）

- npmグローバルインストールが自動更新できない場合に、ワンタイム通知を表示するようになりました。`/doctor`コマンドで修正方法を確認できます。
- `claude doctor`コマンドで、最後の更新試行の結果を表示するようになりました。

## 原文（英語）

- Claude Code now shows a one-time notice when your npm global install can't auto-update; `/doctor` lists the fixes
- `claude doctor` now shows the result of your last update attempt

## 概要

Claude Codeはnpmのグローバルインストール環境では、特定の条件下（パーミッション不足など）で自動更新が失敗することがあります。v2.1.153では、自動更新に失敗した際に一度だけ通知が表示されるようになり、ユーザーが問題を見逃さないよう改善されました。また、`claude doctor`コマンドが最後の更新試行の結果を表示するようになったため、更新の状態をより詳しく診断・確認できます。

## 基本的な使い方

### 更新状態の確認

`claude doctor`コマンドを実行して、インストール環境の状態と最後の更新試行結果を確認します。

```bash
claude doctor
```

または、Claude Codeのチャット内でスラッシュコマンドとして実行することもできます。

```
/doctor
```

### 自動更新が失敗した場合のワンタイム通知

自動更新が失敗すると、次回起動時に一度だけ通知が表示されます。通知には`/doctor`コマンドで修正方法を確認するよう案内が含まれます。

```
⚠ Claude Codeの自動更新に失敗しました。/doctor を実行して修正方法を確認してください。
```

## 実践例

### 自動更新失敗の診断と修正

自動更新に失敗した際のよくあるケースと対処法を示します。

```bash
# まずdoctorで状態を確認
claude doctor

# グローバルインストールのパーミッション問題の場合
# npmのグローバルディレクトリを自分のユーザーに変更する
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH

# 再インストール
npm install -g @anthropic-ai/claude-code

# または sudo を使って更新（環境によっては必要な場合）
sudo npm install -g @anthropic-ai/claude-code
```

### nvm環境での更新管理

nvmを使用してNode.jsを管理している場合、Claude Codeはnvmのグローバルディレクトリにインストールされます。

```bash
# 現在のnvmバージョンを確認
nvm current

# 最新のClaude Codeに手動更新
npm install -g @anthropic-ai/claude-code

# doctorで更新結果を確認
claude doctor
```

### 企業プロキシ環境での更新確認

企業のプロキシ環境では自動更新がブロックされることがあります。

```bash
# doctorで最後の更新試行の詳細を確認
claude doctor

# npmプロキシ設定を確認・修正
npm config get proxy
npm config get https-proxy

# 必要に応じてプロキシを設定して手動更新
npm config set proxy http://your-proxy:8080
npm config set https-proxy http://your-proxy:8080
npm install -g @anthropic-ai/claude-code
```

### CI/CDパイプラインでの更新管理

CI/CDパイプラインでClaude Codeを使用している場合、自動更新の失敗を事前に検出できます。

```bash
# パイプラインの開始時にdoctorで状態確認
claude doctor

# 終了コードで正常性を確認
if ! claude doctor; then
  echo "Claude Codeの環境に問題があります"
  exit 1
fi
```

## 注意点

- 自動更新の失敗通知はワンタイム（一度だけ）表示されます。通知を見逃した場合でも`/doctor`または`claude doctor`で状態を確認できます。
- npmのグローバルインストールでは、インストール先ディレクトリのパーミッションが原因で自動更新が失敗するケースが多いです。
- `claude doctor`の出力には最後の更新試行の日時・結果・エラー内容が含まれるため、問題の特定が容易になります。
- nvm、fnm、voltaなどのNode.jsバージョンマネージャーを使用している場合は、それぞれの環境に合わせた更新手順が必要です。
- システム全体（`/usr/local/lib`など）にインストールされている場合は、`sudo`権限が必要になることがあります。
