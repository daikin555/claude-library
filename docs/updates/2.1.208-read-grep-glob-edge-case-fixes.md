## 原文（日本語訳）

Readが空のファイルを「offsetより短い」と誤って報告する問題、Grepが不正な正規表現パターンに対して黙って「No files found」を返してしまう問題、ページネーションが行われた際にGrepのcountモードが合計数を過小報告する問題、そしてGlobがパターン・パス・作業ディレクトリにnullバイトが含まれる場合に不明瞭なエラーでクラッシュする問題を修正。

## 原文（英語）

Fixed Read reporting empty files as "shorter than offset", Grep silently returning "No files found" for invalid regex patterns, Grep count mode under-reporting totals when paginated, and Glob crashing with an unclear error when the pattern, path, or working directory contained a null byte

## 概要

Claude Codeの基本的なファイル探索・検索ツールであるRead・Grep・Globには、それぞれ細かなエッジケースで誤った結果を返す不具合がありました。今回のアップデートで、空ファイルの読み込み結果表示、不正な正規表現パターンのエラー通知、`count` モードでのページネーション時の集計、null バイトを含む入力へのエラーハンドリングがそれぞれ改善され、これらのツールが返す情報の正確性が向上しました。

## 基本的な使い方

これらはいずれもClaude Codeが内部的に使うツールの挙動修正であり、ユーザーが明示的に呼び出す設定はありません。Claudeが `Read`・`Grep`・`Glob` を使うたびに、修正後の正確な結果や分かりやすいエラーメッセージが返るようになります。

```text
このプロジェクト内で "TODO" を含む行をすべて検索して
```

不正な正規表現を渡してしまった場合でも、修正後は「該当ファイルなし」ではなく、ripgrepの診断情報を含む具体的なエラーが返るため、Claudeがパターンを修正して検索し直せます。

## 実践例

### 空ファイルの内容を確認する場合

新規作成直後でまだ中身のない設定ファイルなどをReadで確認しようとすると、以前は「offsetより短いファイルです」という、あたかも指定した位置がファイル終端を超えているかのような誤解を招く通知が返っていました。

```text
config/local.json の中身を見せて
```

修正後は、ファイルが存在し中身が空であることを正しく示す通知が返るようになり、「ファイルが壊れている」「指定位置が間違っている」といった誤解を避けられます。

### 正規表現の書き方を間違えた場合

Grepはripgrepの正規表現構文を使うため、`interface{}` のようなメタ文字を含む文字列をエスケープせずに渡すと不正なパターンになることがあります。

```text
"interface{}" という文字列を検索して
```

以前はこのような不正なパターンに対して「No files found」という結果が返り、実際には該当ファイルがあるのに検索に失敗したのか、本当に存在しないのか区別できませんでした。修正後はripgrepの診断メッセージを含むエラーが返るため、Claudeが `interface\{\}` のように正しくエスケープして再検索できます。

### 大量のマッチをcountモードでページ分割して集計する場合

`output_mode: "count"` で大量のファイルにまたがる一致件数を数える際、`head_limit` や `offset` で結果を絞り込むと、以前は表示された分のファイルの合計しか計算されず、実際のマッチ総数より少ない数値が報告されていました。

```text
リポジトリ全体で "console.log" が何箇所あるか数えて
```

修正後は、表示上は一部のファイルしか列挙されていなくても、合計は常に全マッチ数を正しく反映します。

### パスにnullバイトが混入した場合

自動生成されたスクリプトや外部データに由来するパス文字列に意図せずnullバイトが混入すると、以前のGlobはこれを処理できず、原因の分かりにくいエラーでクラッシュしていました。

```text
生成されたパス一覧をもとに該当ファイルを検索して
```

修正後は、パターン・パス・作業ディレクトリのいずれかにnullバイトが含まれる場合、Claudeにnullバイトを取り除くよう促す明確なエラーメッセージが返るようになりました。

## 注意点

- これらはいずれもClaude Code v2.1.208で修正されたバグであり、それ以前のバージョンでは上記の誤動作が発生する可能性があります。
- Grepはripgrepをベースにしており、POSIX grepとは正規表現の構文が異なる点は引き続き注意が必要です。メタ文字（`{`、`}` など）を検索する場合はエスケープが必要です。
- countモードの合計数の修正は集計値のみに関するもので、`head_limit`/`offset` で列挙されるファイル一覧自体の件数は変わりません。
