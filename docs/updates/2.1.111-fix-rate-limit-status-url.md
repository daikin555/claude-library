## 原文（日本語に翻訳）

Bedrock/Vertex/Foundryでの429レート制限エラーが、Anthropic運営のプロバイダーのみをカバーする`status.claude.com`を参照していた問題を修正しました。

## 原文（英語）

Fixed 429 rate-limit errors on Bedrock/Vertex/Foundry referencing status.claude.com (it only covers Anthropic-operated providers).

## 概要

Claude Code v2.1.111では、Amazon Bedrock、Google Vertex AI、またはAzure Foundryを使用している際に429（レート制限）エラーが発生した場合、誤って`status.claude.com`（Anthropic直接APIのステータスページ）へのリンクが表示されていた問題が修正されました。これらのサードパーティプロバイダーのステータスは`status.claude.com`ではなく、各プロバイダーのステータスページで確認する必要があります。

## 基本的な使い方

### 修正前後のエラーメッセージ比較

```
# Bedrock使用中に429エラーが発生した場合

# 変更前（誤ったリンク）：
# Error: Rate limit exceeded (429)
# サービスの状態を確認してください: https://status.claude.com
# ← Bedrockには関係のないAnthropicのステータスページ

# 変更後（修正済み）：
# Error: Rate limit exceeded (429)
# Amazon Bedrockのレート制限に達しました
# Bedrockのサービス状態: https://health.aws.amazon.com/
# または、AWSコンソールでサービスクォータを確認してください
```

### 各プロバイダーのステータスページ

| プロバイダー | ステータスページ |
|-------------|----------------|
| Anthropic API（直接） | https://status.claude.com |
| Amazon Bedrock | https://health.aws.amazon.com/ |
| Google Vertex AI | https://status.cloud.google.com/ |

## 実践例

### Bedrockでのレート制限エラーへの対応

```bash
# BedrockでClaude Codeを使用中
export AWS_REGION=us-east-1
# BedrockプロバイダーでClaude Codeを設定済み

claude --print "大量のコードを処理してください"
# 429エラーが発生した場合：
# → AWSのサービスヘルスダッシュボードを確認
# → Bedrockのサービスクォータを確認
```

### Vertex AIでのレート制限対応

```bash
# Vertex AIを使用している場合の429エラー
# → Google Cloud Statusを確認
# https://status.cloud.google.com/

# Vertex AIのクォータを確認
gcloud compute regions describe us-central1 --format="json" | \
  jq '.quotas[] | select(.metric | contains("CLAUDE"))'
```

### レート制限の回避策

```bash
# Bedrockでのレート制限を回避するための設定例

# 1. 複数リージョンを活用（Bedrockは現在クロスリージョン推論をサポート）
export AWS_REGION=us-west-2  # 別リージョンに切り替え

# 2. リクエスト間隔を空ける
for file in src/*.py; do
  claude --print "このファイルをレビューしてください: $file"
  sleep 2  # レート制限を避けるための待機
done

# 3. AWS Service Quotasでクォータ増加をリクエスト
aws service-quotas request-service-quota-increase \
  --service-code bedrock \
  --quota-code L-XXXXXX \
  --desired-value 100
```

### Foundryでのレート制限対応

```
# Azure AI Foundry（旧Azure OpenAI）でのレート制限
# → Azure Service Health Portalを確認
# → デプロイメントのTPM（Tokens Per Minute）設定を確認
```

## 注意点

- `status.claude.com`はAnthropicが直接運営するAPIサービスのステータスのみを表示します。Bedrock、Vertex AI、Foundry経由のサービス状態は表示されません
- 429エラーはレート制限（Too Many Requests）を意味します。多くの場合、一定時間待てば解消されます
- Bedrock/Vertex/Foundryのレート制限はAWS/Google Cloud/Azureのコンソールで確認・調整できます
- サービス自体の障害と自分のアカウントのクォータ超過は別の問題です。エラーメッセージを注意深く読んで区別してください
