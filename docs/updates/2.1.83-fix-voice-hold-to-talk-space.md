## 原文（日本語に翻訳）

音声のホールドトゥトーク解放後、最大 3 秒間スペースキーが反応しない問題を修正しました。

## 原文（英語）

Fixed space key being swallowed for up to 3 seconds after releasing voice hold-to-talk

## 概要

音声入力のホールドトゥトーク（スペースキーを押している間だけ録音）を解放した後、最大 3 秒間スペースキーの入力が無視されてしまう問題が修正されました。音声録音の終了処理が入力をブロックしていたため、音声入力後にテキスト入力に切り替えると最初のスペースが入力されないという不便な問題でした。

## 基本的な使い方

ホールドトゥトークを解放した後、すぐにスペースキーを含むテキスト入力ができるようになりました。

```bash
$ claude
# スペースキー（またはその他のキー）を長押しして音声入力
# キーを離す
# 修正後: 即座にスペースキーを含むテキスト入力が可能
> 音声   と   テキスト   を   素早く切り替えられる
```

## 実践例

### ユースケース: 音声とテキストの混合入力

音声で一部を入力し、テキストで補完する作業フローがスムーズになります。

```bash
$ claude
# 音声: 「このファイルを」（スペース長押し → 離す）
# テキスト: "edit"（スペース離した直後にすぐ入力できる）
> このファイルを edit
```

### ユースケース: 音声入力後のコマンド修正

音声認識の結果を素早くテキストで修正できます。

```bash
$ claude
# 音声認識で入力 → キーを離す
# 認識ミスをすぐにテキストで修正
# 修正後: スペースを含む修正も 3 秒待たずに入力できる
```

## 注意点

- 「ホールドトゥトーク」は音声入力の一形態で、特定のキーを押している間だけ録音します
- 3 秒のブロックはオーディオの後処理タイムアウトに関連していました
- この修正後も音声の転写処理自体の完了を待つ必要がある場合があります
- スペースキー以外のキーへの影響については個別に確認してください
