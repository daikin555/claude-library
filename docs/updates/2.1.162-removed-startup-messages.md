# 起動時の不要なメッセージを削除：よりクリーンな起動体験

## 原文（日本語に翻訳）

「Claude in Chrome が有効」と「マーケットプレースがインストール済み」の起動メッセージを削除しました。モデルの自動更新とチームオンボーディングのヒントは、ロゴの下に静かな通知として表示されるようになりました。

## 原文（英語）

Removed the "Claude in Chrome enabled" and "marketplace installed" startup messages; model auto-updates and the team-onboarding tip now show as quiet notices under the logo

## 概要

Claude Code の起動時に毎回表示されていた「Claude in Chrome が有効です」と「マーケットプレースがインストールされました」というメッセージが削除されました。また、モデルの自動更新通知とチームオンボーディングのヒントは、目立つ大きなメッセージから、ロゴの下に表示される小さな通知に変更されました。起動時のノイズが減り、すぐに作業を開始できるようになります。

## 基本的な使い方

```bash
# 修正前の起動画面:
# ╔══════════════════════════════╗
# ║  Claude Code v2.1.161       ║
# ╚══════════════════════════════╝
# ✓ Claude in Chrome が有効です
# ✓ マーケットプレースがインストールされました
# ℹ モデルが claude-sonnet-4-6 に自動更新されました
# ℹ チームへのオンボーディング: /help を試してみてください
# > _

# 修正後の起動画面:
# ╔══════════════════════════════╗
# ║  Claude Code v2.1.162       ║
# ║  claude-sonnet-4-6 | [更新] ║ ← 静かな通知
# ╚══════════════════════════════╝
# > _
```

## 実践例

### クリーンな起動体験の確認

```bash
# 修正後は起動がすっきりとシンプルになった
claude

# メッセージが減り、すぐに作業に集中できる
```

### CI/CD でのスタートアップ出力

```bash
# CI ログがよりクリーンになった
# 以前は不要な通知メッセージがログに含まれていた
claude --print "コードレビューを実行して" 2>&1

# 修正後: 不要なメッセージがなく、重要な出力に集中できる
```

### スクリプトからの起動での出力処理

```bash
#!/bin/bash
# 起動メッセージのフィルタリングが不要になった
OUTPUT=$(claude --print "タスクを実行して")
echo "$OUTPUT"  # クリーンな出力のみ
```

## 注意点

- Claude in Chrome の状態はメッセージが消えても機能は継続しています
- 確認したい場合は、ブラウザの拡張機能設定で確認できます
- モデルの自動更新は引き続き行われますが、通知がより目立たなくなりました
- チームオンボーディング情報は `/help` コマンドでいつでも確認できます
