## 原文（日本語に翻訳）

`Ctrl+L` が画面クリアと強制再描画を行うようになりました。`Cmd+K` で UI が一部白くなった場合のリカバリーに使用できます。プロンプト入力のクリアには `Ctrl+U` または `Esc` 二回を使用してください。

## 原文（英語）

`Ctrl+L` now clears the screen and forces a full redraw — use this to recover when Cmd+K leaves the UI partially blank. Use `Ctrl+U` or double-Esc to clear prompt input.

## 概要

`Ctrl+L` が画面全体をクリアして完全な再描画を強制するようになりました。`Cmd+K`（macOS）でターミナルをクリアしたときに Claude Code の UI が部分的に白くなる問題が発生した場合のリカバリー手段として有効です。プロンプト入力欄をクリアしたい場合は `Ctrl+U` または `Esc` 二回を使用します。

## 基本的な使い方

| キー | 動作 |
|------|------|
| `Ctrl+L` | 画面クリア + 強制再描画（UI リカバリー） |
| `Ctrl+U` | プロンプト入力欄のみクリア |
| `Esc` 二回 | プロンプト入力欄のみクリア |
| `Cmd+K` (macOS) | ターミナル履歴クリア（UI が崩れる場合あり） |

## 実践例

### UI が崩れたときのリカバリー

`Cmd+K` を使用した後に画面が部分的に白くなった場合：

```
1. Ctrl+L を押す
2. Claude Code の UI が完全に再描画される
3. 正常な状態に戻る
```

### アイドル後の画面リフレッシュ

長時間放置後に画面が乱れた場合：

```
Ctrl+L → 画面全体がリフレッシュされる
```

### プロンプト入力のクリアとの使い分け

```
入力途中のプロンプトをクリアしたい場合：
  → Ctrl+U または Esc 二回を使用

画面全体（UI）をリフレッシュしたい場合：
  → Ctrl+L を使用
```

### tmux・screen 環境での利用

tmux や GNU screen を使用している場合も `Ctrl+L` は有効です：

```bash
# tmux 内で Claude Code を使用している場合でも
# Ctrl+L で Claude Code の UI 再描画が可能
```

## 注意点

- `Ctrl+L` はプロンプトに入力中のテキストには影響しません。入力内容は保持されます
- 以前は `Ctrl+L` と `Ctrl+U` で挙動が違いましたが、今回の変更で `Ctrl+L` が UI 再描画専用になりました
- プロンプト入力のクリアは `Ctrl+U`（readline 標準）または `Esc` 二回を使用してください
- 2.1.83 以前は `Ctrl+L` の動作が異なっていた可能性があります
