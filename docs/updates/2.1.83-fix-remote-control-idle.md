## 原文（日本語に翻訳）

実際には動作中のリモートコントロールセッションが Web セッション一覧でアイドルとして表示される問題を修正しました。

## 原文（英語）

Fixed Remote Control sessions showing as Idle in the web session list while actively running

## 概要

Remote Control（リモートコントロール）セッションが実際には処理を実行中であるにもかかわらず、Web ブラウザのセッション一覧では「アイドル（Idle）」と表示されてしまう問題が修正されました。これにより、Web UI からセッションの実際の状態を正確に確認できるようになります。

## 基本的な使い方

Web セッション一覧でリモートコントロールセッションの正確なステータスが表示されるようになりました。

```
Web UI のセッション一覧:
[ My Claude Session ] - Running  ← 修正後: 実行中に正しく表示
[ Another Session   ] - Idle
```

## 実践例

### ユースケース: 複数セッションの管理

Web ブラウザから複数の Remote Control セッションを管理する際、実際に動作中のセッションを正確に識別できます。

```
claude.ai のセッション一覧:
[ コードレビュー   ] - Running  ← 正しく「実行中」と表示
[ デプロイ作業     ] - Idle
[ ドキュメント作成 ] - Running  ← 正しく「実行中」と表示
```

### ユースケース: セッションのモニタリング

長時間実行するタスクのセッションを Web から確認する場合、ステータスが正確に反映されます。

```
# 大規模なコード生成を実行中のセッション
# Web UI から確認すると:
[ コード生成セッション ] - Running  ← 修正後の正確な表示
```

## 注意点

- Remote Control はリモートマシンで Claude Code を実行し、Web ブラウザや別の端末から操作する機能です
- ステータス表示の問題であり、セッション自体の実行には影響していませんでした
- 修正後は「Running」「Idle」「Completed」などのステータスが実態を正しく反映します
