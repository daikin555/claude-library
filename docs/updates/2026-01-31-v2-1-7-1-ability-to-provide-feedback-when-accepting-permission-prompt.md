## 原文（日本語に翻訳）

パーミッションプロンプトを承認する際にフィードバックを提供できる機能を追加しました。

## 原文（英語）

Added ability to provide feedback when accepting permission prompts

## 概要

Claude Codeがコマンド実行やファイル操作などの権限を要求する際、ユーザーは承認と同時にClaude Codeチームへフィードバックを送信できるようになりました。これにより、不要なプロンプトや改善が必要な権限要求について、直接フィードバックを提供できます。

## 基本的な使い方

パーミッションプロンプトが表示された際、以下の操作が可能です。

### フィードバック付きで承認する

1. パーミッションプロンプトが表示される
2. 承認する際にフィードバック入力欄が表示される
3. 任意でコメントを入力（例：「このプロンプトは毎回表示されるので煩わしい」）
4. 承認を確定

### 通常の承認（フィードバックなし）

フィードバック入力をスキップして、通常通り承認することもできます。

## 実践例

### 頻繁に表示されるプロンプトについてフィードバック

プロジェクト内で`npm install`を実行するたびにパーミッションプロンプトが表示される場合。

**フィードバック例:**
```
「npm installは安全なコマンドなので、毎回確認せずに自動承認してほしい」
```

このフィードバックにより、将来のバージョンで改善される可能性があります。

### セキュリティ関連の懸念を報告

特定のコマンドパターンに対して、より詳細な警告が必要だと感じた場合。

**フィードバック例:**
```
「`rm -rf`コマンドの承認時に、削除対象のパスをより明確に表示してほしい」
```

### ポジティブなフィードバック

新しいパーミッション機能が便利だと感じた場合。

**フィードバック例:**
```
「危険なコマンドの実行前に確認できるのは良い機能です」
```

### 誤検知の報告

安全なコマンドが不必要にブロックされていると感じた場合。

**フィードバック例:**
```
「`git status`のような読み取り専用コマンドまで確認が必要なのは過剰です」
```

## 注意点

- **任意機能**: フィードバックは任意であり、提供しなくても通常通りパーミッションを承認できます
- **改善に活用**: 提供されたフィードバックは、Claude Codeのパーミッションシステム改善に役立てられます
- **プライバシー**: フィードバックを送信する前に、機密情報が含まれていないか確認してください
- **即時反映はされない**: フィードバックは将来のバージョンでの改善に使用されるため、即座にパーミッション動作が変わるわけではありません
