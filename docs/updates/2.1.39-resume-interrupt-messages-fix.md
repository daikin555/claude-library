## 原文（日本語）
/resumeコマンドで中断メッセージがセッションタイトルとして表示されていた問題を修正しました。

## 原文（英語）
Fixed /resume showing interrupt messages as session titles

## 概要
`/resume`コマンドを使用して過去のセッションを再開する際、中断メッセージ（システム内部のメッセージ）がセッションタイトルとして誤って表示されていた問題が修正されました。これにより、セッション一覧がより見やすく、適切なタイトルで表示されるようになりました。

## 基本的な使い方

```bash
# セッション一覧を表示して再開
claude /resume
```

修正前は、セッション一覧に内部的な中断メッセージが表示されていましたが、修正後は適切なセッションタイトルや最後の会話内容が表示されるようになります。

## 実践例

### セッションの再開確認
```bash
# 過去のセッションを確認
claude /resume

# 修正後: 適切なタイトルでセッションが表示される
# 例:
# 1. "Add authentication feature"
# 2. "Fix bug in user registration"
# 3. "Refactor database queries"
```

### 中断したセッションの再開
```bash
# Ctrl+C で中断したセッションを再開
claude /resume

# 修正前: "Session interrupted" のような内部メッセージが表示
# 修正後: 実際の作業内容に基づいたタイトルが表示
```

### 複数セッションの管理
```bash
# セッション一覧から選択して再開
claude /resume

# 修正により、どのセッションを再開すべきかが一目で分かる
# 内部メッセージではなく、実際の作業内容が表示される
```

## 注意点

- この修正は表示の改善であり、セッションの機能自体には影響しません
- 過去に作成されたセッションも正しく表示されるようになります
- セッションの内容や進行状況は変更されません
