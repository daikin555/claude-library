## 原文（日本語訳）

`vimInsertModeRemaps` 設定を追加：vimモードで `jj` のようなINSERTモードの2キーシーケンスをEscapeにマッピングできる。

## 原文（英語）

Added `vimInsertModeRemaps` setting: map two-key insert-mode sequences like `jj` to Escape in vim mode

## 概要

Claude Codeのプロンプト入力欄にはvimスタイルの編集モードがありますが、これまでvimのモーションはキーバインドファイルからリマップできず、`.vimrc` で `jj` や `jk` をEscapeに割り当てる設定に慣れているユーザーはNORMALモードへ戻る手段が物理的なEscapeキーに限られていました。新設定 `vimInsertModeRemaps` により、INSERTモードでの2キーシーケンスをEscapeとして扱えるようになり、使い慣れたvimの操作感をClaude Code上でも再現できます。

## 基本的な使い方

`~/.claude/settings.json` にvimモードを有効化した上で、`vimInsertModeRemaps` に2文字のキーシーケンスとリマップ先を指定します。

```json
{
  "editorMode": "vim",
  "vimInsertModeRemaps": { "jj": "<Esc>" }
}
```

これで、INSERTモード中に `j` を入力した直後（1秒以内）に続けて `j` を入力すると、両方の文字が入力欄に残らずNORMALモードに切り替わります。

## 実践例

### `.vimrc` の `jk` マッピングをClaude Codeでも再現する

普段のエディタで `inoremap jk <Esc>` のような設定をしている場合、同じキーの組み合わせをClaude Codeにも設定できます。

```json
{
  "editorMode": "vim",
  "vimInsertModeRemaps": { "jk": "<Esc>" }
}
```

これにより、Claude Codeのプロンプト編集でも普段の指の動きだけでNORMALモードへ戻れるようになります。

### リマップ対象の文字列を通常通り入力する

1秒のタイムアウトを過ぎてから2文字目を入力した場合や、間に別のキー入力を挟んだ場合は、リマップは発動せずどちらの文字もそのまま入力されます。そのため、例えば `jj` をリマップに設定していても、`java` のように `j` を含む単語を少し間を空けて入力すれば通常通りテキストとして残せます。

```text
j (すぐに) j → Escapeとして扱われ、どちらの文字も入力欄に残らない
j (1秒以上待ってから) j → 通常の文字として "jj" が入力される
```

## 注意点

- キーシーケンスは必ず印字可能な文字2文字で指定する必要があり、リマップ先は `"<Esc>"` のみサポートされます。それ以外の長さや値を指定したエントリは無視されます。
- この設定はユーザー設定ファイル（`~/.claude/settings.json`）、`--settings` フラグ、および管理者が配布するmanaged settingsからのみ読み込まれます。プロジェクトの `.claude/settings.json` や `.claude/settings.local.json` に書いても無視されるため、チェックアウトしたリポジトリが勝手にキー入力をリマップすることはできません。
- 本機能はClaude Code v2.1.208以降が必要です。
