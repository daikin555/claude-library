## 原文（日本語訳）

ブラウザの起動に失敗した際に、`/upgrade` がアップグレードURLではなくログインフローを表示してしまう問題を修正。

## 原文（英語）

Fixed `/upgrade` showing a login flow instead of the upgrade URL when the browser fails to open

## 概要

`/upgrade` はPro・Maxプランのユーザー向けに表示されるスラッシュコマンドで、プランのアップグレードページをブラウザで開くためのものです。これまでは、リモート環境やヘッドレス環境などブラウザの自動起動に失敗する状況で `/upgrade` を実行すると、アップグレード用のURLが表示される代わりに、無関係なログインフローが表示されてしまう不具合がありました。今回のアップデートで、ブラウザが開けない場合には正しくアップグレードURLがテキストとして表示されるようになり、ユーザーが自分でそのURLを開いてアップグレード手続きを完了できるようになりました。

## 基本的な使い方

Pro・Maxプランで契約しているアカウントであれば、Claude Codeのセッション内で次のように入力するだけです。

```text
/upgrade
```

通常はデフォルトブラウザが自動的に開き、プランのアップグレードページが表示されます。

## 実践例

### SSH接続先やヘッドレス環境で作業している場合

GUIを持たないリモートサーバー上やコンテナ内でClaude Codeを使っている場合、ブラウザを自動的に起動することができません。

```text
/upgrade
```

以前はこのような環境でブラウザの起動に失敗すると、無関係なログインフローが表示されて混乱を招いていましたが、修正後はアップグレード用URLがテキストとして表示されるため、そのURLをコピーして手元のPCのブラウザで開けば手続きを進められます。

### ブラウザの既定アプリ設定が壊れている、あるいは制限されている場合

社給PCなどでポリシーによりデフォルトブラウザの自動起動がブロックされている、またはOS側の関連付けが壊れているケースでも、ブラウザの起動は失敗します。

```text
/upgrade
```

修正後は同様にアップグレードURLが表示されるため、ログインフローに誤って迷い込むことなく、表示されたURLを別のブラウザで開いてアップグレードを完了できます。

## 注意点

- `/upgrade` はPro・Maxプランのアカウントでのみ表示されるコマンドです。プランや環境によっては表示されない場合があります。
- Amazon BedrockなどサードパーティのプラットフォームでClaude Codeを使っている場合、認証はAWS側の資格情報で行われるため、`/upgrade` のようなAnthropicアカウントのプラン管理コマンドの扱いが異なる可能性があります。
- この修正はClaude Code v2.1.208以降が対象です。それより前のバージョンでは、ブラウザ起動に失敗した際に誤ってログインフローが表示されていました。
