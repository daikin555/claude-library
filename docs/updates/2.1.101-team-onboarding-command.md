# /team-onboardingコマンド：チームメンバーのオンボーディングガイドを自動生成

## 原文（日本語に翻訳）

ローカルのClaude Code使用状況からチームメンバーのランプアップガイドを生成する `/team-onboarding` コマンドを追加

## 原文（英語）

Added `/team-onboarding` command to generate a teammate ramp-up guide from your local Claude Code usage

## 概要

`/team-onboarding` は、自分のローカルClaude Code使用履歴を分析し、新しいチームメンバーがプロジェクトにすぐ馴染めるようなオンボーディングガイドを自動生成するコマンドです。どのコマンドをよく使うか、どんなワークフローが定着しているかを学習し、チーム向けの実践的なガイドを作成します。ドキュメントを手動で書く手間を大幅に削減できます。

## 基本的な使い方

```
/team-onboarding
```

Claude Code内でこのコマンドを実行するだけで、自分の使用パターンを分析してガイドを生成します。

## 実践例

### 新メンバー向けオンボーディングドキュメントの作成

プロジェクトに新しい開発者が加わったとき：

```
/team-onboarding
```

生成されるガイドには以下が含まれます：
- プロジェクトでよく使われるClaude Codeのコマンド一覧
- 定番のワークフローパターン
- プロジェクト固有のClaude.md設定の説明
- よく使われるスラッシュコマンドと使用例

### チームのWikiやドキュメントへの組み込み

```bash
# 1. ガイドを生成
# Claude Code内で実行
/team-onboarding

# 2. 生成されたガイドをチームWikiに追加
# 生成内容をコピーしてNotionやConfluenceに貼り付けるか、
# ファイルとして保存する
```

### リモートチームへの共有

```
/team-onboarding
```

生成したガイドをSlackやTeamsで共有することで、リモートチームメンバーがローカル環境をセットアップする前にClaude Codeの使い方を理解できます。

## 注意点

- **使用履歴が必要**: 十分な使用履歴がないと生成されるガイドが薄くなる場合があります。ある程度プロジェクトを使い込んでからの実行を推奨します
- **プロジェクト固有の内容**: 生成されるガイドはカレントディレクトリのプロジェクトに基づいた内容になります
- **定期的な更新**: チームのワークフローが変化したら `/team-onboarding` を再実行してガイドを更新することをお勧めします
- **プライバシー**: 生成されるガイドにはコマンド使用パターンが含まれますが、会話内容の機密情報は含まれません
