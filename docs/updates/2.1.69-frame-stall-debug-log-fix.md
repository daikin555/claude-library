## 原文（日本語に翻訳）

デバッグログの同期フラッシュによるタイピング中の断続的なフレームストールを修正

## 原文（英語）

Fixed occasional frame stalls during typing caused by synchronous debug log flushes

## 概要

タイピング中にデバッグログが同期的にフラッシュ（書き込み）されることで、入力レスポンスが断続的に遅延するフレームストールが発生していました。この修正により、デバッグログの書き込みが非同期化され、タイピング体験がスムーズになりました。

## 基本的な使い方

この修正は自動的に適用されます。v2.1.69以降では、Claude Codeへのテキスト入力が常にスムーズに動作します。

```bash
# 長いプロンプトをタイピングする際も
# フレームストール（一瞬の引っかかり）が発生しなくなる
claude
> [長いテキストを入力中...]  # 修正後: 引っかかりなしにスムーズ
```

## 注意点

- この問題は特にデバッグログが頻繁に出力される状況で顕著でした
- 修正は非同期ログフラッシュへの変更により実現されており、ログの内容は変わりません
- タイピングの快適さはハードウェアや環境にも依存しますが、この修正で一般的な引っかかりは解消されます
