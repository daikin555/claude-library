## 原文（日本語に翻訳）

サブエージェントで WebSearch が空の結果を返す問題を修正しました。

## 原文（英語）

Fixed WebSearch returning empty results in subagents.

## 概要

サブエージェント内で WebSearch ツールを使用したときに、実際には検索結果があるにもかかわらず空の結果が返されるバグが修正されました。この問題により、サブエージェントが Web 検索に依存するタスク（調査・情報収集など）を正常に実行できない場合がありました。

## 基本的な使い方

このバグ修正により、サブエージェント内での Web 検索が正常に機能するようになりました：

```
# メインエージェントからサブエージェントを呼び出し
# サブエージェント内で WebSearch を使用するタスク
→ 以前は空の結果が返っていたが、正常に検索結果が返るようになった
```

## 実践例

### サブエージェントでの調査タスク

```
# ワークフロースクリプト内でのサブエージェント呼び出し例
agent("最新のReactのバージョンと変更点を Web で調査してください")
# → WebSearch が正常に動作し、検索結果が返される
```

### deep-research スキルでの活用

```
/deep-research 最新のAI規制動向を調査してください
# → 内部でサブエージェントが Web 検索を実行
# → 修正により検索結果が正常に収集される
```

### ワークフローでの並行調査

```javascript
// ワークフロースクリプト例
const results = await parallel([
  () => agent("セキュリティの最新動向を WebSearch で調査"),
  () => agent("パフォーマンス最適化の手法を WebSearch で調査"),
  () => agent("新しいフレームワークを WebSearch で調査"),
])
// 以前はサブエージェントで WebSearch が空結果を返していたが、修正により正常動作
```

## 注意点

- この修正はサブエージェント経由での WebSearch に限定した問題でした
- メインエージェントでの WebSearch は以前から正常に動作していました
- WebSearch ツールを使用するには、WebSearch が有効な環境（Claude.ai、適切な API 設定など）が必要です
