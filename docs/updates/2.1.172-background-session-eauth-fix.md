## 原文（日本語に翻訳）

デーモンが自動更新された後、古いバージョンで開始したセッションにアタッチしようとするとEAUTHエラーで失敗する問題を修正しました

## 原文（英語）

Fixed background-session attach failing with EAUTH for sessions started on an older version after the daemon auto-updated

## 概要

Claude Codeのデーモンが自動更新された後、更新前のバージョンで開始されたバックグラウンドセッションに再接続しようとするとEAUTH（認証エラー）が発生してアタッチできなくなる問題が修正されました。これにより、デーモンの自動更新をまたいだセッションの継続性が保証されます。

## 基本的な使い方

この修正は自動的に適用されます。デーモンが自動更新されても、既存のバックグラウンドセッションに問題なく再接続できます。

```bash
# バックグラウンドセッションを開始
claude --background "長時間タスクを実行"
# セッションID: session-abc123

# デーモンが自動更新される（バックグラウンドで発生）

# 修正後: 更新後でも問題なくセッションにアタッチできる
claude --attach session-abc123
```

## 実践例

### 長時間タスクの実行中にデーモンが更新されるケース

```bash
# タスクを開始
claude agents
# バックグラウンドエージェントを起動

# しばらく後にClaude Codeが自動更新される
# 修正後: 実行中のセッションを正常に確認・操作できる
claude agents
# セッション一覧が正常に表示され、アタッチも可能
```

## 注意点

- この修正はバックグラウンドセッション（デーモン経由）に限定された問題でした
- 通常のフォアグラウンドセッションには影響しません
- デーモンの自動更新は `~/.claude/daemon/` 配下のプロセスによって管理されています
