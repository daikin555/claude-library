## 原文（日本語訳）

セッションのOAuthトークンに必要なスコープが不足している場合、Claude-in-Chromeが再接続時に403エラーをループし続ける問題を修正しました。

## 原文（英語）

Fixed Claude-in-Chrome 403-looping on reconnect when the session's OAuth token lacks a required scope

## 概要

Claude in Chrome拡張機能を使ってブラウザ操作を行う際、Claude Code側のセッションが保持するOAuthトークンに必要なスコープが不足していると、拡張機能への再接続処理が403 Forbiddenエラーを繰り返し受け取り、再接続の試行がループしてしまう不具合がありました。今回の修正により、スコープ不足による403エラーが正しく検出され、無限ループに陥ることなく適切に処理されるようになりました。

## 基本的な使い方

この修正はバグ修正のため、特別な設定は不要です。Claude in Chromeを通常どおり利用するだけで、修正後の挙動が適用されます。

```bash
claude --chrome
```

## 実践例

### Claude in Chromeを起動してブラウザ操作を行う

```bash
claude --chrome
```

拡張機能との接続が確立された後、セッションが長時間続いてOAuthトークンのスコープが不足する状況になっても、以前のように403エラーで再接続処理がループし続けることはなくなりました。

### 再接続状態を確認する

拡張機能との接続に問題がある場合は、Claude Code内から接続状態を確認できます。

```
> /status
```

もし認証スコープの不足が原因で接続できない場合は、再ログインすることで解消できます。

```bash
claude /login
```

## 注意点

- この修正はスコープ不足による403エラーのループを防ぐものであり、スコープ自体が不足している状態そのものは解消されません。必要な権限を得るには再認証が必要です。
- `CLAUDE_CODE_OAUTH_TOKEN`環境変数を手動で設定している場合、Chrome拡張機能との連携で問題が起きることが知られています。Claude in Chromeを使う場合はブラウザログイン（`~/.claude/.credentials.json`）を利用することが推奨されます。
- 拡張機能側のバージョンとClaude Code CLI側のバージョンに差異があると、スコープの不整合が発生しやすくなるため、両方を最新の状態に保つことを推奨します。
