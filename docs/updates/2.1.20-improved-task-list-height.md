## 原文（日本語訳）
タスクリストをターミナルの高さに基づいて表示アイテム数を動的に調整するよう改善しました

## 原文（英語）
Improved task list to dynamically adjust visible items based on terminal height

## 概要
ターミナルウィンドウのサイズに応じて、タスクリストの表示項目数が自動的に調整されるようになりました。小さなターミナルでも見やすく、大きなターミナルでは多くの情報を一度に確認できます。

## 基本的な使い方

タスクリストは自動的にターミナルサイズに適応します。特別な設定は不要です。

```bash
claude
> /tasks
```

## 実践例

### 小さなターミナルウィンドウ

ターミナルの高さが限られている場合、最も重要なタスクのみが表示されます：

```
Active Tasks (3 total, showing 2):
1. ✓ データベーススキーマ更新
2. ⏳ API エンドポイント実装

+ 1 more task (press ↓ to scroll)
```

### 大きなターミナルウィンドウ

十分な高さがある場合、すべてのタスクが表示されます：

```
Active Tasks (5 total):
1. ✓ データベーススキーマ更新
2. ⏳ API エンドポイント実装
3. ⏳ テストケース作成
4. ⏳ ドキュメント更新
5. ⏳ CI/CD パイプライン設定
```

### 複数のバックグラウンドタスク

バックグラウンドタスクが多数実行中の場合も、スクロールで確認できます：

```bash
claude
> /tasks
```

矢印キーでスクロールして、すべてのタスクを確認できます。

## 注意点

- **レスポンシブ表示**: ターミナルをリサイズすると、表示が自動的に調整されます
- **スクロール機能**: 表示しきれないタスクは矢印キーでスクロール可能です
- **視認性の優先**: 重要なタスク（実行中、エラー発生中）が優先的に表示されます
- **パフォーマンス**: 多数のタスクがあっても表示がスムーズです
