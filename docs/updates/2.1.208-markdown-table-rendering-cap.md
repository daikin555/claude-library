## 原文（日本語訳）

非常に大きなMarkdownテーブルがレンダリングを停止させたり、過剰なメモリを消費したりする問題を修正。200行を超えるテーブルは先頭200行のみを表示し、「…あとN行」という通知を表示するようになった。

## 原文（英語）

Fixed very large markdown tables stalling rendering or using excessive memory; tables over 200 rows show the first 200 with a "… N more rows" notice

## 概要

Claude Codeの応答にMarkdownテーブルが含まれる場合、ターミナル上で罫線付きのテーブルとして描画されます。しかし、数百行に及ぶような非常に大きなテーブルを描画しようとすると、ターミナルの描画処理が止まったように見えたり、メモリ使用量が急増したりする不具合がありました。今回のアップデートで、200行を超えるテーブルは先頭200行のみが表示され、省略された行数を示す「…あとN行」という通知が併記されるようになり、巨大なテーブルでもセッションが固まらなくなりました。

## 基本的な使い方

この修正はClaude Code内部のレンダリング処理の変更であり、ユーザー側で明示的に有効化する設定はありません。Claudeが200行を超えるMarkdownテーブルを出力すると、自動的に先頭200行だけが表示されます。

```text
| id | name | value |
| -- | ---- | ----- |
| 1  | foo  | 100   |
| 2  | bar  | 200   |
...
（201行目以降は省略され、「… 350 more rows」のような通知が表示される）
```

## 実践例

### 大規模CSVやDBの内容を要約させる場合

CSVファイルやデータベースのクエリ結果をClaudeに整形・一覧表示させると、行数が数百件に達することがあります。

```text
このCSVの全レコードをMarkdownテーブルで一覧表示して
```

以前は行数が多いと描画が固まったりターミナルが重くなったりすることがありましたが、修正後は200行までが表示され、残りは「… N more rows」という通知にまとめられるため、セッションが固まらず安全に確認できます。

### 依存関係一覧やログ集計結果を確認する場合

`package.json` の依存パッケージや大量のログ行を集計してテーブル化するようなタスクでも、件数が想定より多くなることがあります。

```text
すべての依存パッケージのバージョン一覧をテーブルで出して
```

行数が200行を超える場合は自動的に切り詰められるため、全件を確認したいときはCSVファイルへの書き出しや `head`/`grep` などのコマンドと組み合わせて確認するとよいでしょう。

## 注意点

- 表示は先頭200行までで打ち切られるため、全データを確認したい場合はファイルへの出力やページネーションを別途Claudeに依頼する必要があります。
- この上限はターミナルでの表示上の制限であり、Claudeが内部で保持・処理しているデータそのものが削られるわけではありません。
