## 原文（日本語に翻訳）

ストリーミング応答中にキューされたコマンドがちらつく問題を修正しました。

## 原文（英語）

Fixed queued commands flickering during streaming responses

## 概要

Claude が応答をストリーミング中に入力したコマンドがキューに入る際、UI がちらついたり、不安定に点滅する問題が修正されました。コマンドキューの表示が安定し、ユーザーが応答を待ちながら次のコマンドを準備する際の体験が向上します。

## 基本的な使い方

Claude が応答を生成中でも、次に実行したいコマンドを安定して入力・確認できるようになりました。

```bash
$ claude
> 大きなタスクを実行してください
# Claude が応答ストリーミング中に...
> 次のコマンドを入力
# 修正前: キューされたコマンドがちらついて表示が不安定だった
# 修正後: 安定して表示される
```

## 実践例

### ユースケース: 連続タスクの事前入力

Claude の応答を待ちながら次のタスクを入力する場合、コマンドが安定して表示されます。

```bash
$ claude
> プロジェクトの構造を分析してください
# Claude が分析中...
> 分析結果をもとに改善提案をまとめてください
# 修正後: 上記のコマンドがちらつかず安定して表示される
# Claude の応答完了後に自動実行される
```

### ユースケース: 長い処理の間の UI 安定性

大きなコードのレビューやファイル処理など、時間のかかる処理中の UI が安定します。

```bash
$ claude
> このプロジェクト全体のコードレビューをしてください
# ストリーミング中の UI が安定
# キュー表示がちらつかない
```

## 注意点

- この問題は純粋に表示上の問題であり、コマンドの実行や機能には影響していませんでした
- コマンドのキューイング自体は正常に動作していましたが、視覚的に不安定でした
- 修正後はキューに入ったコマンドが画面に安定して表示されます
