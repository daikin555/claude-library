## 原文（日本語に翻訳）

大きなページに対する `WebFetch` のピークメモリ使用量を削減しました。

## 原文（英語）

Reduced `WebFetch` peak memory usage for large pages

## 概要

WebFetch ツールで大きなウェブページを取得する際のピークメモリ使用量が削減されました。大量のコンテンツを持つページ（長い仕様書、大規模なドキュメントサイト、コンテンツリッチなページなど）を処理する際に使用するメモリが最適化されます。

## 基本的な使い方

大きなページを取得する際に、より少ないメモリで処理できるようになりました。

```bash
$ claude
> https://large-documentation-site.com/complete-reference を取得して分析してください
# 修正後: 大きなページでもメモリ効率よく処理される
```

## 実践例

### ユースケース: 大規模なドキュメントの取得

長いリファレンスドキュメントや仕様書を取得する際に効果があります。

```bash
$ claude
> React の公式ドキュメント全体を取得して、最新の Hooks の使い方をまとめてください
# 大きなページでもメモリ効率よく処理される
```

### ユースケース: データリッチなページの処理

大量のデータや画像を含むページを処理する場合も効率的に動作します。

```bash
$ claude
> https://github.com/org/repo/blob/main/huge-file.md を取得して要約してください
# ピークメモリ使用量が抑えられる
```

### ユースケース: 複数の大きなページの連続取得

複数のページを連続して取得する場合も、メモリ効率の改善で安定した動作が期待できます。

```bash
$ claude
> 以下の 10 個のドキュメントページを順に取得して比較分析してください:
> 1. https://...
> 2. https://...
# 各ページのメモリが効率的に管理される
```

## 注意点

- この改善はページのコンテンツをストリーミングまたは段階的に処理することでピークメモリを削減しています
- 非常に大きなページの場合は依然として一定のメモリを使用しますが、ピーク値が削減されます
- メモリが限られた環境（コンテナ、小メモリのサーバーなど）での安定性が向上します
