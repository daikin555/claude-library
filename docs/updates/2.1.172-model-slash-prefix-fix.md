## 原文（日本語に翻訳）

`claude agents` のディスパッチ入力での `/model` 候補が誤ったスラッシュプレフィックス付きで表示され、組織で無効化されているモデルが表示される問題を修正しました

## 原文（英語）

Fixed `/model` suggestions in the `claude agents` dispatch input rendering with a misleading slash prefix and showing models disabled for your org

## 概要

`claude agents` コマンドのエージェントディスパッチ画面において、モデル選択の候補が `/claude-sonnet-4-6` のように誤ったスラッシュプレフィックス付きで表示されていた問題と、組織のポリシーで無効化されているモデルが誤って候補に表示されていた問題が修正されました。

## 基本的な使い方

```
claude agents
# エージェントをディスパッチする際のモデル選択画面

修正前の表示:
  /claude-sonnet-4-6  ← 誤ったスラッシュプレフィックス
  /claude-opus-4-8    ← 組織で無効なモデルも表示
  /claude-haiku-4-5   ← 誤ったスラッシュプレフィックス

修正後の表示:
  claude-sonnet-4-6   ← 正しい表示
  claude-haiku-4-5    ← 組織で有効なモデルのみ
```

## 実践例

### 組織管理者向け: モデル制限の正しい反映

```json
// .claude/settings.json または組織設定
{
  "availableModels": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5-20251001"
  ]
}
// 修正後: agentsディスパッチ画面にも正しくフィルタリングされた候補が表示される
```

## 注意点

- この修正はUIの表示問題であり、実際のモデル動作には影響しません
- `availableModels` 設定による制限が `claude agents` ディスパッチ画面にも正しく適用されるようになりました
