## 原文（日本語訳）

project verifyスキルが、ドキュメント化されたコマンドが変更された場合にのみ更新されるべきところ、毎セッション書き換えられてしまっていた不具合を修正。

## 原文（英語）

Fixed project verify skills being rewritten on every session instead of only when a documented command changed

## 概要

`/verify` はビルド済みのバンドルスキルの一つで、コードの変更を実際にアプリを起動して確認するための手順（ビルド・起動・操作のレシピ）を学習し、`.claude/skills/verify/SKILL.md` としてプロジェクトに永続化します。これにより次回セッション以降はコールドスタートを回避し、既知の起動手順をすぐに再利用できるようになっています。今回の修正前は、このproject verifyスキルの中身が本来変更すべきタイミング（README・package.json・Makefileなどに記載された起動コマンド自体が変わった場合）以外の、ほぼ毎セッションで無条件に書き換えられてしまっており、せっかく学習・記録した内容が意図せず上書きされてしまう不具合がありました。

## 基本的な使い方

`/verify` はセットアップ不要で使えるバンドルスキルで、プロジェクトの種類（CLI・サーバー・TUI・ブラウザ駆動など）やREADME、package.json、Makefileなどの内容から起動方法を推測して動作確認を行います。

```text
/verify
```

初回実行時に起動レシピの発見に成功すると、`.claude/skills/verify/SKILL.md` にビルド・起動・操作の手順が保存されます。今回の修正により、このファイルは以降のセッションで無条件に上書きされることはなくなり、ドキュメント上のコマンド（README・package.json・Makefileなどに書かれた起動コマンド）が実際に変更された場合にのみ更新されるようになりました。

## 実践例

### 既存のproject verifyスキルを安心して育てていく

一度 `.claude/skills/verify/SKILL.md` に起動レシピが記録されたプロジェクトでは、以後の `/verify` 実行のたびに毎回ファイルが書き直されることがなくなったため、手動で追記したメモや、これまでの試行で判明した「ハマりどころ」の記述が意図せず消えてしまう心配がなくなりました。

```text
> このリポジトリの変更を確認して
/verify
```

チームで `.claude/skills/verify/SKILL.md` をコミットして共有している場合も、Claude自身のセッションごとの実行で内容が揺れ動くことがなくなり、レビューやdiffの見通しが良くなります。

### package.jsonの起動スクリプトを変更したときは正しく追従する

`npm run dev` から `npm run start:dev` のようにドキュメント化された起動コマンド自体を変更した場合には、そのタイミングでproject verifyスキルの内容が正しく更新されるべき挙動です。今回の修正は「毎回書き換わる」問題を止めるものであり、ドキュメント側のコマンドが実際に変わったときにはきちんと追従して更新される点は維持されています。

```text
# package.jsonのscriptsを変更した後
/verify
```

## 注意点

- この修正はproject verifyスキル（`.claude/skills/verify/SKILL.md`）の更新タイミングに関するものであり、`/verify` コマンド自体の基本的な使い方や、バンドルスキルとして利用するために必要なClaude Codeのバージョン要件（v2.1.145以降）には変更はありません。
- モノレポ構成の場合、project verifyスキルはリポジトリルートではなく、変更が加えられたパッケージ/アプリ側のディレクトリに保存されることがあります。既存のスキルがある場合は新しい学習内容を統合し、重複したファイルを作らないようにする方針は変わりません。
- 本記事作成時点では、この修正内容そのものについての詳細な公式ドキュメント記載は見つからなかったため、changelogの記載内容と `/verify` バンドルスキルに関する一般的なドキュメントをもとに記述しています。詳細は公式のchangelogやドキュメントを参照してください。
