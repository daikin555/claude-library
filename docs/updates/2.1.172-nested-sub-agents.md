## 原文（日本語に翻訳）

サブエージェントが独自のサブエージェントを生成できるようになりました（最大5階層まで）

## 原文（英語）

Sub-agents can now spawn their own sub-agents (up to 5 levels deep)

## 概要

Claude Codeのサブエージェント（Agentツール）が、自分自身の中でさらにサブエージェントを起動できるようになりました。これにより、複雑なタスクを階層的に分割して並列処理する「入れ子型マルチエージェント」アーキテクチャが実現します。最大5階層まで深くエージェントを入れ子にできます。

## 基本的な使い方

ワークフロースクリプト内でエージェントが別のエージェントを起動する例：

```javascript
// ワークフロースクリプト例
export const meta = {
  name: 'nested-research',
  description: 'ネストされたサブエージェントで調査を実施',
  phases: [{ title: 'Research' }, { title: 'Deep Dive' }],
}

phase('Research')
// 第1階層のエージェントが第2階層のエージェントを起動
const overview = await agent(`
  以下のトピックを調査し、各サブトピックについて
  サブエージェントを使って詳細調査を行ってください: ${topic}
`)
```

## 実践例

### 大規模コードベースの並列分析

```javascript
// 各モジュールを専任エージェントに割り当て、
// そのエージェントがさらに細かいファイル分析エージェントを起動
const results = await pipeline(
  modules,
  module => agent(`
    ${module}モジュールを分析してください。
    必要に応じてサブエージェントを使って各ファイルを並列調査してください。
  `)
)
```

### 階層型ドキュメント生成

```javascript
// 第1階層: セクション担当エージェント
// 第2階層: 各セクション内の詳細を書くエージェント
// 第3階層: コード例を生成するエージェント
const doc = await agent(`
  APIドキュメントを生成してください。
  各エンドポイントにサブエージェントを割り当て、
  サブエージェントはさらにコード例生成エージェントを起動してください。
`)
```

### マルチステージ調査タスク

```
Claude Code でのプロンプト例:
「このプロジェクトの技術的負債を調査してください。
  必要であれば各コンポーネントごとにサブエージェントを起動し、
  さらにそれぞれが詳細分析エージェントを使っても構いません。」
```

## 注意点

- **最大5階層まで**: それ以上の入れ子はできません。深い階層が必要な場合はアーキテクチャを見直してください
- **トークン消費に注意**: 入れ子が深くなるほどトークン消費が増大します。`budget.remaining()` で残量を確認してください
- **並列実行の上限**: 同時実行エージェント数は `min(16, CPUコア数 - 2)` に制限されています
- **コスト管理**: ネストされたエージェントの使用は強力ですが、不必要な入れ子は避けてください
