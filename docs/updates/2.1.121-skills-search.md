# /skills にタイプ絞り込み検索ボックスを追加

## 原文（日本語に翻訳）

`/skills` にタイプ絞り込み検索ボックスを追加しました。スクロールせずに長いリストからスキルを見つけられます。

## 原文（英語）

Added a type-to-filter search box to `/skills` so you can find a skill in long lists without scrolling

## 概要

Claude Code v2.1.121 では、`/skills` コマンドの一覧画面に検索ボックスが追加されました。多数のスキルを登録している場合でも、キーワードを入力するだけで目的のスキルをすぐに絞り込めます。スクロールして探す手間がなくなり、スキルの呼び出しが格段に速くなります。

## 基本的な使い方

```bash
# /skills を起動
/skills

# 検索ボックスにキーワードを入力して絞り込む
# 例: "review" と入力すると review 関連のスキルだけが表示される
```

## 実践例

### 多数のスキルから素早く検索

```
# /skills を起動した後、検索ボックスが表示される
# キーボードで直接入力すると絞り込みが始まる

> /skills
┌─────────────────────────────────┐
│ スキルを検索: [review_        ] │
├─────────────────────────────────┤
│ ✓ code-review                   │
│ ✓ security-review               │
│ ✓ pr-review                     │
└─────────────────────────────────┘
```

### チーム共有スキルの管理

```bash
# 大規模チームでは多数のスキルが登録される
# プロジェクト設定に以下のような多数のスキルがある場合でも検索で即座に見つかる

.claude/skills/
├── backend-review.md
├── frontend-review.md
├── database-migration.md
├── security-audit.md
├── performance-test.md
├── api-documentation.md
└── ... (20+ スキル)

# /skills → "migration" と入力 → database-migration が即座にヒット
```

## 注意点

- 検索はスキル名のインクリメンタルサーチ（入力と同時に絞り込み）です
- 大文字・小文字は区別されません
- スキルが少数の場合は検索ボックスも表示されますが、スクロールのメリットがより大きい多数のスキルがある環境で特に効果的です
