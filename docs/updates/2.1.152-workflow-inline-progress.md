## 原文（日本語に翻訳）

Workflow ツールのインライン進行状況表示を簡略化しました。ライブエージェント数はプロンプト下部の永続的なワークフローステータス行にのみ表示されるようになりました。

## 原文（英語）

Simplified the Workflow tool's inline progress display — live agent counts now show only in the persistent workflow status row below the prompt

## 概要

Workflow ツール使用時のインライン進行状況表示が整理されました。以前はワークフローの各インラインメッセージにもエージェント数が表示されていましたが、この情報をプロンプト下部の専用ステータス行に集約することで、ワークフロー実行中の表示がすっきりしました。

## 基本的な使い方

```
# ワークフローを実行すると
→ インラインの進行状況には詳細なステップ情報が表示
→ ライブエージェント数はプロンプト直下の専用ステータス行に表示
   （例："Workflow: 3 agents running"）
```

## 実践例

### 複数エージェントのワークフロー実行

並列処理を含むワークフローの進行状況を確認する例です。

```
# ワークフローを起動
/my-workflow

→ インライン表示（簡略化）：
  "Step 1: analyzing..."
  "Step 2: processing..."

→ プロンプト下部のステータス行：
  "Workflow: 5 agents running | 2 completed"
```

## 注意点

- この変更はビジュアル表示の整理のみで、ワークフローの機能や動作には影響しません
- エージェント数はプロンプト下部のステータス行でリアルタイムに更新されます
- 後述の「バックグラウンドエージェントのタイマー表示」機能と組み合わせると、ワークフローの全体的な進行状況を把握しやすくなります
