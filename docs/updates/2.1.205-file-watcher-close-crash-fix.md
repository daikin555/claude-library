## 原文（日本語訳）

ディレクトリのスキャンが実行中の状態でファイルウォッチャー（file watcher）がクローズされた際に発生していたクラッシュを修正した。

## 原文（英語）

Fixed a crash when a file watcher was closed while a directory scan was still in flight

## 概要

Claude Codeは、作業ディレクトリ内のファイル変更（外部エディタでの編集やgit操作によるファイル更新など）を検知するために、内部的にファイルウォッチャーを利用しています。今回の修正は、このファイルウォッチャーが非同期のディレクトリスキャン処理の完了を待たずにクローズされた場合に、スキャン処理がすでに破棄されたリソースへアクセスしてクラッシュしてしまう競合状態（レースコンディション）を解消したものです。セッションの終了やディレクトリ切り替えなど、ウォッチャーの停止とスキャンの実行タイミングが重なった際の安定性が向上しています。

## 基本的な使い方

この修正はClaude Codeの内部実装に関するバグフィックスであり、ユーザーが明示的に操作する設定項目やコマンドはありません。通常通りClaude Codeを起動し、プロジェクトディレクトリ内でファイルを編集・追加・削除しながら作業するだけで、修正後の安定したファイル監視の恩恵を受けられます。

```text
claude
```

バージョン2.1.205以降にアップデートしていれば、内部的にこの修正が適用された状態でセッションが開始されます。

## 実践例

### 大規模ディレクトリを開いた直後にセッションを終了する

大量のファイルを含むリポジトリを開くと、Claude Codeはバックグラウンドでディレクトリのスキャンを行い、ファイル監視の初期化を進めます。このスキャンが完了する前にセッションを終了したり別のプロジェクトへ切り替えたりすると、従来はファイルウォッチャーのクローズ処理とスキャン処理が競合してクラッシュすることがありました。修正後は、スキャンが実行中でもウォッチャーのクローズが安全に処理されるため、大規模プロジェクトを開いてすぐに終了するような操作でも落ちにくくなっています。

```text
$ cd large-monorepo/
$ claude
> # スキャン完了前に Ctrl+C や /exit でセッションを終了しても安定して終了する
```

### ビルドツールやgit操作でディレクトリ構成が頻繁に変わるプロジェクトでの利用

`npm install` やブランチ切り替えなど、短時間に大量のファイル作成・削除が発生する操作を行っている最中にClaude Codeのセッションを開始・終了すると、ディレクトリスキャンとファイルウォッチャーのライフサイクルが重なりやすくなります。今回の修正により、こうしたタイミングでもクラッシュせずに安定してセッションを継続・終了できるようになりました。

```text
$ git checkout feature-branch &
$ claude
> # ブランチ切り替えでファイルが大量に変化している最中でも安定して動作する
```

## 注意点

- この修正はクラッシュを防ぐための内部的な安定性向上であり、ユーザーが設定できるオプションや新しいコマンドは追加されていません。
- 大規模なリポジトリやCIなどファイル数・変更頻度が多い環境ほど、従来の不具合が顕在化しやすかったため、そうした環境で特に恩恵が大きいと考えられます。
- 本記事作成時点で、Claude Code公式ドキュメント（code.claude.com/docs）内にファイルウォッチャーやディレクトリスキャンの内部動作を具体的に解説したページは見つかりませんでした。挙動の詳細は今後のドキュメント更新を確認してください。
