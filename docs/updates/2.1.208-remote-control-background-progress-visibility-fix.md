## 原文（日本語訳）

ターミナルでホストされているセッションにRemote Controlクライアントが接続した際、タスクが開始・終了するまでバックグラウンドエージェントやワークフローの進捗が見えない不具合を修正した。

## 原文（英語）

Fixed Remote Control clients attaching to a terminal-hosted session not seeing background agents and workflow progress until a task started or stopped

## 概要

Remote Controlは、`claude.ai/code` やClaudeのモバイルアプリから、ローカルで実行中のClaude Codeセッションに接続して操作を継続できる機能です。接続時には、そのセッションで既にバックグラウンド実行されているサブエージェントやダイナミックワークフローの進捗が、接続先のデバイスにも表示される仕様になっています。しかし、ターミナルでホストされたセッション（`claude remote-control` や `--remote-control` フラグで起動したセッション）にRemote Controlクライアントが接続した場合、バックグラウンドタスクが新たに開始または終了するまでは、既に進行中のエージェントやワークフローの状態がクライアント側に表示されないという不具合がありました。今回の修正により、接続した瞬間に現在進行中のバックグラウンド作業の状態が正しく反映されるようになりました。

## 基本的な使い方

ターミナルでRemote Controlセッションを開始します。

```bash
claude --remote-control
```

表示されたセッションURLやQRコードから、別のデバイス（ブラウザやモバイルアプリ）で接続します。

## 実践例

### 長時間のバックグラウンドタスク中にスマホへ切り替える

デスクで作業を開始し、Claudeにバックグラウンドでビルドやテストを実行させたまま、外出先でスマホから状況を確認したい場合があります。

```text
このE2Eテストをバックグラウンドで実行しておいて
```

その後スマホのClaudeアプリからセッションに接続すると、修正前はテストが完了するかタスクの状態が変化するまで進捗が表示されませんでしたが、修正後は接続直後から現在の実行状況が確認できます。

### 既存のダイナミックワークフローに途中から合流する

ターミナルで開始した複数ステップのワークフローが進行中に、別のPCのブラウザから `claude.ai/code` に接続して途中から状況を見たい場合です。

```bash
claude remote-control --name "Release Workflow"
```

修正前はワークフローの各ステップが開始・完了するタイミングまで待たないと最新状態が反映されませんでしたが、修正後は接続した時点のワークフロー進捗がすぐに表示されます。

## 注意点

- この不具合はターミナルでホストされたセッションに限った問題で、Desktopアプリでホストされたセッションでは v2.1.207 で既に同様の修正が入っています。
- Remote Controlはclaude.aiのOAuthログインが必要で、APIキー認証では利用できません。
- Amazon Bedrock、Google CloudのAgent Platform、Microsoft Foundryなど `api.anthropic.com` 以外のエンドポイントを利用している場合、Remote Control自体が利用できません。
