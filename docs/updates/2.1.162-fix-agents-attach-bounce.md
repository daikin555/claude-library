# 修正：バックグラウンドサービス再起動後の claude agents attach で即座にリストに戻る問題

## 原文（日本語に翻訳）

バックグラウンドサービスの再起動後、最初の試行で `claude agents` の attach がセッションリストに即座に戻ってしまうことがある問題を修正しました。

## 原文（英語）

Fixed `claude agents` attach occasionally bouncing straight back to the session list on the first try after a background-service restart

## 概要

バックグラウンドサービス（Claude Code のバックグラウンドプロセス管理デーモン）が再起動した直後に `claude agents` でセッションにアタッチしようとすると、アタッチが成功せず即座にセッション一覧に戻ってしまうことがありました。v2.1.162 でこの競合状態が修正され、サービス再起動直後でも確実にセッションにアタッチできるようになりました。

## 基本的な使い方

```bash
# セッション一覧を表示
claude agents

# セッションを選択してアタッチ（修正後は確実に動作）
# Enter を押してアタッチ → バウンスせずにセッションに接続される
```

## 実践例

### バックグラウンドサービス再起動後のアタッチ

```bash
# バックグラウンドサービスが再起動した後（システム再起動後など）
# 以前: Enter を押してもセッションリストに戻ってしまうことがあった
# 修正後: 確実にセッションにアタッチできる

claude agents
# セッションを選択 → Enter → セッションに正常に接続
```

### システム再起動後のセッション回復

```bash
# システム再起動後にバックグラウンドセッションを回復
claude agents

# 待機中または再開可能なセッションを選択してアタッチ
# 修正後は初回の試行で成功する
```

### CI/CD 環境での利用

```bash
#!/bin/bash
# バックグラウンドサービスの状態に関わらず確実にアタッチ
SESSION_ID=$(claude agents --json | jq -r '.[0].id')
claude agents attach "$SESSION_ID"
# 修正後は再試行なしに動作する
```

## 注意点

- この問題はバックグラウンドサービスが再起動した直後（数秒以内）に特に発生していました
- 修正前は 2 回目の試行では成功することが多かったですが、修正後は初回から成功します
- バックグラウンドサービスは `claude agents` の初回起動時や、システム再起動後に再起動されます
- 問題が続く場合は `claude agents list` でサービス状態を確認してください
