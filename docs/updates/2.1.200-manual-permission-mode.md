## 原文（日本語に翻訳）

CLI、`--help`、VS Code、JetBrainsの全体で、「default」権限モードの名称を「Manual」に変更しました。`--permission-mode manual` と `"defaultMode": "manual"` は、従来の `default` と併せて受け付けられます。

## 原文（英語）

Changed the "default" permission mode to "Manual" across the CLI, `--help`, VS Code, and JetBrains; `--permission-mode manual` and `"defaultMode": "manual"` are accepted alongside `default`

## 概要

Claude Codeには、ツール呼び出し（ファイル編集やコマンド実行など）ごとに許可を求めるかどうかを制御する「権限モード」があります。従来の初期モードは内部的にもUI上でも `default`（デフォルト）と表記されていましたが、今回の変更でCLI・`--help`・VS Code拡張機能・JetBrainsプラグインの表示が統一的に「Manual」という名称に変わりました。これは「各ツール呼び出しのたびに承認を求める」という挙動をより分かりやすく表現するための名称変更であり、動作そのものは変わりません。後方互換性のため、`default` という指定も引き続き使用できます。

## 基本的な使い方

CLIフラグで指定する場合、`manual` と `default` のどちらでも同じ結果になります。

```bash
# 新しい指定方法
claude --permission-mode manual

# 従来の指定方法（引き続き利用可能）
claude --permission-mode default
```

`settings.json` で恒久的に設定する場合も同様に、どちらの値でも動作します。

```json
{
  "permissions": {
    "defaultMode": "manual"
  }
}
```

## 実践例

### CLIから明示的にManualモードで起動する

チーム内のドキュメントやスクリプトを新しい名称に揃えたい場合、`--permission-mode manual` を使って起動します。

```bash
claude --permission-mode manual
```

### settings.jsonでプロジェクト全体の初期モードを固定する

プロジェクト共有設定（`.claude/settings.json`）に以下を追加すると、そのプロジェクトを開いたセッションは常にManualモード（各操作を都度承認する挙動）で開始されます。

```json
{
  "permissions": {
    "defaultMode": "manual"
  }
}
```

### VS Code / JetBrainsのモードセレクターから切り替える

VS CodeやJetBrainsのClaude Code拡張機能では、モード選択UIの表示が「default」から「Manual」に変わります。ドロップダウンやモード切り替えのショートカット（CLIでは `Shift+Tab` で `default`／`acceptEdits`／`plan` などを循環）から選択する操作自体は従来通りです。

## 注意点

- 挙動自体の変更はなく、あくまで表示名の統一です。既存の `"defaultMode": "default"` やスクリプト内の `--permission-mode default` はそのまま動作し続けます。
- `acceptEdits`、`plan`、`auto`、`dontAsk`、`bypassPermissions` など他の権限モードの名称や挙動には影響しません。
- ドキュメントやCI設定、社内Wikiなどで `default` モードについて言及している場合は、今後は「Manual」という表記に置き換わっている可能性があるため、混乱を避けるため表記を揃えておくと良いでしょう。
