# 変更：プランモードで「コンテキストのクリア」オプションがデフォルトで非表示に

## 原文（日本語に翻訳）

プランモードで「コンテキストのクリア」オプションをデフォルトで非表示にするように変更（`"showClearContextOnPlanAccept": true` で復元可能）。

## 原文（英語）

Changed plan mode to hide the "clear context" option by default (restore with `"showClearContextOnPlanAccept": true`)

## 概要

プランモード（`/plan` コマンドや Shift+Tab で起動）でプランを承認する際に表示されていた「コンテキストをクリア」オプションが、デフォルトで非表示になりました。誤ってコンテキストをクリアしてしまうリスクを減らすための変更です。復元したい場合は設定ファイルで有効化できます。

## 基本的な使い方

```
# プランモードの承認画面（修正後のデフォルト）
┌─────────────────────────────┐
│ プランを承認しますか？       │
│                             │
│ [承認] [キャンセル]         │
└─────────────────────────────┘
（「コンテキストをクリア」オプションは非表示）
```

## コンテキストクリアオプションを有効化する方法

```json
// ~/.claude/settings.json
{
  "showClearContextOnPlanAccept": true
}
```

有効化すると以前の表示に戻ります：

```
┌─────────────────────────────────┐
│ プランを承認しますか？           │
│                                 │
│ [承認] [コンテキストをクリア]   │
│ [キャンセル]                    │
└─────────────────────────────────┘
```

## 実践例

### ### 設定ファイルでの有効化

コンテキストをクリアしながらプランを承認するワークフローを好む場合：

```json
// ~/.claude/settings.json
{
  "showClearContextOnPlanAccept": true
}
```

または プロジェクトごとの設定：

```json
// プロジェクト/.claude/settings.json
{
  "showClearContextOnPlanAccept": true
}
```

### ### プランモードの基本的な使い方

```bash
# プランモードを起動（Shift+Tab または /plan コマンド）
# → Claude がタスクを実行する前にプランを提示
# → 承認するかキャンセルするかを選択
```

## 注意点

- **誤操作防止**: 「コンテキストをクリア」は重要な操作であるため、デフォルトで非表示に
- **既存のワークフロー**: コンテキストをクリアしながらプランを承認するワークフローを使っていた場合は、設定で明示的に有効化が必要
- `"showClearContextOnPlanAccept": true` の設定はユーザー設定・プロジェクト設定どちらでも有効
