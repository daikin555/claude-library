## 原文（日本語に翻訳）

`claude agents` のセッションステータスを修正: 完了済みの行が「Done」と「Needs your input」の間でちらつくように切り替わらなくなり、停止したエージェントは「Needs attention」とラベル表示されるようになり、PRに言及する実行結果はクリック可能なリンクとして表示されるようになった

## 原文（英語）

Fixed `claude agents` session status: completed rows no longer flip between "Done" and "Needs your input", stalled agents are now labeled "Needs attention", and results that mention a PR show a clickable link

## 概要

`claude agents` コマンドが表示するバックグラウンドセッション一覧（agent view）のステータス表示に関する3つの不具合がまとめて修正されました。完了済みのセッションが「Done」と「Needs your input」の間で表示がちらついていた問題が解消され、応答が止まってしまったエージェントには新たに「Needs attention」というラベルが付くようになりました。さらに、実行結果にPull Requestへの言及が含まれる場合は、クリック可能なリンクとして表示されるようになり、agent view上でのステータス把握がより正確かつ便利になりました。

## 基本的な使い方

```bash
# バックグラウンドで実行中・完了済みのエージェント一覧を表示
claude agents
```

`claude agents` を実行すると、各バックグラウンドセッションの状態がテーブル形式で一覧表示されます。今回の修正により、このテーブルのステータス列の表示精度が向上しています。

## 実践例

### ちらつきが解消された完了済みセッションの表示

```
修正前の問題シナリオ:
1. バックグラウンドエージェントがタスクを完了
2. claude agents を実行すると一瞬「Done」と表示される
3. ❌ 直後に「Needs your input」に切り替わり、再度「Done」に戻る
4. どちらが正しい状態か判断しづらい

修正後の動作:
1. バックグラウンドエージェントがタスクを完了
2. claude agents を実行すると「Done」と表示される
3. ✅ 表示が安定し、ちらつきは発生しない
```

### 停止したエージェントの「Needs attention」ラベル

```bash
$ claude agents

ID        STATUS            TASK
a1b2c3    Done              READMEの誤字を修正
d4e5f6    Needs attention   依存パッケージの更新（応答が停止）
g7h8i9    Running           テストスイートの実行
```

```
修正前: 応答が止まったエージェントも他の進行中セッションと
        見分けがつきにくい表示だった

修正後: ✅ 「Needs attention」ラベルにより、
        対応が必要なエージェントを一目で識別できる
```

### PRへの言及がある実行結果のリンク表示

```
修正前の問題シナリオ:
1. バックグラウンドエージェントが「PR #482 を作成しました」と報告
2. ❌ 結果欄にはプレーンテキストとして表示されるのみ
3. PRを確認するにはURLをコピーしてブラウザで開く必要があった

修正後の動作:
1. バックグラウンドエージェントが「PR #482 を作成しました」と報告
2. ✅ 結果欄でPRの言及がクリック可能なリンクとして表示される
3. クリックするだけでPRページに直接アクセスできる
```

### 複数のバックグラウンドエージェントを運用するワークフロー

```bash
# 複数タスクをバックグラウンドで並行実行している場合の状態確認
claude agents

# Needs attention のエージェントがいないか定期的にチェック
# → 応答停止に気づきやすくなり、放置を防げる

# Done のセッションについては、PRへの言及があれば
# テーブルのリンクから直接レビューに移動できる
```

## 注意点

- 「Needs attention」は応答が停止（stalled）したエージェントに対して表示されるラベルで、単なる実行中（Running）や完了（Done）とは区別されます
- PRリンクの自動検出は実行結果のテキストにPRへの言及が含まれる場合に動作するもので、すべての結果がリンク化されるわけではありません
- ステータス表示が安定したことで、`claude agents` を継続的に監視するワークフローの信頼性が向上します
- 表示上の修正であり、エージェント自体の実行ロジックや完了判定の仕組みが変更されたわけではありません
