## 原文（日本語に翻訳）

リモートセッションのバックグラウンドタスクがターン間で「まだ実行中」として表示され続ける問題を修正

## 原文（英語）

Fixed remote session background tasks appearing stuck as "still running" between turns

## 概要

リモートセッション（Claude.ai等のリモート接続）でバックグラウンドタスクを実行した場合、タスクが実際には完了しているにもかかわらず「まだ実行中（still running）」として表示され続ける問題がありました。この修正により、バックグラウンドタスクの状態がターンをまたいでも正確に反映されるようになりました。

## 基本的な使い方

この修正は自動的に適用されます。リモートセッションでバックグラウンドタスクを実行すると、完了後に正しいステータスが表示されます。

```bash
# リモートセッションでバックグラウンドタスクを起動
# 例：長時間かかるテスト実行など
claude "テストスイート全体を実行してください"

# タスク完了後
# 修正前：[まだ実行中...] と表示され続ける
# 修正後：[完了] と正しく表示される
```

## 実践例

### CI/CDパイプラインでのリモートタスク監視

```bash
# リモートセッションでビルドタスクを実行
claude "プロジェクト全体をビルドして結果を報告してください"

# バックグラウンドで実行中
# [●] ビルド実行中...

# 完了後（修正後）
# [✓] ビルド完了 - 成功
# 次のターンで正しく「完了済み」として扱われる
```

### 長時間処理の並行実行

```bash
# 複数のバックグラウンドタスクを起動
claude "以下を並行して実行してください：
  1. テスト実行
  2. 依存関係の脆弱性スキャン
  3. コードカバレッジ計測"

# 修正前：全タスクが「実行中」のままになることがあった
# 修正後：各タスクが完了次第、正しいステータスに更新される
```

## 注意点

- この問題はリモートセッション固有の問題で、ローカルセッションでは発生しませんでした
- バックグラウンドタスクの実際の処理結果には影響なく、表示上の問題のみでした
- タスクが本当に実行中の場合と区別がつかなくなるため、修正は重要でした
