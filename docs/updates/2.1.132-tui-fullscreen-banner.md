## 原文（日本語に翻訳）

`/tui fullscreen` の起動バナーを更新し、追加レンダラーのメリット（メモリ使用量の削減、マウスサポート、選択時の自動コピー）を説明するようになりました。

## 原文（英語）

Updated the `/tui fullscreen` startup banner to describe additional renderer benefits (lower memory usage, mouse support, auto-copy on select)

## 概要

Claude Codeの `/tui fullscreen` コマンドで起動するフルスクリーンモードは、通常の表示モードに比べて多くのメリットがあります。このアップデートにより、起動時のバナーにメモリ使用量の削減、マウスサポート、テキスト選択時の自動クリップボードコピーといった具体的なメリットが表示されるようになりました。ユーザーがフルスクリーンモードの利点をより明確に理解できるようになります。

## 基本的な使い方

フルスクリーンモードを起動するには `/tui fullscreen` コマンドを使用します。

```
/tui fullscreen

# 起動後に表示されるバナー（更新後）の例:
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# フルスクリーンモードが有効になりました
# 
# メリット:
#   • メモリ使用量の削減
#   • マウスサポート（クリック、スクロール）
#   • テキスト選択時の自動クリップボードコピー
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 実践例

### フルスクリーンモードの起動と基本操作

```
# フルスクリーンモードを起動
/tui fullscreen

# フルスクリーンモードでのマウス操作
# - 左クリック: カーソル位置の移動
# - スクロールホイール: 会話履歴のスクロール
# - テキストをドラッグして選択 → 自動的にクリップボードにコピー

# フルスクリーンモードを終了
# Ctrl+C または /exit コマンド
```

### メモリ使用量の削減効果

```bash
# 通常モードとフルスクリーンモードのメモリ使用量を比較

# 通常モードでClaude Codeを起動
claude
# メモリ使用量を確認
ps aux | grep claude | grep -v grep

# フルスクリーンモードに切り替え
# /tui fullscreen

# 再度メモリ使用量を確認
# フルスクリーンモードでは一般的にメモリ使用量が削減される
ps aux | grep claude | grep -v grep
```

### テキスト選択と自動コピーの活用

```
# フルスクリーンモードでのテキスト操作

# 1. マウスでコードブロックを選択
#    → 自動的にクリップボードにコピーされる

# 2. コピーしたコードを別のウィンドウに貼り付け
#    → Ctrl+V（Windows/Linux）または Command+V（macOS）

# この機能により、Claudeが生成したコードを
# エディタに素早く貼り付けることができます
```

### デフォルトのフルスクリーンモード設定

```bash
# フルスクリーンモードをデフォルトで起動するように設定
# ~/.claude/settings.json を編集

{
  "tui": {
    "fullscreen": true
  }
}

# または /config コマンドで設定
/config set tui.fullscreen true
```

## 注意点

- フルスクリーンモードはターミナルの種類によってマウスサポートの挙動が異なる場合があります。
- 自動コピー機能はクリップボードへのアクセス権限が必要です。一部のリモートSSH環境では動作しない場合があります。
- メモリ使用量の削減効果は環境やセッションの内容によって異なります。
- フルスクリーンモードを終了するには `Ctrl+C` または `/exit` を使用してください。
- VS CodeやJetBrainsなどの組み込みターミナルでのマウスサポートは、ターミナルの設定に依存します。
