## 原文（日本語訳）

`claude agents` のバックグラウンドセッションで `/model` などのダイアログがブロックされていた問題を修正（過度に広範囲だったガード処理を撤回）。

## 原文（英語）

Fixed /model and other dialogs being blocked in `claude agents` background sessions (reverts an overly broad guard)

## 概要

Claude Codeの `claude agents`（Agent View）は、複数のセッションをバックグラウンドで並行実行・管理できる機能です。直前のバージョン（2.1.208）で、バックグラウンドセッションを保護するためのガード処理が追加されましたが、その範囲が広すぎたために `/model` をはじめとする一部のダイアログまで誤ってブロックされてしまう不具合がありました。今回のアップデートでこの過度なガードが撤回され、バックグラウンドセッション内でも `/model` などのダイアログが正常に開けるようになりました。

## 基本的な使い方

`claude agents` でバックグラウンドセッションにアタッチした状態で、通常のセッションと同じように `/model` を実行するとモデル選択ダイアログが開きます。

```text
claude agents
# 対象セッションにアタッチした状態で
/model
```

## 実践例

### バックグラウンドで走らせているセッションのモデルを切り替える

長時間かかるリファクタリング作業をバックグラウンドで走らせている最中に、特定のセッションだけモデルを変更したい場合、そのセッションにアタッチして `/model` を実行すれば、対象セッション限定でモデルを切り替えられます。

```text
claude agents
# 一覧から対象セッションを選んでアタッチ
/model opus
```

### Agent View一覧からのディスパッチモデル変更と個別セッションの操作を使い分ける

`claude agents` の一覧画面自体で `/model` を実行すると、以降ディスパッチする新規セッションのデフォルトモデルを一時的に切り替えられます（この設定は `claude agents` の実行中のみ有効で、設定ファイルには保存されません）。一方、個別のバックグラウンドセッションにアタッチして `/model` を実行すると、そのセッション自体のモデルを切り替えられます。今回の修正により、いずれの場面でもダイアログが正しく開くようになりました。

```text
claude agents
/model sonnet   # 一覧画面：以降のディスパッチのデフォルトモデルを変更
```

## 注意点

- この不具合は2.1.208で導入されたガード処理が原因で、`/model` の画面が「バックグラウンドセッションでは利用できません」という誤ったブロック対象に含まれてしまっていました。
- 2.1.209では該当のガードが撤回されており、`/model` 以外の一部ダイアログも同様に影響を受けていた可能性があります。
- `/install-github-app` や `/mcp` の認証系アクションなど、意図的にバックグラウンドセッションでは利用できないコマンドも存在するため、すべての制限が撤廃されたわけではありません。
