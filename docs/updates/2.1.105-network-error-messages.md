## 原文（日本語に翻訳）

ネットワークエラーメッセージを改善しました。接続エラーは、サイレントなスピナーの代わりに、すぐにリトライメッセージを表示するようになりました。

## 原文（英語）

Improved network error messages: connection errors now show a retry message immediately instead of a silent spinner

## 概要

ネットワーク接続エラーが発生した場合、以前はサイレントなスピナーが表示されるだけで、ユーザーは何が起きているのかわからない状態でした。今回の改善により、接続エラーが検出された直後にリトライメッセージが表示されるようになり、問題の発生と対処方針をすぐに把握できます。

## 基本的な使い方

この改善は自動的に機能します。特別な設定は不要です。

**改善前:**
```
Claude  [スピナーが回り続けるだけ]
```

**改善後:**
```
Claude  接続エラーが発生しました。リトライ中... (1/3)
```

## 実践例

### オフライン環境での動作確認

```bash
# ネットワーク接続が切れている状態でリクエストを送信
$ claude "コードをレビューしてください"
接続エラー: APIサーバーに接続できません
リトライ中 (1/3)... [3秒後]
リトライ中 (2/3)... [6秒後]
リトライ中 (3/3)... [12秒後]
接続に失敗しました。ネットワーク接続を確認してください。
```

### VPN切断時のフィードバック

VPN接続が切れた場合も即座にエラーメッセージが表示されるため、問題の特定が容易になります。

## 注意点

- スピナーが表示され続ける場合は、接続エラーではなくAPIの応答待ちの可能性があります
- リトライが失敗し続ける場合は、ネットワーク設定やVPNの状態を確認してください
- プロキシ環境では`HTTPS_PROXY`環境変数の設定が必要な場合があります
